## Challenge

In per-block quantization within SiLU-mul kernel, thread 0 computes the group scale and writes it to global memory, then all 128 threads issue a global memory load to read it back, incurring hundreds of cycles of latency and consuming L2 cache bandwidth on the critical path


## Solution

Reuse the already-allocated shared_max[0] reduction buffer to broadcast the computed scale from thread 0 to all threads via shared memory, eliminating the global memory round-trip — shared memory reads complete in a few cycles vs hundreds for global memory loads

## Key Lessons

- Reuse shared memory buffers already allocated for reduction to broadcast per-block scalar results to all threads — shared memory reads complete in a few cycles vs hundreds for global memory, and no additional shared memory allocation is needed
- When one thread produces a scalar needed by all threads in the block, prefer shared memory broadcast over global memory write-then-read to avoid L2 cache pressure and long-latency stalls on the critical path
