## Solution

Widen per-thread vectorization from half2 (2 elements) to 4×half2 (8 elements) by loading four half2 registers in the kernel, computing ELU on each lane independently, and storing four half2 results to increase ILP and reduce kernel launch overhead

## Key Lessons

- Widening per-thread vectorization from half2 to half8 for memory-bound elementwise kernels yields negligible gains when the block size drops from 128 to 32 threads — the severely reduced occupancy eliminates the SM's ability to hide memory latency, negating ILP benefits
- For memory-bound activation kernels at large sizes, half2 vectorization already saturates effective memory bandwidth; further widening to half8 provides diminishing returns unless paired with sufficient thread count to maintain occupancy
