## Solution

Add input and residual in native __half using the hardware __hadd instruction, cast the __half sum once to float solely for variance accumulation, and store the __half directly to residual — eliminating the __float2half back-cast and one of the two __half2float conversions per iteration

## Key Lessons

- Eliminating redundant half↔float conversions in a memory-bandwidth-bound kernel does not improve wall-clock time — the GPU instruction scheduler pipelines conversion latency behind pending memory loads, so the removed instructions were not on the critical path
- When a kernel's working set (e.g., 8KB per array × 2 arrays = 16KB per block) saturates available DRAM bandwidth, instruction-count optimizations are masked; profile memory throughput first before pursuing compute-side cleanups
- Native __half addition (__hadd) is semantically equivalent to float add + round-to-half for the store path — use it to avoid the round-trip through float when the stored value only needs half precision
