## Solution

Fuse sigmoid_accurate() and per-expert bias addition directly into the top-2 search loop body, eliminating the intermediate scores_with_bias buffer and the separate compute_scores_with_bias_kernel launch entirely — the inline arithmetic cost (2-3 SFU/ALU ops per element) is far cheaper than the removed global-memory write+read cycle

## Key Lessons

- Fusing a lightweight elementwise transform (sigmoid+bias) into a downstream reduction/topk kernel eliminates the intermediate global-memory buffer write+read cycle — the inline compute cost is dominated by the saved memory traffic when the transform is only a few ALU/SFU ops per element
- Kernel fusion savings scale with the size of the eliminated intermediate buffer; with fewer total elements (32 experts vs 64), the absolute memory savings are proportionally smaller, yielding a lower but still significant speedup
