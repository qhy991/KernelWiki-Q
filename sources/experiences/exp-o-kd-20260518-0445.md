## Challenge

The after code introduces adaptive max_block_size: 256 for large batches (num_tokens >= 256) and 1024 for small batches. With 512 blocks (num_tokens=512) and 512 threads per b


## Solution

Vectorize memory access using float4 (128-bit) loads/stores so each thread processes 4 elements via reinterpret_cast<float4*>, reducing total memory instructions by 4x and improving coalescing efficiency for HBM-streaming workloads

## Results

- **speedup**: 1.201x (improved)


## Key Lessons

- Use float4 vectorized loads/stores via reinterpret_cast<float4*> for memory-bandwidth-bound elementwise kernels — reduces memory instruction count by 4x and improves coalescing when the working set exceeds L2 cache
- Scale block size inversely with vectorization factor (256 → 64 for float4) to keep total elements-per-block constant and avoid excess register pressure from the wider per-thread state
