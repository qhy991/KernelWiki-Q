## Challenge

Each element FP8 quantization calls scaled_fp8_div which performs a full FP32 division (val / scale) per element; on Ampere, FP32 division has ~16 cycle latency and 1/4 the throughput of FP32 multiply, creating a compute bottleneck in the merge_attn_states kernel


## Solution

Pre-compute the reciprocal scale_inv = 1.0f / scale once at kernel entry, then replace every per-element FP32 division with an FP32 multiplication (val * scale_inv), reducing N divisions to 1 division + N multiplications

## Key Lessons

- Replace per-element FP32 division by a loop-invariant value with a single pre-computed reciprocal (1 division) followed by per-element multiplication — FP32 multiply has 4x the throughput of FP32 division on Ampere, yielding ~10% speedup in compute-bound FP8 quantization kernels
- When a scaling factor is applied to every element in a kernel, always pre-invert the scale once rather than dividing per element; the optimization compounds with element count and is most impactful in compute-bound kernels with large per-thread workloads
