## Solution

Replace serial global-memory scan with cooperative shared-memory load (8 threads load int4 chunks in parallel) followed by warp-level parallel counting (32 threads each check 1 element and reduce via __shfl_down_sync), keeping latency near-constant regardless of block size

## Key Lessons

- Replace serial global-memory scans with cooperative shared-memory loads when the same data is accessed by a reduction — parallel loads amortize global memory latency across threads, and shared memory enables a subsequent warp-level reduction
- Warp-level __shfl_down_sync reduction for small counts (≤32 elements) achieves near-constant latency regardless of element count, since the reduction tree depth is always log2(32)=5 stages — use this pattern when counting over small per-block arrays
- For MoE valid-token counting where valid tokens are contiguous at the front of a sorted array, a parallel count-reduction (count all valid) can replace a serial early-break scan (find first invalid) with identical results and much better scaling
