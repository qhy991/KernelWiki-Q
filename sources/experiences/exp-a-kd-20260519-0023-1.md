## Challenge

Split-KV attention LSE reduction uses two separate kernels (write_partial_lse + host_reduce_lse) launched sequentially from the host, incurring kernel launch overhead and requiring a large intermediate workspace of split_kv * H * B elements for partial LSE values


## Solution

Fuse two kernels into a single fused_lse_reduction kernel using two-phase atomic reduction: Phase 1 uses atomicMax on int-converted LSE values to find global max, Phase 2 uses atomicAdd to sum exp(lse - global_max), with __threadfence() and global counter spin-wait barriers for cross-CTA synchronization

## Key Lessons

- Fusing kernels with atomic cross-CTA synchronization regresses at micro-benchmark scale because spin-wait loops and __threadfence() calls dominate latency for small problem sizes (H=128, B=1, split_kv=4), but can win in full pipelines by eliminating ~5-10us of driver overhead per kernel launch and avoiding large intermediate workspace allocation — evaluate fusion at realistic scale, not micro-benchmarks
- atomicMax on integers (via __float2int_rn conversion) is used for cross-CTA maximum reduction because CUDA lacks native float atomicMax on pre-sm_90 hardware — the int representation preserves ordering for positive values, enabling correct max-finding without custom CAS loops
- Spin-wait barriers using global counters (atomicAdd counter + while loop) provide cross-CTA synchronization without __syncthreads() or separate barrier kernels, but consume SM cycles while waiting — prefer cooperative groups or host-sequenced launches when synchronization overhead exceeds the work being synchronized
