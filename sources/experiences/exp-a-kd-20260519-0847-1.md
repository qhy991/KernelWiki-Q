## Solution

Reorganize LSE storage from transposed [num_heads, num_tokens] to row-major [num_tokens, num_heads] layout, so that adjacent threads sharing the same token_idx but iterating over different head_idx values access consecutive 4-byte addresses, condensing all 32 LSE entries (128 bytes) into one fully-coalesced memory transaction per warp

## Key Lessons

- Store per-token auxiliary arrays (LSE, scales, biases) in row-major [tokens, heads] layout so that the innermost thread dimension (head_idx) indexes contiguous memory, enabling coalesced access — applies whenever threads map to heads within a token in the GPU kernel
- A stride of num_tokens*4 bytes between adjacent head accesses (transposed layout) forces each warp to touch num_heads separate cache lines; swapping to row-major collapses all accesses into ceil(32*4/128)=1 coalesced transaction, improving effective bandwidth by up to num_heads
