## Key Lessons

- Since this kernel is memory-bound (reading 4-byte floats, writing 1-byte uint8 values per thread iteration), the GPU arithmetic units have ample spare throughput.
- One additional multiply does not increase memory traffic or change the load/store pattern, so it gets absorbed into the existing instruction pipeline without measurable latency impact.
