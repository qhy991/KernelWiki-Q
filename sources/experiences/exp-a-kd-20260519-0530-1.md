## Challenge

Enabling block swizzle (BLOCK_SWIZZLE=true) with a 3D grid layout at N=2048 and stride=2048 produces N_SWIZZLE=1, degenerating the 3D grid to 2D and adding ~1.2% overhead from N_SWIZZLE computation without any L2 cache interleaving benefit


## Solution

Change BLOCK_SWIZZLE from false to true and launch with a 3D grid (gridDim.z = N_SWIZZLE) where bx = blockIdx.z * gridDim.x + blockIdx.x interleaves thread blocks across N ranges to improve L2 cache locality on B-matrix K-dimension sweep — but at N=stride, N_SWIZZLE=1 and no interleaving occurs

## Key Lessons

- Block swizzle requires N > stride (N_SWIZZLE >= 2) to actually interleave thread blocks across different N ranges for L2 cache benefit — at N <= stride the 3D grid degenerates to 2D and the optimization is a no-op with slight overhead from extra index computation
- The bx = blockIdx.z * gridDim.x + blockIdx.x formula remaps thread block IDs so that adjacent blocks in the x-dimension sweep different N-column ranges, improving L2 spatial locality during the B-matrix K-dimension access — verify N_SWIZZLE >= 2 before enabling
