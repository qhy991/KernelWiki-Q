## Challenge

GELU array functor uses int(rhs.size()) as the unrolled loop bound instead of the template parameter N directly, which appears to introduce a potential runtime call overhead per loop iteration for the size() member function


## Solution

Replace int(rhs.size()) with the template parameter N directly in the #pragma unroll loop bound, and remove the size() member function from the Array struct — however, the compiler already inlines size() to return N at compile time, producing identical SASS and zero measurable difference

## Key Lessons

- Using int(rhs.size()) vs template parameter N as a #pragma unroll loop bound produces identical SASS when size() is a trivial inline function returning N — the compiler fully resolves both at compile time, making this a style preference rather than a performance optimization
- When benchmarking trivial changes (loop bound source for compile-time-constant bounds), expect delta within timing noise (±0.75%) and do not claim a meaningful speedup
