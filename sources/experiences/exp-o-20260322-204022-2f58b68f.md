## Challenge

Improve RMS Norm throughput beyond 2.17x baseline by better L2 cache reuse for the weight vector (4096 floats = 16KB)


## Solution

Persistent grid-stride loop where each block processes multiple rows, keeping weight vector hot in L2 cache


### Key Changes

- Persistent grid-stride loop: batch_idx = blockIdx.x; batch_idx < batch_size; batch_idx += gridDim.x
- Grid size capped at num_sm*2 to avoid over-subscription
- 256-thread variant for small batches (<=64), 512-thread for large batches
- __launch_bounds__(256,4) and __launch_bounds__(512,2) for occupancy hints
- float4 vectorization + warp shuffle reduction preserved from v1


## Key Lessons

- Persistent grid-stride loop improves L2 reuse when weight vector fits in L2 (16KB for hs=4096)
- Capping grid at num_sm*2 avoids over-subscription while maintaining enough parallelism
- 256-thread blocks with 4 warps/SM hint gives better occupancy for small batches
- 64-thread variants hurt performance due to lower occupancy - avoid for hs=4096
