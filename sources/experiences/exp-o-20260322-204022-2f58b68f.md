RN_04

## Solution

Persistent grid-stride loop where each block processes multiple rows, keeping weight vector hot in L2 cache

## Key Lessons

- Persistent grid-stride loop improves L2 reuse when weight vector fits in L2 (16KB for hs=4096)
- Capping grid at num_sm*2 avoids over-subscription while maintaining enough parallelism
- 256-thread blocks with 4 warps/SM hint gives better occupancy for small batches
- 64-thread variants hurt performance due to lower occupancy - avoid for hs=4096
