## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 pattern) using HALF2() reinterpret_cast macro, so each thread loads 4 half elements per iteration via two half2 reads from both A and X, reducing loop iterations from 4 to 1 and doubling memory transaction width from 2 to 4 bytes

## Key Lessons

- Use half2 vectorized loads (f16x4 pattern) to process 4 half elements per thread per iteration instead of scalar loads — doubles memory transaction width and reduces loop overhead, applicable whenever K is a multiple of 4 and memory access is contiguous
- Compute stride as (warp_index * 4) rather than iterating 4x more warps — keeps the same thread count while each thread does 4x the work, reducing branch and loop-overhead instructions proportionally
- HALF2() reinterpret_cast macro provides a zero-overhead way to load two adjacent half values as a single half2 register on hardware that supports vectorized memory instructions
