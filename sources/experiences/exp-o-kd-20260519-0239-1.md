## Challenge

When beta is zero, the monolithic epilogue compute() still allocates and clears a source_fragment, calls source_iterator.clear_mask(), and passes a zeroed source_fragment to the two-argument output operator on every iteration — wasting memory bandwidth on unnecessary global reads and forcing the output operator to process a dummy argument


## Solution

Split monolithic compute() into compute_source_not_needed_() and compute_source_needed_() with runtime dispatch on output_op.is_source_needed(), so the beta-zero path entirely skips source_fragment allocation, source iterator operations, and uses a single-argument output operator — eliminating ~16MB of unnecessary global memory reads per launch

## Key Lessons

- Split monolithic kernel paths by runtime-invariant conditions (e.g., is_source_needed()) into dedicated functions — the zero-beta path avoids source_fragment allocation, global memory loads, and two-argument operator overhead entirely, yielding up to 1.46x speedup
- When a conditional branch is invariant across all loop iterations (beta-zero check inside a tiled loop), hoist the branch outside the loop and specialize the entire loop body — eliminates per-iteration branch overhead and enables the compiler to optimize each path independently
- Reducing global memory reads in GEMM epilogue directly scales throughput — eliminating the source tile read cuts bandwidth from ~32MB to ~16MB per launch, which is the dominant win over any register or instruction savings
