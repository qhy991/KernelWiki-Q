## Solution

Replace scalar per-element loads with __align__(16) array_t<half,8> packed 128-bit loads that fetch 8 half values in a single ld.128 transaction, upcast all 8 to float via an unrolled template pipeline, accumulate in array_t<float,8> with packed_assign_add, and downcast back — reducing global memory transactions by ~8x and increasing instruction-level parallelism

## Key Lessons

- Use __align__(16) array_t<T,16/sizeof(T)> packed types to issue ld.128/ST.128 transactions instead of scalar loads when processing contiguous half/fp16 data — reduces global memory transaction count by the pack width (8x for half) and is the dominant optimization for bandwidth-bound kernels at multi-MB scales
- Template-based upcast/downcast pipelines (half→float accumulate→half) with #pragma unroll let the compiler fuse the packed load, widen, accumulate, and narrow into a high-ILP inner loop without manual vector intrinsics
- __launch_bounds__(512, 1) hints the compiler to maximize register allocation for the 512-thread configuration of this bandwidth-bound kernel, trading occupancy for per-thread throughput on packed operations
