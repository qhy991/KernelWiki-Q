## Solution

Replace synchronous FLOAT4 global-to-shared loads for B with cp.async.ca asynchronous copies via __cvta_generic_to_shared, committing the async group before A-tile loading so the B transfer overlaps with A loading and transpose in the prologue, then CP_ASYNC_WAIT_GROUP(0) ensures completion before proceeding

## Key Lessons

- Use cp.async.ca to issue asynchronous global-to-shared copies and commit the group before independent work (e.g., A-tile loading/transpose), allowing the async transfer to overlap with compute — applies whenever there is independent memory or compute work that can fill the latency gap
- Always obtain the shared memory pointer via __cvta_generic_to_shared before passing it to CP_ASYNC_CA, as the async copy intrinsic requires a generic-to-shared converted address, not a raw pointer
- Use CP_ASYNC_WAIT_GROUP(0) after the overlapped work to guarantee all pending async copies complete before consuming the data in shared memory
