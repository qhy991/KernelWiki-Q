## Challenge

At 16 KB, the workload fits entirely in a single thread block (1024 vectors processed by 256 threads with 4 iterations each). The before kernel does scalar byte loads, which issues one 1-byte load instruction per element


## Solution

The after kernel's uint4 vector loads issue one 16-byte load per element, reducing the total number of load instructions by 16x. ## Results

- **speedup**: 1.154x (improved)


## Key Lessons

- At this small size the kernel is instruction-limited rather than memory-bandwidth-limited, so reducing the instruction count through vectorization provides a measurable ~15% improvement.
- The key difference from the large case is that register pressure is less of a bottleneck at low occupancy, and the instruction count reduction dominates.
