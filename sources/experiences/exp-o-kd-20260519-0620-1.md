## Solution

Fuse renormalization into the topkGatingSoftmax kernel by accumulating row_sum_for_renormalize during the top-K selection loop (thread_group_idx==0 threads add max_val each iteration) and dividing each output weight by the sum after the loop, eliminating a separate normalization kernel launch and global memory round-trip

## Key Lessons

- Fuse normalization into the selection kernel by accumulating the sum during the existing top-K loop and applying division afterward — eliminates a separate kernel launch and global memory round-trip, applicable whenever a post-selection normalization pass is required
- Use multiply-by-reciprocal (1.f / sum then multiply) instead of repeated division inside loops to avoid expensive float division hardware — applies to any normalization loop where the divisor is loop-invariant
- Guard fused epilogue work behind the lead-thread predicate (thread_group_idx==0) to keep overhead minimal when only a subset of threads own the output — applies to warp-parallel kernels with distributed output ownership
