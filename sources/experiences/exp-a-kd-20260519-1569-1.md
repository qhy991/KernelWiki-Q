## Challenge

Warp-level max reduction in top-2 selection uses manual __shfl_xor_sync loops with descending offset and explicit conditional comparison per step, making the code verbose and harder to maintain without any performance advantage over higher-level abstractions


## Solution

Replace manual __shfl_xor_sync reduction loops with cooperative_groups::reduce(tile, val, cg::greater<float>()) which compiles to identical warp shuffle instructions while providing a cleaner, more maintainable abstraction for warp-tile max reductions

## Key Lessons

- Replacing manual __shfl_xor_sync reduction loops with cg::reduce(tile, val, cg::greater<T>()) compiles to identical warp shuffle instructions — use cg::reduce for readability but expect no performance gain at warp-tile granularity
- cooperative_groups::reduce is a zero-cost abstraction for simple warp-tile reductions where the manual shuffle loop is already optimal; treat it as a maintainability improvement, not an optimization
