## Challenge

Optimization focus: Added float4 vectorized loads for input/bias data in the large token kernel, replacing scalar stride loads.
Problem before: The large token kernel loaded input and bias data using scalar stride loads: each lane iterated with stride WARP_SIZE over the 384-expert dimension, reading


## Solution

pattern also increases pressure on the memory subsystem as adjacent lanes access non-contiguous addresses.
Applied change: Replaced the scalar stride loop with float4 vectorized loads using reinterpret_cast<float4*>. Each lane now loads 3 float4 vectors (VEC_PER_LANE = VPT/VEC_SIZE = 12/4 = 3), each

## Key Lessons

- Expected impact: Vectorized float4 loads reduce the number of memory transactions by 4x and improve memory coalescing, since adjacent lanes now load adjacent 128-bit chunks rather than strided 32-bit values.
- The stride pattern also increases pressure on the memory subsystem as adjacent lanes access non-contiguous addresses.
