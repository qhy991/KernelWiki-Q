## Solution

Warp shuffle reduction + float4 vectorized loads/stores

## Key Lessons

- For hidden_size divisible by 4, float4 vectorized loads give significant bandwidth improvement
- Warp shuffle reduction is faster than shared memory tree reduction for this problem size
- Two-level reduction (warp-level + cross-warp) with only 8 warp sums is very efficient for 256-thread blocks
- hidden_size=1536 with 256 threads: each thread handles 6 float4 elements in the reduction pass
- This pattern (warp-shuffle + float4) achieves 2.33x speedup over baseline tree reduction
