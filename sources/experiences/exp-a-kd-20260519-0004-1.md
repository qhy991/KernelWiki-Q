## Solution

Guard both the MMA loop entry and the producer state advancement with if (k_tile_count > 0), preventing the pipeline from advancing its state counter when no tiles are actually processed — a direct correctness fix modeled on the CUTLASS pattern of checking is_mma_leader_cta && k_tile_count > 0 before issuing MMA operations and incrementing accumulator_pipe_producer_state

## Key Lessons

- Guard MMA loop entry and pipeline state advancement with k_tile_count > 0 to prevent pipeline desynchronization when K == 0 — the branch is cost-free for positive K since the predictor always takes the true path
- In CUTLASS-style pipelined GEMM, the accumulator pipe producer state must only be incremented when tiles are actually processed; unconditional advancement causes the consumer (epilogue) to wait on a stage that was never produced
- Defensive guards on loop iteration counts are correctness-essential in software-pipelined kernels, not just performance-optional — the pipeline state machine couples the producer and consumer, so a spurious increment desynchronizes the entire pipeline
