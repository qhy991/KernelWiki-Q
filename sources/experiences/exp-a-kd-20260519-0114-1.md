## Solution

Distribute the 32 exp2f calls across 8 warps by mapping each warp's lanes 0–3 to 4 rows (warp_id * kLinesPerWarp + lane_id), allowing the warp scheduler to issue exp2f from multiple warps in parallel; additionally guard each exp2f with a conditional skip when m_prime >= mi to bypass unnecessary transcendental computation

## Key Lessons

- Distribute per-row transcendental operations (exp2f, log2f) across multiple warps instead of concentrating them on one warp — the GPU warp scheduler can pipeline SFU instructions from multiple warps, yielding higher throughput even though each warp's latency remains the same
- Guard expensive exp2f calls with a conditional skip when the input is unchanged (m_prime >= mi) to bypass unnecessary computation — particularly effective in flash attention where rows often share the same maximum value across iteration steps
- For small row counts (e.g., 32 rows with 8 warps), the overhead of warp-partitioned indexing and conditional branching may outweigh the parallelism gains, producing neutral or slightly negative results — measure with realistic problem sizes before committing to this pattern
