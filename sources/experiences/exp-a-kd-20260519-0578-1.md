## Challenge

Static 3D shared memory arrays (__shared__ float s_a[K_STAGE][BM][BK]) force the compiler to emit __cvta_generic_to_shared on every async-copy iteration, incurring redundant address-conversion instructions inside the hot K-tile loop of a WMMA TF32 SGEMM kernel


## Solution

Replace static 3D shared memory with extern __shared__ dynamic allocation plus manual pointer arithmetic (stage/row/col byte offsets), and hoist __cvta_generic_to_shared out of the loop by pre-computing smem_a_base_ptr/smem_b_base_ptr once so each async-copy iteration only performs register arithmetic to build the shared address

## Key Lessons

- Hoist __cvta_generic_to_shared out of the K-tile loop by pre-computing a shared-memory base pointer once, then compute per-element byte offsets with pure register arithmetic — the compiler can schedule these integer ops independently from the cp.async pipeline, hiding address-calculation latency
- Dynamic extern __shared__ with manual stage/row/col offset constants (s_a_stage_offset, s_b_stage_offset) gives the same memory layout as static 3D arrays while enabling the pre-computed cvta optimization and allowing the launch site to control shared memory size at runtime
