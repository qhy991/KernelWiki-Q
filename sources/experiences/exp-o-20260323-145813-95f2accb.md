fp32_rms_norm_deepseek_v3_hs7168

## Solution

Use rsqrtf instead of 1/sqrtf for single hardware instruction, removed __fmaf due to --use_fast_math incompatibility

## Key Lessons

- rsqrtf is single hardware instruction vs 1/sqrtf pipeline
- __fmaf incompatible with --use_fast_math flag
- Regular multiply gets fused by compiler with fast_math
