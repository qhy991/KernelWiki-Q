## Key Lessons

- Grid-stride persistent kernels improve performance for large batch RMS norm by reducing block count and improving SM utilization
- __launch_bounds__ hints alone are not sufficient - grid configuration matters too
- For RMS norm with hidden_size=4096, capping grid at 256 blocks for batch_size>128 gives best performance
- Shared memory caching of hidden_states gives only marginal improvement (~0.6%) because GPU L1/L2 cache handles data reuse well
- Grid-stride approach is more effective than streaming loads (__ldcs) for this workload
