## Solution

Vectorize loads/stores with float4 so each thread processes two adjacent RoPE pairs in a single 128-bit transaction, halving memory operations and improving bandwidth utilization

## Key Lessons

- Vectorize memory access with float4 (128-bit loads/stores) when processing consecutive element pairs in elementwise kernels — halves memory transactions and better saturates bandwidth
- When packing two RoPE pairs into one float4, compute separate frequency exponents for even and odd dimension indices since each pair has a distinct positional frequency
- The FLOAT4 reinterpret_cast macro enables coalesced 128-bit vector loads without changing the underlying array layout — apply it at the memory boundary to maximize throughput
