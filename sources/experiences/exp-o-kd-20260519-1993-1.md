## Solution

Fuse RMSNorm and FP8 quantization into a single kernel that computes the normalized value in FP32 and directly quantizes to FP8 without materializing the intermediate __half buffer, eliminating one kernel launch and the 64KB global memory round-trip

## Key Lessons

- Fuse consecutive kernels where the intermediate result is consumed only once — eliminates global memory round-trip, reduces kernel launch overhead, and keeps data in registers/shared memory where bandwidth is orders of magnitude higher
- Kernel fusion yields the largest speedup at small batch sizes where kernel launch overhead (~5us per launch) dominates total latency; as batch size grows the compute-bound portion dilutes the launch savings
- Writing directly to the final output type (FP8) from an FP32 intermediate preserves full precision through the normalization computation while avoiding the cost of a __float2half conversion that would only be read back and converted again
