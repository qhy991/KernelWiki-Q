## Solution

Replace 8-warp CTA block with a single warp (32 threads) where each thread handles 8 elements, using pure warp shuffle reduction via __shfl_down_sync and __shfl_sync broadcast — eliminating shared memory, __syncthreads() barriers, and the serial cross-warp accumulation loop, while boosting occupancy from ~2 blocks/SM to ~16 blocks/SM

## Key Lessons

- For hidden_size ≤ 256, a single warp (32 threads) with kElemsPerThread = hidden_size/32 outperforms a multi-warp CTA approach — warp shuffle replaces shared memory reduction and __syncthreads() barriers, reducing synchronization overhead from O(warps) shared memory round-trips to O(log32) pure-register shuffle
- Reducing threads per block from 256 to 32 dramatically improves occupancy (from ~2 blocks/SM to ~16 blocks/SM on A800), which is the dominant factor in the 1.46x speedup for small hidden dimensions
- Use __shfl_sync(mask, value, 0) to broadcast a warp-reduced scalar (like inv_rms) from lane 0 to all lanes — this replaces the shared-memory write + __syncthreads() + shared-memory read pattern entirely when the reduction fits within one warp
