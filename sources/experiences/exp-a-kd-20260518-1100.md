small_n_64

## Solution

4 of the 128 threads have unique N-positions to compute (TILE_N=64), leaving half the warp idle during the accumulation loop. The shared memory load/store overhead per k-tile iteration (128 elements of sA, 8192 elements of sB) is not amortized over enough output elements for this tiny M=16,N=64 shap

## Key Lessons

- In the real Marlin kernel, this config enables previously unsupported prob_n=64 shapes — the before code would have no valid config at all (determine_exec_config returns {-1,-1,-1} causing a TORCH_CHECK failure), so the after code's tiled kernel is an enabling change rather than a pure latency optim
