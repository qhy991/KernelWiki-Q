## Key Lessons

- The after path eliminates the copy by passing input_stride directly to the RMS norm kernel, so each thread computes the correct input offset as blockIdx.x * input_stride instead of blockIdx.x * hidden_size.
- The stride-aware access pattern may have slightly less cache-friendly access for large strides (stride > hidden_size means rows are farther apart in memory), but this is far outweighed by eliminating the entire copy step.
