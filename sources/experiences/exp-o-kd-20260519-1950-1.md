## Solution

Pack 8 half values into a __align__(16) struct (16 bytes) so the compiler emits LDG.128/STG.128 instructions, fully utilizing memory transaction width and reducing instruction count by 8x

## Key Lessons

- Use __align__(16) packed structs to force 128-bit load/store instructions (LDG.128/STG.128) when operating on narrow types like half — the compiler will not auto-vectorize scalar half loops, so explicit packing is required to fully utilize memory transaction bandwidth
- Scalar half (2-byte) accesses have the worst memory efficiency per transaction because each LDG wastes most of a 32-byte segment; packing 8 halfs into 16 bytes eliminates this waste and yields speedups disproportionate to the width increase (3.8x vs theoretical 8x due to other bottlenecks being exposed)
- The array_t<T,sz> template pattern is reusable: it generalizes vectorized load/store for any element type and pack width (e.g., array_t<float,4> for float4, array_t<half,8> for half8), replacing hand-written reinterpret_cast or union hacks
