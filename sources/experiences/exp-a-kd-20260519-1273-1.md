## Solution

Cast float pointers to P32* (array_t<float,4> with __align__(16)) so each iteration issues one ld.128 and one st.128 instead of four ld.32/st.32 pairs, reducing loop trip count by 4× and cutting memory instruction count by 4× to fully utilize the 128-bit bus width per transaction

## Key Lessons

- Use __align__(16) struct wrappers (e.g., array_t<T, 16/sizeof(T)>) to cast float/int pointers into 128-bit packed types, forcing the compiler to emit ld.128/st.128 instructions — this reduces memory instruction count by 4× for fp32 workloads and applies whenever data is contiguous and naturally aligned
- Packed vectorized access reduces loop trip count proportionally to the packing factor (4× for 128-bit packing of fp32), which lowers loop overhead and improves instruction-level parallelism in addition to memory throughput gains
- Bandwidth-bound element-wise kernels benefit most from vectorized loads/stores — the 1.43× speedup is consistent with cutting memory instruction overhead on a workload where arithmetic is trivial compared to memory traffic
