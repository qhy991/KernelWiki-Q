## Challenge

Standard GEMM C = A * B with M=32, N=1280, K=4096 uses a single kernel configuration for all M<=64 cases without considering whether swapping operand order (computing B^T * A^T instead) could yield better tiling efficiency on the hardware


## Solution

Swap operand order to compute D = B^T * A^T then transpose to recover C = D^T, which in CUTLASS can enable better memory access patterns when M is small relative to N, though in this cuBLAS/PyTorch microbenchmark on SM80 the benefit is negligible without accompanying SM90-specific optimizations like FP8FastAccum and wider K tiles

## Key Lessons

- Swap-AB GEMM (computing B^T*A^T instead of A*B) can improve tiling when M is small relative to N, but the benefit depends on backend-specific features — cuBLAS/PyTorch on SM80 shows no meaningful improvement without SM90 CUTLASS features like FP8FastAccum and wider K tiles
- Operand-order swap is a no-op in isolation on general-purpose BLAS libraries; it becomes an optimization only when the downstream kernel compiler (e.g., CUTLASS) selects different tile schedules based on the swapped dimensions
