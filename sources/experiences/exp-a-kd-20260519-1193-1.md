## Challenge

KV cache stored in bf16 format (2 bytes per element) consumes twice the memory bandwidth of FP8 E4M3 storage, creating a bottleneck in memory-bound decode scenarios where large KV caches must be read from global memory


## Solution

Store KV cache as FP8 E4M3 (1 byte/element) and dequantize on-the-fly via software bit-manipulation: fp8_e4m3_to_float reconstructs IEEE float by extracting sign/exp/mant fields and adjusting bias by 120, then applies per-tensor scale and converts to bf16 — trading ~6-8 extra compute instructions per element for 2x reduction in memory read traffic

## Key Lessons

- FP8 KV cache compression trades memory bandwidth for compute — the 2x traffic reduction only pays off when the working set exceeds L2 cache capacity; for small datasets fitting in L2, the ~6-8 instruction dequantization overhead per element dominates and causes regression
- Software FP8-to-float conversion via bit manipulation (sign/exp/mant extraction + IEEE float reconstruction with bias offset 120) avoids hardware FP8 dependency but adds instruction overhead — prefer hardware FP8 intrinsics when available to reduce the compute cost of on-the-fly dequantization
