## Challenge

warpReduceSum is hardcoded for addition only (val += shuffle), preventing reuse for other reduction operations like max/min without duplicating the entire shuffle loop


## Solution

Generalize the warp reduction by replacing the hardcoded += operator with a function pointer parameter (ReduceFnType<T>), allowing the same shuffle-xor loop body to serve sum, max, or any binary reduction without code duplication

## Key Lessons

- Passing a __inline__ __device__ function pointer for the reduction operator lets NVCC inline the dispatch at compile time, producing identical SASS to a hardcoded operator — safe to use for API generality without performance cost
- Generalizing a warp shuffle reduction via function pointer (sum/max/min) eliminates code duplication across reduction variants while the __shfl_xor_sync loop structure and #pragma unroll guarantee the same instruction pipeline
