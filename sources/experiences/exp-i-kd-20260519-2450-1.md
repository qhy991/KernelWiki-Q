## Challenge

Skeleton requires implementing bidirectional coordinate mapping verification for CUTLASS TensorNHWC layout — computing forward pointer offsets from 4D tensor coordinates and verifying that inverse() correctly recovers the original coordinates from those offsets


## Solution

Compute packed NHWC strides (ldc=C, ldw=C*W, ldh=C*W*H), construct TensorNHWC layout with explicit stride, then in nested loops over N/H/W dimensions use Tensor4DCoord with threadIdx.x as C index, call operator() for forward offset, inverse() for coordinate recovery, and write the round-trip offset to output for verification

## Key Lessons

- CUTLASS TensorNHWC layout supports bidirectional coordinate mapping via operator() (forward: coord→offset) and inverse() (reverse: offset→coord) — use this pair to verify layout consistency when implementing or debugging custom tensor access patterns
- For packed NHWC layout, strides follow a simple product rule: ldc=C, ldw=C*W, ldh=C*W*H — each dimension's stride equals the product of all strictly-right dimension sizes, which generalizes to any row-major packed layout
- Use threadIdx.x as the innermost (C) dimension index in verification kernels to parallelize across the channel dimension while looping over outer dimensions — avoids shared memory and sync overhead for simple layout tests
