## Challenge

Implement Flash Attention with Q8_0 quantized KV cache. Key challenges: understanding Q8_0 format (34-byte blocks with FP16 scale + 32 INT8 values), correct cache indexing for [seq_len, num_heads, head_dim] layout, and online softmax accumulation.


## Solution

Baseline Flash Attention with Q8_0 KV cache dequantization

## Key Lessons

- Q8_0 quantization uses per-block FP16 scale, not per-column scale
- Cache layout for [seq_len, num_heads, head_dim] quantized: seq_idx varies fastest in the flattened view
- INT8 values stored as uint8 require explicit cast to int8 for correct sign interpretation
- Online softmax enables single-pass attention without storing attention weights
- Baseline implementation is correct but needs significant optimization for performance
