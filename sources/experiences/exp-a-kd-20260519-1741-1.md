## Solution

Increasing BLOCK_SIZE from 16 to 32 sets THREAD_GROUP_SIZE=1, eliminating the intra-warp shuffle reduction in QK dot computation entirely and halving the number of KV-block iterations (512 to 256 for 8K tokens); the wider VEC_SIZE (8 vs 4) also enables uint4 vector loads for Q and K instead of uint2, doubling memory throughput per load instruction

## Key Lessons

- Setting BLOCK_SIZE equal to WARP_SIZE (THREAD_GROUP_SIZE=1) in paged attention eliminates intra-group shuffle reduction in QK dot products, saving warp-level communication instructions — applies when each thread can independently compute a full QK dot without exceeding register pressure
- Larger BLOCK_SIZE reduces the number of KV blocks for a given sequence length, halving loop iteration count and block-table lookup overhead — beneficial for long sequences where QK computation dominates
- Wider vector loads (uint4 vs uint2 for Q/K) double memory throughput per load when THREAD_GROUP_SIZE=1 allows VEC_SIZE to grow from 4 to 8, improving arithmetic intensity in the QK phase
