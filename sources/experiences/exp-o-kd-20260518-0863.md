## Challenge

Optimization focus: Changed k_scale and v_scale parameters from scalar pass-by-value (float) to pointer (float*) in reshape_and_cache_kernel, enabling dynamic per-token or per-layer FP8 scaling.
Problem before: In the reshape_and_cache_kernel, k_scale and v_scale were passed as scalar float value...


## Solution

the optimization is in numerical accuracy of FP8 KV cache entries.

## Key Lessons

- Static scaling can lead to significant quantization error when the dynamic range of KV cache values varies across tokens, because a single global scale cannot accurately represent both small and large values in FP8 format.
