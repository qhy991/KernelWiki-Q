## Solution

Process 16 groups per block using 256 threads (16 threads per group via warp-shuffle), computing group and lane IDs from threadIdx to index into the correct group's data, which improves occupancy by doubling threads per block and reducing total block count by 16x

## Key Lessons

- When each group needs only a warp (16-32 threads), pack multiple groups into one block to increase threads per block and reduce block count — 16 groups × 16 threads = 256 threads per block significantly improves occupancy over 1 group × 128 threads
- Derive group ID from threadIdx.x / threads_per_group and lane ID from threadIdx.x % threads_per_group to map a 2D (group, lane) index space onto a linear thread array, enabling multiple independent work items per block
