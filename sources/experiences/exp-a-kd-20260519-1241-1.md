## Solution

Add a per-invocation kv_scale multiplication (input[i] * kv_scale) before quantization to enable e4m3-style scaled FP8 representation; the extra FMA is absorbed by spare arithmetic throughput since the kernel is memory-bound (4-byte reads, 1-byte writes per element)

## Key Lessons

- Adding a single FMA multiply to a memory-bound quantization kernel has negligible performance impact because arithmetic units have spare throughput beyond what memory bandwidth can feed — applies whenever compute-to-memory ratio is low (e.g., narrow-output quantization, type conversion kernels)
- Per-tensor scale multiplication before quantization enables proper FP8 e4m3 range normalization at zero measured cost, making scaled FP8 KV cache quantization preferable over unscaled variants for numerical accuracy without throughput tradeoff
