## Challenge

All-reduce requires a host-side cudaMemcpyAsync to copy input into the shared communication buffer before launching the reduce kernel, adding a separate D2D copy operation and kernel launch overhead that serializes the copy and reduce phases into two sequential GPU submissions


## Solution

Merge the host-side cudaMemcpyAsync into the reduce kernel via a compile-time COPY_INPUT template flag: all GPU threads cooperatively copy their chunk of input into the shared buffer using vectorized int4 stores before performing the reduction, eliminating the separate memcpyAsync and its kernel launch overhead in a single fused kernel

## Key Lessons

- Merging a host-side cudaMemcpyAsync into the compute kernel as a cooperative in-kernel copy eliminates separate kernel launch overhead and allows copy and compute to share the same execution context — applies whenever a D2D copy immediately precedes a kernel on the same stream
- Use compile-time template flags (e.g., bool COPY_INPUT) to conditionally include in-kernel copy paths, enabling the same kernel to serve both capturing and non-capturing modes with zero runtime branching overhead
