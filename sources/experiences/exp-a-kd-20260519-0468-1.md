## Challenge

Scalar ELU kernel processes one float element per thread using separate 32-bit load and store instructions, underutilizing available memory bandwidth for a memory-bound elementwise operation


## Solution

Use float4 (128-bit) vectorized loads/stores via the FLOAT4 macro to process 4 float elements per thread in a single memory transaction, reducing total memory transactions by 4x and improving bandwidth utilization for the memory-bound ELU operation

## Key Lessons

- Vectorize memory access with float4 loads/stores in memory-bound elementwise kernels to reduce transaction count by 4x — applies whenever the operation is bandwidth-limited and elements are contiguous and 16-byte aligned
- Scale thread index by the vectorization factor (idx * 4) and reduce blockSize accordingly (256/4) so total hardware threads cover the same element count with fewer CUDA threads and lower scheduling overhead
