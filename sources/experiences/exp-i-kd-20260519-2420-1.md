## Solution

Instantiate cutlass::maximum<half_t, propagate_NaN> as a callable functor with the propagate_NaN bool template parameter to control NaN behavior, then invoke it as op(*in1, *in2) guarded by a single-thread check on both thread and block indices

## Key Lessons

- cutlass::maximum<T, propagate_NaN> is a callable functor that computes element-wise max with compile-time NaN propagation control via the bool template parameter — when propagate_NaN=true, NaN inputs produce NaN output; when false, the non-NaN operand wins
- CUTLASS functional operators (maximum, plus, multiplies, etc.) are designed to be instantiated as functors and invoked with op(a, b), making them usable directly inside device kernels without CUTLASS kernel launch overhead
