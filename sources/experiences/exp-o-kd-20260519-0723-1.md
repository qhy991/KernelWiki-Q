## Solution

Fuse rope_only_kernel and kv_save_kernel into a single fused_rope_kv_kernel that keeps the RoPE-computed k_out in registers and writes it directly to both k_rope and k_buffer destinations, eliminating the redundant global memory round-trip and one kernel launch

## Key Lessons

- Fuse kernels that produce intermediate results in registers and immediately consume them in a subsequent kernel — keeping data in registers eliminates redundant global memory write-then-reload round-trips
- Kernel launch overhead (~5-10us per launch) dominates execution time for small-batch workloads (nnz ≤ ~16); eliminating even one launch can yield significant speedup
- When fusing producer-consumer kernels, load all inputs needed by the consumer before overwriting shared register variables in the producer stage to avoid register pressure spikes
