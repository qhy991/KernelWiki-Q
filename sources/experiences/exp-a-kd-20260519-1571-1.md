## Solution

Replace the 4-type dispatch macro with a 3-type variant (DISPATCH_3_FLOATING_TYPES) that omits the double case entirely, preventing nvcc from ever instantiating copy_blocks_kernel<double> and reducing binary bloat without affecting runtime behavior

## Key Lessons

- Remove unused type cases from C++ dispatch macros to avoid unnecessary template instantiations — nvcc generates PTX/SASS for every case in the switch regardless of runtime dispatch, so dead cases inflate binary size and compile time
- Compile-time dispatch pruning (removing double from AT_DISPATCH_FLOATING_TYPES_AND2) has no runtime performance effect on the actually-dispatched kernel path — the generated SASS for the used type is identical
