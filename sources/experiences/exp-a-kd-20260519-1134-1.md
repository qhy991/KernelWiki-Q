## Challenge

Block reduction uses integer right-shift (>>5) to compute the number of warps, which truncates the count for non-aligned block sizes (e.g., blockDim.x=100 yields 3 instead of the needed 4), silently dropping partial-warp contributions in the cross-warp reduction phase


## Solution

Replace integer bit-shift (>>5) with float division (/32.f) in the cross-warp guard condition so that blockDim.x / 32.f preserves fractional warp counts (e.g., 100/32.f=3.125), allowing threadIdx.x < 3.125 to correctly include the 4th partial warp's lane-0 threads in the final reduction

## Key Lessons

- Integer right-shift (>>5) for warp-count calculation silently truncates partial warps — use ceil(blockDim.x/32.0) or float division in the cross-warp guard to ensure correctness for non-multiple-of-32 block sizes
- For block sizes that are exact multiples of 32 (the common case), >>5 and /32.f produce identical results and the float division adds one negligible FP instruction; the real benefit is correctness for edge-case block sizes
