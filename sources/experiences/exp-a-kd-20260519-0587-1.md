## Challenge

Scalar FP32 sigmoid kernel performs one load and one store per element via x[idx]/y[idx] indexing, issuing 4x more memory transactions than necessary and underutilizing available HBM bandwidth on a memory-bound workload at 64 MB


## Solution

Vectorize load/store with float4 (16-byte) transactions: each thread loads 4 floats via reinterpret_cast<float4*>, computes 4 sigmoid values in registers, and writes back with a single float4 store, reducing memory instruction count by 4x and improving coalescing efficiency

## Key Lessons

- Float4 vectorized load/store reduces memory instruction count by 4x for elementwise kernels — applies whenever the kernel is memory-bandwidth-bound and data is float-aligned and contiguous
- Scale thread index by the vector width (idx = tid * 4) and reduce blockDim proportionally (256/4 = 64) to maintain the same total element coverage while issuing fewer, wider memory transactions
- reinterpret_cast<float4*> via a FLOAT4 macro is the idiomatic zero-overhead way to perform 128-bit vectorized loads/stores on aligned float arrays in CUDA
