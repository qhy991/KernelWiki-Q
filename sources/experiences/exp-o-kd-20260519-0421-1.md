## Challenge

SGEMM tiled kernel loads B matrix tiles from global memory into shared memory using synchronous FLOAT4 assignments, creating a serialization point each loop iteration that blocks computation until the transfer completes


## Solution

Replace synchronous FLOAT4 B-tile loads with cp.async.ca.shared.global (CP_ASYNC_CA) asynchronous copies, issued and committed at the start of each loop iteration before A loading and the BK compute loop, so B global-to-shared transfers overlap with computation and A loading; a CP_ASYNC_WAIT_GROUP(0) before __syncthreads() ensures completion

## Key Lessons

- Use cp.async.ca.shared.global (CP_ASYNC_CA) for global-to-shared memory copies in tiled GEMM to overlap B-tile loads with compute and A loading — the async pipeline eliminates the serialization bottleneck of synchronous FLOAT4 assignments, yielding significant speedup when BK iterations are numerous
- Issue async copies and commit the group early in the loop body, then CP_ASYNC_WAIT_GROUP(0) only at the end before __syncthreads(), maximizing the overlap window between memory transfers and computation
- Convert shared memory pointers with __cvta_generic_to_shared before passing them to cp.async — the PTX intrinsic requires a shared memory address space pointer, not a generic pointer
