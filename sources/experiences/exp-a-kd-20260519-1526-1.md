## Challenge

MoE grouped GEMM problem-size setup kernel hardcodes the (M, N, K) ordering in problem descriptors, preventing runtime selection of A/B matrix swap when batch dimensions are small and the transposed ordering would enable more efficient GEMM dispatch


## Solution

Add a compile-time bool SWAP_AB template parameter with if constexpr to select between original (M,2N,K)/(M,K,N) and swapped (2N,M,K)/(K,M,N) problem-size orderings, enabling zero-overhead layout adaptation for different expert-batch-size regimes

## Key Lessons

- if constexpr with a bool template parameter provides zero-overhead compile-time branch selection for layout strategies — the compiler eliminates the unused branch entirely, so kernels instantiated with different configurations pay no runtime cost
- SWAP_AB pattern for MoE grouped GEMM: when num_experts * padded_m exceeds a threshold the original (M,N,K) ordering is used; when it falls below, swapping M↔N in problem descriptors lets the GEMM library pick a more efficient tile traversal for tall-skinny transposed problems
