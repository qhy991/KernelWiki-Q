## Solution

Replace raw reinterpret_cast<int4> and __ldg with a typed PackedVec struct providing named .elts[] element access and ld128/st128 template wrappers that encapsulate the 16-byte load/store internally, yielding identical PTX but with type safety and clearer intent

## Key Lessons

- Replacing reinterpret_cast<int4> with a typed PackedVec struct having named .elts[] members produces identical PTX on SM80 — the compiler sees through both patterns since the local variable has no external aliasing, so this refactor is purely a readability and maintainability improvement
- Encapsulating __ldg and int4 store behind typed ld128/st128 template wrappers provides static_assert size checking and hides the reinterpret_cast boilerplate without any runtime cost — applies whenever 128-bit vectorized load/store is used repeatedly across kernels
- A code pattern change that improves type safety but does not alter the instruction sequence will show neutral performance — validate refactors with benchmarks to confirm no regression before assuming equivalence
