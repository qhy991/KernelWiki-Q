## Solution

Map threads to matrix elements using standard 2D CUDA indexing (threadIdx + blockIdx * blockDim), apply triangular fill mode filtering via early return for out-of-triangle elements (i<j for kLower, i>j for kUpper), compute column-major offset, and generate bounded random doubles via linear congruential generator with k=16807, m=16

## Key Lessons

- Use 2D thread indexing (threadIdx.x/y + blockIdx.x/y * blockDim.x/y) for matrix kernels to naturally map thread grid onto matrix dimensions — simplifies bounds checking and reduces address arithmetic compared to 1D linearized indexing
- For triangular fill modes, place the fill-mode check inside the bounds check and use early return for out-of-triangle elements — threads within the same warp that share the same row or nearby columns will often make the same branch decision, reducing divergence
- Use a linear congruential generator (LCG) keyed on memory offset (offset + seed) * k % m for fast, reproducible per-element initialization without shared state or atomics — each thread computes its value independently from its global position
