## Solution

Replace per-thread 2D matrix counting with warp-partitioned shared_counts[32][8] using atomicAdd to eliminate the prefix-sum pass and one __syncthreads barrier; replace per-thread rank tracking with atomicAdd on local_offsets for scatter; increase thread count from 64 to 1024 to reduce per-thread work by 16x

## Key Lessons

- Use warp-partitioned atomic counting (shared_counts[warps][experts_per_warp]) instead of per-thread 2D token-count matrices with prefix-sum for expert-wise token aggregation — atomicAdd directly produces totals, eliminating the O(blockDim.x × num_experts) prefix-sum pass and one __syncthreads barrier
- Use atomicAdd on local_offsets for scatter instead of per-thread prefix count tracking — each thread atomically claims a slot via atomicAdd(&local_offsets[expert_id], 1), simplifying scatter logic and avoiding 2D index bookkeeping
- Increasing thread count (64 → 1024) with less per-thread work yields net speedups when the bottleneck is shared-memory access patterns and barrier synchronization rather than raw computation
