## Solution

Read the stride directly from the tensor's actual leading dimension (LDA) instead of recomputing it from logical dimensions, and set batch_stride to 0 for single-matrix kernels where it is unused, making the code correct for both contiguous and padded layouts

## Key Lessons

- Prefer reading stride from the tensor's actual metadata (e.g., a.stride(0) / LDA) rather than recomputing from logical dimensions — the recomputed value happens to match for contiguous layouts but silently produces wrong indices for padded/strided tensors
- When batch_stride is multiplied by a hardcoded 0 batch index, the value is irrelevant — set it to 0 explicitly to signal intent and avoid misleading computed values
