## Solution

Split the fused kernel into separate Q and K CTAs using blockIdx.y dispatch (dim3(num_tokens,2)) with a single set of 2 vec_t register arrays and 32-float shared memory per CTA, selecting Q or K data/weight pointers via a runtime conditional branch

## Key Lessons

- Fusing Q and K normalization in a single CTA amortizes barrier and reduction overhead across 2 rows per CTA — splitting into separate CTAs doubles total CTA count while the per-CTA barrier cost stays constant, yielding no net reduction in total synchronization work and worse occupancy when SM thread capacity is saturated
- Broadcast-reading a single shared-memory slot (buffer[0]) from all 1024 threads serializes access compared to per-warp slot indexing (buffer[threadIdx.x/32]) where only the 32 threads in each warp read their own slot — per-warp indexing distributes shared-memory bank access across the reduction result phase
- Runtime conditional pointer dispatch (is_q ? q : k) adds register pressure for the branch and pointer selection compared to using separate named pointers for each tensor — when both paths are equally hot in a fused kernel, the compiler generates simpler code with distinct pointers
