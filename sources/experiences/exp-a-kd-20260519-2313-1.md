## Challenge

Scalar per-element half load/store in the merge_attn_states kernel issues one LDG/STG per element, motivating a switch to 128-bit uint4 vectorized access to reduce memory instruction count


## Solution

Replace scalar per-element load/store with 128-bit uint4 (8 half) packed load/store via reinterpret_cast, using a flat 1D grid where each thread processes one pack of 8 elements instead of iterating with stride

## Key Lessons

- Vectorized load/store alone does not guarantee speedup — the flat 1D grid mapping introduces division/modulo index computation (global_idx → token_head_idx, pack_idx, token_idx, head_idx) that adds integer arithmetic overhead which can dominate in sub-10us kernels
- Block-to-head affinity matters for L2 cache locality: when one block processes a single (token, head) pair (before), all threads access contiguous memory from one head; a flat 1D grid (after) mixes threads from different heads in the same block, reducing per-block cache reuse
- For very short kernels (<10us), index computation overhead and cache behavior can outweigh memory bandwidth savings from wider loads — profile before adopting vectorization
