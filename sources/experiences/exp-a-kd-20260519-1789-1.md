## Solution

Replace preprocessor-guarded fp8_e4m3::scaled_vec_conversion with a unified fp8 namespace whose scaled_vec_conversion takes a runtime __nv_fp8_interpretation_t parameter, then wrap it with fp8::scaled_convert<Tout,Tin,kv_dt> that dispatches via constexpr if on an enum (Fp8KVCacheDataType::kFp8E4M3/kFp8E5M2) — the constexpr branch is resolved at compile time so the generated SASS is identical to the preprocessor path

## Key Lessons

- Replacing preprocessor-guarded per-format namespaces with a unified namespace + constexpr enum template dispatch produces identical SASS (the compiler resolves if constexpr at compile time) while eliminating code duplication across FP8 variants — applies whenever multiple FP8 formats share the same conversion logic differing only in the __nv_fp8_interpretation_t argument
- Enum-based template parameters (Fp8KVCacheDataType) are preferable to preprocessor guards (#if defined(ENABLE_FP8_E4M3)) for format selection because they enable type-safe compile-time dispatch, better IDE support, and extensible addition of new formats without adding new namespaces or #ifdef blocks
