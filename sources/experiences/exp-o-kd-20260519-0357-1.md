## Solution

Use float4 (128-bit) vectorized loads/stores via the FLOAT4 macro so each thread processes 4 elements in a single memory transaction, reducing total memory transactions by 4x and improving bandwidth utilization for the memory-bound ELU operation

## Key Lessons

- For memory-bound elementwise kernels (activation functions, pointwise ops), use float4/float2 vectorized loads and stores to process multiple elements per thread — this reduces memory transaction count proportionally and is the highest-impact optimization when compute intensity is low
- Scale thread index by the vector width (idx * 4 for float4) and reduce block size accordingly (blockDim / 4) so total grid coverage remains correct while each thread does more useful work per memory transaction
