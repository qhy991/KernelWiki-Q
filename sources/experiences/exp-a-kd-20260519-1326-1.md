## Challenge

FP8 quantization kernel uses a conditional branch on num_elems % 16 == 0 to select between vectorized and scalar paths, with the vectorized path assuming pointer alignment and length divisibility without runtime alignment handling


## Solution

Replace conditional vectorization with vectorize_with_alignment which unconditionally attempts vectorized access while handling misaligned addresses at runtime via scalar prefix + vectorized body + scalar tail, wrapping the conversion in a device lambda functor dispatched through DefaultVecOp

## Key Lessons

- Alignment-aware vectorization frameworks add runtime pointer-alignment checks and functor/lambda dispatch overhead that nvcc may not fully inline away, causing slight regressions on already-aligned workloads where the simpler conditional path suffices
- For guaranteed-aligned data (e.g., cudaMalloc output with known-divisible lengths), a direct conditional vectorization without runtime alignment probing can outperform a generalized alignment-handling framework by eliminating unnecessary branch and indirection overhead
- Generalized abstraction layers (DefaultVecOp + lambda) over load-convert-store patterns introduce instruction overhead proportional to the framework's generality — when per-element compute cost is low relative to memory ops, abstraction overhead becomes measurable
