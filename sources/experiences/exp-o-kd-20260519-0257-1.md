## Solution

Compute kStages = min(4, ceil(kMaxK/kKeysPerBlock)) = min(4, ceil(128/128)) = 1, reducing shared memory from 32768 to 8192 bytes/block. This raises occupancy from ~5 to ~20 blocks/SM (floor(163840/8192)=20), enabling 4x more concurrent threadblocks to hide memory latency while also eliminating 3 pipeline loop iterations and their associated __syncthreads() barriers

## Key Lessons

- When kMaxK divides evenly into kKeysPerBlock (ceil(kMaxK/kKeysPerBlock)=1), additional pipeline stages are wasted — compute kStages = min(max_stages, ceil(kMaxK/kKeysPerBlock)) to avoid allocating SMEM buffers that never fill with useful data, yielding massive occupancy gains
- Reducing SMEM per block has a compounding effect: fewer bytes/block → more blocks/SM → more concurrent warps → better latency hiding → higher throughput; on A800 (164 KB shared memory/SM), cutting from 32 KB to 8 KB per block quadruples theoretical occupancy
- Each pipeline stage loop iteration adds one __syncthreads() barrier; reducing from 4 stages to 1 eliminates 3 warp-level stalls per kernel and reduces total instruction count in the compute loop
