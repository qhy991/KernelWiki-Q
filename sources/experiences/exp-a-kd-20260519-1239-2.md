## Solution

Set YTILE=1 to eliminate the outer row-loop entirely, so each wavefront processes exactly one row per grid-stride iteration, removing the y-loop overhead and simplifying the control flow for small-M regimes where rows-per-wavefront is already minimal

## Key Lessons

- Setting YTILE=1 in wavefront-split reduction kernels eliminates the outer row-loop when M is small relative to CuCount*WvPrGrp, removing per-iteration loop overhead (index math, bounds checks) that becomes proportionally significant at low row counts
- Row-tiling (YTILE>1) only benefits throughput when each wavefront has enough rows to amortize the loop overhead — for small M or large CU counts, YTILE=1 is preferred
