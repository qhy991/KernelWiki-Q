## Solution

Fuse QK-scoring and softmax-value into a single kernel using online softmax with incremental max-tracking and alpha/beta rescaling, eliminating the intermediate score matrix entirely and reducing global memory traffic from O(M×N) to O(M×K + N×K)

## Key Lessons

- Fuse multi-kernel attention into a single kernel using online softmax to eliminate the M×N intermediate score buffer — avoids O(M×N) global memory writes/reads and kernel launch synchronization overhead, with savings growing linearly with sequence length
- Online softmax with incremental max-tracking (alpha/beta rescaling) replaces three-pass softmax (find-max, compute-denom, accumulate) with a single sequential pass — applicable whenever softmax normalization is followed by weighted accumulation over the same index dimension
- Map one block per query row and one thread per output dimension (qi=blockIdx.x, d=threadIdx.x) so each thread holds a single accumulator and streams through all keys/values, enabling register-level accumulation and eliminating the need for shared-memory reduction
