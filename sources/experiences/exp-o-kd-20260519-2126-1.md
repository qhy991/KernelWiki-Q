## Challenge

Paged-attention K-cache is stored in fp16 (2 bytes/element), causing 2x unnecessary DRAM traffic for long sequences (2048 tokens, 128 blocks) where the kernel is memory-bound and K-cache reads dominate total memory traffic


## Solution

Store K-cache in FP8-E5M2 (1 byte/element) instead of fp16, halving DRAM traffic, and dequantize to fp16 on-the-fly via software integer bit-manipulation (sign/exponent/mantissa reshuffling) before the QK dot product — the conversion cost is negligible compared to the bandwidth savings on long sequences

## Key Lessons

- Store KV-cache in FP8-E5M2 (1 byte/element) instead of fp16 for memory-bound attention kernels — 2x bandwidth reduction on long sequences outweighs the software dequantization cost since DRAM access latency dominates arithmetic latency in paged-attention K-cache traversal
- Implement FP8-E5M2 to fp16 conversion via integer bit-manipulation (sign/exponent/mantissa field reshuffling) rather than float arithmetic — this avoids FPU pipeline stalls and keeps conversion latency predictable on all GPU architectures
- Read FP8 pairs as uint16_t and dequantize two elements at once via fp8x2_to_half2 — pairing matches the half2 vector width used by the downstream QK dot product, avoiding lane-crossing overhead
