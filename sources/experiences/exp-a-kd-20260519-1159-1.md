## Challenge

MoE scoring kernel uses a runtime if(scoring_func == SCORING_SIGMOID) branch inside the per-element hot loop, forcing the compiler to keep both code paths even when the scoring function is SCORING_NONE and the branch always falls through


## Solution

Convert the runtime scoring_func parameter into a compile-time template parameter and use if constexpr to eliminate the dead branch entirely — when SF=SCORING_NONE, apply_scoring<SF> becomes a plain identity pass-through with no branch instruction generated

## Key Lessons

- Use template parameters and if constexpr to convert runtime branch-on-enum into compile-time dispatch — the compiler eliminates dead branches entirely, which matters when the branch is inside a hot loop and the kernel has significant compute beyond memory stalls
- In simple memory-bound kernels where per-iteration branch overhead is negligible compared to memory latency, compile-time dispatch produces no measurable gain — the optimization pays off in compute-bound or branch-heavy kernels where uniform branches add instruction count or warp divergence cost
