## Solution

Replace Pingpong double-buffered schedule with WarpSpecialized single-buffered schedule, removing the redundant second set of shared memory buffers (16KB savings) since the simplified compute kernel cannot exploit the tile-level overlap that Pingpong's two-tile interleaving would provide

## Key Lessons

- Double-buffered Pingpong schedules only improve throughput when the kernel has sufficient compute density and async copy (TMA) to actually overlap mainloop and epilogue across two tiles — allocating double shared memory without the overlap mechanism just wastes SMEM and increases occupancy pressure
- Shared memory footprint reduction (32KB to 16KB) only improves occupancy when shared memory is the occupancy-limiting resource; for simple kernels with low register pressure and few warps, occupancy is already high enough that halving SMEM yields no measurable gain
