## Challenge

GEMM dispatch unconditionally launches a kernel with fixed 32x32x32 tiling requiring 8192 bytes of shared memory without checking device shared-memory capacity, risking launch failure on GPUs with insufficient shared memory per block


## Solution

Query cudaDevAttrMaxSharedMemoryPerBlockOptin at dispatch time to detect available shared memory, then select between a large-tile (32x32x32) kernel and a small-tile (16x16x16) fallback kernel depending on device capability

## Key Lessons

- Query cudaDevAttrMaxSharedMemoryPerBlockOptin at dispatch time and provide a fallback kernel with reduced shared memory — prevents launch failures on GPUs with limited shared memory per block without affecting performance on capable devices
- Provide a small-tile fallback (e.g., 16x16x16 requiring ~2048 bytes) that fits within the guaranteed 48 KB minimum shared memory — applicable to any tiled kernel that uses dynamically sized shared memory beyond the default allocation
