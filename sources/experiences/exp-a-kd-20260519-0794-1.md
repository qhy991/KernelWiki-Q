## Solution

Replace the single-thread serial loop with a parallel segmented prefix sum using shared memory: 16 threads each compute a local prefix sum over a 16-element segment, store segment totals in local_offsets_buf, then apply cross-segment prefix corrections by adding preceding segment totals; the same two-phase pattern is repeated for block-size alignment

## Key Lessons

- Parallelize serial prefix sums via segmented decomposition — each thread computes a local scan over its segment, stores the segment total in a shared buffer, then cross-segment prefixes are added as corrections → applies when N is small enough to fit in shared memory and the serial loop is a bottleneck
- Use a two-phase shared-memory pattern (local scan + cross-segment correction with __syncthreads barriers) to convert an O(N) single-thread serial scan into a multi-thread parallel scan, achieving 2.3x speedup for 256 elements across 16 active threads
