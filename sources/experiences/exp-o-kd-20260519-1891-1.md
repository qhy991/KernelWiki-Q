## Solution

Reduce TILE_M from 128 to 32 to better match M=16, cutting per-thread accumulators from 64 to 16 int32 values (~36 registers vs ~80) and shrinking shared memory from 8800 to 5344 bytes per block, enabling ~7 blocks per SM for dramatically higher occupancy and latency hiding

## Key Lessons

- Match tile dimensions to actual problem dimensions — oversized tiles cause excessive register pressure and shared memory waste when the active data fills only a fraction of the tile, directly reducing occupancy
- Register count per thread is the primary occupancy bottleneck for compute-heavy GEMM kernels: reducing accumulator count from 64 to 16 int32 per thread (~80 to ~36 registers) more than doubles concurrent blocks per SM
- For small-M batched GEMM (M < TILE_M), reducing the M-tile to the next power-of-two above M cuts wasted work without requiring algorithmic changes
