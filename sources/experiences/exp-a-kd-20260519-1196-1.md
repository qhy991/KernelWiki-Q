## Challenge

In GQA rotary embedding (32 query heads, 4 KV heads), a single fused loop iterates over all query elements with a conditional `if (head_idx < num_kv_heads)` guarding key writes, causing warp divergence where 28 out of 32 logical heads skip key work on every iteration


## Solution

Split the fused loop into two independent loops: a query loop over num_heads*embed_dim iterations and a key loop over num_kv_heads*embed_dim iterations, eliminating the conditional branch entirely and letting each loop have uniform control flow

## Key Lessons

- Splitting a fused loop to eliminate warp divergence does not always improve performance — when the GPU can reconverge divergent paths quickly and the split loop has very low thread utilization (128/512 threads active), the overhead of two loop setups can cancel the divergence savings
- In GQA kernels with large num_heads/num_kv_heads ratios, a conditional key guard inside a query loop may be preferable to a separate key loop when the kernel execution time is short (~12us), as the reconvergence cost is amortized while the low-occupancy separate loop is not
- Thread utilization in the split loop matters: with 4 KV heads * 32 embed_dim = 128 iterations in a 512-thread block, 75% of threads are idle, which can negate the benefit of removing divergence
