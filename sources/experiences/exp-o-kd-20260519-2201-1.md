## Challenge

BLOCK_SIZE=16 forces THREAD_GROUP_SIZE=2, requiring two threads to cooperate per QK dot product via __shfl_xor_sync reduction; only one thread per group writes logits, wasting shuffle bandwidth and introducing conditional gating in the hot QK loop, while 128 KV block iterations incur high block-table indirection overhead


## Solution

Increase BLOCK_SIZE from 16 to 32 to set THREAD_GROUP_SIZE=1, eliminating intra-warp QK shuffle reduction entirely so each thread independently computes a full head-size QK dot product, halving KV block iterations from 128 to 64 and using wider uint4 vector loads for improved memory throughput

## Key Lessons

- Increasing BLOCK_SIZE from 16 to 32 in paged attention sets THREAD_GROUP_SIZE=1, eliminating the intra-warp __shfl_xor_sync QK reduction entirely — each thread computes a full head-size dot product independently, removing shuffle overhead and the conditional thread_group_offset==0 gate on logits writes; applies when the kernel has sufficient parallelism to fill warps without multi-thread cooperation per key
- Larger BLOCK_SIZE halves the number of KV block iterations and block-table indirections (128→64 for seq_len=2048), reducing loop overhead — beneficial when the reduced iteration count outweighs the wider cross-thread value reduction (NUM_V_VECS_PER_ROW doubles from 2 to 4)
- THREAD_GROUP_SIZE=1 doubles VEC_SIZE from 4 to 8, upgrading QK vector loads from uint2 to uint4 — wider vector transactions improve memory throughput when the access pattern is already coalesced
