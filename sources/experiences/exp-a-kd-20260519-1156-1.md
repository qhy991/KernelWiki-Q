## Solution

Replace the serial single-thread prefix sum with CUB BlockScan<int32_t, 1024> which distributes the exclusive scan across all 1024 threads via a tree-based reduction with O(log N) parallel depth, eliminating the serial bottleneck entirely

## Key Lessons

- Replace serial single-thread prefix sum loops with CUB BlockScan to distribute scan computation across all block threads at O(log N) depth — applies whenever a prefix sum runs inside a CUDA kernel and block size >= number of scan elements
- CUB BlockScan::ExclusiveSum handles the scan-parallel decomposition automatically including shared-memory temp storage, eliminating manual loop serialization and improving occupancy by keeping all threads active
- For scan sizes (e.g., 256 experts) that fit within a single thread block, BlockScan eliminates the serial bottleneck entirely — the technique is most impactful when the serial loop iteration count approaches or exceeds the number of idle threads
