## Challenge

Skeleton kernel provides only template parameters and TODO markers for implementing D = alpha*(A×B) + beta*C using CUTLASS warp-level tensor operations with FP64 double precision and 8×8×4 instruction shape on SM70+ tensor cores


## Solution

Declare shared memory arrays matching template dimensions, stage data via thread-0-only copy with __syncthreads() barriers, instantiate GemmTensorOp with GemmShape<kM,kN,kK> and InstructionShape<8,8,4> for FP64 tensor cores, execute warp-level GEMM using TensorRef objects with leading dimensions, then write back results to global memory

## Key Lessons

- Use CUTLASS GemmTensorOp with InstructionShape<8,8,4> for FP64 tensor cores on SM70+ — the warp-level API handles HMMA.DP64A scheduling when given TensorRef objects (pointer + leading dimension) for each operand
- Stage GEMM operands in shared memory before invoking warp-level tensor operations — shared memory enables data reuse across the warp and CUTLASS TensorRef-based APIs expect shared memory pointers with correct leading dimensions
- Pass the same TensorRef ({&C[0][0], kN}) as both source accumulator and destination in GemmTensorOp::operator() for in-place D = alpha*(A×B) + beta*C computation — the API accumulates into the destination TensorRef
