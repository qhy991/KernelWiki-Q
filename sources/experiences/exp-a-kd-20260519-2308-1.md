## Solution

Replace the runtime kernel parameter with a type-specialized trait struct (min_scaling_factor<T>::val()) using __device__ __forceinline__, enabling NVCC to fold the constant into an immediate operand and centralizing per-type scaling constants in a single trait definition

## Key Lessons

- Use type-specialized trait structs with __device__ __forceinline__ for per-type constants (e.g., min_scaling_factor, quant_type_max) instead of runtime kernel parameters — NVCC folds the inlined constant into an immediate operand and centralizing constants in traits reduces host-kernel coupling
- Removing a single scalar parameter from a kernel signature is a maintainability improvement rather than a measurable performance optimization at typical kernel execution scales (~6us), as the GPU launch pipeline amortizes the 4-byte parameter buffer overhead
