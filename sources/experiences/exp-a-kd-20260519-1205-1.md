## Solution

Add a conditional branch inside the hot loop to load from shared memory only at group boundaries (k % b_sh_wr_iters == 0) and copy from the previous register buffer on other iterations, reducing shared memory accesses by a factor of b_sh_wr_iters

## Key Lessons

- Conditional shared-memory load reduction regressed in this microbenchmark because NVCC hoists the simple float shared memory read into a register when the loaded address is loop-invariant — the compiler already eliminates redundant shared memory accesses, making the added branch overhead (modulo, comparison, conditional jump) pure waste
- Conditional-load optimizations (group-boundary loading with register copy) are only beneficial when the shared memory load is genuinely expensive (e.g., 128-bit ldsm int4 loads in Marlin tensor-core kernels) and the compiler cannot hoist due to high register pressure from tensor-core fragments — do not apply to simple scalar float reads where the compiler can freely optimize
- Microbenchmarks that isolate a single access pattern may misrepresent optimization impact when the compiler can see and optimize the full access pattern in the simplified code, unlike in real kernels where surrounding tensor-core operations constrain register allocation and prevent such hoisting
