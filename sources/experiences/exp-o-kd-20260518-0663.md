## Challenge

Optimization focus: FP16 scalar ELU kernel progressed to f16x2 vectorized kernel using half2 loads/stores to process 2 half elements per thread.
Problem before: The scalar f16 ELU kernel processes one half element per thread, issuing separate 16-bit load and store instructions. Even with half-precis


## Solution

fill the memory bus width, wasting bandwidth. The total thread count equals N, which increases scheduling overhead.
Applied change: The f16x2 kernel uses half2 (32-bit) vector loads and stores, processing 2 half elements per thread. The two half values are packed into one 32-bit transaction for both

## Results

- **speedup**: 1.44x


## Key Lessons

- Measured: 1.44x speedup at 16M elements, but regressed 0.83x at 1M due to reduced occupancy (128 vs 256 threads/block).
