## Solution

Fuse absmax reduction and quantization into a single per-token kernel where each CUDA block processes one token's hidden_size elements, using warp-shuffle blockReduceMax for the absmax phase and vectorized float4 loads so the quantize phase reuses data already in L1 cache from the reduction pass

## Key Lessons

- Fuse reduction + compute into a single kernel when both passes traverse the same data — eliminates kernel launch overhead and keeps the working set in L1 cache between passes, yielding multiplicative speedup (4.69x here from removing one full global-memory round-trip)
- Per-token block decomposition (one block per token) replaces global atomics with per-block shared-memory writes and per-token scaling, reducing contention and enabling finer-grained quantization accuracy
- Warp shuffle reductions (__shfl_xor_sync) outperform shared-memory tree reductions with __syncthreads barriers for block-level reductions — fewer synchronization points and no shared-memory bank conflicts
