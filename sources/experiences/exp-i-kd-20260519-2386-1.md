## Solution

Map threads to matrix elements via standard 2D CUDA indexing (threadIdx + blockIdx * blockDim), guard with a bounds check, compute column-major offset as i + j*rows, and generate deterministic values using a simple LCG ((offset+seed)*16807 % 16) - 8, yielding coalesced writes since consecutive threads in x access consecutive memory locations

## Key Lessons

- Use 2D thread/block mapping (x for rows, y for columns) with offset = i + j*rows for column-major storage — consecutive threadIdx.x values access consecutive memory, ensuring coalesced global writes
- Guard all 2D kernels with if (i < rows && j < columns) to handle the last tile when matrix dimensions are not exact multiples of block dimensions
- A simple integer LCG ((offset + seed) * k % m) - m/2 with small constants (k=16807, m=16) provides cheap deterministic initialization without expensive math functions like sin/cos or random number APIs
