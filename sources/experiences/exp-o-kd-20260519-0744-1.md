## Challenge

In MoE inference, tokens assigned to no expert (expert_id == -1) still execute the full activation-and-multiply kernel, then get zeroed out by a separate Triton filter_expert kernel, wasting compute on masked tokens and adding an extra kernel launch


## Solution

Add a compile-time template parameter kFilterExpert to act_and_mul_kernel so that, when enabled, each thread checks its token's expert_id and returns early for masked-out tokens (expert_id == -1), fusing the filtering logic into the activation kernel and eliminating the separate filter kernel launch

## Key Lessons

- Fuse token-level filtering into compute kernels via a compile-time template parameter (e.g., kFilterExpert) so masked-out tokens early-return without executing expensive loads and arithmetic — eliminates both wasted compute and a separate filter kernel launch, with savings scaling linearly with mask ratio
- Use if constexpr to guard filtering logic so that when filtering is disabled the check is completely compiled away, avoiding any overhead on the no-masking path (measured ~3% when filter check is active but no tokens are masked)
