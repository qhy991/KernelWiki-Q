## Solution

Split the shared kv_scale parameter into separate k_scale and v_scale arguments, enabling per-tensor scale calibration for asymmetric key/value distributions without altering the quantization arithmetic itself

## Key Lessons

- Splitting a shared quantization scale into per-tensor scales is a structurally neutral API refactoring when scales are equal — the extra float parameter adds zero measurable overhead because it is register-resident and does not increase instruction count or memory traffic
- Per-tensor scale parameters enable future accuracy gains when key and value distributions diverge (e.g., keys spanning [-4,4] vs values spanning [-0.5,0.5]), as each scale can be calibrated independently to maximize quantization resolution within the int8 range
