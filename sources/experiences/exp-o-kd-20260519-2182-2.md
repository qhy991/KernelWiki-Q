## Solution

Replace the shared-memory reduction tree with a two-phase blockReduceMax: first reduce within each warp using __shfl_xor_sync (no shared memory, no __syncthreads), then reduce across warps using a small shared array with a single __syncthreads — cutting synchronization barriers from O(log N) to 1 for the cross-warp phase

## Key Lessons

- Warp shuffle reductions (__shfl_xor_sync/__shfl_down_sync) eliminate shared-memory traffic and __syncthreads within a warp — use them for the intra-warp phase of any block reduction, then only one __syncthreads is needed for the cross-warp phase
- Shared memory for cross-warp reduction only needs WARP_SIZE elements (32 floats), not blockDim.x — since only lane 0 from each warp writes its result, the shared array is an order of magnitude smaller than a full block-sized cache
