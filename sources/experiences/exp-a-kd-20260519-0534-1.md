## Solution

Enable BLOCK_SWIZZLE=true to switch from a 2D grid to a 3D grid that reorders block scheduling via bx=blockIdx.z*gridDim.x+blockIdx.x with an interleaving stride of 2048, so neighboring blocks sharing A-row data execute closer together and maximize L2 cache reuse

## Key Lessons

- Block swizzle (3D grid reordering via bx=blockIdx.z*gridDim.x+blockIdx.x) interleaves block scheduling so that blocks sharing input matrix rows execute close together in time, dramatically increasing L2 cache reuse — especially effective for small problem sizes where the entire working set fits or nearly fits in L2
- The linear 2D grid mapping (blockIdx.x → column-block) causes blocks that access the same A-row tile to be scheduled far apart, evicting shared data from L2 before it can be reused; a 3D swizzle with an appropriate stride (e.g., 2048 bytes) eliminates this spatial locality mismatch
