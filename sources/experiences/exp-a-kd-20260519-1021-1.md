## Challenge

Host-side code queries GPU compute capability at runtime via cudaGetDeviceProperties and uses a ternary branch to select between kernel<16> and kernel<32> template instantiations, forcing the compiler to emit both template paths even though only one is ever taken on a given architecture


## Solution

Replace runtime cc_major detection and ternary dispatch with a compile-time constexpr (kMaxVecBytes=16 for sm_80), producing a single template instantiation and eliminating the dead 32-byte code path entirely from the binary

## Key Lessons

- Replace runtime device-property queries (cudaGetDeviceProperties + ternary dispatch) with compile-time constexpr or macros when the target architecture is known at build time — this eliminates dead template instantiations, reduces binary size and JIT compilation time, with no runtime latency penalty or benefit since the kernel body is identical
- Compile-time dispatch benefits are invisible in kernel microbenchmarks but reduce CUDA driver JIT compilation overhead and binary footprint — measure compilation time separately to capture the win
