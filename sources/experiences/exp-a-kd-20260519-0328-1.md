## Challenge

Two separate kernels compute a per-row dot-product reduction and then consume the result, forcing a global-memory round-trip through an intermediate di buffer and incurring an extra kernel launch overhead between the reduction and consumer stages


## Solution

Fuse the reduction and consumer into a single kernel so the shared-memory reduction result is reused directly from register (float di = smem[0]) instead of being written to and read back from global memory, eliminating the intermediate buffer allocation and the second kernel launch

## Key Lessons

- Fuse a reduction kernel with its consumer into one kernel when the reduction result is per-block (one scalar per block) — the shared-memory value can be read into a register and reused immediately, avoiding a global-memory write, a second kernel launch, and a global-memory read-back, which together account for dominant overhead when the computation per element is light
- Eliminating intermediate global-memory buffers between producer-consumer kernel pairs saves both memory bandwidth and allocation overhead — the savings are largest when the intermediate data is small (e.g., one float per row) but incurs a full kernel launch to materialize
