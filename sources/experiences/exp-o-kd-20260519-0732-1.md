## Solution

Replace the single-thread serial cumsum loop with a parallel Blelloch (work-efficient) prefix scan where all threads participate in up-sweep and down-sweep phases on a shared-memory scan_buf padded to the next power of two, reducing parallel depth from O(num_experts) to O(log(num_experts))

## Key Lessons

- Replace serial single-thread prefix sums with parallel Blelloch scan to reduce work from O(n) serial to O(log n) parallel depth — especially impactful when n (e.g., number of MoE experts) is large (256+) and threads would otherwise sit idle
- Pre-compute padded/aligned values into a contiguous scan_buf before the scan rather than computing CEILDIV inside the scan loop — separates the alignment logic from the parallel prefix sum and simplifies the scan implementation
- Store scan results in a __shared__ variable (s_total_tokens_post_pad) before writing to global memory to avoid redundant global reads in subsequent thread work — reduces global memory traffic when the total is needed by multiple consumers
