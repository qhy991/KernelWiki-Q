## Solution

Extend each preprocessor guard to include '|| defined USE_ROCM' so that HIP compilation activates the same DP4A-optimized code paths, since AMD hardware supports __dp4a as a compatible intrinsic

## Key Lessons

- Extend CUDA architecture guards to include ROCm via '|| defined USE_ROCM' when the underlying intrinsic (e.g., __dp4a) has a compatible HIP equivalent — avoids silently falling through to scalar fallback paths on AMD GPUs
- DP4A-based integer dot products provide ~10% speedup over scalar element-by-element accumulation for quantized matmul; always verify that the optimized path is reachable on all target compilation targets
