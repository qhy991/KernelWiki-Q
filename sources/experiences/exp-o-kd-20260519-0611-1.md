## Solution

Replace the single-block <<<1,1024>>> launch with cudaLaunchCooperativeKernel using MIN(CEILDIV(num_tokens, block_threads), num_experts) blocks of 256 threads each, enabling grid-level synchronization via cg::grid_group so each block independently computes local expert counts then collectively reduces the global cumulative sum across SMs

## Key Lessons

- cudaLaunchCooperativeKernel enables multi-block kernels with grid-level synchronization (cg::grid_group), distributing serial single-block workloads across multiple SMs — applies when computation can be partitioned into per-block local phases followed by a global reduction phase
- Replace single-block <<<1, N>>> launches with cooperative multi-block launches when the workload scales with input size (e.g., token count × expert count), as single-block serialization becomes the bottleneck on GPUs with many SMs
