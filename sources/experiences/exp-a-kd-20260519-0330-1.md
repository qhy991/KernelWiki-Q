## Solution

Fuse QK scoring, softmax, and value weighting into a single kernel using online softmax: each thread maintains a running maximum m, normalization accumulator l, and output accumulator, correcting past contributions with alpha=exp(m-old_m) whenever a new maximum is observed, eliminating the intermediate scores buffer and one kernel launch entirely

## Key Lessons

- Fusing attention sub-operations (QK scoring + softmax + value weighting) into one kernel eliminates an entire global-memory round trip for the attention matrix (M×N writes + reads) and one kernel launch overhead — yields large speedups when the intermediate tensor is non-trivial relative to compute
- Online softmax (running m, l with correction factor alpha=exp(m-new_m)) computes numerically stable softmax in a single pass without materializing the full score matrix — apply whenever softmax inputs are produced and consumed within the same kernel and the reduction dimension fits in a loop
- Thread mapping change from 2D block (16×16) to 1D block with one block per query row improves memory coalescing for Q and output access patterns when each thread owns one output dimension d
