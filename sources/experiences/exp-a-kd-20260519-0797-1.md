## Solution

Replace the exp2f/ceilf/log2f transcendental function chain with integer-only IEEE 754 bit manipulation: fast_log2_ceil extracts the exponent field via bit shift, fast_pow2 constructs a power-of-two float via (x+127)<<23, and extract_ue8m0 directly reads the exponent byte, eliminating all transcendental math from the critical path

## Key Lessons

- Replace exp2f/log2f/ceilf chains with IEEE 754 bit-level manipulation (extract exponent via bit shift, construct power-of-two via (exp+127)<<23) when computing power-of-two scaling factors — reduces each operation from multi-instruction microcode to 2-3 integer instructions, applicable whenever the result must be a power of two
- Precompute reciprocals as compile-time constants (e.g., constexpr MAX_FP8_INV = 1.0f/MAX_FP8) to replace runtime divisions with multiplications — eliminates SFU division instructions in hot loops
