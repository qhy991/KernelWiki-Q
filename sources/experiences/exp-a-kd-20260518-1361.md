## Key Lessons

- Additionally, the after quantization loop uses multiplication by tmp_scale (127.0f / block_absmax_val) rather than division by scale, which avoids the more expensive FP division instruction in the hot inner loop.
- The same structural difference applies: the before code issues two kernel launches (compute_absmax_kernel + static_quant_kernel) requiring two full passes over the input tensor and a global memory round-trip for the scales, while the after code fuses both into one kernel.
