## Challenge

ldmatrix.x4 always loads 4 A-fragment rows (512 bytes per warp, 4 registers per thread) even when m_block_size=8 only requires 2 rows, wasting half the shared memory read bandwidth and 2 registers per thread per pipeline stage


## Solution

Right-size the ldmatrix instruction from .x4 to .x2 via a compile-time template parameter, loading only the 2 A-fragment rows needed for m_block_size=8, which halves shared memory read bandwidth per A-fragment load and frees 2 registers per thread per pipeline stage

## Key Lessons

- Right-size ldmatrix width (.x1/.x2/.x4) to match the actual number of A-fragment rows needed by the tile shape — using .x4 when only 2 rows are required wastes 50% shared memory bandwidth and 2 registers per thread per pipeline stage
- Templating the ldmatrix call with a compile-time count parameter allows the compiler to elide unused branches entirely via if constexpr, producing zero-overhead dispatch to the correct PTX instruction variant
