## Solution

Wrap 4 floats into a __align__(16) array_t<float,4> struct so that each load/store operates on the full 16-byte packed type, causing NVCC to emit LDG.128/STG.128 instructions that transfer 4x the data per memory transaction and better utilize L2 cache line granularity

## Key Lessons

- Use __align__(16) packed structs (e.g., array_t<float,4>) to coerce NVCC into emitting 128-bit LDG.128/STG.128 instructions instead of scalar 32-bit loads/stores — reduces memory transaction count by 4x for bandwidth-bound kernels
- Vectorizing via packed types is more portable and compiler-friendly than manual __ldg() intrinsics or inline PTX — the struct alignment alone guarantees the wider memory transaction without changing the loop logic
- Speedup from vectorized access is typically less than the theoretical 4x because loop iteration count drops proportionally (fewer active warps) and instruction overhead does not scale down
