## Solution

Replace magic-number 16 with named constants (threads_per_group, GROUPS_PER_BLOCK) and remove the always-true runtime guard, producing cleaner kernel launch arithmetic with grid dimensions derived from the named constants

## Key Lessons

- Removing an always-true guard from a kernel has no measurable performance impact when the GPU branch predictor trivially predicts the branch outcome — the compiler and hardware already eliminate the dead path
- Replace magic-number literals with named constants (threads_per_group, GROUPS_PER_BLOCK) for readability even when the numerical value is unchanged, as it documents the mapping between block/thread geometry and logical group assignment
