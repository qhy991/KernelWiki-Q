## Challenge

Attention state merge kernel always performs the full LSE rescaling merge path (two LSE loads, max, two exp, two divisions, per-element FMA loop, log) for every token, even when 87.5% of tokens are suffix-only with no prefix context and produce trivially correct results by just copying the suffix


## Solution

Add an early-exit fast path that checks token_idx >= prefix_num_tokens and copies suffix data and LSE directly, bypassing the full LSE rescaling merge for suffix-only tokens — since these tokens have no prefix contribution, the merged result is simply the suffix itself

## Key Lessons

- When a kernel processes heterogeneous data where a significant fraction of work items follow a trivially simpler path (e.g., suffix-only tokens in attention merge with no prefix contribution), add an early-exit branch before the expensive computation — the branch cost is negligible compared to the saved LSE rescaling, exp, division, and per-element FMA operations
- Prefix-attention merge kernels should accept prefix_num_tokens as a runtime parameter rather than assuming all tokens have prefix context, enabling the fast-path optimization without changing the data layout or requiring separate kernel launches for suffix-only tokens
- Early-exit optimizations yield outsized speedups when the trivially-handled fraction is large (87.5% suffix-only here = 1.16x overall), because the dominant kernel work shifts from compute-bound merge to simple memory copies that are well within the memory subsystem's capability
