## Solution

Scale to 1024 threads using a grid-stride loop with atomicAdd into a flat padded shared_counts array, replacing the per-thread 2D counter layout and eliminating the need to zero-initialize num_thread*num_experts entries — only padded_num_experts entries need zeroing

## Key Lessons

- For histogram/counting kernels in shared memory, a flat array with atomicAdd and many threads (1024) outperforms a per-thread 2D counter layout with few threads (128) because atomicAdd on shared memory is fast and the 8x thread scaling distributes the counting workload more evenly
- Padded_num_experts rounded up to warp boundary (WARP_SIZE=32) for shared_counts reduces bank conflicts during subsequent reads from the flat array
