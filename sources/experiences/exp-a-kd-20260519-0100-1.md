## Solution

Replace scalar per-element index decomposition with cooperative shared-memory tiling: 192 threads coalesce-load input as float4 vectors into shared memory, then each thread assembles a float4 output (3 data channels + 1 zero, or all-zero for odd-index threads), writing results via 16-byte vectorized stores that saturate DRAM bandwidth

## Key Lessons

- For fixed-ratio channel-padding patterns (e.g. 3→8), shared-memory tiling eliminates per-element modular index arithmetic by having threads cooperatively load input tiles and directly map shared-memory offsets to the known output layout — applies whenever the channel ratio and data type are known at compile time
- Vectorized float4 stores (16 bytes per STG) replace four scalar float stores, saturating DRAM bandwidth in memory-bound copy/pad kernels — applies whenever output elements are contiguous and 16-byte aligned
- When a majority of output elements are known zeros (5/8 in this case), a thread-assignment scheme where odd-index threads write all-zero float4 avoids redundant data reads while preserving coalesced store patterns
