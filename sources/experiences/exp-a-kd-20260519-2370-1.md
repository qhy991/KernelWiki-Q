## Solution

Consolidate four separate dp4a accumulators into a single accumulator to reduce register pressure, and replace four distinct grid pointer lookups with a single grid lookup per iteration using nibble extraction via 0x0F0F0F0F masking to produce two packed int values from one uint64_t read, trading a 4x larger grid table (2048 vs 512 entries) for simpler memory access and lower register usage

## Key Lessons

- Consolidating multiple dp4a accumulators into a single accumulator reduces register pressure (4 live int variables to 1), improving occupancy when the dot-product computation allows sequential accumulation into one variable
- Nibble extraction via 0x0F0F0F0F masking from a single grid lookup produces two packed int values per uint64_t read, reducing pointer computation overhead and memory access diversity compared to separate grid pointer casts
- A larger grid table (2048 vs 512 entries, 16KB vs 4KB) is an acceptable L2 cache tradeoff on GPUs with large caches when it enables simpler access patterns and lower register pressure — the cache pressure increase is negligible relative to the compute savings
