## Challenge

When routed_scaling_factor must be applied to MoE gate output weights, the scaling is performed as a separate Python-level tensor operation (topk_weights *= routed_scaling_factor) after the CUDA kernel returns, requiring an extra GPU memory read-modify-write round-trip and potentially a separate kernel launch


## Solution

Fuse the conditional routed_scaling_factor multiplication directly into the existing Rescale Output normalization loop inside the CUDA kernel, so that after dividing by output_sum the kernel immediately multiplies by routed_scaling_factor when the flag is set, merging two sequential operations into a single fused per-element computation that shares the same memory read/write

## Key Lessons

- Fuse conditional post-kernel scaling operations into an existing in-kernel normalization loop to eliminate a separate GPU memory round-trip and potential extra kernel launch — applies whenever a per-element scalar multiply follows a per-element normalize/divide in the same data path
- A uniform conditional branch inside a warp (all threads in thread_group_idx==0 take the same path) introduces negligible divergence overhead, making it safe to add conditional scaling logic to an already-gated loop iteration
