## Challenge

When VPT (16) is smaller than MAX_VPT (32), the direct AccessType<T> struct assignment copies all 32 elements (128 bytes for float) into the local AlignedArray even though only 16 are valid, wasting 50% of global memory bandwidth per load for both input and bias reads


## Solution

Replace the bulk AccessType<T> struct assignment with a #pragma unroll loop that extracts only VPT elements from the vectorized load pointer, avoiding the overshoot read while NVCC can still coalesce the indexed loads into efficient load sequences

## Key Lessons

- When AlignedArray size (MAX_VPT) exceeds the actual elements needed (VPT), bulk struct assignment wastes bandwidth by copying unused tail elements — replace with a #pragma unroll indexed-extraction loop that reads only the needed VPT elements, letting NVCC coalesce the loads
- Templated kernels with compile-time VPT < MAX_VPT should never rely on full-size AccessType assignment for partial data; always extract elements by index to guarantee the load width matches the valid data width
