## Solution

Widen expert_offsets from int32_t* to int64_t* so pointer arithmetic uses the full 64-bit range, preventing silent overflow for large token counts while keeping the same kernel logic unchanged

## Key Lessons

- Widen index/offset types from int32 to int64 before they are used in pointer arithmetic for GPU kernels — a 32-bit offset silently wraps when multiplied by large strides (e.g., k*n), producing wrong addresses with no runtime error
- Pointer arithmetic in CUDA inherits the type width of the offset variable, not the pointer type — casting a truncated int32 value to int64 after the load does not recover the lost upper bits, so the load itself must be 64-bit
