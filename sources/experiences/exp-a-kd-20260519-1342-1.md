## Solution

Add an int64_t input_stride parameter to the kernel signature and replace all occurrences of blockIdx.x * hidden_size in input indexing with blockIdx.x * input_stride, generalizing the kernel to handle arbitrary row strides without affecting the contiguous-case memory access pattern or instruction count

## Key Lessons

- Parameterizing the row stride as a kernel argument generalizes normalization kernels to non-contiguous tensor layouts (padded batches, sliced views) at zero performance cost for the contiguous case, since the compiler generates identical address arithmetic when stride equals hidden_size
- Use int64_t for stride parameters to avoid integer overflow when indexing large tensors with row counts exceeding 2^31 elements
