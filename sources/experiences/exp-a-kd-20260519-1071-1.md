## Solution

Replace three-pass scalar strided loads with AlignedArray<float,4> vectorized packed loads (BYTES_PER_LDG=16), loading 4 floats per instruction; with VPT=16 and THREADS_PER_ROW=1 each thread reads its entire row in a single LDG_PER_THREAD=1 transaction, eliminating 2 redundant read passes and maximizing DRAM bandwidth utilization

## Key Lessons

- Use AlignedArray<float,N> with reinterpret_cast to issue N-wide vectorized loads (16 bytes = 4 floats) in a single instruction instead of scalar strided loads — applies whenever row width is small and each thread can own the entire row
- Load input once into registers (row_chunk) and reuse across multiple softmax phases (max/sum/write) instead of reading input three times through separate loops — reduces total global memory traffic by up to 3x
- When NUM_EXPERTS (row width) is small (e.g. 16 floats = 64 bytes), set VPT=NUM_EXPERTS so one thread owns the full row, enabling a single vectorized load and eliminating inter-thread coordination overhead
