## Solution

Replace scalar per-element loads with vec4_t<half> aligned vector loads (8 bytes = 4 half values per transaction), enabling 4 consecutive threads to coalesce into a single 32-byte memory segment and reducing total transactions by ~4x while adding #pragma unroll to eliminate loop overhead

## Key Lessons

- Use vec4_t<half> (8-byte aligned struct) to vectorize half-precision memory loads in memory-bound reduction kernels — 4 consecutive threads coalesce into one 32-byte transaction, reducing total memory transactions by ~4x
- Prefer reinterpret_cast to a wider aligned vector type over scalar iteration when the input array is contiguous and the element type is smaller than 4 bytes — this maximizes memory segment utilization without changing the algorithm
- Add #pragma unroll on vectorized loops when the trip count is known and small (e.g., 4096/4/1024=1 iteration per thread) to eliminate loop overhead and expose the full unrolled body to the instruction scheduler
