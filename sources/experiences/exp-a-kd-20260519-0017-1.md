## Challenge

CausalMask<false> adds offset_q = N_K - N_Q to the trip count formula, causing each Q-block to visit far more K-blocks than necessary when Q is small relative to K, wasting computation on masked-out attention tiles


## Solution

Replace CausalMask<false> with CausalMask<true> by removing offset_q from the trip count, so max_blocks_q = ceil_div((blk_coord_q+1)*tile_q, tile_k) — when the Q sequence is a prefix of K (IsQBegin=true), no offset correction is needed and the trip count correctly collapses to 1 block for the small-Q case

## Key Lessons

- When Q is a strict prefix of K (e.g., prefill with no prior tokens), use CausalMask<true> (IsQBegin=true) which omits the offset_q term — this eliminates spurious K-block visits proportional to (N_K - N_Q)/tile_k, with dramatic impact under extreme Q/K asymmetry
- The offset_q = N_K - N_Q term in causal mask trip counts exists for sliding-window or suffix-Q scenarios where Q starts mid-sequence; applying it to prefix-Q workloads inflates the tile loop count by N_K/tile_k unnecessarily
