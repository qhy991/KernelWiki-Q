## Solution

Coarsen the grid by assigning 16 groups per block with 256 threads (8 warps), mapping each group to 16 consecutive threads via tid/16 and tid%16 decomposition, and padding shared memory to s_absmax[16][17] to avoid bank conflicts across concurrent group reductions

## Key Lessons

- Block coarsening (multiple work-groups per block) dramatically improves SM occupancy when the original granularity produces blocks with too few warps — each SM can run fewer small blocks due to warp-count limits, so consolidating groups into fewer larger blocks allows the scheduler to hide memory latency through warp-level parallelism
- Pad shared memory array dimensions to odd strides (e.g., [16][17] instead of [16][16]) when multiple independent reduction groups share the same smem region — this prevents bank conflicts when concurrent groups access their respective rows simultaneously
