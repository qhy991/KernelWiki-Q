## Challenge

128-bit vector loads use ld.global.cg (cache-all) hint which retains streamed data in L2 cache even though the data will never be reused, wasting L2 capacity when competing workloads need cache residency


## Solution

Replace ld.global.cg with ld.global.cs (cache streaming) hint on 128-bit loads to minimize L2 retention for data that will not be reused, preserving L2 capacity for other consumers

## Key Lessons

- Use ld.global.cs (cache streaming) instead of ld.global.cg (cache-all) when loading data that will not be reused — the streaming hint reduces L2 pollution, but the benefit only materializes when there is competing L2-resident data from other consumers in the same kernel launch or concurrent streams
- Cache hint selection is a workload-contention optimization, not an absolute improvement — isolated single-kernel microbenchmarks with no competing L2 traffic will show neutral or noisy results because there is nothing to evict from the cache
