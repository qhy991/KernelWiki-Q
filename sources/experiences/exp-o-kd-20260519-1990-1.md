## Solution

Fuse softmax and topk into a single topkGatingSoftmax kernel that keeps all intermediate softmax results in thread-local registers, eliminating the 768 KB intermediate global memory write/read and reducing to one kernel launch

## Key Lessons

- Fusing softmax and topk into a single kernel eliminates intermediate global memory round-trips when the softmax output is only consumed by the topk stage — applies whenever pipeline stages communicate through large temporary buffers that are never reused
- Single kernel launch replaces two launches, eliminating inter-kernel scheduling overhead and stream synchronization latency — significant when kernel execution time is short relative to launch overhead
