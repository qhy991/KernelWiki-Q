## Solution

Merge two separate elect_one_sync() guards into a single __ballot_sync + __syncwarp pair, performing both A and B writes inside one elected-lane block, which halves the warp ballot and barrier count per iteration.

## Key Lessons

- Merging multiple elect_one_sync() guards into a single warp ballot reduces synchronization instruction count, but the benefit is only visible when the synchronization itself is on the critical path — on Ampere without TMA hardware, global memory bandwidth dominates over warp ballot overhead.
- Warp-level barrier consolidation optimizations that target Hopper TMA descriptor issuance patterns do not generalize to plain global memory stores on non-TMA architectures — always validate optimization assumptions against the actual memory access mechanism and hardware generation.
