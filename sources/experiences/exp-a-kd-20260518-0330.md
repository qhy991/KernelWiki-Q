## Solution

Fuse QK scoring, softmax, and value weighting into a single kernel using online softmax (running max m, running sum l, running accumulator acc) so the full M×N attention matrix never leaves registers, eliminating the global-memory workspace and one kernel launch

## Key Lessons

- Fuse separate kernels that communicate through global memory into a single kernel to eliminate intermediate workspace writes/reads and kernel launch overhead — applies whenever intermediate results fit in registers and the producer-consumer chain is short
- Online softmax with a running max (m) and running normalization sum (l) enables single-pass softmax without materializing the full score matrix, reducing global memory traffic from O(M×N) to O(M×K) — applies when softmax is computed over a dimension iterated serially per thread
- Map blockIdx.x to query rows and threadIdx.x to head dimensions for natural memory coalescing on output writes, since adjacent threads write adjacent output elements
