## Challenge

The before source (warp_reduce_max_float_before.cu) has a hand-written warpReduceMax using `val = max(val, VLLM_SHFL_XOR_SYNC(val, mask))` — a direct max() call inside the #pragma unroll shuffle-xor loop


## Solution

The after source (warp_reduce_max_float_after.cu) replaces this with `val = fn(val, VLLM_SHFL_XOR_SYNC(val, mask))` where fn is detail::_max<float>, a trivial __inline__ __device__ wrapper returning `max(a, b)`. - NVCC inlines the function pointer call since the target address is a compile-time constant, producing SASS identical to the direct max() call.
