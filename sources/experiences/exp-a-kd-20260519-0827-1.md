## Solution

Pad each block's signal to 128-byte alignment using alignas(128) struct so each block's volatile accesses hit private cache lines, eliminating cross-SM invalidation traffic despite 32x larger memory footprint

## Key Lessons

- For per-block volatile signals, padding each slot to cache-line size (128-byte alignas) avoids false sharing even though it wastes memory — each SM's volatile accesses hit private cache lines with no invalidation traffic between SMs
- Compact memory layouts that pack multiple blocks' data into shared cache lines cause catastrophic performance degradation under volatile read-modify-write patterns, as each volatile store broadcasts an invalidation to all SMs caching that line, forcing subsequent reads to fetch from higher cache levels or memory
