## Challenge

The A800 GPU has a 40MB L2 cache. With hidden_dim=1024 and fp16 input, each token occupies 2KB and the full batch of 2048 tokens is only 4MB, which fits entirely in L2. When the no-smem kernel reloads from global memory in pass-2, the data is served from L2


## Solution

cache rather than DRAM, making the effective access latency comparable to shared memory. The smem kernel adds overhead: element-wise writes to shared memory during pass-1 (not vectorized like the global load), a __syncwarp() barrier, and element-wise reads
