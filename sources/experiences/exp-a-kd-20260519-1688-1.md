## Solution

Eliminate the downstream compute_expert_offsets and compute_arg_sorts kernel launches when only problem sizes are needed, reducing the pipeline from 3 sequential kernel launches to 1 and cutting launch overhead plus stream synchronization cost by approximately 2/3

## Key Lessons

- At small problem scales where kernel launch overhead dominates computation time, eliminating unnecessary pipeline stages by computing only the results that are consumed yields speedup roughly proportional to the fraction of launches removed — applies when downstream kernel outputs are not needed for the current operation
- Multi-kernel pipelines with sequential dependencies (count → prefix-sum → sort) accumulate launch and synchronization overhead at each stage; skipping stages whose outputs are unused is more effective than trying to fuse or optimize them
