## Challenge

Input loading loop in per-token-group quantization reads one fp16 element at a time via scalar 2-byte loads, issuing 8 separate memory transactions per thread (group_size=128 / 16 threads) and underutilizing the GPU's 32-byte minimum transaction granularity


## Solution

Replace scalar fp16 loads with __half2 packed loads via reinterpret_cast, reading 2 fp16 values per 4-byte transaction and decomposing with __low2float/__high2float, reducing input memory iterations from 8 to 4 per thread

## Key Lessons

- Use __half2 reinterpret_cast loads with __low2float/__high2float to halve memory transactions when reading contiguous fp16 arrays — applies whenever fp16 data is naturally aligned to 4-byte boundaries
- Vectorization speedup is muted when each thread processes few iterations (e.g., 8 scalar elements / 4 packed iterations) because loop overhead dominates over transaction savings; benefit grows with larger group sizes
