## Challenge

Tree attention mask kernel for speculative decoding uses bool-per-element representation (1 byte per mask entry, 65536 bytes total for batch 256, draft 16), initializes each token's mask with one true write plus 15 redundant false writes, and computes per-token indices via an O(bid) cumulative offset loop over verified_seq_len with a FULL_MASK/QLEN_ONLY branch


## Solution

Replace bool-per-element mask with bit-packed uint8_t representation (8x memory reduction from 65536 to 8192 bytes), consolidate initialization to a single byte write per token (write 1 to first byte, rely on bulk memset for remaining bits), and simplify index computation to a fixed-stride formula (bid * draft_token_num + tid) * num_bytes_per_item that eliminates both the cumulative offset loop and the FULL_MASK/QLEN_ONLY branch

## Key Lessons

- Replace bool-per-element masks with bit-packed uint8_t arrays (1 bit per element) when the mask is binary-valued and sparse — reduces memory footprint by 8x, cuts per-token initialization from N writes to 1, and decreases memory bandwidth pressure; applicable to any kernel where boolean masks dominate memory traffic
- Eliminate cumulative offset loops that iterate over per-batch state by switching to a fixed-stride layout — removes O(batch_idx) sequential dependency, avoids redundant reads of verified_seq_len, and collapses multi-branch index logic into a single multiply-add
- Consolidate redundant false-initialization writes into a single semantic write plus bulk memset — when only one element per group is true, writing the first byte to 1 and relying on pre-zeroed memory avoids N-1 redundant stores and their associated memory transactions
