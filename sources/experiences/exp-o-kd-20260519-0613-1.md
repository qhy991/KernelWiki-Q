## Challenge

UE8M0 scale computation in the inner loop uses three expensive GPU floating-point math intrinsics (exp2f, ceilf, log2f) plus an additional log2f for UE8M0 extraction, each requiring multi-instruction microcode sequences that dominate per-group computation cost


## Solution

Replace all four expensive FP math calls with IEEE 754 bit-level manipulation: fast_pow2 constructs a power-of-2 float by directly inserting (x+127) into the exponent field via left shift by 23, fast_log2_ceil extracts the exponent bits and checks for non-zero mantissa to compute ceiling, and extract_ue8m0 reads the exponent byte directly from the float bit representation

## Key Lessons

- Replace exp2f(ceilf(log2f(x))) chains with IEEE 754 bit manipulation — extracting the exponent field via (bits>>23)&0xff and constructing power-of-2 floats via (exp+127)<<23 eliminates expensive FP microcode calls and reduces the computation to a handful of integer instructions
- Precompute reciprocal constants (e.g., MAX_FP8_INV = 1.0f / MAX_FP8) as constexpr to turn runtime divisions into cheaper multiplications in hot loops
- UE8M0 quantized scale format stores only the exponent byte — extract it directly by shifting the float bit representation right by 23 bits, avoiding a full log2f call
