KernelBench level_1 task_4: Matrix-vector multiplication C = A @ B where A(256,131072), B(131072,1), C(256,1). Challenge: accumulating 131072 float values causes precision drift.

## Key Lessons

- For large float accumulations, pairwise summation balances accuracy and performance better than per-element Kahan
- Double precision is not viable for large matrix operations due to performance penalty
- Vectorized loads (float4) are essential for memory bandwidth utilization
- Memory bandwidth is the bottleneck for this problem size, not computation
