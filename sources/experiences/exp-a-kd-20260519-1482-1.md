## Challenge

The KV-cache gather kernel passes entry_size (576) and CTA_SIZE (64) as runtime variables to the inner copy loop, preventing the compiler from unrolling the loop and forcing it to emit loop-counter arithmetic, conditional branches, and runtime stride calculations every iteration


## Solution

Convert entry_size and CTA_SIZE into C++ template parameters (ENTRY_SIZE_T, CTA_SIZE_T) so the inner copy loop has compile-time constant bounds, enabling #pragma unroll to fully unroll the 9-iteration loop into straight-line loads/stores and eliminating runtime loop-counter and branch overhead

## Key Lessons

- Convert known-constant loop bounds (e.g. entry_size=576, CTA_SIZE=64) into template parameters rather than runtime arguments — the compiler can then fully unroll the loop and eliminate loop-counter overhead, yielding straight-line code with better ILP scheduling
- Template-parameterized kernels paired with an assert(CTA_SIZE_T == blockDim.x) guard provide both compile-time optimization and a runtime safety check, ensuring the launch configuration matches the template specialization
- For small fixed-iteration inner loops (9 iterations for 576/64), full unrolling removes branch overhead and exposes memory-load independence to the instruction scheduler, which is especially beneficial for memory-bandwidth-bound copy kernels
