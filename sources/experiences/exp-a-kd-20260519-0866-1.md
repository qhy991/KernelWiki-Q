## Challenge

Direct vector copy via reinterpret_cast<AccessType<T>> assigns the full MAX_VPT-sized chunk (32 elements, 128 bytes) when only VPT elements (16, 64 bytes) are needed, wasting 50% of global memory bandwidth per thread in the MoE gate kernel


## Solution

Replace the bulk AccessType<T> assignment with a #pragma unroll loop that extracts only VPT elements individually from the vectorized load pointer, reading exactly the needed data and letting NVCC coalesce the indexed loads into compact load sequences

## Key Lessons

- When VPT < MAX_VPT, avoid bulk-copying the full AccessType<T> chunk via reinterpret_cast assignment — a #pragma unroll loop extracting only VPT elements cuts wasted bandwidth by (1 - VPT/MAX_VPT) and NVCC still coalesces the indexed loads efficiently
- Per-element indexed access from an AlignedArray pointer (arr[0][ii]) gives the compiler explicit control over load count, whereas bulk struct assignment copies the entire underlying storage regardless of how many elements are semantically valid
