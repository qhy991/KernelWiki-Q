## Challenge

Static 2D shared memory declaration __shared__ int32_t shared_counts[WARP_SIZE][2] hardcodes the experts-per-warp ratio to 2, making the MoE block-alignment kernel inflexible for different expert counts without source recompilation


## Solution

Replace static 2D shared memory with extern __shared__ dynamic allocation and flat 1D indexing shared_counts[warp_id * experts_per_warp + i], passing experts_per_warp as a runtime kernel parameter to support arbitrary expert-to-warp mappings without recompilation

## Key Lessons

- extern __shared__ with flat 1D indexing and a runtime size parameter produces identical bank mappings and zero performance overhead compared to static 2D shared memory, while enabling runtime-configurable dimensions — prefer it when the shared memory layout depends on problem parameters
- 2D shared memory access arr[row][col] with row width W compiles to the same address arithmetic as flat arr[row * W + col]; the compiler emits identical load/store instructions for both patterns, so the choice is purely a code maintainability concern
