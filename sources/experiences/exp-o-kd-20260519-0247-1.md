## Solution

Replace scalar per-element index-decomposition loop with shared-memory tiling: 192 threads cooperatively load float4 values into shared memory via one coalesced 128-bit read per thread, then each thread reads 3 input floats from shared memory, inserts a zero at every 4th position, and writes a full float4 (16 bytes) to global memory, eliminating all modular arithmetic and achieving 4x fewer global memory transactions

## Key Lessons

- Shared-memory tiling with float4 vectorized I/O eliminates per-element modular index decomposition entirely — cooperative loading into shared memory via coalesced 128-bit reads, then reorganizing in shared memory before writing float4 outputs, reduces global memory transactions by 4x and removes expensive % and / operations from the hot path
- For fixed-ratio channel padding (e.g., 3-to-4 channels), the output insertion pattern (insert one zero every 3 input elements) can be expressed as ((k+1)%4==0)?zero:src[k_adj] in a small unrolled loop, avoiding any runtime coordinate computation
- Vectorized output writes (float4) are critical for memory-bound kernels — writing 16 bytes per store instead of 4 bytes saturates memory bandwidth far more effectively, especially when the kernel does negligible compute per element
