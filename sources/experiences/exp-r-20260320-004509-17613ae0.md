KernelBench level_1 task_4: Matrix-vector multiplication C = A @ B where A=(256,131072), B=(131072,1)

## Key Lessons

- For large reduction operations (131072 elements), pairwise summation provides better numerical accuracy than naive summation
- Kahan summation works but has overhead; pairwise tree reduction is a good alternative
- Double precision is not viable for memory-bound kernels due to performance penalty
- float4 vectorized loads improve memory bandwidth utilization
- Matrix-vector multiplication with one thread per row is memory bandwidth limited, not compute limited
