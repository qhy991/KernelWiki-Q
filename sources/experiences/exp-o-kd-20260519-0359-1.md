## Challenge

Scalar ELU kernel processes one float per thread using separate 32-bit load/store instructions, underutilizing memory bandwidth in a memory-bound elementwise operation


## Solution

Replace scalar float load/store with float4 (128-bit) vectorized memory transactions via the FLOAT4 macro, processing 4 elements per thread and reducing total memory transactions by 4x to improve bandwidth utilization

## Key Lessons

- Use float4 (128-bit) vectorized loads/stores for memory-bound elementwise kernels to process 4 elements per thread — reduces memory transaction count by 4x and significantly improves bandwidth utilization when the operation is dominated by memory access rather than compute
- Scale thread index by vector width (idx * 4) and reduce block size proportionally (256/4) to maintain the same total element coverage with fewer threads issuing wider memory transactions
