## Solution

Replace linear scan with binary search over the sorted expert_offsets array, reducing comparisons from O(n) to O(log n) — for 64 experts only 6 comparisons instead of up to 64, with each comparison reading from shared memory (5ns latency) instead of global memory (400ns latency)

## Key Lessons

- Binary search over sorted offset arrays reduces lookup from O(n) to O(log n) comparisons — applies whenever partition boundaries are sorted (as with MoE expert row ranges), especially beneficial when n > 8
- Combine algorithmic complexity reduction (binary search) with memory hierarchy optimization (shared memory caching) for compound speedups — the two optimizations multiply: ~10x fewer comparisons and ~80x lower latency per comparison yield a combined 2.66x kernel speedup
