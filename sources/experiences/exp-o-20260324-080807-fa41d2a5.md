## Challenge

Optimize RMS norm kernel for hidden_size=7168 beyond 2.378x baseline achieved in Round 1


## Solution

__launch_bounds__(256, 2) instead of (256, 4) to allow more registers per thread, fully unrolled 7-element loop (7168/4/256=7 elements/thread), rsqrtf for fast inverse sqrt, float4 vectorization with __ldg

## Key Lessons

- __launch_bounds__(256, 2) beats (256, 4) for register-heavy kernels - fewer blocks/SM means more registers per thread
- For hidden_size=7168 with 256 threads and float4, each thread handles exactly 7 elements - fully unrolling all 7 is better than pragma unroll 4
- rsqrtf is faster than 1.0f/sqrtf for the normalization scale computation
- Shared memory for weights hurts occupancy (limits to 1 block/SM), not worth it
- 2 rows per block adds complexity without benefit for this workload
