## Solution

Replace 16 separate cudaMemcpyAsync D2D calls with a single copy_blocks_kernel launch using a 2D grid (blockIdx.x = layer, blockIdx.y = pair), where each thread block performs the key and value copy via a strided loop, eliminating per-copy CPU dispatch overhead and enabling GPU-side concurrency across pairs

## Key Lessons

- Batching many small cudaMemcpyAsync D2D copies into a single kernel launch eliminates per-call CPU dispatch overhead — especially critical when each transfer is small (≤1 KB) and overhead dominates transfer time
- Use a 2D grid (blockIdx.x = layer, blockIdx.y = pair) to map independent copy operations onto separate thread blocks, letting the GPU scheduler run them concurrently
- Pass cache pointers via device-side int64_t pointer arrays to support multi-layer kernels without separate launches per layer
