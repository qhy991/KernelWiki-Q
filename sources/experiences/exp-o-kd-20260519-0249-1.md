## Challenge

NHWC channel-padding kernel decomposes each linear output index into (n, h, w, c) coordinates via a cascade of modulo and integer-division operations (idx%c_out, /c_out, %w, /w, %h, /h), spending most arithmetic throughput on address calculation rather than data movement


## Solution

Replace per-element modular index decomposition with cooperative shared-memory tiling: 192 threads load input as coalesced float4 vectors into shared memory via direct linear indexing, then each thread reads 3 floats from shared memory and assembles a float4 output with padding zeros, eliminating all modulo/division arithmetic

## Key Lessons

- Per-element modular index decomposition (cascading % and / to recover multi-dimensional coordinates) is extremely expensive on GPUs — cooperative shared-memory loading with direct linear indexing eliminates the entire chain and lets threads access input data by simple offset arithmetic
- For channel-padding kernels where c_out/c_in is a known small ratio, cooperative loading into shared memory followed by per-thread assembly avoids redundant index computation across threads that access the same spatial location
