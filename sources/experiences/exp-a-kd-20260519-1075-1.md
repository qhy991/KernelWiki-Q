## Challenge

Speculative decoding tree-building kernel uses bool-per-element mask (1 byte per boolean) wasting 8x memory, initializes each token mask row with an O(draft_token_num) loop issuing 8 byte-sized stores, and computes mask offsets via a cumulative O(bid) loop over verified_seq_len with mode branching


## Solution

Replace bool-per-element mask with bit-packed uint8_t mask to achieve 8x memory reduction (16384→2048 bytes), use direct multiply-by-stride indexing ((bid * draft_token_num + tid) * num_bytes_per_item) to eliminate the cumulative O(bid) loop and mode branching, and replace the multi-write initialization loop with a single byte write plus bitwise OR for subsequent mask bits

## Key Lessons

- Use bit-packed uint8_t masks with bitwise OR instead of bool arrays for tree/adjacency masks in GPU kernels — reduces memory footprint by 8x and cuts DRAM bandwidth proportionally, especially beneficial when mask values are sparse boolean indicators
- Replace cumulative-offset index computation (O(n) loop over variable-length rows) with direct multiply-by-stride formulas when the data layout permits uniform row sizes — eliminates variable-trip-count loops and conditional branching per thread, both of which hurt warp occupancy
