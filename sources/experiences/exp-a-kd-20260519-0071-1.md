## Solution

Double FragmentsPerPartition from 1 to 2, allocating 2x SMEM (1024 floats) and processing two fragments between each barrier pair, halving total barriers from 32 to 16 while partitioning SMEM into disjoint tiles that reduce bank conflicts and improve locality

## Key Lessons

- Increasing FragmentsPerPartition (FPP) in SMEM-pipelined epilogue loops halves barrier count per processed element — each __syncthreads() costs ~100–200 cycles on modern GPUs, so batching multiple fragments between barriers yields outsized speedup when iteration count is high
- Partitioning SMEM into disjoint per-fragment tiles via offset-based indexing (smem_off = p * smem_pointer_offset) reduces bank conflicts by giving each fragment its own memory region rather than having all threads thrash the same 512-float buffer across 16 iterations
- Barrier reduction and SMEM partitioning are multiplicative effects: fewer barriers means less stall time, and larger partitioned SMEM means better locality per barrier phase — combine both for maximum epilogue throughput
