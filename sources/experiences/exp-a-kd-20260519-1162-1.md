## Challenge

Grouped expert score accumulation kernel uses a runtime variable n_group as the outer loop bound, preventing the compiler from unrolling the 16-iteration loop and forcing it to emit branch, counter-increment, and comparison instructions on every iteration


## Solution

Convert the runtime n_group parameter into a compile-time template parameter NGroup and add #pragma unroll to the outer loop, enabling the compiler to flatten all 16 iterations into straight-line code, eliminate branch overhead, and interleave independent memory loads from different iterations for better instruction-level parallelism

## Key Lessons

- Convert loop-invariant runtime values to compile-time template parameters when the value is known at kernel launch — the compiler can unroll the loop, eliminate branch/comparison overhead, and reorder independent memory loads to hide latency
- Loop unrolling is most effective when the iteration count is moderate (8–32) and the loop body is lightweight — the overhead of branches and counter management becomes a noticeable fraction of total work, making straight-line code worthwhile
- Using a template parameter for the group count also enables compile-time address computation (tid * NGroup * stride), which the compiler can optimize into constant-offset addressing across unrolled iterations
