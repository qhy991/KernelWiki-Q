## Solution

Replace 16 per-group kernel launches (4 blocks each) with a single batched_rotary_embedding_kernel launch of 64 blocks, using a per-token cos_sin_cache_offsets array to handle different LoRA adapter cache offsets, eliminating 15 kernel launch overheads and GPU idle gaps between launches

## Key Lessons

- Batch multiple small kernel launches into a single launch with total thread blocks equal to the sum — eliminates N-1 launch overheads and GPU idle gaps, yielding near-linear speedup when launch overhead dominates compute time
- Use a per-token offset array (cos_sin_cache_offsets) to generalize a per-group token_offset parameter into a batched kernel, keeping the same per-thread work while enabling single-launch execution
- When each group's work is tiny (4 thread blocks for a 108-SM GPU), the multiplicative overhead of N launches far exceeds the compute savings of any per-group optimization — batching is the highest-impact transformation
