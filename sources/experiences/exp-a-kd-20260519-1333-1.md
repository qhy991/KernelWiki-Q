## Challenge

RMSNorm weights (q_weight/k_weight) are read from global memory inside a head loop that iterates HEADS_PER_WARP=4 times, causing redundant global memory accesses for data that is invariant across all loop iterations


## Solution

Hoist loop-invariant weight loads out of the head loop by preloading q_weight and k_weight into per-thread register arrays (q_w[4], k_w[4]) once before the loop, then reading from registers inside each iteration instead of issuing global memory loads

## Key Lessons

- Preloading loop-invariant global memory reads into registers produces no measurable speedup when the data (128 floats = 512 bytes) fits entirely in L1 cache after the first access and the loop trip count (4) is too small to amortize the added register pressure of 8 extra float registers per thread
- Register preload hoisting is beneficial only when L1 cache is heavily contested by concurrent data streams or when the loop trip count is large enough that the cumulative eliminated L1 hits outweigh the occupancy reduction from higher register usage
