## Challenge

Monolithic FP4 E2M1→FP16 dequant function fuses bitwise extraction and bias correction into one body, computing q<<4 twice for Out2 construction and offering no composability for callers that only need the raw shifted result without bias


## Solution

Split the monolithic dequant into two functions: skip_flop_true performs only bitwise extraction (reusing q via in-place q<<=4 to eliminate redundant shift), and the outer function calls it then applies bias via __hmul2 — providing composability without runtime cost since both are __device__ inline and the compiler already CSEs the duplicated shift at -O3

## Key Lessons

- Splitting a monolithic dequant into a skip_flop extraction stage and a bias-correction wrapper improves composability (callers can use raw shifted results) without runtime cost when both are __device__ inline — the compiler inlines the call boundary and produces identical SASS
- NVCC at -O3 already performs CSE on duplicated expressions like q<<4, so hand-eliminating the redundant shift via in-place q<<=4 improves readability but not performance — profile before assuming such micro-refactors matter
