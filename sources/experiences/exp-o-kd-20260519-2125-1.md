## Challenge

Paged attention K-cache is stored and loaded at fp16 precision (2 bytes/element), doubling memory traffic compared to the minimum needed when fp8 precision is acceptable for key vectors during attention computation


## Solution

Store K-cache as FP8-E5M2 (1 byte/element) and dequantize to fp16 via software bit-manipulation (integer shift/mask/or to reconstruct fp16 bit pattern) before the QK dot product, achieving 2x memory traffic reduction that outweighs the small integer conversion cost

## Key Lessons

- Storing K-cache in FP8-E5M2 instead of fp16 halves memory traffic for attention key reads; the 2x bandwidth saving outweighs the software dequantization cost even though FP8-E5M2 to fp16 has no hardware intrinsic on current GPUs
- Software FP8-E5M2 to fp16 conversion via integer bit-manipulation (shift/mask/or on sign/exponent/mantissa fields) compiles to a compact instruction sequence and avoids emulation overhead — prefer this over float-arithmetic conversion paths
- The observed speedup is diluted (9% instead of theoretical ~50% memory-only gain) because the dot-product FLOPs are unchanged; bandwidth-bound kernels will see larger gains from fp8 cache compression
