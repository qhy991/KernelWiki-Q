## Solution

Change k_scale and v_scale from scalar float to pointer (float*) parameters and dereference at the call site, enabling the caller to pass per-token or per-layer scaling factors for dynamic FP8 quantization accuracy in flash-layout KV cache

## Key Lessons

- Pass FP8 scaling factors as pointers rather than scalar values in KV cache kernels to enable dynamic per-token/per-layer quantization — the pointer indirection cost is negligible while the quantization accuracy improvement is significant
- When porting a scaling pattern from one kernel variant to another (e.g., standard to flash-layout), apply the same parameter interface change consistently so all KV cache kernel variants support the same dynamic scaling capability
