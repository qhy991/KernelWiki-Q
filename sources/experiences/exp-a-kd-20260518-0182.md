## Solution

Refactor to CUTLASS 3.0 device_kernel pattern: wrap kernel body in a TransformOperator struct with __device__ operator(), add __launch_bounds__(256, 2) and __grid_constant__ on the template entry point, and replace typed SharedStorage* with plain char* smem cast to float*

## Key Lessons

- Applying __launch_bounds__ and __grid_constant__ to global-memory-bandwidth-bound kernels yields negligible speedup because the dominant memory traffic hides any micro-optimization in register allocation or parameter access paths — verify the kernel's bottleneck before applying these annotations
- The CUTLASS 3.0 device_kernel pattern (operator struct + template wrapper) is primarily a code-organization and maintainability improvement that enforces launch bounds and parameter conventions, not a performance transformation in itself
- NVCC on CUDA 12.8 already routes read-only parameter struct accesses through the cache path for simple loop kernels, making __grid_constant__ redundant when the compiler can prove the parameter fields are read-only within the kernel body
