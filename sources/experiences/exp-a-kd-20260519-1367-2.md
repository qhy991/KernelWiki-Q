## Challenge

Scalar bfloat16 loads and stores (2 bytes per transaction) in the copy kernel severely underutilize available memory bandwidth when transferring the 512-element nope and 64-element rope dimensions per (token, head) pair


## Solution

Replace scalar bfloat16 accesses with inline PTX asm 128-bit (int4) vectorized loads/stores using cache-streaming (.cs) hints, transferring 8 bfloat16 elements per instruction for the nope dimension and 2 elements (b32) per instruction for the rope dimension, saturating memory bandwidth for bulk contiguous transfers

## Key Lessons

- Use int4 (128-bit) vectorized loads/stores via inline PTX asm with cache-streaming hints (.cs) for bulk contiguous transfers — each instruction moves 8 bfloat16 elements instead of 1, maximizing memory throughput for bandwidth-bound copy kernels
- Compute vector load trip counts as compile-time constants (constexpr) and use #pragma unroll to fully unroll the inner copy loop, eliminating loop overhead and enabling the assembler to schedule back-to-back memory transactions
