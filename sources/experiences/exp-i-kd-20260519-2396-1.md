## Challenge

Implement a generic gather kernel that reads elements from an input buffer using an indirect index-mapping function, requiring efficient stride-based index decoding and batch-wise parallelism across arbitrary element types


## Solution

Compute per-batch base pointers via blockIdx.z offset, then use a grid-stride loop where each thread iterates over output elements with stride equal to total thread count, decoding each linear index k into (j, i) via FastDivmod and performing the indirect gather input_b[i + func(j) * stride_divmod.divisor]

## Key Lessons

- Grid-stride loops (k += blockDim.x * gridDim.x) handle arbitrary output sizes without requiring the caller to compute exact grid dimensions — applies to any gather/scatter kernel where work per element is uniform
- FastDivmod replaces integer division/modulo with multiply-shift for stride-based index decoding, removing expensive DIV instructions from the hot path — use whenever a fixed stride is known at kernel launch
- Compute per-batch base pointers once via blockIdx.z offset before the loop instead of re-deriving them per element — reduces address arithmetic in the loop body and improves register usage
