## Challenge

MoE fused gate kernel uses KernelParamsDynamic with runtime int fields for VPT, THREADS_PER_ROW, and ROWS_PER_WARP, preventing NVCC from fully unrolling #pragma unroll loops that iterate over these bounds, incurring loop overhead and limiting instruction-level parallelism across independent VPT elements


## Solution

Replace KernelParamsDynamic (runtime int fields) with a templated KernelParams<VPT, NUM_EXPERTS, THREADS_PER_ROW, ROWS_PER_WARP, ROWS_PER_CTA, WARPS_PER_CTA> struct where all fields are static constexpr, making every #pragma unroll loop bound known at compile time and enabling NVCC to fully unroll the 16-iteration VPT loops (sigmoid, bias-add, argmax) and 4-iteration shuffle reduction into straight-line code

## Key Lessons

- Promote kernel configuration parameters (VPT, THREADS_PER_ROW, ROWS_PER_WARP) from runtime int fields to compile-time static constexpr template parameters when the configuration is known at launch time — this lets NVCC fully unroll #pragma unroll loops, eliminating branch/counter overhead and enabling better instruction scheduling across independent loop iterations
- Full unrolling of moderate-count loops (8-32 iterations) with compile-time bounds produces straight-line code that hides memory latency through instruction-level parallelism — especially effective when each iteration contains independent arithmetic (sigmoid, bias-add, argmax comparisons)
