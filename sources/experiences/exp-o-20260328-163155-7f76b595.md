## Key Lessons

- Parallel Phase 4 V accumulation using all 256 threads with strided access and warp-level reductions was implemented
- Warp-level reduction pattern: threads 0-127 compute first half of partial sums, threads 128-255 compute second half
- Shared memory s_partial[HEAD_DIM * 2] used for storing partial accumulations before warp reduction
- Correctness passed (NMSE=0) for all batch sizes (1, 8, 512)
- Speedup did NOT achieve >1.0 goal (0.0246x for batch_512)
- Possible issue: warp-level reduction adds synchronization overhead that may negate parallelization gains
- Alternative approach needed: consider block-leveltiling or using shared memory to reduce memory access latency rather than just parallelizing the inner loop
