## Challenge

MoE permutation unconditionally clones the topk_ids tensor via cudaMalloc + cudaMemcpy device-to-device before passing it to sortAndScanExpert, even when no expert_map is present and the downstream kernel only reads the data — wasting GPU memory allocation and bandwidth on an unnecessary copy


## Solution

Replace the unconditional cudaMalloc + cudaMemcpy clone with a shallow pointer assignment (int* d_topk_ids_for_sort = d_topk_ids) when expert_map is absent, eliminating both GPU memory allocation and device-to-device memcpy since the downstream consumer only reads the data

## Key Lessons

- Replace unconditional tensor cloning with conditional shallow pointer assignment when the downstream consumer is read-only and no mutation is needed — eliminates cudaMalloc + cudaMemcpy overhead entirely, which is especially significant for per-inference hot paths
- Lazy/conditional copy patterns in MoE routing avoid unnecessary GPU memory traffic: only clone when expert_map rewriting actually requires mutation (has_expert_map case), otherwise use the original tensor directly
