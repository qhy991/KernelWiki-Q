## Challenge

The quantization epilogue loop uses scalar float loads and int8 stores (one element per transaction), underutilizing memory bandwidth, and performs per-element division by scale_val instead of precomputing the multiplicative inverse


## Solution

Replace the scalar quantization loop with vectorize_with_alignment<16> which processes 16 elements per vector transaction, and precompute inv_s = 1/scale_sh outside the loop to convert division into multiplication, reducing both memory transaction count and arithmetic latency per element

## Key Lessons

- Use vectorize_with_alignment<N> to coalesce N scalar loads/stores into single vector transactions in element-wise transformation loops — applies when input and output arrays have compatible alignment and the per-element operation is independent
- Precompute the multiplicative inverse (1/scale) outside the loop and multiply inside instead of dividing per element — replaces a ~20-cycle FP division with a ~5-cycle multiplication on each iteration
