## Challenge

The min_scaling_factor value (1.0f / (448.0f * 512.0f)) is passed as a runtime float kernel parameter, adding an unnecessary kernel argument and preventing the compiler from constant-folding the value at compile time


## Solution

Replace the runtime min_scaling_factor kernel parameter with a template struct (min_scaling_factor<FP8E4M3FN>::val()) using constexpr and __forceinline__ so NVCC constant-folds the expression 1.0f / (448.0f * 512.0f) at compile time, eliminating the parameter slot and producing identical register-level code

## Key Lessons

- Replacing a runtime float kernel parameter with a constexpr + __forceinline__ template struct produces identical SASS after NVCC constant-folding — use this pattern to remove kernel arguments for values derivable from type information, improving code clarity without performance regression
- Eliminating a single kernel parameter has negligible performance impact on compute-bound or memory-bandwidth-bound kernels where each thread processes thousands of elements — the parameter overhead (one register load) is dwarfed by the dominant workload
