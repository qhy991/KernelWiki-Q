## Solution

Trim shared memory to the exact working-set dimensions (kRows*kK + kK*kCols), split the load into two separate unconditional loops over the actual A and B tiles, and use correct strides (kK, kCols) in the compute loop instead of hardcoded 128

## Key Lessons

- Size shared memory buffers to the exact working-set dimensions rather than rounding up to a power-of-two tile — halving shared memory per CTA directly doubles occupancy on shared-memory-bound kernels
- Eliminate conditional guards inside shared-memory load loops by splitting a single oversized loop into separate per-tile loops sized to each matrix dimension — removes warp divergence and zero-initialization overhead from unused elements
- Use named compile-time constants (kK, kCols) instead of hardcoded magic numbers (128) for shared-memory strides — ensures correctness when dimensions are not powers of two and makes the tiling contract explicit
