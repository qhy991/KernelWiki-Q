## Challenge

Scalar int4-to-fp16 dequantization loop extracts each 4-bit value with shift-and-mask, then converts to half via __float2half one element at a time (8 iterations per uint32), resulting in high instruction count and no utilization of the half2 SIMD pipeline


## Solution

Replace the scalar loop with 4 lop3.b32 PTX instructions that extract two int4 pairs per instruction using a lookup-table immediate, then apply sub.f16x2 and fma.rn.f16x2 packed half2 arithmetic to convert all 8 values in 8 total instructions, leveraging the native half2 pipeline

## Key Lessons

- lop3.b32 with a compile-time lookup-table immediate can extract and combine multiple nibble fields from a packed int4 word in a single instruction, replacing scalar shift-and-mask loops — applies whenever multiple sub-byte fields need parallel extraction
- Packed half2 PTX arithmetic (sub.f16x2, fma.rn.f16x2) processes two fp16 values per instruction on the native half2 pipeline, doubling throughput over scalar __float2half conversion — applies to any elementwise fp16 computation where inputs can be arranged in pairs
- Relaxing overly conservative __CUDA_ARCH__ guards (e.g., allowing sm_75 to use lop3.b32 which is supported since sm_70) unlocks hardware capabilities that were always available but artificially gated by incorrect architecture checks
