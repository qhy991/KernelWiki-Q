## Solution

The heuristic correctly selects PersistentScheduler for balanced GEMM shapes (K <= 3*N), confirming that the persistent model already provides full SM utilization when there are enough output tiles — no scheduler switch to StreamK is needed

## Key Lessons

- For balanced GEMM shapes where K <= 3*N, the persistent scheduler has enough output tiles (tiles_m * tiles_n >= SM_count) to keep all SMs busy, so StreamK decomposition provides no benefit — the heuristic should simply fall through to PersistentScheduler
- Dispatch heuristics for GEMM scheduler selection should use K-to-N ratio as a proxy for tile sufficiency: when K <= 3*N the output tile count typically exceeds SM count, making the standard persistent approach optimal
