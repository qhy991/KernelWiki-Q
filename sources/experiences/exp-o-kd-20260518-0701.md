## Challenge

Optimization focus: Parameterized warp reduce_sum with kNumThreads template to support sub-warp reductions, reducing unnecessary shuffle rounds.
Problem before: reduce_sum always performed 5 __shfl_xor_sync shuffle rounds (mask=16,8,4,2,1) regardless of how many threads actually participate in the r


## Solution

consuming instruction slots and increasing latency without contributing to the result.
Applied change: Added a compile-time kNumThreads template parameter (default 32 = kWarpThreads) that controls the initial mask value. The loop now starts at kNumThreads/2, performing exactly log2(kNumThreads) shuf

## Key Lessons

- When only a subset of warp lanes is active (e.g., 8 or 16 threads), the extra shuffle rounds are wasted because they shuffle with lanes holding zero or irrelevant values, consuming instruction slots and increasing latency without contributing to the result.
