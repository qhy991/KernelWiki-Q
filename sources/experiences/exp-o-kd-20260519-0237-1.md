## Challenge

Element-by-element XOR on packed 1-bit arrays uses sub-byte get() (shift+mask) to extract each bit and set() (conditional read-modify-write) to insert the result, requiring 128 iterations with multiple instructions per element instead of operating on the naturally packed uint32_t storage words


## Solution

Replace the 128-iteration element-wise loop (get/set sub-byte access) with a 4-iteration word-level XOR loop that directly XORs the uint32_t storage words, eliminating all shift-mask and conditional read-modify-write overhead since XOR distributes bitwise across packed words

## Key Lessons

- Bitwise operations (XOR, AND, OR, NOT) on packed sub-byte arrays can be applied directly to the storage words since these operations are bitwise-independent — replacing N element extractions with N/32 word-level operations reduces iteration count and eliminates shift-mask overhead by a factor of the word size
- When data is already packed in word-aligned storage, operate on the native word type (uint32_t/uint64_t) rather than unpacking to element granularity — applies to any bitwise logical operation on packed bit arrays, packed boolean masks, or bitmap indices
