## Solution

Remove the redundant cumsum_buffer.zero_() call entirely since align_kernel explicitly writes cumsum[0]=0 and computes cumulative sums for indices 1 through num_experts, fully initializing every entry that subsequent kernels read

## Key Lessons

- Remove redundant host-side buffer initialization (zero_()) when the subsequent kernel already writes every buffer element explicitly — eliminates an extra CUDA kernel launch and avoids potential host-device synchronization overhead, applying to prefix-sum, histogram, and scatter patterns where all output entries are unconditionally written
- Verify downstream kernel write coverage before keeping pre-initialization: if a kernel writes cumsum[0]=0 and cumsum[i]=cumsum[i-1]+count for all i in [1..num_experts], the buffer is fully initialized and no prior zeroing is needed
