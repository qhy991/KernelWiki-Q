## Challenge

Online softmax rescaling stores output accumulators in FP16, requiring __half2float conversion before each FMA and __float2half_rn rounding after — 2 conversion instructions per element per rescale step, accumulating rounding error across the K-seqlen tiling loop


## Solution

Store output accumulators in FP32 (kOStorageAccFloat32=1) so the online rescaling FMA operates directly on float values without intermediate half↔float conversion, eliminating 8 conversion instructions per rescale step and improving numerical precision; conversion to FP16 happens only once at final global store

## Key Lessons

- Store attention output accumulators in FP32 during online softmax rescaling instead of FP16 — eliminates 2 conversion instructions (__half2float + __float2half_rn) per element per rescale step and prevents intermediate rounding error accumulation across the K-dimension tiling loop
- FP32 accumulator storage doubles register footprint (4 vs 2 registers per element) but is worthwhile when the kernel has repeated accumulation steps across a tiled dimension, as the conversion overhead saved outweighs register pressure for typical head dimensions
