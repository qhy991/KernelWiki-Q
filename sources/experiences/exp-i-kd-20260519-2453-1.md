## Challenge

Skeleton kernel requires implementing a complete GEMM threadblock tensor op multiplicand store: constructing a fragment, computing the lane/warp-based memory access pattern for loading input data, and using CUTLASS RegularTileIterator to write tile-aligned output for tensor operations


## Solution

Construct a zero-initialized fragment, compute per-thread input pointers using (lane_id % 8) for contiguous element offset and (lane_id / 8) for strided access, then iterate over ThreadMap's kStrided and kContiguous dimensions loading elements into the fragment, and finally use RegularTileIterator::store() to write the fragment in TensorOpMultiplicandCongruous layout

## Key Lessons

- CUTLASS RegularTileIterator decouples the memory access pattern from the store operation: threads compute their own input offsets via (lane_id % 8) for contiguous element indexing and (lane_id / 8) for strided row indexing, then iterator.store() handles the complex TensorOpMultiplicandCongruous layout remapping automatically
- Fragment indexing follows a fixed layout formula: v + kElementsPerAccess * (c + s * kContiguous) maps the 3D (vector,contiguous,strided) iteration space into a linear fragment array — always use this canonical ordering when manually filling fragments for CUTLASS iterators
- Warp-level offset for multi-warp tiles is computed as (warp_id * Shape::kStrided / kWarpCount) * stride, distributing warps evenly across the strided dimension of the tile
