## Solution

Replace __shfl_sync warp-index broadcast with plain integer division threadIdx.x / 32 wrapped in canonical_warp_idx(), which compiles to a single shift-right instruction with no cross-lane synchronization, since all threads in a warp inherently compute the same value

## Key Lessons

- Avoid __shfl_sync for broadcasting values that all threads in a warp already compute identically (e.g., threadIdx.x / 32) — plain integer division compiles to a single shift-right with no barrier, removing per-iteration synchronization overhead
- Extract repeated index computations into __forceinline__ __device__ helpers like canonical_warp_idx() for clarity and consistent optimization across call sites
