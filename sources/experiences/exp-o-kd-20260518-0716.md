## Challenge

Optimization focus: Added new cvt_fp16_to_fp4_expert kernel with direct thread-to-expert assignment, replacing offset-array search for the mask case.
Problem before: When processing masked expert quantization, the old cvt_fp16_to_fp4 kernel required each thread to search through input_offset_by_expe


## Solution

offset arrays, search loop iterations, and potential shared memory bank conflicts. The continue-based early exit for masked rows also means threads keep iterating across the entire global row range even after their expert's work is done.
Applied change: A new dedicated kernel cvt_fp16_to_fp4_expert 

## Key Lessons

- This eliminates all offset-array lookups, shared memory loading, and search loops.
- The grid size must be a multiple of n_experts for correct thread distribution, which quant_impl enforces by rounding up.
