## Solution

Distribute exp2f across all 4 warps by mapping rows via id = warp_id * kLinesPerWarp + lane_id with guard lane_id < kLinesPerWarp, allowing the warp scheduler to interleave execution and hide the ~20-cycle exp2f latency; also separate the rescale factor into a dedicated out_rescale array so m_prime retains the running max for correctness

## Key Lessons

- Partition per-row exp2f computations across all available warps (id = warp_id * kLinesPerWarp + lane_id) instead of concentrating them on warp 0 — the warp scheduler can interleave independent exp2f pipelines to hide ~20-cycle transcendental latency
- Use a dedicated out_rescale array separate from m_prime so the running maximum is preserved across iterations, preventing correctness bugs from overwriting the max with the rescale factor
