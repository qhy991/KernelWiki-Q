## Challenge

HGEMV kernel processes one scalar half element per thread per iteration, issuing 4 separate scalar load instructions across 4 loop iterations for a K=128 row, wasting memory bandwidth and incurring loop overhead for each element


## Solution

Replace 4 scalar half iterations with 1 vectorized iteration using half2 loads (HALF2 reinterpret_cast), loading 4 elements via two half2 reads each for matrix A and vector x, then manually fusing the 4 multiply-adds via .x/.y field extraction, cutting loop overhead by 4x and doubling memory transaction width

## Key Lessons

- Vectorize HGEMV inner-loop loads from scalar half to half2 (2 elements per load) to halve the number of memory transactions and reduce loop iterations by the vectorization factor — applies when K is a multiple of the vectorization width and addresses are half2-aligned
- reinterpret_cast<half2*>(&value)[0] (HALF2 macro) enables vectorized loads from half-typed arrays without changing the array type or allocation, avoiding pointer-aliasing issues when the underlying memory is contiguous and 4-byte aligned
- Reducing loop iteration count by 4x in the K-dimension eliminates proportionally more branch overhead and index arithmetic instructions, compounding the bandwidth benefit of wider loads
