## Challenge

Stride for a non-packed matrix is derived from the problem shape dimensions (row_stride=K) assuming a packed layout, but the actual data resides in a buffer with leading dimension LDA > K, causing the kernel to read incorrect elements from the padded region of each row


## Solution

Read the actual leading dimension from the input tensor (lda = a.stride(0)) and set row_stride = LDA so the stride matches the real memory layout, producing correct results for both packed and non-packed tensors

## Key Lessons

- Never assume a packed layout (stride = problem dimension) when consuming tensor data — always read the actual stride from the source tensor's leading dimension to handle non-contiguous (padded) memory correctly
- Computing stride via make_cute_packed_stride(shape) is only valid for contiguous tensors; for column slices or views into larger matrices, the stride must come from the original tensor metadata, not the sub-problem shape
