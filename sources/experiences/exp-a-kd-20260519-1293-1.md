## Solution

Replace the per-nibble branching function with a 256-entry __constant__ memory lookup table (kNibbleLUT) that pre-computes the full byte-to-byte mapping, eliminating conditional logic and reducing per-element work to a single table lookup

## Key Lessons

- At -O3, nvcc inlines simple __device__ branch functions into efficient predicated/select instructions that can outperform constant-memory LUT — compiler-predicated branching on data-independent conditions is essentially free
- Constant-memory LUT accesses (one 8-bit load per byte from a 256-byte table) generate actual memory load instructions per element, which may be slower than inlined predicated logic when the branch condition is simple and data-independent
