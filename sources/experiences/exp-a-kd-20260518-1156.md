## Challenge

The before code performs a full CUB DeviceRadixSort on topk_ids (65536 int pairs) plus a device-side index re-initialization kernel every iteration, followed by count/offset/build kernels. The sort itself requires 4 passes over 32-bit keys and additional temporary storage reads/writes


## Solution

The after code replaces the sort with compute_arg_sorts, which uses atomicAdd-based scatter: each thread that finds topk_ids[i] matching its expert_id atomically claims the next slot.
