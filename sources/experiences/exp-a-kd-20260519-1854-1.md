## Challenge

Single kernel launch with dim3(block_num_x, ky, 1) hardcodes ky as the grid y-dimension, which fails silently or crashes when ky exceeds the CUDA grid limit of 65535 along any axis


## Solution

Replace the single kernel launch with a chunked loop that iterates over ky in blocks of at most MAX_BLOCK_SIZE=65535 rows per launch, offsetting input and output pointers for each chunk to handle arbitrarily large row counts within CUDA grid dimension limits

## Key Lessons

- Wrap kernel launches in a chunked loop capped at 65535 per grid dimension to handle arbitrarily large batch sizes — the loop overhead is negligible (nanoseconds) compared to kernel execution time, and for cases below the limit the loop runs exactly once with identical configuration
- When chunking grid launches, correctly offset both input (&x[off * kx]) and output pointers ((int32_t*)vy + off * (kx_padded / 32 * 9)) so each chunk processes a contiguous slice of rows independently
