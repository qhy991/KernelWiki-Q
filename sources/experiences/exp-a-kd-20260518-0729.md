## Challenge

Both before and after source files compute dot(float2, float2) as a two-element multiply-add: compute {a.x*b.x, a.y*b.y} then sum c.x+c.y. The before path chains through sum(mul<float2,float2,float2>(a,b)) where mul computes the component-wise products and sum reduces via v.x+v.y


## Solution

The after path uses the specialized dot(float2,float2) overload that calls mul<float2,float2,float2>(a,b) then explicitly computes c.x+c.y.
