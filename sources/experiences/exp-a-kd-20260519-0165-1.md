## Challenge

The SK-wave scheduling branch in get_blocks() uses condition `full_waves % sm_occupancy == sm_occupancy - 1` without guarding sm_occupancy > 1; when sm_occupancy==1, this condition is trivially always true (N%1==0==0), causing the branch to be taken unconditionally and producing incorrect tile dispatch decisions


## Solution

Add an sm_occupancy > 1 guard to the SK-wave branch condition so that it only fires for genuine multi-occupancy scenarios where the modulo-based wave-alignment check is meaningful

## Key Lessons

- Any condition of the form `x % divisor == divisor - 1` is trivially true when divisor==1 (since N%1==0==0), so always guard such modulo-based branch conditions with a divisor > 1 check to avoid unintended unconditional entry
- Correctness fixes in SM scheduling logic may show neutral performance on multi-occupancy test cases because the guard is already satisfied — the real impact is on single-occupancy configurations where the bug produces wrong dispatch decisions
