## Solution

Reduce pipeline stages from 5 to 3 for large M/N configurations in the shape-aware tile selector, trading prefetch depth for reduced shared memory pressure and higher SM occupancy when wider tiles already capture sufficient working set

## Key Lessons

- Reducing CUTLASS pipeline stages (5 to 3) lowers shared memory per threadblock, increasing occupancy — beneficial when wider tile dimensions already capture enough working set to hide memory latency without deep prefetching
- Pipeline stage count should be tuned jointly with tile shape: wider tiles (larger tile_m/tile_n) can tolerate fewer stages because each threadblock processes more data per pipeline fill, making the prefetch depth-vs-occupancy tradeoff favorable
