## Solution

Decompose the single m16n8k16 MMA into two m16n8k8 MMA instructions — the first processes K-half 0 with a[0],a[1]×b[0] and the second processes K-half 1 with a[2],a[3]×b[1], both accumulating into the same 4 float registers, providing sm75 (Turing) backward compatibility with no performance penalty since kernel launch overhead dominates at microbenchmark granularity

## Key Lessons

- m16n8k16 MMA can be decomposed into two m16n8k8 MMA instructions (splitting along the K dimension) to gain sm75 Turing compatibility — the two halves accumulate into the same 4 float registers, producing numerically identical results with no performance penalty at microbenchmark granularity
- At single-instruction granularity, kernel launch overhead (~2.5us) and global memory access latency dominate over the MMA instruction cost itself, so replacing one MMA with two does not measurably affect latency
- The m16n8k8 PTX instruction takes 2 A-registers and 1 B-register per call (vs 4+2 for m16n8k16), making register pressure analysis easier when decomposing larger MMA shapes for backward-compatible code paths
