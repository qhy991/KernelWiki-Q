## Solution

Replace the runtime bool argument with a template parameter bool is_scale_inverted and use if constexpr to dispatch at compile time, so the compiler emits only the multiply or divide path and eliminates the per-element branch predicate entirely

## Key Lessons

- Move uniform control-flow decisions (same value for all threads, known at kernel launch) into template parameters with if constexpr — eliminates per-element branch predicates and lets the compiler optimize the single remaining path more aggressively
- Wrap the compile-time dispatch in a templated functor struct (ScaledQuant) to keep the kernel body clean and allow partial specialization for different quantization types
