## Solution

Fuse the alignment and padding operations into a single fused_align_and_padding_kernel launch, using __syncthreads() to order the cumsum computation before the padding fill within the same kernel, eliminating one full kernel launch overhead

## Key Lessons

- Fuse short-lived GPU kernels (especially when total runtime is ~10 microseconds) into a single launch — the kernel submission overhead of ~2-3 microseconds per launch becomes a dominant cost for small workloads
- Use __syncthreads() within a fused kernel to enforce ordering between dependent phases (e.g., cumsum computation before padding fill) instead of relying on separate kernel launches for implicit synchronization
