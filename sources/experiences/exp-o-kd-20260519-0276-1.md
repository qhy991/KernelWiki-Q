## Challenge

GELU activation uses erff(scalar / ROOT_TWO_F) where the FP32 division maps to the MUFU.DIV SFU instruction with roughly 1/16th of FFMA throughput, creating a per-element bottleneck in the otherwise compute-light kernel


## Solution

Replace scalar / ROOT_TWO_F with scalar * HALF_ROOT_TWO_F (precomputed 1/sqrt(2)), converting the low-throughput SFU division into a full-throughput FFMA multiply that the compiler can fuse with the surrounding 0.5f * scalar * ... FMA chain

## Key Lessons

- Replace a / constant with a * (1/constant) when the divisor is known at compile time — this converts a low-throughput SFU division (MUFU.DIV, ~1/16 FFMA rate on Ampere) into a full-throughput FFMA multiply that can fuse with adjacent FMA operations
- Precompute reciprocal constants (e.g., HALF_ROOT_TWO_F = 1/sqrt(2)) as static const floats so the compiler hoists them into constant memory or immediate operands, eliminating any runtime overhead
