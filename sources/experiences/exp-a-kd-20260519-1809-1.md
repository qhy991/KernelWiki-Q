## Challenge

The int8 quantization kernel receives its scale factor as a host scalar via kernel launch parameters, requiring a cudaMemcpy(DeviceToHost) synchronization to extract the 4-byte scale value from GPU memory before each kernel invocation, stalling the CUDA stream and forcing the CPU to wait for GPU completion


## Solution

Pass the scale as a device pointer instead of a host scalar; the kernel dereferences *scale_ptr on the GPU, eliminating the per-invocation cudaMemcpy DeviceToHost sync and keeping the entire dataflow on-device

## Key Lessons

- Pass scalar kernel arguments as device pointers (const T* ptr) and dereference inside the kernel (*ptr) instead of performing a D2H cudaMemcpy to extract the value first — eliminates stream-stalling synchronization that dominates latency for small-value transfers
- A single 4-byte cudaMemcpy(DeviceToHost) costs orders of magnitude more than a device-side pointer dereference because it forces full GPU-CPU synchronization, making this pattern especially harmful in repeated invocation loops such as inference serving
