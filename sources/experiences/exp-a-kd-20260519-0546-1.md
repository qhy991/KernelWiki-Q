## Challenge

Scalar elementwise addition kernel processes one float per thread, issuing three separate 4-byte memory transactions (two loads, one store) per element, underutilizing available memory bandwidth


## Solution

Vectorize memory access using float4 (128-bit) loads and stores so each thread processes 4 elements at once, replacing 12 scalar 4-byte transactions with 3 vectorized 16-byte transactions and reducing total memory instruction count by 4x

## Key Lessons

- Vectorizing memory access with float4 reduces instruction count by processing 4 elements per thread — applies to any memory-bandwidth-bound elementwise kernel with contiguous, aligned access patterns
- For small workloads (N=1M ≈ 4MB total), kernel launch overhead and timing noise can mask bandwidth gains from vectorization — measure with sufficiently large N to see the full benefit
