## Challenge

The init_same_group kernel always reads g_idx from shared memory (2 loads per pipe × 8 pipes = 16 loads) and computes pointer offsets even when has_act_order is false and the g_idx values are never used for grouping decisions


## Solution

Convert the runtime has_act_order flag into a compile-time template parameter HAS_ACT_ORDER, then insert an early-continue guard that writes constants and skips all g_idx pointer arithmetic and shared-memory loads; the compiler dead-code-eliminates the entire read path when HAS_ACT_ORDER is false

## Key Lessons

- Promote runtime flags to compile-time template parameters when the flag's value is known at kernel launch — the compiler eliminates entire dead code paths (including shared-memory loads and pointer arithmetic), reducing both instruction count and memory bandwidth consumption
- Early-return or early-continue guards that skip shared-memory reads reduce bandwidth pressure even when the skipped values are semantically unused — shared-memory loads still consume shared-memory bandwidth and occupy load slots in the instruction pipeline
