## Challenge

Vectorized RMS normalization loop lacks #pragma unroll, but with hidden_size=4096 vectorized into 1024 vec4_t elements dispatched across 1024 threads (stride=blockDim.x), each thread executes only one iteration, making the pragma a no-op


## Solution

Add #pragma unroll 4 to the vectorized accumulation loop, but it has no measurable effect because iteration count = num_vec_elems / blockDim.x = 1, leaving the compiler with nothing to unroll

## Key Lessons

- #pragma unroll N has zero effect when each thread executes only 1 iteration (iterations = num_elements / (blockDim.x * stride)) — unrolling requires multiple loop bodies for the compiler to overlap instructions and hide latency
- Before applying loop unrolling pragmas, compute the per-thread iteration count: if iterations <= 1, the pragma is dead code and adds unnecessary compile-time complexity
- Loop unrolling optimizations on reduction-style kernels only benefit configurations where hidden_size >> 4 >> blockDim.x, i.e., where each thread accumulates multiple vec4 chunks before the block-wide reduce
