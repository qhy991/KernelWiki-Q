## Solution

Replace the group-aligned offset formula with direct pipe indexing (S_SH_STAGE * pipe) so each pipeline stage reads from its own distinct shared-memory slot, fixing the aliasing bug while simplifying the address arithmetic

## Key Lessons

- Integer division and remultiplication (pipe / GROUP_RATIO * GROUP_RATIO) truncates to the group-start index — this is a correctness bug, not a performance optimization, when distinct data per pipe is required
- Shared-memory microbenchmarks with simple access patterns may show negligible latency differences between addressing schemes because arithmetic overhead is dominated by memory access latency — the real impact manifests in end-to-end pipeline correctness and downstream computation accuracy
- Prefer the simplest correct indexing expression (direct pipe offset) over group-aligned formulas unless the group aliasing is intentional (e.g., weight sharing across stages)
