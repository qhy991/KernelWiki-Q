## Solution

Replace 8 explicit asm volatile PTX statements with a #pragma unroll for-loop over a reinterpret_cast<half2*> view of the uint4 fields, fusing subtract-multiply via __hsub2/__hmul2 intrinsics into a single expression per iteration, allowing the compiler to potentially schedule operations across iterations

## Key Lessons

- Replacing asm volatile PTX with equivalent __hsub2/__hmul2 intrinsics yields no speedup when nvcc lowers the intrinsics to the same PTX instructions (sub.f16x2/mul.rn.f16x2) — the intrinsic route only helps when it enables the compiler to apply optimizations the asm barrier prevented
- The RAW dependency chain (sub → mul per field) exists identically in both approaches since each iteration's __hmul2 consumes __hsub2's result; cross-iteration scheduling does not materialize because each iteration has independent inputs, not inter-iteration dependencies
- For microbenchmarks at this scale (~7μs kernel time), a 1.27% timing delta falls within measurement noise — treat speedups under ~3% on sub-10μs kernels as neutral
