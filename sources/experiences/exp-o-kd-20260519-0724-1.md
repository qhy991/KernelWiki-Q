## Solution

Fuse rope_only_kernel and kv_save_kernel into a single fused_rope_kv_kernel so that k_out stays in registers after the RoPE computation and is written directly to both k_rope and k_buffer without an intermediate global memory round-trip, while V data is loaded once and reused for the buffer write

## Key Lessons

- Fuse sequential kernels that produce and consume the same intermediate tensor (k_rope) into a single kernel so the intermediate result stays in registers instead of round-tripping through global memory — applies whenever kernel A writes output that kernel B immediately reads as input
- When fusing kernels, load all input data (K and V) and compute all output destinations (k_rope, k_buffer, v_buffer) within the same per-head loop iteration, keeping values in registers across writes to different output buffers
- Kernel fusion delivers speedups even at medium batch sizes (nnz=32) where launch overhead is a smaller fraction, because the primary benefit is reduced global memory traffic rather than eliminated launch overhead
