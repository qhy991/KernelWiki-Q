## Solution

Double-buffer shared memory into 2 stages (sA[2][BM][BK], sB[2][BK][BN]), load first K-tile in prologue, then in mainloop overlap next-tile global load into write_stage with current-tile compute on read_stage, alternating stages with XOR swap

## Key Lessons

- Double-buffering shared memory (2 stages) enables overlapping global load with compute via prologue/mainloop/write_stage XOR swap, but only yields speedup when the compute phase is heavy enough to hide global memory latency — scalar FMAs on 64x64x16 tiles are insufficient
- Doubling shared memory per block (2 stages) reduces SM occupancy, which can offset latency-hiding gains; this tradeoff is only favorable when compute is genuinely dominant (e.g., warp-level tensor core MMA in CUTLASS)
