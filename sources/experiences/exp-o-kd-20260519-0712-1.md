## Challenge

MoE fused gate kernel loads 384 expert scores and biases using scalar stride accesses (12 strided 32-bit loads per warp lane), resulting in poor memory coalescing and underutilized memory bandwidth in the data loading phase


## Solution

Replace scalar stride loads with float4 vectorized loads via reinterpret_cast, reducing each warp lane from 12 strided 32-bit loads to 3 coalesced 128-bit loads and improving memory transaction efficiency in the data loading phase

## Key Lessons

- Replace scalar stride loads with float4 vectorized loads when warp lanes access contiguous array segments — reduces memory transaction count by 4x and improves coalescing with a nested unroll loop over vector elements
- Memory bandwidth optimizations yield smaller relative speedups when compute-heavy phases (e.g., TopK reduction loops) dominate kernel time — profile to confirm the load phase is a meaningful fraction of total latency before investing in vectorization
