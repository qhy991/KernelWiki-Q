## Solution

Replace per-element scalar bf16 loads with vectorized uint4 reinterpret_cast loads (128 bits = 8 bf16 values per instruction), reducing load instructions from 40 to 5 per k-iteration and enabling coalesced memory transactions that reduce L2 cache contention

## Key Lessons

- Replace scalar bf16 loads with vectorized uint4 reinterpret_cast loads (128-bit = 8 bf16 values per transaction) to reduce load instruction count by 8x and improve memory coalescing — applies whenever VPT ≥ 8 and bf16 data is 16-byte aligned
- Multi-token GEMM kernels amplify memory bandwidth bottlenecks because each k-iteration scales load count by the number of tokens, making vectorized loads even more impactful at higher token counts (3.19x for 4 tokens vs smaller speedup for 1 token)
- Preload k_bases array before the main loop (k_bases[ki] = ki * k_elems_per_k_iteration + tid * VPT) to move the address computation out of the hot loop and improve instruction-level parallelism
