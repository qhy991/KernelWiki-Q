## Solution

Fuse sigmoid+bias computation directly into the top-2 selection loop: each thread reads raw scores[i] and group_bias[i], applies sigmoid_accurate() (a single SFU tanhf instruction) and adds bias inline, then immediately compares in the top2 tracking — eliminating the separate kernel, the intermediate buffer, and the global-memory write+read round-trip while keeping transformed values in registers

## Key Lessons

- Fuse element-wise transformations (sigmoid, bias addition) directly into downstream reduction/selection kernels instead of materializing intermediate results to global memory — eliminates one kernel launch, one buffer allocation, and one full global-memory write+read round-trip
- When an intermediate result is consumed exactly once by a subsequent kernel, keeping it in registers via kernel fusion is always preferred over writing to global memory — the register pressure increase is negligible compared to the bandwidth saved
