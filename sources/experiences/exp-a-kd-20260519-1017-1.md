## Challenge

Warp-level RMS norm for head_dim=1024 uses only 32 threads (single warp) per work item with 16 packed bf16x2 pairs per thread, causing high register pressure that forces a two-pass approach: Pass 1 loads input for sum-of-squares, Pass 2 re-loads both input and weight for normalization, doubling global memory traffic


## Solution

Restructure from warp-level two-pass to CTA-level single-pass: use 128 threads (full CTA, 4 warps) per work item with only 4 pairs per thread, reducing register pressure enough to cache input and weight in registers during Pass 1, then apply normalization without re-loading — uses shared memory for cross-warp reduction instead of warp-only shuffle

## Key Lessons

- When per-thread register footprint forces a multi-pass design (re-loading data from global memory), expanding to more threads with fewer elements per thread can reduce registers enough to enable single-pass execution, eliminating redundant memory traffic — the dominant cost in memory-bound kernels
- Using a full CTA (128 threads) instead of a single warp (32 threads) for a work item provides 4x more concurrent memory-level parallelism for global loads, which is especially beneficial when head_dim is large and the kernel is memory-bound
- For cross-warp reductions within a CTA, a two-stage shared memory pattern (per-warp partials → smem → warp-0 final reduction → smem) costs two __syncthreads() barriers but is negligible compared to the savings from eliminating redundant global memory access
