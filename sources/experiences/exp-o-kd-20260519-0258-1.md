## Solution

Reduce pipeline stages from 4 to 1 by computing kStages=min(default_stages, ceil(kMaxK/kBlockK))=min(4,1)=1, which cuts SMEM per block from 32KB to 8KB, raises occupancy from ~5 to ~20 blocks/SM, and eliminates 3 redundant compute iterations with __syncthreads barriers

## Key Lessons

- Compute pipeline stages dynamically as min(max_stages, ceil(kMaxK/kBlockK)) rather than using a fixed default — when the entire K-dimension fits in one tile, additional stages waste SMEM and add redundant sync barriers without any pipelining benefit
- Reducing SMEM per block from 32KB to 8KB can quadruple occupancy (~5→~20 blocks/SM on Sm80), which is often the dominant performance factor for memory-bound kernels even more than the saved instructions
- For small head dimensions (kMaxK≤kBlockK), the CUTLASS CustomMmaMultistage should set kSmemContainsEntireMat=true and skip prologue/mainloop load steps entirely, as the whole matrix resides in a single stage
