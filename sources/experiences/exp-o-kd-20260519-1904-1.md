## Solution

Replace single-pass 512-bin histogram with 4-step progressive refinement using 2048 bins per step (half→11→11→10 bit iterations); each step uses a templated extractBinIdx that extracts different bit ranges of the float representation, narrowing candidates progressively and stopping early when the threshold bin population fits within kNumFinalItems=2048 slots

## Key Lessons

- Use multi-step progressive refinement (half→11→11→10 bit iterations with 2048 bins per step) instead of single-pass coarse binning for top-k selection when logit values span wide ranges or cluster tightly — each step narrows candidates by extracting different bit ranges of the float representation, and early-exit avoids unnecessary work when the threshold bin is small enough
- Implement extractBinIdx as a step-templated function with if constexpr dispatch to extract different float bit ranges per refinement step — enables compile-time optimization and avoids runtime branching overhead per element
- Use the sign-aware complement trick ((bits & sign) ? bits : ~bits & ~sign) to map both positive and negative floats into a monotonically increasing integer ordering, enabling correct radix-based binning across the full float range
