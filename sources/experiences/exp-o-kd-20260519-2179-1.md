## Solution

Move both align_expert_first_token_offset allocation (torch::zeros_like) and getMIndices kernel launch inside the if (align_block_size.has_value()) guard, eliminating a cudaMalloc+cudaMemset and a GPU kernel launch when CUTLASS group gemm is used and block alignment is not required

## Key Lessons

- Guard GPU tensor allocations (torch::zeros_like/empty_like) and kernel launches behind the conditional that determines whether their results are consumed — eliminates cudaMalloc+cudaMemset+kernel launch overhead for code paths that never read the output
- When a function supports multiple backend paths (e.g., DeepGemm vs CUTLASS group gemm), move per-backend setup work into the corresponding branch rather than executing all setup unconditionally — the saved overhead accumulates across many forward passes in production inference
