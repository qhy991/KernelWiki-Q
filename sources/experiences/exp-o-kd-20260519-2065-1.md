## Solution

Templatize the kernel on InputType (float/__nv_bfloat16/__half) and fuse vectorized type conversion into the data loading loop using hardware intrinsics: __bfloat1622float2 and __half22float2 convert two packed values per operation, falling back to scalar __bfloat162float/__half2float when ELTS_PER_LDG==1

## Key Lessons

- Fuse type conversion into existing data-loading loops using vectorized intrinsics (__bfloat1622float2/__half22float2) rather than running a separate conversion kernel — eliminates one full global memory read+write pass and one kernel launch overhead
- Use if constexpr with std::is_same_v to templatize kernel input types without runtime branching — each instantiation compiles to a specialized path with zero overhead for the active type
- Handle ELTS_PER_LDG edge cases with compile-time branching (scalar vs vectorized path) to ensure correctness across all load granularities while maximizing vectorized conversion throughput when alignment permits
