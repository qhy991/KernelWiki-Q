## Solution

Switch to CausalMask<true> (IsQBegin=true) dispatch by removing the offset_q term so max_blocks_q = ceil_div((blk+1)*tile_q, tile_k), producing a correct causal triangular schedule where Q-block i visits exactly i+1 K-blocks

## Key Lessons

- CausalMask<false> (IsQBegin=false) adds an offset_q = N_K - N_Q term that inflates trip counts when Q does not start at the beginning of the sequence — use CausalMask<true> when Q is at the sequence start to get correct triangular scheduling and eliminate unnecessary tile visits
- Inattention causal dispatch, trip count computation directly determines work volume: removing even a single incorrect offset term can reduce total tile visits by orders of magnitude (128→10 trips here) and yield massive speedups without any kernel code changes beyond the dispatch logic
