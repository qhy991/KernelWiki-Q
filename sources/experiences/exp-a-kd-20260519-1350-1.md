## Solution

Switch to an M-appropriate small tile configuration (BM=32, BN=64, BK=64, TM=1, TN=2) that reduces padding waste to 50% for M=16, cuts shared memory from 16KB to 12KB per block for higher occupancy, and doubles BK to amortize each shared memory load over twice as many multiply-accumulate operations

## Key Lessons

- Use M-dimension-aware tile sizing for GEMM: when M is much smaller than BM, most tile rows are zero-padded wasted compute — reducing BM to match M order of magnitude cuts padding waste and shared memory footprint simultaneously
- Increasing BK for small-M GEMM problems improves the compute-to-memory ratio because each shared memory load is amortized over more multiply-accumulate iterations in the K-loop, partially compensating for the reduced parallelism from a smaller tile
- Reducing shared memory per block (16KB to 12KB) enables more concurrent blocks per SM, increasing occupancy — this is especially valuable for small-M problems where per-block work is already limited by padding waste
