## Challenge

Shared memory budget for a tiled GEMM sums all buffer sizes independently (block_meta, A tiles, B tiles, reduction, bias), requesting 41984 bytes per block and limiting SM occupancy to only 3 concurrent blocks on A800 (3×41984=125952 < 167912)


## Solution

Overlap B-loading and reduction buffers in shared memory (they are never live simultaneously), place bias in the unused portion of the larger of the two, and halve block_meta by sharing a single slot instead of two — reducing per-block shmem from 41984 to 32896 bytes and raising occupancy from 3 to 4 concurrent blocks per SM

## Key Lessons

- Overlap shared memory buffers that are never simultaneously live (e.g., B-loading and reduction phases in tiled GEMM) → saves 21.7% per-block shmem → raises SM occupancy when the saved bytes cross a threshold that allows one additional concurrent block per SM
- Compute shared memory budget as max(live_buffers) instead of sum(all_buffers) for phased kernels where buffers have disjoint lifetimes — applies to any kernel with sequential pipeline stages using scratch buffers
- Halve block_meta overhead by sharing a single metadata slot between phases instead of allocating separate slots, since metadata is consumed before the next phase begins
