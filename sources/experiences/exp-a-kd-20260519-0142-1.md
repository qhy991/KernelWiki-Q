## Challenge

GELU kernel computes erf(scalar / sqrt(2)) using double-precision division, which requires a MUFU.DIV special function unit instruction followed by Newton-Raphson refinement, costing multiple cycles with low throughput on the FP64 pipeline


## Solution

Replace the division scalar / sqrt(2) with a multiplication scalar * (1/sqrt(2)) by precomputing the reciprocal constant HALF_ROOT_TWO_D, converting a low-throughput MUFU.DIV + Newton-Raphson sequence into a single DFMA-class instruction issued at full FP64 pipeline throughput

## Key Lessons

- Precompute reciprocals of constants and replace runtime divisions with multiplications — double-precision division (MUFU.DIV + Newton-Raphson) has much lower throughput than a single DFMA multiply, yielding measurable latency reduction even when the surrounding erf() call dominates
- For mathematical identities like x / sqrt(2) = x * (1/sqrt(2)), precomputing the reciprocal constant at compile time eliminates an entire instruction class from the hot path with no loss of numerical accuracy
