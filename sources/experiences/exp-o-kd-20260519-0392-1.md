## Solution

Enable block swizzle via a 3D grid (BLOCK_SWIZZLE=true) that reorders thread blocks using bx=blockIdx.z*gridDim.x+blockIdx.x with interleaving stride=2048, so neighboring blocks sharing A-row data execute closer together and maximize L2 cache reuse

## Key Lessons

- Block swizzle (3D grid reordering via blockIdx.z*gridDim.x+blockIdx.x) dramatically improves L2 cache reuse for small HGEMM problems where the grid has few blocks along one dimension — interleaving blocks that share input data increases L2 hit rate and yields up to 3.8x speedup
- The L2 cache benefit from block swizzle is largest when the problem is small enough that interleaved scheduling can keep shared data (e.g., A-row tiles) resident in L2 across consecutive block launches
