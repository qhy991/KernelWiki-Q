## Solution

Replace the 128-thread shared-memory tree reduction with a 16-thread warp-shuffle GroupReduceMax using __shfl_xor_sync, which performs the reduction entirely in registers within a single warp, eliminating shared memory allocation, bank conflicts, and all __syncthreads barriers

## Key Lessons

- For reductions within a single warp (≤32 threads), prefer __shfl_xor_sync or __shfl_down_sync over shared-memory tree reduction — shuffle instructions operate entirely in registers, eliminating shared memory allocation, bank conflicts, and all __syncthreads barriers
- A 16-thread warp-shuffle reduction needs only 4 shuffle steps (xor with 8, 4, 2, 1) instead of 7 __syncthreads-barriered iterations for a 128-thread shared-memory tree — the reduced thread count works because vectorized loads let each thread cover 8 elements
