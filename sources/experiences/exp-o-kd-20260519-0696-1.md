## Challenge

The CTA-level RMS norm for hidden_size=256 uses 8 warps (256 threads) per block with shared memory reduction, two __syncthreads() barriers per token, and a serial warp-0 accumulation of 8 partial sums, causing high synchronization overhead and poor occupancy (~2 blocks/SM on A800)


## Solution

Replace the 8-warp CTA kernel with a single-warp (32 threads) kernel where each thread handles 8 elements, using pure __shfl_down_sync reduction and __shfl_sync broadcast for inv_rms — eliminating all shared memory, __syncthreads() barriers, and improving occupancy from ~2 to ~16 blocks per SM

## Key Lessons

- For small hidden sizes (≤256), a single warp with multiple elements per thread outperforms multi-warp CTA approaches — eliminates __syncthreads() barriers, shared memory traffic, and dramatically improves occupancy (2→16 blocks/SM on A800)
- __shfl_sync(0xffffffff, value, 0) broadcasts the warp-reduced scalar to all 32 lanes without shared memory or barriers — use for scalar broadcast when the full reduction fits in one warp
