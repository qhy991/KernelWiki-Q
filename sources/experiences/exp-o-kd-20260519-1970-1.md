## Challenge

Scalar byte-by-byte loads issue one 1-byte LDG instruction per element, generating 16x more load instructions than necessary and leaving instruction throughput as the bottleneck for a small 16 KB workload that fits in a single thread block


## Solution

Replace scalar byte loads and stores with uint4 (16-byte) vectorized loads via reinterpret_cast, so each thread processes 16 bytes per iteration — reducing total load instruction count by 16x and cutting store instructions proportionally

## Key Lessons

- Use uint4 (16-byte) vectorized loads instead of scalar byte loads for byte-level elementwise transforms — a single LDG.128 replaces 16 LDG.8 instructions, especially impactful when the kernel is instruction-limited at small data sizes
- reinterpret_cast<const uint4*>(buf)[vec_idx] is the standard CUDA pattern for vectorized loads: cast the base pointer, index by vector count, then reinterpret the register bytes for per-element processing
