## Solution

Change k_scale/v_scale from scalar-by-value kernel parameters to device pointers (const float*) and dereference them inside the kernel body, enabling the scales to reside in GPU global memory rather than being passed through the kernel launch parameter buffer

## Key Lessons

- Changing scalar kernel parameters to device-pointer dereferences for small per-kernel constants (e.g., quantization scales) yields negligible speedup when the kernel's main loop dominates execution time — the pointer read is a single L2-cached 4-byte load amortized across hundreds of iterations, making the parameter passing mechanism irrelevant to overall performance
- When a measured speedup is below ~10% on a microbenchmark with a compute-heavy inner loop, treat it as noise rather than a reliable optimization signal — parameter passing overhead is drowned out by the arithmetic and memory operations in the loop body
- Passing scales as device pointers (const float*) instead of by-value scalars is primarily a design choice for dynamic scale values that vary across calls, not a performance optimization — use whichever interface matches the caller's data layout
