## Solution

Load cos/sin data into per-warp shared memory via cp.async (16-byte cache-global transfers committed as async group 1) before the head processing loop, then read from smem inside the loop; all 4 heads processed by the same warp share the staged data, eliminating 3 out of 4 redundant global reads

## Key Lessons

- When multiple warps/threads read the same read-only data (cos/sin per position), stage it into shared memory once via cp.async and reuse — the async copy hides latency and eliminates redundant global reads across consumers
- On architectures where L2 cache already captures hot data (SM 8.0/A800), explicit shared memory staging of cos/sin yields marginal benefit because subsequent global reads hit L2 cache; the technique is more valuable on SM 9.0 (H100) where cp.async throughput is higher and occupancy trade-offs differ
