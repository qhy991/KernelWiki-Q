## Challenge

RoPE kernel uses a flat linear index across all tokens, causing threads in the same warp to read from interleaved positions spanning different token rows, which defeats memory coalescing and results in scattered global memory transactions


## Solution

Restructure the kernel to use one block per token position (blockIdx.x = token_pos, threadIdx.x = dimension index), so all threads in a warp access contiguous memory within the same token row, enabling fully coalesced global memory loads and stores

## Key Lessons

- Map thread hierarchy to data layout so that consecutive threadIdx values stride through contiguous memory — this enables memory coalescing where a single warp-wide memory transaction covers all 32 threads' reads instead of issuing scattered transactions
- Per-token block mapping (blockIdx.x = token position, threadIdx.x = dimension index) is preferable to flat indexing for per-token operations like RoPE because it naturally groups accesses within a token row and provides sufficient blocks for GPU occupancy
