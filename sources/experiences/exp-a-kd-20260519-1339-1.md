## Solution

Reinterpret input/residual/weight arrays as HalfVec8 (128-bit aligned struct of 8 __half) to process 8 elements per 128-bit vector load, using __half2 packed operators (__hadd2, __hmul2, __half22float2) for arithmetic and a sum_squares() method for variance — reducing loop iterations from 4 to 1 per phase and cutting memory transactions by 8x

## Key Lessons

- Reinterpret __half arrays as 128-bit aligned vector structs (e.g. HalfVec8 = 8 __half) to coalesce 8 scalar loads into one 128-bit transaction — reduces memory instruction count by 8x and is most impactful when kernel is memory-latency bound at small batch sizes
- Use __half2 packed operators (__hadd2, __hmul2, __half22float2) inside vector struct operators to halve the per-element-pair instruction count versus scalar __half arithmetic — the GPU executes these as single instructions on both lanes simultaneously
- Vectorization speedup is amplified at small batch sizes (32 tokens) where latency hiding via occupancy is less effective — the same optimization at larger batches (256 tokens) may show smaller gains because SM pipelining already hides much of the memory latency
