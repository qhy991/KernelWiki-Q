half2 broadcast intrinsic: __halves2half2 vs __half2half2

## Solution

Replace __halves2half2(x, x) with __half2half2(x) — the dedicated broadcast intrinsic that duplicates a single half into both lanes. This makes the broadcast intent explicit to the human reader. However, nvcc with -O3 recognizes identical arguments to __halves2half2 and emits the same PTX/SASS as __half2half2, so there is no measurable performance difference (0.997x, within noise).

## Key Lessons

- __half2half2 is the dedicated lane-duplicate intrinsic for broadcasting a single half into both lanes of half2. __halves2half2 is the two-argument combine. When the compiler can prove both arguments to __halves2half2 are identical, it optimizes to the same PTX — so this is a code-clarity change, not a performance one. Use __half2half2 for explicit broadcast intent and avoid depending on compiler optimization.
- When benchmarking shows a speedup of 0.997x (essentially 1.0), the change is within measurement noise. This is expected for intrinsic substitutions that compile to identical PTX/SASS. The value is in code maintainability and making the programmer's intent clear, not in runtime performance.
