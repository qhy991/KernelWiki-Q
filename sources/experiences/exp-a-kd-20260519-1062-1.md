## Solution

Replace scalar bf16 load/store with uint4 (Pack16B) vectorized load/store processing 16 bf16 elements per iteration, using two uint4 packs per topk loop iteration to accumulate into a float[16] array before writing the output chunk as a single uint4 store

## Key Lessons

- Scalar bf16 loads by 32 contiguous warp lanes already produce naturally coalesced 128-bit memory transactions — explicit uint4 vectorization provides diminishing returns when warp-level coalescing already saturates available bandwidth
- Higher topk values (e.g., topk=8) increase the inner-loop iteration count in vectorized kernels, raising accumulation and loop bookkeeping overhead relative to memory access savings, which can neutralize vectorization gains
- Smaller token counts reduce GPU occupancy, limiting the parallelism advantage of vectorized chunk-based processing — vectorization benefits are most pronounced with high occupancy and large hidden_dim
