## Challenge

The scalar kernel iterates all threads over the entire num_heads*head_size range with a single stride, computing per-element head_idx via division and head_offset via modulo, resulting in strided non-contiguous access patterns and redundant integer arithmetic per element


## Solution

Replace the flat scalar loop with warp-per-head decomposition: each warp (32 threads) owns one head and iterates over heads with warp_id stride, eliminating per-element division/modulo index arithmetic and allowing each warp to focus on a single contiguous memory region in the HND-layout cache

## Key Lessons

- Assign one warp per head in HND-layout KV-cache kernels instead of a flat thread stride over all elements — eliminates per-element division/modulo index computation and gives each warp a contiguous memory region to operate on
- Pre-compute base pointers (key_src, value_src, key_dst, value_dst) once outside the head loop to avoid redundant stride multiplication per element
