## Solution

Fuse all five elementwise kernels into a single silu_bw_fused_kernel that computes sigmoid, x3, dx3, dx2, dx1, and x4 per-element entirely in registers, eliminating the intermediate x3 and dx3 global memory buffers and reducing 5 kernel launches to 1

## Key Lessons

- Fuse sequential elementwise kernels that share intermediate tensors into a single kernel — keeping intermediates in registers eliminates global memory round-trips and can yield 2x+ speedup for bandwidth-bound pipelines
- When multiple kernels write/read the same intermediate buffer (e.g., silu_forward writes x3, then dx2 and x4 read x3), fusion removes the buffer entirely and replaces it with register-local scalars
- Kernel launch overhead scales linearly with the number of launches — fusing N kernels into 1 eliminates (N-1) launch overheads and N-1 global synchronization points, which is significant for small-to-moderate element counts
