## Challenge

MoE align kernel computes a serial prefix sum on a single thread (threadIdx.x==0) iterating over 64 experts, leaving 1023 of 1024 threads completely idle during the cumulative-sum phase while thread 0 performs ~64 sequential iterations with conditional indexing arithmetic into shared memory


## Solution

Replace the single-thread serial prefix sum loop with CUB BlockScan ExclusiveSum distributed across all 1024 threads: each thread reads its own expert count from shared memory, applies the CEILDIV padding, then BlockScan computes the parallel exclusive prefix sum in ~log2(1024)=10 tree-reduce steps, writing cumulative sums directly to global memory

## Key Lessons

- Replace single-thread serial prefix sum loops with CUB BlockScan ExclusiveSum to parallelize the scan across all block threads — even for moderate element counts (64), the parallel scan tree (~log2(N) depth) outperforms a serial loop by keeping all threads active instead of idling all-but-one
- CUB BlockScan handles boundary conditions cleanly: threads whose expert_id >= num_experts contribute zero to the scan, and the exclusive sum naturally produces the correct cumulative-sum array without a separate initialization pass
- The speedup from serial-to-parallel scan grows with expert count but remains meaningful even at 64 experts because the primary gain is eliminating the single-thread bottleneck where 1023 threads sit idle during the prefix-sum phase
