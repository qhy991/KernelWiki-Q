## Challenge

IPC buffer for custom allreduce is allocated with plain cudaMalloc (default cached allocation), which may cause coherence overhead on multi-GPU peer access but is unoptimized for cross-device access patterns


## Solution

Add cudaMemAdviseSetAccessedBy hint after cudaMalloc to advise the driver that the buffer will be accessed by a peer device, enabling the driver to use uncached or coherent mapping where applicable

## Key Lessons

- cudaMemAdviseSetAccessedBy is a no-op on single-GPU NVIDIA environments where no peer device exists — the driver ignores the advice when there is no remote device to optimize for, producing zero performance change
- cudaMemAdvise hints are meaningful only in multi-GPU IPC scenarios (e.g., custom allreduce across GPUs) where cached allocation causes coherence traffic; on local-only access patterns the default cudaMalloc is already optimal
- The hipDeviceMallocUncached optimization that motivated this change is AMD-specific and does not translate to performance gains on NVIDIA hardware where cudaMalloc already provides coherent access by default
