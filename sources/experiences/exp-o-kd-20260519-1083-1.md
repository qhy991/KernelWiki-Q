## Challenge

Blelloch block-level exclusive prefix scan for 64 MoE experts uses up-sweep and down-sweep phases with log2(scan_size) __syncthreads() barriers each, totaling 12 barrier calls for scan_size=64, all mediated through shared-memory round-trips at every step


## Solution

Replace Blelloch scan with a two-level warp-shuffle hierarchical exclusive scan using __shfl_up_sync: intra-warp prefix sums complete entirely in registers without shared memory or barriers, then warp 0 aggregates per-warp partial sums via a second shuffle scan, reducing total __syncthreads() from 12 to ~5 and eliminating all shared-memory round-trips within each warp

## Key Lessons

- Replace shared-memory Blelloch scan with __shfl_up_sync warp shuffle for exclusive prefix sums — intra-warp scan completes entirely in registers with zero __syncthreads() barriers, reducing synchronization from O(log N) per phase to zero within each warp
- For block-level scans spanning multiple warps, use a two-level hierarchy: compute per-warp prefix sums via shuffle, store each warp total to shared memory, then have warp 0 compute an exclusive scan of those totals — cross-warp __syncthreads() drops from 2*log2(N) to O(num_warps)
