## Challenge

With 8 warps computing partial sums across 32 rows, multiple warps concurrently call atomicAdd on the same s_prime entries, causing quadratic contention overhead that grows with warp count


## Solution

Replace atomicAdd cross-warp reduction with a per-warp addition_storage array (num_rows * num_warps slots) where each warp writes its partial sum to its own slot, followed by a deterministic shared-memory reduction loop that sums all warp contributions without atomics

## Key Lessons

- Replace atomicAdd cross-warp aggregation with per-warp slots in shared memory plus a deterministic reduction loop — atomic contention grows quadratically with concurrent warps while the reduction loop cost grows linearly, so the benefit increases with warp count
- Partition shared memory into per-warp slices (addition_storage[row + num_rows * warp_id]) so each warp writes to a conflict-free region, then have designated lanes perform the final cross-warp sum — applies whenever multiple warps contribute partial results to the same output elements
