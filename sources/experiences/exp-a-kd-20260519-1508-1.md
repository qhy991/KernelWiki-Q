## Solution

Replace the custom blockReduceSum with cub::BlockReduce<float,1024>.Reduce(val, cub::Sum{}, blockDim.x), which provides a unified API with runtime-valid-items support, eliminating the conditional branch and the function-pointer dispatch mechanism entirely

## Key Lessons

- Replacing a custom block reduce with cub::BlockReduce does not change performance when both execute the same algorithm (warp shuffle → shared memory barrier → final warp reduce) — the dominant cost is the shuffle network and __syncthreads barrier, which are identical in both implementations
- cub::BlockReduce with runtime-valid-items parameter (blockDim.x) eliminates the need for conditional dispatch over block-size template parameters, improving code maintainability without performance regression
- Function-pointer overhead for trivial reduction operations (FADD) is negligible at -O3 optimization — nvcc generates equivalent PTX for both custom and cub paths, so the choice is purely a software engineering concern
