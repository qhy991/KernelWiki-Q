## Solution

Precompute per-dimension validity as bitmasks at host construction time (one bit per filter position), then replace the per-access coordinate recomputation and bounds comparison chain with a single bitwise AND + shift per dimension in the kernel inner loop

## Key Lessons

- Precomputed bitmask validity replaces per-access coordinate recomputation and multi-clause bounds checks with a single bit-test per dimension — but for small filter sizes (T=R=S=3, 27 total positions), the compiler can optimize the naive coordinate arithmetic effectively, making the speedup negligible
- Bitmask precomputation trades host-side construction overhead and kernel register pressure (additional mask registers) for reduced inner-loop instruction count — this tradeoff favors larger filter sizes where the loop body dominates kernel runtime
- Precompute-then-query patterns must be evaluated against the actual instruction mix and loop trip count: when the original computation is simple integer arithmetic that the compiler can keep in registers, the bitmask indirection may not reduce the critical path
