## Solution

Add sm_occupancy > 1 guard to the conditional branch so the SK wave formation path (full_waves % sm_occupancy == sm_occupancy - 1) is only entered when the SM actually supports multiple concurrent thread blocks, preventing the degenerate sm_occupancy==1 case from taking the wrong code path

## Key Lessons

- Guard modulo-based occupancy branch conditions with an explicit sm_occupancy > 1 check to prevent the degenerate case where N % 1 == 0 always evaluates to true, causing incorrect dispatch into multi-occupancy code paths when only single-occupancy is possible
- When designing scheduler logic that branches on occupancy properties, validate correctness across both sm_occupancy==1 (single-occupancy) and sm_occupancy>1 (multi-occupancy) scenarios, as trivially-true conditions in the degenerate case can mask bugs that only manifest under specific workload configurations
