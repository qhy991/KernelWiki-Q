## Solution

Replace the unconditional return false in the SM89 branch with a CUDA version check (cuda_version >= 12040), enabling FP8 CUTLASS kernel dispatch on Ada Lovelace GPUs with CUDA 12.4+ where hardware support is available

## Key Lessons

- Dispatch gate functions that unconditionally disable hardware paths should be audited when new CUDA versions add support for previously-excluded architectures — the SM89 FP8 path was disabled due to un tuned CUTLASS kernels but CUDA 12.4 brought adequate support, making the unconditional false obsolete
- Microbenchmarking a host-side dispatch gate function yields misleading speedup numbers (the measured 1.41x is noise at sub-nanosecond scale); the real performance impact is downstream when the gate enables hardware-accelerated kernel paths that replace software fallbacks
