## Solution

Add #pragma unroll 1 to both qo_heads and kv_heads iteration loops to explicitly prevent the compiler from unrolling them, reducing register pressure and code size for large head counts

## Key Lessons

- #pragma unroll 1 on loops with runtime-variable bounds is redundant when NVCC compiles with -O3 — the compiler does not aggressively unroll loops whose trip counts are kernel arguments rather than compile-time constants, so both variants generate identical PTX loop code
- #pragma unroll directives become impactful only when loop bounds are compile-time constants (e.g., resolved via C++ templates or macros like DISPATCH_HEAD_DIM) — in that case the compiler may choose to unroll, and #pragma unroll 1 can meaningfully prevent code bloat and register pressure
