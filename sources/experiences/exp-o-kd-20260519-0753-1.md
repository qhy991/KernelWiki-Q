## Challenge

KV-cache transfer kernel unconditionally copies both K-cache and V-cache for every layer, but in MLA (Multi-head Latent Attention) mode the V-cache is not needed, wasting approximately half of global memory bandwidth on unnecessary transfers


## Solution

Introduce a compile-time constant kIsMLA and wrap the V-cache transfer code in if constexpr (!kIsMLA), causing the compiler to completely eliminate all V-cache load/store instructions and pointer indirections when MLA mode is active, halving global memory traffic

## Key Lessons

- Use if constexpr with compile-time flags to eliminate entire code paths in CUDA kernels — the compiler removes dead branches completely, reducing both instruction count and memory traffic with zero runtime overhead
- In memory-bandwidth-bound copy/transfer kernels, eliminating unnecessary data movement (e.g., skipping V-cache in MLA mode) yields near-linear speedup proportional to the reduction in bytes transferred from global memory
