## Solution

Clamp kOffcp to min__(K - A_CHUNK_VAL, k_str + kOff) so that OOB threads read the last valid 8-element chunk (positions 92-99) within the same row as deterministic padding instead of spilling into the next row, fixing correctness to 64/64 match with zero error

## Key Lessons

- When chunk-size (A_CHUNK) does not evenly divide the K dimension, the computed per-thread offset kOff = tid * A_CHUNK can exceed K - A_CHUNK; always clamp to min(K - A_CHUNK, offset) to prevent row-stride address arithmetic from wrapping into the next matrix row — applies to any chunked-load pattern over a strided 2D array
- A single missing clamping guard on an offset variable can cause silent data corruption (reading from wrong rows) rather than a visible crash — extra over-provisioned allocation (K_ALLOC > K) masks the OOB as wrong-but-valid reads, making the bug hard to detect without a per-element reference check
- Latency differences at sub-10us kernel scale are dominated by measurement noise; a correctness fix that adds one min comparison per thread is inconsequential relative to kernel launch overhead
