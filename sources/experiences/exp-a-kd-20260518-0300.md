## Challenge

At M=N=K=512, the grid expands to 4x4 blocks, creating significantly more concurrent warp-level smem traffic. The before code uses B_PAD=8 with linear indexing (load_smem_b_n*(BK+8)+load_smem_b_k), which means consecutive warp rows in B smem always hit predictable banks during LDMATRIX_X4 loads, ...


## Solution

The after code replaces this with swizzle_permuted_B_j which applies XOR permutation (((j>>3)^(i>>2))%2)<<3 to scatter 8-byte chunks across different banks for different row groups. - This eliminates the structural bank conflicts while also keeping the smem layout compact at BK=16 columns per row instead of BK+8=24, freeing 50% of B smem space per tile for potentially higher K_STAGE counts.
