## Solution

Accumulate in float32 via native FFMA instructions using unrolled vector multiply-add (sum_vec[k] += factor * val_vec[k]), converting bf16 inputs to float at load time via cast_load and storing back to bf16 only at the final output

## Key Lessons

- Accumulate reduction sums in float32 instead of bf16 to use native FFMA instructions and avoid precision loss from repeated bf16 rounding — applies to any multi-step reduction chain over bf16 data
- Use #pragma unroll on the inner vector loop to ensure the compiler generates pipelined FFMA without loop overhead
