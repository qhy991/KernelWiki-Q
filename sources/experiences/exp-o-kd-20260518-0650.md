## Challenge

Optimization focus: Replaced __shfl_sync-based warp index broadcast with canonical_warp_idx() which computes the warp index via direct arithmetic (threadIdx.x / NumThreadsPerWarp) without a warp shuffle.
Problem before: The original code used __shfl_sync(0xffffffff, threadIdx.x / 32, 0) to compute t


## Solution

uction. The __shfl_sync acts as a warp-level barrier and consumes extra registers and instruction slots without providing any correctness benefit, since no divergence can exist for this value across lanes in the same warp.
Applied change: Replaced the __shfl_sync broadcast with a call to canonical_w

## Key Lessons

- The __shfl_sync acts as a warp-level barrier and consumes extra registers and instruction slots without providing any correctness benefit, since no divergence can exist for this value across lanes in the same warp.
- In GEMM kernels where this function is called once per threadblock, the absolute cycle savings are small but the pattern removes unnecessary synchronization that could interfere with warp scheduling.
