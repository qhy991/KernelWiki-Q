## Solution

Replace scalar int32_t stores with 4-wide AlignedArray<int32_t,4> vectorized stores via reinterpret_cast, reducing per-thread loop iterations by 4x and issuing fewer store instructions for the same total data volume

## Key Lessons

- Vectorized stores (AlignedArray<T,N>) reduce loop iteration count and instruction overhead for fill/copy kernels → effective when memory bandwidth dominates, but benefits are marginal at small array sizes where kernel launch overhead (~5µs) swamps the actual compute time
- CUDA hardware coalesces scalar 32-bit writes from consecutive threads into full-sector transactions efficiently → for simple constant-fill patterns the per-transaction throughput gap between scalar and vectorized stores is small, making vectorization most beneficial at >=1MB data sizes where bandwidth-bound behavior fully materializes
