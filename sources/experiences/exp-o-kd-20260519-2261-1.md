## Solution

Change k_scale and v_scale kernel parameters from scalar float (pass-by-value) to const float* (pass-by-pointer), enabling dynamic per-head or per-token scaling factor lookup from a device tensor rather than a single constant

## Key Lessons

- Pass KV cache scaling factors as pointers (const float*) instead of scalars to paged attention kernels — enables per-head or per-token dynamic scaling for FP8 dequantization without kernel specialization, applicable whenever quantization scales vary across attention heads or layers
- Pointer-based kernel parameters allow future extension to indexed access (e.g., k_scale[head_idx]) without changing the kernel signature — prefer pointer over scalar when the parameter may need to vary across work items
