## Challenge

Integer division by a non-power-of-2 constant (top_k=3) is performed inside a hot loop repeated 200 times per thread, executing ~800 expensive integer-division ALU operations per thread that each require multiple instruction cycles since the GPU cannot use shift shortcuts for non-power-of-2 divisors


## Solution

Pre-compute sh_ids[i] / top_k into a second shared memory array (sh_precomputed) during the one-time init phase with an extra __syncthreads() barrier, then replace the expensive division in the hot loop with a simple shared memory read of the cached result

## Key Lessons

- Pre-compute loop-invariant expressions (especially integer division by non-power-of-2 constants) into shared memory during a one-time init phase rather than recomputing them on every hot-loop iteration — the cost of one extra __syncthreads() barrier is negligible compared to eliminating hundreds of multi-cycle division operations
- When a division operand is constant across all threads and iterations but the dividend varies, hoist the division out of the repetition loop and cache results in shared memory — applies whenever compute-to-memory-access ratio is high and the computation repeats across loop iterations
