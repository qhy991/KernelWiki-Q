## Challenge

Inner GEMM loop uses conditional break statements (if(k_ >= K) break) to guard against out-of-bounds K-dimension access when K is not a multiple of the thread tile size, introducing divergent branch paths in the final loop iterations


## Solution

Replace conditional break statements with clamped index access using a min__() helper and padded K dimensions: out-of-range indices are redirected to the last valid chunk via min__(k_, K - A_CHUNK) and min__(ki, K - 1), and matrices are padded to tile boundaries (Kbp = ceil(K/256)*256) so redundant reads hit zeroed padding that does not affect the accumulation

## Key Lessons

- Replacing conditional breaks with clamped/padded index access is a branch-elimination technique that benefits AMD ROCm MFMA wavefront execution more than NVIDIA CUDA warp execution, where the branch predictor effectively handles predictable last-iteration exits
- Padded K-dimension (Kbp rounded up to tile boundary) with zeroed padding ensures out-of-range reads produce zero contributions to the accumulation, maintaining correctness without branching
- Clamped index patterns (min__(idx, limit)) remove divergent branches at the cost of redundant memory reads and extra min instructions — the tradeoff is favorable only when branch divergence cost exceeds the redundant compute cost
