## Challenge

MoE fused gate rescale uses two separate kernel launches for normalization and scaling, causing redundant global memory traffic (normalize writes all 32768 elements, scale kernel re-reads them all) plus double kernel launch dispatch overhead across 4096 blocks


## Solution

Fuse normalization and conditional routed_scaling_factor multiplication into a single kernel that reads each element once, applies division and optional scaling in the same loop iteration, and writes the final result once — eliminating one full kernel launch and all redundant global memory round-trips

## Key Lessons

- Fuse sequential elementwise kernels that operate on the same data buffer into a single kernel to eliminate redundant global memory traffic and kernel launch overhead — applies whenever kernels form a producer-consumer chain reading and writing the same array
- Use a boolean flag parameter in fused kernels to conditionally skip operations, preserving caller flexibility while still benefiting from single-launch efficiency
