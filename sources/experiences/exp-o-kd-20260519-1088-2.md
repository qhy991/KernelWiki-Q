## Solution

Replace the single-thread serial rescale loop with a warp-stride batched write (for i = tid_st_x; i < TOPK; i += WARP_SIZE) that reads from shared memory and fuses the normalization into each store via smem_score[i] * fast_rcpf(output_sum), distributing TOPK writes across all warp threads and eliminating the separate read-modify-write pass

## Key Lessons

- Fuse the normalization (divide by sum) into the final output write by multiplying each value by the reciprocal in the same store operation — eliminates a separate read-modify-write pass over global memory
- Use a warp-stride loop (for i = tid; i < TOPK; i += WARP_SIZE) to distribute TOPK output stores across all warp threads — prevents the serial bottleneck of a single-thread rescale loop when TOPK is small relative to warp size
- Replace full-precision division with fast_rcpf (__frcp_rn + one Newton iteration) for output normalization — trades negligible precision loss for lower-latency computation when the result only needs approximate normalization
