## Solution

Pass input_stride as a kernel parameter and read directly from strided memory using blockIdx.x * input_stride + idx addressing, eliminating the copy kernel and intermediate buffer entirely

## Key Lessons

- Parameterize stride instead of hardcoding it — passing input_stride as a kernel argument lets the same kernel handle both contiguous and strided (e.g. packed QKV) layouts without a copy, saving one full global-memory round-trip per invocation
- Eliminating intermediate copy buffers reduces memory allocation pressure and avoids redundant global-memory traffic proportional to num_tokens × hidden_size × sizeof(element)
- Kernel fusion via stride parameterization (replacing a copy kernel + compute kernel with a single stride-aware compute kernel) avoids one kernel launch overhead in addition to the memory savings
