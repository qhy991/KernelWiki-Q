## Solution

Replace the two-kernel copy+norm pipeline with a single rms_norm_kernel that accepts input_stride as a parameter and reads directly from strided input using blockIdx.x * input_stride + idx addressing, eliminating the intermediate buffer and the copy kernel entirely

## Key Lessons

- Parameterizing a kernel with input_stride eliminates the need for a separate copy-to-contiguous kernel, removing redundant global memory traffic and kernel launch overhead — applies whenever input layout differs from computation stride (e.g. packed QKV where hidden_size < row stride)
- Kernel fusion via stride parameterization is more effective than copy-then-compute for memory-bound normalization kernels: removing 128KB of intermediate buffer round-trip plus one kernel launch yielded 1.4x speedup in this rmsnorm case
