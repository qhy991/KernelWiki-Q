## Solution

Fuse both expert_offsets and 128-aligned blockscale_offsets prefix-sum computations into a single GPU kernel that reads problem_sizes1 from device memory and writes both output arrays in-place, eliminating all host-device synchronization (cudaDeviceSynchronize) and data transfer (cudaMemcpy D2H + H2D) overhead

## Key Lessons

- Fusing host-side computation into a GPU kernel eliminates cudaDeviceSynchronize stalls and cudaMemcpy round-trips — applies whenever the CPU computation depends only on data already on the GPU and the arithmetic is lightweight enough for a single GPU thread
- When benchmark timing is dominated by host-device sync overhead rather than actual computation (2.4ms vs 0.02ms kernel time), kernel fusion that removes the sync barrier yields orders-of-magnitude speedups even though the fused computation itself is trivial
- A serial prefix-sum over a few hundred entries (e.g. 256 experts) is fast enough to run on a single GPU thread without needing parallel scan algorithms — the real win is avoiding the host-device boundary, not accelerating the arithmetic
