## Challenge

Scalar half load/store processes one 2-byte element per thread, underutilizing the GPU's 32-byte memory transaction granularity and wasting bandwidth since two separate 2-byte transactions cannot coalesce efficiently


## Solution

Vectorize memory access using half2 (via HALF2 macro reinterpret_cast) to load and store two half elements per thread in a single 4-byte coalesced transaction, doubling per-thread memory throughput and improving coalescing efficiency

## Key Lessons

- Vectorize half memory access with half2 loads/stores (via reinterpret_cast<half2*>) when processing elementwise kernels on fp16 data — two 2-byte halves pack into one 4-byte coalesced transaction, improving memory bandwidth utilization on GPUs with 32-byte transaction granularity
- Halve the thread count (e.g. 256→128) when switching from scalar to half2 vectorized access so total element coverage remains the same while each thread processes 2 elements, maintaining grid-level work不变
