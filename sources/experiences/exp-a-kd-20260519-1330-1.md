## Solution

Replace the per-byte FP32 conversion pipeline with a bit-manipulation technique: extract exponent+mantissa fields from two interleaved FP8 values simultaneously using a precomputed integer MASK (0x7F007F00), widen the 4-bit FP8 exponent to 5-bit FP16 via a single integer shift (RIGHT_SHIFT=1), and correct the exponent bias difference with one __hmul2 half2 multiply by a precomputed bias register — eliminating all std::pow calls, per-element branching, and FP32 intermediate arithmetic

## Key Lessons

- For FP-format conversion (e.g., FP8→FP16), bit-level manipulation (integer AND+shift to widen exponent field, then half2 multiply for bias correction) eliminates FP32 intermediate arithmetic, std::pow calls, and per-element branching — applies whenever source and target share IEEE 754 structure with known exponent/mantissa widths and a simple bias offset
- Processing multiple packed values simultaneously via integer SIMD masking (one MASK to extract two interleaved FP8 exponent+mantissa fields from a 32-bit int) halves instruction count compared to per-byte extraction — applies to any packed data layout where fields can be extracted with a shared bit mask
- Use __hmul2 on half2 reinterpretations of integer registers to keep the entire dequant path in the FP16 ALU pipeline rather than the FP32 pipeline, reducing contention with concurrent FP32 MMA operations in mixed-precision kernels
