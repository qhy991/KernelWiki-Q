## Solution

Replace scalar grid-stride loop with warp-cooperative parallelism: each warp owns one (token, head_chunk) pair and uses vectorized int2 (128-bit) loads for k_nope and int (32-bit) loads for k_rope via register buffers, enabling fully coalesced 256-byte and 128-byte memory transactions while eliminating all per-element division, modulo, and branching overhead

## Key Lessons

- Replace grid-stride loops with warp-cooperative indexing (flat_warp_id = linearized warp index, lane_id = thread-in-warp) for structured memory operations — eliminates expensive integer division/modulo per element and enables each warp to own a natural data tile
- Use vectorized int2/int loads and stores via reinterpret_cast instead of scalar bf16 element access when the data layout permits — each lane's int2 load fetches 8 consecutive bf16 values in a single fully coalesced 128-byte transaction
- Buffer repeated reads of shared data (k_rope is identical across all heads for the same token) into a single register variable per warp, then reuse across the unrolled head loop — eliminates redundant global memory transactions
