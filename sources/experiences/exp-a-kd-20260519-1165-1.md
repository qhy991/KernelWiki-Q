## Challenge

KV-cache block copy uses CPU-side nested loops issuing 2*num_layers*num_pairs individual cudaMemcpyAsync calls (64 calls for 4 layers × 8 pairs), where cumulative per-call dispatch overhead dominates total execution time at 0.162ms


## Solution

Replace 64 individual cudaMemcpyAsync dispatches with a single CUDA kernel launch using a 2D grid (num_layers × num_pairs), where each thread block handles one layer-pair combination, processing all copies on-GPU with constant launch overhead regardless of grid size

## Key Lessons

- Replacing multiple cudaMemcpyAsync calls with a single kernel launch eliminates O(num_layers × num_pairs) CPU-side dispatch overhead — the kernel launch cost is constant regardless of grid size, so scaling to more layers/pairs increases GPU parallelism without increasing dispatch cost
- For batch memory copies between device buffers, a CUDA kernel with a 2D grid (layer × pair) is far more efficient than CPU-loop cudaMemcpyAsync because GPU threads execute copies in parallel while cudaMemcpyAsync serializes dispatch through the CPU driver
- Passing per-layer device pointers via an int64_t pointer array (key_cache_ptrs[layer_idx]) enables a single kernel to operate on multiple independent caches without separate launches per layer
