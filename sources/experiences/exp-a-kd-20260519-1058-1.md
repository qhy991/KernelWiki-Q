## Challenge

Each thread loads VPT=8 float elements one-at-a-time via scalar loop from an oversized MAX_VPT=32 AccessType struct, then writes back element-by-element, producing 24 separate 4-byte memory transactions per thread instead of 3 wider aligned operations


## Solution

Replace scalar load/store loops with single pointer-cast AlignedArray loads and stores using a properly sized Vlen=8 type (ArrayVal = AlignedArray_<float, 8>), enabling the compiler to emit LDG.128/STG.128 instructions and reducing per-thread memory transactions from 24 to 3

## Key Lessons

- Use pointer-cast AlignedArray loads (e.g., *(ArrayVal*)ptr) instead of scalar element-by-element loops when loading contiguous aligned data — enables the compiler to emit wide vector load/store instructions and reduces memory transaction count
- Match the AccessType array width to the actual per-thread data footprint (Vlen=8 not MAX_VPT=32) — oversized type declarations prevent the compiler from emitting a single wide load because the type alignment exceeds the working set
- A single pointer-cast vectorized store (*(ArrayVal*)ptr = chunk) replaces an unrolled scalar store loop, cutting output transactions by the vector width factor
