## Challenge

KV-cache reshape uses a packed layout [num_blocks, num_heads, head_size/x, block_size, x] where value elements are stored with block_size=16 stride, causing uncoalesced memory writes that cannot combine into 128-byte transactions and requiring complex multi-level index arithmetic


## Solution

Replace packed layout with NHD (Natural Head Dimension) layout [num_blocks, block_size, num_heads, head_size] where all head elements for a cache slot are stored contiguously, and use strided thread iteration over the flat n=num_heads*head_size range to enable fully coalesced writes where consecutive threads write to consecutive addresses

## Key Lessons

- NHD layout [blocks, block_size, heads, head_dim] stores all head elements contiguously per cache slot, enabling fully coalesced writes when consecutive threads access consecutive head elements — applies whenever warp threads need to write a full head's worth of cache data
- Packed layout value writes with block_size stride (value_dst[i * block_size]) fragment memory transactions and prevent coalescing — avoid cache layouts where the innermost write dimension has a stride greater than the element size
