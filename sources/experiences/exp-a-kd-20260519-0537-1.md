## Solution

Enable WARP_SWIZZLE template parameter to alternate j iteration direction for odd warp rows via j_s=((i%2)&&WARP_SWIZZLE)?(WARP_TILE_N-j-1):j, reversing ldmatrix access order to potentially improve shared memory bank conflict behavior

## Key Lessons

- Warp iteration order swizzle (alternating ldmatrix column direction for odd/even warp rows) shows no measurable benefit at small GEMM sizes (M=N=K=256 with 2x2 blocks) — the warp scheduler handles predictable ldmatrix patterns regardless of direction at this scale
- Conditional branch overhead from swizzle indexing (i%2 check and ternary select) can introduce slight regression (~1.7%) that falls within noise at small problem sizes — measure before committing swizzle for production kernels
