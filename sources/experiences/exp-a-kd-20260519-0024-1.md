## Solution

Fuse the two-kernel LSE reduction into a single kernel using atomicMax for global max computation, atomicAdd for exponential sum accumulation, and per-batch spin-wait counters (phase1_counter, phase2_counter) to synchronize split_kv CTAs within each batch, eliminating the intermediate lse_acc buffer

## Key Lessons

- Atomic spin-wait barriers between CTAs (atomicAdd counter + while-loop poll) are far more expensive than two sequential kernel launches at small scale — global memory atomics and busy-wait polling consume both bandwidth and compute while blocking SM resources that could otherwise be used
- Fusing kernels that eliminate intermediate buffers and reduce workspace can be beneficial in a full pipeline (e.g., FMHA where LSE reduction overlaps with epilogue), but at the isolated micro-kernel level the synchronization overhead dominates; benchmark fusion in context, not in isolation
- Per-iteration cudaMemset to reset atomic exchange buffers and phase counters adds measurable overhead (~10us) that accumulates across iterations — consider reusing buffers with in-kernel reset when evaluating fused approaches
