## Challenge

The before code executes the full blockReduceSum path even for a 16-thread block: it allocates shared[32], runs a full-warp warpReduceSum, writes to shared memory, calls __syncthreads, reads from shared memory, then runs another full-warp warpReduceSum


## Solution

The after code detects at compile time (via if constexpr maxBlockSize<=WARP_SIZE) that the block fits in one warp and replaces all of that with a single warpReduceSum<T,16> using only 4 shuffle rounds. This eliminates one full warpReduceSum, the shared-memory allocation/write/read, and the __sync...

## Results

- **speedup**: 1.056x (improved)


## Key Lessons

- This eliminates one full warpReduceSum, the shared-memory allocation/write/read, and the __syncthreads barrier entirely.
