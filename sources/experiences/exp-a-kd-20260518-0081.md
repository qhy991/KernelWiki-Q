## Solution

Replace C++ if-branch with explicit PTX @p st.global.v4.u32 predicated store via inline asm (mirroring cutlass::arch::global_store), which guarantees hardware-level predication at the PTX level rather than relying on the compiler to emit it

## Key Lessons

- NVCC at -O3 already translates simple if-guarded scalar/vector global stores into hardware-predicated PTX stores, so explicit inline PTX predication yields no performance gain for straightforward guard patterns — verify SASS equivalence before investing in hand-written predication
- The bottleneck in this workload is global memory bandwidth (256K threads x 64B each), not instruction scheduling, meaning instruction-level optimizations like predication vs branching have negligible effect when the kernel is memory-bound
- Explicit PTX predication (via cutlass::arch::global_store or inline asm) is valuable as a correctness guarantee and portability measure across compiler versions, even when the current compiler already emits equivalent code
