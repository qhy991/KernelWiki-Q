## Challenge

MoE align+sort is implemented as a single CTA (1 block, 1024 threads) where all 32 warps contend on 256 shared-memory local_offsets slots via atomicAdd, and the sort phase is bottlenecked to a single SM


## Solution

Split the monolithic single-block kernel into two kernels: (1) an align-only kernel that computes cumsum and expert_ids without the sort phase, and (2) a multi-block sort kernel (64 blocks x 256 threads) that uses grid-stride loops with global-memory atomicAdd on cumsum_buffer, distributing token scattering across multiple SMs to reduce per-SM atomic contention

## Key Lessons

- Splitting a single-CTA atomicAdd scatter into a multi-block kernel with global-memory atomics distributes contention across SMs — when the scatter target has many independent slots (e.g., 256 experts), multi-SM parallelism outweighs the slower global atomics
- Removing the sort phase from the align kernel shortens single-block execution time, allowing the align kernel to complete faster and the sort kernel to immediately utilize all available SMs
- Grid-stride loops in the sort kernel ensure uniform work distribution regardless of token count vs. block count, preventing tail-end load imbalance
