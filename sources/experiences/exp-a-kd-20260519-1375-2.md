## Challenge

Regular global-memory loads in the inner accumulation loop compete for the same L1 data cache lines as other memory operations, causing cache contention when each thread reads 2 input locations separated by an 8KB stride (HIDDEN_SIZE=2048 floats)


## Solution

Use __ldg() intrinsic to route read-only input loads through the GPU's dedicated read-only texture cache (separate from the L1 data cache), eliminating cache contention between the input accumulation loads and other data-path memory operations

## Key Lessons

- Use __ldg() for read-only global memory accesses to route loads through the dedicated L1 read-only/texture cache instead of the L1 data cache — reduces cache contention especially when threads access non-contiguous or strided addresses
- Read-only cache (via __ldg) is particularly beneficial when threads access memory locations with large strides that would otherwise thrash the L1 data cache used by both loads and stores in the same kernel
