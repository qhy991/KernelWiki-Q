## Solution

Introduce a DSv4-specific transfer path via IsDsv4Layout template flag that calls device::hisparse::transfer_item<HostToDevice> from kvcacheio.cuh, which encapsulates the exact page-padded layout constants (kGPUPageSize=64, kCPUItemBytes=584) and performs warp-cooperative value+scale transfer in a single pass, eliminating the separate V-copy for MLA mode

## Key Lessons

- When device and host memory layouts differ (e.g., page-padded vs linear), use dedicated transfer functions that encode layout constants rather than generic byte-stride arithmetic — prevents silent data corruption from incorrect address computation
- MLA (Multi-head Latent Attention) combines K and V into a single compressed representation, so KV cache transfer paths should skip the separate V-copy when in MLA mode — avoids redundant work and simplifies the transfer logic
- Encapsulate layout-specific constants (page sizes, item byte counts, alignment) in dedicated transfer functions rather than scattering stride arithmetic at call sites — makes the code robust against layout changes and eliminates address-calculation bugs
