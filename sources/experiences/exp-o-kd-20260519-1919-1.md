## Solution

Replace the scalar element-by-element load loop with vectorize_read_with_alignment_impl<__half, 8> that loads HalfVec8 (16 bytes = 8 __half values) per iteration, reducing total global memory transactions from 8192 scalar loads to 1024 16-byte vector loads and improving memory throughput on the wide load path

## Key Lessons

- Vectorized loads (e.g., HalfVec8 = 16 bytes) reduce global memory transaction count by VEC_SIZE compared to scalar loads — applies when hidden_size is a multiple of the vector width and the input pointer is aligned to the vector width boundary
- Separate the memory access pattern from the compute logic via lambda callbacks (vec_op / scalar_op) to keep the vectorized-read utility generic and reusable across different reduction-style kernels
- At larger hidden sizes (8192+) the variance computation loop dominates kernel runtime, making bandwidth savings from wider transactions measurable; at very small hidden sizes the fixed reduction overhead may mask the gain
