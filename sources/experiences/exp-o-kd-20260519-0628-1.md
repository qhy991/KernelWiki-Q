## Solution

Fuse all 5 metadata copy operations into a single kernel that uses a __grid_constant__ parameter struct to pass all source/destination pointers and dimensions, then sequentially processes each field with grid-stride loops annotated with #pragma unroll, eliminating 4 out of 5 kernel launches and improving SM utilization

## Key Lessons

- Fuse multiple small copy kernels that operate on the same logical batch into a single kernel using a __grid_constant__ parameter struct — eliminates per-launch overhead (~5us per launch) and improves SM occupancy when individual data sizes are small
- Pass all kernel parameters (pointers, dimensions) through a single __grid_constant__ struct rather than individual pointer/dimension arguments — enables the compiler to cache the entire parameter block in constant memory, reducing register pressure
- Use #pragma unroll on grid-stride loops over small iteration counts (bs=32) to allow the compiler to fully unroll and eliminate loop overhead, which matters when the fused kernel runs many short sequential loops
