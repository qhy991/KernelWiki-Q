## Challenge

Attention dropout applies a shared-memory mask via 2D index computation (accum_n * num_queries_in_block + accum_m) and bounds-checked branch inside the hot iteration loop, causing 5000×4 redundant smem loads with integer multiply-add address arithmetic per thread


## Solution

Read the shared-memory dropout mask once before the iteration loop and compress each element's keep/drop decision into a single uint32_t bitmask register (dropout_keep_mask), then replace every per-iteration smem load + 2D index + branch with a single BITTEST (dropout_keep_mask & (1u << i)), eliminating all redundant shared-memory traffic and integer multiply-add from the hot path

## Key Lessons

- Hoist invariant shared-memory reads out of iteration loops and compress boolean masks into register-resident uint32_t bitmasks — reduces hot-path smem traffic from N×elements to 1×elements and replaces integer multiply-add addressing with single BITTEST instructions
- When a dropout/inference mask is static across iterations (e.g., same attention pattern reused in GEMM), capture it once into registers with a __syncthreads() barrier afterward so shared memory can be reclaimed by other pipeline stages
- Use 0xFFFFFFFF as the initial bitmask and clear bits for dropped elements (mask &= ~(1u << i)) rather than setting bits for kept elements — avoids needing to know the element count at initialization time and naturally handles the common case where most elements are kept
