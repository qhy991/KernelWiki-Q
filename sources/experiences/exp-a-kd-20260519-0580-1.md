## Challenge

__cvta_generic_to_shared is invoked on every element-level shared memory address inside the cp.async hot loop for each K-tile iteration, adding redundant address-conversion instructions to the async-copy issue path in a double-buffered WMMA TF32 SGEMM kernel


## Solution

Pre-compute __cvta_generic_to_shared once on the shared memory array base pointers (s_a, s_b) before the loop, then derive each cp.async destination address via pure register byte-offset arithmetic (base + stage_offset*stage_idx + row*stride + col), eliminating per-iteration address-conversion overhead and giving the compiler a clearer scheduling hint

## Key Lessons

- Hoist __cvta_generic_to_shared out of hot loops by computing the base pointer once and using byte-offset arithmetic (base + element_index * sizeof(T)) inside the loop — eliminates redundant address-conversion instructions on the cp.async issue path and gives the compiler a clearer scheduling hint
- For multi-stage shared memory buffers accessed via cp.async, pre-compute constexpr stage offsets (e.g., stage_offset = rows * (cols + padding)) so the per-iteration address calculation reduces to a single multiply-add on the stage index, enabling the compiler to fold it into instruction scheduling alongside the async copy
