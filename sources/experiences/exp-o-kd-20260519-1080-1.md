## Solution

Replace Blelloch scan with a two-level warp exclusive scan using __shfl_up_sync: intra-warp communication uses hardware shuffle instructions (no shared memory, no __syncthreads), and only the inter-warp accumulation step requires __syncthreads() barriers, reducing total barrier count from O(2*log(N)) to O(1) per scan level

## Key Lessons

- Replace Blelloch scan with warp-level __shfl_up_sync exclusive scan when scan_size fits within a small number of warps — hardware shuffle eliminates O(log N) __syncthreads() barriers and shared-memory bank conflicts from indirect addressing, reducing synchronization overhead to O(1) per scan level
- Two-level warp scan (intra-warp shuffle + single-warp cross-warp reduction) is the standard pattern for block-wide prefix sum: first scan each warp locally via shuffle, then have warp 0 scan the per-warp partial sums, and finally each thread adds its warp's base offset — applies whenever scan data spans multiple warps but fits in shared memory
