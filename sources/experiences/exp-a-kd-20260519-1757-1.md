## Challenge

The dispatch function clamps the next-power-of-2 tile size to a minimum of 32, forcing even M<=16 workloads into the TILE_M=32 kernel path and preventing use of a smaller, more occupancy-efficient tile configuration


## Solution

Lower the dispatch clamp minimum from 32 to 16, allowing M<=16 cases to select the TILE_M=16 branch instead of always falling into the TILE_M=32 branch when the next-power-of-2 would be 16 or below

## Key Lessons

- Lowering a dispatch clamp minimum from 32 to 16 enables smaller tile configs for small-M GEMM workloads — neutral for M=32 where both paths agree, but beneficial for M<=16 cases that can now use fewer loop iterations per thread
- When the dispatch clamp equals or exceeds a branch threshold, lower branches become dead code — verify that all tile-size branches are actually reachable under the current clamp value
