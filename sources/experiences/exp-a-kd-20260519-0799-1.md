## Solution

Replace the shared memory max-reduction with __shfl_xor_sync warp shuffle reduction, eliminating 1088 bytes of shared memory and both __syncthreads barriers; the max reduction is now performed entirely in registers via warp-level communication across 16 lanes using mask=0xffff

## Key Lessons

- Replace shared memory reductions with __shfl_xor_sync warp shuffle for sub-warp (≤32 thread) reductions → eliminates shared memory allocation and __syncthreads barriers, performing the reduction entirely in registers → applies when the reduction group fits within warp lanes (16-thread groups use mask=0xffff)
- For memory-bound quantization kernels at small problem sizes, __syncthreads removal via warp shuffle provides consistent but modest speedup (≤5%) because memory access latency and kernel launch overhead dominate total execution time
