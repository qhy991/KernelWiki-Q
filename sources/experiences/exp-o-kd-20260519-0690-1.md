## Solution

Introduce a RankData struct that bundles the MAX_RANKS_PER_NODE pointer array into a single GPU-resident object. AllReduceParams now holds a RankData* instead of the embedded array, and the kernel dereferences peer_comm_buffer_ptrs->ptrs[rank]. During graph capture, the host writes a new RankData entry into a pre-allocated GPU array per allreduce call, and the AllReduceParams pointer is set to the correct slot before kernel launch, enabling dynamic buffer assignment across graph replay iterations without host-side memcpy.

## Key Lessons

- Replace embedded fixed-size pointer arrays in kernel parameter structs with pointer indirection to GPU-resident data when the parameters must vary across CUDA graph replay iterations — the host updates the pointer per capture slot rather than the data, and replay resolves the correct slot without illegal host-side memcpy
- One extra pointer dereference in-kernel (RankData* → ptrs[rank]) is a favorable trade-off against the alternative of cudaMemcpyAsync inside a captured stream, which is illegal and would prevent graph capture entirely for multi-call allreduce patterns
