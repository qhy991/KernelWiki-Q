## Challenge

The performance gap widens significantly at 32768 tokens because the multi-block approach scales with input size while the single-block approach has fixed 1024-thread parallelism. ## Solution

Fix the sign in the causal-from-bottom-right diagonal formula: change key_start - num_keys - num_queries to key_start - num_keys + num_queries, correctly positioning the bottom-right causal boundary relative to the key position

## Results

- **speedup**: 1.960x (improved)


## Key Lessons

- CausalFromBottomRight masking requires key_start - num_keys + num_queries (plus, not minus) to correctly compute the bottom-right diagonal boundary — a single sign error in the formula causes queries to attend to incorrectly masked keys across most configurations
- When causal masking validation shows partial correctness (e.g., 2/6 cases), suspect a formula sign error in the diagonal position calculation rather than a logic omission — the cases that pass are typically the ones where the wrong formula happens to produce the same result as the correct one
