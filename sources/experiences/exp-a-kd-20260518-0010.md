## Challenge

Naive 4-bit dequant unpacks each nibble pair using two instructions: AND to mask nibble bits from q, then OR to add the FP16 bias constant 0x64006400. This is 2 instructions per nibble pair, executed twice per uint32_t input (4 nibble pairs total).


## Solution

Use the LOP3 ternary logic instruction: lop3<(0xf0 & 0xcc) | 0xaa>(q, mask, bias) fuses the AND+OR into a single hardware instruction. LOP3 computes arbitrary 3-input boolean functions in one cycle. The 4% speedup is below the 1.05x threshold in the microbenchmark because __hsub2 and __hfma2 FP16 arithmetic dominates execution time, but in the full MarlinMoE kernel where dequant is invoked per weight fragment across thousands of K-dimension iterations, cumulative instruction savings would be significant.

## Results

- **speedup**: 1.040x (4% latency reduction, below 1.05x threshold)


## Key Lessons

- LOP3 (ternary logic instruction) fuses arbitrary 3-input boolean functions into one cycle. Use it to combine bitwise AND+OR patterns: lop3<(0xf0 & 0xcc) | 0xaa> computes (A & B) | C. Applies to any bit-manipulation sequence where two consecutive logic ops can be expressed as a single truth table. Available on SM50+.
- Instruction-count reductions in dequant nibble unpacking have cumulative impact in production kernels. A 4% microbenchmark gain may become significant when dequant is called millions of times in a MarlinMoE-style kernel. Evaluate in the full kernel context, not just isolated microbenchmarks.
- In microbenchmarks where FP16 arithmetic (__hsub2, __hfma2) dominates total execution time, small control-flow/bit-manipulation savings get diluted. The instruction mix matters: optimize the bottleneck category, not all instructions equally.
