## Solution

Replace full O(N log N) sort with multi-pass radix selection: build a coarse 256-bin FP16 histogram in shared memory in one pass to identify the threshold bin separating top-K from the rest, then refine only the ambiguous candidates in that threshold bin using byte-level radix passes on FP32 bits, avoiding global-memory sorting entirely

## Key Lessons

- For top-K selection where K << N, radix-based selection with a coarse histogram followed by refinement of only the threshold bin avoids the O(N log N) cost of full sorting — applies whenever a partial order suffices and the data can be quantized into bins
- Using FP16 quantization for the coarse histogram pass reduces the bin space to 256 entries that fit in shared memory, enabling a single-pass filter that eliminates the vast majority of elements before any detailed comparison is needed
- Converting FP32 values to order-preserving unsigned integers (flipping sign bit for negatives, setting MSB for positives) enables radix-sort techniques on floating-point data without changing the comparison semantics
