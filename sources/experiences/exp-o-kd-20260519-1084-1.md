## Solution

Pack 4 int32 values into a single Vec (int4) struct and write 16 bytes per store instruction via reinterpret_cast<Vec*>, reducing iteration count and store transactions by 4x while improving memory coalescing for large padding fills

## Key Lessons

- Use vectorized int4/float4 stores instead of scalar int32 writes for contiguous memory fills — a single 16-byte store replaces four 4-byte transactions, reducing store-unit pressure and improving coalescing for arrays with thousands of elements
- reinterpret_cast<Vec*>(base_ptr) with a Vec struct containing .x/.y/.z/.w fields lets the compiler emit a single 128-bit store per iteration without changing the underlying memory layout
