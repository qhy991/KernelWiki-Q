## Solution

Consolidate 4 half2 loads/stores into single 128-bit LDST128BITS (float4) transactions using packed local arrays, replace scalar __hadd on .x/.y lanes with SIMD __hadd2 intrinsic in a pragma-unroll loop, and merge 4 boundary checks into one (idx+7 < N), improving memory coalescing, compute throughput, and branch efficiency simultaneously

## Key Lessons

- Use 128-bit (float4) LDST128BITS transactions to load/store 8 half elements in one instruction instead of 4 separate half2 reads/writes — reduces memory instruction count by 4x and improves coalescing for bandwidth-bound elementwise kernels
- Prefer SIMD __hadd2 over scalar __hadd on .x/.y lanes — __hadd2 computes both half additions in a single hardware instruction, doubling arithmetic throughput for half-precision elementwise operations
- Consolidate multiple per-element boundary checks into a single (idx + N_elements - 1 < N) guard — eliminates redundant branches and warp divergence when processing multiple elements per thread
