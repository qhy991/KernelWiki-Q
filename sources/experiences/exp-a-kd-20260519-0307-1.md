## Challenge

MoE gated activation pipeline uses two separate CUDA kernels launched sequentially — one for SiLU-gated activation writing to a 16 MB temporary buffer, and another for per-expert prequant scale multiplication reading from that buffer — incurring an extra kernel launch overhead and a redundant global memory round-trip


## Solution

Fuse the gated activation and per-expert scale multiply into a single kernel that computes apply_gate_vec followed by mul_vec in the same loop iteration, eliminating the intermediate temp buffer and second kernel launch — however, this increases register pressure by requiring both activation intermediates (post_act) and scale data (scale_vec, out_vec) to be live simultaneously

## Key Lessons

- Kernel fusion can regress performance when the fused kernel demands significantly more registers than the separate kernels — the added register pressure (holding scale pointers, post_act intermediates, and out_vec simultaneously) reduces occupancy enough that the lost parallelism outweighs savings from eliminating kernel launch overhead and intermediate buffer traffic
- Two-pass kernels with simple per-pass logic can outperform a fused single-pass kernel when each individual pass has low register pressure and high occupancy, even when the intermediate buffer requires a full global memory round-trip — the higher per-pass occupancy delivers more aggregate memory bandwidth
