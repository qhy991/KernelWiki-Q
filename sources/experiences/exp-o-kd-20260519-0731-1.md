## Challenge

Cumulative sum is computed serially by a single thread (threadIdx.x == 0) iterating over all 256 experts with div/mod per iteration to resolve warp-based two-level addressing, while 1023 threads idle at the barrier — the O(N) serial loop dominates kernel time as expert count grows


## Solution

Replace the serial O(N) cumsum loop with a parallel Blelloch (work-efficient) prefix scan over a power-of-2 scan buffer containing padded expert counts — the up-sweep and down-sweep phases each require only log2(N) barrier rounds, reducing serialization from 256 sequential iterations to 8 parallel rounds per phase

## Key Lessons

- Replace single-thread serial prefix sum with parallel Blelloch scan when the element count is large enough (≥64) that the O(N) serial loop exceeds the cost of log2(N) barrier-synchronized rounds — scales from 256 sequential iterations to 8 parallel rounds
- Use a power-of-2 scan buffer (next_pow2) for Blelloch scan to enable the balanced binary tree addressing pattern, padding excess slots with zero so they contribute nothing to the prefix sum
