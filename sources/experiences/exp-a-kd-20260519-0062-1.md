## Challenge

CUTLASS epilogue kernel unconditionally loads the source tensor from global memory and computes beta*source even when beta=0 (OnlyAlphaScaling mode), wasting memory bandwidth on a read whose result is multiplied by zero


## Solution

Make is_source_needed() a compile-time decision for the OnlyAlphaScaling epilogue type, eliminating the source tensor load and beta multiply entirely so the kernel only reads the accumulator and computes alpha*accum directly

## Key Lessons

- Eliminating dead memory loads (source tensor when beta=0) via compile-time epilogue type dispatch is correct in principle, but measurable speedup requires the saved traffic to exceed the L2 cache resident set — at 256K fp32 elements the dataset fits entirely in L2, so both versions read from cache rather than DRAM
- Source load elimination benefits grow with problem size: when the source tensor exceeds L2 cache capacity, skipping the load saves real DRAM bandwidth and the optimization becomes impactful
- Kernel launch overhead (~0.1ms) can dominate microbenchmarks on small datasets, masking genuine memory bandwidth improvements — validate bandwidth-sensitive optimizations at sizes that exceed the L2 cache (≥40MB on A800)
