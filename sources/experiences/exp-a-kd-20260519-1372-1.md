## Solution

Replace __ldg-based loads and plain stores with PTX inline assembly using .cs (cache-streaming) hints — ld.global.cs.v4.u32 and st.global.cs.v4.u32 — which stream 4x32-bit values through the cache hierarchy without allocating persistent L2 lines, preserving L2 capacity for data that benefits from caching

## Key Lessons

- PTX .cs (cache-streaming) hints bypass L2 allocation for load/store — they benefit streaming workloads only when the working set exceeds L2 capacity or when concurrent kernels compete for L2 space, not when data fits comfortably in cache
- A streaming copy whose total working set (src + dst = 256KB) fits entirely within L2 (5MB on A800) sees no measurable gain from .cs hints because there is no L2 eviction pressure to relieve — memory bandwidth remains the bottleneck
- Use .cs-hinted loads/stores proactively in kernels that process data once alongside other GPU work sharing the same L2, as the real benefit is reducing cache pollution rather than improving single-kernel throughput
