## Solution

Replace scalar per-element kernel with a specialized 3-to-8 channel kernel using shared memory tiling (192 threads cooperatively load float4 input vectors into shared memory) and vectorized float4 output assembly, where even-indexed threads write 3 data channels + 1 zero and odd-indexed threads write all-zero float4, eliminating modular index arithmetic entirely and quadrupling memory throughput

## Key Lessons

- For fixed-ratio channel padding (e.g., 3→8), specialize the kernel to avoid per-element modular index decomposition entirely — use shared memory cooperative loading with vectorized I/O so threads map to output elements via simple arithmetic, eliminating expensive modulo/division cascades
- Use vectorized float4 loads/stores when input/output channels are known multiples — quadruples memory throughput and enables coalesced access compared to scalar per-element stores
- Exploit the data-layout regularity of channel padding: for c_in=3→c_out=8, each pair of output float4 vectors maps to one input float4 (3 data + 5 zeros), allowing even/odd thread logic to handle the zero-fill without branching
