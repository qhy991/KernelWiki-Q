## Solution

Compute tile-aligned skip boundary by floor-dividing the skip offset to CTA_kvL granularity, reducing KV cache tiles to process from ceil(seq_len/CTA_kvL) to ceil(sliding_window_size/CTA_kvL)+1, then distribute only the relevant tile range across splits

## Key Lessons

- Align sliding window skip offset to tile boundaries (floor division by tile size × tile size) instead of using exact token positions — this ensures TMA descriptors remain valid and avoids partial-tile edge cases at the cost of processing at most one extra tile
- When distributing work across splits in a persistent-kernel model, add the skip tile offset to each split's start position so tile IDs map correctly into the global KV cache even after truncating the workload range
