## Challenge

Scalar per-token scale parameter is passed by value and resides in a register, enabling zero-overhead quantization but limiting quantization to a single uniform scale factor across the entire hidden dimension


## Solution

Replace register-resident scalar scale with per-group scale array indexing via integer division (i/group_size) inside the inner loop, adding groupwise quantization support through conditional global memory loads and optional reciprocal computation for inverted scales

## Key Lessons

- Replacing a register-resident scalar parameter with per-group global memory array access inside a tight inner loop regresses performance for short kernels (~6us) because integer division overhead (i/group_size) and additional memory loads cannot be amortized over the kernel's brief execution time
- When consecutive threads in the same warp access different scale values at group boundaries, the small scale array (128 bytes for 32 groups) cannot leverage spatial locality, making each scale load an independent memory transaction rather than a coalesced access
- For groupwise quantization in short kernels, consider loading scale values into shared memory once per block rather than issuing per-element global memory loads with integer division indexing
