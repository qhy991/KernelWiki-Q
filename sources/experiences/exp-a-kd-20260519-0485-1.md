## Challenge

Single-stage (kStage=1) synchronous K/V loading in flash attention split-KV kernel forces each KV tile load to complete before MMA computation can proceed, exposing full memory latency as stall cycles with no overlap between data transfer and arithmetic


## Solution

Upgrade to multi-stage (kStage=2) async pipeline with double-buffered K shared memory, enabling prefetch of the next K tile and V tile via cp.async while current MMA computation proceeds, overlapping memory transfers with arithmetic

## Key Lessons

- Multi-stage async pipelines (kStage=2 double-buffering) yield diminishing returns at intermediate sequence lengths where compute per iteration already dominates memory — the overlap fraction shrinks as arithmetic intensity grows relative to transfer cost
- Pipeline overhead (extra shared memory for double-buffering K tiles, conditional wait-group logic) partially offsets the latency-hiding benefit and can make the optimization net-neutral when compute-to-memory ratio is already favorable
