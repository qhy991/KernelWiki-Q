## Challenge

Row expansion kernel copies float elements one at a time via stride loop (4-byte load/store per iteration), issuing 2048 scalar memory transactions per row and requiring 8 loop iterations per thread with 256 threads, underutilizing available memory bandwidth


## Solution

Reinterpret float pointers as uint4 (128-bit) pointers to issue 16-byte load/store instructions that move 4 floats per memory transaction, reducing total memory instructions by 4x and cutting loop iterations per thread from 8 to 2, allowing the memory controller to coalesce wider transactions into fewer DRAM bursts

## Key Lessons

- Use uint4 (128-bit) vectorized load/store via reinterpret_cast to move 4 floats per memory transaction instead of scalar copies — reduces memory instruction count by 4x and improves bandwidth utilization for contiguous copy kernels
- Vectorized access reduces loop iteration count proportionally (cols/4 iterations instead of cols), cutting loop overhead and index arithmetic in addition to the memory throughput benefit
