## Challenge

Single-query paged KV attention with BLOCK_SIZE=16 forces THREAD_GROUP_SIZE=2, requiring an intra-group shuffle reduction after every QK dot product and doubling the number of paged KV blocks to iterate over, increasing loop overhead and block-table lookup cost per sequence


## Solution

Increase BLOCK_SIZE from 16 to 32 so THREAD_GROUP_SIZE becomes 1, allowing each thread to compute its full QK dot product independently without shuffle reduction, while halving the number of paged KV blocks and increasing per-warp value parallelism

## Key Lessons

- Setting BLOCK_SIZE equal to WARP_SIZE (32) in single-query paged attention makes THREAD_GROUP_SIZE=1, eliminating intra-warp shuffle reduction in QK dot products — each thread handles one full token independently, saving instructions per QK computation
- Larger BLOCK_SIZE halves the number of paged KV blocks for the same sequence length, reducing block-table lookup overhead and loop iteration count in both QK and value accumulation passes
- Increasing BLOCK_SIZE from 16 to 32 widens value parallelism (NUM_V_VECS_PER_ROW 2→4), giving each warp more concurrent work across value cache rows — the extra shuffle steps in reduction are outweighed by reduced block iteration
