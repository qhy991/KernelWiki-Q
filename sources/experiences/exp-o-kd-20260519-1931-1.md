## Solution

Replace the hardcoded SM count with a runtime query via cudaDeviceGetAttribute(&sm_count, cudaDevAttrMultiProcessorCount, device_id) so the block grid is sized to the actual hardware, yielding 216 blocks and 10 groups-per-block on A800 instead of 168 blocks and 13 groups-per-block

## Key Lessons

- Query actual device properties (SM count, memory bandwidth, compute capability) at runtime via cudaDeviceGetAttribute instead of hardcoding assumptions — ensures grid sizing adapts to the deployment GPU and avoids under-utilization on larger hardware
- In grouped/batch workloads where each block processes ceil(num_groups/num_blocks) items, increasing block count reduces per-block sequential iterations and spreads work across more SMs — the benefit is proportional to the ratio of actual to assumed SM count
