## Solution

Set math mode to CUBLAS_TF32_TENSOR_OP_MATH and select CUBLAS_GEMM_DEFAULT_TENSOR_OP algorithm, which instructs cuBLAS to internally cast FP32 inputs to TF32 format (19-bit: 10 mantissa, 8 exponent, 1 sign) and dispatch multiply-accumulate to tensor cores for much higher throughput while keeping input/output arrays in CUDA_R_32F

## Key Lessons

- Enable CUBLAS_TF32_TENSOR_OP_MATH + CUBLAS_GEMM_DEFAULT_TENSOR_OP together to route FP32 GEMM through tensor cores on Ampere+ GPUs — the math mode permits TF32 internal casting and the algorithm hint requests tensor-op dispatch, achieving 1.15x–5.18x speedup with negligible precision loss for deep learning workloads
- TF32 retains the same 8-bit exponent range as FP32 with a reduced 10-bit mantissa (vs 23-bit), making the precision trade-off acceptable for most DNN training and inference workloads while unlocking tensor-core throughput
