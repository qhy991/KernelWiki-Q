## Solution

Fuse the normalize and scale operations into a single kernel that reads each element once from global memory, applies both the row-sum division and the routed scaling factor multiplication in the same thread context, then writes the final result once — cutting global memory traffic by roughly half and eliminating one kernel dispatch

## Key Lessons

- Fuse elementwise operations that traverse the same memory region into a single kernel to halve global memory traffic and eliminate kernel launch overhead — applies whenever two kernels read/write the same array sequentially
- Kernel fusion savings are proportionally larger for smaller workloads where launch overhead dominates; for larger arrays the memory bandwidth savings remain the primary benefit
