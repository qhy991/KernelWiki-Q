## Challenge

Scalar fallback for quantized q4_0×q8_1 dot product decomposes each 32-bit word into individual bytes via pointer casts, extracts 4-bit nibbles with bitmask, sign-extends with subtraction, and accumulates with per-element integer multiply-add in a nested loop — costing 4 IMUL+ADD per vdr element plus byte-level temporaries


## Solution

Replace the scalar byte-decomposition loop with the __dp4a intrinsic which performs a 4-element signed integer dot product in a single hardware instruction, consolidating the bitmask extraction, sign extension, and multiply-add into the GPU's integer dot-product unit (IDP4A on Ampere) and adding #pragma unroll on the outer loop

## Key Lessons

- Use __dp4a intrinsic for 4-element signed integer dot products instead of manual byte decomposition loops — it maps directly to the GPU's integer dot-product hardware unit, eliminating bitmask extraction, sign-extension, and per-element IMUL+ADD sequences into one instruction
- For quantized dot-product kernels (q4_0×q8_1), the full theoretical speedup from instruction elimination may be partially hidden by memory latency when the kernel is memory-bound at the given problem size — the ~10% gain here reflects arithmetic savings rather than the full instruction count reduction
