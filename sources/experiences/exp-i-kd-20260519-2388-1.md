## Challenge

Need to implement a CUTLASS-based debug kernel that loads matrix fragments from global memory via PredicatedTileIterator, stores them to shared memory via RegularTileIterator, and exercises cutlass::debug dump utilities for register and shared memory inspection


## Solution

Construct a PredicatedTileIterator with PitchLinearWarpRakedThreadMap to load a column-major half_t matrix tile into a Fragment, then use cutlass::debug::dump_fragment() with varying N/M/S parameters for register-level inspection, store the fragment to shared memory via a RegularTileIterator with ColumnMajorTensorOpMultiplicandCongruous layout, and call cutlass::debug::dump_shmem() for shared memory inspection

## Key Lessons

- CUTLASS PredicatedTileIterator requires the thread map (PitchLinearWarpRakedThreadMap), matrix shape, element type, and layout as compile-time template parameters — the iterator is then constructed at runtime with params, data pointer, extent, and thread ID to handle boundary predicates automatically
- cutlass::debug::dump_fragment(frag, N, M, S) supports graduated inspection: N=number of threads to dump, M=max elements per thread, S=stride between printed elements — use N=1 to restrict output to the first thread and avoid log spam in large warps
- RegularTileIterator for shared memory uses a specialized smem layout (ColumnMajorTensorOpMultiplicandCongruous) that matches tensor core access patterns, requiring a packed TensorRef constructed from the extern __shared__ array and the smem layout
