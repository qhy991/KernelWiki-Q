## Challenge

In the small_batch_256_8exp before code, the same two-kernel pipeline processes 256 tokens with 8 experts: the align kernel uses 1024 threads with shared-memory atomicAdd, and the sort kernel uses 1 block of 256 threads with global-memory atomicAdd. The after code uses the single moe_align_block_siz


## Solution

, doing per-thread counting without atomicAdd. With 256 tokens spread across 32 threads, each thread processes ~8 elements.
