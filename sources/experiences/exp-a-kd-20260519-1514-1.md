## Solution

Wrap min() in a __device__ inline function (min__()) with explicit uint32_t parameter and unsigned int return types to force LLVM to stay in the unsigned integer domain, preventing implicit double-precision upcast on ROCm while remaining identical on NVIDIA CUDA

## Key Lessons

- Wrapping bare min()/max() in an explicitly-typed __device__ function prevents LLVM from implicitly upcasting integer arguments to double on ROCm — critical for cross-vendor portable code where the same CUDA source must perform well on both NVIDIA and AMD hardware
- On NVIDIA CUDA, min() with uint32_t already compiles to integer IMIN instructions, so an explicit wrapper adds zero overhead — use it as a defensive portability pattern even when targeting NVIDIA-only deployments
