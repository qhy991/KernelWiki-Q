## Solution

Replace packed interleaved layout with flat NHD (num_blocks, block_size, num_heads, head_size) layout using stride-based addressing, where adjacent threads compute sequential tgt_idx values enabling fully coalesced 128-byte memory transactions for both key and value writes

## Key Lessons

- Flat NHD (num_blocks, block_size, num_heads, head_size) cache layout with stride-based addressing enables coalesced writes from adjacent threads, replacing strided access patterns (e.g., value_dst[i*block_size]) that serialize memory transactions — applies to any KV cache or tile-based buffer where multiple threads write to the same logical block
- Thread-stride loops (for i = threadIdx.x; i < n; i += blockDim.x) uniformly distribute work across all active threads and avoid wasted thread slots from per-h_block assignment with early-return guards — use when total element count n is a simple product known at kernel launch
