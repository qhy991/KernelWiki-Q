## Challenge

Out-of-place rotary embedding kernel allocates separate output buffers (out_query, out_key) for write results while reading from const query/key via __ldg, doubling memory footprint from ~4MB to ~8MB for the input+output tensors at 128 tokens × 32 heads × 128 dimensions


## Solution

Convert to in-place operation by removing separate output buffers and reading/writing the same query/key buffers directly, eliminating __ldg for query/key since they are no longer const, and relying on read-before-write ordering within each loop iteration to ensure correctness

## Key Lessons

- Converting out-of-place to in-place operation halves memory footprint by eliminating separate output buffers — applies when each output element depends only on its corresponding input element(s) which are fully consumed before being overwritten
- __ldg routes loads through the separate read-only texture cache on sm_80+, which can partially offset the L1/L2 locality advantage of in-place access; when working sets exceed L2 capacity (~2MB per tensor at this scale), both versions stream from DRAM and the net performance difference is negligible
- In-place rotary embedding is safe because each (x, y) pair is read into registers before either output is written back to the same addresses — the read-before-write ordering within a single thread's iteration guarantees correctness without needing synchronization
