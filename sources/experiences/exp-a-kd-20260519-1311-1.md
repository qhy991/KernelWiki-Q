## Solution

Replace scalar half loads with 128-bit uint4 vectorized loads that fetch 8 half values (16 bytes) per instruction, enabling 32 threads to produce 512-byte warp transactions that fully saturate the L2/DRAM path while keeping __half2float conversion on register-resident data instead of issuing separate global load and conversion instructions

## Key Lessons

- Scalar half loads (2 bytes) produce only 64-byte warp transactions on sm_80, far below the 128-byte coalescing optimum — use uint4 (16-byte) vectorized loads to reach 512-byte warp transactions and fully saturate the memory path, especially for bandwidth-bound reduction kernels
- Vectorized uint4 loads bring data into registers first, allowing __half2float conversion on register-resident values instead of pairing separate global load and conversion instructions, reducing both memory traffic and instruction count by the vectorization factor
- For MoE routing finalization and similar element-wise reduction kernels over half-precision data, the vectorization factor should match the natural width: uint4 = 8 half values = 16 bytes, which aligns to the 128-byte coalescing boundary when 8 threads cooperate
