## Solution

Fuse both kernels into a single topk_fused_ragged_kernel that performs top-K selection with intermediate scatter writes landing in shared memory (s_indices[TopK]) instead of global memory, then adds per-row offset and writes coalesced final results directly to global memory in one pass

## Key Lessons

- Fuse a producer-consumer kernel pair into one kernel when the intermediate data fits in shared memory — eliminates one kernel launch (~5-10us overhead) and removes a full global-memory write+read round-trip for the intermediate buffer
- Use shared memory instead of global memory for intermediate scatter-write buffers (e.g., top-K index arrays built via atomicAdd) — shared memory offers ~30x lower latency and no coalescing penalty for random-access patterns
- When fusing kernels, pass the intermediate buffer as a shared-memory pointer to the sub-function rather than a global-memory pointer, so the same algorithm works unchanged but targets faster memory
