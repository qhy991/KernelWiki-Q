## Challenge

With N=512 at hidden_size=1024, the v2 kernel launches blocks of 512 threads, which exceeds a full warp (32 threads) and provides 16 warps per bloc


## Solution

k. All warps within a block access the same token row sequentially, producing well-coalesced global memory reads. The v1 kernel uses BLOCK_SIZE=256

## Results

- **speedup**: 1.204x (improved)


## Key Lessons

- The v1 kernel uses BLOCK_SIZE=256 with a flat index across all tokens, meaning threads in the same warp frequently cross token boundaries during their 2-wide strided access pattern.
