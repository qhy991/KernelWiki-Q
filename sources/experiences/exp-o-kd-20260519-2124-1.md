## Solution

Replace O(N*K) iterative selection with WarpSelect incremental top-k: read input in rounds of WARP_SIZE, use __ballot_sync to early-exit rounds where no candidate exceeds the current k-th threshold, compact qualifying candidates into a small buffer, and invoke warp-level bitonic sort/merge only when the buffer fills to 32 elements — reducing sync overhead from K __syncthreads to O(N/32) __syncwarp calls and shared memory from ~3KB to 256 bytes

## Key Lessons

- Replace iterative-selection top-k (O(N*K) with K full scans and K __syncthreads) with WarpSelect incremental filtering (O(N log k) with ballot-based early exit and warp-only bitonic sort/merge) — reduces both computation and synchronization overhead when K << N
- Use __ballot_sync to test whether any warp lane has a candidate exceeding the current k-th threshold — when the mask is zero, the entire round is skipped with zero work, providing increasing early-exit benefit as the top-k buffer fills
- Prefer warp-scoped synchronization (__syncwarp) and small per-warp shared buffers (64+64 bytes) over block-scoped __syncthreads and large per-block arrays when each warp processes an independent top-k instance — eliminates unnecessary cross-warp barriers and reduces shared memory pressure
