## Solution

Replace synchronous FLOAT4 stores to s_b with cp.async.ca (copy asynchronous, cache-allocate) inline PTX instructions that issue non-blocking global-to-shared transfers via __cvta_generic_to_shared pointers, commit the async group, then overlap the B-tile transfer latency with A-tile loading and scalar transpose writes before waiting for completion with CP_ASYNC_WAIT_GROUP(0)

## Key Lessons

- Use cp.async.ca (async copy with cache-allocate policy) via inline PTX to issue non-blocking global-to-shared memory transfers, then commit the group and overlap the transfer latency with independent compute or other memory work before CP_ASYNC_WAIT_GROUP — applies when shared memory loading can be overlapped with register loads, transposes, or other arithmetic
- __cvta_generic_to_shared() converts a generic shared memory pointer to the shared memory address space required by cp.async PTX instructions — always use it to obtain the uint32_t smem pointer before issuing async copies
- Separate the async commit from the wait: commit early (CP_ASYNC_COMMIT_GROUP) to start transfers, do independent work, then wait late (CP_ASYNC_WAIT_GROUP) just before the data is consumed — this maximizes latency hiding
