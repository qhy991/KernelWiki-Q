## Challenge

Flat linear thread indexing across all token positions causes threads in the same warp to frequently cross token boundaries during 2-wide strided memory access, resulting in poorly coalesced global memory reads and writes


## Solution

Restructure thread-to-data mapping so each CUDA block handles one token position (blockIdx.x = token_pos) and threads within the block handle different dimension pairs (threadIdx.x = dimension index), producing contiguous sequential memory access within each warp and fully coalesced global reads/writes

## Key Lessons

- Map threads to contiguous memory regions within each warp rather than using a flat index that crosses data-structure boundaries — coalescing improves when adjacent threadIdx values access adjacent memory addresses
- Structure CUDA grids so that one block maps to one logical data unit (e.g., one token position) and threads handle different elements within that unit — this keeps all memory accesses within a warp sequential and coalesced
- When the inner dimension (N=512) is large enough to fill a block, prefer one-block-per-row mapping over flat 1D indexing to avoid warp-level boundary straddling
