## Solution

Add a greedy shortcut clause that unconditionally accepts any draft token whose individual target probability meets or exceeds a threshold (0.5), bypassing stochastic coin comparison for high-confidence tokens while retaining stochastic verification for lower-probability tokens.

## Key Lessons

- Adding a greedy acceptance shortcut for high-confidence draft tokens (target_prob >= threshold) in speculative decoding bypasses stochastic verification for tokens that would almost certainly be accepted anyway, increasing per-step acceptance rate without adding significant compute cost
- A single scalar comparison (prob >= threshold) is trivially cheap relative to global-memory reads, so adding a greedy clause to an acceptance kernel has negligible latency impact while measurably improving acceptance throughput
