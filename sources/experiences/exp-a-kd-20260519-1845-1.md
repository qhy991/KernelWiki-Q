## Solution

Change the compile-time dispatch guard from weight type (b_type == kFE2M1f) to scale type (s_type == kFE4M3fn || s_type == kFE8M0fnu), so that any configuration with FP8-class scales correctly enters the dequantization path regardless of the weight format

## Key Lessons

- Guard dequantization dispatch on the scale type (s_type) rather than the weight type (b_type) — the decision to dequantize depends on whether the scale format is sub-byte FP8, not on what format the weights are in
- When extending quantization support to new formats (e.g., MXFP8), audit all dispatch guards that conflate weight type with scale type — such guards silently skip the correct path for mixed configurations
