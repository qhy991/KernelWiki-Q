## Challenge

Implement flash attention with FP32 query, F16 KV cache, single query position. Key challenges: shared memory limits (48KB), correct online softmax algorithm, mixed precision handling.


## Solution

Per-(batch,head) thread block with online softmax


### Key Changes

- Dynamic shared memory: Q_tile (128 floats) + K_tile (32*128) + V_tile (32*128) = ~32KB
- Online softmax: maintain m_prev, l_prev, acc across tiles
- Mixed precision: convert F16 KV to FP32 for computation


## Key Lessons

- Shared memory layout must match actual head_dim (128), not BLOCK_M
- Dynamic shared memory allocation helps fit within 48KB limit
- Online softmax requires careful handling of m_prev, l_prev updates
- Mixed precision (FP32 query, F16 KV) needs explicit conversion
