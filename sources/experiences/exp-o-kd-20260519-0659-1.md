## Challenge

Direct vector copy via reinterpret_cast<AccessType<T>> on a local array causes misaligned address access when VPT (values per thread) is less than the AccessType alignment requirement (8 elements) and mismatches with a fixed MAX_VPT=32 array size


## Solution

Replace the direct vector copy with a #pragma unroll loop that extracts elements individually from the vectorized pointer into the local array, avoiding misaligned address issues while still allowing the compiler to optimize the unrolled load pattern

## Key Lessons

- Use loop-based element extraction instead of direct vector copy via reinterpret_cast when the runtime element count (VPT) may be smaller than the vectorized access width — avoids misaligned address faults at the cost of slightly degraded performance when VPT exactly equals the vector width
- #pragma unroll on the extraction loop allows the compiler to generate efficient load sequences while correctly handling variable VPT, bridging the gap between alignment safety and vectorized performance
