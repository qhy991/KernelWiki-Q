## Challenge

Index decomposition in quantization SF offset calculation uses integer division and modulo with compile-time constant power-of-2 divisors (128, 32, 4), which may not always be optimized to shift/AND by nvcc when the function is not force-inlined in heavily templated code


## Solution

Replace all div/mod operations on power-of-2 divisors with explicit right-shift (>>) and bitwise AND (&) operations, which always compile to single-instruction ALU ops regardless of inlining depth or compiler context

## Key Lessons

- Use explicit shift (>>) and bitwise AND (&) instead of div/mod for power-of-2 divisors — nvcc may not apply this optimization across non-inlined function boundaries in heavily templated code, making explicit bit operations more robust than relying on compiler strength reduction
- Microbenchmarks of div-to-shift conversions can understate real-world impact because isolated benchmarks create favorable compiler contexts (full inlining, constant propagation) that production code with deep template nesting may not replicate
