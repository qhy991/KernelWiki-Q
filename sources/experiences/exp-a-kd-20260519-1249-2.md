## Challenge

The float-to-int8 conversion uses a software pipeline of rintf for rounding plus fmaxf/fminf for clamping, costing multiple floating-point instructions per element instead of a single hardware-native conversion


## Solution

Replace the multi-instruction software conversion with a single PTX inline asm cvt.rni.sat.s8.f32 instruction that performs rounding-to-nearest-even and saturation to int8 range in one hardware operation

## Key Lessons

- Use PTX inline asm cvt.rni.sat.s8.f32 for float-to-int8 conversion — the .sat modifier provides hardware saturation to [-128, 127] in a single instruction, replacing the software rintf + fmaxf/fminf pipeline
- Prefer hardware saturation (.sat) over manual clamping for narrow integer types — avoids branch-prone min/max chains and reduces instruction count from 3+ to 1
