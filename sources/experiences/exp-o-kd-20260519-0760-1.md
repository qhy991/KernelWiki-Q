## Challenge

Per-token fp8 quantization kernel includes a scalar remainder loop that processes leftover elements not covered by the vectorized uint4 (8-element) path, adding ~15 instructions of dead code when hidden_dim is aligned, bloating the kernel binary and increasing instruction cache pressure


## Solution

Remove the scalar remainder loop entirely and enforce hidden_dim alignment (divisible by vec_size=8) at the host level with a runtime check, eliminating dead code from the kernel binary and allowing the compiler to schedule the main vectorized loop more aggressively without accounting for the transition to the scalar tail path

## Key Lessons

- Remove scalar remainder/fallback loops from vectorized kernels when the data dimension is guaranteed aligned at the host level — the dead code bloats the kernel binary, increases I-cache pressure, and may prevent the compiler from optimally scheduling the main vectorized path
- Enforce alignment preconditions (e.g., hidden_dim % vec_size == 0) at the host/caller level rather than handling misalignment inside the kernel — host-side checks are free at runtime and keep the kernel code path clean
