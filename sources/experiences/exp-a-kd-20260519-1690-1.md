## Solution

Guard the cudaMalloc+cudaMemset allocation, getMIndicesKernel launch, and subsequent cudaMemcpy behind a needs_alignment conditional; when the CUTLASS path is active (needs_alignment=false), all GPU work for alignment is skipped entirely, reducing per-iteration cost to near zero

## Key Lessons

- Guard GPU allocations and kernel launches behind a runtime conditional when the operation is only needed for certain execution paths (e.g., CUTLASS vs non-CUTLASS) — cudaMalloc+cudaMemset+kernel launch overhead is non-trivial and compounds across hot-loop iterations
- Avoid unconditional per-iteration cudaMalloc/cudaFree pairs for temporary buffers when the buffer may not be needed; the driver-level allocation and deallocation cost dominates over the actual kernel compute for small tensors (9 int64 elements in this case)
