## Solution

Introduce a 128-bit int4 vectorized fast path that loads VEC_SIZE elements (4 for fp16, 2 for fp32) per transaction using int4 reinterpret casts, with a scalar cleanup loop for remainder elements and an alignment-check fallback to the original scalar path for unaligned pointers or small dimensions

## Key Lessons

- Replace scalar element-by-element loads/stores with int4 (128-bit) vectorized memory transactions when pointers are 16-byte aligned — reduces total memory transactions by VEC_SIZE (4x for fp16, 2x for fp32) and is critical for memory-bound activation kernels with low arithmetic intensity
- Use a three-tier loop structure (vectorized main loop + scalar cleanup for remainders + original scalar fallback) to safely apply vectorization without breaking correctness for unaligned pointers, small dimensions, or non-divisible element counts
- Compute VEC_SIZE as 16 / sizeof(scalar_t) to make the vectorization width type-adaptive — the same code automatically uses 4-element vectors for fp16 and 2-element vectors for fp32 without specialization
