Use CUTLASS 3.x Epilogue Visitor Tree (EVT) to fuse per-row/col scaling, bias addition, and activation into GEMM epilogue, avoiding separate kernels

## Challenge

Quantized GEMM (FP8/INT8) requires post-GEMM operations: per-row activation scaling, per-column weight scaling, bias addition, and optional activation functions. Running these as separate kernels adds kernel launch overhead and extra global memory round-trips.

## Solution

CUTLASS 3.x Epilogue Visitor Tree (EVT) for fused post-GEMM operations

Build a computation tree in epilogue that chains: accum * scale_b (per-column) -> result * scale_a (per-row) -> + bias. For SM89+ FP8, use KernelTmaWarpSpecializedPingpong for double-buffered TMA loads. For SM100+, use KernelScheduleAuto for automatic stage selection.

## Key Lessons

- Fused epilogue via CUTLASS EVT eliminates separate kernel launches for scaling+bias, saving global memory bandwidth
- FP8 GEMM with EVT on SM89 achieves 200+ TFLOPS, significantly outperforming separate post-processing kernels
- EVT tree structure: Sm90EVT<Compute, InputA, InputB> chains operations; nest EVT nodes for multi-step fusion
- SM90 FP8 uses Pingpong schedule (KernelTmaWarpSpecializedPingpong) for overlapping TMA loads with compute via double buffering
- SM100+ uses KernelScheduleAuto which automatically selects optimal stage count and pipeline schedule
- INT8 GEMM also supports fused epilogue via EpilogueVisitor for per-row/col scaling without separate kernels
