## Solution

Pack half-precision value (16 bits) and index (16 bits) into a single uint32_t using cub::Traits::TwiddleIn for endian-safe bitwise comparison, then use one cg::reduce call with greater<> comparator per top-K iteration to find both max value and winner index simultaneously, eliminating the separate butterfly reduction, ballot-based lane search, and index shuffle

## Key Lessons

- Pack value and index into a single integer word (upper bits for twiddled value, lower bits for inverted index) so a single warp reduction finds both the maximum value and its owner lane — eliminates separate reduction + ballot + shuffle for each top-K iteration
- Use cub::Traits<T>::TwiddleIn/TwiddleOut for endian-safe bitwise comparison of floating-point values in packed integer format — ensures correct ordering when comparing half/fp16 values as unsigned integers without float conversion
- Store the index as (kMaxIdx - idx) in the low bits so that when two packed words have equal value bits, the larger integer (smaller index via inversion) wins the greater<> comparison, providing a deterministic tiebreaker
