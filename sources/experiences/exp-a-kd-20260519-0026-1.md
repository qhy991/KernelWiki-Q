## Challenge

Two-kernel dispatch for split-KV attention epilogue requires a large temporary workspace (tile_size × B × split_kv) and an extra kernel launch, motivating fusion into a single kernel with in-place output accumulation


## Solution

Fused kernel uses per-element atomicAdd on the global output tensor for accumulation across split_kv CTAs, eliminating workspace and extra launch, but on sm_80 this causes severe serialization as 128 threads × 512 elements × 4 CTAs contend for 65536 output locations, regressing 100x — this pattern requires hardware TMA_REDUCE_ADD (sm_100) for efficiency

## Key Lessons

- Per-element atomicAdd on global memory for output accumulation across multiple CTAs causes severe serialization when contending for the same locations — use streaming write + separate reduction on current hardware, or hardware-accelerated TMA_REDUCE_ADD on sm_100
- Kernel fusion that eliminates workspace at the cost of introducing atomic contention can regress performance by 100x — measure on target hardware before deploying fusion patterns that replace streaming access with atomics
