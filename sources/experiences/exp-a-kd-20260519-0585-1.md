## Challenge

When K=16 < WARP_SIZE=32, the before kernel assigns one row per full 32-thread warp but only 16 lanes have valid k indices, leaving lanes 16-31 idle and wasting 50% of the warp's compute capacity


## Solution

Split each 32-thread warp into 2 sub-groups of 16 threads (K_WARP_SIZE=16) via lane index arithmetic (lane % K_WARP_SIZE for k, lane / K_WARP_SIZE for row offset), so each sub-group independently computes a dot product for a different row using a sub-warp reduction, doubling rows per warp

## Key Lessons

- When K < WARP_SIZE, use sub-warp tiling (lane % K_WARP_SIZE for element index, lane / K_WARP_SIZE for row offset) to pack multiple rows into one warp and eliminate idle lanes — applies to any GEMV/reduction kernel where the reduction dimension is smaller than the warp size
- Sub-warp reductions via warp_reduce_sum_f32<K_WARP_SIZE> with K_WARP_SIZE < 32 use fewer shuffle steps and naturally isolate each sub-group's partial sum, enabling independent row computations within a single warp
