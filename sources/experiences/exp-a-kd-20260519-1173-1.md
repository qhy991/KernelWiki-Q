## Solution

Move the max-abs-value reduction to the CPU via a sequential scan over the already-host-resident input array, compute the static scale on the host, and launch only the quant kernel on GPU — eliminating one full kernel launch, 128KB redundant device memory read, 4KB shared memory per block, and atomic contention

## Key Lessons

- For small tensors (~128KB) whose data is already on the host, computing a scalar reduction (max/sum) on CPU and passing the result as a kernel argument eliminates an entire GPU kernel launch, shared-memory allocation, and redundant device memory reads — the CPU scan costs microseconds versus the GPU kernel overhead
- When the reduction output is a single scalar (e.g., a quantization scale factor), a GPU parallel reduction with shared-memory trees and atomics is over-engineered; a host-side sequential scan avoids atomic contention, __syncthreads barriers, and multi-stage reduction complexity
