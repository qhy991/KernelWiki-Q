## Challenge

SiLU activation kernel processes all 8192 tokens unconditionally, loading gate/up float4 vectors, computing silu(gate)*up with expf, and storing results even for masked tokens whose outputs are discarded, wasting ~50% of global memory bandwidth and compute when half the tokens are inactive


## Solution

Add expert filtering via an early-return check on expert_ids[token_id / expert_step] == -1, so threads handling masked tokens skip all global memory loads (2×float4), silu+expf computation, and float4 stores; with BLOCK_SIZE=256 and num_vecs=256 mapping one token per block, masked blocks diverge at full-warp granularity with no intra-warp divergence

## Key Lessons

- Early-return filtering on a per-token predicate (e.g., expert_id == -1) is highly effective when the thread-to-token mapping aligns with CUDA block boundaries (one token per block), because all threads in a masked block take the same branch — zero intra-warp divergence and all global memory/compute for skipped tokens is eliminated
- Expert-filtering early-return scales sub-linearly to the theoretical maximum because the kernel launch grid still covers all tokens (same number of blocks); the speedup approaches the masked fraction (1.63x vs theoretical 2x at 50% masking) with overhead from grid launch and the expert_ids read per masked block
- When adding a filtering predicate to an existing kernel, extend the Params struct with the filter array and stride rather than embedding the filter in the kernel body — this keeps the filtering logic decoupled and reusable across different masking strategies
