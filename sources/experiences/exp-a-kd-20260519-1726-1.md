## Solution

Merge max and min into a single BlockReduce<MinMax,256> pass using a custom MinMax struct with compound operator, eliminating one __syncthreads() and one reduction traversal, but the simultaneous 4x block size reduction (1024 to 256 threads) decreases occupancy from 32 to 8 warps per block and doubles shared memory traffic per element (8-byte MinMax vs 4-byte float), causing net regression

## Key Lessons

- Merging two scalar reductions into one struct-of-scalars reduction eliminates a __syncthreads() barrier but increases per-element shared memory traffic proportionally to struct size — the net benefit depends on whether the saved barrier cost exceeds the extra memory movement
- Reducing block size when merging reductions can cause net regression when grid-level parallelism is limited (few blocks), because the lower per-block occupancy (fewer warps) reduces the SM's ability to hide memory latency during multi-phase kernels
- A MinMax struct (8 bytes) doubles the data moved through shared memory per CUB reduction step compared to a single float (4 bytes) — the merging optimization is most beneficial when the struct overhead is small relative to the eliminated barrier and second reduction pass
