## Solution

Fuse the per-element scale multiplication directly into the fp8-to-float conversion path by introducing scaled_vec_conversion, which converts fp8→half via __nv_cvt_fp8_to_halfraw, then to float, and multiplies by scale in the same function — eliminating the separate dequantization kernel pass entirely

## Key Lessons

- Fusing quantization scale multiplication into the type conversion function eliminates a separate dequantization kernel pass and its memory round-trip — applies whenever dequantization is followed by element-wise scaling in bandwidth-bound pipelines
- Prefer __nv_cvt_fp8_to_halfraw over chained vec_conversion calls for fp8→half conversion when a direct hardware intrinsic is available — fewer intermediate conversions and clearer intent
