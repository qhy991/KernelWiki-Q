## Solution

Use swap_ab strategy: compute C^T = B^T * A^T via cublasHgemm with CUBLAS_OP_T on both operands, making N=4096 the effective M dimension and eliminating all padding/slicing overhead — a single GEMM call replaces the padded pipeline

## Key Lessons

- swap_ab (CUBLAS_OP_T on both operands) eliminates padding overhead but regresses on SM80 (A800) because cuBLAS must internally transpose both input matrices via D2D copies — the implicit transpose cost exceeds the padding overhead it replaces
- On SM100 (Blackwell), the TCGEN05 MMA engine natively supports swap_ab through ScaleConfig major swapping without data transposition, so this technique would be beneficial on that architecture
- cuBLAS GEMM performance is highly sensitive to the combination of op flags and leading dimensions — CUBLAS_OP_N with padded aligned M can outperform CUBLAS_OP_T with natural dimensions when the hardware lacks native transpose support in the tensor core path
