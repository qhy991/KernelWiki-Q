## Solution

Materialize the vector pack into a register-backed local vin_t variable before passing it to the callable, which forces NVCC to emit one wide LD.global instruction (LDG.128 or LDG.32) instead of potentially 16 separate LDG.2 scalar loads, measurably reducing latency for memory-bound read-only kernels

## Key Lessons

- Materialize wide vector packs (≥16 elements) into local variables before passing to read-only callables — the local copy signals NVCC to emit a single wide LD.global instead of decomposing into multiple narrow loads, reducing memory transaction count for memory-bound kernels
- When iterating over reinterpret_cast'ed vector arrays, prefer vin_t tmp = v_in[i]; call(tmp); over call(v_in[i]); to decouple the load width from the callee's internal access pattern — applies whenever VEC_SIZE×sizeof(scalar_t) ≥ 16 bytes
