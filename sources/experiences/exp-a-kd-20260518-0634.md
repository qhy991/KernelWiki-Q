dot22_normal_range

## Solution

s plus 8 __half2float conversions for activations and 4 __low2float/__high2float decompositions for weights. This doubles the arithmetic instruction count and adds significant type-conversion overhead (half->float for each activation element, float->half for the final result via __float2half_rn). FP

## Key Lessons

- The after code decomposes each half2 multiply-add (__hfma2) into two separate FP32 fma operations (__low2float/__high2float extraction plus individual float multiply-adds).
