## Challenge

m16n8k16 BF16-F32 MMA inline asm uses uint32_t* pointers and "r" (integer register) constraints for the f32 accumulator (C) and result (D) operands, potentially forcing cross-bank register moves on older NVCC toolchains that do not optimize across register bank boundaries


## Solution

Change C/D operand pointers from uint32_t* to float* and asm constraints from "r" to "f", correctly annotating the f32 accumulator/result as float-register operands so the compiler allocates them directly into float registers without cross-bank moves

## Key Lessons

- Use "f" (float register) constraints and float* pointer types for f32 accumulator/result operands in mma.sync inline asm — the HMMA instruction natively operates on float registers, so correct annotation avoids potential cross-bank register moves on older toolchains
- Modern NVCC (12.x+) ptxas optimizer handles register bank allocation correctly regardless of constraint letter, producing identical SASS — this annotation fix is only materially beneficial on CUDA 11 and earlier toolchains where the compiler could insert unnecessary MOV instructions across register banks
