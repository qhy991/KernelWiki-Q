## Solution

Replace the shared-memory Blelloch scan with a two-level warp-shuffle scan using __shfl_up_sync for barrier-free intra-warp exclusive prefix sums, then aggregate across warps via a single warp's shuffle scan on warp_sums, reducing total __syncthreads from 16 to 7 by exploiting hardware-guaranteed warp lockstep execution

## Key Lessons

- Use __shfl_up_sync for intra-warp exclusive prefix sums instead of shared-memory Blelloch scan — warp lanes execute in lockstep so no __syncthreads is needed within a warp, eliminating O(log N) barriers per scan phase
- For scan sizes exceeding one warp (N > 32), use a two-level hierarchical approach: intra-warp shuffle scan for each warp's local prefix, then a single warp performs a shuffle scan on the per-warp aggregates — this reduces total __syncthreads from O(2·log N) to O(log(num_warps) + constant), yielding the largest benefit at small-to-medium scan sizes where barrier cost dominates
- Each eliminated __syncthreads saves the full cost of synchronizing all threads in the block — at small scan sizes (128 elements, 1024 threads) the barrier overhead dominates arithmetic, making barrier reduction the primary optimization lever
