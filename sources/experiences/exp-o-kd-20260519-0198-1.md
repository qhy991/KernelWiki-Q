## Challenge

Epilogue accumulator copy unconditionally reads from a global memory buffer even when K=0 (no MMA produced data), loading garbage from uninitialized memory into register fragments and writing incorrect values to output


## Solution

Add a K>0 guard (is_accumulator_needed) that branches between copying valid accumulator data from the buffer and zero-filling register fragments; when K=0 the zero-fill path avoids global memory loads entirely, producing correct identity-element output and reducing instructions

## Key Lessons

- Guard accumulator-buffer reads with a K>0 check in CUTLASS-style epilogues — when K=0 the MMA did not produce data, so the buffer is uninitialized and must be replaced with a zero-fill (the identity element for accumulation) to avoid garbage output
- Register-level zero-fill (reg_frag[i]=0.0f) is cheaper than a global-memory load-copy pipeline, making the K=0 fast path both correct and slightly faster — applies whenever a kernel conditionally consumes data that may not exist
- Treating the K=0 edge case as a distinct code path rather than a boundary condition inside a single loop eliminates branch divergence and avoids loading from potentially unmapped or uninitialized memory regions
