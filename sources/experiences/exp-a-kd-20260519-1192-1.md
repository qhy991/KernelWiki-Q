## Challenge

Simple bf16 element-wise copy kernel is refactored into a templated kernel with if constexpr branching and static_cast, raising the question of whether compile-time dispatch and type casting introduce runtime overhead


## Solution

Use C++17 if constexpr with template parameters and an enum class non-type parameter to dispatch between kAuto (identity static_cast) and FP8 conversion paths at compile time, eliminating any runtime branching overhead

## Key Lessons

- if constexpr with non-type template parameters (enum class) is resolved entirely at compile time in CUDA device code, generating identical PTX to the non-templated equivalent — no runtime branching or divergence overhead is introduced
- When scalar_t and cache_t are the same type (e.g., both __nv_bfloat16), static_cast<scalar_t>(src[i]) is an identity conversion that compiles to the same load-store instruction as direct assignment dst[i] = src[i]
