## Solution

Replace the generic dot<T> template with a specialized dot(float2,float2) overload that directly calls mul then sums c.x+c.y without the intermediate sum() function, but nvcc fully inlines both paths into identical SASS (two FMUL + one FADD), resulting in no measurable performance difference

## Key Lessons

- Marking __device__ __inline__ helper functions (mul, sum, dot) allows nvcc to fully inline the entire call chain at compile time, producing identical SASS regardless of whether a generic template path or a hand-specialized overload is used — template dispatch adds zero runtime overhead when all layers are inlined
- A <1% timing delta (0.36% here) is within GPU measurement noise and should be treated as neutral; specialize for code clarity rather than expected performance gains when the compiler can already inline everything
