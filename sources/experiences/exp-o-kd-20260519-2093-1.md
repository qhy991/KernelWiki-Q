## Solution

Remove the smemLogits shared memory buffer entirely and skip all logit writes to shared memory during histogram-based candidate population and final-sort merge, reducing per-block shared memory by ~8KB and cutting shared memory write traffic by nearly half

## Key Lessons

- Removing unnecessary shared memory buffers (e.g., smemLogits when only indices are needed in output) reduces per-block shared memory by kilobytes, directly improving occupancy — applies whenever intermediate logit storage can be omitted because downstream consumers only need indices
- Eliminating shared memory write traffic during data-intensive loops (histogram population) halves memory bandwidth consumption for the affected phase, as each thread skips one smem write per iteration when the stored value is never read again
