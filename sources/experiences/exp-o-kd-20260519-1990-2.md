## Solution

Use 8-byte vector loads (BYTES_PER_LDG=8, ELTS_PER_LDG=2) so that THREADS_PER_ROW becomes 32 (a power of 2 equal to warp size), enabling warp-level butterfly shuffle reductions via __shfl_xor_sync that operate entirely in registers without shared memory or __syncthreads

## Key Lessons

- Use vector loads (BYTES_PER_LDG=8 for float) to make THREADS_PER_ROW a power of 2 that divides the warp size, enabling __shfl_xor_sync reductions — for 192 experts, 8-byte loads give THREADS_PER_ROW=32 (=warp size), making warp-level reduction possible where scalar loads would give a non-power-of-2 thread count
- Warp-level butterfly shuffle (__shfl_xor_sync) replaces CUB BlockReduce with zero shared memory and zero __syncthreads — applicable when THREADS_PER_ROW is a power of 2 and ≤ warp size, eliminating shared memory bank conflicts and barrier stalls
