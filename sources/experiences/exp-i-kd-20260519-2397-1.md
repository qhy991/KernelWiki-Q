## Solution

Use blockIdx.z for batch dimension offset with per-batch base pointers, grid-stride loop for flexible work distribution, FastDivmod to decode flat index k into (row i, column j), then scatter input[k] to output[i + func(j) * stride] where func maps the column index to the target output column

## Key Lessons

- Use cutlass::FastDivmod to replace expensive integer division/modulo with multiply-shift when decoding flat indices into multi-dimensional coordinates — applies to any kernel that linearizes 2D+ access patterns
- Grid-stride loops (k += blockDim.x * gridDim.x) let a kernel process more elements than total threads, making launch configuration flexible and robust regardless of grid size
- Use blockIdx.z as the batch dimension and compute per-batch base pointers (input + batch * num_elems) at kernel entry — extends scatter/gather kernels to batched inputs without adding parameters
