## Solution

Add a mask array parameter that specifies the number of valid rows per expert, and use `if (rowIdx_in_expert >= mask[expert_idx]) continue` to skip padding rows — however, this introduces warp divergence and an extra global memory read per iteration that outweighs computation savings for lightweight quantization kernels

## Key Lessons

- Mask-based early exit via `continue` causes warp divergence when threads within the same warp hit different branches (~50% skip vs ~50% proceed), and since warps execute in lockstep, total time is dominated by the longest path — this negates per-thread savings for lightweight compute kernels
- Early-exit optimizations are beneficial only when the skipped computation cost substantially exceeds the divergence overhead plus the extra mask memory read — real FP4 quantization with PTX-level cvt instructions, warp reductions, and activation fusion has sufficient per-row cost to justify the mask check, but simplified absmax+scale+pack kernels do not
- Adding a per-iteration global memory read (mask[expert_idx]) in memory-bandwidth-bound kernels with lightweight per-element work can offset or exceed the computation savings from skipping rows
