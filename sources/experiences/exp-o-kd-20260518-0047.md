## Challenge

The before source uses kStages=4 with 64x64 half tiles, allocating 4*64*64*2=32768 bytes of shared memory per block. The after source reduces to kStages=1, needing only 64*64*2=8192 bytes. This mirrors the real optimization: passing kMaxK=64 to MakeCustomMma computes kStages=min(4, ceil(64/64))=1. T


## Solution

factors. First, the occupancy effect: 32768 bytes/block limits occupancy to ~5 blocks/SM, while 8192 bytes/block allows ~20 blocks/SM, quadrupling the number of active warps that can hide memory latency.
