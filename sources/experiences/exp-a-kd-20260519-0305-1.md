## Solution

Fuse the activation and scale kernels into a single kernel that reads GEMM output, applies gated SiLU activation, multiplies by per-expert prequant scale, and writes the final result in one pass — eliminating the intermediate temp buffer write/read and the second kernel launch entirely

## Key Lessons

- Fusing two dependent kernels (activation + scale) that share the same token×column data into a single pass eliminates one full-tensor global memory round-trip and one kernel launch — applies whenever kernel B reads the exact output of kernel A elementwise without cross-thread reduction
- Eliminating the intermediate temp buffer reduces memory pressure and improves cache utilization for adjacent work, especially when the activation tensor is large (4096×2048 fp32 = 32 MB)
