## Challenge

GELU array transformation loop uses int(rhs.size()) as its bound, calling a member function that returns the template parameter N instead of using the compile-time constant directly, which adds a redundant runtime indirection for what is a static value


## Solution

Replace the runtime rhs.size() call with the compile-time template parameter N as the loop bound, eliminating the member function call and making the unrolling hint directly visible to the compiler as a constant expression

## Key Lessons

- Using a compile-time template parameter directly as a loop bound instead of wrapping it in a member function like size() is a cleaner pattern, but modern NVCC at -O3 fully inlines trivial getters returning constants, so the SASS output is identical — the optimization is stylistic rather than performance-relevant
- When a kernel is memory-bandwidth-bound (as is typical for elementwise activation functions like GELU over contiguous arrays), even small instruction-count differences in the compute path are hidden by DRAM latency and cannot produce measurable speedups
