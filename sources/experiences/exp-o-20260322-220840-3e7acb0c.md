## Challenge

Scalar float loads require 4x more memory transactions than necessary for 128-element rows


## Solution

Vectorized float4 loads/stores


### Key Changes

- Replace 4 scalar __ldg() calls with single float4 load
- Replace 4 scalar stores with single float4 store
- Maintain warp reduction for sum-of-squares


## Key Lessons

- Vectorized loads reduce memory transaction overhead for contiguous data
- float4 is optimal for 128-element rows with 32 threads/warp and 4 elements/thread
- 8% improvement shows memory bandwidth was already near-optimal in baseline
