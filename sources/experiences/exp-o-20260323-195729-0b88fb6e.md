RMS Norm LLaMA-3-8B hidden_size=4096

## Solution

Register caching optimization: load input float4 values into registers during reduction pass, reuse them in output pass to eliminate second global memory read. Use 256 threads with 4 unrolled iterations (VEC_COUNT=1024, 256 threads -> 4 iters per thread). __launch_bounds__(256, 4) for better occupancy.

## Key Lessons

- Register caching of input data eliminates second global memory read, significantly improving bandwidth efficiency for RMS norm
- 256 threads with 4 unrolled iterations outperforms 512 threads for HIDDEN_SIZE=4096 (better occupancy)
- __launch_bounds__(256, 4) hints compiler to keep 4 blocks per SM, improving occupancy
- For RMS norm with fixed hidden size, fully unrolled loops with register arrays are very effective
- The reduction + output pattern benefits greatly from caching: read input once, compute norm, write output using cached values
