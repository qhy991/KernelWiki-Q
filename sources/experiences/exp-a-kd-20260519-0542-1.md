## Solution

Replace scalar half loads with half2 vectorized loads, processing 4 half elements per thread per iteration via paired HALF2 loads (reg_x_0, reg_x_1, reg_a_0, reg_a_1), reducing loop iterations from 8 to 2 and issuing 64-bit loads that better utilize L2 cache line bandwidth

## Key Lessons

- Vectorize HGEMV inner-loop loads from scalar half to half2 (f16x4 per thread) to reduce loop iterations by 4x and issue wider 64-bit memory transactions that better saturate L2 cache line bandwidth — applies to any GEMV/GEMM dot-product loop with contiguous half-precision operands
- Reducing loop iteration count in the K-dimension directly cuts per-thread overhead (branch evaluation, index arithmetic, boundary checks), which becomes significant when the arithmetic intensity per iteration is low as in GEMV
