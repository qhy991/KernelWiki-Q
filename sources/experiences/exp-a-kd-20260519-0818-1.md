## Solution

Add #pragma unroll 1 to both head iteration loops to explicitly prevent compiler unrolling, serving as a defensive measure against over-unrolling when loop bounds become compile-time constants via template parameters or macro dispatch

## Key Lessons

- #pragma unroll 1 has no effect when loop bounds are runtime kernel parameters — NVCC at -O3 does not aggressively unroll loops with variable bounds, so the compiler already preserves loop structure and the pragma is redundant
- #pragma unroll 1 becomes meaningful when loop bounds are compile-time constants (template parameters or macro-dispatched values like DISPATCH_HEAD_DIM), where the compiler may fully unroll and cause register spilling — apply it defensively in template-specialized kernels where head counts is a compile-time value
