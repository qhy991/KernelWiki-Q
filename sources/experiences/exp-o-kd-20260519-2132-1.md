## Solution

Accumulate match counts in a register-local variable across all loop iterations, then issue a single atomicAdd per thread after the loop — reducing atomic operations from O(topk_length) to O(THREADS_PER_EXPERT) per expert and eliminating inter-thread serialization on the shared counter

## Key Lessons

- Replace conditional atomicAdd inside loops with register-local accumulation followed by a single atomicAdd after the loop — reduces atomic contention from O(N) to O(T) where T is thread count, applicable whenever threads redundantly update the same counter
- Use implicit int conversion from boolean comparison (topk_ids[i] == expert_id) as a branchless increment to avoid warp divergence in counting loops
