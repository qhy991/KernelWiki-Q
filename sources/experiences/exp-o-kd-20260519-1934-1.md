## Solution

Adaptive heuristic selects YTILE=1 and UNRL=4 for sYT=1 workloads: YTILE=1 eliminates the per-iteration row loop, and UNRL=4 doubles the K-loop stride from 64 to 128 elements per thread, halving iterations from 4 to 2 and improving instruction-level parallelism through more consecutive independent loads

## Key Lessons

- For wavefront-parallel reduction kernels, match YTILE/UNRL to the rows-per-wavefront ratio: when sYT=1 (few rows per wavefront), prefer YTILE=1 with higher UNRL to eliminate row-loop overhead and maximize per-thread work
- Increasing the K-loop unrolling factor (UNRL) reduces loop iteration count and improves instruction-level parallelism by enabling the compiler to schedule more consecutive independent loads per iteration — effective when the working set fits in registers
- Adaptive heuristic tuning based on workload geometry (sYT = ceil(M / (CuCount * 4))) outperforms hardcoded parameter choices because the optimal YTILE/UNRL trade-off depends on the ratio of rows to available compute units
