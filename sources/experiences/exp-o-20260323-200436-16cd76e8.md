## Challenge

Tried to improve beyond 291.6% ratio for rms_norm with hidden_size=4096 on A800 GPU


## Solution

256 threads, 4 float4 per thread, register caching of input (eliminates second global read), warp reduction with shared memory

## Key Lessons

- 256 threads with 4 float4/thread is the sweet spot for rms_norm hs=4096 on A800 - balances occupancy and register pressure
- Reducing to 128 threads with 8 float4/thread increases register pressure and hurts performance (264.7% vs 291.6%)
- cp.async pipeline adds overhead for small rows (4096 floats) - shared memory staging is slower than register caching (269.2% vs 291.6%)
- Streaming stores (__stcs) for output match but don't improve over regular stores (291.6% tie)
- __ldg() for weight loads doesn't measurably improve performance when combined with register caching
- The kernel hits a performance wall at ~291.6% with this register-caching approach - further gains likely require algorithmic changes like persistent kernels or multi-row parallelism
