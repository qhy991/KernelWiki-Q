## Solution

Fuse the entire backward pass into a single CUTLASS-based kernel that tiles over query/key blocks, computes the Di row reduction inside the kernel via computeDi into shared storage, and immediately feeds that state into processBlockIJ to update grad-V, grad-Q, and grad-K without materializing any global intermediate

## Key Lessons

- Fusing separate backward kernels that communicate through global-memory intermediates into a single kernel that computes the intermediate (Di) in shared memory/registers eliminates a full write/read round-trip and one kernel launch — applies whenever two passes share a per-row reduction that the second pass immediately consumes
- Tile-structured iteration (kBlockSizeI × kBlockSizeJ) with explicit remainder handling (processBlockIJ<false> for partial tiles) lets a single fused kernel cover all problem dimensions without requiring that M and N be divisible by tile sizes, removing the conditional launch branches of the original two-kernel approach
