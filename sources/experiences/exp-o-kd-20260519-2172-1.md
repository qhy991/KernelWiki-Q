## Solution

Replace the integer zero-point path with a direct fp16 zero-point path where ZP values are already stored in fp16 format (loaded via vectorized int4 into frag_zpf registers), eliminating all per-iteration unpacking (shift+mask) and int-to-fp16 type conversion arithmetic from the inner loop

## Key Lessons

- Precompute quantized zero-points into fp16 format (e.g., via int4 vectorized load into frag_zpf registers) instead of dequantizing packed integers inside the inner loop — eliminates per-iteration shift+mask+__float2half overhead with a one-time layout conversion
- Moving dequantization out of the inner k-loop and into a pre-processing or register-loading phase reduces instruction count proportionally to loop iterations — applies whenever quantized metadata is invariant across j-iterations
