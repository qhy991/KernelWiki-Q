## Solution

Pass scale_val as a scalar kernel argument by value instead of a global memory pointer, and move the reciprocal division (1.0f / scale) to the host side before kernel launch, so the precomputed value is delivered via the kernel parameter buffer which the GPU hardware broadcasts efficiently to all threads through constant cache or register allocation

## Key Lessons

- Pass read-only scalar parameters by value as kernel arguments instead of global memory pointers — kernel arguments are stored in the parameter buffer and broadcast via constant cache, eliminating per-thread global loads for values uniform across all threads
- Move invariant arithmetic (e.g., reciprocal division) from inside the kernel to the host side before launch — reduces per-thread compute overhead to a single host-side operation and makes the constant immediately available through the parameter block
- Scalar kernel argument passing is most impactful for small-tensor kernels where global load overhead constitutes a larger fraction of total runtime (1.081x at 64K elements vs 1.004x at 4M elements)
