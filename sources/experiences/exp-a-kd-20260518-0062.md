## Solution

Eliminate the source tensor load entirely when OnlyAlphaScaling by using a compile-time predicate (is_source_needed() returns false), replacing the beta*source + alpha*accum pipeline with a direct alpha*accum computation via a dedicated mul_accum function

## Key Lessons

- Dead-load elimination (skipping source tensor read when beta=0) is a valid bandwidth optimization but its measurable benefit depends on the eliminated traffic being a significant fraction of total DRAM access — when the working set fits entirely in L2 cache, the savings may be negligible
- At small problem sizes (256K floats = 1MB) kernel launch overhead dominates execution time at the ~0.1ms scale, making memory-bandwidth optimizations difficult to observe and potentially masked by measurement noise
- Compile-time predicate branching (is_source_needed() evaluated at compile time) cleanly eliminates dead code paths without runtime cost, but the performance impact is only realized when the eliminated path involves significant DRAM traffic relative to the kernel's total execution time
