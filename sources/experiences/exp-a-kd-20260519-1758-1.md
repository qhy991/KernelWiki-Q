## Solution

Replace the hardcoded alignment constant with a computed alignment derived from the element type using 128 / sizeof_bits<ElementAB>::value, adding a sizeof_bits type trait and static_assert checks to validate correctness for int8_t (16), half (8), and float (4)

## Key Lessons

- Compute alignment from the element type (128 / sizeof_bits<T>::value) rather than hardcoding it — the correct TMA vector width depends on element size, and a wrong constant silently produces incorrect transaction widths for any type other than the one assumed during hardcoding
- Use static_assert to validate computed template constants against known expected values for each supported type — catches alignment miscalculations at compile time rather than producing subtle runtime memory-access bugs
