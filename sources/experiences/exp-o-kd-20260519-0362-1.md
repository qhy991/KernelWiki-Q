## Challenge

Scalar FP16 ELU kernel processes one half element per thread using separate 16-bit load and store instructions, which fail to fill the memory bus width and waste bandwidth, while requiring N threads that increase scheduling overhead


## Solution

Replace scalar half loads/stores with half2 (32-bit) vector loads/stores so each thread processes 2 half elements in a single 32-bit memory transaction, doubling per-thread throughput and improving coalescing

## Key Lessons

- half2 vectorization doubles per-thread throughput for FP16 elementwise kernels by coalescing two half elements into one 32-bit memory transaction — applies to any half-precision elementwise operation with contiguous memory access
- Vectorized kernels may regress at small problem sizes (1M elements: 0.83x) due to reduced occupancy from halved thread count; benefit emerges at large sizes where memory bandwidth dominates (16M elements: 1.44x)
