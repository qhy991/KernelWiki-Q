## Challenge

CausalFromBottomRight index computation uses key_start - num_keys - num_queries (double minus), which incorrectly calculates the first query position for the bottom-right causal diagonal, producing wrong attention masking results for most valid configurations


## Solution

Fix the sign error by changing the formula from key_start - num_keys - num_queries to key_start - num_keys + num_queries, which correctly positions the bottom-right causal diagonal so that a key at position key_start first attends to queries starting at key_start - num_keys + num_queries

## Key Lessons

- In CausalFromBottomRight attention, the first_query formula must use key_start - num_keys + num_queries (plus sign on num_queries) to correctly align the causal boundary — a minus sign error causes the bottom-right diagonal offset to shift, silently masking or unmasking incorrect query-key pairs
- When implementing block-sparse causal attention patterns, verify index formulas against concrete cases (e.g., key_start=64, num_keys=128, num_queries=128 should yield first_query=64, not 0) to catch sign errors that pass compilation but produce wrong results
