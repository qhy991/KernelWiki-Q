## Solution

Fuse RMSNorm and FP8 quantization into a single kernel that computes the normalized value and immediately quantizes it to FP8 in the same loop iteration, eliminating the intermediate __half buffer and one kernel launch entirely

## Key Lessons

- Fuse consecutive kernels where one produces an intermediate that the next immediately consumes — the fusion eliminates global memory round-trips for the intermediate buffer and removes one kernel launch, both of which are disproportionately costly at small batch sizes
- Kernel launch overhead (~5us per launch) becomes the dominant cost when per-kernel computation is small (few blocks, light work), making fusion especially valuable for low-batch inference paths
- When the intermediate type differs from the final output type (e.g., __half intermediate → FP8 final), fusion lets the compiler keep the normalized value in registers and convert directly to the target type without ever materializing the intermediate in global memory
