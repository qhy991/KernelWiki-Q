## Challenge

In a multi-group tiled kernel, each group resets the CTA starting offset to blockIdx.x, so when total_tiles per group is not divisible by gridDim.x, the same CTAs always absorb the remainder tiles, producing a skewed work distribution ([12,12,12,12,8,8,8,8] with stddev=2.0) and causing tail synchronization imbalance


## Solution

Track the cumulative tile count across groups via group_total_tiles, and rotate each CTA's starting offset by (gridDim.x - head_cta_id) where head_cta_id = group_total_tiles % gridDim.x, so the remainder tiles alternate between different CTAs across groups and produce a perfectly uniform distribution

## Key Lessons

- Rotate the CTA starting offset across groups by accumulating the total tile count modulo gridDim.x, so remainder tiles distribute to different CTAs in each group instead of always going to the same low-indexed CTAs — eliminates load imbalance when per-group tile counts are not divisible by gridDim.x
- In multi-group tiled kernels, track cumulative tile counts across groups to compute head_cta_id = cumulative_tiles % gridDim.x, then shift each CTA's starting offset by (gridDim.x - head_cta_id) to achieve perfect work balance across all groups
