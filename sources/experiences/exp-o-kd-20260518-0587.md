## Challenge

In the before code, the GEMM uses 256x256 tiles with a 4x4 cluster (16 SMs per CTA), which is massively over-provisioned for M=8. Most threads in the 256-wide M tile have no valid data to process, and the 16-SM cluster coordination overhead is entirely wasted


## Solution

The after code switches to 128x128 tiles with a 1x1 cluster (single SM), matching the small problem geometry. This eliminates wasted thread lanes and cluster synchronization overhead, yielding faster execution for small M.

- **speedup**: 1.713x (improved)
