## Solution

Generalize the shared expert handling by replacing the binary conditional with direct subtraction of num_fused_shared_experts for topk count, and replacing the round-robin single-slot assignment with sequential offset starting from 0 plus a loop that fills all shared expert slots when num_fused > 1

## Key Lessons

- Use the actual variable value (num_fused_shared_experts) directly in arithmetic rather than a binary conditional (num>0?1:0) — the conditional hardcodes an assumption that the count is at most 1, which breaks silently when the count exceeds 1
- When assigning shared expert slots, use sequential offset with a loop over all slots rather than round-robin modulo (row%num_experts) — modulo produces non-deterministic routing across rows and only fills one slot per row when a single assignment is made outside a loop
- Generalization fixes from fixed-count to variable-count logic often produce no measurable performance change for the original count value (num_fused=1) but are essential for correctness at larger counts — validate with multiple count values, not just the default case
