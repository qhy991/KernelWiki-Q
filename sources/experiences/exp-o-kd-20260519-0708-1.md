## Solution

Replace per-warp subset scanning with direct one-thread-per-expert mapping where each thread reads its own single expert value and participates in warp-level shuffle reduction, eliminating the inner scanning loop entirely; add #pragma unroll on fixed-iteration shuffle reductions for better instruction scheduling

## Key Lessons

- When total thread count matches or exceeds data elements (384 threads for 384 experts), assign one element per thread and use warp shuffle reduction instead of having warps iterate through subsets — eliminates loop overhead and reduces strided shared memory bank conflicts
- Adding #pragma unroll to fixed-iteration-count shuffle reduction loops (offset halves from 16 to 1, exactly 5 iterations) lets the compiler fully unroll into independent instructions, improving instruction-level parallelism on shuffle-heavy reduction patterns
