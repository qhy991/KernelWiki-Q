## Solution

Merge the two separate kernels into a single templated kernel (concat_and_cache_mla_kernel) that performs one slot_mapping lookup then uses a device-side lambda to copy both kv_c and k_pe sequentially, eliminating one full kernel launch overhead and all duplicated index computation

## Key Lessons

- Fuse lightweight kernels that share the same index computation (slot_mapping lookup, block decomposition) into a single launch — kernel launch overhead (~5-10us per dispatch) dominates execution time when per-token data volumes are small
- Use device-side lambdas to eliminate code duplication when multiple copy phases differ only in source pointer, stride, size, and destination offset — the lambda parameterizes the varying parts while sharing the common index logic
- Kernel fusion yields the largest speedups when the arithmetic intensity is low (simple strided copies) and launch overhead is a significant fraction of total runtime — the 3.24x improvement here confirms that removing one launch from a pair of memory-bound kernels roughly halves the fixed overhead
