## Challenge

Blelloch exclusive scan uses 2*log2(N) __syncthreads() barriers with shared-memory reads/writes at every step, requiring 12 barrier calls for scan_size=64 and creating significant synchronization and memory round-trip overhead.


## Solution

Replace Blelloch up-sweep/down-sweep with a two-level warp shuffle approach: compute intra-warp exclusive prefix sums via __shfl_up_sync (no shared memory or barriers within each warp), then aggregate inter-warp prefixes using warp0, reducing total __syncthreads() calls from 12 to 5.

## Key Lessons

- Use warp shuffle (__shfl_up_sync) for intra-warp prefix scans instead of shared-memory Blelloch — eliminates shared-memory round-trips and intra-warp __syncthreads() barriers, reducing synchronization overhead by ~60% for scan sizes that fit within a few warps.
- Two-level hierarchical scan (intra-warp shuffle scan + inter-warp aggregation via one designated warp) trades slightly higher register pressure for significantly fewer __syncthreads() calls — applicable whenever the scan problem decomposes cleanly across warp boundaries.
