## Solution

Fuse SiLU(gate)*up activation and FP4-quantization proxy into a single kernel: load gate and up vectors directly from the doubled-width input, compute compute_silu_mul in registers, and pass the result immediately to quant_proxy_fp16 without any intermediate global memory write, eliminating one kernel launch and the 512KB intermediate buffer round-trip

## Key Lessons

- Fuse producer-consumer kernel pairs where the intermediate data fits in registers — eliminates one kernel launch and a full global memory round-trip, yielding significant speedup at small-to-medium problem sizes where launch overhead is a large fraction of total time
- Kernel fusion is most impactful when the intermediate tensor is large relative to compute: 512KB of activation data written then immediately read back wastes bandwidth that register-passing avoids entirely
- When fusing kernels with different access patterns, unify indexing by computing both input offsets from the doubled-width layout (gate||up) in a single loop iteration, computing the output offset separately from the original width
