## Challenge

GELU activation in half precision uses erff (error function) which is a multi-instruction SFU call with high latency that cannot be efficiently pipelined, making the compute-bound kernel bottleneck on the expensive erf evaluation per element


## Solution

Replace the expensive erff SFU call with a GELU Taylor approximation using the single-cycle tanh.approx.f32 PTX hardware instruction via inline asm, converting GELU(x) = 0.5x(1 + tanh(sqrt(2/π)(x + 0.044715x³))) into a short multiply-add chain followed by the fast hardware tanh

## Key Lessons

- Replace erff-based GELU with tanh-approximation GELU using the tanh.approx.f32 PTX instruction — the hardware tanh is a single-cycle operation versus erff's multi-cycle SFU path, yielding ~2x speedup on compute-bound half-precision kernels
- Use inline asm volatile with tanh.approx.f32 to invoke the hardware tanh approximation directly, guarded by __CUDA_ARCH__ for host compatibility — applies to any activation requiring erf or sigmoid-like curves where slight numerical difference is acceptable
- GELU Taylor form 0.5x(1 + tanh(sqrt(2/π)(x + 0.044715x³))) is numerically close to the exact erf form and the additional FMA instructions for the polynomial are well-pipelined, so the net per-element cost drops dramatically when tanh is cheap
