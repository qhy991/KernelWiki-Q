## Challenge

W4A32C8 quantized GEMM kernel was producing incorrect results (NMSE failures) in previous rounds. The test runner was also failing with ModuleNotFoundError for torch.


## Solution

1) Fixed the algorithm to implement W4A32C8 correctly: quantize FP32 activations to Q8_1 per-block, compute integer dot product sumi, apply formula. 2) Used /home/qinhaiyan/anaconda3/bin/python3 for test runner. 3) Added shared memory activation caching (TILE_N=8 warps share one activation row) to improve performance from 6.8% to 32% of baseline.

## Key Lessons

- W4A32C8 formula is: result = d4_0 * (d8_1 * sumi - 8 * s8_1) where s8_1 = sum of FP32 activation values in the block
- The test runner must be invoked with a Python that has torch installed: /home/qinhaiyan/anaconda3/bin/python3
- Shared memory activation caching (loading activation row once per block, shared by TILE_N warps) gives 4.7x speedup over naive per-warp global memory reads
- For K=4096 FP32 = 16KB, fits in shared memory with TILE_N=8 or 16 warps per block
- The entry function name for quant_gemm W4A32C8 is gemm_q4_0_q8_1_forward (auto-detected from first extern C symbol)
