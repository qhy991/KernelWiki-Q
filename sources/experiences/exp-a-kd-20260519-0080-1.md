## Solution

Replace modulo/division index computation with direct multiplication (index = iter_idx * MMA_N) and switch fragment layout from column-major to row-major so produce and consume loops use the same indexing, eliminating the apparent strided access pattern

## Key Lessons

- Modulo and division on compile-time-constant expressions (constexpr template parameters) are fully constant-folded by NVCC, producing identical PTX to direct multiplication — source-level arithmetic simplification has zero runtime benefit when operands are constants
- Register file access is not banked like shared memory, so fragment layout reorganization (column-major vs row-major) in small register arrays (8 elements) produces no measurable access latency difference
- When constexpr parameters satisfy equality constraints (ACC_R == MMA_R), the modulo always yields 0 and the division always yields iter_idx — the compiler eliminates these operations entirely, making source-level simplification a readability improvement rather than a performance optimization
