fp32_rms_norm_llama3_8b_hs4096

## Solution

Strip torch/ATen/c10 headers and PYBIND11_MODULE from baseline. Keep all existing optimizations: float4 vectorization (128-bit loads), warp shuffle reduction (__shfl_down_sync), __ldg read-only cache, __fdividef fast division. Add extern C wrapper with 3-variant dispatch: 128 threads for batch<8, 256 for batch 8-31, 512 for batch>=32.

## Key Lessons

- For rms_norm tasks, float4 vectorization + warp shuffle reduction is highly effective and already present in baselines
- The baseline transplant itself (removing torch headers, adding extern C) can yield 2x+ improvement over the reference baseline
- 3-variant thread-count dispatch (128/256/512) covers latency-sensitive small batches and throughput-oriented large batches
- Always verify >2x improvements with a second run to confirm stability - both runs showed identical 217.5%
- rms_norm entry point must be named exactly 'rms_norm_kernel' for the auto-wrapper to link correctly
