## Challenge

Per-token group quantization kernel and its downstream consumer kernel execute sequentially on the same stream with no mechanism for inter-kernel execution overlap, leaving SM resources idle during kernel transition boundaries on Hopper (SM90+) GPUs


## Solution

Add PDL (Programmatic Dependent Launch) barriers — cudaGridDependencySynchronize() at kernel entry and cudaTriggerProgrammaticLaunchCompletion() at kernel exit, guarded by #if __CUDA_ARCH__ >= 900 — combined with cudaLaunchKernelEx and cudaLaunchAttributeProgrammaticStreamSerialization on the host side to enable hardware-level inter-kernel pipelining on SM90+ Hopper GPUs

## Key Lessons

- PDL (Programmatic Dependent Launch) barriers enable inter-kernel execution overlap on SM90+ Hopper GPUs where the hardware can begin a consumer kernel before the producer fully completes — applies to any pipeline of dependent kernels on the same stream, but requires SM90+ hardware to have any effect
- Guard PDL-specific intrinsics (cudaGridDependencySynchronize/cudaTriggerProgrammaticLaunchCompletion) with #if __CUDA_ARCH__ >= 900 so they compile as dead code on pre-Hopper architectures, ensuring single-source portability with zero runtime overhead on unsupported hardware
- PDL requires both kernel-side barriers AND host-side cudaLaunchKernelEx with cudaLaunchAttributeProgrammaticStreamSerialization — omitting either half silently disables the mechanism with no error or fallback
