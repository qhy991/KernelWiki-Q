## Solution

Remove the entire TopKSort stage including register arrays, smemLogits loads, cub::BlockRadixSort call, and outLogits global writes; replace with a direct strided store from smemIndices to outIndices, eliminating ~8KB+ shared memory temp storage and 8 registers per thread

## Key Lessons

- Eliminate sorting when the downstream consumer does not require sorted output — removing an unnecessary cub::BlockRadixSort in top-K kernels saves shared memory temp storage (~8KB+), register pressure, and compute overhead for the single largest speedup in the kernel
- Direct shared-memory-to-global store with stride indexing avoids intermediate register arrays when no transformation is needed between smem and gmem — cuts both register pressure and unnecessary load/store instructions
