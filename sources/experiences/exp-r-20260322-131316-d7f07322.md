KernelEvalPlus AutoResearch Round 2: RN_04 fp32_rms_norm_llama3_8b_hs4096

## Key Lessons

- Reducing thread count from 512 to 256 can improve occupancy for memory-bound kernels
- rsqrtf() is faster than fdividef(1.0f, sqrtf()) for inverse square root
- #pragma unroll 4 helps compiler optimize loops with float4 vectorization
- __launch_bounds__(256, 4) hints compiler to keep 4 blocks per SM for better occupancy
- Never include cudaDeviceSynchronize() in extern C entry function - it measures sync overhead not kernel time
