## Challenge

GPTQ 8-bit repack kernel uses a two-step shared memory access pattern: first reading 8 values into b1_vals/b2_vals local arrays (8 uint32_t registers), then extracting elements via a two-index scheme (cur_int_local + cur_pos), wasting registers and instructions on intermediate storage


## Solution

Eliminate the b1_vals/b2_vals intermediate local arrays by computing packed indices (cur_n_packed = cur_n/pack_factor, cur_n_pos = cur_n%pack_factor) once outside the loop and reading directly from shared memory in a single step, also reducing sh_stride from 64 to 16 (tile_n_size/pack_factor) for better bank-conflict behavior

## Key Lessons

- Eliminate intermediate local arrays in shared memory processing loops by precomputing packed indices (offset + bit-position) once — this collapses a two-step read-then-extract pattern into a single direct-read step, reducing register pressure and removing an entire unroll loop, with greater benefit when tile_ints is larger (8-bit with 4 ints vs 4-bit with 2 ints)
- Derive sh_stride from data geometry (tile_n_size/pack_factor = 16) rather than hardcoding to the full tile width (64) — a smaller stride reduces the distance between accessed rows in shared memory, improving bank-conflict behavior for column-like access patterns
