## Challenge

The expert_ids fill loop is restricted to only num_experts (256) threads each iterating over their own assigned range, leaving the remaining 768 threads in a 1024-thread block idle


## Solution

Fill expert_ids cooperatively across all 1024 threads using binary search into the prefix sum array — each thread handles multiple blocks by iterating with stride blockDim.x, mapping block indices to expert IDs via O(log N) binary search over the cumsum boundaries

## Key Lessons

- When a per-element output loop is limited to N threads but blockDim.x >> N, restructure as a cooperative stride loop where all threads participate — binary search over the prefix sum array maps output indices to source IDs in O(log N) per element
- Binary search into a sorted prefix-sum boundary array is an efficient inverse mapping technique: given a position, find which segment it belongs to without requiring each segment's owner thread to iterate
