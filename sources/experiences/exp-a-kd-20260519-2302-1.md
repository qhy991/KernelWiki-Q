## Solution

Generalize FP8-specific template names to type-agnostic quant_type_max_v (supporting Float8_e4m3fn, Float8_e4m3fnuz, and int8_t via SFINAE) and replace the local const min_scaling_factor with a template struct min_scaling_factor<T>::val() annotated C10_ALWAYS_INLINE, enabling cross-type reuse while nvcc constant-folds both forms to identical SASS

## Key Lessons

- Renaming FP8-specific template variables to type-agnostic names (quant_type_max_v) with SFINAE enable_if supports multiple quantization types (Float8_e4m3fn, Float8_e4m3fnuz, int8_t) without any performance change — nvcc constant-folds both local const and constexpr template returns to the same immediate value
- C10_ALWAYS_INLINE (__forceinline__) on a constexpr-valued static method does not alter generated SASS versus a local const float — both compile to the same constant-folded immediate, so the annotation serves documentation and cross-type dispatch consistency rather than performance
