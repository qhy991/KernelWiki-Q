## Challenge

Each thread processes a single FP8 byte using scalar fp8_to_bf16_scalar conversion with divergent if/else branches: lanes 0-511 handle dequant while lanes 512-575 handle RoPE copy, leaving most lanes idle during each branch path


## Solution

Replace scalar per-byte dequant with vectorized int4 loads where each lane loads 16 FP8 bytes and converts them to bf16_8_t in batch, while RoPE copy uses int-width loads so all 32 lanes uniformly process both data sections without branch divergence

## Key Lessons

- Vectorize FP8-to-bf16 dequant using int4 loads (16 bytes per lane × 32 lanes = 512 bytes) instead of scalar byte loads — 16x fewer load instructions and full warp participation eliminates branch divergence between dequant and RoPE paths
- Pack RoPE copy into int-width loads (4 bytes per lane × 32 lanes = 128 bytes = 64 bf16 values) so all lanes execute the same store instruction, eliminating the if/else branch divergence that left lanes idle
