## Solution

Implement warp-specialized kernel architecture with 6 warp roles (DMA_Q, DMA_KV, MMA, 4×EPILOG) coordinated through a hierarchy of ClusterTransactionBarrier and ClusterBarrier with precise arrival counts, enabling TMA-based async data movement to overlap with MMA compute on Blackwell SM 100

## Key Lessons

- Warp specialization with dedicated DMA/MMA/EPILOG warps enables compute-memory overlap on Blackwell — assign warp 0 for Q loading, warp 1 for K/V loading, warp 2 for MMA, warps 4-7 for epilogue, and coordinate with typed barriers matching each synchronization point
- Use ClusterTransactionBarrier when TMA hardware arrives on the barrier (arrival count = number of software participants), and ClusterBarrier for pure software synchronization — matching barrier type to arrival mechanism prevents deadlocks
- Initialize all barriers in warp 0 then fence with fence_barrier_init() + cluster_arrive_relaxed()/cluster_wait() to ensure visibility across the entire cluster before any warp begins work
