## Challenge

Single-stage (kStage=1) flash attention pipeline loads K and V tiles synchronously each KV-loop iteration, blocking on CP_ASYNC_WAIT_GROUP(1)+syncthreads before starting Q@K^T and P@V computation, serializing global memory transfers with Tensor Core MMA work


## Solution

Enable double-buffered multi-stage pipeline (kStage=2) so cp.async.cg prefetches the next K tile into an alternate shared memory buffer while the current iteration's Q@K^T and P@V MMA proceeds on data already resident, with selective CP_ASYNC_WAIT_GROUP(1) placement that blocks only for the V needed by P@V while letting the next K async continue

## Key Lessons

- Double-buffered multi-stage pipelines (kStage=2) with alternating smem_sel/smem_sel_next shared memory slots overlap global memory latency with Tensor Core computation, yielding significant speedup when the KV loop has enough iterations (Tc≥8) to amortize pipeline fill/drain overhead
- Strategic CP_ASYNC_WAIT_GROUP(n) placement — waiting only for the required async group rather than all outstanding copies — allows ongoing prefetch transfers to continue while the current computation proceeds, maximizing memory-compute overlap
