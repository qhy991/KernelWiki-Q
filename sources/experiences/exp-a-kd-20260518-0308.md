## Solution

Fuse the gated activation and elementwise scale into a single kernel that keeps the intermediate post-activation result in registers and applies the scale multiply in the same loop iteration, eliminating the intermediate temp buffer write/read round-trip through global memory

## Key Lessons

- Fuse consecutive elementwise kernels that share a producer-consumer temp buffer relationship into a single kernel — the intermediate result stays in registers and avoids a full global memory write+read round-trip, which can yield >2x speedup for memory-bound pipelines
- When fusing kernels in MoE pipelines, combine per-token expert lookup and per-expert scale indexing into the same kernel to avoid redundant binary searches across multiple kernel launches
- Kernel fusion eliminates kernel launch overhead and L2 cache pressure: the first kernel's output is evicted before the second kernel can read it, so keeping data in registers via fusion avoids the cache miss storm on the temp buffer
