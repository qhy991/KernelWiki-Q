## Solution

Replace manual stride assignment with a packed-stride-from-shape helper that derives leading dimensions from logical shape (outer, inner), and compute StrideD independently from StrideC so the epilogue can handle different C-source and D-destination layouts — required for SM100 CUTLASS collective builder compatibility

## Key Lessons

- Derive GEMM strides from logical shape dimensions via a packed-stride helper rather than reading tensor stride fields — this decouples stride logic from tensor metadata internals and matches the SM100 CUTLASS make_cute_packed_stride contract
- Compute StrideD independently from StrideC in the epilogue args even when C and D share the same layout — separating them allows the epilogue to handle cases where source and destination have different memory layouts without code changes
- For contiguous non-batched tensors on SM80/SM90, both manual and packed-stride approaches produce identical values (lda=K, ldb=N, ldc=N), so the refactoring is correctness-motivated for SM100 forward compatibility, not performance-motivated
