## Challenge

In the topk selection loop, each iteration writes results directly to global memory via output[out_idx] = max_val, and a single thread reads back from global memory to accumulate output_sum, creating 2 unnecessary global-memory round-trips per topk element


## Solution

Stage topk results in shared memory arrays (smem_score/smem_idx) instead of writing to global memory per iteration; accumulate output_sum in registers during staging and broadcast to all warp lanes via __shfl_sync after the loop, eliminating all global-memory writes and read-backs during the topk selection phase

## Key Lessons

- Stage topk selection results in shared memory instead of writing directly to global memory during each iteration — eliminates N global stores that would otherwise need to be read back for normalization, reducing global-memory traffic from O(TOPK) round-trips per element to zero during the selection phase
- Accumulate running sums in registers and broadcast via __shfl_sync instead of reading back from global memory — avoids global-memory load latency and lets all warp lanes access the sum in a single shuffle instruction
