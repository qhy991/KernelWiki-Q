## Challenge

During LLM decode, RoPE is computed on Q and K in one kernel, then a separate kernel writes K (post-RoPE) and V into the paged KV cache buffer, requiring two kernel launches and redundant global memory reads of K values already computed in registers


## Solution

Fused KV cache write-back into the RoPE kernel by conditionally dispatching to BatchQKApplyRotaryPosIdsCosSinCacheEnhanced when save_kv_cache is true, computing RoPE on Q and K while simultaneously writing K (post-RoPE) and V into the paged KV cache buffer in a single kernel launch

## Key Lessons

- Fusing a post-kernel write-back (KV cache store) into the computation kernel (RoPE) eliminates one full kernel launch and avoids redundant global memory reads of values already in registers — applies whenever a producer kernel's output is immediately consumed by a short store-heavy kernel
- Conditional dispatch (save_kv_cache flag) preserves the original non-fused path so the optimization can be enabled only in decode-serving scenarios where KV cache write-back is needed, avoiding unnecessary work in prefill or non-serving paths
