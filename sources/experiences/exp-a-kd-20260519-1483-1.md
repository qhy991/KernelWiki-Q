## Solution

Replace single-CTA sequential block-level loop with a per-token direct addressing kernel launched with NUM_TOKENS CTAs, where each CTA computes the source address via token_to_seq mapping and a single block_table lookup (batch_offset/block_size), eliminating nested loops and enabling massive cross-token parallelism

## Key Lessons

- Replace single-CTA sequential loops over independent data units (blocks/tokens) with per-unit CTAs to exploit GPU massive parallelism — applies whenever work items have no inter-dependencies and address computation can be done independently
- Use a token_to_seq lookup array to map flat token indices to batch context (cu_seq_lens), enabling direct per-token address computation that avoids nested block→token iteration and partial-block boundary conditionals
- Precompute batch_offset/block_size and batch_offset%block_size for direct page-table indexing instead of iterating over pages — reduces block_table reads from O(num_blocks) to O(1) per work item and eliminates divergent conditional branches for last-block edge cases
