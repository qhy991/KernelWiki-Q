## Challenge

GELU elementwise kernel processes one float per thread using scalar 32-bit load and store instructions, issuing separate memory transactions per element and failing to leverage 128-bit coalesced access on a memory-bound operation


## Solution

Replace per-thread scalar float load/store with float4 vectorized load/store via the FLOAT4 macro, so each thread reads and writes 4 adjacent floats in a single 128-bit coalesced memory transaction, halving the number of memory transactions and improving bandwidth utilization on the memory-bound GELU operation

## Key Lessons

- For memory-bound elementwise kernels (activation functions, pointwise ops), float4 vectorized load/store processes 4 elements per thread in a single 128-bit coalesced transaction, reducing total memory transactions by 4x and significantly improving bandwidth utilization — applies whenever the operation has low arithmetic intensity and accesses contiguous memory
- Scale the thread index by the vector width (idx = tid * 4) and reduce block size accordingly (block_size / 4) to maintain the same total element coverage while each thread does 4x more useful memory work per instruction
