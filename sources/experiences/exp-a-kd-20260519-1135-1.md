## Solution

Replace integer bit-shift (>>5) with float division (/32.f) so the warp count becomes a ceiling via float comparison — e.g., blockDim.x=100 gives 100/32.f=3.125, and threadIdx.x < 3.125 passes for threads 0-3, correctly reading all 4 warp sums including the partial last warp

## Key Lessons

- Integer bit-shift division truncates toward zero for non-multiple sizes — use float division or explicit ceiling (blockDim.x + 31) / 32 when computing warp counts to preserve partial warps, applying to any block-scope algorithm that indexes by warp ID
- Float division adds one FP instruction but is negligible in reduction kernels dominated by warp shuffle and shared memory latency — the correctness gain far outweighs the single-instruction cost
- When blockDim.x is not guaranteed to be a warp-size multiple, always validate the warp count calculation with edge-case sizes (e.g., 33, 96, 100, 255) rather than only testing powers of two
