## Challenge

In the global iteration pattern, masked rows are skipped via continue but threads must still iterate across all remaining rows in the global range, processing rows from other experts that may have valid work — the continue does not eliminate wasted iteration overhead for masked-out expert regions


## Solution

Restructure the loop to iterate only within a single expert's contiguous row range, so that when a masked row is encountered, break immediately exits the loop since all subsequent rows in that expert's range are also masked — rows are sorted by expert, so the mask threshold is a single cutoff point

## Key Lessons

- When rows are sorted by expert, per-expert iteration allows break instead of continue for masked rows — since masked rows form a contiguous suffix within each expert's range, the first masked row means all subsequent rows are also masked, so break eliminates all wasted iterations
- Per-expert loop scoping converts a conditional skip (continue) into a full loop exit (break), which is especially beneficial when mask ratios are high (~37.5% masked rows in this benchmark)
