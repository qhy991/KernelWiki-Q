## Solution

Map one CUDA block to one token position (blockIdx.x = token_pos, threadIdx.x = tid) so all threads within a warp access consecutive dimension pairs within the same row (x[token_pos*N*2 + tid*2]), enabling the GPU to coalesce adjacent 32-bit loads into fewer 128-bit memory transactions

## Key Lessons

- Per-token block mapping (blockIdx.x = token_pos, threadIdx.x = dimension pair) ensures warp threads access adjacent addresses in the same row, enabling memory coalescing — applies to any per-element kernel operating on 2D [seq_len × dim] data where the inner dimension is contiguous in memory
- Flat linear indexing across multiple logical dimensions (idx = blockIdx.x * blockDim.x + threadIdx.x with idx / N and idx % N decompositions) interleaves threads from different rows in the same warp, producing strided non-coalesced accesses — prefer hierarchical block/thread mapping that preserves spatial locality
