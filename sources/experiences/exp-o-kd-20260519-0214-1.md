## Solution

Move per-sequence dim_k and local_split_kv computation before shared memory allocation and tensor construction, then add an early-return guard `if (local_split_kv == 1) return;` so CTAs for short sequences exit immediately without allocating shared memory or issuing global memory loads

## Key Lessons

- Move lightweight scalar computation (dim_k, local_split_kv) before expensive shared memory allocation (__shared__) and global memory tensor construction so that early-return guards can skip those costly resources entirely — applies to any kernel handling variable-length inputs where a subset of CTAs may have no real work
- In variable-length attention batches, short sequences may collapse to a single effective KV split; an early return when local_split_kv==1 avoids wasted shared memory capacity and register pressure for those CTAs, yielding significant speedup (1.31x) on short-sequence-dominated workloads
