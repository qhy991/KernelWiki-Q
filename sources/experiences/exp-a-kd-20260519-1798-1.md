## Challenge

Two-pass per-token int8 quantization launches two separate kernels: one to compute per-token absmax and write scales to global memory, then a second kernel reads those scales back from global memory and quantizes each element, incurring a full extra pass over the input tensor and a global memory round-trip for the scale buffer


## Solution

Fuse the absmax computation and quantization into a single kernel that computes the block-level absmax via blockReduceMax, stores the result in shared memory with __syncthreads() barrier, then quantizes in-place — eliminating the global memory round-trip for scales and the second full pass over the input tensor

## Key Lessons

- Fuse per-token reduction and elementwise quantization into a single kernel when the reduction result (absmax/scale) is needed only by the same thread block — eliminates global memory round-trip for intermediate scales and avoids a second full pass over the input
- Use __shared__ memory with __syncthreads() to broadcast a block-level reduction result to all threads within the fused kernel, replacing the global memory write-then-read pattern
