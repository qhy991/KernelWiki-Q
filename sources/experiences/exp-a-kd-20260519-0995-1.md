## Challenge

The local array struct AlignedStorage<uint4, 8> uses alignas(sizeof(uint4)*8)=alignas(128), which may force the compiler to allocate an unnecessarily aligned local memory slot for data that never actually resides in local memory, adding a constraint the optimizer must respect


## Solution

Remove the alignas directive from the local storage struct (renamed AlignedStorage→LocalStorage), since the compiler optimizer keeps each uint4 element in registers throughout the load→store pipeline and never materializes the array as a contiguous local memory block, making the alignment attribute irrelevant

## Key Lessons

- Removing alignas from local array structs has no measurable effect when the compiler optimizer enregisters all array elements — applies to fully unrolled load/store pipelines where each element is produced by inline asm and consumed before the next use
- alignas on local arrays only constrains the hypothetical starting address if the object were placed in local memory, but the NVCC optimizer avoids that placement when the data path from load to store does not require materializing the array as a contiguous memory block
