## Solution

Fuse SiLU activation and element-wise multiplication into a single kernel that loads gate and up vectors once, applies SiLU to gate values, multiplies by up values, and writes directly to the output buffer — eliminating the intermediate global memory buffer and second kernel launch entirely

## Key Lessons

- Fuse consecutive elementwise kernels (activation + multiply, bias + activation) into a single kernel to eliminate intermediate global memory round-trips — applies whenever two kernels operate on the same data elements and the intermediate result is never reused elsewhere
- Kernel fusion saves both global memory bandwidth (one fewer write+read per element) and kernel launch overhead (one fewer CUDA launch with its scheduling and context-switch cost), making it especially impactful for memory-bound elementwise operations on large tensors
