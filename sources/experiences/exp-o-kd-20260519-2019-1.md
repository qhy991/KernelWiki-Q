## Solution

Launch with max(num_experts, WARP_SIZE)=32 threads to fill the entire warp, resize shared memory from (num_experts+1)*num_experts to (blockDim.x+1)*num_experts to accommodate the extra threads, and add if(threadIdx.x < num_experts) guards to prevent threads beyond num_experts from corrupting expert-specific accumulation and expert_ids assignment

## Key Lessons

- Always launch at least WARP_SIZE (32) threads in single-block kernels to avoid partial-warp scheduling penalties — even when num_experts < 32, padding to warp size distributes per-thread workload across more lanes and eliminates idle scheduling slots
- When increasing thread count beyond num_experts, guard expert-specific operations with if(threadIdx.x < num_experts) to prevent extra threads from accessing out-of-bounds shared memory rows or corrupting expert_ids assignment
- Size shared memory layouts using blockDim.x rather than hardcoded num_experts when thread count may exceed num_experts, ensuring the prefix-sum accumulation array has enough rows for all participating threads
