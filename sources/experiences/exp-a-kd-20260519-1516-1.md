## Challenge

Per-group fp8 scale computation uses bfloat16 division (warp_max(y_max_bf16) / fp8_max) to normalize the absolute maximum to the fp8 representable range, incurring a division operation per group that is more expensive than multiplication on GPU hardware


## Solution

Precompute fp8_inv = __hdiv(1.0, fp8_max) once outside the per-group loop, then replace per-group division with __hmul(warp_max(y_max), fp8_inv) multiplication, converting an O(N) division cost into a one-time division plus N cheaper multiplications

## Key Lessons

- Precomputing the inverse of a constant divisor and replacing per-element division with multiplication is a standard strength-reduction technique — it amortizes one division across all iterations and replaces each subsequent division with a cheaper multiplication
- In memory-bound kernels where each thread does minimal arithmetic per global memory load, replacing division with multiplication shows no measurable speedup because the arithmetic unit is idle waiting for memory regardless of instruction latency
- This optimization becomes impactful inside pipelined compute stages (e.g., deep_gemm) where multiple groups are processed per token with overlapping async loads, raising compute density enough for arithmetic savings to matter
