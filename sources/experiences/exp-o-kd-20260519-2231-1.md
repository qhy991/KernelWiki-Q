## Solution

Fuse the two-pass quantization into a single kernel that computes per-token absmax via blockReduceMax, stores the result in shared memory, then quantizes using multiplication by the reciprocal (127.0f / block_absmax_val) — eliminating the second kernel launch and the global memory write+read of the scales buffer

## Key Lessons

- Fuse multi-pass kernels that communicate through global memory into a single kernel using shared memory for inter-phase data — eliminates kernel launch overhead and global memory round-trips, especially beneficial when per-block work is small and launch overhead dominates
- Replace division by scale with multiplication by precomputed reciprocal (127.0f / absmax) in quantization kernels — multiplication is cheaper than division on GPU hardware and the reciprocal is computed once per block in shared memory
- Use __shared__ memory to broadcast block-level reduction results to all threads within a block via __syncthreads() barrier — avoids writing intermediate results to global memory and reading them back in a subsequent kernel
