## Solution

Replace single-block <<<1,1024>>> kernel with a multi-block cooperative kernel launched via cudaLaunchCooperativeKernel, distributing token counting across multiple SMs and replacing serial cumsum with a four-phase parallel prefix sum (local scan, cross-block reduction via matrix-fragment loads, block-size alignment, global store) using cooperative_groups::grid.sync() for cross-block synchronization, merging counting, alignment, and sorting into one cooperative kernel

## Key Lessons

- cudaLaunchCooperativeKernel with cooperative_groups::grid.sync() enables multi-block kernels to perform global synchronization, distributing workload across SMs for problems requiring cross-block coordination like parallel prefix sums — applies when a single-block kernel is the serialization bottleneck
- Parallel prefix sum via local scan + cross-block reduction (using matrix-fragment-style loads for coalesced global memory access) + block-size alignment replaces serial cumsum, scaling with SM count rather than being O(N) on one thread — effective when expert count is large enough to amortize synchronization overhead
