## Solution

Replace CUB BlockReduce with a warp-level butterfly design where each thread owns all VPT=8 elements of a row (THREADS_PER_ROW=1), enabling thread-local max/sum with no cross-thread reduction, while packing 32 rows per warp via a 2D thread layout (dim3(WARP_SIZE,WARPS_PER_CTA)) for massive row-level parallelism without shared memory or barriers

## Key Lessons

- For small row sizes (num_cols <= WARP_SIZE * reasonable_VPT), pack all elements per row into a single thread (VPT=num_cols) to make reductions purely thread-local — eliminates shared memory, syncthreads barriers, and wasted threads carrying sentinel values
- Use 2D warp-level thread layout (dim3(WARP_SIZE, WARPS_PER_CTA)) with ROWS_PER_WARP = WARP_SIZE / THREADS_PER_ROW to process many rows in parallel per CTA, maximizing occupancy for narrow-row kernels
- When THREADS_PER_ROW=1 (single thread owns full row), the shfl_xor butterfly loop has mask starting from 0 and immediately exits — the reduction is free, operating entirely in registers with zero cross-thread coordination
