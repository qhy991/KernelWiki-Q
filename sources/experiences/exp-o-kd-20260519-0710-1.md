## Challenge

Per-warp subset scanning for TopK selection assigns ~32 experts per warp and iterates them with a strided inner loop over shared memory, causing redundant reads and branch-heavy comparisons per warp per topk iteration


## Solution

Replace per-warp strided scanning with one-thread-per-expert assignment where each thread reads only its own expert score once, then uses direct warp-level SHFL reduction with #pragma unroll to find the global maximum, storing results in shared arrays for deferred batch output and thread-local invalidation

## Key Lessons

- Assign one thread per element instead of per-warp strided scanning for TopK selection — each thread reads its own shared-memory slot once and reduces via SHFL, eliminating the inner strided loop and its ~32 shared-memory reads per warp per topk iteration
- Use #pragma unroll on warp-level SHFL reduction loops to convert the iterative shuffle chain into a fixed instruction sequence, which amortizes better across many blocks and reduces branch overhead
- Prefer thread-local invalidation (if tid == selected set own slot to -FLT_MAX) over cross-thread shared-memory writes — the owning thread already has its element in a register-friendly cache line, avoiding bank conflicts from random-warp writes
