## Solution

Pad each threadblock's workspace region up to the next 128-byte cacheline boundary using a compile-time alignment macro (ALIGNED_ACCUM_SIZE), so that each block's accumulator tile starts at an independent cacheline and false-sharing traffic between adjacent blocks is eliminated.

## Key Lessons

- Cacheline-aligning per-block workspace regions prevents false sharing only when the natural workspace size is not already a cacheline multiple — verify the alignment math actually changes the stride before applying this pattern
- False-sharing mitigation via cacheline padding requires concurrent access with high temporal locality on adjacent regions to produce measurable gains; synchronous microbenchmarks with __threadfence between write and read phases may not exhibit the sharing pattern the optimization targets
- The compile-time alignment macro ((bytes + CACHELINE_SIZE - 1) / CACHELINE_SIZE) * CACHELINE_SIZE / sizeof(float) is a reusable pattern for rounding up per-block allocation to cacheline boundaries in any multi-block workspace layout
