## Challenge

Baseline had torch/pybind11 dependencies that needed to be stripped and replaced with pure CUDA extern C entry point


## Solution

Rewrote kernel.cu from scratch with pure CUDA: float4 vectorization (128-bit loads), warp shuffle reduction (__shfl_down_sync), __ldg read-only cache, adaptive kernel selection (128/256/512 threads based on batch size), __fdividef fast division

## Key Lessons

- For rms_norm with hidden_size=4096, float4 vectorization gives 4x memory bandwidth efficiency
- Adaptive kernel selection (128/256/512 threads) based on batch size is important for latency vs throughput tradeoff
- Warp shuffle reduction is faster than shared memory reduction for this problem size
- The baseline stripping approach (sed) leaves torch-dependent code fragments; better to rewrite the entry function from scratch
- Use Python scripts instead of shell printf for TSV logging to avoid quoting issues with apostrophes
