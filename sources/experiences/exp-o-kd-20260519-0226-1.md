## Challenge

CUTLASS 3D dgrad convolution uses the kAnalytic iterator with kStrided stride support, which recomputes NDHWC coordinates via at() on every tile access and checks validity through coordinate-based arithmetic including stride divisibility tests, incurring significant per-tile overhead in the hot inner loop


## Solution

Switch from kAnalytic iterator to kOptimized iterator with kUnity stride support, which precomputes a delta table (inc_next[]) for pointer advancement and bitmask predicates (masks_[s][3]) for validity checks, converting runtime coordinate computation into simple pointer arithmetic in the hot path

## Key Lessons

- Prefer kOptimized over kAnalytic iterator in CUTLASS convolutions when strides are uniform (1x1x1) — the optimized iterator precomputes delta tables and bitmask predicates at initialization, replacing per-tile coordinate recomputation with single-pointer arithmetic in the hot path
- Use kUnity stride support instead of kStrided when convolution strides are all 1, as this eliminates stride divisibility checks and enables the precomputed delta table optimization
