## Challenge

The epilogue kernel unconditionally computes intermediate = beta * source via a dedicated mul_source function even when beta equals 1.0, wasting arithmetic instructions on an identity multiplication that produces the same values as the input


## Solution

When beta is known to be 1.0 (NoBetaScaling path), eliminate the mul_source call entirely and use source_frag directly as the intermediate, removing FRAGMENT_COUNT unnecessary FFMA operations per thread

## Key Lessons

- Eliminating identity-scale multiplications (beta=1.0 * source) in epilogue paths is a valid specialization but the arithmetic savings are negligible when memory latency and kernel launch overhead dominate the critical path — the optimization becomes meaningful only at larger fragment counts or higher arithmetic intensity
- Specializing kernel variants for known constant values (NoBetaScaling path) is a standard CUTLASS epilogue technique to reduce instruction count, even when the per-kernel speedup is below measurement noise
