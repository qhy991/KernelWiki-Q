## Solution

Swap grid/block dimensions to <<<N=1024, M=64>>> and reverse the indexing from row+col*ld to col+row*ld, achieving stride-1 coalesced access that collapses each warp's 8 partition reads into 8-16 cache lines, while simultaneously improving SM occupancy via 64-thread blocks and filling 108 SMs with 1024 blocks

## Key Lessons

- Swapping grid/block dimensions to achieve stride-1 coalesced access is the highest-impact optimization for memory-bound reduction loops — when consecutive threads access contiguous addresses, each warp's memory transactions collapse from hundreds of partial cache lines to a handful of full ones
- Multiple reduction iterations (split-K partitions) multiply the per-access coalescing penalty linearly — ensuring coalesced access in the inner loop becomes critical when the partition count is high, as the total wasted bandwidth scales with iterations
- Smaller thread blocks (64 vs 1024 threads) combined with more blocks (1024 vs 64) simultaneously improve SM occupancy and grid-level parallelism — especially beneficial on GPUs with many SMs when each thread's work is memory-bound rather than compute-bound
