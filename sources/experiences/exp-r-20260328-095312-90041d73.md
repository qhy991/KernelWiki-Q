## Challenge

Metal kernel for Q4_0 GEMM failed correctness test with NMSE ~2.0. Root causes: (1) Nibble ordering was reversed — byte stores q[i]|q[i+16]<<4, should use low nibble for positions 0-15, high nibble for 16-31. (2) memcpy() not available in Metal Shading Language.

### Symptoms

- `NMSE ~2.0 across all batch sizes`
- `Compile error: use of undeclared identifier 'memcpy'`

## Key Lessons

- Q4_0 byte layout: byte[i]=q[i]|(q[i+16]<<4) — low nibble→positions 0-15, high nibble→positions 16-31
- Metal Shading Language lacks memcpy — use as_type<T>() for bit reinterpretation
- Metal half-to-float must handle subnormals, inf, NaN correctly
- Generated random test data essential for local kernel verification
