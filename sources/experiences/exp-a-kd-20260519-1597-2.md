## Solution

Replace per-thread offset tracking with atomicAdd on a flat local_offsets array: initialize local_offsets[e] = cumsum[e] once, then each thread atomically claims its rank via atomicAdd(&local_offsets[expert_id], 1), eliminating 2D index arithmetic and decoupling the placement from the counting data structure

## Key Lessons

- atomicAdd on a flat offset array is simpler and more scalable than per-thread 2D matrix offset tracking for scatter/gather placement — each thread atomically claims its slot without computing 2D indices, and the pattern scales with any thread count without resizing the per-thread dimension
