## Solution

Fuse BF16-to-float conversion directly into the softmax kernel by inlining __bfloat162float at each access point (max-reduction, exp-sum, output loops), eliminating the intermediate float buffer and its global memory round-trip entirely

## Key Lessons

- Fuse type-conversion kernels with compute kernels when the intermediate buffer is read exactly once by the next kernel — eliminates a full global memory write+read round-trip, saving bandwidth proportional to the intermediate buffer size
- Inlining __bfloat162float at each access point in the fused kernel is cheaper than a separate conversion kernel launch because the conversion cost (a single instruction) is amortized by eliminating kernel launch overhead and the intermediate buffer allocation
- For row-wise softmax over narrow rows (64 elements), global memory traffic dominates compute cost, so reducing memory round-trips by fusing adjacent kernels yields measurable speedups even when the fused kernel performs the same arithmetic
