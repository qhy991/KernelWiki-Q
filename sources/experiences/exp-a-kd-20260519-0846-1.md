## Solution

Replace scalar half-element load/store with uint4 (128-bit) packed memory transactions that load 8 half values per thread in a single 16-byte transaction, reducing total global memory transactions by ~8x and achieving near-full utilization of each 128-byte memory transaction segment

## Key Lessons

- Use uint4 (128-bit) packed load/store via reinterpret_cast to process multiple elements per thread in a single memory transaction — reduces global memory transaction count by N× (where N = pack_size) and dramatically improves effective bandwidth for memory-bound kernels
- For half-precision elementwise kernels, pack_size = 16/sizeof(half) = 8 elements fit in one uint4 transaction, aligning well with GPU 128-byte transaction granularity and reducing per-element addressing overhead
- Remap thread indexing from per-element to per-pack (threads_per_head = head_size / pack_size) when vectorizing — this maintains the same logical work distribution while each thread processes pack_size contiguous elements
