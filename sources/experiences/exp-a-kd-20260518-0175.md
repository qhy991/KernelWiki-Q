## Challenge

The before kernel performs a software transpose using four __shfl_sync warp-shuffle operations per iteration (lines 38-41), which add register-to-register exchange latency and instruction count before the shared-memory writes


## Solution

The after kernel bypasses the shuffle sequence entirely on SM80 because its SM90_U16x8_STSM_T::copy() takes the #else fallback branch and writes four scalar stores directly to smem without any transpose step. The ~9% speedup comes from eliminating the four warp-shuffle instructions per iteration;...

- **speedup**: 1.099x (improved)
