## Solution

Add input_stride parameter to rms_norm_kernel so it computes blockIdx.x * input_stride + idx for input indexing, reading directly from strided tensors in-place and eliminating the separate strided-to-contiguous copy kernel launch and its associated global memory traffic

## Key Lessons

- Prefer stride-aware kernel parameters (input_stride) over a separate contiguous-copy kernel launch when operating on non-contiguous tensors — the copy kernel adds an extra launch plus redundant global memory read/write traffic that a single stride-aware kernel avoids entirely
- Replacing two kernel launches (copy + compute) with one stride-aware kernel yields compounding savings: fewer kernel launch overheads, less global memory traffic, and lower pressure on the CUDA stream scheduler
