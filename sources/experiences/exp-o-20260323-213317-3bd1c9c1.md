fp32_rms_norm_llama3_8b_hs4096

## Solution

256-thread kernel with rsqrtf, #pragma unroll 4, 8-warp smem reduction, __launch_bounds__(256,4), float4 vectorization, __ldg cache

## Key Lessons

- 256 threads with __launch_bounds__(256,4) is the sweet spot for hidden_size=4096 RMS norm
- Reducing to 128 threads hurts performance despite more unrolling per thread - parallelism matters more
- Caching 16KB weight in shared memory reduces occupancy and hurts small-batch latency - L2 cache is sufficient
- Warp-per-row approach (8 warps × 1 row per block) is much slower for small batches due to reduced parallelism per row
- rsqrtf is faster than fdividef+sqrtf for the normalization factor
- 8-warp smem reduction (smem[8]) is optimal for 256-thread blocks
- float4 vectorized loads with __ldg are key to achieving high memory bandwidth
- The Round 2 kernel at 2.692x is likely near-optimal for this problem size and GPU architecture
