## Challenge

Optimize flash attention kernel beyond ratio=241.5 (v2 baseline)


## Solution

Increase V_CHUNK from 64 to 128, doubling V shared memory from 16KB to 32KB half. Add __launch_bounds__(512, 2) for better register allocation. Total smem ~34.7KB within 48KB limit.


### Key Changes

- V_CHUNK=64 -> 128
- s_v_chunk[128*128] = 32KB half
- __launch_bounds__(BLOCK_SIZE, 2)
- Same float4 load pattern (8 halves per transaction)


## Key Lessons

- Doubling V_CHUNK from 64 to 128 reduces V global memory bandwidth by 2x, significantly improving performance
- 32KB half smem for V fits within 48KB limit alongside other shared memory
- __launch_bounds__(512, 2) helps compiler optimize register usage
- V_CHUNK=128 with TILE_SIZE=512 means 4 V_CHUNK sub-tiles per tile, reducing sync overhead
