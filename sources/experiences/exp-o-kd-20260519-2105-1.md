## Solution

Add a TileSchedulerArguments parameter with default initialization and construct KernelHardwareInfo inside the caller, forwarding both into GemmKernel::Arguments so callers can select StreamKScheduler for tall-skinny shapes (k > 3n) while preserving backward compatibility for the default PersistentScheduler path

## Key Lessons

- Add TileSchedulerArguments as a defaulted parameter to CUTLASS GEMM callers to enable StreamK scheduling for tall-skinny shapes (k > 3n) — StreamK splits individual tiles across multiple SMs, eliminating idle SMs when tile count < SM count, with a default value preserving backward compatibility
- CUTLASS 3.x GemmKernel::Arguments requires both KernelHardwareInfo and TileSchedulerArguments fields for proper SM-level work decomposition — construct KernelHardwareInfo inside the caller and pass it alongside the scheduler to avoid argument-struct mismatches
