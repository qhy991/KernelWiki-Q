## Challenge

GEMM epilogue loads beta from device memory at runtime and uses a conditional branch to decide whether to load the source tensor; when beta=0 the branch is uniformly taken but the source-load path remains compiled into the kernel, and host-side source memory must still be allocated and passed as a kernel argument


## Solution

Replace the runtime beta check with a compile-time bool template parameter (SourceNeeded) so that when false the compiler eliminates the entire source-load branch via dead code removal, reducing kernel arguments from 6 to 4 and removing the requirement to allocate or pass a source tensor

## Key Lessons

- Use compile-time template parameters (e.g., template<bool SourceNeeded>) instead of runtime conditionals for epilogue scaling mode selection — the compiler eliminates dead branches entirely, reducing instruction count and kernel argument overhead, even when the runtime branch is uniformly taken
- Compile-time branch elimination's primary value is correctness guarantees (source is never read regardless of beta's runtime value) and host-side memory savings (no source tensor allocation), rather than measurable kernel latency improvement when the runtime branch is already uniformly coherent
- For memory-bound epilogue kernels where both paths access the same accumulator tensor, eliminating a dead branch and reducing kernel arguments yields only marginal latency gains (~2%) since memory bandwidth — not instruction count — dominates
