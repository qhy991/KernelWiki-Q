## Challenge

Two sequential dense mma.sync.m16n8k16 instructions are required per k-tile to cover k=32, doubling MMA instruction dispatch count and fragment load overhead, while failing to leverage Ampere's dedicated 2:4 sparse tensor core datapath


## Solution

Replace two dense mma.sync.m16n8k16 instructions with a single mma.sp.sync.aligned.m16n8k32 that covers k=32 in one tensor core dispatch, using a FragM metadata register to encode 2:4 sparsity selectors, halving MMA instruction count and reducing register pressure per k-tile

## Key Lessons

- mma.sp.sync.aligned.m16n8k32 doubles the effective K-dimension per tensor core dispatch versus mma.sync.m16n8k16 — use it when the weight matrix has 2:4 structured sparsity to halve MMA instruction count per k-tile
- Sparse MMA requires a FragM metadata register (Vec<uint,1>) encoding 2:4 sparsity selectors as an extra operand — the metadata load overhead is small and amortized by eliminating one entire MMA dispatch per k-tile
- Dense-to-sparse MMA conversion reduces register pressure by consolidating two MMA operations into one, but fragment loads and accumulator writeback remain constant, so measured speedup is typically less than the theoretical 2x (here 1.5x)
