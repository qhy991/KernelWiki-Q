## Challenge

CUTLASS GEMM with TileShape<128,128,64> on M=64 only utilizes 50% of the M tile (64/128), wasting half the warps in the M direction while leaving K-pipeline depth shallow at 64


## Solution

Shrink TileShape M from 128 to 64 to exactly match the actual M dimension (eliminating wasted warp rows) and deepen K from 64 to 128 to improve pipeline utilization, achieving full M tile utilization with 2 warps instead of partially-used 4 warps

## Key Lessons

- Match TileShape M to the actual problem M dimension to avoid wasted warp computation — when M < tile_M, only a fraction of warps produce valid output, directly reducing utilization
- Deepening the K tile dimension (64→128) when shrinking M frees shared memory budget and improves pipeline depth, partially compensating for reduced thread-level parallelism
- TileShape selection for small-M GEMMs should prioritize M-fit first (tile_M ≤ M) then maximize K depth with the remaining shared memory budget
