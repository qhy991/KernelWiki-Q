## Solution

Replace scalar bf16 load/store with 128-bit vectorized int4 load/store via inline PTX (ld.global.cs.v4.u32 / st.global.cs.v4.u32), where each of 32 lanes transfers 8 bf16 values per iteration with cache-streaming hints (.cs) to bypass L1 cache for write-once data, reducing load/store instruction count by ~8x for the nope bulk

## Key Lessons

- Use 128-bit (int4 = 8×bf16) vectorized load/store via PTX inline asm to reduce memory instruction count by ~8x for bulk copy operations — applies when the copy region size is known at compile time and aligned to 16 bytes
- Cache-streaming hints (.cs) on ld.global/st.global bypass L1 cache allocation for write-once data patterns (e.g., concatenation output), preventing cache pollution and improving L2 hit rates for other data
- Separate differently-sized copy regions (nope=512 vs rope=64 elements) into independent loop phases to eliminate per-element branching and allow each phase to use the optimal vector width
