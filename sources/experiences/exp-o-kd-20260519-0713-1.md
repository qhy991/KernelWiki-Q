## Solution

Map each thread to exactly one expert (tid == expert index) so each thread already holds its value without scanning, then use direct warp-level reduction on that held value, defer global output writes to a single consolidated pass after all top-k iterations, and move shared memory arrays to kernel scope

## Key Lessons

- When thread count naturally matches element count (e.g., 384 threads for 384 experts), use a one-thread-per-element mapping to eliminate inner scanning loops and shared memory read bursts — each thread already holds its value and participates directly in warp reduction
- Defer global memory writes until after all reduction iterations complete and store intermediate results in shared memory arrays — this consolidates scattered global stores into a single batch, reducing global memory traffic in the hot loop
- Declare __shared__ arrays at kernel/function scope, not inside loops — in-loop shared memory declarations are not valid CUDA C++ and produce undefined behavior
