## Solution

Replace num_ints_per_thread scalar 4-byte loads with a single 16-byte int4 vectorized load per k-iteration via reinterpret_cast<int4*>(&frag)[0] = sh_zp_src[offset], cutting load instructions from 256 to 128 and allowing the GPU to issue wider memory transactions

## Key Lessons

- Consolidate multiple scalar shared/global memory loads into a single vectorized load (int4/float4/uint4) when consecutive elements are accessed — halves the load instruction count and enables wider memory transactions, applying to any loop that reads 2+ adjacent 4-byte values
- The measured speedup in microbenchmarks with single-thread launches is amplified by launch overhead and loop instruction dominance; in production kernels with full thread blocks the relative gain is smaller but the vectorization principle and instruction count reduction remain valid
