## Solution

Merge both copy kernels into a single templated kernel using a device-side lambda that performs kv_c and k_pe writes sequentially, eliminating one full kernel launch overhead and amortizing driver dispatch cost across both copy phases

## Key Lessons

- Fuse multiple lightweight kernels into a single launch when per-kernel GPU execution time is shorter than driver dispatch overhead (~5-10us) — applies to small-token, low-dimensional workloads where launch cost dominates
- Device-side lambdas in CUDA enable clean kernel fusion by abstracting repeated scatter-copy patterns into a reusable callable within one kernel, avoiding code duplication while consolidating multiple launches into one
