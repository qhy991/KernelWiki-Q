## Challenge

The RoPE kernel uses a flat 1D thread index (idx = blockIdx.x * blockDim.x + threadIdx.x) where consecutive threads span different token positions, causing strided memory accesses across row boundaries that require separate memory transactions per warp


## Solution

Restructure the thread-to-data mapping so blockIdx.x corresponds to token_pos and threadIdx.x to the dimension pair index within that token, making all threads in a warp read from the same contiguous row and producing fully coalesced 32-byte memory accesses

## Key Lessons

- Map blockIdx.x to the outer loop dimension (token position) and threadIdx.x to the inner contiguous dimension (feature pairs) so consecutive warp threads access adjacent memory locations — this produces coalesced accesses whenever the inner dimension is laid out contiguously in memory
- Flat 1D thread indexing that interleaves work across rows (idx / N, idx % N) breaks memory coalescing because consecutive threads cross row boundaries, requiring separate memory transactions per boundary within a single warp
