## Solution

Pass scale as a device pointer (const scale_type* scale_ptr) instead of a scalar kernel argument; the kernel reads it once from global memory via a single load, eliminating the host-side .item() synchronization at the caller site while keeping kernel-level performance identical since the single global load is hidden by the dominant input/output memory workload

## Key Lessons

- Passing scalar values as device pointers instead of kernel arguments eliminates the need for host-side .item() synchronization at the caller site — applies whenever the scalar originates from a device tensor and the kernel workload dominates over the single global memory read
- A single global memory load for a scalar parameter is completely hidden by the input/output memory access pattern when each thread processes multiple elements in a strided loop (hidden_size >> blockDim.x)
