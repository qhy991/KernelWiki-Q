## Solution

Swap A and B matrices via host-side transpose (A_swap=B^T, B_swap=A^T) so that C^T = B^T * A^T makes M_swap=N=4096, creating 4096 block rows instead of 1 and fully utilizing the GPU

## Key Lessons

- Swap A and B (C^T = B^T * A^T) for small-M GEMM workloads to convert the tiny M dimension into a large parallel dimension, enabling full GPU occupancy — mirrors CUTLASS 3.x swap_ab optimization for decode/low-batch scenarios
- Occupancy gains from increasing thread block count by orders of magnitude (1→4096) can dominate cache-friendliness penalties from transposed access patterns when original parallelism is extremely low
