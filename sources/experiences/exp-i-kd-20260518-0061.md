## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: SM 80 (A100); Uses CUTLASS library types (cutlass::layout::RowMajor, cutlass::MatrixCoord)
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
