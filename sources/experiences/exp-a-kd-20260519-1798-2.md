## Challenge

The quantization inner loop uses FP division by scale for each element, which is significantly slower than multiplication because FP division requires multiple hardware micro-ops on GPU


## Solution

Precompute the reciprocal tmp_scale = 127.0f / block_absmax_val once after the block reduction, then use multiplication in the hot inner loop (value * tmp_scale) instead of division (value / scale), replacing an expensive FP division per element with a single FP division followed by cheap multiplications

## Key Lessons

- Replace division by a loop-invariant value with multiplication by its reciprocal in hot inner loops — a single FP division costs ~4x the latency of an FP multiply on GPU, so precomputing the reciprocal amortizes the cost across all loop iterations
