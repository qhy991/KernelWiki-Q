## Solution

Replace the sequential scan with warp-level parallel counting: 32 lanes each count a slice of sorted IDs from shared memory, then aggregate via __shfl_down_sync butterfly reduction in O(log 32) steps; widen rd_ids computation to all MOE_BLOCK_SIZE threads instead of 1/4

## Key Lessons

- Replace sequential early-exit scans with warp-level parallel counting + __shfl_down_sync reduction when counting elements matching a predicate — achieves O(log warpSize) latency instead of O(n) and distributes work across 32 lanes
- Widen thread participation for post-load elementwise computation (e.g., division) from a subset of threads to all available threads — removes unnecessary divergence and better utilizes warp slots
