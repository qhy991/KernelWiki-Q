## Solution

Replace the single scalar scale with a per-group scale array in global memory, computing a group index (i / group_size) at each element to load the appropriate scale value; this adds conditional branching, integer division, and global memory loads that regress latency by ~23% but enables groupwise quantization accuracy

## Key Lessons

- Replacing a register-resident scalar with per-group global memory lookups adds latency from L1 cache misses and integer division (i / group_size) per element — groupwise quantization is an accuracy feature that trades off latency and should be benchmarked before deployment
- When per-element group index computation is needed, precomputing group_size as a power-of-2 allows replacing integer division with a bit-shift, and batching scale loads via shared memory can amortize global memory latency across the warp
