## Solution

Implement a CuTe-based GEMM with three-phase structure: (1) partition gmem/smem/register tensors and thread roles via ThrCopy/ThrMMA, (2) prefetch K_PIPE_MAX-1 tiles via cp.async before a pipelined main loop that overlaps gmem-to-smem copies with warpgroup MMA using double-buffered pipe indices, and (3) apply axpby epilogue to fuse alpha/beta scaling

## Key Lessons

- Software pipelining with K_PIPE_MAX stages hides gmem latency by prefetching tiles via cp.async while computing on previously loaded tiles — applies to any tiled kernel with regular K-dimension iteration where smem fits multiple tiles
- warpgroup_fence_operand / warpgroup_arrive / warpgroup_commit_batch / warpgroup_wait form the mandatory synchronization protocol for Hopper tensor core GMMA: the fence pairs prevent register hazards while arrive/commit establish the producer-consumer barrier across the warpgroup
- Prefetch K_PIPE_MAX-1 tiles before the main loop to fill the pipeline before steady-state begins, avoiding startup bubbles where compute would stall waiting for the first smem tiles to arrive
