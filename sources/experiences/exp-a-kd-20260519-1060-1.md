## Solution

Stage topk results in shared memory arrays (smem_score/smem_idx) instead of writing to global memory each iteration, and accumulate output_sum in registers directly from the selected value, eliminating both the per-iteration global store and the global-memory read-back for accumulation

## Key Lessons

- Stage per-iteration results in shared memory instead of writing directly to global memory when results will be consumed again within the same block — shared memory is ~20-100x lower latency than global memory and avoids unnecessary L2 traffic
- Accumulate reduction results in registers rather than reading them back from global memory — register accumulation (output_sum += thread_max_val) followed by shfl_sync broadcast eliminates an entire global-memory read round-trip per iteration
