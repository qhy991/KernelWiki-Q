## Solution

Set VPT=16 so each thread owns all 16 expert values of a row (THREADS_PER_ROW=1), eliminating all cross-thread reductions and shared-memory TempStorage; use WARPS_PER_CTA=4 with each warp processing 32 rows to improve occupancy from 1 row/block to 128 rows/block

## Key Lessons

- When the reduction width (num_cols) fits within a single thread's register capacity (≤32 floats), set VPT=num_cols and THREADS_PER_ROW=1 to own the entire row per thread — this eliminates all cross-thread reductions, shared-memory TempStorage, and __syncthreads() barriers, yielding significant latency savings
- Replace one-row-per-block kernels (blockIdx.x=row) with multi-row-per-block kernels using warp tiling (ROWS_PER_WARP=32, WARPS_PER_CTA=4) to improve occupancy and amortize block launch overhead across 128 rows per block instead of 1
- The butterfly (shfl_xor) reduction loop naturally compiles to zero iterations when THREADS_PER_ROW=1 (mask starts at 0), making the template general across row widths while incurring no cost when the thread owns the full row
