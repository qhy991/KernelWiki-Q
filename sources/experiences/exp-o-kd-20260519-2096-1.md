## Solution

Remove the outLogits output parameter and smemLogits[kTopK] shared memory buffer entirely, keeping only the index output path — this frees 8KB of shared memory per thread block, increasing occupancy and eliminating shared memory writes that were never consumed downstream

## Key Lessons

- Remove unused output parameters and their associated shared memory buffers from kernels — if downstream consumers only need indices (not values), eliminating the value output path frees shared memory and increases occupancy
- Audit kernel signatures and shared memory allocations for parameters that are written but never consumed by the caller — removing dead output paths reduces shared memory pressure which directly affects occupancy and scheduling
