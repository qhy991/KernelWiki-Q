## Challenge

MOE fused gate kernel loads input and bias arrays using scalar stride accesses (one float per iteration with WARP_SIZE stride), issuing 12 separate 32-bit global memory transactions per lane and underutilizing available memory bandwidth


## Solution

Replace scalar stride loads with float4 vectorized loads via reinterpret_cast<float4*>, loading 3 float4 vectors per lane (12 elements total) in coalesced 128-bit transactions, and unroll both the outer vec-per-lane loop and inner float4 processing loop to let the compiler schedule LDG.128 instructions efficiently

## Key Lessons

- Replace scalar stride loads with float4 reinterpret_cast loads when each warp lane processes multiple contiguous elements — this collapses N separate 32-bit global loads into N/4 coalesced 128-bit LDG.128 transactions, quadrupling effective memory bandwidth utilization
- Use lane_id * VEC_PER_LANE + i as the float4 vector index so adjacent lanes load adjacent 128-byte segments, ensuring fully coalesced warp-level access patterns
- Apply #pragma unroll on both the outer vector loop and inner element extraction loop to eliminate loop overhead and allow the compiler to pipeline multiple LDG.128 instructions
