## Challenge

The dSij backward kernel uses two separate 32-element fragment arrays (fragment_attn and fragment_di) in addition to the 32-element accumulator, consuming 96 registers per thread and risking register spilling under high occupancy


## Solution

Fuse the attention and di fragment loads into the computation loop: replace fragment_di[32] with a scalar current_di (broadcast across all 32 elements since di only depends on tid/32), and read attn per-element inline, eliminating two 32-element register arrays and reducing register pressure from 96 to 32 registers

## Key Lessons

- Eliminating fragment arrays by fusing load and compute loops reduces register pressure (96→32 registers), but NVCC -O3 may already keep all registers in the register file without spilling at moderate occupancy, making the optimization neutral in isolated benchmarks
- Register pressure reduction from fragment elimination becomes impactful only when the kernel has additional competing register demands (e.g., multiple GEMM stages, shared memory pointers, loop state) that push total usage over the occupancy limit — validate under realistic conditions, not synthetic microbenchmarks
- When a value is broadcast across all loop iterations (like di depending only on tid/32), recognize it as a scalar and hoist it out of the array to avoid wasting 31 register slots on identical values
