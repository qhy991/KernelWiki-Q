## Solution

Remove the zero_buffer_kernel entirely because moe_align_block_size_kernel's serial section (threadIdx.x == 0) already writes cumsum[0] = 0 and computes cumsum[i] = cumsum[i-1] + padded_count for all i from 1 to num_experts, fully initializing every entry that count_and_sort_expert_tokens_kernel subsequently reads via atomicAdd

## Key Lessons

- Eliminate redundant buffer-zeroing kernels when a subsequent producer kernel already fully initializes the same buffer via serial writes (e.g., prefix sum from cumsum[0]=0 through cumsum[num_experts]) — each avoided kernel launch saves 5-10us of command submission overhead plus execution time
- When a producer kernel writes every element of a buffer before a consumer reads it via atomicAdd, pre-zeroing is unnecessary because the producer's explicit writes define all base values before the consumer increments them
