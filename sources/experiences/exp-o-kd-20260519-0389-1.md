## Solution

Enable block-level swizzle by setting BLOCK_SWIZZLE=true and launching with a 3D grid (N_SWIZZLE=2 z-slices), where bx=blockIdx.z*gridDim.x+blockIdx.x remaps adjacent x-dimension blocks to distant column tiles separated by gridDim.x*BN columns, spreading B-matrix access across different L2 cache regions and reducing set contention

## Key Lessons

- Block-level swizzle via 3D grid remapping spreads sequential thread blocks across distant column tiles (separated by gridDim.x*BN columns), reducing L2 cache set contention — applies to any GEMM tiling where the number of column tiles exceeds the L2 associativity and adjacent blocks share B-matrix rows
- A compile-time BLOCK_SWIZZLE template parameter combined with bx=blockIdx.z*gridDim.x+blockIdx.x enables zero-overhead swizzle toggle: when false, bx reduces to blockIdx.x (identity); when true, the z-dimension remapping activates without branching
