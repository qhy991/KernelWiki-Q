## Solution

Use the compute_source_not_needed_() path: completely eliminate all source-related operations when beta=0 by removing the source pointer from the kernel signature, skipping source_fragment allocation and global memory loads, and replacing the two-argument output operator (alpha*acc + beta*src) with a single-argument form (alpha*acc), halving total global memory traffic from ~32MB to ~16MB

## Key Lessons

- When beta=0 in a GEMM epilogue, the source matrix contributes nothing to the output — skip all source global memory loads, source_fragment allocation, and use a single-argument output operator (alpha*acc) instead of the two-argument form (alpha*acc + beta*src) to halve memory bandwidth and reduce register pressure
- Compile-time dispatch paths (e.g., compute_source_not_needed_() vs compute_with_source()) are more effective than runtime beta==0 checks because the compiler cannot assume a runtime parameter is zero and must generate code for the full load-multiply-add pipeline
- Eliminating dead memory traffic at the algorithmic level (removing the source pointer entirely from the kernel signature) prevents future regressions where source loading might be accidentally reintroduced, compared to relying on conditional branches inside the loop
