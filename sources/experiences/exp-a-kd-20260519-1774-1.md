## Challenge

Standard GEMM call C=matmul(A,B) with small M=16 and large N=4096, K=4096 uses a single GEMM configuration without considering that swapping operand order (computing B^T*A^T instead of A*B) could enable more efficient tile scheduling on hardware with Tensor Memory Accelerator


## Solution

Swap operand order to compute D=matmul(B^T, A^T) then C=D^T, which yields the same result but on SM90 with CUTLASS 3.x enables more efficient TMA tile loads by making the first operand have a larger M dimension; however on SM80 with cuBLAS the backend internally selects the optimal algorithm regardless of operand ordering, resulting in neutral performance

## Key Lessons

- Swap-AB operand reordering (computing B^T*A^T instead of A*B) benefits CUTLASS 3.x on SM90 where TMA tile loads are more efficient when the first operand has a larger M dimension — but is neutral on SM80 cuBLAS which internally selects the optimal algorithm regardless of operand order
- Small speedups (~4%) in GEMM benchmarks are typically within measurement noise and should not be claimed as improvements without statistical validation across multiple runs
