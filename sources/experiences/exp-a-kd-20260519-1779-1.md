## Challenge

Swap-AB GEMM optimization (reordering A*B into B^T*A^T) is applied for a small-M case (M=32, N=1280, K=4096) to potentially improve tiling efficiency when the M dimension is tiny, but the transposed operand ordering shows no measurable benefit on SM80 cuBLAS


## Solution

Swap operand order by computing D = B^T * A^T instead of C = A * B, which transposes the tiling dimensions so the small M=32 becomes the trailing dimension; however cuBLAS on SM80 handles both orderings equally well for these dimensions, resulting in neutral performance

## Key Lessons

- Swap-AB GEMM reordering (A*B → B^T*A^T) does not guarantee improvement for small-M problems on SM80 cuBLAS — the library already handles both operand orderings efficiently when N is reasonably sized (N=1280)
- When applying swap-AB, scale arguments must be reordered to match the transposed epilogue layout, but this is a correctness requirement with no performance effect
