## Solution

Fuse the bias addition into the GEMM epilogue by computing C = scaleA * (scaleB * (A * B)) + bias in a single kernel, eliminating one kernel launch and the intermediate global memory round-trip for the output matrix

## Key Lessons

- Fuse elementwise epilogue operations (bias add, activation, residual) into the preceding GEMM kernel to eliminate a full global memory round-trip and one kernel launch — the savings are proportionally largest when the GEMM computation itself is fast relative to launch overhead
- Kernel launch overhead (parameter setup, driver submission, implicit inter-kernel dependencies) becomes a significant fraction of total latency for small matrices, making kernel fusion especially valuable in batched or small-tile scenarios
