## Challenge

Unconditional accumulator copy in a CUTLASS epilogue kernel always reads from the accumulator buffer regardless of whether K>0, offering no protection against the K=0 edge case where no valid MMA results exist


## Solution

Add a runtime bool guard (is_accumulator_needed = K > 0) that wraps the accumulator copy in a conditional branch, with an else path that fills registers with zeros for the K=0 identity case

## Key Lessons

- Adding a uniform branch guard (K > 0) around accumulator copy introduces negligible real overhead — in SM100 CUTLASS epilogue pipelines, TMEM synchronization (consumer_wait, consumer_release) dominates, dwarfing any single branch cost
- Microbenchmarks of very small kernels (~0.02ms per launch) are unreliable for measuring branch overhead — 10%+ swings from measurement noise can mask or invert the true signal
- Guard-on-K patterns are correct and practical for CUTLASS epilogue code even when the guard is always true for the common K>0 case, since the cost is amortized to zero in real workloads
