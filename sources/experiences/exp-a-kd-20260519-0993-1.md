## Solution

Fuse QKNorm and RoPE into a single kernel so that RMSNorm+weight-normalized values stay in the thread-local register array e[EPT]; RoPE rotation is applied directly on register-held elements before a single final write to global memory, eliminating the intermediate buffer and its ~32MB round-trip

## Key Lessons

- Fuse consecutive kernels that operate on the same data when the intermediate result fits in registers — this eliminates global-memory round-trips and saves kernel launch overhead (~5-10us per launch), yielding speedups proportional to the eliminated traffic for memory-bound workloads
- Register-resident data can carry forward across computation phases within a single kernel (RMSNorm → RoPE) without spilling, provided per-thread working set (e[EPT] where EPT=4 floats) stays within register budget — validate register pressure with --ptxas-options=-v
