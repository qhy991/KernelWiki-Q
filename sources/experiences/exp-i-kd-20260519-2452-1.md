## Solution

Use CUTLASS PredicatedTileIterator to decompose the copy into threadblock-level tiles along the strided dimension, with ceiling-divided iteration count, zero-initialized fragment for predicated load/store, and separate first-tile processing followed by a decrementing loop for remaining tiles

## Key Lessons

- PredicatedTileIterator encapsulates boundary predicate logic inside load()/store() — the kernel body never needs explicit bounds checking, which keeps the inner loop tight and branch-free
- Compute tile iteration count via ceiling division (extent[strided] + Shape::kStrided - 1) / Shape::kStrided to cover all rows including the final partial tile, then use a decrementing loop (iterations > 1) to process the remaining tiles after the first
- Zero-initialize the fragment before the first load to avoid uninitialized data in predicated-out elements at tile boundaries — the iterator only writes active elements, leaving the rest at zero
