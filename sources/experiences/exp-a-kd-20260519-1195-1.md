## Solution

Replace the 70-line software FP8 emulation struct with the native __nv_fp8_e4m3 hardware type from cuda_fp8.h, which compiles to a single cvt.rn.fp8_e4m3.f32 PTX instruction, eliminating approximately 20 instructions per element and removing the entire software conversion path

## Key Lessons

- Use native hardware FP8 types (__nv_fp8_e4m3 / __nv_fp8_e5m2) instead of software emulation — a single PTX cvt instruction replaces ~20 bit-manipulation branches, and the instruction savings scale linearly with per-thread element count
- Software type emulation overhead scales with per-thread workload — at group_size=256 (16 elements per thread) the measured improvement was 4%, more pronounced than group_size=128's 2.8%, confirming that conversion path cost is proportional to the number of elements each thread processes
