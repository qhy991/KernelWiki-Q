## Solution

Use half2 (32-bit) vectorized loads/stores via the HALF2 macro to process two half elements per thread in a single 32-bit memory transaction, coalescing two 16-bit accesses into one wider transaction and doubling per-thread memory throughput

## Key Lessons

- Vectorize half-precision memory access with half2 loads/stores to coalesce two 16-bit elements into one 32-bit transaction — applies to any memory-bound half-precision elementwise kernel where elements are contiguous and 32-bit aligned
- Scale the thread index by 2 (idx = 2 * linear_id) and halve the block size to maintain the same data coverage while doubling per-thread memory throughput
