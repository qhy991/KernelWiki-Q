## Solution

Replace 8 scalar bf16 loads with a single uint4 (128-bit) vectorized load via reinterpret_cast, consolidating eight narrow 16-bit memory transactions into one wide 128-bit transaction that fully utilizes the memory bus width

## Key Lessons

- Use uint4 (128-bit) vectorized loads via reinterpret_cast instead of scalar bf16 loads when reading 8 contiguous bfloat16 values — a single wide memory transaction replaces 8 narrow ones, dramatically reducing memory request overhead on bandwidth-bound kernels
- Vectorized loads are most impactful in memory-bandwidth-bound kernels with small batch sizes where the transaction count dominates; the 1.57x speedup here reflects an 8x reduction in load transactions for the inner dot-product loop
