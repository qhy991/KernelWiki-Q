## Challenge

CUTLASS MMA promotion interval validation hardcodes a divisibility-by-4 check, assuming every mainloop iteration issues exactly 4 MMA instructions; this misvalidates tile configurations where K_tile / MMA_K_atom differs from 4 (e.g., 2, 6, or 8 instructions per iteration), accepting invalid intervals and rejecting valid ones


## Solution

Replace the hardcoded % 4 divisibility check with a dynamically computed divisor: mma_instructions_per_iter = k_tile / mma_k_atom (i.e., size<2>(TileShape{}) / TiledMma::tile_size_mnk<2>()), then validate mma_promotion_interval % mma_instructions_per_iter == 0

## Key Lessons

- Never hardcode MMA instruction counts per mainloop iteration — compute dynamically from size<2>(TileShape{}) / TiledMma::tile_size_mnk<2>() to handle all tile configurations correctly, as the K-dimension ratio varies across hardware and tile shapes
- CUTLASS promotion interval validation must be parameterized by the actual tile shape rather than assuming a fixed multiple — different K-tile / MMA-atom combinations yield instruction counts of 2, 4, 6, 8, etc.
