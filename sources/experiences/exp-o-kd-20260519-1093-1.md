## Solution

Fuse sigmoid into the TopK kernel using vectorized 128-bit AlignedArray loads (4 floats per transaction) with interleaved thread addressing for coalescing, computing sigmoid in registers on the loaded data and eliminating the 1MB global workspace entirely

## Key Lessons

- Fuse producer-consumer kernel pairs when intermediate data fits in registers — eliminates global workspace allocation, write, and read pass, saving both memory bandwidth and kernel launch overhead
- Use 128-bit aligned vector loads (4 x float via AlignedArray) with interleaved thread addressing instead of scalar per-element accesses to saturate memory bandwidth when each thread processes multiple elements
