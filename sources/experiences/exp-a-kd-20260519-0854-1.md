## Challenge

The Blelloch exclusive prefix sum kernel performs log2(scan_size) iterations in both the up-sweep and down-sweep phases, each requiring a __syncthreads() barrier — totalling ~17 barriers for scan_size=256 with 1024 threads, causing all threads to stall at every barrier


## Solution

Replace the Blelloch scan algorithm (O(log n) __syncthreads barriers per phase, ~17 total) with a two-level warp-shuffle-based exclusive prefix sum using __shfl_up_sync, which completes intra-warp scans without any barriers and aggregates cross-warp partial sums with only 6 fixed __syncthreads() barriers regardless of scan_size

## Key Lessons

- Use __shfl_up_sync for intra-warp exclusive prefix sum to eliminate all __syncthreads barriers within each 32-element group — warp shuffle is hardware-synchronized and requires no software barrier
- Apply two-level scan hierarchy (intra-warp via shuffle + cross-warp aggregation via __syncthreads) to scale prefix sum to N > 32 while keeping __syncthreads count constant at ~6 regardless of scan_size
- For single-CTA scan workloads, __syncthreads barrier cost dominates arithmetic cost — reducing barriers from O(log n) × 2 to a fixed 6 yields 1.30x speedup even when total operation count is comparable
