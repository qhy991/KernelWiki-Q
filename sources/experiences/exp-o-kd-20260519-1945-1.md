## Challenge

SiLU+Mul and FP4 quantization are implemented as two separate kernels communicating through a 2MB intermediate global memory buffer (512×2048 half), requiring a full write-then-read round-trip that wastes global memory bandwidth


## Solution

Fuse SiLU+Mul and FP4 quantization into a single kernel where compute_silu_mul output stays in registers and feeds directly into quant_proxy_fp16, eliminating the 2MB intermediate global memory write and subsequent read

## Key Lessons

- Fuse producer-consumer kernels that communicate through a global memory intermediate buffer into a single kernel — keeping intermediate results in registers eliminates write+read round-trips and saves 2x the intermediate buffer size in global memory traffic
- Kernel fusion speedup scales with problem size: at M=512/K=2048 the 2MB intermediate buffer makes memory bandwidth the dominant bottleneck (1.47x gain), whereas smaller working sets may see limited benefit from fusion alone
