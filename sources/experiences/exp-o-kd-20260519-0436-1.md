## Solution

Enable TF32 tensor core path by setting CUBLAS_TF32_TENSOR_OP_MATH as the handle math mode and CUBLAS_GEMM_DEFAULT_TENSOR_OP as the algorithm hint, which forces cuBLAS to select a TF32-based tensor core kernel for SGEMM

## Key Lessons

- Setting CUBLAS_TF32_TENSOR_OP_MATH on the handle combined with CUBLAS_GEMM_DEFAULT_TENSOR_OP as the algorithm hint explicitly directs cuBLAS to use TF32 tensor cores for FP32 GEMM, yielding a ~1.15x speedup on Ampere+ GPUs — apply when reduced mantissa precision (~10-bit) is acceptable for the workload
- cuBLAS auto-dispatch under CUBLAS_DEFAULT_MATH may already use tensor cores on some GPU generations, but the explicit TF32 math mode ensures consistent kernel selection and can influence which specific tensor-core kernel variant is dispatched
