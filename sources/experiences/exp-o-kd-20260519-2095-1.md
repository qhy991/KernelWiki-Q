## Solution

Eliminate the TopKSort radix sort stage entirely and write smemIndices directly to global memory in a single strided loop, removing the smemLogits shared memory array (8KB), the TopKSort temp storage from the union (~several KB), the per-thread topKLogits/topKIndices register arrays (8 registers), and the outLogits global memory writes (2048 floats per row)

## Key Lessons

- When downstream consumers only need indices (not sorted logit values), eliminate the sort stage entirely — the histogram binning already guarantees that the top-K items are in smemIndices, and the FinalSort on the threshold bin ensures correct ordering within the boundary region
- Removing a cub::BlockRadixSort stage eliminates both its shared memory temp storage (several KB freed for higher occupancy) and per-thread register pressure (8+ registers for key+value arrays), compounding into significant speedup
- Audit whether output arrays (outLogits) are actually consumed downstream — if only indices are needed, removing the output write path saves 2048 float global memory stores per row
