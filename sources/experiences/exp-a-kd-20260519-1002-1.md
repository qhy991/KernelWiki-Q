## Solution

Introduce an adaptive dispatch that selects between a cooperative kernel (BM=16, BK=16, BN=16) for large M and a pingpong kernel (BM=8, BK=8, BN=16) for small M (<=64), using a runtime boolean to route to the appropriate implementation; for M=256 the cooperative path is selected, identical to the before version

## Key Lessons

- Adaptive kernel dispatch (selecting different tiled-GEMM strategies based on M dimension) is structurally sound but only benefits problem sizes that actually exercise the alternate path — for M=256 the cooperative kernel is selected in both versions, producing neutral performance
- When adding conditional dispatch, verify that the benchmark workload actually triggers the new code path; otherwise the change adds binary size and dispatch overhead with no performance gain
