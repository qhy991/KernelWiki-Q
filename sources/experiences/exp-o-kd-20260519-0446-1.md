## Challenge

Scalar FP16 sigmoid kernel processes one element per thread with individual half loads and stores, underutilizing memory bandwidth on a memory-bound kernel operating on 32 MB of data that exceeds L2 cache


## Solution

Vectorize the kernel to use half2 loads and stores via reinterpret_cast, processing two elements per thread to halve the number of memory instructions and improve coalescing efficiency on a bandwidth-bound workload

## Key Lessons

- Use half2 vectorized loads and stores via reinterpret_cast<half2*> to process two FP16 elements per thread — halves memory instruction count and significantly improves bandwidth utilization on memory-bound kernels
- For FP16 elementwise kernels where the working set exceeds L2 cache, vectorization from scalar half to half2 yields proportionally larger speedups than FP32 because the narrower data type makes instruction overhead more significant relative to useful data movement
