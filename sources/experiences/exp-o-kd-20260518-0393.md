half_4096rows

## Solution

The after code's uint4 loads issue 16-byte memory transactions that the hardware can coalesce into full 32-byte L2/DRAM transactions, achieving much higher effective bandwidth. With 4096 half columns and 256 threads, the scalar version needs 16 iterations while the vectorized version needs only 2...

## Key Lessons

- The half-precision case shows an even larger speedup than float because 128-bit vectorization packs 8 half elements per uint4 load instead of 4 float elements.
