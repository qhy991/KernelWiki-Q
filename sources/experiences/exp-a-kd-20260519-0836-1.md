## Challenge

Three separate kernel launches each independently read the same source metadata arrays (cache_seqlens, cu_seqlens_k, page_indices, nsa_cache_seqlens, nsa_cu_seqlens_k) from global memory, consuming 3x the read bandwidth and incurring additional kernel launch overhead per speculative decoding step


## Solution

Fuse 3 separate kernel launches into a single multi-destination kernel that reads each source element once into a register, then broadcasts the value to all 3 destination buffers, eliminating 2 kernel launches and reducing source read bandwidth by 3x

## Key Lessons

- Fusing multiple identical kernel launches that read the same source data into a single multi-destination kernel eliminates redundant global memory reads and kernel launch overhead — applies whenever the same source must be copied to N independent destinations
- Reading source data once into a register (int32_t v=src.field[i]) and broadcasting to multiple destinations exploits the GPU's high register-to-register throughput and keeps source data warm in L2 cache for subsequent writes
