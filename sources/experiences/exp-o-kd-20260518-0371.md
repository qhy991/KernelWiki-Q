## Solution

Switch output accumulator R_D to FP32 storage (kOStorageAccFloat32=1) so online rescaling FMA operates directly on float values without __half2float/__float2half_rn conversion, deferring the single FP32-to-FP16 conversion to the final global memory store

## Key Lessons

- Use FP32 accumulators for online softmax rescaling in tiled attention kernels instead of FP16 storage — eliminates repeated half-to-float and float-to-half conversion (2 instructions per element per rescale step) and improves numerical precision, at the cost of doubled register footprint
- Defer precision downcasting to the final output store rather than performing it at every intermediate accumulation step — reduces both instruction count and accumulated rounding error in tiling loops
