## Challenge

Embedding lookup kernel uses 4 separate scalar f32 load/store instructions per thread (8 total memory instructions), underutilizing available memory bandwidth and increasing instruction dispatch pressure


## Solution

Replace 4 scalar f32 load/store pairs with a single 128-bit packed load/store via reinterpret_cast<float4*>, emitting LDG.128/STG.128 instructions that coalesce 4 elements into one memory transaction, reducing total memory instructions from 8 to 2

## Key Lessons

- Use reinterpret_cast<float4*> (128-bit vectorized load/store) to replace groups of 4 contiguous scalar f32 accesses — reduces memory instruction count by 4x and enables the GPU to issue wider LDG.128/STG.128 transactions, applying to any kernel with contiguous coalesced memory access patterns
- The LDST128BITS macro pattern (reinterpret_cast<float4*>(&(value))[0]) is a portable way to express 128-bit memory ops without explicit pointer arithmetic — works for any 16-byte-aligned contiguous f32, f16×8, or int32×4 access
