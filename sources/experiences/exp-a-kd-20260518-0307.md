## Challenge

Warp swizzle causes a small regression at M=N=K=1024 (3%). The same alternating-direction pattern (j_s reversed for odd i rows) that regressed at 512 also regresses at 1024, but the impact is smaller


## Solution

Fuse the gated activation and per-expert scaling into a single kernel, combining the SiLU gating loop and the elementwise multiply into one pass that reads gemm_result, applies gated activation, multiplies by per-expert scale, and writes directly to output — eliminating the temporary buffer and one kernel launch

## Results

- **speedup**: 0.971x (regression)


## Key Lessons

- Kernel fusion that combines compute-heavy operations (SiLU gating with exp) with additional memory reads (per-expert scale) can increase register pressure enough to reduce occupancy and degrade performance — the saved kernel launch and temp-buffer write is not always worth the extra register and latency cost
- Separate simple kernels with low register footprints (like a dedicated elementwise multiply) can achieve higher occupancy and better memory throughput than a single fused kernel that must keep many live values (linear, gate, post_act, scale, alpha/beta/limit) across the loop body
- Fusion is beneficial when the intermediate result (temp buffer) is large enough that the extra global memory round-trip dominates; when the per-thread work is already compute-heavy (SiLU involves __expf), adding more arithmetic into the same loop may push the kernel from memory-bound to register-latency-bound
