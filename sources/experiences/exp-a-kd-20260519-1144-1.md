## Challenge

CUDA __hadd2 intrinsic for half2 addition relies on the compiler to emit the HADD2 instruction; in heavily templated code contexts the compiler may struggle with type resolution and generate suboptimal code


## Solution

Replace __hadd2 CUDA intrinsic with inline PTX asm 'add.f16x2' using explicit register constraints, providing a code-generation guarantee that forces the HADD2 instruction regardless of compiler type-resolution behavior

## Key Lessons

- Inline PTX asm for half2 arithmetic (add.f16x2) generates the same HADD2 SASS instruction as __hadd2 intrinsic — no throughput gain in simple contexts, but guarantees correct instruction emission in heavily templated code where the compiler may struggle with half2 type resolution
- PTX asm wrappers operating on uint32_t registers bypass the compiler's half2 type system, which can prevent unwanted type-conversion overhead in complex template hierarchies — use when compiler codegen for half2 intrinsics is unreliable, not for raw performance
