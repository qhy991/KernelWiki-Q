## Solution

Use a strided thread-indexed loop where each thread copies elements at stride blockDim.x, followed by cp_async_fence() to commit pending async operations, cp_async_wait<0>() to drain the async queue, and __syncthreads() to ensure all threads have completed before the bulk store stage

## Key Lessons

- Use a strided loop (threadIdx.x stride = blockDim.x) to distribute GMEM-to-SMEM copies across all threads uniformly — this pattern ensures coalesced access when tensor elements map to contiguous global memory addresses
- cp_async_fence() + cp_async_wait<0>() + __syncthreads() forms the three-stage synchronization barrier for async copy pipelines: fence commits pending async ops, wait<0> drains the queue to completion, syncthreads ensures all threads are ready before the next stage
