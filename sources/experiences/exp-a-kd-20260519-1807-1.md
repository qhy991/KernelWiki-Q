## Solution

Pass scale as a device pointer (const scale_type* scale_ptr) and dereference it inside the kernel, keeping the value entirely on the device side and eliminating the host-side .item() synchronization roundtrip

## Key Lessons

- Passing a scalar as a kernel argument (constant memory via parameter buffer) vs. dereferencing a device pointer (global memory → L2 cached) produces nearly identical kernel-level latency for a single 4-byte read — the real benefit of the device-pointer approach is eliminating host-side synchronization, not kernel throughput
- Kernel microbenchmarks that measure only GPU execution time cannot capture host-side optimization gains such as eliminating .item() device-to-host synchronization — end-to-end pipeline benchmarks are needed to measure such improvements
