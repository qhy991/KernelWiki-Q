## Solution

Replace the scalar variance loop with vllm::vectorize_read_with_alignment<8>, which issues 8-element vector loads (16 bytes for fp16/bf16) to reduce total memory transactions by roughly 8x in the aligned case, while handling misaligned addresses and non-vector-multiple lengths via scalar prefix/tail fallback paths

## Key Lessons

- Replace scalar element-by-element memory loops with vectorized reads (e.g., vec_n_t<scalar_t, 8> for 16-byte loads) to reduce memory transaction count by the vector width — applies to any bandwidth-bound kernel reading contiguous arrays from global memory
- Use a vectorize_read_with_alignment utility with scalar prefix/tail fallback rather than hand-rolling alignment checks — it correctly handles misaligned base pointers and lengths that are not exact multiples of the vector width without sacrificing the aligned fast path
- Precompute row pointers (input + blockIdx.x * input_stride) outside the loop to avoid redundant address arithmetic across all threads in the block
