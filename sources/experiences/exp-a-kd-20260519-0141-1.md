## Solution

Wrap tile selection in a compile-time if-constexpr guard that checks compatible(shape(CTA_Tile{}), shape(SLayout{})); when incompatible, apply product_each(cta_tile) to flatten each hierarchical sub-tuple into its product (e.g., ((4,32),64) → (128,64)), yielding a flat shape compatible with SLayout

## Key Lessons

- When constructing TiledCopy or TiledMMA in CuTe, always guard hierarchical CTA_Tile shapes with a compatible() check against the SMem layout shape — mismatched rank structures (nested tuple vs flat integer) produce inconsistent tile partitioning that silently corrupts TMA addresses
- product_each() flattens hierarchical shapes by collapsing each sub-tuple to its product (4*32=128), converting ((4,32),64) to (128,64) — use it as a fallback when CTA_Tile hierarchy does not match SLayout hierarchy
- Compile-time if-constexpr with compatible() incurs zero runtime cost while preventing shape incompatibility bugs that only manifest at specific CTA tile configurations (GETT 3-level, mixed hierarchy)
