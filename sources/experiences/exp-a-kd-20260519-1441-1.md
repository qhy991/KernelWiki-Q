## Solution

The before code uses PTX createpolicy.fractional.L2::evict_first to mark 100% of 16-byte cache lines for immediate L2 eviction, preventing read-once streaming data from polluting the L2 cache; removing this hint causes the data to linger under the default retention policy, degrading performance by ~9.6% on workloads where 512 blocks stream ~64MB through a 40MB L2

## Key Lessons

- L2::evict_first cache hints on cp.async are critical for streaming workloads where data is read once and total data volume exceeds L2 capacity — without the hint, read-once data pollutes L2 lines and degrades memory throughput for other concurrent blocks
- Removing an L2 eviction hint is a regression when the workload is streaming-heavy (64MB data >> 40MB L2): the hint prevents cache pollution, and its absence forces the default retention policy to hold stale lines that serve no reuse
