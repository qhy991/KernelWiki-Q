## Solution

Replace erff-based exact GELU with a Taylor approximation using fast_tanh backed by the single-instruction PTX tanh.approx.f32 hardware intrinsic on sm_75+, trading a small accuracy loss for significantly fewer compute instructions

## Key Lessons

- Replacing erff with tanh.approx.f32 via inline PTX reduces GELU from a multi-instruction SFU sequence to a single hardware instruction — beneficial when the kernel is compute-bound, but neutral when memory-bandwidth-bound
- GELU_taylor (0.5*z*(1+tanh(sqrt(2/pi)*z*(1+0.044715*z^2)))) is a standard CUTLASS approximation that trades ~1e-3 accuracy for faster compute — use when the downstream model tolerates this error
- Always profile before optimizing activation functions: an elementwise fp32 kernel reading/writing 8 bytes per element saturates DRAM bandwidth before the ALU becomes the bottleneck, making compute-side instruction reduction invisible
