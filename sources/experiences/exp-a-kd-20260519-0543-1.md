## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 pattern), loading 4 half elements per thread per iteration via paired half2 registers (reg_x_0/reg_x_1/reg_a_0/reg_a_1), reducing loop iterations from 16 to 4 and doubling per-warp memory transaction width from 64 to 128 bytes for better L2 cache utilization

## Key Lessons

- Use half2 vectorized loads (f16x4 pattern) to process 4 half elements per thread per iteration in GEMV/GEMM inner loops, reducing loop count by 4x and doubling per-warp memory transaction width — applies whenever K is a multiple of 4 and memory bandwidth is the bottleneck
- Vectorized loads improve L2 cache utilization by issuing wider memory transactions per warp (128 bytes vs 64 bytes), which amortizes cache line fetch overhead across more useful data — effective for bandwidth-bound kernels with strided or contiguous access patterns
- Loop iteration count reduction directly cuts per-iteration overhead (branch, address computation, loop counter update) across all active threads, compounding with the bandwidth benefit for a multiplicative speedup
