## Challenge

Backward attention kernel launches with grid(1, num_batch*num_heads) so that each (batch,head) pair uses a single block that serially iterates over all 32 key blocks via a for-loop, severely underutilizing available SMs and leaving GPU parallelism untapped


## Solution

Split the key dimension across multiple CUDA blocks via blockIdx.x (num_splits_key=4), so each block handles a strided subset of key blocks (start=split_idx*BlockSizeJ, stride=num_splits*BlockSizeJ), turning serial key iteration into parallel execution across SMs

## Key Lessons

- Split a serial loop dimension across CUDA blocks via blockIdx to expose parallelism — when one block iterates over N independent work units, launching N_split blocks each handling a strided subset (stride=N_split) directly multiplies SM utilization by N_split
- The measured speedup (4.53x) can slightly exceed the theoretical split factor (4x) because multiple blocks across SMs maintain independent memory pipelines, improving overall instruction-level parallelism versus a single block's limited occupancy
- Atomic accumulation overhead for partial gradient results across splits is negligible when the per-tile compute (dot products over head_dim) dominates the kernel runtime
