## Challenge

RoPE kernel launches a single thread block that serially loops over all 64 Q and 8 KV heads, leaving 107 out of 108 SMs idle on A800 when batch size is small (nnz=1)


## Solution

Introduce adaptive occupancy-based dispatch: query cudaOccupancyMaxActiveBlocksPerMultiprocessor at runtime, and when token-level blocks cannot saturate SMs, launch a head-parallel kernel that maps each head to blockIdx.y, distributing 72 independent thread blocks across dozens of SMs simultaneously

## Key Lessons

- For kernels with many independent heads (e.g. RoPE with 64 Q + 8 KV heads) but small batch size, distribute heads across blockIdx.y instead of serializing them in a loop within a single thread block — this fills idle SMs when token-level parallelism is insufficient to saturate the GPU
- Use cudaOccupancyMaxActiveBlocksPerMultiprocessor at runtime to decide between kernel variants (token-parallel vs head-parallel) — this adapts automatically to different GPU architectures and batch sizes without hardcoding thresholds
- Map Q heads to blockIdx.y < num_qo_heads and KV heads to blockIdx.y >= num_qo_heads in a single kernel with a branch — avoids maintaining two separate kernels while keeping per-head RoPE computation identical
