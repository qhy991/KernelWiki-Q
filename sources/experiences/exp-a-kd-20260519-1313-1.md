## Solution

Load the entire expert_first_token_offset array into shared memory via __ldg (read-only texture cache path) once per block, then service all subsequent reads from shared memory, eliminating repeated global loads and bypassing L1 pollution when multiple blocks access the same entries

## Key Lessons

- For small read-only arrays (under 128 bytes) accessed repeatedly by all threads in a block, shared memory caching via __ldg provides a modest but consistent benefit by routing reads through the texture cache and eliminating L1 conflict misses across multiple blocks
- __ldg forces reads through the read-only texture cache path which does not pollute the L1 data cache — useful when the same small data structure is accessed by multiple concurrent blocks that would otherwise thrash L1
- The speedup from shared memory caching scales with the ratio of repeated reads to one-time load cost; for very small arrays (9 elements) with short accumulation loops (≤9 iterations), the improvement is modest (1.07x) since the dominant cost shifts to the downstream global memory writes
