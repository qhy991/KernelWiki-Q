## Challenge

RoPE kernel uses flat linear thread indexing across all token-position pairs, causing consecutive threads in a warp to access pairs from different token positions, which results in non-coalesced memory transactions and poor bandwidth utilization


## Solution

Remap threads so each block handles one token position (blockIdx.x = token_pos, threadIdx.x = pair index), making consecutive threads in a warp read/write contiguous pairs within the same token row and achieving coalesced memory accesses

## Key Lessons

- Remap thread-to-data mapping so consecutive threads in a warp access contiguous memory addresses — coalesced global memory transactions can deliver significantly higher effective bandwidth than strided or interleaved patterns
- When each data element has a 2D structure (e.g., token position × hidden dimension), assign one block per row and threads within the block to columns, so each warp naturally coalesces along the contiguous dimension
- Thread remapping alone (no arithmetic changes) can yield substantial speedups when the original mapping causes poor memory access patterns — always profile memory coalescing before optimizing compute
