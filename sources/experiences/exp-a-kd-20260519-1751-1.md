## Solution

Fuse SiLU+mul and per-block quantization into a single kernel that computes the activation in registers, performs shared-memory absmax reduction on the in-register result, broadcasts the group scale via shared memory, and quantizes without ever materializing the intermediate activation in global memory

## Key Lessons

- Fuse activation (SiLU+mul) with its consumer (quantization) into one kernel to keep intermediate results in registers and eliminate a global memory round-trip — applies whenever an activation is followed by a per-element or per-group reduction/transform that can share the same thread mapping
- When the intermediate tensor would be written as FP16 and read back as float, fusion also avoids the redundant half-to-float re-conversion, saving both bandwidth and arithmetic
- Kernel fusion removes one full kernel launch overhead plus any associated stream synchronization, which is particularly valuable for memory-bound operations where the compute overhead of a single extra launch is non-trivial relative to total runtime
