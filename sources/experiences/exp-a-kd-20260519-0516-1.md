## Solution

Vectorize loads/stores using float4 so each thread processes 4 elements via a single 16-byte coalesced transaction, reducing total memory instructions by 4x and increasing per-thread work to better amortize instruction overhead

## Key Lessons

- float4 vectorization replaces 4 scalar loads/stores with one 16-byte coalesced transaction, but on GPUs with wide L2 caches (A800) the scalar path may already be coalescing efficiently — vectorization yields measurable gains mainly when the scalar kernel is genuinely bandwidth-starved
- For small-to-medium problem sizes where the working set fits in L2 cache, memory-level vectorization often falls within noise margins; the benefit becomes visible at larger N or when the kernel is the primary bandwidth consumer
- Thread count reduction (256→64 threads) from vectorization reduces scheduling overhead but also lowers occupancy — on memory-bound kernels the net effect may be neutral when the original occupancy already hid latency adequately
