## Challenge

2D nested loop maps rows to blockIdx.x and columns to threadIdx.x; when numRows (m_topk) is small relative to gridDim.x, most thread blocks sit idle, causing severe GPU underutilization


## Solution

Flatten the 2D row/col loop into a 1D global-index range over numRows * colsPerRow total elements; each thread strides through the flat range with gridDim.x * blockDim.x step, deriving rowIdx and colIdx via integer division and modulo to achieve uniform work distribution across all threads regardless of row/column ratio

## Key Lessons

- Flatten 2D nested loops where one dimension (rows) can be much smaller than gridDim.x into a 1D global-index range with uniform stride — ensures all threads contribute equal work regardless of dimension ratios, especially effective when the outer-loop count is small relative to GPU parallel capacity
- Derive 2D indices from a flat global index via integer division and modulo (rowIdx = globalIdx / colsPerRow, colIdx = globalIdx % colsPerRow) to maintain logical addressing while gaining the load-balancing benefits of a flat iteration space
