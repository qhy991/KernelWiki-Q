## Challenge

Skeleton code requires implementing vectorized register-to-global tensor copy using CUTLASS/CuTe abstractions: partition global memory into tiles, allocate a matching register tensor, fill with a deterministic pattern, and issue vectorized stores via a copy policy


## Solution

Use CuTe's local_tile(gC, tiler, 0) to partition the global tensor into an 8x8 tile, make_tensor_like to allocate a register tensor matching the tile layout, fill it with T(i % 13), and invoke copy(policy, rC, tCgC) which applies the vectorization assumptions from the copy policy (e.g., vectorizing to half2/float4) for coalesced global stores

## Key Lessons

- local_tile(gC, tiler, coord) partitions a global CuTe tensor into per-thread tiles using compile-time tiler shape — the returned tensor tCgC has the same layout as the tile and maps directly to global memory addresses for the thread's assigned block
- make_tensor_like(source_tensor) allocates a register tensor with identical shape and value type to the source — provides a zero-copy register buffer whose layout matches the tiled global tensor for direct elementwise copy
- copy(policy, src, dst) applies a CopyPolicy that encodes vectorization assumptions (e.g., align to half2/float4 boundaries) so the backend emits vectorized STG instructions — prefer this over bare copy(src, dst) when static analysis alone cannot prove alignment
