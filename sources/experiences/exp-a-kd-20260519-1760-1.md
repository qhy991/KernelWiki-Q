## Solution

Replace the hardcoded alignment constant with a compile-time computed expression 128 / (sizeof(ElementAB) * 8) that derives the vector width from the element type's bit width, ensuring correct 128-bit vectorized load sizing for any element type while producing identical code for int8_t

## Key Lessons

- Derive alignment from element type bit width (128 / sizeof_bits<T>) rather than hardcoding a magic number — this is a zero-cost compile-time refactoring that produces identical machine code for the original type but prevents misaligned vector widths when the template is instantiated with different element types
- Hardcoded template constants that happen to be correct for one instantiation are a latent bug: the compiler cannot catch the mismatch when a new type is introduced, and the error manifests as silent misalignment or out-of-bounds vectorized loads
