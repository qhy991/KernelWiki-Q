## Solution

Delegate all reduction logic to CUTLASS's ReduceSplitK functor pattern: declare __shared__ SharedStorage for inter-thread-block data exchange, instantiate the ReductionKernel functor, and invoke it with (params, shared_storage) — CUTLASS internally handles shared memory tiling, warp shuffle reduction, and LinearCombination epilogue transformation

## Key Lessons

- Prefer CUTLASS functor-based kernel patterns over hand-coded reduction loops — ReduceSplitK encapsulates shared memory tiling, warp shuffle reduction, and epilogue transformation in a single reusable template that adapts to SM architecture via SharedStorage
- Declare __shared__ storage using the kernel's nested SharedStorage typedef rather than manually sizing shared memory arrays — CUTLASS computes optimal shared memory layout from Shape, OutputOp, and ReductionOp template parameters
- Avoid duplicate using declarations when defining CUTLASS type aliases — redefining Shape, ElementOutput, or ElementWorkspace causes compilation errors; each alias must be defined exactly once with the desired configuration
