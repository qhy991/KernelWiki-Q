## Key Lessons

- The output stride remains at the original (non-doubled) colsPerRow because after silu_and_mul the data dimension returns to the original size.
- When a mask pointer is provided, the kernel reads both the gate and value halves of the input (the input last dimension doubles to accommodate both), applies `silu(x) * y` in-place on the packed vector, then quantizes the activated result directly to FP4.
