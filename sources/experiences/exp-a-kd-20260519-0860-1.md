## Challenge

Three separate CUDA kernels (shuffle gather, weight multiply, topk sum-reduce) are launched sequentially for MoE-style gather-and-weighted-sum, incurring two kernel launch/sync overheads and two intermediate buffer writes (~1MB each) that dominate execution time at small batch sizes where launch overhead is a significant fraction of total time


## Solution

Fuse the three separate kernels into a single kernel that reads input directly via the permutation index, multiplies by the weight factor, and accumulates into a register sum_val across topk iterations in one pass, eliminating both intermediate buffer allocations and two kernel launch/sync gaps

## Key Lessons

- Fuse sequential kernels that produce intermediate buffers into a single kernel when intermediate data can be held in registers — eliminates kernel launch overhead, device sync gaps, and redundant global memory round-trips, especially effective at small batch sizes where launch overhead is a large fraction of total execution time
- Kernel fusion with register accumulation (sum_val += factor * val) avoids two full DRAM write-then-read round-trips per element compared to writing intermediates to global memory, compounding launch overhead savings with memory bandwidth savings
