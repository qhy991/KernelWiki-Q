## Solution

Pre-compute the aligned prefix-sum of expert token offsets on the host into aligned_expert_first_token_offset[], then each GPU thread independently computes its result in O(1) time via two __ldg reads from global memory, completely eliminating the serial lane-0 loop, shared memory usage, __syncthreads barrier, and __shfl_sync broadcast.

## Key Lessons

- Move prefix-sum computations that depend only on static/small input data to host-side pre-computation — this converts O(N) serial GPU loops into O(1) per-thread lookups, especially critical when the loop runs on a single warp lane while others idle
- Serial accumulation on lane 0 with __shfl_sync broadcast is a severe warp divergence bottleneck when iteration count scales with problem parameters (e.g., num_local_experts=64); replacing it with direct __ldg reads lets all threads run in parallel without barriers
- Eliminating shared memory and __syncthreads from a kernel reduces register pressure, frees shared memory capacity for higher occupancy, and removes a warp-level synchronization cost — worthwhile when the data can be pre-processed externally
