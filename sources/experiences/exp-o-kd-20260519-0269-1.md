## Solution

Replace the triple-nested unrolled loop with direct integer division and modulo on ldsm_vec_num to recover the three indices (access_m_idx, inner_idx, inst_m_idx) in O(1), eliminating the loop overhead and conditional branch entirely and producing fewer constructor instructions in the final binary

## Key Lessons

- When an unrolled loop iterates over compile-time-constant bounds and a conditional selects one iteration based on a runtime index, replace the loop with direct integer division/modulo to recover the iteration variables — this eliminates loop overhead and produces fewer instructions even when the compiler could theoretically constant-fold
- O(1) index recovery via divmod is preferable to O(n) enumeration-with-compare for mapping a linear index back to multi-dimensional coordinates when all dimensions are compile-time constants
