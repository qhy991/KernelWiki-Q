## Solution

Vectorize memory access using half2 reinterpret_cast loads (HALF2 macro) to load 4 half elements per thread per iteration via two half2 loads each for x[] and a[], collapsing the K-dimension loop from 4 iterations to 1 and quadrupling per-instruction memory throughput

## Key Lessons

- Vectorized half2 loads improve memory throughput but increase register pressure (4 registers reg_x_0/1, reg_a_0/1 vs 0 extra registers in scalar) — for small K dimensions the register overhead and extra .x/.y deconstruction instructions can outweigh the bandwidth benefit
- At sub-0.01ms kernel execution times (small problem sizes like M=512, K=128), CUDA event timing jitter dominates measurement and vectorization overhead cannot be amortized — benchmark vectorization optimizations only at problem sizes large enough to be compute or bandwidth bound
- Loop iteration collapse from 4 to 1 removes the amortization of loop overhead across iterations — a single iteration with 4 half2 loads and manual .x/.y extraction costs more total instructions than 4 simple scalar load-and-multiply iterations for very short loop counts
