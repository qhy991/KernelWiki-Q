## Challenge

C++ if-branch guarding a scalar float store is used instead of PTX-level predicated store, raising the question of whether explicit PTX predication via @p st.global.u32 can outperform the compiler's branch-lowering for guarded global memory writes


## Solution

Replace the C++ if-guarded store with inline PTX using setp.ne.b32 to set a .pred register and @p st.global.u32 for the conditional store, mirroring cutlass::arch::global_store<AccessType,4> — but the compiler at -O3 already emits identical predicated stores for simple boolean guards, yielding no measurable difference

## Key Lessons

- For a simple scalar store guarded by a boolean, nvcc at -O3 already emits a predicated store instruction identical to explicit PTX @p st.global — hand-writing PTX predication provides no benefit for this pattern because the compiler optimization is sufficient
- When a kernel is memory-bandwidth-bound (16 scalar float stores per thread across 256K threads), minor instruction-level differences are hidden by the memory bottleneck, making predication-vs-branch microbenchmarks neutral even if the SASS differed
- Explicit PTX predication (cutlass::arch::global_store style) is valuable for complex guard expressions or multi-instruction sequences where the compiler may not predicate optimally, but not for single-store boolean guards
