## Solution

Replace hardcoded dimension products with explicit int64_t stride parameters (page_stride, head_stride) passed from the caller, parameterizing the address computation so the same kernel handles both NHD and HND layouts without modification

## Key Lessons

- Parameterizing stride values as kernel arguments instead of hardcoding dimension products enables layout-agnostic kernels that work across NHD and HND cache formats — the performance-neutral overhead from larger parameter blocks (40 vs 12 bytes) is negligible compared to the flexibility gained
- For short-duration kernels with few loop iterations per thread, minor parameter-block overhead (int vs int64_t strides) falls within measurement noise — do not over-optimize kernel parameter size when the kernel's arithmetic intensity is low
