## Challenge

A three-function inline device template chain (mul specialization → sum → generic dot) computes a simple scalar float multiply, introducing unnecessary template indirection for the common float×float case


## Solution

Replace the generic template chain with a specialized dot(float, float) overload that directly returns a * b — however, because all three functions in the chain are __device__ __inline__, nvcc already inlines the entire chain into a single FMUL, so the SASS is identical and the specialization has no runtime effect

## Key Lessons

- Template chain dispatch through __device__ __inline__ functions adds zero runtime overhead — nvcc fully inlines the entire call graph and resolves template specializations at compile time, producing identical SASS to a hand-written specialized overload
- Specializing template overloads for performance is only justified when the specialized path uses fundamentally different instructions (e.g., vectorized loads or hardware intrinsics); for simple scalar arithmetic the compiler already generates optimal code from the generic template
