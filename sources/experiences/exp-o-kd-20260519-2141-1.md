## Challenge

Per-thread 2D token-count matrix tokens_cnts[blockDim.x+1][num_experts] stored in dynamic shared memory grows to O(blockDim.x * num_experts), exceeding shared memory limits for large expert counts (e.g., 256 experts requiring ~33KB), forcing a global-memory fallback that is orders of magnitude slower


## Solution

Replace the large 2D per-thread token-count matrix with a warp-partitioned shared_counts[32][8] compact array where each warp owns 8 expert slots and uses atomicAdd for counting, reducing shared memory from O(blockDim.x * num_experts) to O(32 * 8) = 1024 bytes and eliminating the global-memory fallback path entirely

## Key Lessons

- Warp-partitioning a large 2D shared-memory array into a compact warp-slotted structure (32x8) with atomicAdd replaces O(blockDim.x * N) memory with O(32 * ceil(N/32)) and avoids global-memory fallback — applies whenever per-thread counting matrices exceed shared memory capacity for large problem dimensions
- atomicAdd on shared memory has minimal overhead on modern GPUs (a few clock cycles) and is an acceptable trade-off when it enables a compact data layout that keeps the entire working set in shared memory, avoiding global-memory fallback that costs orders of magnitude more
