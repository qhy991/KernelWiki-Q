## Challenge

Scalar loop iterates over all heads with thread stride blockDim.x, computing per-element head_idx/head_offset via integer division and modulo, resulting in non-coalesced memory access across scattered head regions in the HND-layout KV cache


## Solution

Distribute heads across warps (warp-per-head decomposition) so each warp owns a single contiguous head segment, then use 128-bit vectorized loads/stores (VEC_SIZE=8) within each warp to perform wide memory transactions, eliminating per-element division/modulo and improving coalescing in the HND-layout cache

## Key Lessons

- Warp-per-head decomposition (assigning each warp to a single contiguous head segment) eliminates per-element division/modulo index arithmetic and enables vectorized access within each head — applies whenever data has a natural head/channel dimension with contiguous segments
- 128-bit vectorized loads/stores (VEC_SIZE=8 for half) consolidate 8 narrow scalar transactions into one wide memory transaction per thread, doubling or quadrupling effective bandwidth for contiguous copy kernels when alignment permits
- Pre-computing source and destination base pointers outside the loop avoids redundant stride multiplication per element, reducing integer ALU pressure in memory-bound copy kernels
