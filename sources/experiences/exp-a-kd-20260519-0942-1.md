## Solution

Add __launch_bounds__(512, 1) to the kernel declaration, informing NVCC that maxThreadsPerBlock=512 and minBlocksPerMultiprocessor=1, which allows more registers per thread and reduces spills for the vals[4] and sums float4 structures

## Key Lessons

- Adding __launch_bounds__(maxThreads, minBlocks) alone may yield negligible improvement in single-GPU microbenchmarks where global memory bandwidth dominates — the annotation's benefit is more visible in multi-GPU contexts where barrier synchronization latency makes per-invocation register-spill overhead more impactful
- Memory-bound kernels (128-bit vector loads across multiple buffers) are bottlenecked by global memory throughput, not register allocation — launch_bounds cannot overcome a memory wall
- __launch_bounds__ is most effective when the kernel has high register pressure (large per-thread arrays like vals[RANKS_PER_NODE]) and runs in a latency-sensitive context where spill overhead compounds with synchronization costs
