## Solution

The after code fuses these into a single kernel: reduce_kernel_fused_fp8 computes the softmax normalization, applies the out_scale inline as acc *= out_scale, and writes FP8 output directly via the same __nv_cvt_float_to_fp8 call. The speedup comes from eliminating one full kernel launch overhead...

## Key Lessons

- Each kernel launch has fixed scheduling overhead, and the intermediate buffer occupies global memory bandwidth that the fused approach avoids entirely.
- The before code requires two separate GPU kernel launches: reduce_kernel_float writes 4-byte float intermediate results to a 16KB global memory buffer, then quantize_kernel_fp8 reads that same buffer back from global memory, applies the FP8 scale factor, converts each value to FP8 via __nv_cvt_float
