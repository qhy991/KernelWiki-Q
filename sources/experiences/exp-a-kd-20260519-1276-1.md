## Solution

Replace scalar element-wise loads with packed 128-bit array_t<half,8> loads (8 half elements per transaction), upcast to fp32 for accumulation via __half2float in an unrolled loop, and downcast only the final result — the 8x reduction in memory instructions dominates the speedup while fp32 conversion arithmetic overlaps with memory latency on Ampere architecture

## Key Lessons

- Use packed 128-bit loads via array_t<T, 16/sizeof(T)> with __align__(16) instead of scalar element-wise loads for half-precision reductions — reduces memory instruction count by 8x and the extra conversion arithmetic overlaps with memory latency on Ampere, yielding net speedup even with additional instructions
- Upcast fp16 inputs to fp32 before accumulation and downcast only the final result — preserves full fp32 precision through chained additions without changing the output format, eliminating the mantissa overflow error inherent in __hadd reduction chains
- Stride the packed loop by n/8 instead of n to process 8 elements per iteration, eliminating per-element loop overhead as a bonus side effect of the packed access pattern
