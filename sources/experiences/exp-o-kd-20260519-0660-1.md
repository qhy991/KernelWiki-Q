## Solution

Reduce dynamic shared memory from 128KB to 32KB (kSmem = 8*1024*sizeof(uint32_t)), which is sufficient because each radix pass holds at most TopK=64 candidates in the threshold bin, so 4K entries per round (2 rounds = 8K entries = 32KB) covers the working set, allowing 3-4 blocks per SM and boosting occupancy from ~512 to ~1536-2048 concurrent threads per SM

## Key Lessons

- Profile shared memory usage against actual working-set requirements — allocating 128KB for a TopK=64 radix that only needs ~TopK candidate slots per round wastes SM memory budget and caps occupancy at 1 block/SM; sizing to the true working set (4K entries/round × 2 rounds = 32KB) yields 3x+ speedup from occupancy alone
- On GPUs with 164KB shared memory per SM (A800/A100 sm_80), the sweet spot for per-block dynamic shared memory is ≤32KB to allow 3-4 concurrent blocks per SM; crossing the 64KB threshold typically halves occupancy
