## Solution

Replace the three-operation software path with a single inline PTX ASM instruction cvt.rni.sat.s8.f32 that performs rounding-to-nearest-even, saturation to [-128, 127], and int8 conversion in one hardware operation, reducing per-element instruction count and improving ILP in the kernel inner loop

## Key Lessons

- Use inline PTX ASM cvt.rni.sat.s8.f32 instead of rintf + fmaxf/fminf + cast for float-to-int8 conversion — the single PTX instruction combines rounding, saturation, and narrowing into one hardware operation, reducing instruction count and register pressure in tight quantization loops
- Use reinterpret_cast<const int8_t&>(dst) to extract the sign-extended int8 result from the 32-bit PTX register destination without additional conversion instructions
