## Key Lessons

- The half-precision case shows the largest speedup because scalar half loads (2 bytes each) are the worst case for memory coalescing on sm_80: 32 threads loading 2 bytes each yield only 64 bytes per warp-level transaction, far below the 128-byte optimal transaction size.
