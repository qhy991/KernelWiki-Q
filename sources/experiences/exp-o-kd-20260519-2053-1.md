## Challenge

The init_same_group kernel always reads g_idx from shared memory (2 loads per pipe iteration × 8 pipes = 16 accesses) even when has_act_order is false, wasting shared-memory bandwidth on data that is never consumed for any grouping decision


## Solution

Convert has_act_order from a runtime flag to a compile-time template parameter HAS_ACT_ORDER; when false, the compiler elides the entire g_idx-read branch and emits only two constant stores per pipe iteration, eliminating 16 shared-memory loads and all associated pointer arithmetic

## Key Lessons

- Promote runtime boolean flags that control entire code paths into compile-time template parameters so the compiler can dead-code-eliminate unused branches — applies whenever a flag partitions the kernel into disjoint code paths that never execute together
- When a kernel has conditional functionality (e.g., act-order vs no-act-order dequant), early-exit with trivial constant writes replaces expensive shared-memory reads + arithmetic + comparison, reducing bandwidth and instruction count for the inactive path
