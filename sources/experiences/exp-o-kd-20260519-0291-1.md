## Solution

Replace the multi-instruction isnan+branch sequence with PTX inline asm max.NaN.f32 which compiles to a single FMNMX.NAN SASS instruction on SM80+, eliminating all conditional branches and warp divergence while preserving NaN propagation semantics, with a __CUDA_ARCH__ fallback to the original branch code on older architectures

## Key Lessons

- PTX max.NaN.f32 compiles to a single FMNMX.NAN SASS instruction that handles NaN propagation in hardware — use it instead of isnan()+if-branch sequences on SM80+ to eliminate multi-instruction overhead and warp divergence in reduction patterns
- Branchless NaN-propagating operations are critical in warp-level shared-memory reductions where NaN values cause divergent branches across threads in the same warp, serializing execution — a single hardware instruction keeps all threads in lockstep
- Guard PTX inline asm with __CUDA_ARCH__ checks to maintain correctness on older architectures where the instruction is not available, falling back to the C++ implementation
