## Challenge

Per-thread linear scan through a 256-entry prefix-sum offset array to find the expert assignment for a token, requiring up to 256 serial comparisons per thread and suffering from divergent early-exit paths across threads in a warp


## Solution

Replace per-thread linear scan with warp-cooperative search: each of 32 lanes probes one offset entry per iteration, __ballot_sync votes on whether to continue, and __clz identifies the boundary lane — reducing iterations from O(E) to O(E/32) and converting divergent sequential reads into coalesced warp-wide parallel reads

## Key Lessons

- Use __ballot_sync + __clz for warp-cooperative search on sorted arrays: each lane probes one element, __ballot_sync collects pass/fail votes into a bitmask, and __clz finds the boundary in one instruction — reduces O(E) serial search to O(E/32) warp iterations
- Warp-cooperative search improves memory coalescing: 32 lanes reading contiguous entries produces coalesced wide memory transactions, whereas per-thread linear scan issues divergent serial reads with unpredictable early exits
- Speedup from warp-cooperative search scales with problem size: at 256 experts the 9.3x speedup far exceeds the 4.7x at 64 experts because serial overhead grows as O(E) while cooperative overhead grows as O(E/32), making this technique increasingly valuable for large expert counts in MoE models
