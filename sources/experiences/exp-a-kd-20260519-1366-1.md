## Challenge

The naive concat kernel uses a scalar grid-stride loop where every thread performs three integer division/modulo operations per element to decompose a flat index into (token, head, dim), resulting in high arithmetic overhead and poor memory access coalescing when threads in the same warp straddle different (token, head) pairs


## Solution

Assign one warp to each (token, head) pair, computing source and destination base pointers once per warp; each of the 32 lanes then handles a contiguous stripe of elements, eliminating all per-element index decomposition arithmetic

## Key Lessons

- Assign one warp per natural work-item (e.g., per token-head pair in attention concatenation) and pre-compute base pointers once — eliminates per-element integer division/modulo overhead and ensures all lanes in a warp access contiguous memory
- Use bitwise operations (>>5 for divide-by-32, &31 for modulo-32) instead of arithmetic division/modulo for warp-level index decomposition — single-instruction on GPU
