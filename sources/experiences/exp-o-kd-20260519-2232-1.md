## Challenge

MLA attention cache writes use two separate CUDA kernel launches (copy_kv_c_kernel and copy_k_pe_kernel), incurring double kernel launch overhead (~5-10μs per launch) and duplicating slot_mapping lookup and block_idx/block_offset computation across both kernels, which dominates execution time for small per-token data volumes (544 floats)


## Solution

Consolidate both kv_c and k_pe copy phases into a single kernel launch using a device-side lambda that performs the slot_mapping lookup once then sequentially copies kv_c at offset 0 and k_pe at offset kv_lora_rank, eliminating one full kernel launch cycle and duplicating no address computation

## Key Lessons

- Fuse lightweight kernels that share the same grid dimensions and address computation into a single launch using a device-side lambda or sequential phases — eliminates kernel launch overhead (~5-10μs each) which dominates when per-token data volume is small
- Duplicate slot_mapping lookups and index computations across multiple kernels targeting the same cache structure can be consolidated into a single shared computation block, reducing both launch overhead and redundant integer arithmetic
