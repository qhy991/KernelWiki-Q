## Solution

Fuse rms_norm and fp8_quant into a single kernel that computes the normalized value in-register and immediately quantizes to FP8, eliminating the intermediate __half buffer and one kernel launch entirely

## Key Lessons

- Fuse producer-consumer kernel pairs where the intermediate data is consumed exactly once — keeping values in registers avoids a full global-memory round-trip that can easily dominate latency for moderate buffer sizes (1–4 MB)
- Kernel fusion that eliminates an intermediate buffer simultaneously removes one kernel launch overhead and the associated L2/global memory traffic, compounding into significant speedups (1.4x observed here)
- When the producer writes fp16 intermediates and the consumer immediately reads them back, the half-precision store-load pipeline is pure overhead — fusing to keep the value in fp32 across both operations also avoids the precision loss of the intermediate truncation
