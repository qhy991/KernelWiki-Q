## Challenge

Naive sigmoid implementation 1/(1+expf(-x)) uses expf SFU call followed by negation, addition, and division (4 post-SFU operations), which is more arithmetic-heavy than necessary for a standard sigmoid activation


## Solution

Replace naive 1/(1+expf(-x)) with the mathematically equivalent tanhf-based formula 0.5f*tanhf(0.5f*x)+0.5f, which reduces post-SFU arithmetic from 4 ops (negation+addition+division+expf) to 2 ops (scaling+addition+tanhf), and enables downstream kernel fusion opportunities by eliminating intermediate buffers

## Key Lessons

- Replacing 1/(1+expf(-x)) with 0.5f*tanhf(0.5f*x)+0.5f reduces post-SFU instruction count from 4 to 2 but yields no throughput gain in isolation because SFU call latency dominates — applies when the SFU is the bottleneck rather than surrounding arithmetic
- The tanhf-based sigmoid formula enables kernel fusion (sigmoid+bias in a single pass) by eliminating the intermediate buffer required when sigmoid is a separate kernel — the real performance gain comes from fusion, not from the per-element formula change
- Both expf and tanhf map to a single SFU hardware call on NVIDIA GPUs, so swapping between SFU-backed transcendental functions does not change SFU throughput utilization
