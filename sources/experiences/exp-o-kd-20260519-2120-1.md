## Solution

Eliminate the cudaMemcpy2D intermediate copy by passing a runtime q_stride parameter to the kernel so it computes row base addresses directly on the original QKV buffer (q + seq_idx * q_stride + head_idx * HEAD_SIZE), exploiting the fact that within each (seq, head) pair the 128 contiguous floats are unaffected by the inter-row stride

## Key Lessons

- Eliminate cudaMemcpy2D strided copies by passing the stride as a runtime kernel parameter — pointer arithmetic cost is identical (one multiply-add) but avoids moving entire rows across the memory subsystem, yielding direct speedup proportional to the copy volume removed
- Non-contiguous row access is cheap when per-row reads are contiguous: within each (seq_idx, head_idx) pair the kernel reads 128 sequential floats (512 bytes) as a single contiguous segment regardless of the inter-row stride, so stride-aware addressing has no throughput penalty vs. contiguous access
