## Challenge

The cp.async.cg.shared.global copy uses an L2::evict_first cache hint via createpolicy.fractional PTX, adding instruction overhead for cache policy management that is unnecessary when the working set fits entirely within L2 cache capacity


## Solution

Remove the L2::evict_first cache hint and use plain cp.async.cg.shared.global without cache policy directives, since the 8MB working set fits entirely within the A800's 40MB L2 cache and the eviction hint adds PTX instruction overhead with no benefit

## Key Lessons

- L2 cache eviction hints (L2::evict_first) have no measurable effect when the working set fits within L2 cache capacity — the hint only matters when cache pressure forces eviction decisions
- Adding createpolicy.fractional PTX instructions introduces overhead (extra .reg.b64 allocation and policy creation) that is wasted when the hardware never needs to apply the policy — prefer plain cp.async without cache hints for small working sets
- Measure the data footprint vs L2 cache size before applying L2 cache policies; on A800 (40MB L2), workloads under ~30MB are unlikely to benefit from eviction hints
