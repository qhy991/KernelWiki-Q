## Solution

Swap operands to compute D=B^T*A^T using CUBLAS_OP_T/CUBLAS_OP_N, which is mathematically equivalent to C^T; this makes the first operand B^T have N=4096 rows instead of M=16, transforming a narrow-first-operand problem into a wide one for better memory access patterns and GPU occupancy

## Key Lessons

- Swap cuBLAS operands (swap-AB) when one dimension is much smaller than the others — computing D=B^T*A^T instead of C=A*B gives the GEMM kernel a wider first operand, improving memory throughput and occupancy for small-M or small-N problems
- cuBLAS GEMM performance is sensitive to the aspect ratio of the first operand; always benchmark both AB and swapped-AB orderings when M << N or N << M, since the transpose-based swap is mathematically equivalent but can yield drastically different hardware utilization
