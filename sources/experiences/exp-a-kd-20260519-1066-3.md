## Challenge

The before TopK kernel uses CUB BlockReduce with shared-memory tempStorage and __syncthreads barriers for each of 8 k-iterations, and each thread redundantly reads topk_indices from global memory up to k times to mask already-selected experts, causing O(k²) global reads per thread per iteration


## Solution

Replace CUB BlockReduce + global-memory exclusion checks with register-only TopK: threads perform local argmax on their register-resident chunk, then use warp-shuffle __shfl_xor_sync butterfly reduction across THREADS_PER_ROW (8) lanes to find the row-wide maximum, and mask selected experts by writing -10000.f to the corresponding register slot

## Key Lessons

- Replace CUB BlockReduce + shared memory + __syncthreads with warp-shuffle XOR butterfly reduction for small group reductions — when THREADS_PER_ROW ≤ WARP_SIZE, shuffle eliminates shared-memory pressure, synchronization barriers, and temporary storage
- Mask already-selected TopK elements by writing a sentinel value (-10000.f) to the register array instead of checking a global-memory exclusion list — this converts O(k²) global reads per thread into O(1) register writes
- Use __shfl_xor_sync with a width parameter equal to THREADS_PER_ROW to limit the butterfly reduction to only the collaborating threads within a warp, allowing multiple rows (ROWS_PER_WARP=4) to share the same warp
