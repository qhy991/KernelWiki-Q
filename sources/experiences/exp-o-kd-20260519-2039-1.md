## Solution

Replace the stride computation prob_k/8 with lda/8, where lda is the actual leading dimension passed as a new kernel parameter, so pointer arithmetic correctly steps over the full padded row width instead of assuming contiguous layout

## Key Lessons

- Use the actual leading dimension (lda) rather than the logical dimension (prob_k) for stride computation in tiled memory access — prob_k only equals the physical row stride in the contiguous case, and any padding breaks the assumption
- When kernel correctness depends on memory layout, always accept the stride/lda as an explicit parameter rather than deriving it from logical dimensions — this avoids silent data corruption on non-contiguous inputs
