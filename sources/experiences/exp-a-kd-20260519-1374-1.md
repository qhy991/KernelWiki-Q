## Solution

Launch max(num_experts, 32) threads to fill all warp lanes, distributing token-counting and index-mapping across 4x more threads; adjust shared memory layout to (blockDim.x+1)*num_experts entries and guard expert-specific loops with if(threadIdx.x < num_experts) to prevent out-of-bounds writes from excess threads

## Key Lessons

- Launch blocks with at least WARP_SIZE threads even when the logical work count is smaller — sub-warp launches waste SIMT lanes and reduce arithmetic intensity across the board
- When padding thread count beyond the logical work dimension, guard expert-indexed loops with if(threadIdx.x < num_experts) and size shared memory relative to blockDim.x not the logical count, preventing out-of-bounds shared memory writes and incorrect expert_ids assignments
- Distributing fixed work across more threads (CEILDIV(numel, blockDim.x) per thread) reduces per-thread sequential loop iterations proportionally — 4x threads means 4x fewer iterations per thread, yielding near-linear speedup when memory bandwidth is not the bottleneck
