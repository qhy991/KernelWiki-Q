## Challenge

Branch-based absolute value computation uses ternary operators introducing warp divergence in the absmax scan, while BlockReduce<1024> with a 1024-thread block allocates large shared memory and limits the number of concurrent blocks per SM


## Solution

Replace branch-based ternary absmax with fabsf/fmaxf intrinsics for branch-free computation and reduce block size from 1024 to 256 with BlockReduce<256> to lower shared memory footprint, though the reduced warp occupancy (8 warps per block vs 32) ultimately degrades throughput for hidden_size=4096

## Key Lessons

- fabsf/fmaxf intrinsics replace ternary-based abs/max with dedicated hardware instructions, eliminating branch divergence in reduction scans — applies to any kernel computing per-thread max or absolute-value reductions
- Reducing block size from 1024 to 256 cuts BlockReduce shared memory (roughly 4KB → 1KB) but reduces warp occupancy per SM from 32 to 8 warps, which can dominate performance for memory-bound kernels where higher occupancy hides latency
