## Solution

Pre-allocate d_tokens_cnts and d_cumsum once before the benchmark loop and reuse them across all iterations; the kernel safely reuses these buffers because it zeroes the per-thread rows of tokens_cnts at the start of each invocation and fully recomputes cumsum, so no stale data persists between calls

## Key Lessons

- Pre-allocate GPU scratch buffers once outside hot loops and reuse them across kernel invocations instead of per-call cudaMalloc/cudaFree — each cudaMalloc/cudaFree forces a synchronous device-wide serialization stall (~4-5ms) that dwarfs microsecond-scale kernel execution time for small kernels
- A kernel that zeros its own scratch buffers at the start of each invocation (self-resetting workspace) is safe to reuse pre-allocated buffers without host-side cudaMemset between calls, eliminating all per-iteration allocation overhead
- cudaStreamSynchronize called once after a batch of asynchronous kernel launches measures total GPU-side time without host-side synchronization overhead, whereas per-iteration cudaMalloc/cudaFree implicitly synchronizes the stream on every iteration and inflates wall-clock time by orders of magnitude
