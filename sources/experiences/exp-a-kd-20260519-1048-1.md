## Solution

Pack 8 tokens per CTA with one warp per token, replacing blockReduceMax + __shared__ scale + __syncthreads with warpReduceMax and register-local scale computation to eliminate synchronization overhead and reduce CTA count from 2048 to 256

## Key Lessons

- CTA count is a first-order performance factor for memory-bandwidth-bound kernels — reducing from 2048 to 256 CTAs by packing multiple tokens per CTA starves the GPU scheduler of independent work units across 108 SMs, outweighing any per-CTA synchronization savings
- When both kernel variants perform identical global-memory bandwidth (two full passes without shared-memory caching), CTA-level parallelism and occupancy dominate performance over blockReduceMax vs warpReduceMax overhead differences
- Eliminating __syncthreads and __shared__ intermediates via warp-level reduction provides negligible benefit when the kernel is memory-bandwidth-bound with high arithmetic intensity already hidden by vectorized loads
