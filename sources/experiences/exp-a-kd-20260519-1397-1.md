## Challenge

Binary 2SM/1SM tile heuristic only selects between two fixed tile sizes (256x128 for 2SM or 128x128 for 1SM), which can leave SMs underutilized for medium-sized GEMM problems that could benefit from smaller tiles distributed across more SMs


## Solution

Replace the binary 2SM/1SM heuristic with a three-tier adaptive tile_m selection that tries tile_m=64, 128, or 256 in order of increasing SM utilization, plus an alignment-based TMA epilogue flag, enabling finer-grained SM occupancy tuning for medium problem sizes on SM100+ hardware where CUTLASS tile scheduling is exposed

## Key Lessons

- Multi-tier adaptive tile selection (trying smaller tiles first to maximize SM occupancy) provides finer-grained scheduling than a binary 2SM/1SM heuristic, but the benefit is only measurable when the underlying library exposes tile scheduling control — CUTLASS does on SM100+ while cuBLAS handles it internally on SM80, making the improvement invisible in this benchmark
- TMA epilogue selection based on output alignment ((m*n) % 4 == 0) enables shared-memory-efficient writes when the output tensor meets alignment requirements — applicable to SM90+ TMA-capable hardware where epilogue store strategy affects measurable throughput
