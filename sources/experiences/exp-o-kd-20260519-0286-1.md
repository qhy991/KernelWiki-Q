## Solution

Add an `sm_occupancy > 1` guard to the SK wave formation branch condition, so it is only entered when the GPU actually supports multiple CTAs per SM; when sm_occupancy==1 the code falls through to the correct single-occupancy fallback that combines the last full wave with the partial wave

## Key Lessons

- Guard modulo-based branch conditions against the divisor-equals-1 edge case (N%1==0 is always true) — the guard eliminates a class of spurious dispatch decisions when the occupancy constraint makes the branch semantically invalid
- When a code path depends on multi-occupancy semantics (e.g., SK wave formation for load balancing across CTAs per SM), always guard with an occupancy > 1 check so single-occupancy configurations fall through to their correct fallback behavior
