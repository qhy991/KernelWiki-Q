## Solution

Wrap the per-element expf computation inside templated SigmoidArray and MultipliesArray functors operating on Array<float,8>, with #pragma unroll on all inner loops, to mimic the CUTLASS epilogue functor pattern — however nvcc at -O3 unrolls both the scalar loop and the functor-based loop identically, producing the same SASS instruction stream with no throughput gain

## Key Lessons

- Wrapping scalar element-wise math in CUTLASS-style Array<T,N> functors does not produce vectorized instructions for float types — Array<float,8> occupies 8 individual scalar registers with no wider SIMD packing, so nvcc compiles both scalar-loop and functor-loop variants to identical SASS
- CUTLASS functor patterns provide code structure and composability benefits for epilogue fusion, but actual throughput gains come from hardware-backed vectorized types (half2, float4) or tensor core operations, not from templating scalar float loops
- expf vs __expf: the before code uses the slower expf (library precision) while after uses __expf (fast intrinsic), but the difference is absorbed by compiler optimizations at -O3 and the tiny workload makes it unmeasurable
