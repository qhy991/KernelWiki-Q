## Solution

Swap A and B operands to restructure the GEMM as C^T(NxM) = B^T(NxK) x A^T(KxM) + bias_col, enabling tile dimensions TILE_M_SWAP=8 and TILE_N_SWAP=128 that achieve 100% thread utilization for M=8, with ScaledEpilogueColumnBias maintaining mathematical equivalence to the original row-broadcast bias

## Key Lessons

- AB-swap tiling can eliminate thread utilization waste when M is small (M < TILE_M) by matching TILE_M_SWAP to M, but the resulting larger shared memory footprint (5.1KB → 9KB) may reduce occupancy and offset the utilization gain — always profile the net effect on the target hardware
- Shared memory budget directly controls occupancy: increasing smem from 5.1KB to 9KB per block can reduce resident blocks per SM on GPUs with 48KB or 64KB shared memory limits, causing overall throughput regression even when per-thread utilization improves
- Grid size matters for small batch workloads: reducing total blocks from 64 to 32 by using larger tiles can leave SMs idle on high-SM-count GPUs like A800 (108 SMs), producing regression that outweighs any utilization improvement within each block
