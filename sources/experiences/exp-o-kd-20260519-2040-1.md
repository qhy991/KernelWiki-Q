## Challenge

The permute_cols_kernel assumed the input matrix A had row stride equal to size_k (contiguous layout), so it used a single row_stride derived from size_k for both input and output pointer advancement, producing incorrect column permutation when A is non-contiguous (e.g. padded or a sliced view with stride > size_k)


## Solution

Add an explicit lda (leading dimension of A) parameter and decouple input row stride (lda-based) from output row stride (size_k-based), so the kernel reads non-contiguous input rows at their true memory offsets while writing contiguous output rows

## Key Lessons

- Decouple input and output row strides via explicit lda parameter rather than assuming stride == column count — this is essential whenever the kernel may operate on non-contiguous views (padded tensors, sliced submatrices) where stride(0) > size_k
- When a kernel reads from one memory layout and writes to another, always parameterize both strides independently even if they happen to be equal in the common case — the cost is zero extra runtime overhead but prevents silent correctness errors on non-contiguous inputs
