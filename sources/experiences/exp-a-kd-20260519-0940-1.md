## Solution

Move the reciprocal scale computation (1.0f / scale) to the host side before kernel launch and pass the precomputed scale_val as a scalar kernel argument (const float) instead of a pointer — eliminating the per-thread global memory load and FP division entirely

## Key Lessons

- Precompute invariant scalar values on the host and pass them as scalar kernel arguments instead of device pointers — eliminates redundant global memory loads and per-thread arithmetic when the value is identical across all threads
- For memory-bandwidth-bound kernels processing large tensors, eliminating a single scalar global memory load yields negligible speedup because the cost is dominated by bulk data transfers — focus optimization effort on reducing the dominant memory traffic instead of per-thread constant overhead
