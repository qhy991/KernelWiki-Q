## Key Lessons

- Vectorization with float4 requires careful alignment - naive implementation breaks correctness
- Shared memory limits on sm_80 are 0x29000 bytes - 128x128 float tiles exceed this
- cuBLAS (used by PyTorch) is highly optimized for GEMM - custom kernels often slower
- Register tiling (4x4 per thread) improves arithmetic intensity but still can't beat cuBLAS
- For correctness-critical tasks, prefer simple working implementations over complex optimizations
