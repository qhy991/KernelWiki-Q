## Solution

Fuse the bias addition into the GEMM kernel epilogue by computing scaled + __half2float(bias[col]) before the final __float2half store, eliminating one kernel launch and avoiding the intermediate global-memory write of C that the separate bias-add kernel would need to read back

## Key Lessons

- Fuse elementwise operations (bias add, activation, scale) into the GEMM epilogue to eliminate separate kernel launches and avoid intermediate global-memory round-trips — the savings are proportionally largest when the GEMM computation itself is fast relative to launch overhead
- A separate bias-add kernel must read back every element of C from global memory that the GEMM just wrote, doubling memory traffic for that tile — fusing the bias addition in the same thread before the store removes this redundant load
