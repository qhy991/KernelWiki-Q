## Challenge

SiLU and elementwise multiply are implemented as two separate kernel launches communicating through an intermediate global memory buffer, causing a full global memory round-trip per element (write by SiLU kernel, read by Mul kernel) plus kernel launch overhead for the second kernel


## Solution

Fuse SiLU activation and elementwise multiply into a single kernel that reads gate and up once, computes SiLU on gate in registers, multiplies by up, and writes the result directly to output—eliminating the intermediate global memory buffer and the second kernel launch entirely

## Key Lessons

- Fuse elementwise activation + multiply into a single kernel when the intermediate result is only consumed once — keeping it in registers eliminates one global memory write+read round-trip and one kernel launch, producing significant speedup for memory-bound elementwise operations
- Kernel fusion is especially beneficial for bfloat16 elementwise kernels where compute is cheap relative to global memory traffic — removing the intermediate buffer cuts total global memory transactions by ~33% (from 3 reads + 1 intermediate write + 1 read + 1 output write → 2 reads + 1 output write)
