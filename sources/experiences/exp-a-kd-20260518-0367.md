## Challenge

At 2048 tokens the register buffering shows effectively zero improvement. The before kernel's 16 repeated k_rope reads per warp-chunk all hit L2 cache because the per-token rope data (128 bytes) stays resident across the short unrolled loop


## Solution

The after kernel's register buffer (k_rope_buf, a 4-byte int per lane) does not measurably reduce latency because the L2-to-register path for a 4-byte load is already fast, and the compiler may even recognize the read redundancy in the before version and optimize it into a register reuse pattern....

## Results

- **speedup**: 0.998x (neutral)


## Key Lessons

- The before kernel's 16 repeated k_rope reads per warp-chunk all hit L2 cache because the per-token rope data (128 bytes) stays resident across the short unrolled loop.
- The after kernel's register buffer (k_rope_buf, a 4-byte int per lane) does not measurably reduce latency because the L2-to-register path for a 4-byte load is already fast, and the compiler may even recognize the read redundancy in the before version and optimize it into a register reuse pattern.
