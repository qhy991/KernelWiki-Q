## Solution

Replace Blelloch shared-memory scan with a two-level warp shuffle scan: use warp_exclusive_scan (based on __shfl_up_sync) for barrier-free intra-warp prefix sums, then accumulate warp-level partial sums via a second warp_exclusive_scan on warp 0, reducing __syncthreads from 14 to 7

## Key Lessons

- Replace Blelloch shared-memory scan with warp shuffle (__shfl_up_sync) based scan when scan_size is small (1-2 warps) — warp shuffle is barrier-free within a warp, eliminating up to 50% of __syncthreads calls and their associated cross-warp synchronization overhead
- Two-level warp shuffle scan scales to multi-warp prefix sums by computing per-warp partial sums, then scanning warp-level aggregates on warp 0, then adding warp offsets back to each thread's local result — applies whenever data fits in a few warps (<= 1024 elements)
- __shfl_up_sync provides a lockstep inclusive scan primitive within a warp; computing v - original after the shuffle loop converts it to an exclusive scan without extra shared memory traffic
