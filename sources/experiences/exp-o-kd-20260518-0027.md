## Challenge

The before code extracts each of the 128 individual 1-bit elements using shift-and-mask (get), applies logical NOT to each element, and writes it back with a conditional set operation that does a read-modify-write on the packed word. This produces 128 iterations of shift+mask+compare+conditional ...


## Solution

The after code performs a single ~ (bitwise NOT) on each of the 4 uint32_t storage words, which flips all 32 bits per word in one instruction. The elimination of per-element extraction/insertion and the 32x reduction in loop iterations explains the significant speedup.

## Results

- **speedup**: 2.375x (improved)
