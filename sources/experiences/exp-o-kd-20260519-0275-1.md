## Solution

Replace erf(scalar / sqrt(2)) with erf(scalar * (1/sqrt(2))) by precomputing HALF_ROOT_TWO_D = 0.7071067811865476, converting the expensive double-precision division into a single DFMA-class multiplication issued at full FP64 pipeline throughput

## Key Lessons

- Replace a / b with a * (1/b) by precomputing the reciprocal constant when b is known at compile time — multiplication uses the full-throughput DFMA pipeline while division requires the low-throughput MUFU.DIV unit plus Newton-Raphson refinement, saving multiple cycles per invocation
- Precompute mathematically equivalent constants (e.g., 1/sqrt(2) instead of sqrt(2)) to shift work from runtime special-function units to compile-time constant folding — applies to any expression where the divisor or denominator is invariant
