## Solution

Raise AlignmentCD to 8 for fp16 output and explicitly pack 8 fp16 values into a float4 for a single 128-bit vector store per iteration, doubling the per-transaction payload from 64 bits to 128 bits and halving the number of store instructions issued

## Key Lessons

- Use float4 (128-bit) vector stores for fp16 output in CUTLASS epilogues by setting AlignmentCD = 128 / sizeof_bits<ElementD> (= 8 for fp16) — this enables the compiler to issue full-width store transactions instead of narrower scalar or half-width stores, delivering up to ~2x bandwidth improvement
- Explicitly pack narrow-type elements (fp16, bf16) into a float4 union and write via a single pointer cast rather than relying on the compiler to auto-vectorize contiguous scalar stores — the alignment hint alone may not be sufficient for NVCC to widen stores when element type is smaller than 32 bits
