## Solution

Load k_rope once per warp into a register buffer (k_rope_buf) and reuse it across all 16 heads in the chunk during the store phase, eliminating 15 redundant global memory reads per warp-chunk and reducing total k_rope traffic from 128 loads to 8 loads per token

## Key Lessons

- When a tensor dimension is broadcast/shared across multiple output dimensions (e.g., k_rope has no head dimension but is replicated across 128 heads), load it once into a register and reuse — applies to any concat/broadcast kernel where input lacks an output axis
- Use static_assert(sizeof(BufType) == DIM * sizeof(element) / WARP_SIZE) to verify at compile time that the register buffer type exactly covers the per-lane portion of the data, preventing silent misalignment bugs
