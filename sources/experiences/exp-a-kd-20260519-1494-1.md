## Solution

Replace static 2D shared memory with dynamically allocated flat array sized at runtime ((num_experts+1)*num_experts ints = 4360 bytes), accessed via index(row,col) helper — reducing memory footprint but introducing stride-32 bank conflicts and per-access multiply overhead

## Key Lessons

- Static C 2D shared arrays with row stride ≡ 1 (mod 32) — e.g., [65][65] — naturally avoid bank conflicts because consecutive rows' same-column elements fall in different banks; flat arrays with stride ≡ 0 (mod 32) cause n-way conflicts when threads access the same column across rows — use padding or odd stride to preserve conflict-free access
- Runtime-sized dynamic shared memory via extern __shared__ reduces wasted allocation but introduces per-access integer multiply overhead in index() that replaces compile-time-constant 2D array address arithmetic — for tight loops like prefix sum, this instruction overhead is measurable
- For single-block kernels, reducing shared memory footprint does not improve occupancy (only one block can run regardless), so the only effects of the change are the indexing overhead and bank conflict behavior — shared memory reduction only helps when it enables additional concurrent blocks
