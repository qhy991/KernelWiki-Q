## Solution

Vectorize memory access by processing 2 FP16 elements per thread using half2 reinterpret_cast loads/stores, consolidating two 2-byte scalar transactions into one 4-byte half2 transaction to halve total memory requests

## Key Lessons

- Use half2 vectorized loads/stores via reinterpret_cast to coalesce two 2-byte FP16 scalar transactions into one 4-byte transaction — this halves memory request count for memory-bandwidth-bound elementwise operations
- Scale thread index by the vectorization factor (idx = 2 * linear_tid) and halve the grid size accordingly to maintain correct work distribution when processing multiple elements per thread
- For memory-bound kernels, reducing total memory transaction count through wider vectorized access yields direct latency improvement even when compute throughput is unchanged
