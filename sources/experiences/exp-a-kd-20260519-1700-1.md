## Challenge

FP8 quantization uses two separate kernels over the full tensor: a shared-memory tree reduction for global absmax followed by a scalar quantization pass, requiring two full traversals of input data, a global atomicMax synchronization, and a separate kernel launch with redundant memory reads


## Solution

Fuse absmax reduction and quantization into a single per-token kernel where each block processes one token's hidden_size elements: warp-shuffle blockReduceMax replaces the shared-memory tree with O(log N) __syncthreads barriers, and the token's data stays in L1 cache between the absmax and quantize passes, eliminating kernel launch overhead and redundant global memory reads

## Key Lessons

- Fusing a reduction kernel with a consumer kernel into a single per-token block eliminates kernel launch overhead, redundant global memory traversal, and global synchronization (atomicMax), yielding large speedups when the per-token working set fits in L1 cache — applies to any quantization or normalization with per-token granularity
- Per-token (per-block) scaling is preferable to global scaling for quantization because it preserves per-token dynamic range and avoids cross-block atomics; each block computes its own scale independently via blockReduceMax, eliminating contention
- Warp shuffle reduction (__shfl_xor_sync) outperforms shared-memory tree reduction with __syncthreads barriers for block-level reductions because shuffle instructions bypass shared memory entirely and avoid synchronization stalls between tree levels
