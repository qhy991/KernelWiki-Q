## Challenge

Three separate kernel launches of fused_metadata_copy_kernel_single each independently read the same source data (4096 page_indices elements for bs=32, max_len=128) from global memory, consuming 3x read bandwidth, plus ~10us of launch overhead per iteration from two redundant kernel launches


## Solution

Fuse three separate single-destination kernel launches into one multi-destination kernel that reads each source element once into a register and broadcasts to all three destination buffers (dst0/dst1/dst2), eliminating 2 of 3 kernel launches and reducing global memory reads by 3x

## Key Lessons

- Fuse multiple kernel launches that read the same source data into a single kernel that reads once into a register and writes to multiple destinations — eliminates redundant global memory reads and removes per-launch overhead (~10us per eliminated launch)
- Read-once-write-many pattern is especially effective when source data (e.g., page_table at bs*max_len elements) is much larger than per-element computation, as global memory bandwidth dominates and 3x read reduction translates directly to ~3x speedup
- Broadcast source data to register before multi-destination writes so the GPU L2 cache serves subsequent writes from cached source data rather than re-fetching from global memory
