## Solution

Fuse SiLU+Mul and FP4 quantization into a single kernel so compute_silu_mul results stay in registers and feed directly into quant_proxy_fp16, eliminating the intermediate global memory write/read and its associated 2× activation-size bandwidth cost

## Key Lessons

- Fusing a producer-consumer kernel pair (SiLU+Mul → quantization) into one kernel eliminates the intermediate global memory buffer, saving 2× activation-size in DRAM traffic — applies whenever two kernels communicate through a temp buffer that fits in registers per thread
- Kernel fusion speedup scales with problem size because global memory bandwidth becomes the dominant bottleneck at larger sizes; the same fusion may show diminishing returns on very small problems where kernel launch overhead already dominates
- Fused kernels retain coalesced memory access when both input reads come from the same contiguous buffer (gate||up), improving data locality compared to separate kernels that must reload from a scattered intermediate buffer
