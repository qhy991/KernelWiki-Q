## Solution

Replace scalar element-wise kernel with shared-memory-tiled float4 vectorized kernel: 192 threads cooperatively load input as float4 vectors into shared memory, then each thread assembles one float4 output containing 3 data floats plus 1 zero-padding float and writes it as a single coalesced 128-bit store, eliminating per-element modular index arithmetic

## Key Lessons

- Vectorized float4 loads and stores reduce global memory transaction count by 4x compared to scalar float access — applies whenever data is naturally aligned to 128-bit boundaries or can be padded to them
- Shared-memory tiling enables cooperative loading and channel rearrangement (3-in → 4-out with zero padding) at shared-memory bandwidth rather than global-memory bandwidth, avoiding redundant global reads
- Eliminating per-element modular index decomposition (idx%c_out, cascaded divisions) in favor of direct shared-memory indexing removes expensive integer division instructions from the hot path, freeing ALU throughput for actual data movement
