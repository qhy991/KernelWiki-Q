## Solution

Vectorize to half2 (two FP16 values per thread) via reinterpret_cast, halving thread count per block (256→128) to process two elements per thread with paired __hadd operations

## Key Lessons

- half2 vectorization yields no benefit when the scalar half-width access pattern already coalesces into full-width memory transactions on sm_80 — 32 consecutive threads each reading one half already merge into a 64-byte coalesced transaction per warp
- Halving thread count (256→128) to process two elements per thread can reduce occupancy and concurrent memory requests, offsetting the doubled per-thread throughput for kernels already saturating bandwidth
- For memory-bound kernels whose working set fits in L2 cache, bandwidth is already saturated by the scalar pattern; vectorization only helps when the scalar pattern fails to fully utilize the memory subsystem (e.g., misaligned or strided accesses)
