## Challenge

Warp-level RMS norm for head_dim=1024 uses only 32 threads each handling 16 bf16x2 pairs; storing all pairs for single-pass would exceed register capacity, forcing a two-pass approach that loads input data twice and doubles global memory traffic


## Solution

Replace warp-level two-pass (32 threads, 16 pairs/thread) with CTA-level single-pass (128 threads, 4 pairs/thread): the reduced per-thread register footprint allows caching input and weight in registers across a single load, while 4x more threads provide higher memory-level parallelism and cross-warp shared-memory reduction replaces warp-only shuffle

## Key Lessons

- Reducing per-thread work granularity (16 pairs → 4 pairs) enables single-pass execution: the lower register footprint allows caching all input and weight values in registers across the reduction barrier, eliminating redundant global-memory re-loads — applies whenever register pressure forces a multi-pass approach
- Scaling from warp-level (32 threads) to CTA-level (128 threads) provides 4x more memory-level parallelism for global loads and enables cross-warp shared-memory reduction with negligible __syncthreads() overhead — applies to bandwidth-bound normalization kernels where a single warp underutilizes memory bandwidth
- Two-stage shared-memory reduction (per-warp shuffle → smem → warp-0 final reduction → smem) is a clean pattern for cross-warp reductions within a CTA when the number of warps is small (≤4) — avoids launching a second kernel or using atomics
