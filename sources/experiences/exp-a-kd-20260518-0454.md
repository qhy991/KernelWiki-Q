## Key Lessons

- The after kernel regresses on A800 (Ampere sm_80) because its PTX-level cache control primitives add net overhead.
- The before kernel's two-phase approach uses simpler control flow—two clean unrolled loops with direct pointer recomputation via head_id * stride—and keeps loaded data in the fastest memory tier (registers via k_nope_buf[16]), which is more efficient on Ampere for this streaming copy pattern.
