## Challenge

Rotary embedding kernel iterates over num_heads*head_size elements per thread with an is_first_half conditional branch that causes warp divergence — threads in the same warp processing first-half and second-half head dimensions must serialize both branch paths, and each iteration writes only one output element


## Solution

Restructure the loop to iterate over num_heads*embed_dim instead of num_heads*head_size, computing both x-rotated and y-rotated outputs per iteration via direct multiply-subtract/multiply-add pairs — this halves iteration count, eliminates the is_first_half branch and conditional ternary selections entirely

## Key Lessons

- Restructure rotary embedding loops to iterate over embed_dim (half the head dimension) and compute both x-rotated and y-rotated outputs per iteration — halves loop count and eliminates warp-divergent conditional branches, applicable whenever paired rotation outputs share the same cos/sin values
- Replace conditional ternary selections (is_first_half ? q_x : q_y, is_first_half ? -q_y : q_x) with direct unconditional multiply-subtract/multiply-add pairs — avoids branch divergence inside warp-stride loops when both outputs can be computed from the same inputs
