## Solution

Switch to MQA layout with NKV=1, making block_stride=NKV*HS*BS (1024 half-elements per block, ~1MB total, 8× memory reduction), and remap all query heads to KV head 0 via head_mapping — the kernel logic stays identical but operates on a compacted cache with improved L2 locality

## Key Lessons

- MQA reduces KV cache memory by num_heads/num_kv_heads factor with identical per-kernel compute workload — applies whenever multiple query heads share the same KV projections, with cumulative bandwidth savings across all layers and sequences at the model level
- Per-kernel microbenchmarks may show neutral speedup for MQA vs MHA since the dot-product FLOPs and access pattern per head are identical — real MQA benefits manifest through reduced total memory allocation and improved cache residency at the system level, not at the single-attention-kernel level
