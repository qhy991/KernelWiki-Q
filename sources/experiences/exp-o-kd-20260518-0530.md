## Challenge

The before code uses CUB DeviceRadixSort::SortPairsDescending to fully sort all 8192 float-int32 pairs in descending order and then takes the first TopK=2048 entries. CUB radix sort performs multiple global-memory passes over the entire dataset, writing keys and values through temporary buffers


## Solution

The after code avoids the full sort entirely: it builds a single coarse 256-bin FP16 histogram in shared memory (one pass over the data), identifies the threshold bin separating the top-K from the rest, and only refines the small set of ambiguous candidates in that bin using byte-level radix pass...

## Results

- **speedup**: 42.831x (improved)
