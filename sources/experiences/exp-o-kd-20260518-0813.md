## Challenge

Optimization focus: Remove outLogits parameter and smemLogits shared memory buffer from topKPerRow kernel.
Problem before: The topKPerRow kernel allocated a shared memory buffer smemLogits[2048] (8KB) to store logit values alongside indices, even though only the indices were ultimately needed by t


## Solution

he caller. This wasted 8KB of shared memory per block and forced the kernel to track and write logit values through the entire pipeline.
Applied change: Remove the smemLogits shared memory buffer and the outLogits output parameter entirely. Only indices are stored and output. This frees 8KB of sha

## Key Lessons

- Eliminates shared memory writes that were never consumed downstream.
