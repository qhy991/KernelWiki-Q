## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 style), loading 4 half values per iteration via paired half2 loads for both matrix and vector, reducing loop iterations from 16 to 4 and widening warp-level memory transactions from 64 to 128 bytes for better L2 cache utilization

## Key Lessons

- Replace scalar half loads with half2 vectorized loads in GEMV/GEMM dot-product loops to halve the number of load instructions and double effective memory bandwidth per transaction — applies when K is a multiple of 4 and pointers are half2-aligned
- Pack 4 half multiply-accumulate operations per loop iteration (2 half2 loads × 2 lanes each) to reduce loop iteration count by 4x, cutting per-iteration overhead (index arithmetic, branch, loop counter) across all active threads
- Use reinterpret_cast<half2*> to load consecutive half pairs as a single half2 transaction for coalesced 128-byte warp-level memory access instead of scattered 64-byte scalar access
