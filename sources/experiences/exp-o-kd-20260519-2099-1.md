## Solution

Fuse the RoPE computation on k_pe with the KV-cache write into a single kernel: after computing RoPE in registers, immediately write the result to the paged KV cache in the same thread, eliminating the intermediate global-memory store/load round-trip for k_pe and removing one kernel launch entirely

## Key Lessons

- Fuse RoPE with KV-cache write when both operate on k_pe: computing RoPE in registers and immediately writing to the paged cache eliminates one global store/load round-trip per token and one kernel launch overhead — applies whenever a post-processing kernel consumes a tensor that a preceding kernel just produced
- Thread block size for a fused kernel should cover the max of the two original kernel dimensions (max(rope_dim, kv_lora_rank) capped at 512) so that warp-level parallelism is sufficient for both phases without requiring separate launches
- Eliminating inter-kernel global memory traffic is especially impactful for small tensors (k_pe is only ROT_DIM=64 elements per token) where the store/load overhead dominates the actual compute cost
