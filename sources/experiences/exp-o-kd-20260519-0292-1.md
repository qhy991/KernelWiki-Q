## Solution

On SM80+ (Ampere and later), use PTX inline asm max.NaN.f32 which maps to a single FMNMX.NAN hardware instruction that performs NaN-propagating maximum in one operation, falling back to the ternary+isnan expression on pre-SM80 hardware

## Key Lessons

- Use PTX max.NaN.f32 inline asm for NaN-propagating maximum on SM80+ instead of C++ ternary + std::isnan — compiles to a single FMNMX.NAN instruction versus a 4-instruction divergent branch sequence (FSETP.GTU+FSETP.GT+FSEL+BRA), delivering major speedup in compute-bound reduction kernels
- Guard SM80+ PTX intrinsics with #if defined(__CUDA_ARCH__) && (__CUDA_ARCH__ >= 800) and keep the generic fallback in the #else branch — ensures correctness across all architectures while exploiting hardware support where available
- Hardware instruction savings matter most in compute-bound workloads (11.88x speedup) but are hidden in memory-bound regimes (1.01x) — prioritize single-instruction replacements in hot reduction loops over memory-bound kernels
