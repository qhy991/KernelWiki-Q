## Solution

Reduce kCount from 8 to 4 to double grid blocks (16 to 32) and halve per-thread register footprint, though performance remains neutral because both variants are memory-bound and already achieve near-peak streaming bandwidth

## Results

- **speedup**: 0.997x (neutral)


## Key Lessons

- Reducing per-thread accumulator count (kCount) in memory-bound split-K reduction kernels does not improve performance when both variants already achieve near-peak streaming bandwidth — the bottleneck is memory throughput, not register pressure or occupancy
- Grid configurations that already provide enough blocks to fill all SMs yield equivalent performance for streaming memory workloads; increasing block count beyond SM saturation provides no measurable benefit when the kernel is strictly memory-bound
