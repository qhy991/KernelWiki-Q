## Challenge

QKV and cos/sin data are loaded synchronously from global memory in separate phases, causing the warp to stall on each load without any overlap between memory transfers and computation


## Solution

Use cp.async two-group pipelining: group 0 async loads QKV tiles into shared memory, group 1 async loads cos/sin into shared memory, then wait_group<1> allows QKV processing to begin while cos/sin transfers are still in flight, overlapping memory latency with RMSNorm computation

## Key Lessons

- cp.async two-group pipelining can overlap independent global memory load phases (QKV vs cos/sin), but the benefit is marginal when baseline occupancy is already high and per-warp computation is lightweight — the async latency hiding covers too small a fraction of total kernel time
- Reducing grid size via multi-head-per-warp (HEADS_PER_WARP=4) trades occupancy for per-warp reuse of shared cos/sin data; this can counterbalance the latency hiding gained from pipelining on already-saturated GPUs
