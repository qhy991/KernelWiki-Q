## Challenge

With 4 tokens, the kernel processes 4x more A-matrix loads per k-iteration, making memory bandwidth the dominant bottleneck. The before code's 8 scalar bf16 loads per matrix access become 8*5=40 scalar loads per iteration (1 B load + 4 A loads, each loading 8 values individually)


## Solution

The after code replaces these with 5 uint4 vector loads, cutting load instructions by 8x. The 3.19x speedup is larger than the single-token case because the increased memory pressure amplifies the benefit of coalesced 128-bit loads: fewer memory transactions means less contention for the L2 cache...

## Results

- **speedup**: 3.189x (improved)
