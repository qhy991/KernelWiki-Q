## Solution

Replace synchronous FLOAT4 B-tile loads with cp.async.ca (async copy from global to shared memory with L2 cache hint), using commit_group/wait_group to allow B-tile transfers to proceed in the background while A-tile loads and BK=16 compute steps execute, hiding global memory latency in the steady-state loop

## Key Lessons

- Use cp.async.ca.shared.global to issue asynchronous global-to-shared memory copies for tile loads, overlapping them with compute via commit_group/wait_group — works when the compute per BK step is large enough (e.g., TM=8/TN=8 gives 64 FMAs/thread/step) to cover the async transfer latency
- Separate B-tile async copy from A-tile register load by issuing cp.async for B first, then loading A into registers; this maximizes overlap because the A load latency is hidden behind the already-in-flight B async transfer
- Use __cvta_generic_to_shared to obtain a shared-memory pointer suitable for cp.async destination addressing, then offset by element stride (i * 4 bytes for float) rather than indexing the shared array directly
