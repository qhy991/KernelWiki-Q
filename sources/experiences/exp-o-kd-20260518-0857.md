## Challenge

Optimization focus: Replace hardcoded pipe_stages (4) with runtime stages parameter: 2 for Turing sm75, 4 for Ampere+; propagate stages through all shared-memory sizing functions.
Problem before: The Marlin MoE WNA16 kernel used a hardcoded compile-time constant pipe_stages=4 for shared memory sizin


## Solution

(pipe_stages * tb_k * tb_n / pack_factor * 4 bytes) alone exceeds 64KB, making the kernel impossible to launch on Turing.
Applied change: The commit replaces the global constant pipe_stages with a runtime stages parameter. For Ampere+ (sm80+), stages remains 4; for Turing (sm75), stages is set to 2.
