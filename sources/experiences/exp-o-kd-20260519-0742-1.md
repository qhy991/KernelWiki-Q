## Challenge

SiLU activation kernel processes all tokens unconditionally, wasting global memory bandwidth (two float4 loads + one float4 store per thread) and compute (expf-based silu) on masked tokens that contribute no useful output


## Solution

Add expert-based early-return filter after the bounds check: threads whose token maps to expert_id==-1 skip all global memory loads, silu computation, and output stores, eliminating wasted work for masked tokens at block granularity with zero intra-warp divergence

## Key Lessons

- Insert early-return guards based on expert/token masks immediately after bounds checks — when each CUDA block maps to exactly one token (BLOCK_SIZE=256, num_vecs=256), the entire block exits coherently with zero warp divergence, yielding near-theoretical speedup proportional to the masked fraction
- Design block-to-token mapping so that BLOCK_SIZE equals hidden_dim/VEC_SIZE, making per-token filtering a whole-block decision rather than a per-thread branch — avoids the divergence penalty that would arise from fine-grained masking within a warp
- For MoE-style workloads where many tokens are routed away from a given expert, the cost of one extra global read (expert_ids array) per block is negligible compared to the saved global loads, compute, and stores for masked tokens
