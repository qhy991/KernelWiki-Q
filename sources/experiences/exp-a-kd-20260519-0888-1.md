## Solution

Replace scalar half loads with vectorized 16-byte loads via vec_t<T,8>::cast_load(), enabling each thread to process 8 half values per memory transaction and reducing the threads-per-group requirement from 64 to 16

## Key Lessons

- Use vectorized loads (vec_t + cast_load/memcpy) to issue 16-byte memory transactions instead of scalar 2-byte loads — reduces total load instructions by sizeof(vec)/sizeof(T)x and improves memory throughput when data is contiguous and type-aligned
- Vectorized loads reduce the threads-per-group requirement (64 to 16 for group_size=64 with half), which enables packing multiple groups per block (16 groups/block, 256 threads) for higher GPU occupancy
