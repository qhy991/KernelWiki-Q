## Solution

Replace scalar __ldg loads and stores with int4 (128-bit) vectorized loads and stores: each int4 transaction moves VEC_SIZE=2 floats at once, cutting memory transactions from 3*d to ~3*d/2, with a runtime alignment check to fall back to scalar for unaligned pointers and a scalar tail loop for remainder elements

## Key Lessons

- Use int4 (128-bit) vectorized loads/stores instead of scalar __ldg + scalar store in memory-bound activation kernels — the wider transactions halve memory operation count and improve cache line utilization, yielding speedups beyond the theoretical vector width factor
- Guard vectorized paths with a runtime alignment check (is_16byte_aligned on all pointers) plus a d >= VEC_SIZE guard, falling back to scalar for unaligned or too-small dimensions — preserves correctness without assuming allocator alignment
- Include a scalar tail loop (vec_end to d) after the vectorized loop to handle remainder elements when d is not a multiple of VEC_SIZE — prevents silent element drops in the output
