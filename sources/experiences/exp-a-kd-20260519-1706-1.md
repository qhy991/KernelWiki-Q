## Challenge

Scalar fused_add_rms_norm kernel loads/stores one __half element at a time, issuing 4096 individual half-width memory transactions per array per token and performing all arithmetic through float casts, resulting in high instruction count and underutilized memory bandwidth


## Solution

Introduce a vectorized _f16Vec<__half,8> wrapper providing 128-bit aligned loads/stores (8 FP16 per transaction), packed half2 arithmetic for add/multiply, and a sum_squares() method that converts pairs to float2 for accumulation — reducing memory transactions by 8x and cutting instruction count while keeping numerically critical variance in float

## Key Lessons

- Wrapping FP16 arrays in an alignas(16) _f16Vec<__half,8> struct enables 128-bit vectorized loads/stores that coalesce 8 half elements into a single memory transaction — applies to any bandwidth-bound kernel over contiguous half-precision arrays
- 
- Vectorization speedup is bounded by grid-level parallelism: with only 64 blocks (small token counts), the GPU cannot saturate all SMs, so the gain comes primarily from reduced instruction count rather than occupancy improvement — expect larger speedups at higher token counts
