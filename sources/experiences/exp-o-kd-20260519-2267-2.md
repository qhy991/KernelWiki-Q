## Challenge

Exclusive prefix sum of aligned expert counts is computed serially by a single thread (thread 0) iterating over all num_experts entries, creating a sequential bottleneck of 64 iterations while other threads remain idle


## Solution

Replace the serial two-phase prefix sum (per-expert column scan by num_experts threads, then serial cumsum by thread 0) with CUB BlockScan::ExclusiveSum across all 1024 threads, fusing the alignment (DIV_CEIL rounding) into the per-thread input so the scan produces the final aligned cumsum in O(log n) steps

## Key Lessons

- CUB BlockScan::ExclusiveSum replaces serial prefix sum loops with O(log n) parallel scan, eliminating single-thread bottlenecks when the reduction count fits within a single block — applies whenever a small-array cumsum is computed by one thread while others wait
- Fuse alignment or transformation logic (DIV_CEIL rounding) into the per-thread input to BlockScan rather than performing it as a separate pass, reducing kernel phases and shared memory round-trips
