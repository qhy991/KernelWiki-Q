## Solution

Introduce a compile-time maxBlockSize template parameter: when maxBlockSize > WARP_SIZE, right-size shared memory to maxActiveLanes = ceil(maxBlockSize/WARP_SIZE) and use a partial-warp reduce of _nextPow2(maxActiveLanes) lanes for the final step; when maxBlockSize <= WARP_SIZE, an if constexpr branch skips shared memory and __syncthreads entirely, collapsing to a single partial-warp reduce

## Key Lessons

- Add a compile-time maxBlockSize template parameter to block reduction primitives so shared memory and final-warp shuffle width can be right-sized at compile time — eliminates wasted shared slots and excess shuffle iterations when the actual block count is known
- Use if constexpr to short-circuit blockReduceSum to a single partial-warp reduce when maxBlockSize <= WARP_SIZE, removing shared-memory allocation, __syncthreads, and the cross-warp round-trip entirely — most beneficial for small kernels with blockDim <= 32
- Pass _nextPow2(actualCount) as a template argument to warpReduceSum so the shuffle reduction uses only the needed lanes instead of always iterating over all 32 — reduces unnecessary __shfl_xor iterations when fewer warp-level results exist
