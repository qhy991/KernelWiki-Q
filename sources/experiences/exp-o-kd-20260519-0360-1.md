## Solution

Replace scalar half loads/stores with half2 (32-bit) vectorized memory transactions via the HALF2 reinterpret_cast macro, processing two half elements per thread in a single load/store pair and halving the total number of memory instructions

## Key Lessons

- For memory-bound half-precision elementwise kernels, vectorizing to half2 (32-bit) loads/stores halves the number of memory transactions and significantly improves bandwidth utilization — the computation per element stays the same but memory throughput doubles
- Use reinterpret_cast<half2*> to load two adjacent half values in one 32-bit transaction (HALF2 macro pattern), apply the scalar operation to each .x/.y lane, then store back the same way — this avoids needing native half2 arithmetic intrinsics while still getting vectorized memory access
- Halve the block size (256→128) when switching from scalar to half2 vectorization to keep total elements processed per block constant, avoiding register pressure increase
