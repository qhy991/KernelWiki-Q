KernelBench level_1 task_10: Batched matrix multiplication (N, M, K) x (K, L) -> (N, M, L) with N=16, M=1024, K=2048, L=768

## Key Lessons

- 64x64 tiles with 4x4 register tiling is a good balance for this problem size
- Float4 vectorization requires careful alignment - naive implementation breaks correctness
- Shared memory limits (48KB on sm_80) constrain maximum tile size
- PyTorch matmul uses cuBLAS which is hard to beat for standard GEMM
- Register tiling significantly reduces shared memory bandwidth pressure
- Boundary handling with if-checks is necessary for non-multiple dimensions
