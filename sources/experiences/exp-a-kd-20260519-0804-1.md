## Solution

Make the trailing __syncthreads() conditional via if constexpr (start || need_fence), so it is compiled out entirely when both template parameters are false, eliminating the redundant barrier without affecting correctness for other template instantiations.

## Key Lessons

- A single redundant __syncthreads() at kernel exit may produce no measurable speedup when the kernel's dominant cost is shared-memory reduction and global memory access — the barrier overhead is too small relative to total kernel time to be statistically significant at typical measurement granularity.
- Use if constexpr with boolean template parameters to conditionally compile out synchronization barriers that are only needed for specific instantiation paths — this eliminates dead barriers at compile time with zero runtime cost for the paths that don't need them.
