## Solution

Replace the scalar byte-by-byte decomposition and multiply-add loop with the __dp4a intrinsic, which performs a 4-element signed integer dot product in a single hardware instruction (IDP4A on Ampere), eliminating the inner loop, reducing register pressure, and consolidating 4 IMUL+ADD sequences into one instruction per vdr element

## Key Lessons

- Use __dp4a intrinsic for 4-element signed integer dot products instead of manual byte decomposition loops — the hardware unit performs the entire 4-wide multiply-accumulate in one instruction, eliminating byte extraction, sign-extension, and individual IMUL+ADD overhead
- Keep accumulation in integer domain (int sumi) across all vdr iterations rather than casting to float per iteration (float sumf += (float)sumi) — deferring the single float conversion to the end reduces unnecessary F2I/I2F round-trips
- Memory-bound kernels may only show partial speedup from arithmetic simplification (~10% vs theoretical larger gains) — the full benefit of instruction elimination is masked when memory latency dominates, but the register pressure reduction still improves occupancy
