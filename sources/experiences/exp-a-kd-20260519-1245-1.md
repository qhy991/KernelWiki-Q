## Solution

Reinterpret the half input array as vec4_t<half> (8-byte aligned struct of 4 half values) to load 4 elements per single transaction; consecutive threads loading consecutive vec4_t elements produce fully coalesced 32-byte memory transactions (4 threads × 8 bytes), reducing total memory transactions by ~4x

## Key Lessons

- Reinterpret scalar input arrays as vec4_t<scalar_t> (8-byte aligned) to load 4 elements per transaction — consecutive threads then produce coalesced 32-byte memory accesses, reducing transaction count by ~4x for memory-bound kernels
- Use hidden_size >> 2 to compute the vectorized element count and pair with #pragma unroll to minimize loop overhead when the iteration count per thread is small and known at compile time
- Vectorized loads are most impactful for memory-bound reduction kernels (RMS norm, layer norm) where the arithmetic intensity is low and performance is dominated by global memory throughput
