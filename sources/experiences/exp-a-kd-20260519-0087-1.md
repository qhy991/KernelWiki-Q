## Solution

Replace 128 per-element bit extractions with 16 byte-level reads and a simple non-zero check per byte (item || bits), reducing iterations 8x and enabling short-circuit behavior where once item becomes true, subsequent iterations are trivially predicted branches

## Key Lessons

- For logical OR reduction on packed 1-bit booleans, scan raw bytes instead of extracting individual bits — if any byte is non-zero the OR is true, reducing iterations by 8x with simpler per-iteration logic
- Sequential accumulation via item || bits enables hardware branch prediction short-circuiting: once item becomes true the remaining iterations degenerate to trivially predicted branches, further reducing effective work
- When the data is already packed (1 bit per element in consecutive bytes), operate on the native packing granularity rather than unpacking to individual elements — this avoids per-element index/shift/mask overhead entirely
