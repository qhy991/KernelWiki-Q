RMS Norm LLaMA-3-8B hidden_size=4096

## Solution

Remove cudaDeviceSynchronize() from extern C wrapper - the test framework handles synchronization itself

## Key Lessons

- NEVER put cudaDeviceSynchronize() inside the extern C entry point - the KernelEvalPlus test framework handles sync itself, and including it adds ~2.4ms overhead per benchmark call
- The performance_ratio metric is measured at batch_1 (smallest batch), so small-batch latency is critical
- float4 vectorized reads with __ldg and warp shuffle reduction is an effective baseline for rms_norm
- Adaptive thread count (128 for small batch, 256 for medium, 512 for large) helps but the sync removal was the dominant fix
