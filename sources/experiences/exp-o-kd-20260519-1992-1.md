## Solution

Fuse rms_norm and fp8_quant into a single kernel (rms_norm_static_fp8_quant_kernel) that computes variance, normalizes, and quantizes to FP8 within the same thread loop, writing directly to the FP8 output buffer and eliminating the intermediate __half allocation entirely

## Key Lessons

- Fuse a normalization kernel with its downstream consumer (quantization, activation, projection) into a single kernel when the intermediate tensor is only read once — eliminates the global-memory round-trip and one kernel launch, which is especially significant at small-to-medium batch sizes where launch overhead is proportionally large
- When a producer writes to an intermediate buffer in one format (e.g., __half) only for the consumer to immediately read and convert it, keep the data in registers and apply the conversion inline — the intermediate buffer becomes pure waste
