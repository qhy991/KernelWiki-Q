## Solution

Replace per-iteration runtime cudaGetDeviceProperties lookup with a compile-time constexpr int kMaxVecBytes = 16 constant, eliminating the redundant host-side query entirely; the GPU kernel is unchanged since the queried value was never passed to it

## Key Lessons

- Prefer compile-time constexpr constants over runtime cudaGetDeviceProperties queries for architecture-dependent parameters (e.g., max vector width, shared memory capacity) when the target architecture is known at compile time — eliminates host-side overhead and enables compiler optimization
- Dead host-side variables computed from runtime queries but never passed to kernels have zero GPU performance impact; CUDA event timing only measures GPU execution, so removing such dead code is a cleanliness improvement, not a throughput optimization
