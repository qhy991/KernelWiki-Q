## Solution

Fuse the three-kernel gather-scale-reduce pipeline into a single kernel where each thread accumulates the weighted sum in a register across the topk loop, reading input directly via permutation index and writing only the final result — eliminating two intermediate global memory buffers and two redundant memory passes

## Key Lessons

- Fuse sequential kernels that produce intermediate buffers into a single kernel — register accumulation eliminates global memory round-trips for temporary data, which dominates cost in memory-bound pipelines
- Eliminating intermediate buffers reduces memory allocation pressure and avoids redundant write-then-read cycles across kernel boundaries where the compiler cannot optimize away global memory traffic
- For memory-bound multi-stage pipelines (gather → scale → reduce), speedup from fusion is proportional to the number of eliminated global memory passes — removing 2 intermediate writes and 2 redundant reads yields an order-of-magnitude improvement
