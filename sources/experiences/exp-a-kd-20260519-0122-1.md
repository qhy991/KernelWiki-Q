## Challenge

The old int32_t-only warp_uniform API requires callers to manually type-pun float values through memcpy into int32_t, call the int32_t overload, then memcpy back — adding boilerplate, a separate wrapper function, and obscuring intent


## Solution

Replace the int32_t-only overload and manual memcpy wrapper with a single templated warp_uniform<T> that uses an anonymous union { T value; uint32_t asInt; } for inline type punning, eliminating the need for a separate wrapper and making the API generic over any 4-byte type

## Key Lessons

- Templated warp_uniform<T> with anonymous-union type punning compiles to identical SHFL.U32 PTX as memcpy-based punning — nvcc optimizes both paths identically, so prefer the template for API cleanliness without performance risk
- For 4-byte types (float, int32_t, uint32_t), no masking or packing is needed when passing through __shfl_sync — the raw uint32_t representation is sufficient for warp-wide broadcast
