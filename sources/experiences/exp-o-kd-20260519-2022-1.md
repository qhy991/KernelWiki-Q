## Solution

Convert topk to a compile-time template parameter with #pragma unroll to fully unroll the inner loop (eliminating branch overhead and enabling the compiler to overlap independent loads), and replace plain global loads with __ldg to route reads through the read-only cache path, avoiding L1 data cache pollution for the strided 8KB-spaced accesses

## Key Lessons

- Make small iteration counts (e.g. topk=4) compile-time template parameters with #pragma unroll to eliminate loop overhead and expose independent memory operations for ILP scheduling — the benefit is largest when the loop body is a single load-add dependency chain
- Use __ldg() for read-only global memory accesses whose stride exceeds a cache line (here 8KB between topk slots) to route loads through the read-only/texture cache and avoid polluting the L1 data cache — this is especially effective when multiple such loads are unrolled and issued simultaneously
- Combining compile-time unrolling with __ldg is synergistic: unrolling gives the compiler 4 independent __ldg operations to schedule in parallel through the read-only cache, whereas a runtime loop forces sequential issue and prevents cache-path specialization
