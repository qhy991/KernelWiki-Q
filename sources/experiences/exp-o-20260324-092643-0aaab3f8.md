fp32_rms_norm_llama3_8b_hs4096

## Solution

Register blocking: load all 8 float4 input values into registers during phase 1 sum-of-squares computation, then reuse those register values in phase 2 normalization instead of re-reading from global memory

## Key Lessons

- Register blocking (caching input in registers) eliminates second memory pass and improves performance from 2.461x to 2.647x
- 64 threads is too few for hidden_size=4096 - reduces GPU occupancy and parallelism, causing regression to 2.342x
- 128 threads remains the sweet spot for this workload
- For two-pass algorithms (sum then normalize), storing intermediate values in registers avoids redundant global memory reads
