## Challenge

Each thread independently performs an O(E) linear scan through a prefix-sum offset array to find the expert assignment for a token, reading up to 64 entries serially with per-thread branching and no warp-level cooperation


## Solution

Replace per-thread O(E) linear scan with warp-cooperative search where all 32 lanes simultaneously probe one offset entry each, use __ballot_sync to vote on which entries satisfy the predicate (offset <= token_id), and __clz on the ballot mask to find the last qualifying entry in O(E/32) iterations

## Key Lessons

- Use __ballot_sync + __clz to convert per-thread O(N) linear scans into warp-cooperative O(N/32) searches where each lane probes one element in parallel and the ballot mask encodes all predicate results for instant aggregation
- Warp voting intrinsics (__ballot_sync, __any_sync, __all_sync) enable single-instruction aggregation of 32 boolean conditions across a warp, eliminating serial branching and redundant memory reads when searching sorted or prefix-sum arrays
- When all warp lanes read contiguous elements from the same array, the hardware coalesces the 32 reads into fewer memory transactions, yielding additional bandwidth savings beyond the algorithmic reduction from O(E) to O(E/32)
