## Solution

Replace per-thread counter + cumsum lookup with a single atomicAdd on shared-memory local_offsets[expert_id], which returns the current offset as the rank and increments it atomically in one operation — reducing three global-memory accesses per token to one shared-memory atomic

## Key Lessons

- For parallel sorting where each element needs a unique rank within its bucket, atomicAdd on a shared-memory offset array returns the current offset as the rank and increments it in one operation — replaces separate counter-read, cumsum-lookup, and counter-write with a single atomic
- Shared-memory atomicAdd is well-suited for token-to-bucket assignment in MoE sorting because conflicts are naturally distributed across experts and shared-memory atomics are orders of magnitude faster than global-memory atomics
