Metal FP16 GEMM optimization - Round 2: Implemented 32x32 tiling with 2x2 thread coarsening (each thread computes 4 output elements).

## Key Lessons

- Thread coarsening requires careful shared memory indexing - each thread must load enough elements to fill the entire tile
- With BLOCK_ROWS=2 and BLOCK_COLS=2, each thread loads 4 elements from each matrix
- 32x32 tiles with 16x16 threads = each thread handles 2x2 region of the tile
