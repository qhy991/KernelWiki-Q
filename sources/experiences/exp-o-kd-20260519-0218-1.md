## Solution

Replace two separate Gemm kernel launches with a single fused B2bGemm kernel that keeps GEMM0 output in the register file for direct consumption by GEMM1, using smaller threadblock shapes (64x64x32 and 64x128x32) to ensure the intermediate fits in RF

## Key Lessons

- Fuse back-to-back GEMMs with elementwise epilogues (ReLU) into a single B2bGemm kernel when the intermediate tensor fits in the register file — eliminates DRAM round-trip and kernel launch overhead, particularly beneficial for small problem sizes where launch overhead dominates
- Use smaller threadblock tile sizes (e.g., 64x64 instead of 128x64) when fusing kernels for RF residency — smaller tiles reduce register pressure per threadblock, enabling the intermediate result to stay on-chip without spilling
