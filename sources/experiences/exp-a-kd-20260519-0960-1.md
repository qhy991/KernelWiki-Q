## Solution

Extract the inline block reduction logic (shared memory allocation, lane/warp ID computation, warpReduceMax + __syncthreads + second warpReduceMax) into a standalone __forceinline__ __device__ blockReduceMax() utility function, called in one line from the kernel

## Key Lessons

- Extracting inline block reduction logic into a __forceinline__ __device__ utility function improves code readability and reusability with zero performance cost — NVCC inlines the function identically, producing bit-equivalent PTX/SASS
- For small-grid kernels (e.g., 32 blocks on 8K elements), the block reduction phase contributes negligible overhead relative to the global memory scan phase, so refactoring the reduction has no measurable timing impact
