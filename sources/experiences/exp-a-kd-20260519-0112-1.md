## Solution

Replace contended atomicAdd with per-warp disjoint slots in addition_storage[] (one slot per row per warp), then perform a deterministic serial reduction where one lane per row reads all warp contributions via an unrolled loop — eliminating atomic lock/retry overhead in favor of pipelined shared-memory reads

## Key Lessons

- Replace contended atomicAdd on shared memory with per-warp disjoint storage slots followed by a deterministic serial reduction — shared-memory reads are pipelined and avoid the lock/retry overhead inherent to atomics, yielding a net win even with extra smem usage and one extra reduction pass
- When the number of cross-warp contributors is small (e.g., 4 warps), an unrolled loop reading per-warp slots is cheaper than N atomic operations — the O(N) serial reads have deterministic latency while atomics suffer unbounded contention
- Partition shared memory by warp ID (index as row + num_rows * warp_id) to guarantee write-disjointness and enable contention-free accumulation without synchronization primitives
