## Solution

Replace the software SoftFP8E4M3FN type with NVIDIA's native __nv_fp8_e4m3 hardware type, which maps float-to-fp8 conversion directly to the single cvt.rn.fp8_e4m3.f32 PTX instruction, eliminating all bit manipulation branches and reducing ~20 instructions per element to 1 hardware instruction

## Key Lessons

- Use native hardware FP8 types (__nv_fp8_e4m3/__nv_fp8_e5m2) instead of software bit-manipulation conversion — the cvt.rn.fp8_e4m3.f32 PTX instruction handles rounding, clamping, and subnormal cases in a single hardware operation, eliminating ~20 instructions and multiple conditional branches per element
- FP8 conversion savings are proportionally small when the kernel's dominant cost is elsewhere (memory loads, reductions, stores) — at group_size=128 with only 8 conversions per thread, the ~150 saved instructions are a minor fraction of the ~300-400 total instructions per thread
