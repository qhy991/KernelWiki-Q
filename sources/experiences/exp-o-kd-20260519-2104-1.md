## Challenge

Persistent cooperative GEMM scheduler assigns whole tiles to grid-threads in a strided loop with cg::grid_group synchronization between tile iterations, incurring heavy barrier overhead that dominates runtime for 256 tiles across 108 SMs


## Solution

Replace whole-tile persistent scheduling with StreamK-style decomposition that breaks each tile into 64 independent K-iteration units (16384 total), distributing them across all 108 SMs without inter-tile grid synchronization; partial results are accumulated via atomicAdd into the output matrix

## Key Lessons

- Decompose tile-level scheduling into K-iteration-level granular work units to eliminate inter-tile grid synchronization barriers in persistent kernels — the finer granularity removes the dominant overhead when tiles exceed SMs and the tail wavefront causes load imbalance
- Use atomicAdd for partial-result accumulation when tiles are split across SMs in StreamK-style scheduling — the atomic cost is negligible when amortized over large K dimensions (K=2048 multiply-adds per output element)
- Prefer work-unit counts proportional to tiles × (K/TK) over raw tile counts for cooperative kernel scheduling, producing ~64× more schedulable units that improve SM utilization and load balance across wavefronts
