## Challenge

MOE gate rescale uses two separate kernel launches — one for normalizing weights by row sum and a second for multiplying by routed_scaling_factor — incurring double kernel dispatch overhead and forcing an intermediate global memory write from the normalize kernel that the scale kernel must re-read


## Solution

Fuse the normalize and conditional scale operations into a single kernel (rescale_fused_kernel) that reads each element once, divides by the row sum, and conditionally multiplies by routed_scaling_factor in the same loop iteration — eliminating the second kernel launch and the redundant global memory load-store cycle

## Key Lessons

- Fuse sequentially-dependent elementwise kernels (normalize → scale) into a single kernel launch to eliminate kernel dispatch overhead and redundant global memory round-trips — applies whenever two kernels operate on the same data in a producer-consumer pattern with no intervening synchronization
- Add a boolean flag (apply_routed_scaling_factor_on_output) to make the fused kernel conditionally skip the scaling step, preserving correctness when the scaling factor should not be applied without needing a separate code path
