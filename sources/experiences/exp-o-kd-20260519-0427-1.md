## Solution

Replace synchronous B-matrix global-to-shared FLOAT4 stores with cp.async.ca (copy asynchronous, cache-allocate) PTX instructions using commit/wait groups, so that B-matrix DMA transfers overlap with A-matrix register loads and computation in the same loop iteration

## Key Lessons

- Use cp.async.ca.shared.global to issue asynchronous global-to-shared copies instead of synchronous FLOAT4 loads — the hardware DMA engine handles the transfer while the SM continues executing independent instructions, hiding memory latency
- Use commit/wait groups (CP_ASYNC_COMMIT_GROUP / CP_ASYNC_WAIT_GROUP) to fence async copies: commit marks the batch, wait(n) stalls until only n groups remain outstanding, enabling fine-grained overlap control
- Convert shared memory pointers to generic-to-shared via __cvta_generic_to_shared before passing to cp.async PTX — the async copy intrinsic requires a shared memory address in the generic address space
