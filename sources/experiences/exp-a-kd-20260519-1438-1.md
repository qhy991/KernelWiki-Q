## Solution

Introduce a lazy-clone pattern: alias the original buffer via a shallow pointer (d_topk_ids_for_sort = d_topk_ids) and only allocate+copy a clone when expert_map is actually present, skipping the cudaMalloc/cudaMemcpy/cudaFree overhead in the no-expert_map path.

## Key Lessons

- Lazy-clone (shallow alias first, clone only when mutation is needed) is a correct structural refactor for MoE topk-id preprocessing — when expert_map is present the clone still occurs so the hot path is unchanged, but the no-expert_map path avoids cudaMalloc/cudaMemcpy/cudaFree overhead entirely
- Neutral speedup in the clone-mandatory path confirms that introducing the aliasing indirection (one extra pointer variable and branch) adds negligible overhead compared to the dominant cost of the device memcpy and kernel launch
