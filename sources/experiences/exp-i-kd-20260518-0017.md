## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: SM90 (NVIDIA Hopper); Shared memory layout determined by SmemLayoutA and SmemLayoutB; TMA transaction bytes must be computed from tensor sizes

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: SM90 (NVIDIA Hopper); Shared memory layout determined by SmemLayoutA and SmemLayoutB
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
