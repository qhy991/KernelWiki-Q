## Solution

Remove the smemLogits[kTopK] shared-memory buffer entirely; only store indices in shared memory (smemIndices) during Phase 1 and Phase 2, then read logits directly from global input using the selected indices as a lookup table in the final store, eliminating ~4096 shared-memory float writes per row and reducing per-block shared memory from ~20 KB to ~12 KB

## Key Lessons

- Removing a redundant shared-memory intermediary buffer (smemLogits) and instead reading output data directly from global memory via index lookup can reduce shared-memory pressure and instruction count, even when the net speedup is neutral because occupancy is already limited by the thread-count cap (2048 threads/SM) rather than shared-memory capacity
- When occupancy is already capped by a different resource (thread count), reducing shared-memory usage alone may not yield measurable speedup — the freed shared memory does not translate into more concurrent blocks per SM
- Reading selected elements directly from global input (gathering via index) leverages L2 cache residency from the earlier histogram-building pass, avoiding a write-then-read shared-memory round-trip for data that was already in cache
