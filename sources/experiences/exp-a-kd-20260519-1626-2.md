## Solution

Replace scalar scale+clamp+store loop with bfloat162 vectorized intrinsics: __hmul2 for per-pair scaling, __hmin2/__hmax2 for per-pair clamping, and __nv_fp8x4_e4m3 to pack 4 bf16 values into a single 32-bit FP8x4 store, eliminating 4 scalar stores in favor of one coalesced write

## Key Lessons

- __nv_fp8x4_e4m3 packs 4 bf16 values into a single 32-bit store, reducing 4 scalar STG instructions to 1 — applies whenever writing 4+ contiguous fp8 values from bf16 intermediate results
- __hmul2 / __hmin2 / __hmax2 operate on two bf16 values in one instruction, replacing two scalar fmaxf/fminf calls — use bfloat162 vector intrinsics for any per-element arithmetic that can be paired on contiguous data
