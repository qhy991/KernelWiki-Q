## Solution

Fuse both GEMMs into a single B2bGemm kernel using register-file residency: GEMM0's accumulator stays in registers, FragmentIteratorA1 slices it in-place while applying the epilogue (scale + bias + ReLU) inline during load, feeding GEMM1's warp MMA directly without intermediate global memory access or a second kernel launch.

## Key Lessons

- Back-to-back GEMMs where one's output feeds the next's input can be fused via CUTLASS B2bGemm with RF (register-file) residency, keeping the accumulator in registers and applying the epilogue inline — eliminates intermediate global memory traffic and kernel launch overhead, applicable when the intermediate tensor fits in the register file
- When fusing GEMMs with RF residency, widen the first GEMM's WarpShape to align its accumulator N dimension with the second GEMM's K dimension (e.g., WarpShape0 N from 32 to 64 to match GEMM1 K=64), ensuring FragmentIteratorA1 can directly slice the register-held accumulator without shared-memory staging
