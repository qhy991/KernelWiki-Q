## Challenge

Element-wise logical AND reduction on packed 1-bit booleans extracts each bit individually via byte-index, bit-shift, and mask (storage[byte_idx] >> bit_idx) & 1) across all N=128 elements, incurring 128 loop iterations with per-element bit extraction overhead and narrow memory accesses


## Solution

Reinterpret the packed 1-bit storage as a byte pointer and scan entire bytes: if any byte is 0x00 (all 8 bits false), the logical AND result is false. This reduces 128 element-level iterations to 16 byte-level checks (8x fewer iterations), eliminates bit-shift/mask extraction, and uses wider byte-width memory accesses that the GPU issues more efficiently

## Key Lessons

- For packed sub-byte data types, operate on the native container width (bytes) instead of extracting individual elements — this reduces loop iterations by the packing factor (8x for 1-bit types) and eliminates bit-shift/mask instruction overhead
- A logical AND over N packed 1-bit values can be computed by checking if any storage byte equals 0x00 — equivalent to testing whether any group of 8 consecutive bits contains a zero, which is a simple integer comparison rather than N separate boolean operations
