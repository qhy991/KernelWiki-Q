## Challenge

The before code unconditionally computes exp2f for every row on every iteration even when the running maximum mi has not changed, wasting transcendental hardware cycles on redundant rescale-factor calculations


## Solution

Add a conditional changed = (m_prime_id < mi_id) guard that skips the exp2f computation entirely when the row maximum is unchanged, setting out_rescale = 1.0 instead; also adds a NaN guard for the -inf initialization case by resetting mi to 0.0

## Key Lessons

- Guard expensive transcendental operations (exp2f, log2f) with a conditional check on whether the input actually changed — skipping exp2f when out_rescale would be 1.0 avoids ~20-cycle transcendental latency per row per iteration
- Add NaN guards for -inf initialization values that can propagate through exp2f(0 - (-inf)) = exp2f(+inf), producing NaN in downstream accumulation
