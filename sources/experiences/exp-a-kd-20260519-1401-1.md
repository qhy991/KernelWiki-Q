## Solution

Increase max_par from 16 to 64 so that all column slices for N=4096 can be batched into a single kernel launch, eliminating serial launch overhead and allowing the GPU scheduler to distribute all threadblocks across SMs at once

## Key Lessons

- Increasing the host-side parallelism parameter (max_par) to batch more column slices per kernel launch eliminates serial launch overhead — applies whenever the workload dimension can be covered by a single launch without exceeding GPU resources
- Kernel launch overhead (host enqueue + GPU idle between launches) dominates execution time for small-tile GEMM dispatch loops; a single large launch is preferable to many small serial launches when the total threadblock count fits within SM capacity
