## Solution

Lower the dispatch clamp minimum from 32 to 16 so that M=8 maps to next_pow_2(8)=8 which falls into the M16 tile path, selecting the smallest fitting tile configuration and avoiding allocation for unused rows

## Key Lessons

- When dispatching tile-shaped GEMM kernels via power-of-2 sizing, the clamp minimum must match the smallest available tile size — a clamp that is too large forces small M values into oversized tiles, wasting shared memory and prefetching unused rows
- For simple scalar inner-product kernels where the per-element K-loop dominates compute cost, tile size selection has minimal effect on wall-clock time — the benefit of correct dispatch appears in production tiled GEMM with shared-memory prefetching
