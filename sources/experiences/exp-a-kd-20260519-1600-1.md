## Solution

Add an explicit input_stride kernel parameter and replace all hardcoded hidden_size stride calculations with blockIdx.x * input_stride for input reads, making the kernel correct for both contiguous and non-contiguous memory layouts without changing the computational logic

## Key Lessons

- Decouple input stride from hidden size by accepting input_stride as a kernel parameter — ensures correctness for padded/strided tensor layouts (e.g., KV-cache pages, batched sequences with different lengths) while incurring zero performance overhead when stride equals hidden_size
- Distinguish input_token_offset (row start in input, uses input_stride) from token_offset (row start in output/residual, uses hidden_size) to handle cases where input and output have different leading dimensions
