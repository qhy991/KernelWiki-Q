## Solution

Replace scalar loads with vectorized int4 loads where each of 32 lanes loads 16 bytes (128 bits), converts fp8x8→bf16x8 via bf16_8_t struct, and writes 2×int4 per lane; RoPE copy uses int-width warp-cooperative stores (32 lanes × 4 bytes = 128 bytes) with all lanes participating uniformly

## Key Lessons

- Vectorized int4 loads (128-bit per lane × 32 lanes = 512 bytes per warp) replace 512 scalar byte loads for FP8 dequant, achieving 16x wider memory transactions per thread — applies whenever data is contiguous and type-aligned
- Batch fp8x8→bf16x8 conversion via bf16_8_t structs processes 8 FP8 values per call, reducing per-element conversion overhead compared to scalar fp8_to_bf16_scalar in a loop
- Int-width reinterpret_cast copy (32 lanes × 4 bytes) for bf16 RoPE data eliminates the separate 64-thread branch and merges RoPE copy into the uniform 32-lane cooperative pattern
