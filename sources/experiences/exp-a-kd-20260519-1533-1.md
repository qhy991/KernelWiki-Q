## Solution

Fuse RoPE computation on k_pe with the paged KV-cache write into a single kernel — compute the rotary embedding in registers and immediately reinterpret-cast and write the result to the cache offset, eliminating the intermediate global-memory round-trip and one kernel launch

## Key Lessons

- Fusing a compute kernel with a subsequent memory-write kernel eliminates the intermediate global store/load round-trip — applies when the second kernel only reads data produced by the first and writes it to a different memory region
- Kernel launch overhead and the synchronization gap between back-to-back launches is eliminated by fusion — most impactful when each individual kernel is lightweight relative to launch cost
- Thread block size in the fused kernel should cover the maximum dimension across both original kernels (max(rope_dim, kv_lora_rank)) to avoid underutilization, capped at 512 threads for occupancy
