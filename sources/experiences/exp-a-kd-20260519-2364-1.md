## Challenge

Chunked load kernel computes kOffcp = k_str + kOff without bounds checking, which can produce out-of-bounds global memory addresses when K does not perfectly tile with A_CHUNK, leading to undefined behavior and incorrect reduction results for non-tiling K dimensions


## Solution

Add a min__ clamping guard so kOffcp = min__(K - A_CHUNK_VAL, k_str + kOff) ensures the chunked load base offset never exceeds the valid K-dimension range, functioning as a correctness-only safeguard that adds a single always-false comparison when K perfectly tiles

## Key Lessons

- Add min-clamping guards to chunked load offsets (kOffcp = min(K - chunk_size, raw_offset)) to prevent out-of-bounds access when K does not evenly divide by the chunk factor — the guard is free when K perfectly tiles because the comparison is always-false and the compiler may eliminate it
- Observed speedups at sub-10us kernel scale (1.127x on a ~6us kernel) fall within measurement noise and should not be attributed to the code change — validate with larger workloads or statistical tests before claiming performance improvement
- Bounds-checking clamps are correctness optimizations, not performance optimizations — they activate only for edge-case dimensions where out-of-bounds access would occur, producing identical results and negligible overhead in the common (perfect-tiling) case
