## Solution

Reduce cluster shape from 4x4 (16 SMs) to 2x1 (2 SMs) and simplify tile indexing from 2D cluster decomposition to 1D, cutting inter-SM coordination overhead while retaining sufficient parallelism for large-M GEMM

## Key Lessons

- Reducing cluster shape from 4x4 to 2x1 for large-M GEMM cuts inter-SM barrier synchronization overhead by 8x — applies when tile counts already provide sufficient parallelism without needing large cooperative groups
- Simplify cluster indexing from 2D decomposition (cluster_id_x/y + local_id_x/y) to 1D (single cluster_id) when CLUSTER_N=1 — reduces address arithmetic and eliminates unnecessary modulo operations
- Grid dimension calculation must match cluster shape: grid_y should scale by CLUSTER_N only when the cluster spans the M dimension; removing the spurious multiplier corrects the block count when CLUSTER_N=1
