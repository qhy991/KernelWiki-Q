## Challenge

Two separate GPU kernels — a softmax reduce kernel writing float intermediate output to a global memory buffer, followed by a quantize kernel reading that buffer back to apply FP8 scale and write FP8 output — incur an extra kernel launch overhead and a full global memory write-read round-trip on the intermediate float buffer


## Solution

Fuse the reduce and quantize kernels into a single kernel that computes softmax normalization, applies the FP8 out_scale inline (acc *= out_scale), and writes FP8 output directly via __nv_cvt_float_to_fp8, eliminating one kernel launch and the intermediate float buffer's entire global memory write-read round-trip

## Key Lessons

- Fusing a computation kernel with a subsequent quantization kernel into a single pass eliminates both kernel launch overhead and the global memory round-trip for the intermediate buffer — applies whenever a quantization step (FP8, INT8, INT4) follows a float-producing computation and the intermediate data has no other consumers
- Inline FP8 scale application (acc *= out_scale before __nv_cvt_float_to_fp8) replaces a dedicated quantize kernel when the pre-quantized float value is already available in registers — no separate kernel or intermediate global memory storage is needed
