## Challenge

The scalar kernel computes head_idx = i / head_size and head_offset = i % head_size inside every loop iteration to build a multi-stride target index, even though the NHD layout (head_stride == head_size) means the source-to-destination mapping is actually linear, wasting arithmetic throughput on redundant division and modulo operations


## Solution

Pre-compute source and destination base pointers once outside the loop using token_idx and slot arithmetic, reducing the inner loop to a plain linear copy with no per-element div/mod or multi-stride index computation

## Key Lessons

- When head_stride equals head_size (NHD layout), the multi-stride index mapping block_offset*page_stride + head_idx*head_stride + head_offset collapses to a simple linear offset, making per-element div/mod arithmetic entirely redundant → recognize when layout parameters create linear mappings and pre-compute base pointers instead
- Hoist invariant address arithmetic out of loops by computing source and destination base pointers once before the loop body, then iterate linearly → eliminates repeated integer division/modulo and multi-term address computation from the hot path
