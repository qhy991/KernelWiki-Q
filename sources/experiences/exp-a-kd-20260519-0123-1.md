## Solution

Replace the int32_t-only API and manual punning wrapper with a templated warp_uniform<T> that encapsulates type punning internally via an anonymous struct containing a union of T and uint32_t, eliminating caller-side bit manipulation while producing identical PTX

## Key Lessons

- Templating warp-level primitives over the value type with an internal union-pun to uint32_t eliminates per-type wrapper boilerplate while generating identical PTX — the compiler already optimizes zero-extend and mask on sub-32-bit shuffle operands
- When a __shfl_sync operates on a 32-bit register containing a 16-bit value, the upper 16 bits are undefined but irrelevant; reading back only the lower bits via the union's typed member preserves correctness without masking
