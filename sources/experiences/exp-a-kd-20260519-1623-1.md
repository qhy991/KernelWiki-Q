## Challenge

SiLU+mul+FP8 quantization kernel reads gate and up projections directly from global memory in each token iteration, creating a serial load-then-compute dependency that stalls the pipeline and underutilizes available memory bandwidth


## Solution

Introduce cp.async.cg.shared.global PTX instructions to initiate asynchronous 16-byte global-to-shared memory copies, managed by a 3-stage shared memory buffer with cp_async_fence/cp_async_wait_group synchronization, so that each compute iteration reads from shared memory while the next iteration's global load is already in flight

## Key Lessons

- Use cp.async.cg.shared.global with multi-stage shared memory buffers to overlap global memory loads with compute — the async copy engine operates independently of SM compute units, hiding global memory latency when compute and load stages are balanced
- Prefetch NUM_STAGES-1 iterations before entering the main loop, then issue one async load per compute iteration, to ensure the pipeline is always full and compute never stalls waiting for data
- Use cp_async_commit_group + cp_async_wait_group<N> (not __syncthreads alone) to order async copies — wait_group<N> only blocks until the Nth-most-recent group completes, allowing newer copies to remain in flight
