## Challenge

Attention state merge uses two separate kernel launches: merge_attn_states_kernel writes BF16 to an intermediate global memory buffer, then quantize_fp8_kernel reads that buffer to produce FP8 output, incurring an extra kernel launch overhead and an unnecessary BF16 round-trip through global memory


## Solution

Fuse FP8 quantization into the merge kernel by adding a compile-time USE_FP8_OUTPUT template flag and output_t type parameter: after computing the weighted merge in float, convert directly to FP8 via float_to_fp8e4 with a device-side scale factor, eliminating the intermediate BF16 global memory buffer and the second kernel launch entirely

## Key Lessons

- Fusing a post-processing kernel (quantization) into the compute kernel eliminates intermediate global memory round-trips and kernel launch overhead — applies whenever a producer kernel writes results that a consumer kernel immediately reads element-wise
- Use compile-time template flags (bool USE_FP8_OUTPUT) with if constexpr to zero-cost branch between output formats in a fused kernel, avoiding runtime divergence while keeping one kernel implementation for multiple output types
- For small working sets, kernel fusion yields proportionally smaller speedups because kernel launch overhead is a smaller fraction of total time, but the global memory traffic savings from eliminating the intermediate buffer remain
