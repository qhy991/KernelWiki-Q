fusion_large_128q

## Solution

The after path fuses all six output computations into a single kernel where each thread reads seq_len and block_tables once, then derives both the base outputs and the two additional outputs (block_offset+1 and div_ceil(next_seq_len, block_size)) from the same register-held seq_len, block_index, ...

## Key Lessons

- Eliminating one kernel launch saves the second launch's driver overhead entirely, and the fused kernel also avoids re-loading seq_lens_ptr and block_tables_ptr from global memory in a second pass, yielding a consistent ~1.4x speedup across all batch sizes.
