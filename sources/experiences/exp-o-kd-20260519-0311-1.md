## Challenge

MoE gated activation and prequant scale are implemented as two separate kernels with an intermediate temp buffer: the first kernel writes activation results to temp, then the second kernel reads temp, multiplies by scale, and writes to output — costing one full extra global memory read/write round-trip over the entire activation tensor


## Solution

Fuse the gated activation and prequant scale into a single kernel that computes apply_gate_vec and mul_vec in the same vectorized loop, writing the final scaled result directly to the output buffer — eliminating the intermediate temp tensor and one full global memory round-trip

## Key Lessons

- Fuse elementwise kernels that share the same iteration domain and data layout into a single kernel to eliminate intermediate global memory round-trips — applies whenever two kernels pass data through a temp buffer with matching thread/block dimensions
- When fusing kernels that both use vectorized float4 access, the fused loop can chain apply_gate_vec → mul_vec in registers without any extra global traffic, turning two full tensor read-writes into one read (input) + one write (output)
