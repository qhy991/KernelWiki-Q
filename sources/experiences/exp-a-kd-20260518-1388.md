## Challenge

The large-batch config (thread_m_blocks=2, tb_n=64) has larger per-tile buffers that scale linearly with stages. In the before code, sh_a_size = 4 * (32 * 128) * 2 = 32768 bytes and sh_b_size = 4 * (128*64/8) * 4 = 16384 bytes for the stages-dependent portion alone


## Solution

The after code halves these to stages=2, giving sh_a_size = 16384 and sh_b_size = 8192. The stages-dependent terms dominate the total, so the overall shared memory drops by ~42%.
