## Solution

Convert the runtime bool parameter to a template parameter bool is_scale_inverted and use if constexpr inside a ScaledQuant struct specialization, so the compiler emits only the required multiply or divide path at compile time, eliminating per-element branch overhead

## Key Lessons

- Promote uniform runtime booleans to template parameters and dispatch via if constexpr — eliminates per-element branch instructions in tight loops where all threads take the same path, allowing the compiler to optimize the single emitted path more aggressively
- Encapsulate the compile-time dispatched logic in a templated struct (ScaledQuant) with SFINAE specialization to keep kernel launch syntax clean while retaining full compile-time resolution
