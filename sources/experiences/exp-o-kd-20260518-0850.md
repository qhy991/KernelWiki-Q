## Key Lessons

- This was suboptimal because: (1) the 64-wide N-tile wasted computation when N is small (e.g., 1280), (2) the 128-deep K-tile had lower arithmetic intensity than a 256-deep tile, and (3) the Pingpong schedule adds overhead without benefit for small-M kernels where there is not enough work to overlap.
- The swap_ab flag is the key enabler that allows the M/N transpose, while the tile tuning extracts additional performance from better memory access patterns and reduced overhead.
