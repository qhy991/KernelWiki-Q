## Challenge

Fused metadata copy kernel passes 13 separate pointer and scalar arguments through the default kernel parameter mechanism (global memory), consuming more launch parameter space and preventing the compiler from caching parameter accesses across the kernel's multiple copy loops


## Solution

Bundle all kernel parameters (10 pointers + 3 scalars) into a single FusedMetadataCopyParams struct annotated with __grid_constant__, which places the entire parameter block in constant memory so the compiler knows values are immutable during execution and can aggressively cache and reorder accesses

## Key Lessons

- __grid_constant__ places kernel parameters in constant memory, signaling to the compiler that values are immutable during execution and enabling aggressive caching and access reordering — most beneficial for kernels with many parameters accessed repeatedly across multiple loops
- Bundling many kernel arguments into a single struct parameter reduces launch setup overhead (fewer arguments to marshal) and improves code maintainability, even when the measured speedup is neutral for small launch-heavy workloads
- For kernels where data volume dominates (bs*max_len elements) vs parameter access cost (a few dozen scalars/pointers), __grid_constant__ optimization alone yields negligible speedup — combine with compute or memory optimizations for meaningful gains
