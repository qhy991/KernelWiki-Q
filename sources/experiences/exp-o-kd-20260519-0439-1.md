## Solution

Replace static 3D shared arrays with dynamic extern __shared__ plus manual pointer arithmetic and pre-computed cvta base pointers; the compiler can then fold stage/element offsets into pure register arithmetic scheduled independently from the cp.async pipeline, eliminating per-iteration cvta instructions

## Key Lessons

- Pre-compute __cvta_generic_to_shared() base pointers once before the hot loop instead of calling it per iteration with multi-dimensional array references → eliminates redundant cvta/mov instructions that the compiler cannot hoist when array indices vary per iteration → applies to any cp.async or ldg/sts pipeline that reuses the same shared memory base across loop iterations
- Use dynamic extern __shared__ with manual pointer arithmetic and constexpr stage offsets instead of static multi-dimensional shared arrays → gives the compiler explicit byte-level offsets it can fold into register arithmetic, improving scheduling freedom around async copy operations → applies to multi-stage pipelined kernels where shared memory is partitioned into K stages
