## Solution

Centralize the scale/bias epilogue: write GEMM0 accumulator to shared memory with scale/bias pre-applied (M*K1 operations), then add a __syncthreads() barrier so GEMM1 reads already-transformed A1 values directly, reducing GEMM1 inner-loop FLOPs from 3 to 1 per iteration

## Key Lessons

- In B2B GEMM with elementwise operators (scale/bias, activation), hoist the elementwise transform into a centralized smem epilogue between the two GEMMs — this replaces O(M*N*K) redundant inline operations with O(M*K) one-time operations, a factor-of-N reduction in arithmetic
- The __syncthreads() barrier cost between GEMM0 epilogue and GEMM1 mainloop is negligible compared to the arithmetic savings when the K dimension is large, because the barrier latency is fixed while the redundant FLOPs eliminated scale with M*N*K1
- Shared-memory accumulator pattern (EpilogueSmemAccumulator in CUTLASS) trades register pressure for smem usage and simplifies the second GEMM's mainloop by removing per-warp scale/bias fragment management — applicable whenever an elementwise transform sits between two GEMM stages
