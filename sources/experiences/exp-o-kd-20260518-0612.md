## Solution

exceeds L2 cache capacity, so memory bandwidth dominates. The uint4 vectorized path reduces total memory transactions by 16x: each warp of 32 threads doing uint4 loads produces 4 coalesced 128-byte transactions (32 * 16 = 512 bytes), whereas the byte-by-byte warp would issue 32 separate 1-byte reads

## Key Lessons

- The before kernel processes one byte per thread: each thread reads in[i], applies kNibbleLUT, and writes out[i].
