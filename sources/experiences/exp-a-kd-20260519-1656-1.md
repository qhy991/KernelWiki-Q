## Solution

Replace the entire manual bit-unpacking routine with the vendor-provided __nv_fp8_e4m3 type and static_cast<float>, which compiles to a single hardware instruction on FP8-capable GPUs or a highly optimized software emulation, eliminating the branch and powf call

## Key Lessons

- Use vendor-provided FP8 intrinsics (__nv_fp8_e4m3 / __nv_fp8_e5m2 with static_cast<float>) instead of manual bit-level unpacking — the intrinsic compiles to a single hardware conversion instruction on FP8-capable GPUs, eliminating branches, powf calls, and register pressure
- Manual floating-point format conversion via integer bit-extraction and powf is both slower and harder to maintain than vendor intrinsics — always prefer the SDK type when targeting a known precision format
