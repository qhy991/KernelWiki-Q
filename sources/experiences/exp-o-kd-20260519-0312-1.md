## Challenge

The original doActivationKernel used a 1D launch with a strided inner loop over columns, producing non-coalesced scalar memory accesses. AWQ FC2 prequant scaling was performed in a separate kernel pass, requiring every activation element to be written once by activation then read and written again by applyPrequantScale.


## Solution

Rework doActivationKernel into a 2D row/column launch where blockIdx.x covers token groups and blockIdx.y covers column vectors, with each thread owning one packed vector lane. Wrap all global loads/stores with CUTLASS global_load/global_store via loadVec/storeVec to force 128-bit vectorized transactions. Fuse optional FC2 prequant scale multiplication into the hot loop using compile-time constexpr specialization to eliminate the separate post-processing kernel.

## Key Lessons

- Replace 1D strided column-walk loops with a 2D row/column launch where each thread owns one vector lane — this eliminates strided non-coalesced access and enables CUTLASS global_load/global_store to emit 128-bit vectorized memory transactions
- Fuse prequant scale multiplication into the activation kernel hot loop to eliminate a separate post-processing kernel pass, saving one full round of global memory read and write per activation element
- Use if constexpr with compile-time bool template parameters to specialize away unused bias/scale/prequant branches in the hot loop, removing runtime predicate overhead for configurations that don't need them
