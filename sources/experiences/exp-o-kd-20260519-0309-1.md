## Challenge

Gated activation and per-expert prequant scaling are implemented as two separate kernels: before_activation_kernel writes SiLU-gated output to a temp buffer, then scale_kernel reads the same buffer to apply expert-specific scaling, incurring an extra kernel launch and a full-tensor global memory round-trip between them


## Solution

Fuse the activation gate and per-expert prequant scale into a single kernel so that each thread computes apply_gate_vec then immediately multiplies by the expert's scale vector, eliminating the intermediate temp buffer write, the second kernel launch, and the full-tensor global memory reload

## Key Lessons

- Fusing a producer-consumer kernel pair (activation → scale) into a single kernel eliminates one full-tensor global memory write/read round-trip and one kernel launch — applies when the consumer reads exactly what the producer writes and no other kernel needs the intermediate result
- Eliminating the temp buffer reduces memory pressure by the size of the activation tensor (num_tokens × inter_size × sizeof(float)), which is significant for large batch × hidden-dim configurations
