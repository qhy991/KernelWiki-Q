## Solution

Replace row-based add_tile_offset with add_pointer_offset(kSmemPointerOffset) computed as StorageShape::kCount / kSmemTiles, which derives the same effective stride for simple layouts but generalizes correctly to multi-fragment SMEM layouts where partitions are not row-aligned

## Key Lessons

- Prefer add_pointer_offset (flat element-count offset) over add_tile_offset (2D row-based offset) for SMEM navigation in CUTLASS epilogues — pointer offsets generalize to multi-fragment layouts while tile offsets assume FragmentsPerIteration=1
- When two offset computation methods yield identical effective values for a simple layout (smem_total_count / partitions_k == smem_stride), choose the one whose derivation matches the more general data layout model to future-proof the code
