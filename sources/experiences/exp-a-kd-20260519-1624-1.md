## Solution

Replace the single global scale with per-group dynamic scale computed via warp-level __shfl_xor_sync max reduction: each lane tracks local max_abs during SiLU+mul, the warp reduces to a group_max via 5 shuffle iterations, derives a per-group FP8 scale, and quantizes in a second pass — trading one global memory load for warp shuffle overhead to gain finer quantization granularity

## Key Lessons

- Per-group dynamic FP8 quantization via warp shuffle reduction improves quantization granularity but adds overhead from 5 shuffle iterations and intermediate result storage — for small workloads (H=128, T=32, single warp) the shuffle cost balances the saved global-scale memory load, yielding no net speedup
- Storing intermediate results in a bf16 register array (res[4]) to enable two-pass computation (first pass: compute + track max, second pass: quantize) is necessary when the scale depends on the data but doubles the register pressure per element
- Warp-level __shfl_xor_sync max reduction with log2(32)=5 iterations is a fixed-cost operation regardless of data size — it becomes amortized over larger group sizes or more elements per lane
