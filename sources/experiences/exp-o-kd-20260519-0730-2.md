## Challenge

Expert counting step uses warp-based index decomposition (integer div/mod + 2D indexing into shared_counts) for every atomicAdd, adding unnecessary arithmetic overhead when expert_id directly indexes a flat array


## Solution

Use direct expert_id indexing into a flat shared_counts array (atomicAdd(&shared_counts[expert_id], 1)), eliminating the per-element div/mod and 2D address computation

## Key Lessons

- Use direct flat-array indexing instead of 2D warp-based div/mod decomposition when the index space is naturally contiguous — eliminates integer division/modulo instructions inside hot loops
- When shared memory layout mirrors a natural 1D index (expert_id), prefer 1D indexing; warp-based 2D decomposition only adds value when warp-level bank-conflict avoidance is proven necessary
