## Solution

Redesign the cache layout from packed interleaved [num_blocks, num_heads, head_size/x, block_size, x] to NHD contiguous [num_blocks, block_size, num_heads, head_size], and replace per-h_block threading with stride-based iteration, so that both key and value stores target a single contiguous tgt_idx producing fully coalesced 128-byte transactions

## Key Lessons

- NHD cache layout [num_blocks, block_size, num_heads, head_size] places all head elements for a block slot contiguously, enabling fully coalesced writes when blockDim.x threads write at stride blockDim.x — applies to any flash-attention compatible KV-cache design
- Strided value cache writes (value_dst[i*block_size]) fragment memory transactions into partial 128-byte segments; replacing with contiguous target indexing eliminates the fragmentation and achieves full transaction utilization
- Stride-based thread iteration (for i = threadIdx.x; i < n; i += blockDim.x) with blockDim=min(n,512) provides better occupancy than per-h_block dispatch when the element count is large and regular
