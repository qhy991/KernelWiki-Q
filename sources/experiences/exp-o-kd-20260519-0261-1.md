## Solution

Replace bool kSingleValueIteration with int kMaxK in run_grouped() and run_attention() templates; DefaultFMHAGrouped re-derives kSingleValueIteration as kMaxK <= kKeysPerBlock and forwards kMaxK to MakeCustomMma which computes kStages = min(Stages, ceil(kMaxK / Shape::kK)), capping shared-memory pipeline stages to the minimum needed for the actual head dimension

## Key Lessons

- Replace boolean dispatch flags with the actual numeric bound they approximate — the downstream template can re-derive the boolean while also using the numeric value for finer-grained tuning such as shared-memory pipeline stage count
- Compute kStages = min(MaxStages, ceil(kMaxK / Shape::kK)) in GEMM threadblock-level MMA to cap pipeline depth to the minimum required for the actual problem dimension, reducing SMEM consumption and improving occupancy for small problem sizes
- Carry the head-dimension bound (kMaxK) through the kernel template chain rather than baking a worst-case assumption, so that small-head and large-head workloads each get their optimal pipeline configuration without separate kernel instantiations
