## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: CUTLASS epilogue framework; Mixed precision: FP32 accumulator, FP16 input/output; ArrayPlanarComplex<T, 4> for vectorized access

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: CUTLASS epilogue framework; Mixed precision: FP32 accumulator, FP16 input/output
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
