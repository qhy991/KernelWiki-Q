## Solution

Replace the global-memory per-thread counter matrix with shared memory shared_counts[32][8] partitioned by warp (8 experts per warp), using atomicAdd for counting instead of per-thread exclusive counters, eliminating all global-memory writes during counting and removing the cudaMemset overhead entirely

## Key Lessons

- Partition shared memory by warp (e.g., shared_counts[32][8] = 1KB) to handle up to 256 experts without bank conflicts — each warp owns a fixed slice, enabling atomicAdd without cross-warp contention
- When the global-memory footprint of per-thread counters exceeds shared memory capacity, restructure to use warp-partitioned shared-memory atomics instead of per-thread exclusive counters, trading sequential consistency for orders-of-magnitude lower latency
