## Solution

Replace CUB BlockReduce with a __shfl_xor_sync butterfly XOR reduction scoped to THREADS_PER_ROW=2 threads, reaching argmax consensus through register-to-register exchanges with no shared memory, no tempStorage, and no __syncthreads barriers

## Key Lessons

- For sub-warp reductions where the active thread group is smaller than the warp, use __shfl_xor_sync with a lane_mask parameter equal to the group size — it avoids shared-memory staging and __syncthreads entirely while keeping data in registers
- When only a few threads hold meaningful data (2-of-32 in this case), CUB BlockReduce over the full block wastes shared memory and synchronization; a scoped warp-shuffle reduction limited to the active subgroup is both faster and simpler
