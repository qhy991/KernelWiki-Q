## Challenge

The 256-expert MoE alignment kernel stores a per-thread expert count matrix tokens_cnts[(blockDim.x+1)*256] in global memory, requiring global writes for counting, a multi-row prefix sum across all threads also in global memory, and a host-side cudaMemset to clear the large matrix before every kernel invocation


## Solution

Replace the global-memory tokens_cnts matrix with warp-partitioned shared_counts[32][8] (1KB static shared memory) where each warp owns 8 experts and uses atomicAdd for counting — this aggregates counts per-expert directly, eliminating the separate prefix-sum-across-threads step and the host-side cudaMemset

## Key Lessons

- For MoE token-counting with many experts (256+), warp-partitioned shared memory counting (each warp owns N experts) with atomicAdd eliminates the need for a per-thread count matrix in global memory and the associated prefix-sum-across-threads step — applies when expert count exceeds shared memory capacity for per-thread rows
- Moving the counting data structure from global to shared memory eliminates the host-side cudaMemset overhead required to clear the matrix before each kernel launch — shared memory is re-initialized inside the kernel at near-zero cost
