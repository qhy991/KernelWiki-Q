## Solution

Replace ^row with ^(row % 8) in the XOR remapping to constrain the XOR operand to the warp-tile row range [0,7], ensuring all remapped indices stay within the valid row range and eliminating the need for clamping branches

## Key Lessons

- When using XOR-based shared-memory index remapping for bank-conflict avoidance, constrain the XOR operand to the warp-tile dimension (e.g., row % 8 for 8-row tiles) rather than using the full row index — this guarantees remapped indices remain in-bounds without clamping
- Eliminating conditional bounds-checking branches in shared-memory access loops removes warp divergence and branch overhead, yielding measurable speedup even when the branch was rarely taken
