## Solution

Vectorize global memory access using float4 so each thread loads/stores 4 contiguous elements, halving the total memory instruction count from 2N to N/2 and ensuring all accesses are 128-bit aligned and coalesced

## Key Lessons

- Float4 vectorization reduces memory instruction count by 4x and cuts thread scheduling overhead by 4x for memory-bound elementwise kernels — applies whenever the kernel performs independent per-element operations on contiguous float data
- Grid size must be divided by the vectorization factor (BLOCK*4 instead of BLOCK) to avoid launching 4x too many threads when each thread handles 4 elements
