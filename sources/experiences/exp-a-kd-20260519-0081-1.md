## Solution

Replace the C++ if-branch with an explicit PTX-level predicated store using inline asm (@p st.global.v4.u32), guaranteeing hardware predication rather than relying on the compiler's branch conversion; however, NVCC at -O3 already emits equivalent predicated SASS for simple if-guarded stores, yielding no measurable performance difference

## Key Lessons

- For simple if(guard) { *ptr = val; } store patterns, NVCC -O3 already generates hardware-predicated PTX stores identical to hand-written inline asm — explicit PTX predication only guarantees the behavior but provides no speedup for trivial guards
- When a kernel is memory-bandwidth-bound (each thread stores 64B per iteration across 256K threads), instruction-level optimizations like predication vs branching have negligible effect because the bottleneck is DRAM throughput, not instruction scheduling
- Explicit inline asm predicated stores are valuable for code clarity and guaranteeing predication semantics across compiler versions, but should not be expected to improve performance over simple if-guarded stores that the compiler handles correctly
