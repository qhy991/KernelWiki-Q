## Challenge

Sub-warp 8-lane max reduction uses __shfl_down_sync in a loop from delta=16 to 1 with width=8, executing 5 shuffle instructions where deltas 16 and 8 wrap within the 8-lane segment and return the lane's own value (NOP), wasting 2 shuffle rounds per reduction


## Solution

Parameterize the reduction with a kNumThreads template and switch to __shfl_xor_sync with full warp width=32, starting the mask loop at kNumThreads/2 so only log2(kNumThreads) productive rounds execute (3 for kNumThreads=8), eliminating the 2 NOP shuffle rounds

## Key Lessons

- Parameterize sub-warp reduction loops with the actual group size (kNumThreads) rather than hardcoding delta=16 — starting mask at kNumThreads/2 ensures only log2(group_size) productive shuffle rounds execute, eliminating NOP rounds when group_size < 32
- __shfl_xor_sync with width=32 and a group-scoped mask correctly isolates sub-warp groups for butterfly reduction, unlike __shfl_down_sync with a narrow width which requires extra care to avoid wrapping NOPs at large deltas
- A compile-time kNumThreads template with static_assert(has_single_bit) makes the reduction generic, safe, and optimizable — the compiler unrolls exactly the right number of rounds at compile time
