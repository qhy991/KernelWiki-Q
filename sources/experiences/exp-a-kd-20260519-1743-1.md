## Challenge

Single-query cached KV attention with BLOCK_SIZE=16 creates THREAD_GROUP_SIZE=2, requiring __shfl_xor_sync reduction for QK dot products and a conditional gate (thread_group_offset==0) for logits writes, adding per-iteration synchronization overhead across 512 KV blocks


## Solution

Increase BLOCK_SIZE to 32 (matching WARP_SIZE) so THREAD_GROUP_SIZE=1, enabling each thread to independently compute a full QK dot product with wider uint4 vector loads (VEC_SIZE=8), eliminating shuffle reduction and the conditional gate while halving KV block iterations from 512 to 256

## Key Lessons

- Setting BLOCK_SIZE equal to WARP_SIZE (THREAD_GROUP_SIZE=1) eliminates intra-warp shuffle reduction in QK dot products — each thread independently computes a full dot product with wider uint4 vector loads, removing __shfl_xor_sync overhead from every KV block iteration
- Larger BLOCK_SIZE halves the number of KV block iterations (512→256 for seq_len=8192 at BLOCK_SIZE=32), reducing block-table lookup overhead and loop control cost — applies when paged KV cache granularity allows larger blocks
- THREAD_GROUP_SIZE=1 removes the conditional gate (thread_group_offset==0) on logits writes, enabling all threads to contribute to softmax computation without warp divergence on the store path
