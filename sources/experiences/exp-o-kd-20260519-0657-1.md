## Solution

Fuse all three operations into a single kernel that reads input directly via the permutation index and accumulates factor*val into a register sum_val across topk iterations, eliminating two kernel launches and two intermediate buffer allocations

## Key Lessons

- Fuse sequential kernels that form a producer-consumer chain into a single kernel when intermediate results are never reused — eliminates kernel launch overhead and intermediate global memory traffic
- Accumulating weighted sums in registers inside a fused kernel is especially beneficial at small batch sizes where kernel launch overhead dominates total execution time
- Eliminating intermediate buffers (d_shuffled, d_multiplied) reduces memory allocation overhead and frees GPU memory for other uses, which matters when the pipeline runs repeatedly
