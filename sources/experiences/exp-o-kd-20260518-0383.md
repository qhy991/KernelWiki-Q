## Challenge

In the before code, each element FP8 conversion calls scaled_fp8_div which performs val / scale (a full FP32 division). The after code pre-computes scale_inv = 1.0f / scale once at kernel entry, then each element calls scaled_fp8


## Solution

_mul which performs val * scale_inv (an FP32 multiply). On A800 (Ampere), FP32 division has ~16 cycle latency and 1/4 the throughput of FP32 multiply (~4 cycle latency). With 4.2M elements, replacing N divisions with 1 division p

## Results

- **speedup**: 1.101x (improved)


## Key Lessons

- The effect is larger in the large case because the kernel is more compute-bound (more elements per thread block means the division throughput bottleneck is more exposed).
