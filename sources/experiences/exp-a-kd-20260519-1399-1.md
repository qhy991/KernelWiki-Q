## Challenge

GELU activation is implemented using the exact erf-based formula, which relies on the ::erf SFU instruction; an alternative tanh-approximation using ::tanhf plus extra FMA operations is proposed, but the net effect on performance is unclear due to differing instruction counts and SFU characteristics


## Solution

Replace erf-based exact GELU with the tanh polynomial approximation: 0.5f * f * (1.0f + tanhf(BETA * (f + KAPPA * f^3))), which adds 3 extra FMA operations (x_cube, inner, BETA*inner) but uses tanhf instead of erf as the SFU call — both have similar throughput on sm_80, making the change performance-neutral in memory-bound workloads

## Key Lessons

- In memory-bound kernels processing large tensors (128MB input), swapping one SFU instruction for another plus extra FMA operations has negligible performance impact because the kernel is dominated by DRAM traffic rather than compute throughput
- erf and tanhf have similar SFU throughput on sm_80, so the tanh GELU approximation's additional 3-4 FMA instructions do not create a bottleneck when the activation is a small fraction of total work per element
- Activation function choice (exact vs approximate GELU) should be driven by numerical accuracy requirements, not performance expectations, in memory-bound fused kernels at large problem sizes
