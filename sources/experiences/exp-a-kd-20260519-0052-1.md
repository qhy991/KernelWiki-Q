## Solution

Replace plain integer div/mod with FastDivmod objects that use __umulhi (single multiply-high instruction) plus a shift to compute quotient/remainder in ~4 cycles each, avoiding the ~20-cycle integer divide path entirely

## Key Lessons

- Use FastDivmod (multiply-high + shift via __umulhi) instead of plain integer div/mod for hot-loop index decomposition — avoids the ~20-cycle integer divide pipeline and reduces each decomposition to ~4 cycles on SM80+
- Precompute FastDivmod objects once outside the loop rather than using inline div/mod on every iteration — the one-time setup cost amortizes over all accesses
