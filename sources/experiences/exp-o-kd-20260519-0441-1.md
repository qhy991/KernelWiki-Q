## Challenge

Static shared memory arrays declared as __shared__ float s_a[K_STAGE][BM][BK+A_PAD] are limited to 48 KB total on most GPUs, blocking higher pipeline stage counts (K_STAGE=5+) that would increase global-memory/compute overlap for large-K GEMM workloads


## Solution

Replace static 3D shared memory arrays with a single extern __shared__ float smem[] allocation partitioned via pointer arithmetic into s_a and s_b regions, removing the 48 KB static ceiling and enabling use of up to 96 KB (or architecture maximum) shared memory for higher pipeline stages

## Key Lessons

- Use extern __shared__ with manual pointer arithmetic instead of static __shared__ arrays when shared memory needs approach or exceed 48 KB — static shared memory has a hard 48 KB limit while dynamic shared memory can use up to the architecture maximum (96 KB on Ampere+, 164 KB on Hopper), enabling higher pipeline stage counts
- When switching to dynamic shared memory, must set cudaFuncAttributeMaxDynamicSharedMemorySize via cudaFuncSetAttribute() and pass the byte size as the third kernel launch argument — omitting either step causes silent fallback to 48 KB or launch failure
- Pre-compute stage offsets as constexpr (s_a_stage_offset, s_b_stage_offset) to avoid repeated multi-dimensional index calculations in the hot loop — this keeps indexing overhead comparable to the static array version
