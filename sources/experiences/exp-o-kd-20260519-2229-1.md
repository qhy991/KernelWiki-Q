## Solution

Fuse the absmax computation and quantization into a single kernel that computes per-token absmax via blockReduceMax, shares the result via __shared__ memory with a __syncthreads() barrier, then quantizes using multiplication by precomputed reciprocal tmp_scale = 127.0f / block_absmax_val instead of division by scale loaded from global memory

## Key Lessons

- Fuse multi-pass kernels (compute-scale + apply-scale) into a single kernel when the intermediate result (per-token absmax) fits in shared memory — eliminates kernel launch overhead and global memory round-trip for intermediate data
- Use __shared__ memory + __syncthreads() to broadcast block-level reduction results to all threads within a fused kernel, avoiding the need to write and re-read intermediate values from global memory
- Replace division by a per-block scale with multiplication by precomputed reciprocal (127.0f / absmax) — floating-point multiplication is faster than division and the reciprocal is computed once per block in the fused kernel
