Bit-level half construction for 4-bit dequant zero-point constants (zero=8 variant)

## Solution

Identical fix: half_uint16(0xe400 | 8) for z1 (bit-level OR on known -1024 pattern), __hsub(__int2half_rn(-64), __int2half_rn(8)) for z16, __half2half2 for broadcast. The technique is zero-value-independent — works for any runtime zero-point by ORing into the known base bit pattern.

## Key Lessons

- The bit-level constant construction pattern (0xe400 | zero) works for ANY runtime zero-point value, not just edge cases. The key insight is that -1024.0f in IEEE 754 half-precision is 0xe400, and OR-ing in an integer zero ≤15 modifies only the low 4 bits. This is a general pattern for half-precision dequant constant construction with per-group zero-point offsets.
- Consistency across zero-point values (8 and 15 showing identical 12.6% speedup) validates that this optimization targets the constant setup path itself, not an edge-case specific code path. Constant-setup instruction count reduction applies uniformly regardless of the zero-point magnitude.
