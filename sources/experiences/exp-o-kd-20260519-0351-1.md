## Challenge

Two separate kernel launches for attention — qk_scores_kernel writes the full M×N score matrix to global memory, then softmax_value_kernel reads it back to compute softmax and value aggregation, causing an unnecessary global-memory round-trip for the attention matrix plus an extra kernel launch overhead


## Solution

Fuse Q·K scoring, softmax, and value aggregation into a single kernel using the online softmax algorithm — each thread maintains a running max (m), running normalization term (l), and running output accumulator, updating all three per key slot so the full attention matrix never materializes in global memory

## Key Lessons

- Fuse multi-pass attention (score → softmax → value) into a single kernel using the online softmax trick (running max m and running sum l) to avoid materializing the full attention matrix in global memory — eliminates O(M×N) workspace traffic and one kernel launch
- Online softmax update formula acc = acc * exp(m - new_m) + exp(score - new_m) * v[j*K+d] maintains numerically stable running sums without requiring a two-pass reduction — applies whenever softmax must be computed incrementally over a streaming sequence
- Launch configuration change from 2D grid of thread blocks (one per score element) to 1D grid (one block per query row, one thread per output dimension) maps naturally to the fused kernel's per-query-row data parallelism and eliminates thread divergence from boundary checks in the score kernel
