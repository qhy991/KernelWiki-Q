## Key Lessons

- The speedup is smaller (1.36x vs 3.57x for kDim=128) because each row's computation takes longer (8 elements per thread vs 4), making the block scheduling overhead a smaller fraction of total latency.
- The per-row arithmetic and memory access patterns are identical between before and after — only the launch configuration and block residency differ.
