## Challenge

Branch-based absolute value and max computation in the absmax scan uses ternary operators (val > zero ? val : -val and val > absmax_val ? val : absmax_val), causing warp divergence when threads in the same warp take different paths for sign flip and comparison


## Solution

Replace ternary-operator absmax with fabsf/fmaxf intrinsics which compile to single hardware instructions without branching, eliminating warp divergence in the absmax scan phase

## Key Lessons

- fabsf/fmaxf replace ternary-based absmax with branchless hardware intrinsics, eliminating warp divergence in reduction scans — applies whenever computing elementwise absolute maximum across a vector
- Branchless intrinsics (fabsf, fmaxf) are always preferred over manual ternary (x > 0 ? x : -x) for abs/max on GPU since the hardware has dedicated single-instruction implementations
