## Challenge

Two separate kernel launches — normalize_kernel then scale_kernel — each traverses the full 128KB output array over global memory, forcing the second kernel to re-read every element that the first kernel just wrote, doubling memory traffic and incurring an extra kernel dispatch overhead


## Solution

Fuse normalize and scale into a single fused_rescale_kernel that reads each element once, divides by the row sum, multiplies by the scaling factor in the same thread context, and writes the final result once — cutting global memory traffic by roughly half and eliminating one kernel dispatch

## Key Lessons

- Fuse elementwise kernels that operate on the same data into a single kernel launch to halve global memory traffic — applies whenever kernel B reads output that kernel A just wrote, and the operations can be composed per-element in the same thread
- Kernel launch overhead savings diminish with larger workloads because compute time dominates dispatch cost, but global memory traffic savings remain substantial regardless of problem size
