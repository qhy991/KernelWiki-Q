## Challenge

The before source launches a single block and walks the full key range in one CTA, so only 256 threads contribute while the rest of the GPU sits idle


## Solution

The after source slices the same reduction across 64 split blocks and combines the partial sums with one atomic add per split, which exposes much more parallel work on a long sequence and matches the commit's goal of parallelizing across the key dimension.

## Results

- **speedup**: 17.534x (improved)
