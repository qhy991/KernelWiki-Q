## Solution

Reserve a compile-time fill_threads subgroup (256 threads) for initialization while remaining threads handle counting/alignment concurrently, using three __syncthreads() barriers to synchronize the fill threads with the working threads' phase boundaries so sorted_token_ids is ready before the sorting phase reads it

## Key Lessons

- When a kernel must initialize a buffer it later reads, overlap initialization with computation by splitting threads into fill and working subgroups within the same block — avoids inter-block synchronization while hiding init latency
- Use compile-time template parameters (e.g., fill_threads) for thread-role splits so the compiler can eliminate branch divergence and optimize each path independently
- Match __syncthreads() count between fill and working thread paths to keep all threads synchronized at the same barrier phases, preventing deadlocks in single-block kernels
