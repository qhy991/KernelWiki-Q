## Solution

Propagate DefaultIterators::kFragmentsPerIteration to the epilogue when kPartitionsK==1, setting kFPI=2 for float/half to process two fragments per iteration, halving __syncthreads() barriers from 16 to 8 at the cost of doubling shared memory from 34KB to 68KB and reducing occupancy from 4 to 2 blocks/SM

## Key Lessons

- Increasing kFragmentsPerIteration halves sync barrier count but doubles shared memory consumption — the net speedup depends on whether the compute between barriers is heavy enough (as in real CUTLASS epilogues) to outweigh the occupancy loss from reduced per-SM block count
- CUTLASS epilogue optimizations designed for heavy warp-level accumulator iteration can appear as regressions in microbenchmarks with trivial smem copy loops, because the microbenchmark's compute-to-barrier ratio is too low for barrier reduction to help — validate with realistic workload characteristics before concluding an optimization regresses
