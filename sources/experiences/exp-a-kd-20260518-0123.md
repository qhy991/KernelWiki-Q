## Solution

Replace the int32_t-only overload plus manual punning wrapper with a single templated warp_uniform<T> that uses a union of T and uint32_t internally, letting the compiler handle type representation while issuing the same SHFL.U32 instruction

## Key Lessons

- Templating warp-level primitives over the value type eliminates boilerplate punning wrappers while generating identical PTX — the NVPTX ABI already passes small types in 32-bit registers, so the compiler folds zero-extend and mask into no-ops
- When shuffling sub-32-bit types (__half, __bfloat16) via __shfl_sync, placing them in a union with uint32_t and shuffling all 32 bits is safe: the upper garbage bits are ignored by subsequent half-precision operations that only read the lower 16 bits
