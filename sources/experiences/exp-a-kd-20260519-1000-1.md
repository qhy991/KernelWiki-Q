## Solution

Hoist all three redundant host-side operations (cudaStreamCreate/Destroy, cudaGetDeviceProperties, cudaSetDevice) from per-stage call_stage() to the pipeline_after() caller, passing the stream and is_h20_device as parameters to each stage, eliminating 4x redundant driver round-trips per pipeline invocation

## Key Lessons

- Hoist cudaStreamCreate/Destroy, cudaGetDeviceProperties, and cudaSetDevice out of per-call helper functions to the top-level caller when multiple stages share the same device and stream context — each cudaStreamCreate/Destroy round-trip costs ~10-15us of driver-level context allocation, which dominates latency when kernel work is small
- Pass pre-created streams and pre-computed device flags as function parameters rather than querying/creating them inside each callee — applies to any multi-stage pipeline where the device context does not change between stages
