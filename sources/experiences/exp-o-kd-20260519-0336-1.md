## Solution

Remove all cudaMemset calls from the warmup and benchmark loops since the write_once kernel performs a full write to every output element, making zero-initialization of the output buffer unnecessary before each invocation

## Key Lessons

- Remove cudaMemset on output buffers when the subsequent kernel performs a full write to every element — the memset adds an unnecessary device-memory traversal that the kernel already supersedes, costing ~25% overhead in this benchmark
- When benchmarking a kernel that fully initializes its output, omit any pre-fill of the output buffer in the timed loop; the memset is at best redundant and at worst hides the kernel's true steady-state cost by adding a competing memory pass
