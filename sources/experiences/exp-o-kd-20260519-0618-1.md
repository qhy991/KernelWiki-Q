## Solution

Fuse renormalization into the topkGatingSoftmax kernel by accumulating row_sum_for_renormalize during the top-K selection loop (when thread_group_idx==0) and dividing each output weight by this sum in a post-loop block, eliminating the separate kernel launch and keeping the renormalization in-register since the output values are likely still in L2 cache from the prior write

## Key Lessons

- Fuse lightweight post-processing (renormalization, scaling) into the producer kernel when the output data is likely still in L2 cache or registers — eliminates a separate kernel launch and avoids a full global memory round-trip for small per-row data
- Accumulate a running sum during the top-K selection loop (row_sum_for_renormalize += max_val in the output-writing branch) to avoid a second pass over the data for normalization
- For warp-level kernels where THREADS_PER_ROW=1, the thread that writes the output is the same thread that can renormalize — no cross-thread communication needed for the fused post-processing step
