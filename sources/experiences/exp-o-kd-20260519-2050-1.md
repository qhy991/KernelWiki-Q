## Solution

Change the first parameter from int* to const int* so the caller can pass topk_ids directly when expert_map is absent, cloning only inside the expert_map.has_value() branch where preprocessTopkIdLauncher modifies the data in-place

## Key Lessons

- Prefer const-qualifying pointer parameters when the function only reads the data — this enables callers to avoid unnecessary GPU clones and device-to-device copies for the read-only path
- Conditionally clone GPU tensors only in branches that actually mutate the data (e.g., expert_map preprocessing), rather than unconditionally cloning at the call site — reduces host-side overhead (cudaMalloc + cudaMemcpy) proportional to tensor size
