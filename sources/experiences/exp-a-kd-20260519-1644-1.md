## Solution

Replace named struct members (u0..u7) with a fixed-size array member (d[8]) to enable generic indexed access in loops and template code, while producing identical SASS because NVCC resolves both access patterns to the same memory offsets at compile time

## Key Lessons

- Named vs array struct members for wide register types (u0..u7 vs d[0]..d[7]) produce identical SASS when accessed with constant indices — NVCC resolves both to the same memory offsets, so choose based on code ergonomics not performance
- Array members (d[8]) enable generic indexed-access loops in template code (for j in 0..8) that named members cannot support without macros or hand-unrolling — prefer array layout when the struct will be used in templated or loop-based contexts
