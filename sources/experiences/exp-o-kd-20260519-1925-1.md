## Solution

Always pass the GPU pointer (data_ptr<float>()) directly to the CUTLASS epilogue visitors with a boolean broadcast flag (numel != 1) that indicates whether the scale is a per-channel vector or a per-tensor scalar; the epilogue visitor reads the scalar from GPU memory instead of extracting it to host, eliminating the device-to-host transfer entirely

## Key Lessons

- Never use tensor.item<T>() on GPU-resident scalars inside hot dispatch paths — it triggers a synchronous cudaMemcpy (DeviceToHost) that serializes the stream; instead pass the GPU pointer directly and let the kernel read the scalar from device memory
- A boolean broadcast flag (numel != 1) combined with an always-GPU-pointer pattern unifies per-tensor and per-channel scale handling into a single code path, eliminating branches that cause torch.compile graph breaks when scalars migrate to CPU
