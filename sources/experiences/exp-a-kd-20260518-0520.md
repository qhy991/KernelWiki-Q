## Challenge

GELUTanh involves a more complex computation chain (tanhf plus cubic term x^3), which dominates the execution time. The before code casts bf16->float inside apply_activation, computes the full tanh-based CDF in float32, casts back to bf16, then multiplies bf16*bf16


## Solution

The after code casts both inputs to float, computes activation in pure float32, multiplies float*float, and casts back to bf16.
