## Challenge

GEMM epilogue always loads source data from global memory even when beta=0, wasting 16MB of memory bandwidth per kernel launch since the loaded source values are multiplied by zero and contribute nothing to the output


## Solution

Use compute_source_not_needed_() path that skips all source-related global memory loads, eliminates source_fragment allocation, and switches to a single-argument output operator (alpha*acc) instead of the two-argument form (alpha*acc + beta*src), halving epilogue memory bandwidth from ~32MB to ~16MB

## Key Lessons

- Skip source data loading entirely when beta=0 in GEMM epilogue by dispatching to a compute_source_not_needed_() code path — eliminates wasted global memory bandwidth, reduces register pressure, and avoids unnecessary arithmetic on zero-multiplied values
- Use compile-time or dispatch-level branching to select specialized epilogue paths (with-source vs source-not-needed) rather than runtime beta checks inside the loop — the compiler cannot assume beta=0 from a runtime parameter, so the loads persist as mandatory memory operations
- Halving global memory traffic in a memory-bound epilogue yields nearly proportional speedup — when the kernel is bandwidth-limited, eliminating unnecessary reads is more impactful than any arithmetic optimization
