## Solution

Switch the template parameter kOStorageAccFloat32_ from 0 to 1, keeping accumulators in F32 throughout the rescaling loop to eliminate per-step half conversion instructions

## Key Lessons

- Eliminating a small number of conversion instructions (__half2float/__float2half_rn) in the rescaling path yields no measurable speedup when kernel latency is dominated by MMA computation and global memory traffic — the conversion overhead is negligible relative to arithmetic and memory bottlenecks
- F32 accumulator storage primarily improves numerical accuracy (reduced rounding in softmax rescaling) rather than performance; the FP16/FP32 conversion pipeline on modern GPUs executes quickly enough that instruction count reduction alone is not a reliable optimization lever
