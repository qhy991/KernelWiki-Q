## Solution

Query cudaDevAttrMaxSharedMemoryPerBlockOptin once at static-init time and branch: launch the large-tile kernel when its shared memory fits, otherwise fall back to a smaller-tile (16x16x16) kernel that stays within the device limit

## Key Lessons

- Query cudaDevAttrMaxSharedMemoryPerBlockOptin at runtime to guard shared-memory-heavy kernel launches — the static-init + branch pattern adds zero measurable overhead on devices where the primary path is taken, while preventing launch failures on constrained devices
- Provide a smaller-tile fallback kernel for shared memory portability across GPU generations — tile size reduction trades compute efficiency for compatibility, ensuring correctness when the primary tile exceeds device limits
