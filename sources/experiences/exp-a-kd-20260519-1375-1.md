## Solution

Make TOPK a compile-time template parameter and add #pragma unroll so the compiler fully expands the 2-iteration inner loop into two independent load-add sequences with no loop control or branch overhead

## Key Lessons

- Pass small fixed iteration counts (e.g., top-K) as compile-time template parameters instead of runtime integers, enabling #pragma unroll to eliminate loop overhead and unroll address arithmetic — applies whenever the loop bound is a known constant at kernel launch time
