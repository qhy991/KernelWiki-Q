## Challenge

Blelloch shared-memory exclusive prefix scan for 8 expert counts uses an up-sweep and down-sweep phase totaling 8 __syncthreads barriers, but when scan_size=8 fits entirely within a single warp (8 < WARP_SIZE=32), the intra-warp prefix sum can theoretically be computed without any barriers via warp shuffles


## Solution

Replace the Blelloch shared-memory scan with a two-level warp shuffle exclusive scan using __shfl_up_sync: an intra-warp prefix sum (barrier-free) followed by a cross-warp correction step, reducing total __syncthreads from 8 to 7, though the saving is negligible at this tiny problem size where kernel launch overhead dominates

## Key Lessons

- Warp shuffle-based exclusive scan (__shfl_up_sync) eliminates intra-warp __syncthreads but provides no measurable speedup when kernel launch overhead (~3.5us) dominates at tiny problem sizes — the technique only pays off when scan work is a significant fraction of total kernel execution time
- For scan_size <= WARP_SIZE, a single warp shuffle scan pass suffices without cross-warp coordination, making the implementation simpler than multi-warp Blelloch, but the barrier reduction (8 to 7 __syncthreads) is too small to overcome launch overhead
