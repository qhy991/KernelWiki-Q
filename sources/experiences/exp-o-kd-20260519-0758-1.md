## Solution

Replace scalar bfloat16 loads with 128-bit vectorized loads via uint4 reinterpret_cast, fetching 8 bf16 values per LDG.128 instruction; the inner unrolled loop unpacks and compares all 8 elements, reducing total load instructions by 8x and dramatically improving cache-line utilization

## Key Lessons

- Use uint4 (128-bit) reinterpret_cast loads to batch 8 bfloat16 elements into a single LDG.128 transaction instead of scalar loads — reduces memory instruction count by 8x and saturates cache-line bandwidth when elements are contiguous and 16-byte aligned
- Vectorized load-and-unroll pattern is the standard approach for bandwidth-bound reduction kernels (max/sum) over half/bfloat16 arrays — the inner compile-time-bounded unroll adds negligible register pressure compared to the memory throughput gained
