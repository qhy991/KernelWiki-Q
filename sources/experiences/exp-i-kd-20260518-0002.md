## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: Any CUDA-capable GPU; Thread layout: 2D block and grid; Bounds checking required for irregular dimensions

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: Any CUDA-capable GPU; Thread layout: 2D block and grid
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
