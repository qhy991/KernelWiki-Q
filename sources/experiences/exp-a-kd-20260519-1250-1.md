## Challenge

Two separate GPU kernels perform softmax normalization followed by FP8 quantization: reduce_kernel_float writes 4-byte float intermediate results to a ~16 KB global memory buffer, then quantize_kernel_fp8 reads the entire buffer back from global memory, applies the FP8 scale factor, converts each value via __nv_cvt_float_to_fp8, and writes 1-byte FP8 output


## Solution

Fuse the reduce and quantize kernels into a single kernel that computes softmax normalization, applies the FP8 out_scale inline (acc *= out_scale), and writes FP8 output directly via __nv_cvt_float_to_fp8, eliminating one kernel launch overhead and one full global memory write-read round-trip for the intermediate float buffer

## Key Lessons

- Fuse FP8 quantization into the producer kernel when the output path is FP8 — eliminates one kernel launch and one global memory write-read round-trip, which dominates cost for small-to-medium output tensors (e.g., per-head attention output of 128 elements)
- A null-pointer guard on the scale pointer (fp8_out_scale_ptr != nullptr ? scale : 1.0f) makes the fused kernel a drop-in replacement that degrades gracefully when quantization is not needed
- Kernel fusion savings are proportional to intermediate buffer size relative to compute — the 4x storage expansion (float32 intermediate for FP8 output) makes fusion especially valuable for bandwidth-bound epilogue kernels
