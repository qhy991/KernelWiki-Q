## Challenge

Broadcast FMA kernel (scalar * half2 vector + half2 vector) decomposes half2 operands into individual float lanes via __low2half/__high2half + __half2float, performs fma() in float domain, then converts back to half2 via __floats2half2_rn and again to float for accumulation — incurring 8 half<->float conversion operations per iteration and processing lanes independently


## Solution

Replace the entire float-domain pipeline with native PTX f16x2 packed operations: use h0_h0 (PTX mov.b32 {a,a}) to broadcast the scalar half into both lanes of a half2, then fma.rn.f16x2 to compute scalar*a+c for both lanes in a single instruction, and add.f16x2 for accumulation — eliminating all half<->float conversion round-trips

## Key Lessons

- Use PTX fma.rn.f16x2 to compute packed half2 FMA in a single instruction instead of decomposing into float lanes — eliminates 8 half↔float conversion round-trips per iteration and exploits SIMD hardware for both lanes simultaneously
- Use h0_h0 (PTX mov.b32 {a,a}) to broadcast a scalar half into both lanes of a half2 register — cheaper than __halves2half2(x,x) and avoids any float-domain intermediate
- Operate on uint32_t bit representations of half2 values throughout compute loops to avoid implicit conversions and reduce register pressure from 4+ float registers down to 1 uint32_t per half2 value
