## Solution

Replace the single-thread serial loop with a CUB BlockScan-based parallel prefix sum launched as <<<1,64>>>, distributing the 64 element reads across 64 threads and computing the inclusive scan in O(log N) sequential steps via cooperative shared-memory reduction

## Key Lessons

- Replace <<<1,1>>> serial cumulative-sum kernels with CUB BlockScan parallel prefix sum when the element count fits in a single block — the O(log N) cooperative scan reduces serial steps from N to log N with minimal shared-memory overhead
- Use a templated BLOCK_SIZE parameter for the scan kernel to adapt the block dimension to the problem size at compile time, avoiding runtime branching and ensuring CUB's shared-memory allocation matches the actual thread count
- Carry a base_prefix_sum across chunk iterations by having the last thread broadcast its partial sum via shared memory and __syncthreads — this extends a single-block scan to input sizes larger than BLOCK_SIZE without a second kernel launch
