## Challenge

All 16 heads use the global-maximum NNZ_V=64 and NNZ_S=64 as loop bounds in the vertical/slash index merge loop, forcing sparse heads with only 4-8 actual entries to iterate through 64 slots with redundant global memory reads and conditional branch evaluations


## Solution

Override global NNZ_V/NNZ_S with per-head actual topk counts read from device arrays (per_head_vertical_topkv, per_head_slash_topkv) before entering the merge loop, allowing sparse heads to exit after 4-8 iterations instead of 64, reducing wasted global memory reads and branch evaluations for 62.5% of heads

## Key Lessons

- Replace global-max loop bounds with per-head actual counts when workload is unevenly distributed across heads — the single extra device-memory read per head is negligible compared to saving 50-60 redundant iterations for sparse heads
- Per-head parameter specialization via device array lookup is a lightweight pattern that yields speedups proportional to the fraction of under-utilized heads in the batch — applicable whenever heads have variable sparsity (e.g., top-k attention, mixture-of-experts routing)
