## Solution

Replace compile-time pipe_stages=4 with a runtime stages parameter set to 2 on memory-constrained GPUs, halving all pipeline buffer allocations (sh_a_size, sh_b_size, sh_s_size, sh_g_idx_size) while retaining one-stage-ahead prefetching for latency hiding

## Key Lessons

- Replacing compile-time pipe_stages with a runtime stages parameter allows the same kernel binary to adapt pipeline depth to the GPU's shared memory budget — 2 stages on constrained devices (64KB Turing) vs 4 on larger devices, without recompilation
- Reducing pipeline stages from 4 to 2 still provides one-stage-ahead prefetching for latency hiding while cutting all pipeline-buffer shared memory (sh_a, sh_b, sh_s, sh_g_idx) by 50%, enabling thread configs like {128,128,256} to fit within 64KB
- Adding is_a_8bit to sh_a_size sizing supports future 8-bit activation quantization by halving A-matrix buffer footprint when applicable, composable with the stages reduction
