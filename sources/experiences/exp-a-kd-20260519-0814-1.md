## Challenge

Scale output pointer is computed with a hardcoded row-major addressing formula (output_s + (block_group_id + local_group_id)), preventing the same kernel from supporting column-major tensor layouts without source-level kernel duplication


## Solution

Add a bool IS_COLUMN_MAJOR template parameter and use if constexpr to dispatch between row-major and column-major scale addressing at compile time, emitting only the matching branch with zero runtime overhead

## Key Lessons

- if constexpr with a bool template parameter enables compile-time branch elimination — the compiler emits only the matching path, producing identical SASS to a hand-written single-path kernel, avoiding the need to maintain duplicated kernel variants for different memory layouts
- Introduce a named intermediate variable (global_group_id) to replace repeated (block_group_id + local_group_id) sub-expressions — improves readability without affecting codegen since the compiler common-subexpression-optimizes them anyway
