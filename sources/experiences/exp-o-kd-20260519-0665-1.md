## Solution

Add a greedy shortcut clause (target_prob_single >= threshold_single with threshold=0.5) that unconditionally accepts draft tokens whose individual target-model probability exceeds the threshold, bypassing stochastic coin comparison entirely for high-confidence tokens

## Key Lessons

- Adding a greedy acceptance shortcut (prob >= threshold) alongside stochastic verification increases speculative decoding acceptance rate with negligible compute cost — the scalar comparison is trivially cheap compared to global-memory reads
- Using a combined acceptance criterion (stochastic OR greedy) lets high-confidence tokens bypass random rejection while preserving stochastic properties for borderline tokens, improving pipeline throughput without sacrificing correctness
- Parameterizing thresholds (threshold_single, threshold_acc) rather than hardcoding allows tuning the greedy-vs-stochastic tradeoff per deployment scenario
