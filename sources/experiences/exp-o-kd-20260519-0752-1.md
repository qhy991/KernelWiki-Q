## Solution

Add a compile-time constexpr bool kIsMLA template parameter and guard the entire V-cache load/store path with if constexpr (!kIsMLA), so that when kIsMLA=true the V-cache pointer arithmetic, load_vec, and store_vec calls are completely eliminated from the binary — halving global memory traffic from 4 to 2 transfers per work item

## Key Lessons

- Use if constexpr with a compile-time bool flag to eliminate unnecessary data paths from CUDA kernels at zero runtime cost — when kIsMLA=true, the V-cache load/store code is not instantiated at all, removing memory traffic, pointer arithmetic, and register allocation in a single construct
- In memory-bandwidth-bound kernels, eliminating half the global memory transfers (4→2 per work item) yields a near-linear speedup (~2x), confirming the kernel is fully memory-bound and that the L1::no_allocate load/store hints let the hardware saturate available bandwidth
- Guard unused function parameters (v_cache_src, v_cache_dst) behind the same if constexpr when the API contract requires them but they may be unused in certain modes — prevents wasted pointer computation even though the parameters remain in the signature
