## Challenge

Top-K sampling kernel with large vocabulary (128K) and multiple batch sizes. Initial implementation failed correctness tests due to using random sampling instead of deterministic sampling.


## Solution

Simple selection sort for top-k, deterministic argmax sampling

## Key Lessons

- For sampling operations in test frameworks, check if deterministic sampling is expected
- Read test handler code to understand reference implementation behavior
- Large vocabularies (128K+) exceed shared memory limits - use global memory
- Selection sort O(n*k) is acceptable for small k (k=6) even with large n (128K)
