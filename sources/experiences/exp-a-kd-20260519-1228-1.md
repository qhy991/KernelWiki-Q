## Challenge

The original kernel uses a 2D batch-split grid (batch_size × num_splits) with 576-thread blocks where each thread serially loops over tokens in its split, causing load imbalance because short requests generate inactive splits that waste SM slots


## Solution

Replace 2D batch-split grid with a 1D flat warp-per-token model using binary search on workspace_starts to map each warp to its owning request, enabling uniform work distribution where every launched warp produces real output without inactive splits

## Key Lessons

- Replace 2D batch-split grids with 1D flat token grids when processing variable-length sequences — flat indexing eliminates load imbalance from uneven split sizes and avoids wasting SM slots on short requests
- Binary search on prefix-sum workspace_starts maps flat token IDs to owning requests in O(log N) with no divergence across warps — suitable when num_reqs is small (tens to hundreds) relative to total_tokens
- Warp-per-token assignment (one warp per output token) eliminates serial token loops and ensures all lanes are active, improving both occupancy and instruction throughput
