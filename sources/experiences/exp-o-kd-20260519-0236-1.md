## Challenge

Packed 1-bit array NOT operation uses per-element get/set accessors that extract each bit via shift-and-mask, apply logical NOT, then insert back with conditional read-modify-write, resulting in 128 iterations of sub-byte manipulation per array instead of operating on whole storage words


## Solution

Replace the 128-iteration per-element get/NOT/set loop with a 4-iteration word-level bitwise NOT loop that applies the ~ operator directly on each uint32_t storage word, flipping all 32 bits per word in a single instruction and eliminating shift-mask-compare-conditional-set overhead

## Key Lessons

- Operate on packed storage words directly with bitwise operators (~ | & ^) instead of extracting and inserting individual sub-byte elements — eliminates shift-mask-conditional overhead and reduces loop iterations by the packing factor (32x for 1-bit elements)
- When data is stored in packed word-aligned containers, prefer whole-word operations over per-element accessors because bitwise NOT on a uint32_t is a single ALU instruction that correctly flips all packed bits simultaneously
