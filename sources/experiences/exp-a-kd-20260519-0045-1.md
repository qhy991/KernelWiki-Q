## Solution

Replace all six integer division/modulo operations with CUTLASS-style FastDivmod using precomputed multiply-shift (__umulhi + shift), which replaces each slow integer division with a single high-multiply and shift that executes in one pipeline cycle

## Key Lessons

- Replace integer division/modulo with precomputed multiply-shift (FastDivmod pattern: __umulhi(dividend, multiplier) >> shift) when divisors are loop-invariant — integer division on GPU takes ~20 cycles vs ~4 for multiply-shift, and the precomputed constants are computed once on the host
- Precompute FastDivmod structs on the host and pass them as a Params value-type struct to the kernel — this avoids per-thread divisor recomputation and keeps the hot loop body to pure multiply-shift arithmetic
- Multi-dimensional index decomposition (e.g., flat → NDHWWC) benefits disproportionately from FastDivmod because each dimension introduces a division/modulo pair, so N dimensions replace 2N slow divisions with 2N fast multiply-shifts
