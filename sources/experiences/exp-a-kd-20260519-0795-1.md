## Solution

Reorganize reduction using int4 vector loads (4× int32 per access) with warp-level work distribution where each warp processes a contiguous range of 128 experts per pass, aiming to improve memory throughput via wider 128-bit load transactions instead of scalar 32-bit loads

## Key Lessons

- Vectorized int4 loads for reading source data do not reduce atomicAdd contention at the destination — when the bottleneck is atomic operation throughput on global memory, wider reads add indexing overhead without addressing the true bottleneck
- The original `if (count > 0)` guard skips unnecessary atomicAdd calls for zero-valued elements, reducing atomic contention; removing it to issue unconditional atomicAdds on every loaded element can increase contention and regress performance
- For small workloads where data fits in L2 cache (256 experts × 32 blocks ≈ 32KB), scalar access patterns already achieve good cache behavior; complex warp-based indexing and wider loads add instruction overhead without measurable bandwidth gains
