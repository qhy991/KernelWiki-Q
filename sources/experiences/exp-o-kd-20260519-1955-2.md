## Solution

Upcast each bf16 value to float32 via __bfloat162float in an unrolled loop, accumulate using native fp32 addition with full 23-bit mantissa, then downcast only the final result back to bf16 via __float2bfloat16 — preserving numerical accuracy through the entire reduction chain

## Key Lessons

- Use upcast-reduce-downcast pattern (__bfloat162float → fp32 accumulate → __float2bfloat16) for bf16 reductions to avoid compounding mantissa rounding errors — applies whenever reducing more than 2-3 bf16 values where precision matters
- Wrap upcast/downcast in __forceinline__ unrolled helpers operating on packed arrays to amortize function call overhead and enable the compiler to fuse the conversion pipeline with arithmetic
