block_fp8_e2e_4096

## Solution

The optimization's actual production benefit is eliminating the host-side .T.contiguous() memory allocation and data copy during model loading, not measurable kernel-level speedup during inference.

## Key Lessons

- Because the scale tensor is tiny relative to the weight matrix, the transpose overhead (a few microseconds of kernel launch and 4KB copy) is negligible compared to the ~0.12ms block-wise scaling traversal of 16M weight elements.
