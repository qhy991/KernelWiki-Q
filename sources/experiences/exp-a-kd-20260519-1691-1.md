## Solution

Pack the float value and int32_t index into a single uint64_t word using cub::Traits::TwiddleIn for sign-correct comparison, then use a single cg::reduce warp primitive per iteration — eliminating the butterfly reduction, ballot, lane-scan, and index broadcast steps, reducing per-iteration shuffles from O(5×WARP_SIZE) to O(5×log2(WARP_SIZE))

## Key Lessons

- Pack value and index into a single integer word (value bits << bitwidth | inverted_index) so that a single integer greater-than reduction simultaneously finds the max value and its associated index — eliminates separate value reduction, ballot-based lane identification, and index broadcast
- Use cub::Traits<T>::TwiddleIn/TwiddleOut to handle signed floating-point comparison via unsigned integer arithmetic — sign bit is inverted so that standard unsigned greater-than produces correct floating-point max semantics
- Cooperative groups cg::reduce on a tiled_partition<kWARP_SIZE> replaces manual butterfly __shfl_xor_sync loops with a single library call that the compiler can optimize as a whole, reducing shuffle instruction count from O(WARP_SIZE) to O(log2(WARP_SIZE)) per reduction
