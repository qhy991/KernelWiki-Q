## Solution

Vectorize memory access using half2 reinterpret_cast to load and store two half values (4 bytes) per thread in a single memory transaction, halving load/store instruction count and improving memory coalescing efficiency for the bandwidth-bound FP16 kernel

## Key Lessons

- Use half2 reinterpret_cast to vectorize FP16 memory access (4 bytes per load/store) instead of scalar half loads — halves memory instruction count and improves coalescing when the working set exceeds L2 cache and the kernel is bandwidth-bound
- Each thread processes 2 elements with half2 (idx * 2 stride) while reducing thread count proportionally (blockDim from 256 to 128) to keep total work constant — applies to any element-wise FP16 kernel where memory throughput is the bottleneck
- FP16 vectorization yields larger speedups than FP32 vectorization because narrower data types make instruction overhead proportionally more significant relative to useful data movement
