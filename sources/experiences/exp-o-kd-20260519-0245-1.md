## Solution

Apply scale/bias once per accumulator element in a centralized epilogue pass that writes the transformed values to shared memory (M*K1 operations instead of M*N*K1), then let GEMM1 read A1 from shared memory with ~10x lower latency and no per-iteration scale/bias overhead

## Key Lessons

- Hoist elementwise transforms (scale/bias) out of the GEMM inner loop into a centralized epilogue pass — this reduces operations from O(M*N*K) to O(M*K) and eliminates redundant global-memory reads of the accumulator
- Write the transformed accumulator to shared memory before the next GEMM stage reads it — shared memory has ~10x lower latency than global memory and the __syncthreads() barrier cost is amortized by the reduced memory traffic and simplified inner loop
- In B2b GEMM (e.g., fused quantize-then-matmul), the EpilogueSmemAccumulator pattern decouples the scale/bias transform from GEMM1's mainloop, simplifying warp-level fragment management and removing per-warp scale/bias arrays
