## Solution

Replace custom FP8Wrapper bit-manipulation conversion with NVCC's native __nv_fp8_e4m3 type using static_cast, and template the kernel on the destination data type for generality, removing the custom FP8Wrapper struct entirely

## Key Lessons

- Custom integer bit-manipulation for float-to-fp8 conversion outperforms NVCC's internal __nv_fp8_e4m3 software emulation on sm_80 Ampere GPUs lacking native FP8 hardware — the hand-written bit-manipulation path uses fewer instructions per element and NVCC inlines it efficiently
- static_cast<__nv_fp8_e4m3> only provides a benefit on sm_89+ (Ada Lovelace) or sm_90+ (Hopper) GPUs with native FP8 cvt instructions — always verify target architecture before replacing custom conversion code with native FP8 types
