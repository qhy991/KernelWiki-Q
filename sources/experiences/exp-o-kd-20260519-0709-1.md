## Solution

Replace per-warp subset scanning with one-thread-per-expert direct mapping where each of 384 threads holds one expert value, eliminating the inner scanning loop and its shared memory read iterations; add #pragma unroll on the fixed-iteration warp shuffle reduction loops to generate straight-line SHFL instruction sequences without loop control overhead

## Key Lessons

- When thread count can match element count (384 threads for 384 experts), use direct one-thread-per-element mapping instead of per-warp subset scanning — eliminates inner iteration loops and their shared memory read/branch overhead, applicable whenever element count is a small multiple of warp count
- #pragma unroll on fixed-iteration warp shuffle reduction loops (5 iterations for offset 16→1) lets the compiler emit straight-line SHFL sequences without loop counter overhead — applies to any warp-level reduction with known iteration count
- Use per-thread direct invalidation (if tid == selected) instead of concentrating all writes in warp 0 lane 0 — distributes shared memory writes across threads and avoids serialization in the output path
