fp32_rms_norm_llama3_8b_hs4096

## Solution

256-thread kernel with rsqrtf, #pragma unroll 4, 8-warp smem reduction, __launch_bounds__(256,4)

## Key Lessons

- rsqrtf() is significantly faster than 1.0f/sqrtf() for RMS norm inverse computation
- 256 threads with min_blocks_per_sm=4 can outperform 512 threads for memory-bound kernels
- Reducing shared memory footprint (8 vs 16 floats) improves L1 cache efficiency
- #pragma unroll 4 with 4 iterations per thread is optimal for hidden_size=4096 with 256 threads
- For RMS norm at hs=4096: 256 threads, unroll-4, rsqrtf gives ~2.7x over PyTorch baseline
