## Solution

Apply N-aware dispatch with swap_ab: transpose both A and B matrices on the host, then compute C^T = B^T * A^T with swapped problem shape (N, M, K) so the now-larger M dimension drives the grid configuration, intending better tile utilization for tall-skinny problems

## Key Lessons

- swap_ab (transposing problem dimensions to swap M and N) only benefits TMA-based kernels on SM90 where the TMA descriptor reconfigures tile fetching along the now-larger M dimension — applying it to naive one-element-per-thread kernels degrades coalescing by introducing strided column-major access to the transposed B matrix
- N-aware dispatch is a valid strategy but must be paired with hardware-appropriate kernels: the transpose-and-swap pattern assumes the underlying kernel can exploit the new memory layout, otherwise the host-side transpose cost and worse access pattern cause net regression
- Memory coalescing dominates naive GEMM performance: the original layout (row-major A, row-major B with column-indexed access) coalesces well for contiguous threads along N, while the transposed layout forces strided reads from B^T that halve effective memory bandwidth
