## Challenge

Implement a complete CUTLASS SIMT warp-level GEMM kernel (D = alpha*A*B + beta*C) from a skeleton, requiring correct CUTLASS type configuration with MmaSimtPolicy, shared memory tiling for A/B/C matrices, and proper warp-level GEMM invocation with alpha/beta scaling


## Solution

Define shared memory tiles for A (row-major), B (column-major), C matrices with single-thread (threadIdx.x==0) copy loops annotated with CUTLASS_PRAGMA_NO_UNROLL; configure GemmSimt with MmaSimtPolicy<MatrixShape<4,8>, RowMajorInterleaved<2>, GemmShape<4,4,1>> and warp shape GemmShape<16,32,8>; invoke gemm(alpha, A_ref, B_ref, beta, C_ref, C_ref, threadIdx.x) for warp-level D = alpha*A*B + beta*C

## Key Lessons

- CUTLASS GemmSimt warp-level GEMM requires MmaSimtPolicy<MatrixShape<4,8>, RowMajorInterleaved<2>, GemmShape<4,4,1>> for the fundamental 8x8x4 SIMT instruction shape — this policy maps warp threads onto a 4×8 grid with 2-element row interleaving for bank-conflict-free shared memory access
- The GemmSimt operator takes (alpha, A_tensor_ref, B_tensor_ref, beta, C_src_ref, C_dst_ref, thread_id) where C serves as both source accumulator and destination — pass the same shared C tile reference for both parameters when computing D = alpha*A*B + beta*C in-place
- Annotate shared-memory copy loops with CUTLASS_PRAGMA_NO_UNROLL to prevent the compiler from unrolling large tile copies, which avoids register pressure spikes and instruction cache thrashing for tiles sized kM×kK or larger
