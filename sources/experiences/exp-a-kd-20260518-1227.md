scaled_fp8_to_float

## Solution

The optimization is relevant on AMD/ROCm where the old hip_fp8 class used a multi-instruction software emulation path for the from_bits() constructor and float cast operator, whereas the new fp8_type.__x assignment allows the compiler to use native FP8 conversion instructions on MI300+ hardware.

## Key Lessons

- The optimization is relevant on AMD/ROCm where the old hip_fp8 class used a multi-instruction software emulation path for the from_bits() constructor and float cast operator, whereas the new fp8_type.__x assignment allows the compiler to use native FP8 conversion instructions on MI300+ hardware.
- On NVIDIA hardware, both construction methods (memcpy vs __x assignment) compile to identical PTX - the compiler sees through both patterns and generates the same load+convert instruction sequence.
