## Solution

Pack multiple rows per warp via sub-warp tiling: each warp processes ROW_PER_WARP=2 rows, with K_WARP_SIZE=16 lanes per row, fully utilizing all 32 lanes. The lane index maps to a (row, k) pair via lane/K_WARP_SIZE and lane%K_WARP_SIZE, and reduction uses the smaller K_WARP_SIZE instead of full WARP_SIZE

## Key Lessons

- When K < WARP_SIZE, pack multiple rows into a single warp via sub-warp tiling (ROW_PER_WARP = WARP_SIZE / K) to fully utilize all lanes — this reduces block count by ROW_PER_WARP and eliminates wasted lanes, compounding benefits as M grows
- Map lane index to a (row, k) pair using integer division/modulo on the lane ID (lane / K_WARP_SIZE gives the sub-row, lane % K_WARP_SIZE gives the k index), then reduce over K_WARP_SIZE lanes instead of the full WARP_SIZE
