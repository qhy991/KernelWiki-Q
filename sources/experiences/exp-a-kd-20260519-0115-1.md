## Solution

Pre-scale input values by (scaling * kLog2e) before computing the row-wise max so atomicMax stores the already-scaled max directly, eliminating the multiply from the atomicCAS critical section and reducing instruction count inside the contended path; the pre-scaled value is then reused in the exp2f summation phase

## Key Lessons

- Pre-scale values before entering atomicCAS critical sections — removing even a single FMUL from the atomic loop reduces the contention window and potential CAS retries, which is valuable when many warps contend for the same shared-memory addresses
- Factor out repeated arithmetic (scaling * kLog2e) into a single pre-computation per element rather than computing it separately in the max-finding and exp-summation phases — reduces total FP instruction count and enables register reuse of the pre-scaled value
