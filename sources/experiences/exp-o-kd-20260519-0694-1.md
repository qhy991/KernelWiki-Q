## Solution

Replace the multi-warp CTA block with a single-warp (32-thread) block: the sum-of-squares reduction completes entirely via __shfl_down_sync within the warp, and inv_rms is broadcast from lane 0 using __shfl_sync — eliminating all shared memory and both __syncthreads() barriers, while the smaller 32-thread block enables higher SM occupancy (up to 16 blocks/SM vs 6)

## Key Lessons

- For small reductions (≤32 elements), a single-warp block can complete the entire reduction via __shfl_down_sync and broadcast the result with __shfl_sync — eliminating shared memory, __syncthreads() barriers, and cross-warp coordination overhead entirely
- When per-thread compute is lightweight (e.g., 1–2 elements per thread), synchronization overhead from __syncthreads() dominates kernel time — switching to warp-only execution removes this bottleneck and enables higher SM occupancy via smaller block sizes
- __shfl_sync(mask, value, 0) broadcasts lane 0's value to all lanes in the warp, replacing shared-memory write + __syncthreads() + shared-memory read patterns for single-value distribution
