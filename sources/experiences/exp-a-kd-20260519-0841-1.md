## Solution

Replace linear interleaved layout with DSv4 page-padded layout: group 64 items per page with contiguous value region at 576-byte stride (72×8, aligned to cache-line boundaries) followed by a separate contiguous scale region, and use bitwise page decomposition (index >> 6 and index & 63) to compute per-field pointers instead of linear multiply addressing

## Key Lessons

- Page-padded layout with stride aligned to cache-line multiples (576 = 72×8) improves memory coalescing over interleaved stride (584 = 73×8) — applies when transferring structured multi-field records on GPU
- Bitwise page decomposition (>> 6, & 63) replaces multiply-based linear addressing with cheaper shift/and operations — applies when page size is a power of 2, common in paged KV cache designs
