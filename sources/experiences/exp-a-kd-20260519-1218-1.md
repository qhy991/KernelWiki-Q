## Challenge

Scalar variance kernel reads each __half element individually via stride-based indexing (one 2-byte global load per iteration), issuing 4096 separate 2-byte transactions to compute x*x for RMS norm variance


## Solution

Replace scalar stride loop with vectorize_read_with_alignment_impl<__half, 8> which reinterpret_casts the input pointer to HalfVec8 (8 __half values, 16 bytes) and loads 8 elements per vector transaction, reducing total load transactions from 4096 to 512 for the aligned case

## Key Lessons

- Vectorizing global memory reads (8-wide HalfVec8) reduces load transaction count by 8x but yields negligible speedup when per-thread work is tiny (4 elements each) and kernel launch overhead plus block reduction dominate total execution time — vectorization benefits are most pronounced when memory latency is the actual bottleneck
- vectorize_read_with_alignment_impl provides a safe generic pattern for vectorized loads with alignment fallback (prefix scalar → vector body → tail scalar), but the alignment check and branching overhead further reduce benefit for very short loops where the scalar path would execute in ~4 iterations anyway
