## Challenge

The RMS norm variance kernel uses a scalar strided loop where each of 1024 threads performs individual 2-byte global loads for __half elements (4096 scalar loads total), incurring high per-element latency overhead and underutilizing available memory bandwidth


## Solution

Replace scalar strided loads with vectorize_read_with_alignment<8> which casts the input pointer to vec_n_t<__half, 8> (128-bit chunks), reducing 4096 scalar 2-byte loads to 512 vector 16-byte loads and cutting global memory transaction count by 8x via coalesced wide loads

## Key Lessons

- Use vectorized loads (vec_n_t<scalar_t, 8> for 128-bit chunks) instead of scalar element-wise reads in reduction/accumulation loops over contiguous arrays — each wide load is serviced as a single memory transaction, reducing total transaction count by the vectorization factor and lowering per-element latency overhead
- vectorize_read_with_alignment handles misaligned prefix/tail with scalar fallback and takes a fast path when cudaMalloc guarantees alignment and the element count is divisible by VEC_SIZE — safe to use without manual alignment checks
- When hidden_size is a multiple of the vector width and memory is cudaMalloc-aligned, the fast path issues only len/VEC_SIZE vector loads with no scalar overhead, maximizing memory throughput for reduction-style kernels
