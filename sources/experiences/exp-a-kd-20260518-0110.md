## Challenge

The before code uses 4 individual uint32_t pointer loads in a triple-nested loop (inst_m_idx, inner_idx, access_m_idx) to read 128 bits from shared memory per thread. The after code replaces these 4 scalar loads with a single ldmatrix.x4 instruction that loads the same 128 bits in one warp-cooperati


## Solution

p includes a __syncthreads() barrier on every iteration which dominates the kernel runtime, hiding the instruction-count reduction from ldmatrix. The __syncthreads() cost is ~100-200 cycles per call on Ampere, while the load instruction savings from ldmatrix.x4 vs 4 scalar loads is only a few cycles
