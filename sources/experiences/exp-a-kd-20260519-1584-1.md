## Solution

Fuse the half-to-bf16 conversion directly into the dequantize kernel's store point using a templated convert_from_half helper, eliminating the intermediate half buffer, the second kernel launch, and the redundant global memory round-trip

## Key Lessons

- Fuse type-conversion into the store point of a producer kernel rather than using a separate conversion kernel — eliminates one kernel launch, one intermediate buffer allocation, and a full global memory round-trip, yielding significant latency reduction when the conversion is a single hardware intrinsic (__float2bfloat16 on SM80+)
- Template the output type via a convert_from_half specialization to keep the dequantize logic type-agnostic — the same kernel body serves half, float, or bfloat16 output without code duplication
