## Solution

Replace AT_DISPATCH_FLOATING_TYPES_AND2 (4 types including double) with VLLM_DISPATCH_FLOATING_TYPES (3 types: float, half, bfloat16), removing the explicit double template instantiation to eliminate unused PTX/SASS code generation and reduce binary size by ~4KB per kernel

## Key Lessons

- Remove explicit template instantiations for unused types (e.g., double when only fp16/bf16 are used at runtime) — each instantiation forces nvcc to generate a full PTX/SASS path that inflates binary size and compile time for zero runtime benefit
- For a single kernel the savings are modest (~0.4% binary, ~2% compile time), but applying this across all dispatch sites in a production codebase yields cumulative reductions that are significant — prefer narrow dispatch macros over broad ones
