## Solution

Replace hardcoded bit-width constants with a compile-time template parameter num_bits, deriving pack_factor, tile_ints, and mask via constexpr arithmetic, and unifying the two separate extraction loops into a single loop that computes cur_int and cur_pos from pack_factor

## Key Lessons

- Templating bit-width as a compile-time int parameter generalizes quantized weight repacking kernels across bit widths (2/4/8-bit) with zero runtime cost — the CUDA compiler resolves constexpr mask and pack_factor to the same PTX shift-and-mask instructions as hardcoded constants
- Unifying two separate extraction loops (one for i<2 and one for i>=2) into a single loop with cur_int=elem/pack_factor and cur_pos=elem%pack_factor eliminates code duplication and makes the loop structure self-adapting to any bit width
- Refactoring hardcoded constants to constexpr template-derived values is safe for performance when all derived values are compile-time computable — NVCC folds them into immediate operands in PTX, producing identical SASS
