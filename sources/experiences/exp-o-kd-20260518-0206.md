vec_copy_vpt16

## Solution

The AFTER loop `row_chunk[ii] = vec_thread_read_ptr[0][ii]` for ii=0..15 accesses only the 16 valid elements. With #pragma unroll, NVCC can coalesce these 16 indexed loads into compact load sequences while still avoiding the 64-byte overshoot. The 7.2% speedup reflects the reduced memory traffic ...

## Key Lessons

- In the VPT=16 configuration, the BEFORE code assigns the full 32-element AccessType<T> (128 bytes for float) via `row_chunk_vec_ptr[0] = vec_thread_read_ptr[0]`, which reads 16 extra elements beyond the 16 needed (VPT=16 < MAX_VPT=32).
