## Solution

Replace per-element expert search with upfront round-robin thread-to-expert assignment: each thread computes its expert_idx and tid_in_expert purely from its global tid via integer division/modulo by stride (gridDim*blockDim/n_experts), with remainder handling so the first remainder experts get stride+1 threads — this eliminates all offset-array lookups and enables break-based early exit since threads iterate only within their assigned expert

## Key Lessons

- Assign threads to experts upfront via round-robin (tid / stride) instead of per-element offset-array search — eliminates global/shared memory reads and search loop overhead, and is most beneficial when expert count is large (64+ experts: 1.41x) while being neutral for few experts (6 experts: ~1.0x)
- Round-robin thread assignment enables break-based early exit within a single expert's iteration range, versus continue which skips the current element but keeps the thread iterating across all experts — break immediately stops the loop when the mask threshold is reached
- Thread-to-expert mapping via pure arithmetic (division/modulo by stride) replaces data-dependent search, making the assignment O(1) per thread regardless of expert count and removing the need for offset arrays as kernel parameters
