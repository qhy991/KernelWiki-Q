## Challenge

Scalar FP16 sigmoid kernel processes one element per thread with separate 2-byte loads/stores, incurring double the memory instruction count and underutilizing available HBM bandwidth for a 32 MB working set that exceeds L2 cache


## Solution

Vectorize memory access by processing two half values per thread via half2 reinterpret_cast loads/stores (HALF2 macro), halving the number of load/store instructions and improving memory coalescing efficiency for the bandwidth-bound kernel

## Key Lessons

- Use half2 vectorized loads/stores via reinterpret_cast for elementwise FP16 kernels to halve memory instruction count — applies whenever the kernel is memory-bandwidth-bound and data is naturally aligned to 4-byte boundaries
- Scale thread index by the vectorization factor (idx * 2 for half2) and halve the block size to maintain the same total thread-to-element coverage, preserving memory coalescing across the grid
- FP16 kernels benefit disproportionately from vectorization compared to FP32 because the narrower data type makes per-element instruction overhead proportionally larger relative to actual data movement
