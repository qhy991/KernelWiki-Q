## Solution

Fuse the two-kernel write-then-reduce sequence into a single kernel that uses atomicAdd to accumulate each CTA's partial output directly into the final output tensor, eliminating the intermediate workspace and separate kernel launch — designed for SM100 Blackwell's hardware-accelerated TMA_REDUCE_ADD instruction, with atomicAdd as a software fallback on older architectures

## Key Lessons

- Fusing a two-kernel write-then-reduce pattern into a single atomicAdd-based kernel eliminates workspace memory and launch overhead, but software atomicAdd on global memory causes severe contention when multiple CTAs target the same output locations — hardware TMA_REDUCE_ADD on SM100 Blackwell avoids this entirely by performing the reduction in the TMA unit
- A software atomicAdd fallback for a hardware-specific reduction instruction (TMA_REDUCE_ADD) can be orders of magnitude slower than the original two-kernel approach on legacy hardware, so always validate the fallback path independently and guard architecture-specific optimizations with runtime checks
