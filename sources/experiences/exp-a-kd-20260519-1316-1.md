## Challenge

Split-KV attention partitions the KV sequence across 4 chunks (SPLIT_FACTOR=4), requiring partial result buffers (partial_max, partial_sum, partial_out) and a separate reduction kernel to merge per-split online softmax states, adding inter-kernel synchronization and extra memory traffic


## Solution

Replace split-KV with a single-pass kernel that processes the entire KV sequence in one thread block using online softmax, normalizing the output directly (output[tid] / running_l) and eliminating the reduction kernel, partial result buffers, and inter-kernel synchronization

## Key Lessons

- Removing split-KV in favor of single-pass attention simplifies the code (one kernel instead of two, no partial buffers) but does not guarantee speedup — at batch=32 and seq_len=4096, the split version's 2048 thread blocks already saturate the A800 GPU, making the reduction overhead negligible relative to per-split compute
- Split-KV parallelism benefits depend on workload balance: when per-split compute is heavy enough (long sequences, high dimension), the reduction cost is small and the extra thread blocks provide no additional occupancy advantage — single-pass is competitive in this regime due to simpler memory footprint and no inter-kernel sync
