## Solution

Fuse bias addition into the GEMM epilogue by computing the scaled dot-product result in registers and adding bias before a single __float2half store to global memory, eliminating the second kernel launch and the output-tensor reload

## Key Lessons

- Fuse elementwise post-ops (bias add, activation, residual) into the GEMM epilogue write to eliminate a separate kernel launch and the associated output-tensor global memory round-trip — applies whenever the epilogue operation is per-element and independent across output locations
- Keeping the GEMM accumulator in registers and applying post-ops before the single global store reduces output memory traffic from write+read+write (3 transactions) to write-only (1 transaction), which is especially beneficial for bandwidth-bound epilogues
