## Solution

Split the single-step __hfma2 into a two-step pattern: skip_flop=true stores raw lop3 results via reinterpret_cast (no FP operation), then a separate __hsub2 applies bias subtraction — enabling downstream callers to intercept raw values and fuse custom scale+bias before correction

## Key Lessons

- __hfma2(x, 1.0, bias) and __hsub2(x, bias) lower to identical HFMA2 instructions with negated operand at the NVPTX level — the identity multiply is free via hardware fusion, so replacing one with the other has zero performance impact in isolation
- Separating raw lop3 result storage from bias subtraction into a two-step (skip_flop) pattern creates a composable API where downstream kernels can intercept raw dequantized values and fuse custom scale+bias operations before bias correction — applies when dequant output feeds into further arithmetic that could absorb the bias term
