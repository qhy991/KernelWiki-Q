## Solution

Fuse the FP8 output scale into the reduce kernel by computing out_scale = 1.0f / *fp8_out_scale_ptr when the pointer is non-null and multiplying the accumulated softmax result before writing to output, eliminating the separate quantization kernel and its associated memory round-trip

## Key Lessons

- Fusing quantization scaling into the compute kernel (applying out_scale before the existing FP8 conversion intrinsic) eliminates a separate quantization kernel launch and one intermediate global memory write-read round-trip — applies whenever a post-processing scale can be folded into an existing arithmetic chain before the type conversion
- Guard optional fusion with a runtime null-pointer check (fp8_out_scale_ptr != nullptr) and a fallback scale of 1.0f to keep the kernel functional for non-FP8 output paths without requiring separate kernel variants
