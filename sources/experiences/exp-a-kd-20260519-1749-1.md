## Challenge

AWQ INT4 dequantize kernel applies zero-point subtraction and scale multiplication via a scalar loop of 8 separate __hsub/__hmul half-precision intrinsics, potentially underutilizing the GPU's native f16x2 dual-half arithmetic units


## Solution

Replace the scalar __hsub/__hmul loop with explicit inline PTX sub.f16x2 and fma.rn.f16x2 instructions operating on the uint4 .x/.y/.z/.w fields, performing two half operations per instruction to directly express vectorized FP16 arithmetic

## Key Lessons

- NVCC -O3 auto-vectorizes scalar __hsub/__hmul intrinsics into f16x2 PTX operations, producing equivalent SASS for both scalar-source and explicit-PTX-source paths — hand-written PTX f16x2 only helps when the compiler fails to auto-vectorize
- AWQ INT4 dequantize at 11008x768 scale is memory-bandwidth bound (loading weight and scaling factor data dominates runtime), so reducing FP16 arithmetic instruction count alone does not translate to wall-clock improvement — optimize memory access patterns instead
