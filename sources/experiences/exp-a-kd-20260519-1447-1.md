## Solution

Replace hardcoded loads and split loops with a templated approach using constexpr pack_factor and tile_ints, computing element position via division/modulo (cur_int/cur_pos) to unify extraction into a single loop — but the compiler resolves these to identical PTX shift-and-mask instructions, producing no measurable performance change

## Key Lessons

- Templated constexpr generalization (pack_factor, tile_ints, computed cur_int/cur_pos) improves code maintainability for multi-bitwidth support but does not change runtime performance — the NVCC compiler fully resolves division/modulo by constexpr powers of two into the same shift-and-mask PTX as handwritten code
- Splitting a single unified extraction loop into two hardcoded sub-loops vs. one computed loop is a stylistic refactoring with zero performance impact when all loop bounds and divisors are compile-time constants
