## Challenge

In the before source (fp16_128_4tok_before.cu), the pipeline requires two separate __global__ kernel launches: silu_and_mul_kernel writes SiLU(x)*y as __half to a 1KB intermediate global-memory buffer, then scaled_quant_kernel reads that same buffer back and quantizes each element to int8


## Solution

The after source (fp16_128_4tok_after.cu) uses a single fused_silu_mul_quant_kernel where the float-valued silu_result = silu_fn(x)*y is computed in registers and immediately quantified via scaled multiplication and clamping without ever reaching global memory.
