## Solution

Fuse the two kernels into a single fused_rope_kv_kernel that keeps k_out in registers after RoPE computation and writes it directly to both k_rope and k_buffer, eliminating the redundant global memory round-trip and halving kernel launch overhead

## Key Lessons

- Fuse sequential kernels that share producer-consumer data through global memory into a single kernel so intermediate results stay in registers — eliminates redundant global memory traffic and halves launch overhead, especially impactful for small batches where launch overhead dominates
- When nnz is small (≤4 tokens), kernel launch overhead (~5-10us per launch) can account for a large fraction of total execution time, making kernel fusion disproportionately beneficial compared to the memory savings alone
- In a fused kernel, load all needed inputs (V values, KV cache offsets) before the compute-heavy section (RoPE) so that register-held results can be written to multiple destinations without re-reading from global memory
