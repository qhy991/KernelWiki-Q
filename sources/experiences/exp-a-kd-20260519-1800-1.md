## Solution

Replace the hardcoded += with a function pointer dispatch pattern: val = fn(val, VLLM_SHFL_XOR_SYNC(val, mask)), where fn is a __device__ function like _sum or _max. NVCC resolves the function pointer to a compile-time constant, producing identical SASS to the original hardcoded version while enabling code reuse across reduction operations

## Key Lessons

- Replacing a hardcoded operator with a __device__ function pointer dispatch adds zero runtime cost when NVCC can resolve the pointer to a compile-time constant — enables code reuse across reduction types (sum, max, min) without SASS divergence
- Refactoring warp-level reduction from a single-operation template (warpReduceSum) to a generalized warpReduce(fn) is a safe generalization: the compiler inlines the function pointer and produces identical machine code, confirmed by sub-noise-level timing delta (0.19%) on a sub-3μs kernel
