## Solution

Vectorize I/O from scalar float to float2 when both c_in and c_out are even: halve channel dimensions, cast pointers to float2*, and process two adjacent channels per thread, doubling per-transaction bandwidth from 4 to 8 bytes and halving total loop iterations

## Key Lessons

- When channel dimensions are even, cast float pointers to float2 and halve channel counts to double per-thread I/O width from 4 to 8 bytes — reduces global memory transactions and loop iterations by 2x with no change to index arithmetic logic
- Vectorization speedup is bounded by non-memory overhead: when per-element integer arithmetic (mod/div for coordinate decomposition) dominates, doubling I/O width yields sub-linear gains rather than the theoretical 2x
