## Challenge

TopK histogram construction loop loads one float per iteration using scalar global memory accesses, underutilizing available memory bandwidth when processing contiguous logit arrays during the histogram binning phase


## Solution

Introduce float4 vectorized loads via PTX inline asm with cache-global hint (ld.global.cg.v4), processing 4 logits per load instruction and binning them into the histogram, reducing total load instructions by 4x in the histogram construction phase

## Key Lessons

- Vectorized float4 loads via PTX ld.global.cg reduce load instruction count by 4x for contiguous float arrays — the cache-global hint bypasses L1 for streaming access patterns where data is used once, applicable to histogram and reduction kernels
- Even a sound micro-optimization (vectorized loads) can be masked when the overall kernel design change (larger histogram, heavier scan) introduces dominating overhead — profile at the kernel level to distinguish per-technique contribution
