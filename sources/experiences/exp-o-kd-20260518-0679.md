## Key Lessons

- Since threads in the same warp read adjacent addresses within the same token row, the GPU can coalesce these 32-bit accesses into fewer 128-bit memory transactions, reducing the effective memory bandwidth cost per access.
- When consecutive threads in a warp belong to different token positions, their global memory accesses are strided across token rows (e.g., thread 0 reads x[0], thread 1 reads x[2*N], thread 2 reads x[4*N]), producing a non-coalesced pattern that wastes memory bandwidth.
