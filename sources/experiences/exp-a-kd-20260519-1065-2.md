## Challenge

Each TopK iteration uses CUB BlockReduce with shared-memory tempStorage and __syncthreads barriers to reach argmax consensus across 256 threads, incurring shared-memory staging cost and a full-block synchronization per iteration even though only 8 elements per row need comparison


## Solution

Replace CUB BlockReduce with a warp-shuffle butterfly XOR reduction (__shfl_xor_sync) scoped to THREADS_PER_ROW=2 threads per row: each thread finds a local argmax over its row_chunk[] elements, then a two-iteration XOR reduction reaches consensus in registers without shared memory or __syncthreads, and the winning expert is cleared by setting its value to -10000.f for subsequent iterations

## Key Lessons

- Replace CUB BlockReduce + __syncthreads with __shfl_xor_sync scoped to THREADS_PER_ROW when the reduction group is smaller than the warp — avoids shared-memory staging and full-block barriers, reducing per-iteration overhead from O(blockSize) sync to O(log2(threadsPerRow)) register shuffles
- Clear selected TopK elements by overwriting with a large negative sentinel (-10000.f) in the thread-local register array instead of masking via global-memory reads of prior indices — avoids the O(k_idx) global-memory comparison loop and keeps all TopK state in registers
- Scope __shfl_xor_sync with the width parameter (THREADS_PER_ROW) so multiple row-groups within the same warp perform independent reductions without interference, enabling ROWS_PER_WARP concurrent TopK operations per warp
