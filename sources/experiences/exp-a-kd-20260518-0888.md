## Challenge

For 1021 elements (not divisible by 16), the before code falls back entirely to scalar (can_vectorize=false), while the after code vectorizes 1008 elements via vec_n_t loads and processes only 13 tail elements with scalar. In principle, the after code should be faster because it vectorizes ~98.7%...


## Solution

in the after code offsets the modest vectorization benefit on the ~4 vector packs per thread. The expected improvement from partial vectorization would become visible at larger sizes with multi-block grids.
