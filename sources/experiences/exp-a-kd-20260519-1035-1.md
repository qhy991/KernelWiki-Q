## Solution

Replace the Blelloch shared-memory prefix scan with a two-level warp-exclusive scan using __shfl_up_sync: each warp computes a local scan via shuffle intrinsics (no shared memory), warp-level sums are scanned by warp 0, then per-warp offsets are added back — reducing __syncthreads() barriers from ~18 to ~5-6 and eliminating bank-conflict-inducing addressing

## Key Lessons

- Warp shuffle intrinsics (__shfl_up_sync) replace shared-memory Blelloch scan for intra-warp prefix sums, eliminating bank conflicts and reducing __syncthreads() barriers from O(2·log N) to O(log_warps) — applies when scan_size ≤ single-block and WARP_SIZE divides the problem into manageable sub-scans
- Two-level scan decomposition (per-warp local scan → cross-warp prefix scan → add offsets) extends warp shuffle scans to arbitrary problem sizes that exceed a single warp's 32-thread capacity
