## Challenge

Before each invocation of moe_align_block_size_kernel, a separate zero_buffer_kernel is launched to memset the 65-element cumsum_buffer to zero, adding an extra CUDA kernel launch (~5-10us overhead) plus GPU execution time that is entirely redundant


## Solution

Remove the zero_buffer_kernel entirely because moe_align_block_size_kernel already writes every cumsum entry via cumsum[0]=0 and cumsum[i]=cumsum[i-1]+padded_count for i=1..num_experts, fully initializing the buffer that count_and_sort_expert_tokens_kernel subsequently reads via atomicAdd

## Key Lessons

- When a kernel fully writes all entries of a buffer, preceding memset operations are redundant and can be eliminated — moe_align_block_size_kernel writes cumsum[0..num_experts] completely, making the prior zero_buffer_kernel unnecessary
- Eliminating even small redundant kernel launches saves ~2-3us per invocation from launch submission overhead alone — for frequently called kernels in MoE routing paths, this accumulates to measurable throughput gains
