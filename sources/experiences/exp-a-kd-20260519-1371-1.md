## Challenge

Streaming copy kernel uses __ldg loads and plain stores that allocate L2 cache lines, potentially polluting L2 cache for other concurrent workloads when data is read once and never reused


## Solution

Replace __ldg/plain stores with ld.global.cs.b32/st.global.cs.b32 PTX instructions using the .cs (cache-streaming) hint, which bypasses L2 cache line allocation for data that is read or written once and never reused

## Key Lessons

- Use ld.global.cs/st.global.cs PTX hints for streaming data that is accessed once and never reused — the .cs hint bypasses L2 cache line allocation, reducing cache pollution when L2 capacity is under contention
- Cache-streaming hints show no measurable benefit when the working set fits entirely within L2 cache (128KB vs 5MB L2 on A800) with no competing L2 pressure — apply .cs only when streaming data exceeds L2 capacity or concurrent kernels share the L2 cache
