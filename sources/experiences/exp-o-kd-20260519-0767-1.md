## Challenge

Two separate kernel launches — rescale_kernel divides output by row sum, then scale_kernel multiplies by routed_scaling_factor — each requiring its own full global-memory read-modify-write pass over 24576 float elements plus separate launch overhead (~5us per launch)


## Solution

Fuse the rescale (divide by sum) and scale (multiply by factor) operations into a single fused_rescale_scale_kernel that performs both in one per-element loop, eliminating one kernel launch and halving global-memory traffic by avoiding a second full read-modify-write pass over the output array

## Key Lessons

- Fuse sequential elementwise kernels that operate on the same data into a single kernel to eliminate redundant global-memory round-trips — applies whenever two kernels read-modify-write the same array and the operations commute or can be composed in one pass
- Kernel launch overhead (~5us per launch) is significant for small-to-medium workloads; fusing two kernels into one removes this overhead entirely and compounds with the memory-traffic savings
