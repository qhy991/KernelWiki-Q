## Solution

Replace the two-phase deferred pattern with inline conversion: load each fp8 chunk into a small local uint8_t fp8_temp[8] temporary and immediately convert to Half8 within the same loop iteration, eliminating the persistent Vlocalb8[][][] buffer entirely and reducing peak register pressure by 512 bytes

## Key Lessons

- Fuse fp8-to-half load and conversion into a single loop iteration using a local temporary instead of a persistent buffer when the intermediate format (raw fp8) is consumed only once — eliminates the register pressure from holding both raw and converted buffers simultaneously
- Deferred (load-all-then-convert-all) patterns are appropriate when the intermediate buffer is small and reuse is needed (e.g., K-cache 1D buffer), but for V-cache's larger 2D buffer shape the register cost of persisting the raw buffer outweighs any scheduling benefit
- Peak register pressure from two coexisting buffers (raw + converted) reduces warp occupancy in real multi-warp kernels even when single-thread microbenchmarks show neutral timing — the benefit manifests at the SM level, not the instruction level
