## Challenge

SiLU-and-multiply kernel loads two half-precision input vectors per iteration using plain pointer dereference (input[...]), which does not explicitly route reads through the read-only/texture data cache path


## Solution

Wrap both read-only input loads with __ldg() to explicitly route them through the read-only data cache (texture path), potentially freeing L1/L2 cache for other data and reducing load-use latency

## Key Lessons

- Adding __ldg() to const __restrict__ pointer dereferences produces no measurable improvement on Ampere-class GPUs — the compiler already auto-routes such loads through the read-only cache, making explicit __ldg() redundant on modern architectures
- Half-precision (2-byte) elements have lower bandwidth pressure than float (4-byte), so cache-path routing optimizations have even less impact since the memory subsystem is less saturated
- Short kernels (~6.5us) exhibit high measurement noise; a <1% delta is within noise margin and should not be interpreted as a real performance change
