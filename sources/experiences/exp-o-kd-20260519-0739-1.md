## Challenge

Scalar peer-reduction kernel loads one half element at a time via __half2float, issuing narrow 32-bit load instructions per peer per element — at 2MB (1M half elements) this produces ~4M separate global memory transactions that severely underutilize available memory bandwidth


## Solution

Replace scalar per-element loads with __align__(16) array_t<half,8> packed structs to issue single 128-bit ld.128 transactions covering 8 half elements per load, accumulate in vectorized array_t<float,8> with fully unrolled packed_assign_add, and write back in packed form — reducing total memory transactions by ~8x

## Key Lessons

- Use __align__(16) struct wrappers (array_t<T, 16/sizeof(T)>) to generate 128-bit ld.128/st.128 transactions — the compiler emits a single wide memory instruction instead of multiple narrow scalar loads, yielding ~8x transaction reduction for half-precision data
- Batch type conversion and accumulation through packed array operations with #pragma unroll — unrolled vector upcast/downcast + packed_assign_add maximizes instruction-level parallelism and eliminates per-element function call overhead
- Loop iteration count scales inversely with pack width (total_elements → packed_size = total_elements / (16/sizeof(T))) — wider packs reduce both memory traffic and loop overhead proportionally
