## Challenge

Optimize top-k sampling kernel (k=8) for LLaMA3-8B vocab size. Baseline was very slow. Need to maximize throughput for batch_512 workload.


## Solution

Block-per-batch with 256 threads, 2xfloat4 per loop iteration (8 elements/step), __ldg() read-only cache, warp shuffle reduction + shared memory inter-warp reduction

## Key Lessons

- 256 threads outperforms 512 threads for block-per-batch top-k on A800 (less register pressure, better occupancy)
- Processing 2 float4 vectors per thread per iteration (8 elements/step) improves ILP and memory throughput vs 1 float4/step
- __ldg() provides marginal improvement for read-only probability arrays
- Warp shuffle reduction + shared memory merge is the right pattern for block-per-batch top-k
- The 2xfloat4 approach requires careful handling of remainder elements when vocab_size is not divisible by 2*blockDim.x*4
