## Solution

Replace inline per-warp scale/bias with a centralized EpilogueSmemAccumulator pass that applies scale/bias once per accumulator element into shared memory, then GEMM1 reads from smem — eliminating redundant global memory traffic and reducing per-iteration FLOPs in the inner loop

## Key Lessons

- In back-to-back GEMM (B2b) pipelines, apply elementwise transforms (scale/bias) in a centralized epilogue pass to shared memory rather than inline in the second GEMM mainloop — this eliminates M*N redundant transform operations and moves the data path from global memory to shared memory
- Using shared memory as an inter-GEMM accumulator buffer decouples the epilogue from the mainloop: the barrier cost of __syncthreads() is amortized by reduced global memory traffic (~10x lower latency per access) and a simpler inner loop with fewer FLOPs per iteration
- When the same transformed accumulator is consumed K1 times across N output columns, pre-computing the transform into smem reduces total operations from O(M*N*K1) scale/bias applications to O(M*K1) — a factor-of-N reduction
