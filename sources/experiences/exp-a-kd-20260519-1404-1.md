## Solution

Remove reverse_map and permuted_idx side-effect writes entirely and simplify the kernel to a pure row shuffle that only reads dst2src_map and copies row data, eliminating 2 auxiliary global stores and 2 kernel parameters

## Key Lessons

- Eliminating small-volume auxiliary global stores (~64KB) from a kernel dominated by bulk data transfer (~64MB of fp16 row copy) has negligible performance impact — the memory subsystem handles scalar stores in parallel with the dominant vectorized copy loop
- Side-effect writes guarded by threadIdx.x==0 do not stall warp execution because the remaining threads proceed independently to the main vectorized copy work without waiting for thread 0's stores to complete
