## Solution

Replace __shfl_sync broadcast with canonical_warp_idx() which simply returns threadIdx.x / NumThreadsPerWarp without any warp-level synchronization, removing the unnecessary shuffle barrier while preserving the correct warp-uniform index value

## Key Lessons

- Avoid __shfl_sync for broadcasting warp-uniform values (e.g., threadIdx.x / 32) — every thread in the same warp already computes the same quotient, so the shuffle barrier is redundant and prevents independent warp forward progress
- When a value is guaranteed to be identical across all lanes in a warp by construction (e.g., integer division of threadIdx.x by the warp size), use a plain arithmetic computation or a non-syncing helper like canonical_warp_idx() instead of a warp shuffle broadcast
