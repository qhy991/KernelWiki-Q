## Challenge

cuBLAS GEMM wrapper uses a global singleton handle with per-call null-pointer checks and static alpha/beta variables, adding unnecessary host-side branching and preventing multi-handle or multi-stream usage patterns


## Solution

Refactor to pass the cuBLAS handle as an explicit parameter, removing the global singleton, per-call null checks, and static alpha/beta variables so the caller owns handle lifecycle and can support multi-stream concurrency

## Key Lessons

- Host-side branching (null checks on a global handle) is negligible compared to GPU kernel execution time for compute-bound operations like GEMM — such refactoring improves code structure and multi-stream safety rather than measurable performance
- Pass library handles (cuBLAS, cuDNN) as explicit parameters rather than using global singletons — this enables multi-stream concurrency, avoids hidden state, and makes handle lifecycle ownership clear without changing GPU-side behavior
- static local variables for scalar arguments (alpha, beta) provide no performance benefit over regular locals in cuBLAS calls and introduce subtle shared-state risks in multi-threaded scenarios
