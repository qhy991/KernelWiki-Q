## Challenge

With BLOCK_SIZE=16, THREAD_GROUP_SIZE=2 threads cooperate on each QK dot product via warp-shuffle reduction, adding latency per KV block; additionally, 128 KV blocks are iterated for seq_len=2048, increasing block-table lookup overhead


## Solution

Double BLOCK_SIZE from 16 to 32 so THREAD_GROUP_SIZE becomes 1, eliminating intra-group QK shuffle reduction entirely and halving the number of KV block iterations; each thread computes its QK dot product independently with wider vector loads (uint4 vs uint2), reducing shuffle instructions and block-table overhead

## Key Lessons

- Increasing BLOCK_SIZE to make THREAD_GROUP_SIZE=1 eliminates intra-group QK shuffle reduction and its associated mask-guarded logits writes, reducing instruction count per KV block — effective when the wider vector load (uint4 vs uint2) is still efficiently coalesced
- Doubling BLOCK_SIZE halves the number of KV blocks iterated (128→64 for seq_len=2048), reducing block-table lookup overhead and loop iteration cost; the net reduction in shuffle instructions and block iterations outweighs the increased value reduction stages (NUM_V_VECS_PER_ROW 2→4)
