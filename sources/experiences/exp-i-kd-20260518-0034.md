## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: SM 70/80/90; Count should be a power of 2 for vectorization (8, 16, 32); Use aligned memory accesses

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: SM 70/80/90; Count should be a power of 2 for vectorization (8, 16, 32)
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
