## Solution

Decompose work so each warp owns one head (warp_id selects head, lane indexes within head_size), then use vectorized loads/stores with VEC_SIZE=8 (128-bit transactions) via a generic vectorize_with_alignment template to copy contiguous head segments in bulk, eliminating per-element division/modulo and consolidating scalar 2-byte transactions into 16-byte vector transactions

## Key Lessons

- Warp-per-head decomposition assigns one head to each warp so threads access contiguous head_size segments — enables vectorized loads/stores and eliminates per-element division/modulo for head index computation, applying whenever num_heads is a multiple of warps_per_block and data is laid out with contiguous heads
- Use vectorize_with_alignment<VEC_SIZE> with VEC_SIZE=8 for fp16 to issue 128-bit (16-byte) memory transactions instead of scalar 2-byte stores — applies whenever source and destination addresses are aligned and length is a multiple of VEC_SIZE, yielding up to 8x fewer memory transactions
- Precompute source and destination base pointers outside the head loop (key_src, key_dst) so the inner loop only adds per-head offsets — reduces redundant address arithmetic when the same base repeats across iterations
