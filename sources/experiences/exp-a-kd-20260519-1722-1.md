## Solution

Replaced per-element division with precomputed reciprocal multiplication (inv_s = 1.0f/scale) and wrapped the quantization loop in vectorize_with_alignment<16> for 64-byte vectorized memory transactions, but simultaneously reduced launch block size from 1024 to 256 threads, dropping warps per block from 32 to 8 and starving the SM scheduler

## Key Lessons

- Precomputing a reciprocal (1.0f/scale) and multiplying instead of dividing saves ~4 cycles per element — valid and universally applicable when the divisor is loop-invariant
- Reducing block size from 1024 to 256 threads (32 warps to 8 warps per block) starves the SM warp scheduler of independent warps needed to hide memory and arithmetic latency — occupancy loss can dominate arithmetic savings even when individual operations are faster
- Vectorized memory access via vectorize_with_alignment<16> improves per-thread bandwidth utilization, but the benefit is negated when the total active thread count drops proportionally, reducing overall memory-level parallelism across the SM
