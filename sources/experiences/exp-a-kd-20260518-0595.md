## Challenge

The before code uses fmax/fmin which are C standard library double-precision functions, requiring implicit float→double promotion of each operand (the scaled value, FP8_E4M3_MAX, and -FP8_E4M3_MAX) and a double→float conversion of the result


## Solution

The after code uses fmaxf/fminf which operate directly on float operands. However, the measured speedup is only 0.34%, which is within measurement noise. ## Results

- **speedup**: 1.003x (neutral)


## Key Lessons

- On sm_80 GPUs, nvcc with -O3 likely optimizes the fmax/fmin calls to use single-precision hardware instructions when all operands are float, since the double promotion is unnecessary and the compiler can see through it.
