## Challenge

In the v2 kernel, blockIdx.x maps directly to token_pos and threadIdx.x to the dimension pair index tid within that token. This means all threads in a warp read from the same c


## Solution

Replace scalar loads with float4 vectorized loads to coalesce 4 adjacent floats into one 128-bit transaction, reduce thread count from 256 to 64, and replace the shared-memory tree reduction with __shfl_xor_sync warp shuffle, cutting shared memory from 256 floats to 2 floats and eliminating all but one __syncthreads() barrier

## Results

- **speedup**: 1.213x (improved)


## Key Lessons

- Use float4 vectorized loads to coalesce 4 contiguous float accesses into one 128-bit memory transaction, reducing total memory transactions by 4x for stride-1 access patterns — applies to any reduction or elementwise kernel with aligned contiguous data
- Prefer __shfl_xor_sync warp shuffle for intra-warp reductions over shared-memory tree reductions — eliminates __syncthreads() barriers within warps, reduces shared-memory footprint, and leverages dedicated hardware shuffle paths
- Reducing block thread count when using vectorized loads maintains effective memory throughput while lowering register and shared-memory pressure per block, often improving overall occupancy
