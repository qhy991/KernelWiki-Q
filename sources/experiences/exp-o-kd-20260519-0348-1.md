## Challenge

Row-wise dot-product kernel loads out and grad arrays one float at a time, issuing 256 separate load instructions per row and underutilizing available memory bandwidth


## Solution

Cast the row-aligned float pointers to float4 and iterate over kCols/4 elements, loading four values per instruction and fusing the four products into a single accumulator — cutting load count by 4x and improving memory throughput

## Key Lessons

- Use float4 (or other vectorized) loads instead of scalar loads when input arrays are 16-byte aligned and accessed sequentially — reduces load instruction count by 4x and saturates memory bandwidth
- reinterpret_cast<const float4*>(ptr + row*stride) is the idiomatic way to express vectorized access over a row-major layout without changing the outer allocation
