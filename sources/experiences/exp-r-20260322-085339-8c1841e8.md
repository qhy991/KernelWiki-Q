KernelEvalPlus AutoResearch Round 3: RN_01 rms_norm optimization (fp32, hidden_size=7168)

## Key Lessons

- For rms_norm with large hidden_size (7168), shared memory input caching HURTS - the 28KB smem reduces occupancy more than it helps
- __ldcs (streaming cache hint) is better than __ldg for weight vectors in rms_norm - weight is read once per row, streaming avoids L1 pollution
- The combination of __ldg for input + __ldcs for weight + no smem cache is near-optimal for this kernel
- Adaptive block size dispatch (256/512/1024) based on batch_size is effective
- Performance ratio of 168.5x over baseline is very high - further gains require fundamentally different approach (e.g., persistent kernel, multi-row-per-block)
