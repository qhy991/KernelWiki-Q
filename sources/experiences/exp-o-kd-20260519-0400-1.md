## Solution

Consolidate four separate half2 loads/stores into single 128-bit LDST128BITS (float4) transactions, replace scalar __hadd on .x/.y fields with SIMD __hadd2 intrinsic in a pragma-unroll loop, and merge four boundary checks into one (idx+7 < N) to eliminate branch divergence

## Key Lessons

- Use 128-bit (float4) load/store transactions via reinterpret_cast instead of multiple separate half2 loads when processing 8 or more consecutive fp16 elements — halves the number of memory instructions and improves coalescing
- Prefer __hadd2 SIMD intrinsic over scalar __hadd on .x/.y components — one hardware instruction computes two half additions simultaneously, doubling arithmetic throughput
- Consolidate multiple adjacent boundary checks (idx+k < N for k=0,2,4,6) into a single (idx+7 < N) guard around a 128-bit store — eliminates branch divergence and reduces instruction count
