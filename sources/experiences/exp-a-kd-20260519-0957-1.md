## Solution

Replace Blelloch shared-memory scan with a two-level warp shuffle scan using __shfl_up_sync for barrier-free intra-warp exclusive prefix sums, then accumulate warp-level partial sums via a second warp shuffle pass on warp0, reducing total __syncthreads from 14 to 7 and yielding ~22% latency reduction

## Key Lessons

- Use __shfl_up_sync for intra-warp prefix sum instead of shared-memory Blelloch scan to eliminate all intra-warp __syncthreads barriers — warp shuffle executes in hardware without synchronization, applicable whenever scan/reduction data fits within warp boundaries (≤32 elements per warp)
- Hierarchical two-level decomposition (intra-warp exclusive scan → inter-warp accumulation via warp0 → combine) naturally maps to GPU warp structure and reduces barrier count by ~2× for small prefix sums spanning multiple warps
- Blelloch scan's log₂(N) up-sweep + log₂(N) down-sweep iterations each requiring a full __syncthreads barrier is prohibitively expensive when N is small (e.g., 64 elements) and block size is large (1024 threads) — most threads are idle but still pay the barrier cost
