## Solution

Introduce a head-parallel kernel where each block processes exactly one head (blockIdx.y indexes heads), distributing 72 independent blocks (64 qo + 8 kv heads) across available SMs instead of serializing all heads in a single block

## Key Lessons

- Replace per-block head loops with blockIdx.y head indexing when the number of tokens is too small to saturate SMs — each head becomes an independent thread block dispatched to a different SM, converting serial loops into parallel execution
- Map Q and KV heads into a single blockIdx.y dimension (by < num_qo_heads for Q, by - num_qo_heads for KV) to launch a uniform grid covering all heads without separate kernel launches
