## Solution

Size the grid by totalWorkSize (numRows*colsPerRow) instead of just numRows, and apply a balancing loop that doubles grid count while halving block size (down to 64 threads) until the grid fills available SMs, increasing occupancy and work distribution

## Key Lessons

- Size kernel grids by total work elements (rows * cols) rather than a single dimension — when one dimension is small, the full workload is still large and needs commensurate grid sizing
- A grid/block balancing loop (double grid, halve block until grid fills SMs or block hits 64 threads) is a simple heuristic to maximize occupancy without exceeding hardware limits
