## Solution

Upgrade from kStage=1 to kStage=2 to enable a multi-stage async pipeline: issue cp.async.cg for the current V tile and next K tile at the start of each iteration while the current iteration's Q@K^T and P@V MMA proceeds on data prefetched in the previous iteration, using smem_sel/smem_sel_next double-buffered K shared memory slots to avoid read-write conflicts

## Key Lessons

- Increasing kStage from 1 to 2 in flash attention split-KV MMA kernels enables double-buffered async K/V loading that overlaps global memory latency with Tensor Core computation — effective when the KV loop has enough iterations (e.g., Tc>=8) to amortize the pipeline startup cost
- Use smem_sel/smem_sel_next alternation for double-buffered shared memory slots so that writing the next K tile into the alternate buffer does not conflict with reading the current K tile during MMA computation
- Selective CP_ASYNC_WAIT_GROUP(1) before P@V allows the next K async copy to continue in flight while only blocking for the V needed for the current P@V step — finer-grained synchronization than a full syncthreads barrier
