## Solution

Replace KernelParamsDynamic (runtime int fields) with a templated KernelParams<VPT, NUM_EXPERTS, THREADS_PER_ROW, ROWS_PER_WARP, ROWS_PER_CTA, WARPS_PER_CTA> struct where all fields are static constexpr, giving NVCC exact compile-time knowledge of every loop bound and enabling full unrolling of the 32-iteration sigmoid/bias-add/argmax loops and the 3-iteration shuffle-reduction loops into straight-line code with better instruction scheduling

## Key Lessons

- Convert kernel configuration parameters (VPT, THREADS_PER_ROW, etc.) from runtime int fields to compile-time static constexpr template parameters when the values are known at launch time — this enables NVCC to fully unroll #pragma unroll loops with known bounds, eliminating per-iteration branch/counter overhead and allowing instruction-level parallelism across loop iterations
- Compile-time specialization yields large speedups (33% here) in kernels dominated by small fixed-count loops (sigmoid, argmax, warp shuffle reduction) because the compiler can flatten them into straight-line code and interleave independent operations for latency hiding
