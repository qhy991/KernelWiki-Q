## Solution

Replace the scalar per-element loop with a compile-time AlignedArray<T, Vlen> type whose width exactly matches the per-thread data footprint, enabling a single pointer-cast vectorized load that avoids the MAX_VPT alignment mismatch and eliminates loop overhead

## Key Lessons

- Use a compile-time-sized AlignedArray whose width matches the exact per-thread data footprint rather than a fixed MAX_VPT AccessType — this eliminates alignment mismatches when the runtime element count is smaller than the maximum, enabling a single wide vector load instead of a scalar loop
- When a direct vectorized copy via reinterpret_cast causes misalignment, the root cause is often a width mismatch between the cast type and the actual data footprint — fixing the type width to match the data (not the maximum possible width) resolves the misalignment and unlocks the vectorized path
