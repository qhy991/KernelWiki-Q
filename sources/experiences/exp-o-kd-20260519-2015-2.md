## Challenge

Scalar bf16 load/store per element underutilizes memory bandwidth — each iteration issues one 16-bit STG instruction, leaving most of the available memory throughput wasted on a memory-bound concat kernel


## Solution

Use inline PTX ld.global.cs.v4.u32 / st.global.cs.v4.u32 (128-bit vector loads/stores with cache-streaming hint) so each lane copies 8 bf16 values per instruction, and the .cs hint bypasses L2 cache allocation for write-once data to reduce cache pollution

## Key Lessons

- Use inline PTX ld.global.cs.v4.u32 / st.global.cs.v4.u32 (128-bit vector transfers) instead of scalar bf16 loads/stores — each instruction moves 8 bf16 values, achieving 8x higher memory throughput per transaction on memory-bound kernels
- The .cs (cache-streaming) hint bypasses L2 cache allocation for write-once data, reducing cache pollution — apply when the destination buffer is written but never re-read by the same kernel
