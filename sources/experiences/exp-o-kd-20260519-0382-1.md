## Challenge

Scalar GELU kernel processes one half element per thread, leaving each 32-bit coalesced memory segment half-utilized since a single 16-bit store only fills half the minimum transaction width


## Solution

Convert scalar half load/store to half2 vectorized load/store via the HALF2 reinterpret_cast macro, processing 2 adjacent half values per thread in a single 32-bit coalesced transaction, and halve the block size to keep total thread count equivalent

## Key Lessons

- Use half2 vectorized load/store (via reinterpret_cast<half2*> or HALF2 macro) in memory-bound elementwise kernels to double per-thread data width and halve total memory transactions, achieving up to 1.5x speedup when processing fp16 data
- When converting scalar half kernels to half2, multiply thread index by 2 and halve blockDim to keep the same total element coverage and occupancy
