## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: Any CUDA-capable GPU; No shared memory required; Block size: 2D grid with blockDim.x * blockDim.y threads

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: Any CUDA-capable GPU; No shared memory required
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
