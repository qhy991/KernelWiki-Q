## Solution

Convert entry_size and CTA_SIZE into compile-time template parameters (ENTRY_SIZE_T=576, CTA_SIZE_T=64), enabling #pragma unroll on the inner copy loop so the compiler generates 9 unrolled load-store pairs as straight-line code, eliminating loop counter overhead and branch misprediction penalty

## Key Lessons

- Promote loop-invariant runtime values (entry sizes, CTA sizes) to compile-time template parameters so the NVCC compiler can fully unroll inner loops — eliminates loop counter overhead and produces straight-line code with better ILP scheduling
- Use assert(blockDim.x == TEMPLATE_CTA_SIZE) at kernel entry to catch launch-time mismatches between the runtime grid config and the compile-time template value, preserving safety without sacrificing optimization
- #pragma unroll is only effective when the iteration count is a compile-time constant; with runtime bounds the hint is silently ignored by NVCC, yielding no benefit over a plain loop
