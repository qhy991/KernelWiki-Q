## Solution

Replace the hard-coded stride expression with a runtime q_stride parameter, allowing the kernel to correctly index into query tensors with arbitrary memory layouts while producing identical addresses in the contiguous case

## Key Lessons

- Replacing hard-coded stride assumptions with a runtime stride parameter generalizes the kernel to support non-contiguous tensor layouts with zero performance cost in the contiguous case — the compiler produces identical address arithmetic when q_stride == num_heads * HEAD_SIZE
- Hard-coded stride assumptions in CUDA kernels are a correctness hazard for frameworks that may transpose or pad tensors between operations — always accept stride as a parameter when the memory layout is not guaranteed by the caller
