## Solution

Cache input in shared memory during Pass-1 (with bank-conflict padding via 32-byte stride alignment), synchronize with __syncwarp(), then read from shared memory in Pass-2 to eliminate the redundant global memory load

## Key Lessons

- Shared memory caching is unnecessary when the per-warp working set (e.g., 1024 bytes at hidden_dim=512) fits entirely in the 128KB L1 cache — the second global read already hits L1, and the smem store plus __syncwarp overhead cancels any benefit
- Before applying shared memory caching to eliminate redundant global loads, evaluate the actual memory hierarchy footprint: if per-warp data is small enough to remain L1-resident between passes, the optimization adds overhead without measurable gain
