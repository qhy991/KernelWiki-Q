## Solution

Fuse the two-pass attention into a single kernel using the online softmax algorithm: each thread iterates over keys, computes one QK dot product per iteration, and updates a running max/normalizer/accumulator triplet (m, l, acc) in registers, eliminating the M×N intermediate score matrix entirely

## Key Lessons

- Online softmax (running max + running sum with corrective scaling factors alpha/beta) enables single-pass attention by rescaling the accumulator at each step, eliminating the need to materialize the full attention matrix — applies whenever softmax is followed by a weighted sum and intermediate storage is the bottleneck
- Fusing two kernels that communicate through a large global-memory intermediate into one kernel that keeps state in registers removes O(M×N) global memory traffic, yielding speedups that grow with sequence length
- Map attention dimensions to grid/thread hierarchy so that each block handles one query row (blockIdx.x = qi) and each thread handles one output dimension (threadIdx.x = d), giving natural coalescing on the value and output tensors
