## Solution

Replace uniform kChunkSize-based chunk iteration with variable-length chunk iteration driven by pre-computed cu_chunk_seqlen (CSR-style cumulative token offsets) and last_chunk_indices metadata arrays, using a tokens_processed accumulator for pointer arithmetic instead of multiplying chunk index by a constant. Falls back to uniform block_size chunking when APC metadata is unavailable.

## Key Lessons

- Use pre-computed CSR-style cumulative offset arrays (cu_chunk_seqlen) for variable-length chunk iteration when kernel chunk boundaries must align with external metadata boundaries (e.g., cache blocks) → ensures correct state save/restore at block boundaries → applies to any kernel whose chunking must respect prefix caching or tiling alignment constraints
- Replace index-times-constant pointer arithmetic (chunk * kChunkSize) with a running accumulator (tokens_processed) when chunk sizes vary → avoids stale pointer calculations for non-uniform chunks → applies whenever loop iterations process variable-length segments
- Provide a runtime null-check fallback path when optional metadata pointers are absent → preserves backward compatibility for non-cached modes without code duplication → applies to any kernel adding optional metadata-driven behavior
