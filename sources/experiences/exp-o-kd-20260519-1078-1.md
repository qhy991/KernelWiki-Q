## Solution

Replace Blelloch algorithm with a two-level warp exclusive scan using __shfl_up_sync hardware shuffle instructions: each warp computes an inclusive scan locally via shuffle, warp-level prefix sums are scanned by a single warp, then per-thread results are adjusted — reducing __syncthreads() barriers from O(2*log N) to ~5-6 and eliminating shared-memory bank conflicts

## Key Lessons

- Replace Blelloch (work-efficient) prefix scan with two-level warp shuffle scan using __shfl_up_sync for small data sizes fitting in a single thread block — reduces __syncthreads() barriers from O(2*log N) to a constant (~5-6) and eliminates shared-memory bank conflicts from indirect addressing patterns
- __shfl_up_sync enables warp-level inclusive scan in O(log WARP_SIZE) steps without any shared-memory traffic — ideal for per-warp partial scans that are then combined via a second scan of warp-level aggregates
- For single-block prefix scans (N ≤ 1024), the constant-factor overhead of Blelloch's two-phase algorithm outweighs its O(N) work-efficiency advantage over the O(N log N) but barrier-light warp-shuffle approach
