## Solution

Replace tanh-approximation GELU with exact erf-based GELU: cast to float once, compute f * M_SQRT1_2, call ::erf once, then combine as f * 0.5 * (1 + erf(f * ALPHA)). This reduces arithmetic from 5 ops + 1 SFU to 4 ops + 1 SFU and eliminates the x^3 computation that forced extra half-to-float promotion chains

## Key Lessons

- Prefer exact erf-based GELU (f * 0.5 * (1 + erf(f * M_SQRT1_2))) over tanh-approximation for __half kernels — it uses fewer arithmetic ops (4 vs 5 + x^3), avoids cascading half↔float promotion from x*x*x, and produces results closer to the CPU reference
- In memory-bound __half activation kernels, reducing per-element arithmetic count directly lowers register pressure and instruction latency even when the operation count difference is small (1–3 instructions), because each extra half-precision multiply forces an implicit widening cast
