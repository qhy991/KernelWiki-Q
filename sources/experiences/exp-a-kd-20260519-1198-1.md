## Solution

Split the single combined query+key loop into two independent loops — one iterating over nq=num_heads*embed_dim for query rotation and a second over nk=num_kv_heads*embed_dim for key rotation — removing the conditional branch entirely

## Key Lessons

- Splitting a combined loop into two separate loops to remove a conditional branch that is already always-true (MHA where num_heads==num_kv_heads) provides no benefit — the branch predictor already handles it, and the split introduces loop overhead plus an implicit barrier between loops that can slightly degrade performance
- When a conditional guard is uniform across all threads in a warp (all threads take the same path), removing it by restructuring into separate loops may increase instruction count and synchronization cost rather than reducing it
