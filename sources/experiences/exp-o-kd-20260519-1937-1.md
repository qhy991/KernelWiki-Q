## Solution

Replace scalar loads with vec4_t<scalar_t> vectorized loads that reinterpret the array as 4-element aligned structs, loading 4 elements per 128-bit transaction and processing all four components (x, y, z, w) in unrolled float operations, reducing memory transactions by 4x

## Key Lessons

- Use vec4_t<scalar_t> reinterpret-cast to pack 4 scalar elements into a single 128-bit aligned load, reducing memory transactions by 4x — applies when hidden_size is divisible by 4 and elements are contiguous in memory
- Vectorized loads benefit half-precision (fp16) far more than float32 because 2-byte half elements coalesce poorly without explicit vectorization, while 4-byte float loads already coalesce well natively
- Guard vectorized paths with a hidden_size % 4 == 0 check and provide a scalar fallback to maintain correctness for non-aligned dimensions
