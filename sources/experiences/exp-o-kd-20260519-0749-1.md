## Solution

Extract the sorting loop into a separate moe_token_sort_kernel that uses the global cumsum_buffer as atomic counter bases with a grid-stride loop, enabling up to 65535 blocks of 256 threads each (16.7M threads) instead of a single block

## Key Lessons

- Replace single-block shared-memory atomic sorting with a separate multi-block global-atomic kernel using a grid-stride loop when the workload scales with input size — shared memory limits parallelism to one block (1024 threads) while global atomics unlock GPU-wide parallelism (millions of threads)
- Reuse an already-populated global cumsum_buffer as the atomic counter base for the sorting phase instead of copying offsets into shared memory — avoids shared-memory initialization overhead and eliminates the __syncthreads barrier between initialization and sorting
