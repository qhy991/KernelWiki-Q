## Solution

Replace the Coord2Int extent_ member with a single int extent_row_ (4 bytes), eliminating the unused column component from the register set so the compiler needs one fewer live register across the fully-unrolled epilogue store loops

## Key Lessons

- Store only the actually-needed scalar field from a coordinate/extent struct instead of the full struct when the hot loop accesses just one component — reduces register pressure by eliminating dead live ranges across unrolled loops
- Register-saving optimizations such as removing unused struct members only produce measurable speedups when the kernel is at or near the register spill threshold; in real CUTLASS epilogues composed with element-wise transforms, reductions, and shared-memory staging, this single-register saving can be decisive in avoiding local-memory spills
