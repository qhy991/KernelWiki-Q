## Solution

Replace the O(N) linear scan with O(log N) binary search on the sorted array, reducing each thread from thousands of sequential memory accesses to ~15 random-access comparisons regardless of expert ID

## Key Lessons

- When searching a sorted array for a boundary or count, always prefer binary search O(log N) over linear scan O(N) — the algorithmic improvement dominates any micro-architectural concerns when N is in the thousands
- Linear scans with early exit on sorted data still degrade to O(N) for the highest-valued keys, creating severe thread work imbalance across expert IDs in MoE offset computation
- Binary search's random-access pattern is far cheaper than sequential scan's cache-line-sequencing when the search space is large and only a handful of comparisons (~log2 N) are needed
