## Challenge

For N=8, the same analysis applies: rhs.size() is inlined to return 8, and #pragma unroll with a known constant bound produces identical unrolled SAS


## Solution

Replace int(rhs.size()) with the template parameter N directly in the loop bound and remove the size() member function from the Array struct, eliminating the indirection entirely

## Results

- **speedup**: 1.008x (neutral)


## Key Lessons

- A trivial inline member function (like size() returning a compile-time constant N) is fully inlined by nvcc, producing identical SASS to using the template parameter directly — no measurable performance gain from removing the indirection when the function body is a single return of a constant
- When #pragma unroll is present and the loop bound resolves to a compile-time constant through inlining, the compiler generates fully unrolled code regardless of whether the bound is expressed as size() or a template parameter — verify with SASS inspection before assuming the change matters
- Prefer template parameters directly in loop bounds as a code clarity and maintainability practice, but recognize that for trivially inlineable functions the compiler already achieves the same result
