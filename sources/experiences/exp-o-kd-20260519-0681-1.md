## Challenge

RMSNorm kernel reads input x from global memory twice — once in Pass 1 for sum-of-squares and again in Pass 2 for output computation — wasting 4KB of fp16 global-memory traffic per row and incurring redundant L2 cache hits across 2048 concurrent blocks


## Solution

Cache each fp32 input element in a per-thread register array (xi_cache[kElemsPerThread]) during Pass 1 and reuse those registers in Pass 2, eliminating the second global-memory load; each thread stores only 8 fp32 values (32 bytes), well within register capacity for a 512-thread CTA

## Key Lessons

- Cache input values in per-thread register arrays when the same data is consumed in multiple passes — eliminates redundant global-memory loads and avoids L2 cache-line contention across concurrent blocks, with negligible register pressure when elems_per_thread is small (e.g., ≤8)
- Prefer #pragma unroll on fixed-trip-count loops that populate register arrays — the compiler can eliminate loop overhead and coalesce register assignments into a single load phase
- Register caching yields modest but consistent gains even when L2 partially covers redundant reads — the saving comes from avoiding L2 hit latency and reducing cache-line contention, not just raw bandwidth
