## Challenge

For __half type, the before code passes __half by const reference (a 16-bit value behind a pointer) and the after code passes __half by value plus uses the single-arg overload. At -O3, the compiler inlines the functor an


## Solution

Change functor parameter from const reference to pass-by-value for __half, add a single-arg overload with default-constructed threshold (T()), and eliminate the redundant local variable — the compiler inlines both paths identically at -O3

## Results

- **speedup**: 0.999x (neutral)


## Key Lessons

- For 16-bit GPU-native types like __half, const-ref vs pass-by-value has no performance impact at -O3 because the compiler inlines both into identical register operations — prefer pass-by-value for clarity and to avoid unnecessary pointer indirection in the source
- Memory-bound elementwise kernels (especially half-precision at 2 bytes/element) are dominated by memory throughput, making instruction-level calling convention differences completely undetectable in practice
- Deltas under 0.5% in CUDA event timing on bandwidth-bound kernels should be treated as measurement noise, not attributed to the code change
