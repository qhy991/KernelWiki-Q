## Solution

Replace bool* tree_mask with uint8_t* bit-packed representation where each row uses ceil(draft_token_num/8) bytes instead of draft_token_num bytes, encoding self-bit as bit 0 and connection bits in subsequent positions via bitwise OR read-modify-write, reducing per-row storage from 8 bytes to 1 byte for draft_token_num=8

## Key Lessons

- Bit-packing boolean arrays into uint8_t (8 bits per byte) reduces memory footprint by up to 8x and can reduce global memory traffic — applies when managing sparse boolean connectivity matrices where each element carries one bit of information
- At small problem scales (e.g., draft_token_num=8, batch_size=4), bitwise OR read-modify-write overhead for setting individual bits can offset bandwidth savings from bit-packing — the optimization is most effective when the packed array constitutes a significant fraction of total kernel memory traffic
- When kernel execution is dominated by other memory-intensive loops (e.g., parent_list traversal, retrive_index construction), optimizing a minor data structure like tree_mask yields neutral results — profile to identify the dominant bottleneck before optimizing
