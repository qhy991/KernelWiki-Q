int8_m128_n1024_k512

## Solution

en the bias-add kernel re-reads the entire output tensor from global memory, adds bias in float, and writes back. This extra kernel launch adds ~5-10us of dispatch overhead, and the separate bias-add pass incurs M*N*2 bytes of global memory read traffic plus another M*N*2 bytes of write traffic

## Key Lessons

- By keeping the GEMM result in registers and applying bias inline, the fused kernel eliminates one kernel launch and one full output-tensor read/write round-trip, reducing global memory traffic for the output by roughly half (one write instead of write+read+write).
