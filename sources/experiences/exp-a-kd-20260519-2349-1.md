## Solution

Widen M tile to 64 and narrow N tile to 16 (blockDim 16x64) to increase M-axis parallelism; however blockDim.x=16 < TILE_K=32 forces cooperative loading loops for sA and blockDim.y=64 > TILE_K=32 leaves threads idle during sB load, increasing shared memory from 8KB to 10KB and reducing sB reuse from 32 to 16 columns per cycle

## Key Lessons

- Asymmetric tile dimensions (64x16 vs 32x32) that break the exact blockDim-to-tile coverage ratio force cooperative loading loops and leave threads idle — the loading overhead and reduced sB reuse outweigh the M-axis parallelism gain in simple shared-memory tiled GEMM
- Adaptive tile heuristics from CUTLASS/Hopper TMA-based implementations do not transfer directly to cooperative-loop shared-memory GEMM because TMA hardware handles tile loading without thread overhead — always benchmark tile tuning changes rather than porting heuristics across different memory subsystems
- Prefer balanced tile dimensions where blockDim exactly covers TILE_M, TILE_N, and TILE_K to guarantee single-iteration loads with zero wasted thread cycles — deviate only when profiling confirms the tradeoff is favorable
