## Solution

Replace thread-private shared memory row counting with a grid-stride loop using atomicAdd into a compact shared_counts array padded to WARP_SIZE boundaries, enabling the kernel to scale from 128 to 1024 threads and distribute token counting across 8x more threads

## Key Lessons

- Grid-stride loops with atomicAdd into shared memory scale counting kernels to arbitrarily many threads — the atomics are cheap in shared memory because they resolve at L1 cache speed, and the stride pattern ensures even work distribution regardless of thread count
- Padding shared memory arrays to WARP_SIZE boundaries (padded_num_experts = ((num_experts + 31) / 32) * 32) prevents bank conflicts when multiple warps access adjacent expert counts simultaneously
