## Solution

Replace static __shared__ declarations with extern __shared__ dynamic allocation using a SmemLayout struct that computes total byte sizes and manual pointer offsets, enabling a single contiguous shared memory block with explicit layout control

## Key Lessons

- Converting static __shared__ to dynamic extern __shared__ with manual pointer offsets is neutral when the total shared memory footprint is small (under 2KB) — NVCC already lays out static arrays contiguously, so the manual layout replicates the compiler's work with added pointer-arithmetic overhead
- Dynamic shared memory layout is beneficial for flexibility (supporting multiple kernel variants or larger allocations near the shared memory limit), not for performance — only apply when the layout needs to vary at launch time or when approaching smem capacity limits
