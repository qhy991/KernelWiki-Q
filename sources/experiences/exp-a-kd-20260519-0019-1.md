## Challenge

In the attention split-KV reduction kernel, shared memory (512-byte sLseScale[kMaxSplits]) and global memory tensor pointers are allocated before computing local_split_kv, so short sequences (local_split_kv==1) that need no reduction still consume shared memory and pointer setup overhead on every CTA invocation


## Solution

Move the local_split_kv computation before shared memory declaration and tensor pointer construction, then add an early return when local_split_kv==1 so that short-sequence CTAs skip all resource allocation and reduction work entirely

## Key Lessons

- Reorder computations so that per-CTA decision variables (e.g., local_split_kv) are derived before __shared__ declarations and tensor pointer construction, enabling early returns that skip resource allocation for no-work cases — applies to any kernel with variable-length or conditional work per CTA
- In split-KV attention with variable sequence lengths, short sequences (dim_k <= tile_shape_s) produce local_split_kv==1 requiring no reduction — early-exiting these CTAs avoids wasted shared memory and global pointer setup, improving throughput for mixed-length batches
