## Challenge

The original allgather kernel uses two separate code paths: a main loop over partition elements followed by a small conditional block for remaining elements, which is clean and branch-free inside the unrolled inner loop


## Solution

The after code unifies both paths into a single loop over largest_part with a conditional (gather_from_rank == NGPUS-1 || idx < part) inside the #pragma unroll inner loop; this generalizes correctness for uneven partitions but introduces a branch predicate that the compiler cannot fully eliminate even when the condition is always true, causing ~13% regression

## Key Lessons

- Adding conditionals inside #pragma unroll inner loops adds per-iteration branch-predicate overhead that the compiler may not eliminate even when the condition is always true — prefer separate loop paths for different cases over unified loops with runtime guards
- Specialization wins over code uniformity in tight memory-copy inner loops: a branch-free inner loop with a separate small tail handler outperforms a generalized loop that folds both paths into one conditional
- Computing an extra variable (gather_from_rank) inside an unrolled loop body that previously only computed dst_idx inline increases register pressure and instruction count per unrolled iteration, compounding the cost when the loop is fully unrolled
