## Challenge

Optimization focus: Added gather_cached_kv_kernel_optimized with loop unrolling (factor=4) and separated read/write phases using local index/data arrays, compared to the baseline gather_cached_kv_kernel that processes each element one-at-a-time with immediate read-then-write.
Problem before: The ...


## Solution

The optimization is most effective at smaller token counts where the kernel has fewer concurrent warps and the explicit read batching provides latency hiding that the GPU cannot achieve naturally.

## Results

- **speedup**: 0.965x


## Key Lessons

- In heavily templated CUDA code with scattered memory access patterns (the key cache uses a 5D layout with interleaved x sub-elements), this serialized read-then-write pattern can limit instruction-level parallelism and cause the memory pipeline to stall on read-after-write hazards.
