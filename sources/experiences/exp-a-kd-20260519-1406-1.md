## Challenge

MoE topk gating kernel always applies softmax scoring which requires two warp-level reductions (max + sum) across 8 threads per row, but the alternative sigmoid scoring is explored as a potential simplification that eliminates cross-thread communication


## Solution

Replace always-softmax scoring with compile-time branched scoring via if constexpr and a ScoringFunc enum, adding a sigmoid path (1/(1+exp(-x))) that eliminates warp-level reductions entirely, making each element independent

## Key Lessons

- Replacing softmax with sigmoid in MoE gating can regress performance despite eliminating warp reductions, because sigmoid requires expf+division per element whereas softmax amortizes division to one reciprocal per row via warp sum-reduction — per-element division cost dominates when VPT is small
- Even when an element-wise activation seems simpler (no cross-thread communication), the arithmetic instruction count (8 expf+8 div+8 add vs 8 expf+1 div+8 mul) determines performance in memory-bandwidth-bound kernels where the working set fits in cache
- if constexpr with a ScoringFunc enum is an effective pattern for compile-time dispatch between scoring strategies in template kernels, incurring zero runtime overhead while keeping both paths in the same source
