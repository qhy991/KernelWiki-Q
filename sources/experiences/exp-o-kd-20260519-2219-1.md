## Challenge

Naive GEMM with M=1 (decode batch size 1) launches only grid.y=1 thread block row, leaving the GPU severely underutilized since the entire N=4096 output is computed by a single row of thread blocks


## Solution

Swap A and B operands by transposing inputs so that M_swap=N=4096 and N_swap=M=1, turning grid.y from 1 into 4096 block rows. Since C^T = B^T * A^T, the correct output is recovered from C_swap[j*M+i], mirroring the CUTLASS 3.x swap_ab optimization for small-M decode workloads

## Key Lessons

- For small-M decode GEMM (M=1), swap A and B by computing C^T = B^T * A^T so that the former N dimension becomes the new M dimension, multiplying thread block rows from 1 to N and restoring GPU occupancy — the occupancy gain dominates any cache access pattern penalty
- The AB-swap trick mirrors the CUTLASS 3.x swap_ab optimization: when M << N, the transposed formulation exposes far more parallelism than the natural M×K×N layout, making it the preferred strategy for single-batch decode workloads
