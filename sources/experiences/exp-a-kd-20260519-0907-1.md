## Challenge

MoE alignment kernel host code allocates and frees two GPU buffers (tokens_cnts and cumsum) on every invocation via cudaMalloc/cudaFree, causing four synchronous device-wide stalls per iteration that add ~18ms overhead regardless of kernel workload


## Solution

Pre-allocate the reusable tokens_cnts and cumsum buffers once before the benchmark loop and reuse them across all iterations, eliminating the per-call cudaMalloc/cudaFree overhead that dominated execution time

## Key Lessons

- Pre-allocate reusable GPU buffers once outside the iteration loop instead of cudaMalloc/cudaFree per call — synchronous allocation forces device-wide stalls whose overhead can exceed kernel execution time by orders of magnitude
- cudaMalloc/cudaFree are synchronous operations that serialize the stream; when kernel execution time is small (< 1ms), allocation overhead dominates total latency, making pre-allocation the highest-impact optimization available
- The speedup ratio decreases as kernel workload grows (30x at 4096 tokens vs 90x at 256 tokens) because the fixed allocation overhead becomes a smaller fraction of total time — but the absolute time savings remain constant
