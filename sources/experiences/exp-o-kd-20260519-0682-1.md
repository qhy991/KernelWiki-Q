## Solution

Cache each fp32-converted input element in a per-thread register array (xi_cache[16]) during Pass 1 so that Pass 2 reuses registers instead of re-reading from global memory, eliminating the redundant 8KB fp16 load per row; the 16×float (64-byte) register footprint per thread stays within register budget for a 512-thread CTA, and #pragma unroll on both loops enables the compiler to fully unroll the fixed-trip-count loops

## Key Lessons

- Cache input elements in per-thread register arrays when a kernel needs the same data in multiple passes — for RMSNorm with kDim=8192 and 512 threads, 16 floats (64 bytes) per thread fits in registers and eliminates a full redundant global read, saving ~9.5% latency
- Prefer register caching over relying on L2 cache reuse when the working set per-SM is large — 1024 rows × 8192 elements × 2 bytes = 16 MB far exceeds L2 capacity, so the second read will miss L2 and go to DRAM
- Use constexpr trip counts and #pragma unroll on fixed-size loops to help the compiler fully unroll register-cache loops, converting indexed array access into direct register references
