## Solution

Add the missing tile_m * ldr row-stride offset to the vector pointer computation so each tile reads bias values from the correct row in the 2D Vector array; this is a correctness fix with negligible performance impact since it adds one integer multiply-add before the per-element loop

## Key Lessons

- When a 2D row-major bias array is broadcast per tile-row in CUTLASS epilogues, the pointer must include both the column offset and the tile-row stride (tiled_coord.m() * ldr) — omitting the row stride causes all tiles beyond the first row to silently read incorrect bias values
- A single integer addition to a pointer base computed once before a per-element loop has negligible performance impact compared to the loop's memory traffic, so correctness fixes of this nature are effectively free
