## Challenge

The aligned cumsum computation runs on a single thread (threadIdx.x==0) with a serial O(num_experts) loop, creating a bottleneck that scales linearly with expert count


## Solution

Replace the single-thread serial cumsum loop with CUB BlockScan::ExclusiveSum, computing the aligned exclusive prefix sum across all experts in O(log N) parallel steps using warp-level primitives

## Key Lessons

- Use CUB BlockScan::ExclusiveSum to replace serial cumsum loops on a single thread — reduces prefix sum from O(N) serial to O(log N) parallel and is especially impactful when N (num_experts) is large (e.g., 128+)
- When using BlockScan, each thread computes its own input value (e.g., aligned_count from its expert slot) and the scan distributes results to all threads, eliminating the single-thread bottleneck entirely
