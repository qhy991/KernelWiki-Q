## Challenge

cuBLAS GEMM wrapper uses a global singleton handle with a per-call null check on every invocation, adding a host-side branch on the hot path before each cublasGemmEx call


## Solution

Pass the cuBLAS handle as an explicit function parameter created once at application startup, eliminating the global singleton and per-call null check from the GEMM wrapper entirely

## Key Lessons

- Replacing a global singleton handle with an explicit parameter is a software-engineering improvement (better testability, no hidden state), but a single predictable host-side branch (null check) is negligible compared to GPU kernel execution time — applies to any library wrapper where host overhead is dwarfed by device work
- Static local variables (static half alpha/beta) in hot-path functions add no measurable cost in cuBLAS calls — the real optimization opportunity is in algorithmic choices (math mode, tensor cores) not micro host-side cleanup
