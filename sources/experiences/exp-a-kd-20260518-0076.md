## Solution

Replace the 128-iteration sub-byte element loop with a 4-iteration word-level OR loop that applies bitwise OR directly on uint32_t storage words, exploiting the fact that OR on packed words correctly computes all 32 per-bit ORs in a single native instruction

## Key Lessons

- For bitwise operations (AND, OR, XOR) on packed sub-byte arrays, apply the operation directly on whole storage words — a single uint32_t OR computes 32 simultaneous 1-bit ORs, eliminating all shift-mask and read-modify-write overhead — applies whenever the logical operation maps to a native word-level bitwise instruction
- A loop over N individual bit accesses costs O(N) shift+mask+branch instructions; replacing it with O(N/W) word-level operations where W is the word width reduces instruction count by the packing factor (here 32x from 128 iterations to 4) — applies to any packed data layout where the per-element operation is lane-independent
