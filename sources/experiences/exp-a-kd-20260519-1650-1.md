## Solution

Fuse both kernels into a single advance_step_flashinfer_kernel that reads seq_len once into a register, derives all six outputs from that value and a single block_tables lookup, eliminating the second kernel launch and its redundant global-memory reads

## Key Lessons

- Kernel fusion eliminates per-launch driver overhead (~5-10us) and redundant global-memory reads when consecutive kernels operate on overlapping data — applies whenever a chain of lightweight kernels read the same input arrays
- When a second kernel depends on values written by a first kernel to global memory, fusing lets the second kernel's logic reuse the first kernel's register-resident values directly, bypassing a store-load round trip through DRAM
