## Solution

Precompute all 256 byte-to-byte mappings into a constant-memory lookup table (kNibbleLUT[256]) and replace per-nibble branching with a single constant-memory read per byte, eliminating branch divergence and reducing per-element computation to a table lookup

## Key Lessons

- Precompute per-element lookup tables into __constant__ memory when the input domain is small (256 entries for byte-to-byte mapping) — constant memory provides cached broadcast reads and eliminates branch divergence overhead from per-element conditional logic
- A 256-entry byte LUT covers all possible byte→byte transformations, turning two per-nibble branch evaluations into one constant-memory read per byte, which also simplifies vectorized kernel design by removing the need for per-nibble function calls in the inner loop
