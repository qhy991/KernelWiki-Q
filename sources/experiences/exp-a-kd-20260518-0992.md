## Key Lessons

- The real benefit of the vec_n_t pattern in the original commit is that it makes vectorized load intent explicit to the compiler in heavily templated code where coalescing heuristics may be less effective, and it enables the template-parameter VEC_SIZE to adapt to different hidden_size alignments.
- Both before and after versions have identical memory access patterns at the hardware level: the before code's unrolled contiguous scalar loads at input_row[idx*8+j] are coalesced by NVCC into the same 128-bit global memory transactions as the after code's explicit vec_n_t packed loads.
