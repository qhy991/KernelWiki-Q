## Solution

Replace round-robin single-slot assignment with sequential assignment starting from offset 0, adding a loop to fill all shared-expert slots when num_fused_shared_experts > 1, ensuring correct behavior for arbitrary numbers of fused shared experts

## Key Lessons

- Round-robin assignment (row % N) and sequential assignment (offset starting at 0) produce identical results when N=1 since both compute offset=0 for every row — validate multi-slot logic by testing with N > 1 to expose functional differences
- When adding a loop to fill multiple shared-expert slots, guard it with an if (N > 1) check to avoid the loop overhead for the common single-slot case, keeping the fast path branch-free
