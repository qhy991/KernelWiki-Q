## Solution

Add an early-return guard that checks if cache_index equals pad_slot_id and immediately returns, skipping the entire SSM recurrence computation for padding slots

## Key Lessons

- Adding an early-return guard for padding/invalid slots is a zero-cost defensive pattern when no padding exists — the single integer comparison is negligible compared to the kernel's memory and arithmetic workload, but it saves significant computation when padding ratios are high
- Early-return guards in GPU kernels do not cause measurable warp divergence overhead when the branch is uniformly not taken across all threads in a warp (all valid entries take the same path)
