## Solution

Replace the triple-nested loop with direct index arithmetic: access_m_idx = ldsm_vec_num % kTilesPerInstruction, inner_idx = (ldsm_vec_num / kTilesPerInstruction) % kAccessesInner, inst_m_idx = ldsm_vec_num / (kTilesPerInstruction * kAccessesInner), computing the same offset in constant-time without any loop or conditional branch

## Key Lessons

- Replace search loops that iterate to find a known index with direct integer division/modulo decomposition — when the loop variable is reconstructed from a flat index via row-major ordering (idx = c + C*(r + R*d)), the inverse (c = idx % C, r = (idx/C) % R, d = idx/(C*R)) computes the same result in O(1) without any branch
- Constructor hot paths in template-heavy libraries (e.g., CUTLASS warp iterators) benefit from eliminating unrolled loops even when branches are predictable — the compiler may not always eliminate dead iterations across all GPU architectures
