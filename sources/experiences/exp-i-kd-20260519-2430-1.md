## Challenge

Implement a CUTE-library-based kernel that copies data from global memory to shared memory via cute::copy, synchronizes threads, scales each element by 2, and writes the result back to global memory


## Solution

Use CUTE tensor abstractions (make_tensor + make_smem_ptr/make_gmem_ptr + Int<1> layout) to create 1D tensor views over global and shared memory, then apply cute::copy for the global-to-shared transfer followed by __syncthreads() and a scalar scale-and-store to global output

## Key Lessons

- CUTE's make_tensor(make_smem_ptr(...), Int<1>{}) creates a 1D shared-memory tensor view and make_tensor(make_gmem_ptr(...), Int<1>{}) creates a global-memory view — cute::copy(source, dest) handles the element-wise transfer between them without manual indexing
- extern __shared__ with dynamic allocation is required for CUTE shared-memory tensors when the shared memory size is determined at kernel launch time — the smem pointer is partitioned per-thread using threadIdx.x offsets
- __syncthreads() is mandatory between a shared-memory write (via cute::copy) and any shared-memory read to prevent data races — CUTE's copy abstraction does not implicitly synchronize across threads
