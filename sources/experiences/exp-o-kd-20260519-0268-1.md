## Solution

Change the sign from minus to plus in the formula (key_start - num_keys + num_queries), correctly positioning the bottom-right causal diagonal relative to the key position so that the smallest query index is derived from the proper geometric relationship between key offset and query count

## Key Lessons

- Sign errors in causal masking boundary formulas are particularly insidious because they produce silently wrong results (negative values clamped to zero) rather than crashes — always verify causal diagonal formulas against a geometric diagram of the attention pattern
- For CausalFromBottomRight masking, the smallest query index for a given key must account for the query range growing with key position: key_start - num_keys + num_queries, not key_start - num_keys - num_queries
