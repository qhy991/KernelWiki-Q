## Challenge

Two separate kernels — silu_and_mul_kernel writes a 512KB __half intermediate buffer (64 tokens x 4096 hidden) to global memory, then scaled_quant_kernel reads it back for int8 quantization, incurring ~1MB of redundant global memory traffic and an extra kernel launch


## Solution

Fuse SiLU-and-mul and scaled int8 quantization into a single kernel so the float silu_result stays in registers and the final int8 is written directly, eliminating the entire 512KB write + 512KB read intermediate round-trip and one kernel launch

## Key Lessons

- Fuse consecutive elementwise kernels that pass intermediate results through global memory into a single kernel — keeping intermediates in registers eliminates write-read round-trips that scale linearly with data size
- Kernel fusion removes both the global memory traffic for intermediates and the launch overhead of the second kernel, compounding into significant speedups when global memory bandwidth is the bottleneck
- When the intermediate type differs from both input and output (here fp16→fp32→int8), fusion avoids unnecessary half-precision store/load of intermediate values that would lose precision or waste bandwidth
