## Solution

Replace the two-pass global memory access pattern with a single cp.async bulk load where only MOE_BLOCK_SIZE/4=8 threads each issue one asynchronous 16-byte copy into shared memory, eliminating all redundant global reads

## Key Lessons

- Use cp.async.bulk (via inline PTX cp.async.ca.shared.global) to batch small fixed-size global-to-shared copies into a single non-blocking operation — eliminates redundant global reads when the same data is accessed by multiple threads or in multiple passes
- When all threads redundantly read the same global memory addresses in a scan loop, replace with a single cooperative async load where each thread owns one unique chunk, then synchronize once with cp.async.wait_group
