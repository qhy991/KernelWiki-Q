## Challenge

Optimization focus: Fused renormalize into moeTopK fallback kernel after topk selection loop.
Problem before: The moeTopK fallback kernel (used for non-power-of-2 expert counts) did not support renormalize. After topk selection, a separate renormalize step was needed to normalize the k selected weig


## Solution

kernel launch and global memory read+write.
Applied change: A renormalize boolean parameter was added to moeTopK. During the topk selection loop, when renormalize is true, each winning weight value is accumulated into selected_sum. After the loop, thread 0 of each block divides each of the k output 

## Key Lessons

- Expected impact: Eliminates one kernel launch and one global memory read+write cycle over the topk weight output in the fallback path.
