## Solution

Replace erff-based GELU with a Taylor approximation using the single-cycle PTX hardware instruction tanh.approx.f32 via inline asm, computing GELU as 0.5f * f * (1.0f + fast_tanh(k0 * f * (1.0f + k1 * f * f))) where the multiply-add chain is well-pipelined alongside the fast tanh

## Key Lessons

- Replace erff-based GELU with a Taylor approximation using tanh.approx.f32 PTX hardware instruction — the single-cycle hardware tanh eliminates the expensive multi-instruction SFU erff call, yielding up to 2x speedup in compute-bound fp16 activation kernels
- Use inline PTX asm (tanh.approx.f32) to access hardware approximation instructions that are not exposed through standard CUDA math APIs, guarded by __CUDA_ARCH__ for host compatibility
- GELU can be accurately approximated as 0.5f * f * (1.0f + tanh(k0 * f * (1.0f + k1 * f * f))) where k0=0.7978845608028654f and k1=0.044715f — suitable whenever slight numerical differences (<0.05 absolute error) are acceptable
