## Key Lessons

- For tall-skinny GEMM problems (where K >> N), the default persistent scheduler leaves SMs underutilized because the number of output tiles along M and N is small relative to the number of available SMs, resulting in idle hardware and lower throughput.
- Measured performance improvement depends on problem shape and should be validated via microbenchmark for the specific k > 3n regime on SM90+ hardware.
