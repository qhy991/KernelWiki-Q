## Solution

Implement scalar class with unique_ptr RAII wrapping cudaMalloc and custom scalar_deleter, upload the device pointer to a global __device__ variable via cudaMemcpyToSymbol for zero-argument side effects; in the kernel, call check_cluster_shape template for dimension validation and use cute::block_id_in_cluster() to identify the first thread of the first block for the write

## Key Lessons

- Use cudaMemcpyToSymbol to upload a device pointer to a global __device__ variable, enabling zero-argument kernels to produce observable side effects through a pre-published global pointer — essential for testing cluster launch where kernel arguments may be constrained
- Combine check_cluster_shape<dims...>() with assert() for compile-time-parameterized runtime cluster dimension validation, ensuring the launched cluster geometry matches expectations on SM 90+ hardware
- Use cute::block_id_in_cluster() (not blockIdx) to get the block's position within its cluster when writing side effects gated on the first block, since blockIdx indexes globally across the grid while block_id_in_cluster indexes within the cluster group
