## Challenge

With __half precision and d=4096, the kernel is still memory-bandwidth-bound — each thread reads two half values (x and y) and writes one, and the SiLU computation casts to float for the expf/reciprocal then back to half.


## Solution

The function-pointer template parameter generalization does not change the instruction stream: NVCC instantiates act_and_mul_kernel<__half, silu_kernel<__half>> and fully inlines silu_kernel, producing the same SASS as t

## Key Lessons

- The function-pointer template parameter generalization does not change the instruction stream: NVCC instantiates act_and_mul_kernel<__half, silu_kernel<__half>> and fully inlines silu_kernel, producing the same SASS as the direct silu call in the before version.
