## Solution

Fuse absmax reduction and quantization into a single per-token kernel (one block per token), eliminating the redundant full-input re-read between two kernel passes, removing global atomics via block-local shared memory scale, and reducing kernel launch overhead

## Key Lessons

- Fusing absmax reduction and quantization into a single per-token kernel halves global memory reads (input read once for absmax, once for quant in same kernel vs. read twice across two kernels) — applies whenever a reduction result feeds into an elementwise transformation on the same data
- Per-token block assignment (blockIdx.x = token index) enables block-local scale computation without global atomics, since each block independently processes its own token's hidden dimension
