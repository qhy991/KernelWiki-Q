## Solution

Replace coalesce() with a structured 2D layout that groups 256 consecutive 4-bit elements (128 bytes) into a contiguous inner dimension, preserving 32-byte TMA sector boundaries so Hopper TMA can issue full-sector loads instead of partial ones

## Key Lessons

- Use structured multi-dimensional CuTe layouts instead of coalesce() for packed low-precision data on sm_90 (Hopper) — TMA hardware uses the layout descriptor structure to determine 32-byte sector boundaries, and coalesce destroys the alignment information needed for full-sector loads
- CuTe layout changes that preserve logical-to-physical offset mapping may still affect performance on architectures where the layout descriptor is consumed by hardware (TMA on sm_90) — benchmarking on the target architecture is essential since sm_80 (Ampere) shows no difference
- Group packed low-precision elements so the inner dimension spans at least 64 values (32 bytes) to match TMA sector granularity — 256 consecutive 4-bit values = 128 bytes = 4 full TMA sectors
