## Challenge

Column permutation kernel assumes input matrix is contiguous (lda == size_k) by using a single row_stride for both input and output addressing, preventing correct operation on padded/strided input matrices where the leading dimension differs from the logical column count.


## Solution

Decouple input and output row strides by introducing an explicit lda (leading dimension) parameter: input rows are indexed via lda while output rows use size_k, enabling correct permutation on non-contiguous padded matrices without changing the contiguous-case memory access pattern.

## Key Lessons

- Separate input and output row strides via an explicit lda parameter to support non-contiguous (padded) input matrices — when lda == size_k the compiler produces identical PTX, so there is zero overhead in the contiguous case
- Always design kernels with an explicit leading dimension parameter rather than assuming a contiguous layout, since padded matrices arise naturally from batched GEMM, quantization padding, and memory alignment requirements without incurring any performance penalty when the strides happen to match
