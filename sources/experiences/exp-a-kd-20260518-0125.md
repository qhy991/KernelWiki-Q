## Challenge

The before code reads warp_id/lane_id/thread_id directly from threadIdx via __forceinline__ getters each iteration, which NVCC compiles to simple register moves from the special threadIdx register


## Solution

The after code replaces these with warp_uniform_func (a __shfl_sync broadcast from lane 0), called 3 times per iteration inside the rematerializeThreadIds lambda.
