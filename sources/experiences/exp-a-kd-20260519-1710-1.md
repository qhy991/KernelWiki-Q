## Solution

Eliminate redundant round-trip casts by performing the addition in native scalar_t type (scalar_t z = input[...]; z += residual[...];), casting only once to float for the variance accumulation, and storing the native-type sum directly without a cast-back

## Key Lessons

- When scalar_t is a reduced-precision type (e.g. __half/bfloat16), redundant float↔scalar_t round-trip casts in tight reduction loops generate actual hardware conversion instructions (F2F in PTX) — eliminating them by computing in native type and casting once reduces instruction count
- This FP32 control case (scalar_t=float) confirms the optimization is a no-op when types already match: the compiler folds identity casts away, producing identical PTX, validating that the cast-elimination benefit is exclusive to reduced-precision instantiations
- For generic templated kernels, structure arithmetic to use the narrowest necessary type (scalar_t) for storage-path operations and cast to the wider type (float) only where precision is required (variance accumulation)
