## Solution

Add a compile-time kNumThreads template parameter (default 32) that controls the initial mask value to kNumThreads/2, performing exactly log2(kNumThreads) shuffle rounds instead of always 5, with static_asserts enforcing valid power-of-2 range [1,32]

## Key Lessons

- Parameterize warp shuffle reduction with a compile-time thread count template to eliminate unnecessary shuffle rounds — when only k threads participate, log2(k) rounds suffice instead of always 5, yielding large speedups for sub-warp patterns
- Enforce power-of-2 constraints via static_assert on the thread count template parameter — butterfly reduction (__shfl_xor_sync) requires power-of-2 group sizes for correctness, and the static_assert catches misuse at compile time
- Default the thread count to full warp size (32) for backward compatibility — callers that need full-warp reduction continue working unchanged while sub-warp callers explicitly specify smaller counts
