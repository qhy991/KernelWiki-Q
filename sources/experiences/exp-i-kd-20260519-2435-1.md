## Challenge

Implement a multi-tier memory copy kernel (gmem→rmem→smem→gmem) using the Hopper STSM primitive via cute::copy_stsm, requiring proper uint128_t alignment for shared memory, register file staging, and correct synchronization barriers between memory tiers


## Solution

Stage data through three tiers using uint32_t register arrays for gmem→rmem loads, cute::copy_stsm with uint128_t-aligned shared memory pointers for the STSM hardware primitive, __syncthreads() barrier between smem write and read, then scatter from smem back to gmem

## Key Lessons

- cute::copy_stsm invokes the Hopper STSM hardware primitive for efficient register-to-shared-memory transfer — prefer it over manual shared memory writes when targeting SM90+ to leverage the dedicated store path
- STSM requires uint128_t-aligned shared memory pointers: reinterpret_cast the __shared__ buffer to uint128_t* before passing to copy_stsm, ensuring the hardware receives a properly aligned destination
- The gmem→rmem→smem→gmem pipeline requires __syncthreads() between the STSM write phase and the smem read-back phase so all threads complete their shared memory stores before any thread reads
