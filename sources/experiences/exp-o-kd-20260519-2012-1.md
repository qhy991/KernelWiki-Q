## Solution

Swap A and B operands and use CUBLAS_OP_T for both to compute C^T = B^T[N×K] * A^T[K×M] with leading dimension N=512 as the m-dimension, eliminating all M-padding by letting the small M=32 become the n-dimension instead

## Key Lessons

- When a GEMM dimension is small (e.g., M=32) and requires padding for cuBLAS efficiency, swap operands using CUBLAS_OP_T to make the larger dimension (N=512) the leading m-parameter — this eliminates padding waste while keeping the large dimension as the primary parallelism axis
- Computing C^T = B^T * A^T produces the transpose of C = A * B using the same data with no extra copy, since the transpose is absorbed into the cuBLAS operand descriptors — the result must be indexed as C_T[n + m*N] instead of C[m + n*M]
- Operand-swapping speedup scales with the padding ratio but is bounded by the small dimension becoming the n-parameter in cuBLAS — the 4x FLOP reduction yielded only 1.22x wall-clock improvement because n=32 limits per-SM throughput compared to the padded m=128
