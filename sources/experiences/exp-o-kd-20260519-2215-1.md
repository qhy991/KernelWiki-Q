## Solution

Pre-compute the aligned expert prefix offsets on the host into a dedicated array (O(num_local_experts) sequential pass), then replace the GPU-side serial accumulation loop with two __ldg reads per thread — one for the pre-computed aligned base offset and one for the token offset within the expert — yielding O(1) per-thread work with no shared memory, no __syncthreads, and no warp shuffle

## Key Lessons

- Pre-compute prefix sums on the host when the data is static and available before kernel launch — eliminates GPU-side serial accumulation loops, shared memory allocation, __syncthreads barriers, and warp shuffle broadcasts, trading a cheap O(N) host pass for massive per-thread work reduction
- When only one lane in a warp performs useful computation (serial loop on lane 0 + __shfl_sync broadcast), warp utilization drops to 1/32 — replacing with independent O(1) __ldg lookups per thread achieves near-full warp utilization
- For index-mapping kernels where the mapping table is constant across all threads, a pre-computed lookup table with __ldg reads outperforms cooperative shared-memory loading followed by serial per-warp computation, even though shared memory has lower raw latency than global memory
