## Challenge

In merge_attn_states_kernel, every token thread performs the full LSE rescaling merge (loading prefix/suffix LSE, computing max, exponentiating, dividing, per-element FMA) even when the token has no prefix KV context, causing wasted prefix loads and unnecessary exp/div/FMA operations


## Solution

Add an early-exit branch at the top of the kernel: when token_idx >= prefix_num_tokens, the thread copies suffix output directly to output via a single 128b pack load/store and writes suffix LSE, bypassing all prefix loads, LSE comparisons, exp, division, and FMA operations for suffix-only tokens

## Key Lessons

- Add early-exit fast paths when a kernel branch reduces to a simple copy (e.g., merge with empty prefix yields identity) — the savings scale with the fraction of threads taking the fast path, from eliminating unnecessary exp/div/FMA and redundant data loads
- Use 128b pack load/store for direct memory copies in early-exit paths instead of element-wise loops — a single pack transaction moves 4 halfs or 2 floats at once, matching the memory subsystem's natural width
- Guard LSE writes with output_lse != nullptr && pack_idx == 0 so only one thread per head writes the scalar LSE value, avoiding warp-divergent redundant writes in the fast path
