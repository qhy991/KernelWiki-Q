## Solution

Add an early-exit branch for suffix-only tokens (token_idx >= prefix_num_tokens) that directly copies suffix_output via a single 128b pack load/store and reads suffix LSE, bypassing prefix LSE load, max/exp/division arithmetic, and the per-element FMA merge loop entirely

## Key Lessons

- Add early-exit fast paths for tokens that skip the merge (suffix-only with no prefix contribution) — when 50% of tokens are suffix-only, skipping prefix LSE load, exp/div/FMA arithmetic, and prefix data load for those threads reduces average per-thread instruction count and memory traffic proportionally
- When p_scale is provably 0 (prefix_lse=-inf), the merge reduces to a pure copy — detecting this condition structurally (token_idx >= prefix_num_tokens) before entering the arithmetic path is cheaper than letting the hardware compute through the zero-weighted path
- Data-dependent early exits in GPU kernels are effective when a significant fraction of threads can skip expensive computation — the warp divergence cost is outweighed by the saved arithmetic and memory bandwidth when the skip ratio is high (here 50%)
