## Solution

Replace the scalar stride loop with a vectorized loop using vec_n_t<float,4> (16-byte aligned struct) and a DefaultVecOp wrapper that applies the scalar multiply per element inside the vector, causing nvcc to emit 128-bit LDG.128/STG.128 instructions and reducing total memory transactions by 4x

## Key Lessons

- For memory-bound elementwise kernels, cast aligned float pointers to vec_n_t<float,4>* and process 4 elements per iteration — nvcc emits LDG.128/STG.128, cutting memory transactions by 4x and improving L2 cache line utilization
- Use a DefaultVecOp wrapper with #pragma unroll to apply a scalar lambda per vector element — keeps the code generic while ensuring the compiler vectorizes the inner loop
- Always include a tail loop (scalar fallback) after the vectorized loop to handle remaining elements when hidden_size is not a multiple of VEC_SIZE, avoiding out-of-bounds access
