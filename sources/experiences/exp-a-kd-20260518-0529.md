## Solution

Enabled BLOCK_SWIZZLE=true and introduced a 3D grid launch where blockIdx.z selects swizzle groups interleaving across N-dimension columns (stride=2048) to improve L2 cache spatial locality between adjacent thread blocks

## Key Lessons

- Block swizzle via 3D grid interleaving only produces L2 cache benefits when N_SWIZZLE > 1, i.e., the N dimension spans multiple stride-sized groups; for problem sizes smaller than the stride it degenerates to identity mapping and adds minor overhead → always verify N_SWIZZLE > 1 before expecting gains
- When the swizzle stride (2048) exceeds the problem dimension (N=1024), the 3D grid has gridDim.z=1 and bx reduces to blockIdx.x, making the optimization a no-op; the measured 2.27% regression comes from host-side N_SWIZZLE computation and 3D grid setup cost, not from kernel execution changes
