## Challenge

Per-tensor FP8 quantization kernel receives scale as a device pointer and every thread performs a global memory load followed by a reciprocal division (1.0f / (*scale_ptr)) to compute scale_val, wasting memory bandwidth and arithmetic units redundantly across all threads


## Solution

Move the reciprocal division to the host side and pass scale_val as a scalar kernel argument instead of a device pointer, so the GPU hardware broadcasts the value to all threads via the constant cache or register file without any global memory load or per-thread division

## Key Lessons

- Pass per-tensor scalar parameters (scale, bias, epsilon) as scalar kernel arguments instead of device pointers — the CUDA driver places scalar args in the kernel parameter buffer which is broadcast via constant cache or registers, eliminating redundant global loads and per-thread arithmetic
- Move constant-once computations (reciprocal of a single scalar) to the host side before kernel launch — avoids N threads redundantly computing the same value and removes a global memory round-trip for a single float
