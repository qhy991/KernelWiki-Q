## Challenge

Attention is implemented as two separate kernels (QK scoring + softmax-value) with an intermediate scores[M*N] buffer in global memory, forcing both kernels to stream through the full M×N attention matrix and incurring two kernel launch overheads plus a large intermediate memory allocation


## Solution

Fuse the two separate QK-score and softmax-value kernels into a single kernel using the online softmax algorithm: each thread computes one (qi, d) output by iterating over the N sequence positions, maintaining a running maximum m and normalization factor l, and rescaling the accumulator on each step — eliminating the intermediate M×N scores buffer entirely

## Key Lessons

- Fuse separate QK-scoring and softmax-value kernels into a single kernel using the online softmax algorithm to eliminate the intermediate M×N attention score buffer in global memory — the speedup grows with sequence length because memory savings are O(N²) while compute stays O(N)
- The online softmax maintains a running maximum m and normalization sum l, rescaling the accumulator by alpha=exp(m-new_m) on each iteration — this avoids the two-pass requirement (find-max then normalize) and enables single-pass fusion without materializing the full attention matrix
- Thread mapping for fused attention: one block per query row (blockIdx.x = qi), one thread per head dimension (threadIdx.x = d) — each thread iterates independently over N keys, requiring only registers for m, l, acc with zero shared memory
