## Solution

Replace synchronous global memory reads with cp.async.cg.shared.global PTX instructions that initiate asynchronous 16-byte global-to-shared copies running concurrently with compute, using a 3-stage shared memory buffer so each compute iteration reads from shared memory while the next iteration's global load is already in flight, hiding global memory latency

## Key Lessons

- Use cp.async.cg.shared.global PTX instructions with multi-stage shared memory buffers to overlap global memory loads with compute — preloads NUM_STAGES-1 iterations ahead, then each compute iteration issues wait_group + new load to sustain the pipeline
- Prefer bfloat162 (nv_bfloat162) vectorized pairs with __bfloat16222float2 / __float22bfloat162_rn over scalar bf16 conversion for paired element processing — halves conversion overhead and enables __hmul2 / __hmin2 / __hmax2 vector intrinsics
- Pack 4-element FP8 output with __nv_fp8x4_e4m3(results[0], results[1]) and write via a single reinterpret_cast store instead of 4 individual __nv_fp8_e4m3 writes — reduces store instructions from 4 to 1 in the quantization epilogue
