## Solution

Replace synchronous B-matrix global-to-shared loads with PTX cp.async.ca.shared.global asynchronous copies, using __cvta_generic_to_shared for shared pointer conversion, CP_ASYNC_COMMIT_GROUP to issue the async group, and CP_ASYNC_WAIT_GROUP(0) before __syncthreads() so that B-tile loads overlap with the compute loop on the current tile

## Key Lessons

- Use cp.async (cp.async.ca.shared.global) for global-to-shared memory copies in tiled GEMM to overlap data movement with computation — the async copy runs on the memory controller while SM executes the compute loop, hiding load latency without consuming registers
- Always pair CP_ASYNC_COMMIT_GROUP() + CP_ASYNC_WAIT_GROUP(0) around the compute section: commit before compute so the async transfer begins, then wait before __syncthreads() to ensure shared memory is ready for the next tile
- Use __cvta_generic_to_shared() to convert generic shared memory pointers to the shared address space required by cp.async PTX instructions — direct reinterpret_cast is insufficient for the async path
