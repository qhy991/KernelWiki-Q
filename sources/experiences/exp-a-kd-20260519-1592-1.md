## Challenge

MoE problem-size computation uses atomicAdd inside the per-element loop for every matching token, causing O(topk_length) atomic operations per expert block and serializing threads on the shared atomic_buffer[expert_id] address


## Solution

Accumulate match counts in a per-thread register variable across all loop iterations, then perform a single atomicAdd per thread after the loop — reducing atomic operations from O(topk_length) to O(THREADS_PER_EXPERT) per expert and eliminating serialization on the shared memory address

## Key Lessons

- Replace conditional atomicAdd inside loops with register-local accumulation followed by a single atomicAdd after the loop — reduces atomic contention from O(N) to O(threads_per_block) and converts expensive global memory atomics into free register arithmetic
- Use predicate-to-int conversion (expr == value) to accumulate boolean matches as integer counts in a register, avoiding branch divergence and atomic overhead simultaneously
