## Challenge

A separate zero_buffer_kernel is launched to zero the cumsum_buffer (257 int32 elements) before every invocation of the align+sort kernel pipeline, wasting a full kernel launch submission and a small memset despite the align kernel already overwriting every cumsum entry during its serial prefix-sum computation


## Solution

Remove the zero_buffer_kernel call entirely: moe_align_block_size_kernel already writes cumsum[0]=0 and fills all subsequent prefix-sum entries in its serial loop, making prior zeroing of cumsum_buffer redundant and saving one full kernel launch submission per pipeline invocation

## Key Lessons

- Eliminate redundant buffer-zeroing kernel launches when the downstream consumer already fully initializes every element of the buffer — the savings come from removing the fixed kernel launch overhead (~2.7us), not from the memory operations themselves
- When a kernel performs a serial prefix-sum that writes cumsum[0]=0 and then every cumsum[i] from the previous value, no prior zeroing of the cumsum array is needed — the prefix-sum itself is a complete initialization
