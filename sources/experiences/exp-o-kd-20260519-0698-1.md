## Challenge

MoE alignment kernel benchmark allocates temporary buffers (tokens_cnts, cumsum) via cudaMalloc inside each loop iteration and frees them with cudaFree after the kernel launch, causing four synchronous device-wide stalls per iteration that dominate total runtime (~18ms overhead vs ~0.6ms kernel execution)


## Solution

Pre-allocate the fixed-size temporary buffers (tokens_cnts and cumsum) once before the benchmark loop and reuse them across all iterations, eliminating the per-call cudaMalloc/cudaFree overhead that forced four synchronous stream stalls per iteration

## Key Lessons

- Pre-allocate fixed-size GPU temporary buffers outside hot loops instead of cudaMalloc/cudaFree per iteration — synchronous cudaMalloc/cudaFree force device-wide stalls that can dominate runtime when kernel execution is fast relative to allocation overhead
- When benchmark GPU time (~18ms) is nearly invariant across input sizes, suspect allocation/deallocation overhead rather than kernel compute — the kernel itself was only ~0.6ms, with ~17.4ms consumed by four synchronous malloc/free round-trips per iteration
