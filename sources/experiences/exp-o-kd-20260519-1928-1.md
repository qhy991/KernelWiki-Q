## Solution

Unify both branches by always passing scales as a device pointer with a boolean is-vector flag (numel() != 1); Sm90ColOrScalarBroadcast/Sm90RowOrScalarBroadcast epilogue types interpret this flag at runtime to broadcast a single device value or load per-row/column data, eliminating the GPU→CPU sync entirely

## Key Lessons

- Never call .item<float>() on a GPU tensor inside a dispatch path — it triggers an implicit cudaMemcpy D2H that forces a full device synchronization; instead, keep data on-device and use a boolean flag to communicate scalar-vs-vector semantics to the kernel
- CUTLASS Sm90ColOrScalarBroadcast/Sm90RowOrScalarBroadcast accept a device pointer plus an is_scalar boolean — leverage this interface to avoid host-side value extraction and keep the entire pipeline on-device
- Conditional branches that move data between device and host (e.g., nullptr + CPU value vs device pointer) should be unified into a single device-side representation with a runtime flag — this also prevents torch.compile graph breaks caused by cross-device data movement
