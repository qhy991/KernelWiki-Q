## Solution

Consolidate all three kernels into a single templated advance_step_kernel that reads seq_lens once, computes next_seq_len and next_input_pos locally, and writes tokens, positions, seq_lens, and slot_mapping in one pass per thread, eliminating 2 of 3 kernel launch overheads

## Key Lessons

- Consolidate multiple trivial kernels operating on the same data indices into a single kernel to eliminate redundant launch overhead — applies when per-kernel GPU work is small and launch cost dominates total execution time
- When kernels share intermediate results (e.g., next_input_pos computed by update_lens_pos and consumed by update_slot_mapping), fusion enables passing values through registers instead of global memory round-trips, reducing both latency and memory traffic
