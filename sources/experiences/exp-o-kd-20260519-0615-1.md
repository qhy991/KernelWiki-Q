## Solution

Fuse renormalization into the moeTopK kernel by accumulating the row sum as each top-K weight is written, then performing the inverse-multiply normalization in-place at the end of the same kernel, eliminating the separate renormalize kernel launch and its global memory round-trip

## Key Lessons

- Fuse a lightweight post-processing pass (e.g., renormalization) into the producer kernel when the per-row data fits in a few registers — eliminates a separate kernel launch and avoids a global memory round-trip, with the re-read likely hitting L2 cache
- Accumulate running statistics (sum, max) during the main computation loop rather than computing them in a second pass — turns an O(n) re-read into an O(1) register-based accumulation
