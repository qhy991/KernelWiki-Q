## Solution

Restructure the loop to iterate over num_heads*embed_dim (half the count) and have each thread directly compute both rotated outputs (x and y) without any conditional, halving total iterations and eliminating warp-divergent branching while also removing __ldg from now-mutable query pointers

## Key Lessons

- Replace conditional single-output loops with branch-free dual-output loops when the conditional selects between two symmetric computations (e.g., rotation x/y halves) — halves iteration count and eliminates warp divergence, which serializes execution within a warp
- Remove __ldg from pointers that may be written to (in-place modification), since __ldg directs the load through the read-only texture cache which is incorrect for mutable memory and may produce undefined behavior
- When a loop index maps to two symmetric halves via modular arithmetic and a boolean condition, restructure to iterate over the base dimension (embed_dim) and explicitly compute both outputs per iteration — applies to any paired-element transformation like rotary embeddings, complex multiplications, or interleaved layouts
