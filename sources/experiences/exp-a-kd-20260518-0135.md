## Challenge

The before code computes logsumexp as mi + logf(s_prime) where mi = max * kLog2e is in log2-scaled space. The after code correctly divides mi by kLog2e before adding to logf(s_prime), converting back to nat


## Solution

Divide mi by kLog2e before adding to logf(s_prime) to convert from log2-scaled space back to natural log space, fixing the numerical correctness of logsumexp

## Results

- **speedup**: 0.996x (neutral)


## Key Lessons

- When computing logsumexp in log2 space (using exp2f with kLog2e scaling), the max term mi = mx * kLog2e must be divided by kLog2e before combining with logf(s_prime) to return to natural log space — omitting this conversion inflates results by factor kLog2e ≈ 1.44
- A single FLOP correctness fix (one division) adds negligible overhead compared to the dominant cost of 128 exp2f calls per thread, so correctness fixes in log-space arithmetic rarely impact performance
- Always track the logarithmic base convention throughout a kernel: mixing log2-scaled intermediates with natural-log operations is a common source of silent numerical errors in attention kernels
