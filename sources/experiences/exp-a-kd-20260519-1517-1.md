## Challenge

Each thread independently performs O(E) serial linear scan through a prefix-sum array to find which expert a token belongs to, issuing up to 64 conditional loads and branches per token with no inter-thread cooperation


## Solution

Replace per-thread O(E) linear scan with warp-cooperative search: all 32 lanes simultaneously probe one offset entry each via coalesced loads, vote with __ballot_sync on whether offset[i]<=token_id, and use __clz on the ballot mask to locate the last qualifying entry — reducing search from O(E) serial iterations to O(E/32) warp iterations

## Key Lessons

- Use __ballot_sync + __clz to turn O(N) serial search into O(N/32) warp-cooperative search — each lane probes one element, ballot voting aggregates results in one instruction, and __clz finds the highest qualifying index without branching — applies whenever threads search a sorted array for a boundary predicate
- Prefer warp-cooperative search over per-thread binary search for small-to-medium arrays (32–256 elements) — the 32-wide coalesced memory access and single-instruction ballot reduce both memory latency and branch overhead compared to independent serial or binary searches
- Pass lane_id as the starting index into the search array so each warp lane reads a naturally coalesced consecutive element (lane 0 reads arr[0], lane 1 reads arr[1], etc.), maximizing memory throughput during the probe phase
