## Solution

Replace scalar stride loop with vectorize_with_alignment<4> that computes pointer alignment at runtime, processes a small scalar prefix to reach alignment boundary, then performs 128-bit wide vec_n_t<float,4> loads/stores in the main body, followed by a scalar tail for remainder elements — reducing total memory transactions and improving L2 cache line utilization

## Key Lessons

- Use vec_n_t<float,4> (128-bit) vectorized loads/stores with alignment-aware prefix/tail handling instead of scalar loops for memory-bound elementwise kernels — reduces memory transaction count by 4x and improves L2 cache line utilization
- Compute pointer misalignment at runtime via uintptr_t masking (addr & (WIDTH-1)) and process a small scalar prefix before entering the vectorized main loop — ensures correctness for any input alignment without wasting bandwidth
- Separate scalar_op and vec_op lambdas allow the same elementwise logic to be shared between the scalar prefix/tail and the vectorized body, keeping the code DRY while achieving maximum memory throughput in the main loop
