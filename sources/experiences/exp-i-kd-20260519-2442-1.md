## Solution

Use cutlass::epilogue::thread::LinearCombinationPlanarComplex<half_t, 4, half_t, half_t> functor initialized with params, load accum and source via reinterpret_cast to ArrayPlanarComplex<half_t, 4> for vectorized 128-bit memory transactions, apply functor in a #pragma unroll 1 loop, and store result with the same vectorized cast

## Key Lessons

- Use reinterpret_cast with ArrayPlanarComplex<Element, N> for vectorized load/store of planar complex data — enables single-instruction 128-bit memory transactions (4 × half = 8 bytes) critical for epilogue throughput
- LinearCombinationPlanarComplex encapsulates alpha/beta scaling and planar complex accumulation in a single functor call — prefer CUTLASS functor composition over manual element-wise arithmetic for correctness, type safety, and hardware-specific optimization
- #pragma unroll 1 prevents the compiler from unrolling the epilogue loop, which is appropriate when N is typically 1 and unrolling would add unnecessary register pressure without benefit
