## Solution

Reinterpret scalar __half pointers as aligned _f16Vec<__half,8> structs (128-bit / 16-byte per load) so each iteration processes 8 FP16 values via a single vectorized load, uses packed __half2 addition for the residual add, and __half22float2 for variance accumulation — reducing iteration count from 2048 to 256 and cutting global memory transactions by 8x

## Key Lessons

- Replace scalar __half memory access with 128-bit aligned vector loads (_f16Vec<__half,8>) to coalesce 8 FP16 values into a single 16-byte transaction — reduces global memory traffic by the vector width factor and is applicable whenever FP16 arrays are accessed sequentially
- Use packed __half2 arithmetic (native += and *= on __half2) inside vectorized structs to double arithmetic throughput per instruction compared to scalar float conversion — works when adjacent FP16 elements can be processed independently
- Reduce block size (1024→256) when vectorization cuts iteration count proportionally — fewer threads with larger per-thread work improves SM occupancy and reduces scheduling overhead for memory-bound kernels
