## Challenge

Fused add RMS norm kernel requires contiguous input, forcing a separate copy_strided_half_to_contiguous_kernel launch before the main kernel to convert stride-512 rows into contiguous layout, adding an entire kernel launch and 64KB of redundant global memory read+write traffic


## Solution

Add input_stride parameter to the kernel, compute vec_input_stride = input_stride/width, and use strided_id = blockIdx.x * vec_input_stride + idx for all input vector loads and stores, eliminating the separate copy kernel entirely while preserving 128-bit vectorized memory access within each row's contiguous cache-line segments

## Key Lessons

- Prefer passing a stride parameter to vectorized kernels and computing vec_stride = stride/width over materializing a contiguous copy — eliminates an entire kernel launch and its associated global memory traffic, especially when the strided rows still contain contiguous cache-line segments wide enough for full vector loads
- When input rows have stride > hidden_size but the valid columns within each row are contiguous, vectorized loads at the scaled-down vec_stride offset still hit contiguous cache-line segments, preserving memory throughput without requiring physical contiguity
