## Challenge

MoE sum kernel uses regular global-memory loads (input[...]) for accumulating across the topk dimension, competing for L1 data cache lines with other memory operations in the same kernel


## Solution

Use __ldg(&input[...]) to route each read-only global load through the dedicated GPU read-only texture cache (L1 read-only path on sm_80), separating read-only accumulation traffic from the L1 data cache

## Key Lessons

- Use __ldg() for read-only global-memory loads to leverage the dedicated read-only texture cache path (separate from L1 data cache) — applies whenever a kernel reads but never writes to a buffer, especially with large strides between accessed elements
