## Solution

Load k_rope once per warp into a register buffer (k_rope_buf as int, covering 2 bf16 values per lane) and reuse it across all HEAD_CHUNK_SIZE=16 heads in the unrolled store loop, eliminating 15 redundant global memory reads per warp-chunk

## Key Lessons

- When a source tensor lacks a dimension present in the output (e.g., k_rope has no head dimension but k has 128 heads), load it once into a register and reuse across the missing dimension's loop — eliminates N-1 redundant global reads per work unit
- Register buffering combined with warp-cooperative partitioning maximizes reuse: one warp owns a token×head-chunk, so shared data (like k_rope per token) stays in registers for the full chunk iteration
