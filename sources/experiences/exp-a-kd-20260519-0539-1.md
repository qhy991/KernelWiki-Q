## Solution

Enable WARP_SWIZZLE (template param true) so that odd i-rows reverse their j iteration direction, alternating the MMA access pattern to reduce shared memory bank conflicts across adjacent warps in the same thread block

## Key Lessons

- Warp swizzle (alternating MMA iteration direction per row) can regress performance when the HMMA16816 pipeline is already well-optimized by hardware and ldmatrix bank-conflict handling is already efficient — the added branch and disrupted compiler instruction scheduling outweigh any bank-conflict reduction at larger tile sizes
- The impact of warp swizzle is problem-size dependent: at larger K dimensions the penalty shrinks because more K-tile iterations amortize the per-iteration branch overhead, but the optimization still does not yield a net gain over the baseline fixed-order pattern
