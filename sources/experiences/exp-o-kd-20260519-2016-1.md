## Challenge

Flat grid-stride loop with per-element index decomposition (division/modulo on flat tid) causes non-coalesced scalar memory access when the source tensor has transposed strides (nope_s1=32768 bytes), as adjacent threads in a warp access different (token, head) pairs with widely separated addresses


## Solution

Map each warp to one (token, head) pair via flat_warp_id = tid >> 5, compute the non-contiguous base pointer once per warp (token_id*nope_s0 + head_id*nope_s1), so all 32 lanes access contiguous stride-1 data from that base and the transposed-stride penalty is amortized from per-element to per-warp

## Key Lessons

- Map warps to logical work items (token,head) instead of using flat grid-stride loops when source tensors have non-unit inner strides — computing the non-contiguous base pointer once per warp amortizes the stride penalty across all 32 lanes, enabling fully coalesced contiguous access within each work item's data
- Prefer bitwise operations (>> 5 for divide-by-32, & 31 for modulo-32) over integer division/modulo for warp-level index decomposition — they compile to single instructions and make the warp-centric intent explicit
