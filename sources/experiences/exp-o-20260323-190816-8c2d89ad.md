fp32_rms_norm_llama3_8b_hs4096

## Solution

Shared memory weight cache: load 16KB weight vector into __shared__ float4 smem_w[1024] once per block, reuse for both sum-sq accumulation and output scaling passes. Applied to all 3 thread-count variants (128/256/512).

## Key Lessons

- Shared memory weight cache (16KB) improves RMS norm performance from 2.175x to 2.323x baseline
- Multi-row kernels with 128 threads caused severe regression - avoid unless carefully tuned
- The 16KB weight vector for hs=4096 fits perfectly in shared memory (1024 float4)
- Always test v1 (baseline transplant) before applying optimizations to avoid regression baseline confusion
- The 3-variant dispatch (128/256/512 threads by batch size) is a good pattern for RMS norm
