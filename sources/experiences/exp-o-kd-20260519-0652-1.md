## Solution

Pre-compute the integer division once during the shared memory loading phase into a separate shared memory array (sh_rd_block_sorted_ids), then replace the per-lookup division with a simple shared memory read (~5 cycles) in the hot loop, trading a small amount of extra shared memory for a significant reduction in arithmetic throughput pressure

## Key Lessons

- Pre-compute invariant integer divisions into shared memory once during the loading phase rather than recomputing them inside hot loops — GPU integer division takes ~20 cycles vs ~5 cycles for a bank-conflict-free shared memory read, yielding outsized speedups when the loop iterates many times
- When an expensive arithmetic result depends only on shared memory data that does not change across iterations, hoist the computation out of the loop and store it in a companion shared memory array, accepting a modest shared memory footprint increase for reduced arithmetic throughput pressure
