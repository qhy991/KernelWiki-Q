## Solution

Apply float4 (128-bit) vectorized loads and stores so each thread processes 4 elements, reducing thread count by 4x and cutting per-element instruction overhead, with a fallback scalar loop for the tail when N is not a multiple of 4

## Key Lessons

- Float4 vectorization of elementwise kernels reduces instruction count but yields diminishing returns when the scalar version already achieves effective memory coalescing — consecutive threads accessing consecutive 32-bit addresses are automatically coalesced into wide transactions on sm_80+ hardware
- For memory-bound kernels where scalar coalescing is already near-optimal, vectorization gains are marginal (~3%) and may not justify the added code complexity of the tail-case fallback loop
