## Challenge

Per-block reduction uses atomicAdd to a shared global-memory address, serializing 512 threads on a single atomic location per block, and additionally requires a cudaMemset to zero the buffer before each invocation plus a subsequent global-memory read by thread 0 to retrieve the result


## Solution

Replace atomicAdd-to-global-memory with CUB BlockReduce, which performs the sum cooperatively in shared memory using a hierarchical warp-level reduction tree, eliminating atomic contention, the per-invocation cudaMemset buffer reset, and the extra global-memory read

## Key Lessons

- Replace per-block atomicAdd-to-global with CUB BlockReduce when each block reduces to a single scalar — BlockReduce uses shared-memory warp-level reduction trees that avoid atomic serialization and eliminate the need for a separate global buffer and its memset reset
- CUB BlockReduce requires only a small shared-memory TempStorage allocation sized to the block, and its Sum() method handles the full reduction hierarchy (warp shuffle → inter-warp → cross-warp) in a single call, producing cleaner and more maintainable code than hand-rolled atomic patterns
- When all threads in a block contribute to one aggregate value, cooperative shared-memory reduction is strictly superior to atomic accumulation on global memory due to eliminated contention, fewer global-memory transactions, and no host-side buffer management overhead
