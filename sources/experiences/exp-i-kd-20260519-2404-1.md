## Solution

Build the cute tensor hierarchy bottom-up: global tensors via make_tensor(make_gmem_ptr(...), select<...>(shape_MNK), stride) to extract M-K, N-K, M-N subshapes; CTA tiling via local_tile with Step<_1,X,_1> for A (M,K), Step<X,_1,_1> for B (N,K), Step<_1,_1,X> for C (M,N); shared memory sized by cosize_v<Layout>; per-thread copy partitions via TiledCopy.get_slice().partition_S/D with make_fragment_like for registers; MMA partitions via TiledMMA.get_slice().partition_A/B/C with make_fragment_A/B/C

## Key Lessons

- Use select<I,J>(shape_MNK) to extract the correct 2D subshape for each matrix: select<0,2> for A (M,K), select<1,2> for B (N,K), select<0,1> for C (M,N) — the index order encodes which dimensions of the 3D problem shape each matrix spans
- local_tile Step patterns control which dimensions the CTA tile covers: Step<_1,X,_1> tiles M and K but not N (for A), Step<X,_1,_1> tiles N and K but not M (for B), Step<_1,_1,X> tiles M and N but not K (for C output)
- Use separate register fragments for copy (make_fragment_like on TiledCopy partition) vs MMA (make_fragment_A/B/C on TiledMMA partition) — copy registers buffer gmem traffic while MMA registers hold shared-to-register tiles for the gemm operation
