## Challenge

Every A matrix row lookup in the sorted-token-id inner loop performs integer division by top_k (sorted_row = sh_block_sorted_ids[row] / top_k), costing ~20 instruction cycles per lookup on GPU where the division is repeated across all pipeline stages and rows


## Solution

Pre-compute sorted_id / top_k once during the block data load phase into a dedicated shared memory array (sh_rd_block_sorted_ids), replacing every per-lookup integer division with a simple ~5-cycle shared memory read in the hot inner loop

## Key Lessons

- Pre-compute integer divisions with a constant divisor into shared memory when the result is read repeatedly in a hot loop — GPU integer division costs ~20 cycles vs ~5 cycles for a bank-conflict-free shared memory read, and the one-time setup cost amortizes across all subsequent lookups
- Adding a small dedicated shared memory array for pre-divided values is worthwhile when the lookup count per loaded value is high (multiple pipeline stages × multiple rows), as the trade-off favors reduced arithmetic over increased shared memory footprint
