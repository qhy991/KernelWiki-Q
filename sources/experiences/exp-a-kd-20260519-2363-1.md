## Solution

Clamp kOffcp to K - A_CHUNK using min__(K - A_CHUNK_VAL, k_str + kOff), so out-of-bounds threads read the last valid chunk (positions 92-99) as deterministic padding within the same row instead of wrapping into the next row, eliminating cross-row data corruption.

## Key Lessons

- When tiling the K-dimension with a fixed chunk size, threads whose offsets exceed K - chunk_size must be clamped to the last valid chunk — without clamping, the row-stride address formula wraps into the next matrix row, producing silent cross-row data corruption
- A correctness fix that adds one integer min comparison per thread has negligible performance impact (sub-1us difference is measurement noise) — the latency regression here is within noise at sub-10us kernel scale
- Duplicate the last valid chunk as padding rather than leaving OOB threads undefined — this produces deterministic results for all threads without requiring conditional branches inside the inner loop
