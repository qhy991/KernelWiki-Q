## Challenge

MoE block-alignment kernel launches with only num_experts (8) threads per block, leaving 24 of 32 warp lanes idle at 25% SIMD utilization, forcing each thread to sequentially process 2048 of 16384 tokens


## Solution

Increase block thread count from num_experts (8) to max(num_experts, WARP_SIZE=32) to achieve full warp utilization, expand shared memory layout from (num_experts+1)*num_experts to (blockDim.x+1)*num_experts entries, and add if(threadIdx.x < num_experts) guards around expert-specific accumulation and expert_ids assignment to prevent out-of-bounds writes from extra threads

## Key Lessons

- Launch CUDA kernels with at least WARP_SIZE (32) threads even when fewer logical entities (e.g., num_experts=8) exist — a sub-warp block wastes SIMD lanes and forces disproportionately long per-thread serial loops, costing 2x+ performance
- When scaling thread count beyond the logical entity count (num_experts < blockDim.x), guard expert-specific shared-memory writes and expert_ids assignment with if(threadIdx.x < num_experts) to prevent out-of-bounds indexing from helper threads
- Size shared memory dynamically using blockDim.x rather than a compile-time constant like num_experts — this keeps the layout correct when the launch thread count changes at runtime
