## Solution

Replace eight scalar half load/store assignments with two 128-bit packed transactions using the LDST128BITS macro (reinterpret_cast<float4*>), reducing memory instruction count from 16 to 2 for the same 8-element workload and shifting thread addressing from threadIdx.x*8 to threadIdx.x with 8*tx stride for 128-bit alignment

## Key Lessons

- Use 128-bit packed load/store (LDST128BITS / reinterpret_cast<float4*>) instead of scalar element-wise copies when processing contiguous half elements — a single 16-byte wide transaction replaces eight 2-byte transactions, reducing memory instruction count by 8x
- When vectorizing memory access, adjust thread addressing so each thread strides by the vector width (threadIdx.x with 8*tx stride) rather than pre-multiplying by element count (threadIdx.x * 8) — this ensures correct 128-bit alignment
