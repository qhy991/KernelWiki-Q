## Solution

Change scale parameters from scalar float (by-value) to float pointers, allowing the kernel to dereference device-memory scale values via *k_scale/*v_scale — this enables dynamic per-invocation scaling computed on the GPU without host round-trips, at the cost of an extra global memory load per thread and additional host-side cudaMalloc+cudaMemcpy setup

## Key Lessons

- Changing scalar kernel parameters to device pointers enables dynamic per-invocation scaling computed on-GPU, but introduces extra global memory loads and host-side allocation overhead — favor pointers when scales vary per-invocation or are computed on-device, favor scalars when scales are static constants
- Microbenchmarks with undersized workloads (1 thread block) are dominated by kernel launch overhead and cudaEvent timing granularity, making apparent regressions unreliable — use sufficiently large batch sizes to isolate kernel-level parameter-passing effects from measurement noise
