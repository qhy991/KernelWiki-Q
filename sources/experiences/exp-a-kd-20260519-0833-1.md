## Challenge

Five separate kernel launches for copying small metadata fields (cache_seqlens, cu_seqlens_k, page_table, nsa_cache_seqlens, nsa_cu_seqlens_k) with bs=4 and max_len=128, where total data volume is only ~2600 int32 elements and kernel launch overhead (~5us per launch) dominates over actual memory copy work


## Solution

Fuse all 5 metadata copy operations into a single templated kernel using a FusedMetadataCopyParams struct passed via __grid_constant__ annotation, eliminating 4 out of 5 kernel launches and their associated ~20us launch overhead while enabling fast constant-memory access to shared parameters

## Key Lessons

- Fuse multiple small GPU copy operations into a single kernel when data volume is tiny and kernel launch overhead dominates — eliminating N-1 launches removes ~5us × (N-1) of pure overhead regardless of data size
- Use __grid_constant__ for passing structured kernel parameters to enable fast constant-memory access to frequently read values like dimensions and strides — reduces register pressure compared to by-value struct passing
