KernelBench level_1 task_9: Tall-and-skinny GEMM C = A @ B where A=(16384,16), B=(16,16384), C=(16384,16384). K=16 is very small.

## Key Lessons

- For tall-and-skinny GEMM with very small K (e.g., K=16), shared memory tiling is ineffective because threads in same block don't share data
- Vectorized loads (float4) help but provide limited speedup when K is small
- Thread-per-tile (2x2) reduces kernel launch overhead and provides modest speedup
- Problem characteristics (K=16) fundamentally limit optimization potential - memory bandwidth bound
- Always verify data sharing patterns before applying shared memory optimizations
