## Solution

Implement the complete CUTE tensor pipeline: construct global and shared memory tensors using make_tensor with gmem/smem pointers and the provided layout, partition source and destination tensors via TiledCopy thread slices, allocate a register tensor matching the partitioned shape, then execute the three-stage copy pipeline (copy gmem→rmem, copy tiled rmem→smem for STSM, strided loop smem→gmem)

## Key Lessons

- CUTE's partition_S / partition_D naming convention: partition_S is for source tensors (read side) and partition_D is for destination tensors (write side) — using them correctly ensures proper thread-to-data mapping for tiled copy operations
- Allocate register tensors via make_tensor<T>(shape(partitioned_source)) to create a register-backed tensor matching the partitioned source shape, providing the necessary intermediate buffer for two-stage gmem→rmem→smem copies
- The copy(tiled_copy, src, dst) overload dispatches hardware-optimized STSM/STS instructions via the TiledCopy object, while copy(src, dst) without a TiledCopy performs a plain element-wise register copy — choose the overload based on whether the copy crosses memory hierarchy levels
