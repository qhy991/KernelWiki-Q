## Solution

Replace scalar stride loads with float4 vectorized loads via reinterpret_cast<float4*>, reducing 12 scalar transactions per lane to 3 coalesced 128-bit loads; each lane loads VPT/VEC_SIZE=3 float4 chunks and iterates over the 4 elements within each vector with #pragma unroll

## Key Lessons

- Replace scalar stride loops with float4 vectorized loads via reinterpret_cast when iterating over contiguous arrays with fixed element count — reduces memory transactions by 4x and improves coalescing when each lane processes VPT elements and VPT is divisible by 4
- Compute vec_idx = lane_id * VEC_PER_LANE + i rather than a global stride loop to achieve naturally coalesced 128-bit loads where adjacent lanes read adjacent memory chunks
- Use #pragma unroll on inner loops over float4 components to eliminate loop overhead and allow the compiler to schedule the 4 element-wise operations with independent sigmoid/bias arithmetic
