## Solution

Add shared memory caching to read input from global memory once into shared memory (computing absmax during the copy), then quantize from shared memory — however this results in a slight regression because the 1MB working set fits entirely in the A800's 40MB L2 cache, making the double-read pattern a cache-hit sequence with minimal additional latency

## Key Lessons

- Shared memory caching to eliminate redundant global reads only helps when the working set exceeds L2 cache capacity — for small tensors (1MB) that fit entirely in L2 (40MB on A800), the 'double read' hits L2 cache with minimal latency, and the added smem copy + __syncthreads() barrier overhead causes a net regression
- L2 cache residency must be verified before applying shared memory caching optimizations — if the input working set is well within L2 capacity, the hardware cache already provides near-zero-cost redundant access, making explicit software caching counterproductive
- __syncthreads() barriers are not free and add measurable overhead when the optimization they enable provides no real benefit — always verify the assumed bottleneck (DRAM bandwidth) actually exists before adding synchronization
