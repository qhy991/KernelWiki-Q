## Solution

Replace hardcoded VEC_SIZE with a dynamic computation using gcd(max_vector_bytes / sizeof(scalar_t), hidden_size) which guarantees the vector size evenly divides hidden_size, and derive block_size directly from the computed vector size instead of a manual power-of-2 loop

## Key Lessons

- Compute vectorized access width via gcd(natural_vector_width, hidden_size) rather than hardcoding it — guarantees divisibility for all hidden sizes and selects the largest safe vector width automatically
- For hidden_size=3072 with fp16, gcd(8, 3072)=8 matches the hardcoded value, so performance is neutral; the real benefit is correctness portability across model configurations with different hidden sizes
