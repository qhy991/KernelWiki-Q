## Solution

Vectorize to float4 (kAlignment=4) so each thread processes 4 logits per iteration via 128-bit loads/stores, adding a fast path that writes packed -inf (float4) when all 4 bits in a group are masked, and a partial-mask path that loads, selectively overwrites, and writes back

## Key Lessons

- Vectorizing a bitmask kernel from scalar to float4 is neutral when the bitmask is sparse/random (~50% set): the scalar kernel skips ~50% of positions via early-exit (m==0), while the vectorized kernel only skips ~6.25% of groups, forcing ~87.5% of groups through the expensive partial-mask path (load + per-element check + write-back) that generates more total memory traffic than the scalar skips save
- Vectorized access is most beneficial when the mask is dense (most/all bits set per group) so the fast packed-write path (m==kPackedMask) dominates — with random sparse masks the partial-mask overhead negates the coalescing gain
