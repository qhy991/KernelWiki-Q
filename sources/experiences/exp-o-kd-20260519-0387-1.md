## Solution

Vectorize to half2 (4-byte) loads/stores so each thread processes two half elements in one transaction via reinterpret_cast<half2*>, doubling per-thread memory throughput and halving thread count to N/2 for reduced scheduling overhead

## Key Lessons

- Vectorize scalar half element-wise kernels to half2 loads/stores via reinterpret_cast<half2*> to pack two 2-byte elements into one 4-byte transaction — doubles memory throughput and reduces thread scheduling overhead by half, yielding measurable speedup on large arrays where 2-byte accesses underutilize bandwidth
- Scale thread index by 2 (idx = 2 * linear_id) when vectorizing from scalar to half2 so that N/2 threads cover the same range, avoiding out-of-bounds access while halving launch overhead
