## Solution

Remap output addressing to coalesced layout: g_mindx = m*4 + (tid%64) groups 4 adjacent rows contiguously (64-element wide base), and g_adr = g_mindx + M*g_nindx*4 groups 4 adjacent N-columns so adjacent threads target the same cache line for each g_nindx value

## Key Lessons

- Remapping atomic operation addresses from scattered (M-strided) to coalesced (grouped-4 contiguous) layout aligns thread writes to the same cache line — improves coalescing on hardware with expensive non-coalesced atomics, but shows neutral effect when the output buffer fits entirely in L2 cache (16KB here)
- Address coalescing for atomics requires a data layout permutation: both the row index (g_mindx = m*4 + tid%64) and column stride (M*g_nindx*4) must be remapped together so adjacent threads map to adjacent memory locations within the same cache line
- Coalescing optimizations that target global memory atomics are most impactful on larger output buffers that exceed L2 cache capacity, or on architectures (like AMD CDNA) where non-coalesced atomic operations have higher overhead than NVIDIA GPUs
