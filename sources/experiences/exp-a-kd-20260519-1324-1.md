## Solution

Replace the conditional thread_max_vec call with vectorize_read_with_alignment, a generalized abstraction that unconditionally handles both aligned and misaligned addresses via runtime address checks and functor+lambda dispatch — however, the abstraction introduces residual instructions for alignment checks, address computation, and lambda dispatch that nvcc cannot fully eliminate even with --extended-lambda and -O3, causing 18.86% regression in the aligned case

## Key Lessons

- Generic vectorized-read abstractions with runtime alignment checks and lambda/functor dispatch add instructions that nvcc may not fully eliminate even with --extended-lambda and -O3 — when all inputs are known-aligned, prefer direct reinterpret_cast vectorized loops to avoid abstraction overhead
- Per-thread loop body overhead from abstraction layers accumulates proportionally with the number of vector packs each thread processes — the regression is most pronounced when each thread handles many packs (256 threads × 16 packs for 4096 elements) because the extra instructions execute per iteration
