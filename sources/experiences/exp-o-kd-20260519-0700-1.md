## Solution

Replace scalar int32 loads with int4 vector loads (128-bit transactions fetching 4 int32 values per access), reducing total memory transactions by 4x in the aligned path while preserving correctness with an alignment check and scalar fallback for remainder elements

## Key Lessons

- Use int4 vectorized loads (128-bit transactions) to fetch 4 int32 elements per memory transaction in memory-bound loops — reduces transaction count by 4x and improves bandwidth utilization when the pointer is 16-byte aligned
- Always include an alignment check (reinterpret_cast<uintptr_t>(ptr) % 16 == 0) and scalar fallback when using vectorized loads — ensures correctness on unaligned inputs while maximizing throughput on the aligned path
- Add a remainder loop (vec_length * 4 + threadIdx.x to topk_length) after vectorized processing to handle tail elements when the array length is not a multiple of the vector width
