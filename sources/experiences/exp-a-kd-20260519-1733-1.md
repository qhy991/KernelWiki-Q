## Solution

Replace modulo-2 double-buffered peer_counter[2] arrays and multi-thread read-modify-write on self_counter with separate start/end arrays plus a compact _flag[kMaxBlocks] array; all threads read _flag (single load) but only thread 0 writes it back after __syncthreads(), eliminating multi-writer cache-line contention and reducing global store traffic from O(ngpus) RMW operations to one conditional write

## Key Lessons

- Replace multi-thread read-modify-write on shared cache-line counters with a single-reader/single-writer flag pattern (all threads read, only thread 0 writes after __syncthreads) → eliminates L2 write-back traffic from false sharing across threads hitting the same 128-byte-aligned cache line → applies when barrier state is per-block and only needs to be consistent after synchronization
- Use structurally separate arrays (start/end) instead of modulo-indexed double-buffering (arr[2][...]) to distinguish barrier phases → removes modulo arithmetic overhead and halves the memory footprint of the signaling arrays → applies when two-phase alternation can be encoded in the array selection rather than a runtime index
- Consolidate per-barrier counter writes to a single thread via __syncthreads() + conditional write → ensures exactly one store to _flag per barrier invocation with no atomicity concerns → applies when post-synchronization visibility is sufficient and only one thread needs to persist the barrier state
