## Solution

Enlarge the tile to ThreadblockShape<128,128,64> and WarpShape<64,64,64>, computing 16384 output elements per threadblock with only 16384 int8 global loads, raising arithmetic intensity to ~1:1 and eliminating the warp-level tensor core underutilization caused by only 16 MMA rows per warp in the small-tile configuration

## Key Lessons

- Small Cutlass tile shapes (e.g. ThreadblockShape<32,64,128>) create many tiny threadblocks with low arithmetic intensity on large-M GEMM problems — prefer large tiles (128x128) that keep SM resources saturated and reduce global memory round-trips per compute operation
- WarpShape must be sized to fully utilize tensor core MMA instructions; WarpShape<16,64,64> leaves tensor cores underutilized because only 16 MMA rows per warp provides insufficient work, while WarpShape<64,64,64> spreads 64 rows across the warp for full utilization
- Arithmetic intensity (outputs produced / global bytes fetched) is the key metric for tile sizing — a ratio near 1:1 (16384 outputs / 16384 loads) dramatically outperforms ~0.17:1 (2048 outputs / 12288 loads)
