## Solution

Replace scalar bf16 loads/stores with 128-bit uint4 vectorized loads/stores via a Pack16B union (8 bf16 values per transaction, 2 packs covering 16 elements per topk iteration), and route input reads through the read-only cache with __ldg, reducing global memory transactions by ~8x

## Key Lessons

- Use uint4 (128-bit) vectorized loads/stores via a union overlay (e.g., Pack16B mapping uint4 to 8×bf16_raw) to process 8 elements per memory transaction — reduces transaction count by 8x and fully saturates the memory bus on aligned accesses, critical for memory-bound kernels where computation is trivial
- Route input reads through the read-only data cache with __ldg when the data is consumed once (no write-then-read), improving L1 utilization and freeing the standard cache path for other data
- Chunk the hidden_dim loop by vector width (hidden_dim / VEC) instead of iterating element-by-element, enabling aligned 128-bit accesses and reducing loop overhead by the same factor
