## Solution

Convert compile-time pipe_stages=4 constant into a runtime stages parameter (set to 2 for Turing), halving all pipeline-buffer shared memory (sh_a_size, sh_b_size, sh_s_size, sh_g_idx_size) while retaining one-stage-ahead prefetching for latency hiding

## Key Lessons

- Converting compile-time pipeline stage constants to runtime parameters enables per-GPU tuning of shared memory usage — stages=4 works on A100 (164KB shared) but stages=2 is needed on Turing (64KB), and the 2-stage pipeline still provides one-stage-ahead prefetching for latency hiding
- Pipeline stage count scales shared memory linearly (sh_a, sh_b, sh_s, sh_g_idx all multiply by stages) — reducing stages from 4 to 2 cuts total shared memory by ~50%, which can be the difference between a thread config being valid or rejected on memory-constrained GPUs
