## Solution

Increase BLOCK_SIZE from 16 to 32 so that THREAD_GROUP_SIZE becomes 1, eliminating the per-QK shuffle reduction entirely since each thread independently computes one full QK dot product; this also halves the number of KV blocks for the same sequence length and naturally doubles VEC_SIZE to 8 (uint4 loads) for wider memory throughput

## Key Lessons

- Setting BLOCK_SIZE equal to WARP_SIZE (32) makes THREAD_GROUP_SIZE=1, eliminating intra-group __shfl_xor_sync reduction in QK dot products — each thread processes one token independently, saving log2(THREAD_GROUP_SIZE) shuffle instructions per QK computation
- Larger BLOCK_SIZE halves the number of KV blocks for the same sequence length, reducing block-table indirection overhead and outer loop iteration count — particularly beneficial for long sequences where the QK loop dominates
- THREAD_GROUP_SIZE=1 naturally doubles VEC_SIZE (4→8) and widens QK loads from uint2 to uint4, improving memory throughput for both Q and K load paths without any code restructuring
