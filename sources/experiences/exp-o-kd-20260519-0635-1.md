## Solution

Restructure device-side layout into page-padded format: group 64 items' value data contiguously at 576-byte stride (576 = 72×8, cache-line aligned) within each page followed by a separate contiguous scale region, using bitwise page decomposition (index >> 6 and index & 63) to replace multiply-based addressing

## Key Lessons

- Use page-padded memory layout with stride that is a multiple of 8 (cache-line aligned) instead of interleaved value+scale stride — reduces cache-line straddling for warp-cooperative 8-byte stores when transferring mixed-size fields like KV-cache items
- Replace multiply-based linear addressing (index * stride) with bitwise page decomposition (index >> bits for page number, index & mask for page offset) for marginally faster address computation in warp-cooperative transfer kernels
