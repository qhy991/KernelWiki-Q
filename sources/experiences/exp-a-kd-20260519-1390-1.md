## Solution

Fuse bias addition into the GEMM kernel epilogue so D = scaleA*(scaleB*(A*B)) + bias is computed in a single kernel launch, eliminating the extra kernel launch and the output tensor global memory round-trip

## Key Lessons

- Fusing a lightweight epilogue operation into a compute-bound kernel eliminates one kernel launch and an output tensor memory round-trip — applies whenever the epilogue adds negligible FLOPs relative to the main computation
- For large untiled GEMM workloads where compute dominates latency, the extra memory round-trip of a separate bias-add pass is negligible relative to GEMM time — kernel fusion yields measurable speedup primarily when the auxiliary kernel's memory traffic is significant relative to the main kernel's execution time
