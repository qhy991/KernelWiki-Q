## Solution

Replace single-pass 512-bin approach with progressive multi-step refinement using 2048 bins per step (half-precision initial step, float-precision refinement), round-based prefix scan across bins/CASE_THREADS rounds with __syncthreads_or early exit to terminate scanning as soon as the threshold bin is found

## Key Lessons

- Progressive histogram refinement (coarse half-precision bins → fine float-precision bins across multiple steps) reduces total work for topk thresholding — the initial step quickly narrows candidates and subsequent steps refine within the threshold region, applying whenever bin granularity must scale beyond a single BlockScan capacity
- __syncthreads_or enables block-wide early exit from iterative prefix scan rounds — once any thread finds the threshold bin, all threads exit immediately, avoiding unnecessary scan work on remaining bins
- Using half-precision for the initial binning step provides sufficient coarse granularity at lower computational cost when subsequent float-precision refinement steps will correct any precision loss from the coarse approximation
