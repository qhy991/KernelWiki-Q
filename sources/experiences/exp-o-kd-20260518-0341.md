## Solution

Added a runtime maxK dispatch macro that buckets max(query_dim, value_dim) into compile-time constants {64, 128, inf} via DISPATCH_MAXK, threading kMaxK into AttentionBackwardKernel so the template selects smaller 64x64 tiles (4 warps) when head dimensions are 64 or less

## Key Lessons

- Dispatching runtime dimension values into compile-time buckets (e.g., ≤64, ≤128, unbounded) and threading them as template parameters lets the kernel select smaller tile sizes and fewer warps for small inputs — apply this whenever a templated kernel's tile/block configuration depends on head or feature dimensions
- Adding a max-dimension compile-time bound to attention backward kernels trims thread count, shared-memory footprint, and scheduling overhead for small head dimensions while preserving full-width paths for larger K — especially beneficial when short or low-K backward attention workloads are common
