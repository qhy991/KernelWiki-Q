## Solution

Replace scalar float loads with vec4_t aligned loads (4 floats per struct) to reduce the number of load instructions by 4x, adding manual unpacking of .x/.y/.z/.w fields and accumulating four partial sums per iteration

## Key Lessons

- Vectorized vec4_t loads provide no benefit for fp32 data when blockDim already equals hidden_size (1024 threads / 256 vec4 elements = 75% thread idle), negating the wider-load advantage — thread utilization must be considered alongside memory width
- For 4-byte (fp32) types, 32 consecutive threads in a warp already coalesce scalar loads into a full 128-byte memory transaction, so vectorized loads do not improve coalescing efficiency for fp32 — they primarily help sub-4-byte types like fp16 or int8
