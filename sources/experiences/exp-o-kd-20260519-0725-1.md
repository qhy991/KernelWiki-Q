## Solution

Fuse rope_only_kernel and kv_writeback_kernel into a single fused_rope_kv_kernel where post-RoPE K values (y1, y2) remain in registers after computation and are written directly to both k_rope and k_buffer, eliminating one global memory round-trip and one kernel launch

## Key Lessons

- Fuse producer-consumer kernel pairs when intermediate results fit in registers — keeping values in registers across logical stages eliminates both global memory round-trips and kernel launch overhead simultaneously
- Kernel launch overhead (argument setup, enqueue, stream synchronization) is a significant fraction of total latency for small-batch GPU workloads (nnz=8), making kernel fusion disproportionately beneficial when per-kernel work is small
