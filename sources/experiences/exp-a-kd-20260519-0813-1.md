## Solution

Replace __shared__ float scale with const float scale computed directly from each warp's own warp_max, making the scale variable register-local per warp and eliminating the cross-warp race condition entirely

## Key Lessons

- Never use __shared__ variables for per-warp data when multiple warps in the same CTA operate on independent tokens — each warp should hold its own scale in registers to avoid inter-warp races and eliminate unnecessary shared-memory traffic
- When each warp processes an independent work item (e.g., one token per warp), prefer register-local const variables over __shared__ memory for scalar reduction results — the warp shuffle already produces the final value per-warp, so shared memory adds overhead and introduces correctness hazards
- A shared-memory race condition may not manifest as incorrect results on specific hardware or schedules but is still a latent bug that can fail under different GPU architectures, occupancy levels, or scheduler decisions
