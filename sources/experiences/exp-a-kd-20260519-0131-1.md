## Solution

Fix the sign error in the causal boundary formula from key_start - num_keys - num_queries to key_start - num_keys + num_queries, which correctly positions the bottom-right causal diagonal so that a key at position key_start first attends to queries starting at the algebraically correct offset

## Key Lessons

- In causal attention boundary arithmetic, the correct CausalFromBottomRight formula is max(0, key_start - num_keys + num_queries) — the plus sign on num_queries ensures the causal diagonal aligns with the bottom-right offset of the attention matrix, not top-left
- Index boundary bugs in attention kernels often produce silently wrong results rather than crashes — always validate against known test vectors (e.g., key_start=64, num_keys=128, num_queries=128 should yield 64, not 0)
