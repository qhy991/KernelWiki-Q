## Solution

Add a compile-time SWAP_AB template parameter to conditionally swap M and N in problem_sizes (writing (2N, M, K) and (K, M, N) instead of (M, 2N, K) and (M, K, N)), enabling CUTLASS GEMM to use TMA-based epilogue for small-M MoE workloads on SM90 hardware

## Key Lessons

- Swapping M and N in CUTLASS GEMM problem sizes via a compile-time SWAP_AB template is a zero-overhead abstraction at the utility-kernel level (identical store count and access pattern) that enables TMA-based epilogue for small-M MoE workloads on SM90 — apply when num_experts * padded_m <= 64
- Use if constexpr for compile-time layout variants in problem-size setup kernels — both code paths emit the same number of global stores, so the branching cost is eliminated at compile time and the real performance gain appears downstream in the GEMM consumer
