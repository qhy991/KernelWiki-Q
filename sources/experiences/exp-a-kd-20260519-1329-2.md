## Challenge

The before kernel uses 20 floats of shared memory for cross-tile scale communication and two __syncthreads() barriers — one to collect per-warp max-abs values into shared memory and another to broadcast the computed tile scale back to all threads, adding ~100-200 cycles of warp stall per invocation


## Solution

Eliminate shared memory entirely by using half-warp shfl_xor reductions with width=16 so each half-warp independently computes its tile's max-abs and scale, then designated lanes write the scale directly to global memory without any __syncthreads() barriers

## Key Lessons

- Use half-warp shfl_xor with width=16 to confine reductions within 16-lane subgroups, enabling independent per-tile scale computation without shared memory or __syncthreads() barriers — each half-warp handles one tile's reduction independently
- Eliminating __syncthreads() saves ~100-200 cycles per barrier on modern GPUs; when per-tile reductions can be confined to warp subgroups via shuffle width parameter, shared memory and barriers become unnecessary
- When each thread processes multiple elements (8 bf16 via int4), the local max-abs over those elements can be computed with register-only fmaxf chains before entering the warp shuffle reduction, reducing the number of shuffles needed
