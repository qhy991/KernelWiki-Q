## Solution

Precompute the four increment values into a Params struct (inc_next[4]) on the host once, pass them to the kernel as constant arguments, and replace the if-else chain that selects among four local variables with a single array-index lookup using next_idx — eliminating all per-iteration offset recomputation

## Key Lessons

- nvcc -O3 performs loop-invariant code motion (LICM) effectively — when all operands in a per-iteration expression are loop-invariant constants or kernel arguments, the compiler hoists the computation out of the loop automatically, making manual precomputation redundant in this case
- Precomputing stride offsets into a lookup table (inc_next[]) is a valid structural pattern for CUTLASS-style pointer advance, but only yields measurable gains when the offset expressions cannot be proven loop-invariant by the compiler (e.g., when they depend on per-iteration state or indirect indexing)
