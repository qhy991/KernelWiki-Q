## Solution

Pre-compute all division results (sh_ids[i] / top_k) into a second shared memory array during a one-time initialization phase, then replace every hot-loop division with a shared memory read of the pre-computed value, trading 1600 integer divisions per thread for 8 divisions plus 1600 shared memory loads

## Key Lessons

- Hoist invariant integer divisions out of hot loops by pre-computing results into shared memory — the one-time division cost is amortized across all repetitions, and shared memory reads replace expensive integer divide instructions (~20 cycles each on CUDA) with ~1-cycle loads
- Pre-computation via shared memory doubles the shared memory footprint (8KB vs 4KB here), which may slightly increase bank conflict latency, but the division elimination benefit dominates for non-power-of-2 divisors that the compiler cannot optimize via bit-shift
- Division by non-power-of-2 constants (e.g., 7) cannot be reduced to shift instructions by the compiler, making pre-computation especially valuable compared to power-of-2 divisors where the hardware already handles them efficiently
