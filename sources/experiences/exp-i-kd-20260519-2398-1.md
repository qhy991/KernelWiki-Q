## Challenge

A generic GPU permutation kernel is needed that supports both batched and non-batched tensor reordering, requires efficient division-free coordinate decomposition from linear indices to 2D PitchLinear coordinates, and must accept an arbitrary functor for flexible output address mapping


## Solution

Implement a grid-stride loop where each thread strides by blockDim.x*gridDim.x to cover all elements; use cutlass::FastDivmod for O(1) coordinate decomposition of the linear index into (stride_coord, intra_stride_coord); apply the Permute functor to map PitchLinearCoord to output address, with batched mode offsetting only the input pointer (CUTLASS 2.x convention: target tensor has 0 batch stride)

## Key Lessons

- Use cutlass::FastDivmod(j, i, k) for O(1) linear-to-2D coordinate decomposition without integer division — the divisor field stores the stride and the operator() computes quotient and remainder in one step, applicable to any strided tensor indexing pattern
- Grid-stride loops (k += blockDim.x * gridDim.x) let a fixed grid cover arbitrarily large element counts while maintaining coalesced access — applicable when each element is processed independently and memory bandwidth is the bottleneck
- Template the batch dimension offset with a bool parameter and use ternary dispatch (Batched ? 0 : offset) to let the compiler eliminate dead branches at compile time — avoids runtime branching cost in batched permutation kernels
