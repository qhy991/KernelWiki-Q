## Solution

Implement cluster shape validation via a templated check_cluster_shape function, assert all parameters against compile-time expected values, use cute::block_id_in_cluster() to get the cluster-aware block ID, and write a sentinel float value to a global device pointer (scalar_ptr_gpu) from thread 0 of the origin block to confirm execution

## Key Lessons

- Use a global device pointer (scalar_ptr_gpu) as a side-effect channel for test kernels that need to confirm execution without polluting the kernel signature — applies to any test scenario where zero-argument or fixed-signature kernels must report back to host
- Use cute::block_id_in_cluster() instead of blockIdx for cluster-aware kernels — it returns the block's position within its cluster (not the grid-level block index), which is the correct identifier for per-cluster logic
- Guard test side-effects with a compound condition (threadIdx + block_id origin check) to ensure exactly one thread across the entire cluster writes the sentinel value, avoiding race conditions in multi-block cluster launches
