## Solution

Switch from kAnalytic/kStrided to kOptimized/kUnity iterator: the optimized iterator precomputes a delta table (inc_next[]) for pointer advancement across T,R,S filter dimensions and uses bitmask predicates (masks_[s][3]) for validity checks, converting runtime coordinate computation into simple pointer arithmetic and bitwise AND operations, while kUnity eliminates stride divisibility checks entirely

## Key Lessons

- Prefer kOptimized iterator over kAnalytic for CUTLASS convolutions when stride={1,1,1} — precomputed delta tables replace per-tile coordinate recomputation with a single pointer offset, and bitmask predicates replace multi-instruction coordinate validity arithmetic with a 3-way bitwise AND
- When all convolution strides are 1, use kUnity stride support instead of kStrided to eliminate runtime stride divisibility checks in the iterator hot path
- CUTLASS iterator algorithm selection can yield large speedups (2.5x here) even without changing tile sizes or instruction shapes, because iterator overhead in the address computation path dominates when filter dimensions are small and tiles are visited frequently
