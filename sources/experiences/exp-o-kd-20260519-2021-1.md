## Challenge

MoE sum kernel passes TOPK as a runtime integer parameter, preventing the compiler from unrolling the inner loop and forcing loop-control overhead (branch, increment, compare) on each of the 2 iterations


## Solution

Convert TOPK from a runtime parameter to a compile-time template constant, enabling #pragma unroll to fully expand the 2-iteration inner loop into two independent load-add sequences with no loop overhead

## Key Lessons

- Convert small fixed iteration counts (e.g., TOPK=2) into compile-time template constants with #pragma unroll — eliminates loop overhead and exposes independent load-add sequences to the instruction scheduler, applies whenever the iteration count is known at compile time
