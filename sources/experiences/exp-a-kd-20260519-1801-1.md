## Challenge

Warp-level reduction is implemented as separate hardcoded template functions for each operation (warpReduceSum with +=, warpReduceMax with max()), requiring duplicate shuffle-xor loop logic for every new reduction operation


## Solution

Consolidate duplicate warp reduction templates into a single warpReduce that accepts a ReduceFnType function pointer (e.g. detail::_sum or detail::_max); NVCC inlines the call since the target is a compile-time constant, producing identical SASS while eliminating code duplication

## Key Lessons

- Passing a compile-time-constant function pointer to a __device__ function for dispatch (e.g. detail::_max vs detail::_sum) produces zero runtime overhead because NVCC inlines the call — safe to use for API generalization without performance regression
- Consolidating N nearly-identical template specializations (sum, max, min) into a single template parameterized by a ReduceFnType function pointer eliminates code duplication and maintenance burden while preserving identical SASS output
