## Challenge

MoE alignment kernel uses a single-block <<<1,1024>>> launch for token counting and serial cumulative sum computation by thread 0, serializing all work on one SM and requiring a second separate sort kernel, leaving other SMs idle during counting and cumsum phases


## Solution

Replace single-block launch with multi-block cooperative kernel via cudaLaunchCooperativeKernel, distributing token counting across multiple SMs and replacing serial cumsum with parallel prefix sum using cooperative_groups grid.sync() for cross-block synchronization, fusing counting, alignment, and sorting into one kernel

## Key Lessons

- For large-count workloads (32K+ tokens), replace single-block <<<1,1024>>> launches with multi-block cudaLaunchCooperativeKernel to distribute work across all available SMs — applies when the algorithm can be restructured to use cooperative_groups grid.sync() for cross-block barriers
- Replace serial cumulative sum by a single thread with parallel prefix sum where each thread computes a local segment and cross-block reduction merges segments, enabling multi-SM utilization for scan operations over hundreds of elements
- Fuse counting, alignment padding, and sorting into a single cooperative kernel to eliminate the launch overhead and memory round-trip of separate kernels, enabled by cooperative_groups grid.sync() for global synchronization within one kernel
