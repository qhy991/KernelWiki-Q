## Challenge

Warp-level max reduction in top-2 kernel uses manual __shfl_xor_sync loops with explicit conditional updates, resulting in verbose code that is harder to maintain and potentially less optimizable by the compiler


## Solution

Replace manual __shfl_xor_sync reduction loops with cooperative_groups::reduce using cg::greater<float>() comparator, which produces identical O(log W) shuffle steps in a more expressive and maintainable API

## Key Lessons

- cg::reduce with cg::greater<T>() produces the same O(log W) shuffle steps as a manual __shfl_xor_sync loop for warp-level max finding — prefer the cooperative_groups API for readability and maintainability when performance is equivalent
- Using __syncwarp() before cooperative_groups warp-level operations ensures correct execution barrier semantics, especially when preceding code has divergent branches
