## Challenge

The before code writes the scan running prefix to shared memory at index smem_rp[si] without the row offset r * MAX_DSTATE_VAL, while the read on subsequent chunks correctly uses smem_rp[si + r * MAX_DSTATE_VAL]. ## Solution

The after code aligns the write index with the read index by adding r * MAX_DSTATE_VAL, giving each row its own dedicated shared memory slot. ## Key Lessons

- Since the sole code change is adding an integer offset to a shared memory write index - a single addition that does not alter the instruction count, memory access width, or computation flow - the measured latency is essentially identical between both versions.
