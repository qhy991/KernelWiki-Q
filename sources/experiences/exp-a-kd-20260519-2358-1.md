## Solution

Replace manual stride assignment with cutlass::make_cute_packed_stride(StrideType{}, shape) to derive packed strides from tensor shape dimensions following SM100 conventions, and compute StrideD independently from StrideC to support separate C-source and D-destination layouts in the epilogue

## Key Lessons

- Use cutlass::make_cute_packed_stride(StrideType{}, shape) instead of manually assigning leading dimensions — it produces strides in the packed format SM100 collective builders expect and generalizes correctly to batched and transposed layouts without conditional logic
- Separate StrideD from StrideC in GEMM epilogue arguments (c_ptr, c_stride, c_ptr, d_stride) to support different source and destination tensor layouts — conflating them via ldd = ldc only works when C and D share identical memory layout
