## Solution

Replace the variable-count loop with a fixed 3-load/store pattern per lane: 2 value qwords (at lane_id and lane_id+32) plus 1 conditional tail/scale qword (value tail for lane_id < 8, scale for lane_id >= 8), eliminating loop overhead and enabling better instruction scheduling

## Key Lessons

- Replace variable-count transfer loops with fixed-pattern loads when item size is known at compile time — eliminates loop overhead (branch prediction, counter increments) and enables the compiler to schedule loads and stores more efficiently
- Use a conditional tail-load pattern (lane_id < threshold loads from value tail, else from scale) to handle remaining bytes after fixed-width loads, avoiding wasted bandwidth when value and scale regions are separated in memory
