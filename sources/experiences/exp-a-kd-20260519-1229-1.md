## Solution

Replace the divergent scalar bf16 copy with a warp-cooperative int-width vectorized copy: each warp processes one token and all 32 lanes load one 4-byte int each (32×4=128 bytes=64 bf16 values), eliminating the if-branch and enabling fully coalesced 4-byte memory transactions

## Key Lessons

- Warp-cooperative vectorized access eliminates divergence by ensuring all lanes participate → but for small per-token transfers (128 bytes) the total working set (16KB for 128 tokens) fits in L2 cache, making coalescing benefits negligible
- When total data volume is small enough to reside in L2 cache, divergence elimination and coalescing optimizations yield neutral results → focus optimization effort on kernels with larger memory footprints where DRAM bandwidth is the bottleneck
- Mapping one warp per token with int-width (4-byte) loads is an elegant pattern for copying N×4-byte regions → reuse when per-token data is a multiple of 128 bytes (32 lanes × 4 bytes) and the kernel is memory-bound on DRAM
