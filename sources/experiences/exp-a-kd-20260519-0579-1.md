## Solution

Replace static shared memory with dynamic shared memory (extern __shared__ float smem[]) and manual pointer arithmetic for multi-stage address computation, but the host side incorrectly sets cudaFuncAttributeMaxDynamicSharedMemorySize to 98304 bytes instead of the actual ~32 KB requirement, causing the driver to reserve 3x more shared memory per block than needed and reducing concurrent block occupancy

## Key Lessons

- When converting static to dynamic shared memory, cudaFuncAttributeMaxDynamicSharedMemorySize should be set to the actual required size, not a fixed maximum (e.g. 98304) — the driver reserves the full requested budget per block, reducing occupancy when the requested amount exceeds the real need
- Dynamic shared memory pointer arithmetic (manual byte offsets from a base pointer) adds complexity over static 3D array indexing but is necessary when shared memory layout must be determined at runtime; however, always compute and pass the exact smem_size to the kernel launch to avoid over-reservation
