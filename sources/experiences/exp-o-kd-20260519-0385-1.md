## Challenge

Element-wise hardswish activation kernel processes one half element per thread using scalar 2-byte loads and stores, underutilizing the GPU's 32-byte memory transaction granularity and wasting memory bandwidth


## Solution

Vectorize to half2 (4-byte) loads/stores so each thread processes two half elements in one coalesced transaction, doubling memory throughput by packing two 2-byte values into a single aligned 4-byte access via a HALF2 reinterpret_cast macro

## Key Lessons

- Vectorize half loads/stores to half2 (4-byte aligned access) to improve memory coalescing on GPUs with 32-byte transaction granularity — applies to any elementwise kernel operating on half-precision data where the computation is lane-independent
- Halve the thread count (256→128) when doubling per-thread work via half2 vectorization to keep total grid coverage and shared resource usage balanced
- Use reinterpret_cast<half2*>(&val)[0] (HALF2 macro) for zero-copy type-punning between adjacent half pairs and half2 registers when the memory is naturally aligned
