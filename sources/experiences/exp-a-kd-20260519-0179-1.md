## Solution

Replace local threadIdx.x / 128 with __shfl_sync(0xffffffff, threadIdx.x / NumThreadsPerWarpGroup, 0) to broadcast the warp-group index from lane 0 to all lanes, ensuring warp-wide agreement — but this introduces a shuffle barrier per invocation that dominates cost in tight loops

## Key Lessons

- __shfl_sync broadcast ensures warp-wide value agreement but forces warp convergence and adds synchronization latency — only use when threads may actually observe divergent values; a plain register-local integer division has zero cross-thread dependency and is faster when divergence is impossible by construction
- Warp-group index utilities (canonical_warp_group_idx) are correctness primitives for SM90+ cooperative kernels, not performance optimizations — evaluate whether the shuffle is needed given your launch configuration before applying
