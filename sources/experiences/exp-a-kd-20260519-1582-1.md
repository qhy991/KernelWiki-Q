## Solution

Remove __float2half() so the float computation result is assigned directly to dst_t*, eliminating the intermediate half conversion when dst_t=float

## Key Lessons

- Removing an apparently redundant __float2half() can regress performance because the half intermediate may reduce register pressure or enable better PTX scheduling in tight inner loops — the saved conversion instruction does not always dominate when memory bandwidth is the bottleneck
- When dst_t=float (4 bytes/element) vs half (2 bytes/element), store bandwidth doubles and can overshadow instruction savings from eliminating a conversion, making PTX-level scheduling artifacts the deciding factor
- A regression from removing a type conversion is likely a compiler scheduling artifact rather than a fundamental disadvantage — profile with nvcc --ptxas-options=-v to compare register counts before assuming the change is universally harmful
