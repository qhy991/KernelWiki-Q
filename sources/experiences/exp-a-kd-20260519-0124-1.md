## Solution

Replace the int32_t-only function with a templated warp_uniform<T> using a union of T and uint32_t for type punning, enabling a single implementation to work for any 4-byte type while compiling to the identical SHFL.U32 PTX instruction

## Key Lessons

- Union-based type punning between T and uint32_t in __shfl_sync wrappers is a zero-cost abstraction for any 4-byte type — nvcc emits the identical SHFL.U32 PTX instruction regardless of whether the cast is explicit or union-based, since sizeof(T)==sizeof(uint32_t) makes the union a register-level no-op
- Templating warp-level primitives over type T with a union punning pattern enables code reuse across int32_t, float, and other 4-byte types without per-type specialization or performance regression
