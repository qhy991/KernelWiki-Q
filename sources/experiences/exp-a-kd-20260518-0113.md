## Solution

Distribute exp2f computations across all 4 warps via warp-partitioned indexing (warp_id * kLinesPerWarp + lane_id), add a conditional 'changed' guard to skip exp2f when the running max is unchanged, and separate the rescale factor into a dedicated out_rescale array so m_prime retains the running max

## Key Lessons

- Distribute expensive transcendental operations (exp2f, ~20 cycles on A100) across multiple warps via lane_id-based partitioning instead of serializing on a single warp — allows the warp scheduler to interleave and hide latency
- Guard exp2f with a conditional 'changed' check on the running max to skip the transcendental entirely when the max has not increased — avoids unnecessary math on stable iterations
- Use a dedicated out_rescale array separate from m_prime (running max) to avoid overwriting state needed across iterations — keeps each variable's meaning single-purpose and avoids subtle correctness issues
