## Solution

Track cumulative tile counts across groups with group_total_tiles and head_cta_id accumulators, then compute blk_offset = (blockIdx.x + (gridDim.x - head_cta_id)) % gridDim.x to rotate which CTAs receive remainder tiles per group, reducing work distribution to [9,8,8,8,8,8,8,8] with stddev=0.331

## Key Lessons

- Rotating CTA starting offsets across independent work groups distributes remainder tiles to different CTAs per group, reducing workload variance and tail synchronization latency — applies whenever tile counts per group are not divisible by gridDim.x
- Computing head_cta_id = cumulative_tiles % gridDim.x provides a zero-cost deterministic rotation that requires no inter-CTA communication, making it suitable for any multi-group strided scheduling pattern
