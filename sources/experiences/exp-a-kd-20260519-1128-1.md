## Solution

Fuse the two kernels into a single candidates_after_kernel that reads raw scores and bias directly, computes sigmoid+bias inline in registers, and uses an if-block pattern for bounds and is_finite checks — eliminating one kernel launch and the entire intermediate scores_with_bias global-memory buffer allocation, write, and read cycle

## Key Lessons

- Fusing a producer kernel that writes an intermediate buffer with its consumer kernel into a single kernel eliminates one kernel launch, one global-memory allocation, and one full write+read cycle — applies whenever the intermediate data is used only once and fits in registers per thread
- Inline elementwise computation (sigmoid+bias) inside the reduction loop keeps transformed values in registers instead of writing to and reading from global memory, improving data locality for bandwidth-bound patterns
- Prefer if-block control flow over ternary expressions when the computation inside the condition is expensive (e.g., sigmoid), so the GPU can skip the SFU instruction entirely for out-of-bounds or non-finite elements
