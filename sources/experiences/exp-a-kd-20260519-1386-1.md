## Solution

Replace atomicOr word-level packing with direct byte-level write via reinterpret_cast<uint8_t*>, since each group writes to a unique byte position within the uint32 word — atomics are unnecessary when byte offsets are naturally disjoint

## Key Lessons

- When each writer targets a unique, non-overlapping byte within a packed word, direct byte-level writes via reinterpret_cast<uint8_t*> are correct without atomics — atomicOr is unnecessary when write positions are naturally disjoint by construction
- Removing atomics on a low-contention path (1 thread per group, ≤4 writers per word) may not improve measurable latency when the dominant cost is elsewhere (shared memory loads, warp reductions over group elements) — profile to confirm atomics are a bottleneck before optimizing them
- Padded grid iteration (iterating over padded_groups_per_row with is_valid_group guards) eliminates the need for separate cudaMemset zero-initialization of the scale buffer, since the kernel writes every byte including padding positions — this structural simplification can be more valuable than the latency improvement
