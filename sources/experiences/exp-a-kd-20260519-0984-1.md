## Solution

Add expert-based early-return filtering: each thread checks expert_ids[token_id / expert_step] == -1 before loading data, skipping the two float4 global loads (32 bytes), 4 expf-based SiLU computations, and float4 store (16 bytes) for masked tokens, reducing active memory traffic from ~96 MB to ~24 MB with 75% masking

## Key Lessons

- Early-return filtering based on expert/sparse masks yields bandwidth-proportional speedups in MoE-style activation kernels — the speedup scales with mask percentage because memory traffic dominates compute
- Block-per-token thread mapping (BLOCK_SIZE=256 threads == hidden_dim/VEC_SIZE vectors) ensures entire warps share the same token_id, making the early-return filter divergence-free — applies whenever hidden_dim evenly divides into one block's worth of vector threads
- The observed speedup (2.55x) is below the theoretical 4x for 75% masking because the same grid size is still launched — masked blocks incur scheduling and expert_ids comparison overhead even though they skip the heavy memory operations
