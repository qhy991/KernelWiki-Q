## Solution

Replace scalar byte-by-byte load/store with uint4 vectorized 16-byte load/store: each thread loads 16 bytes via reinterpret_cast<uint4>, processes all bytes in registers using the same kNibbleLUT, then writes back as a single uint4 store, reducing memory transactions by 16x and enabling full 128-byte coalesced transactions per warp

## Key Lessons

- Use uint4 (16-byte) vectorized load/store instead of scalar byte access for bandwidth-bound elementwise kernels — each warp of 32 threads produces 4 coalesced 128-byte transactions (512 bytes total) instead of 32 separate single-byte reads that cannot fully coalesce
- Processing vectorized data in registers (load uint4 → iterate bytes in local array → store uint4) eliminates redundant global memory round-trips while preserving the same per-element compute logic
- Vectorized access also reduces total thread-block count (fewer grid iterations), lowering kernel launch overhead and improving throughput on large datasets that exceed L2 cache
