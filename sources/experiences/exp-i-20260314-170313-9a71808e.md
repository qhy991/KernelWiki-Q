## Challenge

Implement top-k sampling with two stages: (1) find top-k indices from 160 probabilities, (2) categorical sample from them. Initial attempts failed due to race conditions in shared memory during parallel top-k selection.


## Solution

Simplified single-thread top-k selection in shared memory


### Key Changes

- Fixed function signature to include k parameter
- Used thread 0 for all top-k selection to avoid race conditions
- Fixed seed (42) for deterministic sampling
- Simple selection algorithm: find max, mark as used, repeat k times


## Key Lessons

- For small vocab sizes (160), single-thread top-k in shared memory is simpler and more reliable than complex warp-based parallel algorithms
- Race conditions in shared memory can cause non-deterministic behavior leading to test failures
- Fixed seed is essential for reproducible sampling in tests
- Function signature must match test framework expectations exactly
