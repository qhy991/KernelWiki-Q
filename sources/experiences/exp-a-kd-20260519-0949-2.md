## Solution

Replace warp-based div/mod index decomposition with direct shared_counts[expert_id] indexing, eliminating integer division and modulo operations from the hot atomicAdd counting loop since the shared memory layout now matches the expert_id range directly

## Key Lessons

- Use direct array indexing instead of div/mod decomposition when the logical index (expert_id) maps directly to the physical array index — eliminates expensive integer division/modulo instructions from hot loops that execute per-element
