## Challenge

Prefix sum over expert counts is computed serially by a single thread (threadIdx.x == 0) iterating all 64 experts while 1023 other threads idle at a __syncthreads() barrier, wasting SM occupancy


## Solution

Replace serial single-thread cumsum with a parallel Blelloch prefix scan: all threads cooperatively perform log2(N)=6 up-sweep and 6 down-sweep phases on a power-of-2 scan buffer, distributing work across the entire block

## Key Lessons

- Replace serial single-thread prefix sums with Blelloch parallel scan when element count fits in shared memory — O(log N) steps with all threads participating versus O(N) on one thread, though gains are modest when the serial loop is already short and dominated by other phases
- Blelloch scan requires a power-of-2 buffer; use next_pow2() to pad non-power-of-2 element counts, filling excess slots with zero to preserve correctness
