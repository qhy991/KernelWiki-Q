## Solution

Replace four scalar 32-bit global loads/stores per thread with one float4 (128-bit) vectorized load and one float4 store, processing two adjacent RoPE pairs simultaneously and halving global memory transactions

## Key Lessons

- Use float4 vectorized loads/stores to halve global memory transactions when processing multiple adjacent element pairs — applies to any elementwise or pairwise kernel where consecutive threads access contiguous memory
- When packing two RoPE pairs into one float4, compute separate freq/sin/cos values for each dimension index (token_idx*2 and token_idx*2+1) to maintain correct rotary angle per dimension, since adjacent pairs map to different frequency bands
