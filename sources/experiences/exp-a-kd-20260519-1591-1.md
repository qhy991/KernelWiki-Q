## Challenge

Each thread issues atomicAdd(&atomic_buffer[expert_id], 1) on every loop iteration where topk_ids[i] matches expert_id, causing up to O(topk_length) serialized atomic operations per block since all 512 threads target the same memory address


## Solution

Replace per-element atomicAdd inside the strided loop with register-local accumulation in a per-thread variable, then issue a single atomicAdd per thread after the loop, reducing atomic operations from O(topk_length) to at most THREADS_PER_EXPERT=512 per expert and eliminating severe serialization on the shared counter address

## Key Lessons

- Accumulate per-thread partial results in registers before issuing a single atomicAdd per thread — reduces atomic contention from O(iterations) to O(threads) and is applicable whenever multiple threads atomically update the same address inside a loop
- Converting a conditional atomicAdd(if match) to an unconditional register addition (occurrences += condition) also eliminates branch divergence inside the loop, since the boolean cast produces 0 or 1 without a divergent branch
