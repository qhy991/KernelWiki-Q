## Challenge

Per-token FP8 quantization kernel writes quantized bytes using scalar per-element stores in an unrolled loop (8 individual byte stores per vector), issuing 8192 separate store transactions for hidden_dim=8192 and severely underutilizing available memory bandwidth


## Solution

Double the vector size to 16 elements (matching 128-bit = 16 uint8_t output bytes) and replace the 8 scalar byte stores with a single uint4 128-bit vector store using a union for alignment, reducing total store transactions from 8192 to 512 and halving the loop iteration count

## Key Lessons

- Replace scalar byte stores with uint4 128-bit vector stores when writing contiguous uint8_t output arrays — a single uint4 store writes 16 bytes in one transaction, reducing store instruction count by 16x and improving memory throughput significantly
- Use a union (uint8_t[16] aliased with uint4) to build byte-level output in registers and issue the vector store without memcpy overhead — the union guarantees 16-byte alignment required by uint4
- Choose VEC_SIZE to match the target store width in bytes: VEC_SIZE=16 produces exactly 16 FP8 bytes = 128 bits = one uint4, making the vector store natural and avoiding partial-write handling
