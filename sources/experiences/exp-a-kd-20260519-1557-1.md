## Solution

Wrap the two global loads with __ldg() to explicitly request read-only cache routing, but on Ampere the const __restrict__ qualifier already achieves the same effect, resulting in no measurable performance improvement beyond measurement noise

## Key Lessons

- On Ampere (sm_80+) and later architectures, const __restrict__ pointer annotations already cause the compiler to emit LDG (read-only cache) instructions, making explicit __ldg() wrappers redundant and producing no performance gain
- Short-duration kernels (~6-7 us) exhibit measurement variance of ~3%, so speedups below the 5% threshold should be treated as noise rather than real improvement
