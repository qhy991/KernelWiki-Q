## Solution

Guard the entire allocation, memset, and kernel launch with a needs_alignment condition; when the CUTLASS group gemm path determines alignment is unnecessary, all GPU work is skipped entirely, eliminating per-iteration cudaMalloc/cudaMemset overhead and kernel launch cost

## Key Lessons

- Guard GPU allocation and kernel launch with a runtime condition (e.g., needs_alignment) to skip all GPU work when the computation result is unused — eliminates cudaMalloc/cudaMemset overhead and kernel launch latency entirely
- The allocation fixed cost (cudaMalloc + cudaMemset) dominates transfer cost for small buffers; skipping the allocation entirely when possible yields 2x+ speedup even when the kernel itself is lightweight
- Apply the same conditional guard to persistent allocations outside the hot loop (e.g., d_m_indices) to avoid unnecessary memory consumption when the code path is not active
