## Solution

Transpose the LSE array layout from head-major [num_heads][num_tokens] to token-major [num_tokens][num_heads], so adjacent threads (same token, consecutive heads) access contiguous addresses with stride num_heads (32 floats = 128B), enabling coalesced warp-level memory transactions for both reads and writes

## Key Lessons

- Transposing small metadata arrays (LSE) from head-major to token-major layout enables coalesced warp-level access but may yield no measurable speedup when the metadata traffic is negligible compared to packed value loads (uint4) that dominate kernel runtime
- When auxiliary data arrays (e.g., LSE, scale factors) are small enough to fit entirely in L2 cache (16KB each on A800 with 40MB L2), the coalescing benefit is hidden because even scattered reads are serviced from L2 without DRAM overhead
- Always profile the memory bandwidth breakdown before optimizing data layout — optimizing coalescing for arrays that represent <1% of total traffic provides negligible returns regardless of access pattern quality
