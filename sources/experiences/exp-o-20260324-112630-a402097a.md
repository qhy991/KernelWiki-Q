## Challenge

Optimize Q8_0 flash attention kernel for Llama3-8B with seq_len=512, baseline uses 1 warp per block (32 threads)


## Solution

4 warps per block, each warp processes seq_len/4=128 KV positions independently, then merge partial flash-attention states (m, l, o) via shared memory

## Key Lessons

- For KernelEvalPlus flash_attention, flash_attn_kernel must be extern C void (host function) that launches the actual __global__ kernel internally
- 4-warp parallel KV processing with shared memory merge gives ~2x speedup for small batches
- Large batch (512) may be slightly slower due to warp merge overhead - consider adaptive warp count
- The auto_wrapper.cu declares flash_attn_kernel as extern C void - always check this file before writing kernel.cu
