## Solution

Replace shared-memory tree reduction with a hardcoded __shfl_xor_sync warp-shuffle GroupReduceMax for 16-thread sub-warp groups, eliminating all __syncthreads barriers and shared-memory bank accesses, and pack 16 groups per block (256 threads) to increase occupancy over the original 1-group-per-block design

## Key Lessons

- Prefer warp-shuffle reduction (__shfl_xor_sync) over shared-memory tree reduction for sub-warp-sized groups (≤32 threads) — eliminates __syncthreads barriers and shared-memory bank contention, with the benefit growing as group size shrinks and reduction overhead dominates
- Pack multiple groups per block when per-group thread count is small (16 threads/group × 16 groups = 256 threads/block) to improve occupancy compared to one-group-per-block launches that leave SM resources underutilized
