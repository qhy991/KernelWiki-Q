## Challenge

Warp-level top-K uses separate butterfly reduction on float values followed by O(WARP_SIZE) ballot+ffs+shfl lane-search to correlate each max value with its index, requiring multiple warp shuffle rounds and conditional index broadcast per iteration


## Solution

Pack half-precision value bits and index into a single uint32_t via cub::Traits::TwiddleIn comparison-friendly encoding (value in upper 16 bits, inverted index in lower 16 bits), enabling one cg::reduce call per top-K iteration that simultaneously resolves both the max value and its originating index, eliminating the separate ballot/ffs/shfl lane-search entirely

## Key Lessons

- Pack value and index into a single comparison-friendly word (upper bits = twiddled value, lower bits = inverted index) so one warp reduction resolves both max value and its origin — eliminates O(WARP_SIZE) ballot/ffs/shfl lane-search per top-K iteration and applies to any associative reduction where auxiliary data must travel with the extremum
- Use cub::Traits<T>::TwiddleIn/TwiddleOut to convert floating-point values into unsigned integers that preserve ordering under integer comparison, enabling standard integer max/min reductions to work correctly on float/half/bfloat16 data
- Store index as (kMaxIdx - idx) in the compact word so that ties on value are broken by smaller index via the same integer greater-than comparison — no extra tie-breaking logic needed
