## Solution

Replace per-thread read-modify-write with a centralized _flag[blockIdx.x] read by all threads and written back only by thread 0 after __syncthreads(); split the unified barrier into separate start/end functions with dedicated arrays, eliminating modulo-2 alternation and reducing the peer_counter double-buffer to a single _flag[kMaxBlocks] per block

## Key Lessons

- Replace per-thread read-modify-write counters with a single centralized flag read by all threads and written by one designated thread after __syncthreads() — eliminates cache-line thrashing when multiple threads write different offsets within the same aligned cache line
- Splitting a unified barrier template with modulo-2 alternation into separate start/end functions with dedicated arrays removes the double-buffer indexing overhead and reduces struct memory footprint
