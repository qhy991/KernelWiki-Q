## Challenge

Per-token group quantization kernel uses atomicOr to pack 8-bit exponent values into uint32 scale words, causing per-word atomic contention when multiple thread groups target the same packed uint32 and requiring the entire buffer to be pre-zeroed since atomicOr only ORs bits without clearing them


## Solution

Replace atomicOr with direct byte-level writes via reinterpret_cast<uint8_t*> to store each exponent at its exact byte offset within the packed uint32, eliminating atomic contention and removing the dependency on a pre-zeroed buffer since each byte is written independently

## Key Lessons

- Use direct byte-level writes via reinterpret_cast<uint8_t*> instead of atomicOr when packing sub-word fields into uint32 buffers — byte stores are naturally atomic on GPU hardware and avoid both contention and the requirement for a pre-zeroed buffer
- When packing N-bit fields into aligned containers (e.g., 4 × uint8 into uint32), prefer byte-addressed stores over shift-and-atomicOR — the hardware guarantees single-byte write atomicity without needing explicit atomic operations
