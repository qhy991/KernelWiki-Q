## Solution

Replace the scalar element-by-element min/max loop with vectorize_read_with_alignment<16>, loading 16 floats (128 bytes) per thread via a vec_n_t aligned load and DefaultReadVecOp functor that unrolls each element through the same MinMax operator+= lambda

## Key Lessons

- Vectorized loads do not improve throughput when the scalar access pattern is already fully coalesced (one element per thread, contiguous stride-1 reads) — the memory system already saturates at one transaction per warp
- Lambda functor paths through template abstractions (DefaultReadVecOp, std::decay_t) increase register pressure and add indirection that can degrade performance on tiny single-block kernels where launch overhead dominates
- Vectorization benefits require headroom in memory throughput to exploit; when hidden_size equals blockDim.x (256 threads each reading 1 float = 1 KB total), the working set fits in a single cache line and wider loads offer no advantage
