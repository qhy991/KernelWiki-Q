## Challenge

Non-contiguous input (stride > hidden_size) forces a separate strided_to_contiguous copy kernel launch before the RMS norm+quant kernel can run, adding an extra full pass over all input elements, an extra kernel launch overhead, and a redundant contiguous buffer allocation


## Solution

Pass input_stride directly to the RMS norm kernel so each thread computes the correct input offset as blockIdx.x * input_stride, eliminating the separate copy kernel, the extra memory pass, and the contiguous buffer allocation entirely

## Key Lessons

- Make kernels stride-aware by accepting an input_stride parameter and computing row offsets as blockIdx.x * input_stride instead of hardcoding stride == hidden_size — eliminates a separate copy kernel launch, extra memory traffic, and a redundant contiguous buffer when input layout is non-contiguous
- The cost of an extra kernel launch + full data copy outweighs the slightly less cache-friendly strided access pattern in the compute kernel itself, because the copy also reads strided memory and adds a full extra write pass to contiguous memory
