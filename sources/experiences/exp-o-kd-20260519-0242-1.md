## Solution

Replace 128 per-element bit extractions with 16 direct byte-level reads via reinterpret_cast to uint8_t*, checking each byte for non-zero — since any set bit in any byte means logical OR is true, reducing iteration count 8x and eliminating per-element index/shift/mask overhead

## Key Lessons

- For logical reductions (OR/AND) on packed bit arrays, operate at byte granularity instead of extracting individual bits — any non-zero byte implies OR=true, any byte != 0xFF implies AND=false, reducing work by 8x with no per-element overhead
- Short-circuit evaluation (item || bits) combined with byte-level scanning creates naturally predicted branches: once item becomes true, subsequent iterations become trivially predicted, further reducing effective work
