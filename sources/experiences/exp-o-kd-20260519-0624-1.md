## Solution

Replace the entire shared-memory butterfly reduction with a parameterized reduce_sum<kNumThreads> template using __shfl_xor_sync warp shuffle intrinsics, performing 3 register-to-register shuffle instructions (~4-8 cycles each) with no shared memory access, no barriers, and no warp divergence

## Key Lessons

- Replace shared-memory butterfly reduction with __shfl_xor_sync warp shuffle for sub-warp reductions — eliminates __syncthreads() barriers and shared-memory traffic, yielding dramatic speedups for hot reduction loops
- Parameterize the reduction thread count as a compile-time template constant (kNumThreads) with static_assert power-of-2 check — enables #pragma unroll and produces optimal shuffle sequences for any sub-warp group size
- Use an active_mask bitmask ((1u << kNumThreads) - 1) to constrain __shfl_xor_sync participation to only the active threads — avoids undefined behavior when reducing across fewer than 32 threads
