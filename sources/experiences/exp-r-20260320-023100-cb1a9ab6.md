## Key Lessons

- Small K dimension (< 64) severely limits GEMM optimization potential
- Shared memory tiling benefit scales with K - more tiles = more reuse
- Memory bandwidth is primary bottleneck for small-K GEMM
- Register tiling can hurt performance when K is too small
- For K=32, simple 16x16 tiling is optimal despite modest speedup
