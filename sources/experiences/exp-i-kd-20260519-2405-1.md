## Solution

Implement a three-level pipelined GEMM: (1) construct CuTe global/shared/register tensors with static assertions for shape compatibility, (2) prefetch K_PIPE_MAX-1 shared memory stages via async copy (cp.async) before the main loop, and (3) run a pipelined main loop that overlaps gmem-to-smem async copies, smem-to-register prefetch, and register-level warp MMA, finishing with an axpby epilogue for alpha/beta scaling

## Key Lessons

- Prefetch K_PIPE_MAX-1 shared memory pipeline stages before entering the main GEMM loop so that the first compute iteration finds data ready in smem — eliminates initial stalls and allows full overlap of gmem→smem, smem→rmem, and MMA from the first iteration
- Use a three-level pipeline (gmem→smem async copy via cp.async, smem→rmem register prefetch with s2r atoms, rmem MMA compute) where each level operates on a different pipeline stage simultaneously — the k_block loop interleaves smem→rmem load of k_block+1 with MMA on k_block, while k_block==0 issues the next gmem→smem async copy
- Rotate shared memory pipeline indices (smem_pipe_read/smem_pipe_write) with modular arithmetic and synchronize with cp_async_wait<K_PIPE_MAX-2>() + __syncthreads() only at the pipeline stage boundary (k_block == K_BLOCK_MAX-1) to minimize barrier overhead while guaranteeing data readiness
