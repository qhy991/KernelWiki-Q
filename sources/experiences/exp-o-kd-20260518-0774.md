## Challenge

Optimization focus: Replaced conditional scale argument construction that uses .item<float>() (GPU→CPU sync) with uniform device-pointer + boolean-flag construction.
Problem before: When scales are per-tensor (numel() == 1), the dispatcher calls a_scales.item<float>() and b_scales.item<float>() to e


## Solution

ons where kernel execution time is short, this synchronization overhead can dominate total latency. Additionally, moving scales to the CPU caused torch.compile graph breaks.
Applied change: The dispatcher now always passes scales via a device pointer (a_scales.data_ptr<float>()) together with a bool

## Key Lessons

- This eliminates the GPU→CPU synchronization entirely because no host-side value extraction is needed.
