## Solution

Fuse SiLU and element-wise multiplication into a single kernel that loads gate and up vectors once, applies SiLU to gate values in registers, multiplies by up values, and writes directly to output — eliminating the intermediate global memory buffer and second kernel launch entirely

## Key Lessons

- Fuse elementwise activation (SiLU) with follow-up elementwise operations (Mul) into a single kernel to eliminate intermediate global memory round-trips — applies whenever the activation output is only consumed by another elementwise op in the same pipeline
- Eliminating intermediate global memory buffers reduces both memory bandwidth pressure (one fewer write+read per element) and kernel launch overhead, yielding compounding speedup on bandwidth-bound workloads
- Loading both input vectors (gate, up) in the same fused kernel keeps their values in registers between the SiLU and Mul stages, avoiding the store-to-global-then-reload pattern that dominates cost in separate-kernel approaches
