## Challenge

The min_scaling_factor value (FLT_EPSILON) is passed as a runtime kernel launch parameter, adding a float argument to the kernel signature that the compiler cannot fold into an immediate operand, requiring parameter buffer setup at each launch


## Solution

Remove the runtime min_scaling_factor parameter and replace it with a type-specialized compile-time trait (min_scaling_factor<int8_t>::val()) that returns FLT_EPSILON via __device__ __forceinline__, allowing NVCC to fold the constant into an immediate operand in the fmax instruction

## Key Lessons

- Replacing runtime kernel parameters with compile-time trait specializations enables constant folding by NVCC, producing identical SASS with cleaner kernel signatures — applicable when the parameter value is a type-dependent constant known at compile time
- Moving compile-time-known values from kernel launch parameters to __device__ __forceinline__ traits eliminates per-launch parameter buffer overhead and makes the value's dependence on the data type explicit in the type system
