## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: SM 70+ (Volta and newer); Shared memory limit: 48KB (SM 70) / 96KB (SM 80+); Use warp-level primitives (__shfl_* operations)

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: SM 70+ (Volta and newer); Shared memory limit: 48KB (SM 70) / 96KB (SM 80+)
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
