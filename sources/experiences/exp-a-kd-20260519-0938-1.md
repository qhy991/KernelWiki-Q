## Solution

Remove the redundant always-true guard and dedent the body, compute topk_excluding_share_expert_fusion to decouple shared expert slots from the topk loop iteration count, add a conditional shared expert assignment block gated on n_share_experts_fusion > 0, and simplify rescale to direct float division

## Key Lessons

- Redundant guards whose condition duplicates the enclosing loop bound are always true and compile away — removing them improves readability without changing semantics, but produces negligible performance difference since the optimizer eliminates dead branches
- Computing an effective topk count (topk_excluding_share_expert_fusion) that subtracts reserved shared expert slots decouples the selection loop iteration count from the total output slot count, enabling a clean shared-expert extension without restructuring the core topk logic
- In backward-compatibility scenarios (n_share_experts_fusion == 0), the effective topk equals the original topk and the shared expert assignment branch is dead code — the compiler treats the always-false conditional as a no-op, yielding neutral performance impact
