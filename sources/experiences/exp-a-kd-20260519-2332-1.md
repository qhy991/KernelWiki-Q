## Solution

Insert __threadfence_block() before every __syncthreads() call — both after the initial shared memory load and inside each tree reduction iteration — to explicitly commit all pending shared memory writes before reaching the barrier, reducing warp stall time at synchronization points

## Key Lessons

- Adding __threadfence_block() before __syncthreads() in shared memory reduction kernels can yield measurable speedup by committing writes earlier and reducing warp stall time at the barrier — even though __syncthreads() already includes an implicit fence for shared memory, the explicit fence helps the hardware pipeline settle writes sooner
- This NVIDIA pattern is the direct analogy to AMD's s_waitcnt 0 before __syncthreads() — both ensure pending async memory operations complete before the barrier, preventing threads from entering synchronization while stores are still in flight
