## Solution

Parameterize the reduction via a kNumThreads non-type template parameter with static_assert guards (power-of-2, within [1,32]), deriving the initial mask from kNumThreads/2 so the same function works for any sub-warp power-of-2 width while preserving full-warp behavior when kNumThreads=32

## Key Lessons

- Parameterize warp shuffle reductions via a compile-time kNumThreads template with static_assert(has_single_bit) so one function handles full-warp and sub-warp cases without runtime overhead — the compiler unrolls identical shuffle sequences for any power-of-2 thread count
- Hardcoded magic numbers like mask=16 in butterfly reductions limit reusability; deriving mask from kNumThreads/2 at compile time is zero-cost generalization that preserves full-warp performance (0.18% delta is measurement noise)
