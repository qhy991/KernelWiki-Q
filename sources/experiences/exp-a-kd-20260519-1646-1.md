## Challenge

The cast-and-scale kernel embeds an inline if constexpr type-conditional branch directly inside the per-element conversion loop, mixing type dispatch logic with computation and making the kernel harder to extend with additional packed types


## Solution

Extract the inline if constexpr branch into a standalone __device__ __forceinline__ helper (cast_to_float2) that dispatches the correct conversion intrinsic per type, keeping the kernel loop body clean while producing identical PTX (cvt.rn.f32.bf16x2 for bfloat162 specialization)

## Key Lessons

- Extracting if constexpr type dispatch into a __device__ __forceinline__ helper produces identical PTX because if constexpr is resolved entirely at compile time regardless of whether it is inline or in a separate function — safe refactoring with zero performance risk
- Sub-1% timing deltas on memory-bound conversion kernels at 1M elements are within measurement noise — verify PTX instruction-level equivalence before claiming a performance difference
- Adding extra if constexpr branches (e.g., float2 passthrough) that are not selected for a given template instantiation are eliminated by the compiler and do not affect generated code or register pressure
