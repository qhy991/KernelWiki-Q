## Solution

Fuse scale multiplication and zero-point subtraction into a single __hfma instruction: __hfma(w, s, __hneg(zp)) computes w*s + (-zp) = w*s - zp in one FMA, reducing instruction count and register pressure in the inner dequant loop

## Key Lessons

- Fusing __hmul + __hsub into __hfma(w, s, __hneg(zp)) reduces per-element instruction count from 2 to 1 FMA — the throughput gain is negligible in isolated microbenchmarks where the scheduler can overlap independent half-precision ops, but becomes meaningful under register pressure when multiple B-fragment elements are processed concurrently in the inner loop of kernels like Marlin
- __hfma(a, b, c) computes a*b+c in one instruction — use __hneg to flip the sign of the subtract operand (zp) to express a*b - c as a*b + (-c), enabling single-instruction fusion of multiply-subtract patterns
- Microbenchmark results showing neutral speedup for instruction fusion do not always predict production kernel behavior — the real benefit depends on surrounding code context, particularly register pressure and instruction-level parallelism saturation
