## Challenge

In the before code, get_num_blocks() rounds work_blocks (600) up to gpu_occupancy (432) multiples, yielding 864 blocks — 264 idle CTAs


## Solution

The after code uses fast_max(600, 432)=600, launching only the blocks that do real work. Fewer launched blocks reduces scheduler overhead and kernel launch time while maintaining full GPU utilization since 600 blocks across 108 SMs provides ample work.

## Results

- **speedup**: 1.226x (improved)
