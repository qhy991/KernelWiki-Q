## Challenge

The before code uses if(guard) wrapping a scalar 32-bit float store, while the after code uses inline PTX @p st.global.u32 with a setp predicate. For a single scalar store guarded by a boolean, the CUDA compiler at -O3 already emits a predicated s


## Solution

Replace the C++ if-branch with inline PTX predicated global store (setp.ne.b32 + @p st.global.u32), mirroring cutlass::arch::global_store<AccessType, 4> to explicitly control hardware predication at the PTX level

## Results

- **speedup**: 1.000x (neutral)


## Key Lessons

- For simple scalar predicated stores, nvcc at -O3 already emits hardware-predicated instructions equivalent to explicit PTX @p st.global — manual PTX provides no measurable benefit over compiler-generated code in this case
- When a kernel is memory-bandwidth-bound (16 scalar stores per thread across 256K threads), minor instruction-level differences such as branch-vs-predicate are hidden by the memory bottleneck, making such micro-optimizations undetectable in isolation
- The cutlass::arch::global_store pattern via inline PTX is designed for vectorized wider accesses (128-bit etc.) where the compiler may not always emit optimal predication — for single 4-byte scalar stores the compiler is already optimal
