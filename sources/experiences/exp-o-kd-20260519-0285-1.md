## Solution

Replace the occupancy-rounded waveset formula (ceiling division into gpu_occupancy multiples) with a simpler fast_max(work_blocks, avail_sms*4) strategy that launches exactly the blocks needed for real work while guaranteeing a minimum grid size of 4 SM waves for hardware utilization

## Key Lessons

- Avoid rounding grid size up to full gpu_occupancy (sm_occupancy * avail_sms) multiples when work_blocks already provides sufficient SM coverage — the ceiling-division inflation launches idle CTAs that add scheduler overhead without throughput benefit
- Prefer fast_max(work_blocks, minimum_grid) over waveset rounding for grid sizing — launching exactly the blocks that do real work reduces kernel launch latency while a small minimum floor (avail_sms * 4) ensures hardware occupancy is maintained
