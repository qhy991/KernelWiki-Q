## Solution

Insert __threadfence_block() before every __syncthreads() call — both after the initial shared memory load and inside each tree reduction iteration — to ensure all pending shared memory writes are committed before the barrier, reducing warp stall time at synchronization points

## Key Lessons

- Inserting __threadfence_block() before __syncthreads() in shared memory reduction kernels allows the memory subsystem to settle writes earlier, reducing warp stall time at the barrier — applies whenever shared memory is written and then immediately read across threads in a barrier pattern
- Although __syncthreads() includes an implicit memory fence for shared memory on NVIDIA hardware, an explicit __threadfence_block() can still improve pipeline efficiency by committing writes before the barrier is reached, analogous to AMD's s_waitcnt 0 before __syncthreads()
