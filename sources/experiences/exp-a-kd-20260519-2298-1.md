## Solution

Apply loop unrolling with factor=4 and separate the read phase (batch all __ldg loads into local arrays) from the write phase (batch all stores), allowing the GPU memory subsystem to issue multiple outstanding read requests concurrently and avoid read-after-write resource conflicts on the memory pipeline

## Key Lessons

- Separating read and write phases (batch all __ldg loads into local arrays before issuing stores) allows the GPU memory subsystem to issue multiple outstanding read requests concurrently and hide read latency — applies when a kernel performs interleaved read-then-write operations in a tight loop
- Loop unrolling with #pragma unroll combined with local array buffers gives the compiler full scheduling freedom to reorder independent loads for maximum throughput — applies when loop iterations are independent and the iteration count is divisible by the unroll factor
- Using register-allocated local arrays (fixed-size arrays indexed by unroll constant) as buffers between load and store phases eliminates pipeline stalls between dependent load and store instructions — applies when the unroll factor is small enough to fit in registers
