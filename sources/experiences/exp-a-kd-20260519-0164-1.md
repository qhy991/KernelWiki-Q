## Solution

Add an `sm_occupancy > 1` guard to the modulo-based SK wave formation branch so that when only one wave fits per SM the code skips the buggy branch and falls through to the correct fallback that combines the last full wave with partial-wave tiles into a single larger dispatch

## Key Lessons

- Guard modulo-zero-derived conditions with an occupancy > 1 check — when sm_occupancy==1, `x % 1 == 0` and `1 - 1 == 0` collapse to a tautology, so any branch predicated on that modulo test will always be taken regardless of the intended logic
- In multi-occupancy dispatch planners, always verify the degenerate single-occupancy case separately: the modulo-based wave-boundary detection pattern assumes at least 2 concurrent waves per SM to produce meaningful results
