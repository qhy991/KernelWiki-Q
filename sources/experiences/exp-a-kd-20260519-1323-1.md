## Solution

Replace conditional can_vectorize dispatch with unconditional vectorize_read_with_alignment<16> that uses address-based alignment detection (addr & (WIDTH-1)) and a prefix/main/tail decomposition, wrapped in a lambda-based scalar operator — this generalizes correctness for unaligned inputs but adds functor indirection and an alignment-check branch over the tight direct for-loop in thread_max_vec

## Key Lessons

- Generalizing vectorization with unconditional alignment handling adds lambda/functor indirection and alignment-check branches that prevent nvcc from generating as tight a loop body as a direct for-loop — for simple aligned cases the conditional path produces better PTX
- When each thread processes few vector packs (e.g., 4 packs with 256 threads on 1024 elements), fixed per-iteration overhead (alignment check, lambda dispatch) amortizes poorly compared to a tight direct vectorized loop
