## Challenge

Scalar per-element quantization loop issues one float load and one int8 store per iteration, underutilizing memory bandwidth and using division by scale in the hot path instead of multiplication by precomputed inverse


## Solution

Use vectorize_with_alignment<16> to process 16 elements per vector iteration, coalescing float loads and int8 stores into wide memory transactions, and precompute inv_s = 1/scale to replace division with multiplication in the quantization expression

## Key Lessons

- Use vectorize_with_alignment<N> to coalesce N scalar loads/stores into single wide memory transactions in per-element transform loops — applies to any quantization/cast kernel with contiguous memory access patterns
- Precompute reciprocal (1/scale) outside the loop to replace per-element division with multiplication — division costs 5-10x more FLOPs than multiply on GPU
