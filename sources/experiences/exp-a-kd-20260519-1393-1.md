## Solution

Always load scales into shared memory on every pipeline iteration (guarded only by the thread-bound predicate tid < SCALE_STAGE_INT4) and advance the global read pointer exclusively at group boundaries using (pipe+1) % group_ratio == 0, ensuring every stage receives valid scale data while maintaining the same scale value for all stages within a group

## Key Lessons

- In pipelined quantized GEMM, scale fetching must happen on every pipeline stage, not just at group-start boundaries — stages within the same group share the same scale pointer but still need their own shared-memory copy loaded
- Separate the load predicate (thread-level parallelism guard) from the pointer-advance predicate (group boundary) to ensure correctness: load always, advance only at group boundaries
- Correctness fixes in pipelined kernels may add small overhead (~0.83% for redundant shared-memory loads) but are essential — stale or uninitialized scale data produces silently incorrect dequantization results
