## Challenge

Padding fill of sorted_token_ids array uses scalar per-element int32 stores, issuing one 4-byte STG per iteration and underutilizing available memory bandwidth for the 8192-element array


## Solution

Replace scalar int32 stores with int4 (Vec) vectorized stores that pack 4 int32 values into a single 16-byte write per iteration, reducing total store transactions by ~4x and improving memory coalescing for arrays >= 8192 elements

## Key Lessons

- Use int4/float4 vectorized stores instead of scalar stores for uniform-value fill kernels to reduce store transaction count by 4x and improve memory coalescing — applies whenever filling contiguous arrays with a constant value and the size is a multiple of the vector width
- Pre-compute the fill vector once (fill_vec.x = fill_vec.y = fill_vec.z = fill_vec.w = value) outside the loop to avoid redundant packing per iteration
