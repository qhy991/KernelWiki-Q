## Solution

Fuse the pre-scale division and half-to-fp8 conversion into a single kernel: read half, divide by scale in float domain, then directly convert float to fp8 via __nv_cvt_float_to_fp8 — eliminating the intermediate half buffer, one kernel launch, and the float→half→float round-trip

## Key Lessons

- Fusing a pre-scale kernel with a type-conversion kernel into a single launch eliminates the intermediate buffer allocation, halves global memory traffic, and removes one kernel launch overhead — applies whenever a chain of elementwise transformations can be composed arithmetically
- Using __nv_cvt_float_to_fp8 directly from float domain bypasses the intermediate half representation (__nv_cvt_halfraw_to_fp8), avoiding a float→half round-trip and reducing conversion instructions — applies when the input is already in float domain after arithmetic
