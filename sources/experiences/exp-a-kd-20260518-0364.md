## Challenge

The after kernel reorganizes memory access from per-element bf16 reads/writes (naive before) to warp-cooperative vectorized int2/int transactions. In the naive version, each thread computes a linear index and reads one 2-byte bf16 value with a conditional branch (NOPE vs ROPE), resulting in uncoales


## Solution

tter reads from k_nope and k_rope plus scatter writes to k. The after version assigns one warp per 16-head chunk; each lane reads a single int2 (8 bytes) covering 4 bf16 values of k_nope per head and one int (4 bytes) covering 2 bf16 values of k_rope per token, producing fully coalesced 32-wide memo
