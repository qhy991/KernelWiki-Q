## Challenge

Flat TopK warp reduction scans all N=16 candidates per thread in each of K=8 iterations to find the local maximum, then performs one warp reduction per iteration — simple but the code attempts to improve it via hierarchical grouping and sorted extraction


## Solution

Replace flat scan-all-N with hierarchical strategy: split N=16 into 4 groups of 4, pre-sort each group via a 5-comparator sorting network (Sort4), extract top-K from each group using sorted-list shift-down, then merge results via a final warp reduction — but the extra warp synchronizations (4 per-group plus 1 merge) outweigh the reduced per-group work for small K=8 and N=16

## Key Lessons

- For small K and N (K<=8, N<=16), flat single-scan TopK with K warp reductions outperforms hierarchical partitioning — the overhead of N/4+1 additional warp synchronizations exceeds savings from smaller per-group candidate sets
- Sorting network pre-processing (Sort4) adds instruction overhead without payoff when N is small enough that a flat O(N) per-thread scan is already cheap relative to the warp synchronization cost
- Warp reduction calls dominate TopK performance for small candidate sets — minimizing the total number of cg::reduce invocations matters more than reducing per-thread comparison count
