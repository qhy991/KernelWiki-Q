## Solution

Enable block swizzle (BLOCK_SWIZZLE=true) by switching to a 3D grid launch with interleaving stride=2048, which remaps thread block indices so adjacent blocks share L2 cache lines for both A and B matrix tiles, improving L2 locality and temporal reuse

## Key Lessons

- Enable block swizzle via 3D grid launch with interleaving stride to remap thread block indices so adjacent blocks share L2 cache lines for both A and B — applies to tiled GEMM kernels where consecutive blocks exhibit poor L2 temporal reuse on shared matrix tiles
- Block swizzle benefit scales with problem size and number of column-blocks; at M=N=K=512 with 4 column-blocks (512/128), the interleaving yields 2.54x speedup by converting sequential column-block access into L2-friendly interleaved access
