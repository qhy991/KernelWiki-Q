## Solution

Implement a two-level pipelined loop: outer loop iterates k_tiles (gmem tiles, CUTE_NO_UNROLL for across-tile overhead), inner loop iterates k_blocks (smem register tiles, CUTE_UNROLL for within-tile optimization). At k_block==K_BLOCK_MAX-1, drain register pipeline to smem with double syncthreads barriers. At k_block==0, prefetch next k_tile from gmem to registers. Every iteration, copy next k_block from smem to registers and compute gemm on current k_block registers concurrently

## Key Lessons

- Use CUTE_NO_UNROLL on the outer k_tile loop (across gmem tiles) to avoid code bloat from unrolling, and CUTE_UNROLL on the inner k_block loop (within a smem tile) for maximum ILP — the unrolling strategy matches the memory hierarchy boundary
- Schedule gmem→rmem prefetches at k_block==0 (beginning of each k_tile) and rmem→smem writes at k_block==K_BLOCK_MAX-1 (end of each k_tile) so that global memory latency is hidden behind K_BLOCK_MAX MMA iterations
- Guard the last k_tile prefetch with (k_tile + 1 < K_TILE_MAX) to avoid out-of-bounds global memory reads — reuse the current tile's data as a safe no-op for the final iteration
