## Solution

Switch the alignment predicate from stride(-1) to stride(2), the per-head stride in the [batch, seqlen, heads, dim] layout, which reflects the actual spacing between adjacent heads in memory and correctly matches the vectorized alignment requirement of the packed Q/K/V accesses

## Key Lessons

- When dispatching between aligned and generic kernel paths, check the stride dimension that controls the actual vectorized access pattern (e.g., stride(2) for per-head spacing in BMHK layout), not the innermost stride(-1) which is often 1 for contiguous tensors and produces false negatives on the alignment test
- For multi-dimensional tensor dispatch logic, the correct stride to validate is the one governing the packed dimension that the kernel vectorizes over — the innermost stride is a misleading proxy when the memory layout packs along a higher axis
