## Solution

Reduce TILE_M from 128 to 64 to exactly match the problem dimension M, eliminating all padding rows; this halves the per-thread accumulator count from 64 to 32 int32s, drops register pressure from ~80 to ~52 registers per thread, and shrinks shared memory from 8800 to 6496 bytes per block, enabling 4-5 concurrent blocks per SM

## Key Lessons

- Match tile dimensions to the actual problem size to eliminate padding overhead — when TILE_M exceeds M, half the threads compute zero-padded rows that waste registers, shared memory, and arithmetic
- Reducing per-thread accumulator count directly lowers register pressure, which is often the primary occupancy limiter for GEMM kernels — cutting accumulators from 64 to 32 int32s increased concurrent blocks per SM from ~3 to ~5
- Occupancy-driven speedups compound: fewer registers per thread enables more blocks per SM, smaller shared memory per block lifts the shared-memory occupancy ceiling, and both together improve latency hiding for memory-bound tile loads
