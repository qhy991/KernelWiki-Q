## Solution

Pack two half elements into a half2 register per thread using reinterpret_cast-based HALF2 macro, processing 2 elements per thread with 4-byte load/store instead of 2-byte, and halving thread count (256→128) while keeping the same grid coverage

## Key Lessons

- Scalar half kernels accessing consecutive 2-byte elements already achieve near-full memory coalescing on modern GPUs (A800) because the coalescing hardware merges warp-wide 2-byte accesses into wider transactions — explicit half2 vectorization yields negligible gains for this pattern
- half2 vectorization of elementwise activation kernels provides measurable benefit only when the per-element computation is compute-bound enough that reduced thread overhead matters, or when the scalar access pattern is misaligned — for memory-bound hardswish on already-coalesced fp16 data, the technique is redundant
