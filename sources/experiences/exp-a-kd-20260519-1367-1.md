## Solution

Assign one warp to each (token, head) pair and compute the non-contiguous base pointer once per warp via stride arithmetic, so all 32 lanes access contiguous inner-dimension memory from a shared base, eliminating per-element index decomposition and enabling fully coalesced transfers

## Key Lessons

- Assign one warp per (token, head) pair when copying non-contiguous tensors with transposed strides — computing the base pointer once per warp amortizes stride arithmetic across all 32 lanes and guarantees contiguous inner-dimension access
- Flat grid-stride loops with per-element div/mod index decomposition are especially harmful for non-contiguous layouts because integer arithmetic overhead and memory non-coalescing compound on every iteration
