## Challenge

Naive 4-bit dequantization uses a scalar pipeline of shift/mask to extract four nibbles individually, conditional sign-extension via if/else, float conversion with explicit zero-point subtraction, and __floats2half2_rn — producing many instructions per value, float temporaries, and high register pressure from per-nibble arrays


## Solution

Replace the entire shift/mask/sign-extend/float-convert pipeline with lop3.b32 inline PTX that fuses nibble extraction and fp16 exponent injection into one instruction per nibble pair, then use __hsub2/__hfma2 with IEEE 754 magic constants to complete dequantization via native half2 arithmetic

## Key Lessons

- lop3.b32 (lookup-table ternary logic) fuses mask + OR into a single PTX instruction, replacing separate AND/shift sequences and reducing instruction count and register pressure — the benefit is most pronounced in full kernels where dequant shares issue slots with MMA pipelines, not in isolated microbenchmarks where nvcc auto-optimizes the naive path
- Use IEEE 754 magic constants (e.g., 0x64006400 for fp16 exponent injection, 0x64086408 for subtraction baseline) to convert integer nibbles into valid fp16 values via bitwise manipulation, enabling native __hsub2/__hfma2 half2 arithmetic instead of scalar float conversion and __floats2half2_rn
