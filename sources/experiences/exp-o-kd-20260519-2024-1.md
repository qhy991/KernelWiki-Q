## Solution

Add an early-exit fast path that checks token_idx >= prefix_num_tokens at the top of the kernel: for suffix-only tokens, directly copy suffix_output via a single 128b pack load/store and read suffix_lse, bypassing the full LSE rescaling, exp, division, and per-element FMA merge computation

## Key Lessons

- Add early-exit fast paths for tokens that don't need the full computation (e.g., suffix-only tokens with no prefix data) — even when the general path would produce the correct result (p_scale=0), skipping the exp/div/FMA saves both arithmetic and memory bandwidth proportional to the fraction of fast-path tokens
- When a parameter like prefix_num_tokens enables a data-dependent shortcut, pass it into the kernel and check before entering the expensive path — the branch divergence cost is negligible when entire warps of threads hit the same path (tokens are processed in contiguous groups)
