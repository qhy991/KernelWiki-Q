## Key Lessons

- The measured ~1% slowdown is negligible because this microbenchmark kernel is memory-bound: each thread loads 4 half2 weights (32 bytes) and 8 half activations (16 bytes) from global memory, and the arithmetic latency is hidden behind memory access latency.
- Comparing the before source (dot22_8_h_half2acc, lines 12-19) and after source (dot22_8_h, lines 11-31): the before accumulates in half2 via 4 __hfma2 instructions (8 FP16 multiply-adds packed into 4 dual-issue operations), then reduces with __hadd and applies the scale with __hfma.
