## Solution

Replace 4 separate half2 loads/stores with a single LDST128BITS (float4) 128-bit packed load/store pair, consolidating 8 memory instructions into 2 coalesced transactions, and replace 4 boundary checks with a single (idx+7<N) check

## Key Lessons

- Use 128-bit (float4) LDST128BITS load/store to consolidate multiple half2 memory transactions into a single coalesced access when processing 8+ half elements per thread — reduces memory instruction count from 2N to 2 and improves bandwidth utilization
- A single boundary check (idx+7 < N) replaces multiple per-pair checks when using packed 128-bit loads/stores, reducing branch overhead and producing cleaner control flow
- #pragma unroll over a local half[8] array produces a compact instruction sequence that may improve instruction cache utilization compared to manually expanded per-register field access
