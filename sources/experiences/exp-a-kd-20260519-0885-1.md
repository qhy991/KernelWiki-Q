## Solution

Replace global iteration with direct thread-to-expert partitioning: compute expert_idx and tid_in_expert from thread ID via integer division/modulo with remainder handling, eliminating the offset array entirely and mapping each thread to one expert in O(1)

## Key Lessons

- For MoE kernels with uniform expert sizes, partition threads across experts via integer division/modulo of thread ID instead of searching offset arrays at runtime — reduces per-element overhead from O(n_experts) comparisons and global memory reads to O(1) arithmetic
- Grid size should be a multiple of n_experts when using thread partitioning to avoid load imbalance from uneven remainder distribution
