## Solution

Cache Pass-1 input data into shared memory with 32-byte bank-conflict-padded stride per warp, then read from shared memory in Pass-2, eliminating the redundant global memory load; shared memory delivers ~5TB/s effective bandwidth versus ~50-80GB/s from HBM, and the __syncwarp() barrier cost (~10 cycles) is negligible compared to the saved global reads

## Key Lessons

- When a kernel makes multiple passes over the same input data, cache the data in shared memory on the first pass and read from smem on subsequent passes — shared memory (~5TB/s) provides ~60-100x higher effective bandwidth than HBM, and the warp barrier cost is negligible
- Use per-warp bank-conflict-padded shared memory stride (round up to 32-byte boundary) when multiple warps share a single smem buffer — prevents cross-warp bank conflicts that would serialize shared memory accesses
- Do not rely on L1 cache to retain data between kernel passes under concurrent CTA execution — total CTA working sets (e.g., 8 warps × 4KB = 32KB) can exceed L1 capacity, causing evictions that force expensive re-reads from L2 or HBM
