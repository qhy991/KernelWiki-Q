## Challenge

Per-block sum reduction uses atomicAdd to a single global-memory address (atomic_buffer[expert_id]) contested by all 512 threads in the block, causing severe atomic serialization; additionally requires cudaMemset to zero the buffer before each launch and a subsequent global read by thread 0 to retrieve the result


## Solution

Replace global-memory atomicAdd + __syncthreads + global read with CUB BlockReduce, which performs the sum cooperatively in shared memory using a hierarchical warp-level reduction pattern, eliminating atomic contention, the separate atomic buffer, and its cudaMemset reset

## Key Lessons

- Replace global-memory atomicAdd with CUB BlockReduce for per-block reductions — BlockReduce uses shared-memory warp-level hierarchical reduction, avoiding atomic serialization when all threads in a block write to the same address
- CUB BlockReduce eliminates the need for a separate atomic buffer and its cudaMemset reset between kernel launches, removing both memory allocation overhead and an extra API call per invocation
- When every thread in a block contributes exactly one partial sum, cooperative shared-memory reduction is always faster than global atomic contention — applies to any per-block histogram, counting, or summation pattern
