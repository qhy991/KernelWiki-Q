## Challenge

An epilogue kernel processes one fragment per loop iteration, issuing two __syncthreads() barriers per iteration (32 barriers for 16 iterations), causing cumulative barrier latency that dominates kernel runtime as the copy work itself is lightweight


## Solution

Coarsen the loop by processing FPP=2 fragments per iteration using partitioned shared memory (each fragment gets its own smem_offset region), halving the number of __syncthreads() barriers from 32 to 16 while keeping total copy work identical

## Key Lessons

- Loop coarsening that batches multiple fragments between __syncthreads() barriers reduces synchronization overhead proportionally — applies when shared memory can be partitioned to hold multiple fragments and the per-barrier work is lightweight relative to barrier latency
- Each __syncthreads() forces all threads in a block to converge; when a kernel has many short iterations the cumulative barrier cost dominates, so coarsening the loop to process multiple items per barrier phase is the key lever
