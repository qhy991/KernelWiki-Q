## Solution

Replace the single M-threshold with arithmetic intensity (AI = 2·M·N·K / (M·K + K·N + 2·M·N)) and route via a compound condition (m <= 32 || AI < 70), ensuring only truly memory-bound problems take the swap-A/B path while compute-bound problems stay on the higher-throughput default path

## Key Lessons

- Arithmetic intensity (FLOPs per byte transferred) is a more accurate routing metric than a single dimension threshold — it directly captures whether a problem is compute-bound or memory-bound, enabling correct kernel-path selection across arbitrary (M,N,K) shapes
- Routing/filtering kernels add negligible overhead when dominated by launch cost and sparse global memory writes — the real performance impact manifests downstream in the compute kernels they dispatch to, so microbenchmarking the routing kernel alone may show neutral results even when the optimization is beneficial
