## Solution

Promote VEC_SIZE from a hardcoded constexpr inside the kernel body to a template parameter, computing the optimal value at launch time via std::gcd(16/sizeof(half), hidden_size), enabling correct vectorization across different hidden sizes without sacrificing compile-time unrolling

## Key Lessons

- Promoting a constexpr inside a kernel to a template parameter yields identical PTX when the value matches — NVCC already treats inner constexpr as a compile-time constant, so the optimization is about generality not speedup
- Use std::gcd(memory_access_width / sizeof(element), hidden_size) to compute a safe vector size that adapts to any hidden dimension, preventing out-of-bounds access when hidden_size is not a multiple of the default vector width
- Template-parameter vector sizes enable the same #pragma unroll and reinterpret_cast optimizations as hardcoded constexpr while supporting multiple hidden sizes from a single compilation unit
