record_benchmark_artifacts fails with 'results.tsv is only written by state_core projection rebuild' when the agent tries to record results before the state_core projection has been rebuilt.

## Challenge

Agent calls record_benchmark_artifacts but gets error: 'results.tsv is only written by state_core projection rebuild.' The results.tsv file is generated by the state_core projection system, not by direct file writes.

### Symptoms

- `results.tsv is only written by state_core projection rebuild`
- `record_benchmark_artifacts fails`
- `Agent wastes steps trying to create results.tsv manually`

## Solution

Do not try to create results.tsv manually. Instead, ensure the verification pipeline completes successfully - the state_core projection will generate results.tsv automatically as part of the verification flow.

### Implementation

```cuda
1. Do NOT try to write results.tsv manually via mcp_filesystem_write_file
2. Do NOT try to create results.tsv via sandbox_execute
3. Run the canonical verification command first
4. After verification passes, the state_core projection rebuilds automatically
5. results.tsv will be generated by the projection system
6. If verification hasn't run, results.tsv won't exist - this is expected behavior
7. Focus on getting verification to pass, not on creating results.tsv
```

## Key Lessons

- results.tsv is generated by the state_core projection system, not by manual file writes.
- record_benchmark_artifacts requires the state_core projection to have been rebuilt.
- The projection rebuild happens automatically after successful verification.
- Do not try to create results.tsv manually - it will be overwritten anyway.
- Focus on getting verification to pass rather than trying to create output files.
