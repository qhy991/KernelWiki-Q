## Solution

Replace scalar float loads/stores with float4 (128-bit) vectorized access via cast_load/cast_store, processing 4 floats per memory transaction to achieve fully coalesced 128-bit wide accesses that match the GPU memory subsystem's native transaction granularity

## Key Lessons

- Use float4 (128-bit) vectorized loads/stores instead of scalar float accesses in bandwidth-bound inner loops — reduces memory transaction count by 4x and matches GPU memory subsystem's native 128-byte transaction granularity, directly translating to higher effective throughput
- Wrap vectorized types (float4/float2/half2) in a struct with cast_load/cast_store methods for clean, reusable vectorized access patterns that handle alignment and reinterpretation internally
