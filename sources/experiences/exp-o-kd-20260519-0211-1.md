## Solution

Dispatch between CausalMask<true> (IsQBegin=true) and CausalMask<false> (IsQBegin=false) based on the causal_q_begin flag, so that when Q is at the beginning of the sequence the kernel uses a tighter trip count calculation without offset_q, reducing K-tile visits per Q-row from 128 to 10 in the benchmarked case

## Key Lessons

- Use the correct template parameter for causal mask types based on Q position (IsQBegin=true vs false) rather than defaulting to one variant — CausalMask<true> computes max_blocks_q = ceil_div((blk_coord_q+1)*tile_q, tile_k) without offset_q, eliminating unnecessary K-tile visits when Q is at the sequence beginning
- Compile-time mask type dispatch (CausalMask<true> vs CausalMask<false>) can yield order-of-magnitude trip count reductions by letting the kernel skip tiles that the looser mask would conservatively include — always check whether the mask precision matches the actual data layout
