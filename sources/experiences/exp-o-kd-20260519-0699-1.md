## Challenge

The moe_align_block_size host function allocates tokens_cnts and cumsum buffers via cudaMalloc inside the function body and frees them with cudaFree after each kernel launch. cudaMalloc and cudaFree are synchronous CUDA runtime calls that force stream synchronization, blocking the host thread until the GPU finishes all prior work, creating per-call allocation overhead that dominates kernel execution time.


## Solution

Extend the function signature to accept pre-allocated torch.Tensor buffers (token_cnts_buffer, cumsum_buffer) from the caller, removing cudaMalloc/cudaFree calls entirely so buffers are allocated once and reused across all MoE inference iterations, eliminating per-call synchronous allocation overhead and stream synchronization stalls.

## Key Lessons

- Replace per-call cudaMalloc/cudaFree with caller-managed pre-allocated tensor buffers in hot-path functions called per batch — cudaMalloc/cudaFree are synchronous calls that force stream synchronization and add allocator overhead on every invocation, which dominates execution time when the kernel itself is fast
- Pre-allocate workspace buffers once at the Python/caller level and pass them as arguments for MoE inference loops — the same buffers can be reused across all iterations, turning per-call synchronous allocations into zero per-call allocations
