## Solution

on is insufficient to overlap meaningfully with the async copy latency; the dominant cost is the inner TILE_K multiply-add loop, not the global-to-shared copy. The 3x shared memory reduces SM occupancy and the cp.async commit/wait sync

## Key Lessons

- The after code uses multi-stage cp.async pipelining which requires 3x the shared memory of the before code (3 stage buffers vs 1: 3*(64*32+32*64)*4=48KB vs 16KB).
