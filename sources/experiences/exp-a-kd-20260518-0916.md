small_m_m2_n1024_k1536

## Solution

The after code swaps A/B to compute (N,K)x(K,M)=(1024,1536)x(1536,2), so the effective M dimension is now N=1024 which fills the 128-wide tile rows efficiently, while the effective N dimension is M=2 padded to just 16. This reduces total padded work from 64*1024*1536=100M to 1024*16*1536=25M MACs...

## Key Lessons

- In the before code, the 64x64 tile shape forces M=2 rows to be padded to 64, wasting 62 out of 64 rows per tile.
