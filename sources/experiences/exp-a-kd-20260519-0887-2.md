## Solution

Replace the shared-memory tree reduction with a warp-shuffle GroupReduceMax using __shfl_xor_sync across a 16-thread warp (mask=0xffff), eliminating all shared-memory usage and __syncthreads barriers for the reduction

## Key Lessons

- Warp-shuffle reductions (__shfl_xor_sync/__shfl_down_sync) eliminate shared-memory allocation, bank conflicts, and __syncthreads barriers compared to shared-memory tree reductions — applicable whenever the reduction fits within a single warp (≤32 threads)
- For small fixed-size groups (16 threads), hardcode the shuffle offsets (8,4,2,1) with a partial warp mask instead of using a generic loop — this allows full compiler unrolling and avoids branch divergence
