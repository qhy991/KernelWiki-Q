## Challenge

B weight tiles (sh_b) and the reduction output buffer (sh_red) are counted as separate shared memory allocations in the total, but they are never simultaneously live during pipelined Marlin MoE execution, wasting shared memory capacity


## Solution

Allocate B weight tiles and reduction buffer at the same shared memory offset using max(sh_b_size, sh_red_size), since only one is live at any pipeline stage, eliminating their combined allocation overhead

## Key Lessons

- Identify shared memory buffers with disjoint lifetimes in pipelined kernels and overlap them via max(size_a, size_b) instead of summing — applies whenever two buffers are used in different pipeline stages and never co-live
- Shared memory reduction from overlapping disjoint-lifetime buffers directly translates to higher SM occupancy (5→8 blocks/SM in this case), producing measurable speedup on shared-memory-bound configurations
