## Solution

Replace unscaled vec_conversion with scaled_vec_conversion that fuses scale multiplication into the fp8→half path: convert fp8 to half, promote to float for the multiply by a per-element scale factor, then convert back to half — eliminating the separate dequantization pass and reducing precision loss by scaling in float before downcasting

## Key Lessons

- Fuse per-element scale multiplication into the fp8→half conversion path (half→float→multiply→half) rather than performing dequantization as a separate kernel pass — the small arithmetic cost of float promotion trades favorably against a full extra memory round-trip and kernel launch, especially for bandwidth-bound KV-cache operations
- Scaling in float before downcasting to half reduces precision loss compared to applying a global scale separately to intermediate half values — the wider float mantissa preserves more significant bits during the multiply
