## Solution

Replace the redundant __shfl_sync warp shuffle with canonical_warp_idx(), which directly returns threadIdx.x / NumThreadsPerWarp without any warp-level synchronization, eliminating the shuffle instruction and allowing the compiler to schedule the computation independently of warp convergence

## Key Lessons

- Using __shfl_sync to broadcast a warp-uniform value (same result for all threads in the warp) is redundant — prefer a direct computation like canonical_warp_idx() which avoids shuffle synchronization overhead and improves instruction-level parallelism
- Removing unnecessary __shfl_sync calls allows forward progress independence between warps and is most impactful in kernels with many small tiles where per-invocation overhead is proportionally significant
