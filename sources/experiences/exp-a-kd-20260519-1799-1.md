## Solution

Fuse absmax reduction and quantization into a single kernel launch, storing the block-level absmax in shared memory via __syncthreads() barrier and computing the reciprocal scale (127.0f/absmax) for multiplication instead of division, eliminating the second kernel launch and the global memory write/read of scales

## Key Lessons

- Fuse multi-pass kernels that share per-block intermediate results (e.g., reduction outputs used as scaling factors) into a single kernel using shared memory and __syncthreads() — eliminates kernel launch overhead and global memory round-trips, especially beneficial when per-block work is small relative to launch cost
- Replace division by scale with multiplication by reciprocal (tmp_scale = 127.0f / absmax) inside fused kernels — multiplication is faster than division on GPU hardware and avoids re-reading the scale from global memory
- Kernel fusion yields the largest relative speedup when launch overhead dominates actual computation, such as small hidden sizes where each block does minimal work
