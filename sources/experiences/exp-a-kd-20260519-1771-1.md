## Challenge

The kernel uses cooperative shared-memory load of expert offsets, a __syncthreads() barrier, a serial lane-0 accumulation loop iterating up to num_local_experts times, and a __shfl_sync broadcast — serializing computation on one lane while 31 warp lanes sit idle, and stalling the entire threadblock at the barrier


## Solution

Pre-compute the aligned expert offset prefix-sum on the host side into a separate table, then each GPU thread independently reads its two needed values (aligned_base and token_offset) via __ldg — eliminating shared memory, __syncthreads barriers, serial lane-0 accumulation loops, and __shfl_sync broadcasts entirely

## Key Lessons

- Pre-compute small prefix-sum tables on the host (e.g., ≤9 elements for 8 MoE experts) instead of recomputing per-GPU-thread — host cost is negligible and eliminates shared memory, barriers, and serial accumulation on the device
- Per-thread __ldg reads from pre-computed tables replace cooperative shared-memory load + __syncthreads + serial accumulation when each thread needs only 1-2 values from a small lookup table — the read-only cache makes random access efficient
- Eliminating lane-0 serialization (31 idle warp lanes) and __syncthreads barriers yields significant speedup (2x) even for small expert counts, because barrier stall cost and warp divergence overhead dominate the actual computation
