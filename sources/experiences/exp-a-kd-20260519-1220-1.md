## Solution

Replace scalar loads with an alignment-aware vectorized read utility (vectorize_read_with_alignment_impl<__half, 8>) that attempts 16-byte vector loads and falls back to a three-phase scalar-prefix / vectorized-middle / scalar-tail path when the input length is not aligned to VEC_SIZE=8

## Key Lessons

- Alignment-aware vectorized read utilities (scalar-prefix + vectorized-middle + scalar-tail) add branching and instruction overhead that can negate bandwidth gains when the misaligned remainder is very small (1 element out of 4097) — the scalar fallback cost outweighs the vectorization benefit for near-aligned sizes
- Vectorization of memory-bound loops yields clear speedups only when the vectorized portion dominates the workload; when len % VEC_SIZE is tiny, the overhead of the alignment check and fallback paths produces neutral or negative results
- For hidden sizes that are almost aligned (e.g., 4097 vs VEC_SIZE=8), consider padding to alignment and masking the excess in the computation rather than adding runtime alignment logic to the load path
