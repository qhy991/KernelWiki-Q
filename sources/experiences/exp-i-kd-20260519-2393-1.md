## Solution

Map each thread to one output element via 2D index (threadIdx + blockIdx*blockDim), skip upper-triangle and out-of-bounds threads with i >= j check, accumulate the dot product A[i,k]*A[j,k] over K in column-major layout (A[row + k*lda]), then apply alpha/beta scaling in-place

## Key Lessons

- Exploit symmetry (i >= j guard) to skip the upper triangle entirely — for SYRK the output is symmetric so computing only the lower half halves total work without any correctness loss
- Column-major matrix access uses the pattern A[row + k*lda] where lda is the stride between rows — this matches BLAS convention and ensures coalesced reads when consecutive threads share the same k column
- For reference kernels, a one-thread-per-element 2D grid mapping (blockIdx.x*blockDim.x + threadIdx.x) is the simplest correct decomposition and provides a baseline for verifying optimized implementations
