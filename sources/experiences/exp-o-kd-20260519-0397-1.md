## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 pattern): each thread loads 4 half values per iteration via two half2 loads for both A and x, reducing loop iterations from 16 to 4 and doubling per-warp memory transaction width from 64 to 128 bytes for better L2 cache utilization

## Key Lessons

- Replace scalar half loads with half2 vectorized loads in GEMV/GEMV-style dot-product loops to reduce loop iteration count by 4x and widen per-warp memory transactions — applies whenever K is divisible by 4 and addresses are half-aligned
- Use reinterpret_cast<half2*>(&val)[0] (HALF2 macro) for type-punned half2 loads from half arrays — avoids separate pack intrinsics and lets the compiler emit a single 128-bit LDG instruction for two consecutive half2 loads
- Precompute NUM_WARPS as ceil(ceil(K/WARP_SIZE)/4) to match the f16x4 iteration stride, ensuring the vectorized loop covers exactly the same K range as the scalar version
