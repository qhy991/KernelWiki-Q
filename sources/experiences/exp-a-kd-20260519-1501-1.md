## Solution

Replace the custom blockReduceSum with cub::BlockReduce<float, 1024>, which internally uses the same warp-shuffle + shared-memory strategy for full-block reductions, producing nearly equivalent PTX while eliminating all hand-written reduction boilerplate

## Key Lessons

- cub::BlockReduce internally implements the same warp-shuffle + shared-memory two-phase reduction as hand-written blockReduceSum for full-block configurations — replacing custom reduction code with CUB yields equivalent performance while removing ~30 lines of boilerplate
- When all threads in the block participate in the reduction (blockDim.x == BLOCK_DIM template parameter), CUB and hand-written shuffle reductions compile to nearly identical PTX; any measured difference is within noise
- Prefer CUB primitives over hand-rolled reductions for maintainability — they handle edge cases (non-power-of-2 thread counts, partial blocks) correctly and auto-tune to the GPU architecture
