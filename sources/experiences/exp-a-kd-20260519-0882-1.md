## Challenge

MoE gate rescale uses two separate kernel launches — one for weight normalization (divide by sum) and another for routed_scaling_factor multiplication — causing each element to be written to global memory by the first kernel and re-read by the second, incurring redundant global memory traffic and double kernel launch overhead


## Solution

Fuse normalization and conditional scaling into a single kernel that reads each element once, applies division and optional routed_scaling_factor multiplication in the same loop iteration, and writes the final result once — eliminating the redundant global memory round-trip and one full kernel launch

## Key Lessons

- Fusing two elementwise kernels that operate on the same data into one kernel eliminates redundant global memory writes and reads — applicable whenever consecutive kernels read/write overlapping memory regions with no inter-kernel dependencies on the modified data
- Kernel launch overhead becomes proportionally more significant at larger row counts; fusing kernels that each launch thousands of blocks yields larger speedups because the per-launch dispatch cost is amortized away
- Adding a boolean flag parameter for conditional execution (apply_routed_scaling_factor_on_output) preserves flexibility while still gaining the fusion benefit, avoiding the need for separate kernel variants
