## Solution

Replace synchronous B-tile global-to-shared FLOAT4 stores with cp.async.ca.shared.global PTX instructions, issue and commit the async group before starting A-tile loads so the two transfers proceed concurrently, then wait for completion with CP_ASYNC_WAIT_GROUP(0) — this hides B-tile global memory latency behind A-tile loading and shared-memory transpose across all BK iterations

## Key Lessons

- Use cp.async.ca.shared.global to issue asynchronous global-to-shared copies for one tile (e.g., B) and commit the group before loading the other tile (e.g., A) synchronously — the async DMA overlaps with the synchronous load path, hiding global memory latency without additional hardware resources
- __cvta_generic_to_shared converts a generic shared-memory pointer to the special shared-memory address space required by cp.async PTX instructions — always use it when the destination is a shared-memory array element
- Issue CP_ASYNC_COMMIT_GROUP after all async copies for a tile and CP_ASYNC_WAIT_GROUP(0) after the dependent work completes (A load + transpose) to maximize overlap while guaranteeing data is ready before consumption
