## Challenge

Fixed CUTLASS 2.x int8 GEMM uses TileShape<128,128,64> regardless of problem dimensions, requiring more threadblocks to cover large M (512) and resulting in suboptimal L2 cache locality along the M dimension


## Solution

Replace fixed tile configuration with shape-aware selection: for large M (mp2>256) and N<=8192, use TileShape<256,128,64> which doubles M-dimension coverage per threadblock, halving threadblock count along M and improving L2 cache reuse

## Key Lessons

- Wider tile M-dimensions (256 vs 128) reduce threadblock count for large M problems, improving L2 cache locality — applies when M is large enough that the wider tile is fully utilized and does not leave trailing partial tiles
- Shape-aware tile selection based on next-power-of-2 of M and N dimensions enables the CUTLASS kernel to adapt threadblock mapping to problem geometry, avoiding one-size-fits-all configurations
