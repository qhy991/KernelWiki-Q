KernelBench level_1 task_8: Matrix multiplication C = A * B with irregular shapes (M=8205, K=2949, N=5921)

## Key Lessons

- Shared memory tiling provided 6% improvement (54.59s → 51.43s) for irregular shapes
- Larger tile size (32x32) performed worse due to reduced occupancy and synchronization overhead
- Vectorized loads (float4) caused misaligned address errors - need careful alignment handling
- PyTorch native matmul (likely cuBLAS) is 3.15x faster due to highly optimized libraries
- For irregular shapes, smaller tiles (16x16) work better than larger tiles (32x32)
