## Challenge

Activation shared memory buffer in get_kernel_cache_size always allocates 2 bytes per element regardless of actual activation precision, wasting 50% of sh_a_size when INT8 activations are used


## Solution

Add an is_a_8bit boolean parameter to get_kernel_cache_size and is_valid_config, conditionally sizing the activation shared memory buffer to 1 byte per element for INT8 activations versus 2 bytes for FP16, halving sh_a_size for INT8 workloads

## Key Lessons

- Conditionally size shared memory buffers based on actual data precision — sh_a_size *= (is_a_8bit ? 1 : 2) halves activation buffer footprint for INT8 workloads, freeing shared memory for larger tile sizes or more pipeline stages
- Use bool instead of int for boolean flags in function signatures (is_zp_float changed from int to bool) — improves type safety and communicates intent clearly
