## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Use cutlass::Array<T, N> data structure; Each thread handles one array element; Use .at(i) for element access

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Use cutlass::Array<T, N> data structure; Each thread handles one array element
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
