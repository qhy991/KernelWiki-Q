## Solution

Use vectorize_with_alignment<16> to process 16 float elements (64 bytes) per vectorized load and 16 int8 elements per vectorized store, handling alignment via prefix/tail scalar fallback, reducing total memory transactions in the quantization phase

## Key Lessons

- vectorize_with_alignment<16> handles misaligned pointers by computing a scalar prefix, then processing aligned vector chunks, then a scalar tail — provides vectorized memory access without requiring aligned allocation guarantees
- Vectorized loads alone cannot compensate for a 4x reduction in block parallelism (1024→256 threads); the bandwidth benefit of wider loads is overshadowed when too few warps are available to hide memory latency
- Guarding the inverse scale with a zero-check (absmax == 0.f ? 0.f : 127.f/absmax) prevents division by zero when a token is all-zeros, a correctness improvement over the unguarded version
