## Solution

Increase FragmentsPerPartition from 1 to 2, allocating double SMEM (1024 floats) partitioned into per-fragment tiles, so two fragments are processed per barrier phase — halving __syncthreads() count from 32 to 16 while improving SMEM access locality within each partition

## Key Lessons

- Increasing FragmentsPerPartition in SMEM-staged epilogue kernels reduces __syncthreads() barrier count proportionally (barriers = 2 * iters/FPP) — trading SMEM capacity for fewer barriers yields dramatic speedup when barrier overhead dominates, as seen with the 11.5x improvement from halving 32 barriers to 16
- Partitioning SMEM into per-fragment tiles (smem_off = p * smem_pointer_offset) eliminates cross-fragment SMEM reuse conflicts and improves bank access locality when multiple fragments are processed within a single barrier phase
