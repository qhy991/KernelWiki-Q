## Challenge

Flash attention baseline had V accumulation bottleneck: only 128/512 threads active (25% utilization). V loaded scalar from global memory.


## Solution

Cache V (and K) in half-precision shared memory using float4 coalesced loads with all 512 threads. Progressively increase KV_CHUNK: 32→64→128 tokens per sub-tile.

## Key Lessons

- Caching V in half-precision shared memory with float4 loads is the dominant optimization for flash attention with FP16 KV cache
- Larger KV_CHUNK (32→64→128) consistently improves performance by increasing smem reuse
- Reusing same smem buffer for K and V (load K, compute QK, then load V) saves smem and allows caching both
- Reducing BLOCK_SIZE from 512 to 256 hurts performance due to less QK parallelism even with better occupancy
- float4 loads (8 halves per transaction) maximize memory bandwidth for FP16 data
