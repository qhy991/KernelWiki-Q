## Challenge

Scalar element-by-element float loads and int8 stores in the absmax reduction and quantization loops waste memory bandwidth by issuing one 4-byte load per element instead of utilizing the full 16-byte memory transaction width available on GPU


## Solution

Use float4 (vec4_t_f) vectorized loads to process 4 elements per memory transaction for absmax computation and int8x4_t vectorized stores for quantization output, reducing the number of load/store instructions by 4x and improving memory throughput utilization

## Key Lessons

- Use reinterpret_cast to float4/vec4_t_f vectorized types for memory-bound loops processing contiguous float arrays — a single 16-byte load replaces four 4-byte loads, reducing load instruction count by 4x and improving memory throughput without changing the computation logic
- Pack int8 quantization outputs into int8x4_t structs for vectorized stores when writing contiguous quantized values — this coalesces four 1-byte stores into one 4-byte store, reducing store instruction pressure on memory-bound kernels
