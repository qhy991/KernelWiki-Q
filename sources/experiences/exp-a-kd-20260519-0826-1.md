## Solution

The attempted simplification stripped the Semaphore struct's alignas(128) padding in favor of a plain uint32_t array, but since __syncthreads() only synchronizes within a block and cannot prevent cross-SM L2 cache line invalidation, every exit() write by thread 0 invalidates the L2 cache line shared by 15 other blocks, accumulating false-sharing penalties across all 10,000 iterations and causing a 3.66x slowdown

## Key Lessons

- Use alignas(64) or alignas(128) padding on per-block signal/counter arrays to ensure each block's data occupies its own cache line — prevents false sharing when multiple SMs write adjacent indices in tight iteration loops
- __syncthreads() only synchronizes threads within a single block; it provides no protection against cross-SM L2 cache line invalidation, so false sharing between blocks can only be prevented by memory layout alignment, not by thread synchronization
- Simplifying struct wrappers to plain arrays can introduce severe regressions when the original struct's alignment was load-bearing for cache line isolation — always verify that struct simplification preserves the original memory stride assumptions
