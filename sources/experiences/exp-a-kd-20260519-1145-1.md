## Solution

Replace __hmul2 intrinsic with explicit PTX inline asm mul.f16x2 using uint32_t register constraints, and replace __float2half2_rn with cvt.rn.f16x2.f32 PTX asm, ensuring the compiler always emits the exact HMUL2 hardware instruction without intermediate moves

## Key Lessons

- Replacing __hmul2 with explicit PTX asm mul.f16x2 and uint32_t register constraints is a defensive optimization that ensures the HMUL2 instruction is always emitted directly — useful in template-heavy code where compiler type resolution may insert extra moves, though no measurable gain in simple microbenchmarks
- Using uint32_t instead of half2 as the register-level type for half2 pairs gives explicit control over register allocation and avoids implicit compiler-generated packing/unpacking instructions
