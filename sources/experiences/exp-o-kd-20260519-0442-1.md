## Solution

Pre-compute shared memory base pointers once outside the loop via __cvta_generic_to_shared on the base of s_a/s_b, then compute byte offsets arithmetically inside the loop: base_ptr + (stage * stage_offset + row * stride + col) * sizeof(float), eliminating repeated cvta calls

## Key Lessons

- For static shared memory, the NVCC compiler already folds repeated __cvta_generic_to_shared calls into compile-time constants, so pre-computing a base pointer and using manual byte-offset arithmetic yields no measurable improvement — the optimization is only worthwhile for dynamic shared memory where the base address is truly runtime-variable
- Manual byte-offset arithmetic (base_ptr + computed_offset * sizeof(T)) can replace 3D array indexing when dynamic shared memory is declared as a 1D array, but the compiler handles this equivalently for static shared memory declarations
