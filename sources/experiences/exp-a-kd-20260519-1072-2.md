## Challenge

The three-pass softmax structure (max-finding, exp+sum, output-write) re-reads each row element from global memory three times because scalar loads cannot retain data across passes — producing 3x the necessary global memory traffic for the read portion


## Solution

Load the row chunk into registers once (row_chunk[VPT]), then perform all three computation phases (max, exp+sum, normalize) on the register-resident data, eliminating 2 redundant global memory reads per element and reducing total traffic from 3 reads + 1 write to 1 read + 1 write

## Key Lessons

- When a kernel performs multiple passes over the same row data (e.g., max→sum→normalize in softmax), load the row into registers once and reuse across all phases — this eliminates (num_passes - 1) redundant global memory reads per element
- Register-resident multi-phase computation is most effective when VPT (values per thread) is small enough that row_chunk[VPT] fits entirely in registers without spilling — typically VPT ≤ 8 for fp32
