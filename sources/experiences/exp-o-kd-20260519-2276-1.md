## Solution

Pre-compute source and destination base pointers once per token to eliminate per-element div/mod arithmetic, then delegate the copy to a generic vectorize_with_alignment<8> helper that issues 128-bit (8×half) vector loads and stores with automatic misalignment handling via prefix-scalar/vector-body/tail-scalar decomposition

## Key Lessons

- Pre-compute source and destination base pointers once before the element loop when the layout makes all elements contiguous — eliminates per-iteration integer div/mod overhead and enables vectorized access in a single transformation
- Use 128-bit vector loads/stores (8×half) instead of scalar 2-byte accesses for contiguous memory copies in KV-cache kernels — reduces memory transaction count by 8× and dramatically improves bandwidth utilization (1.67× speedup observed)
- The vectorize_with_alignment pattern (aligned fast path with prefix-scalar/vector-body/tail-scalar fallback) handles arbitrary pointer alignment robustly while still benefiting from vectorized access on the aligned majority of elements
