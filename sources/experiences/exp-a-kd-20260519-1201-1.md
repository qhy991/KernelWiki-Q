## Challenge

Marlin-style quantized GEMM kernel always uses m_block_size=16 for shared memory A-tiles and reduction buffers, even when prob_m <= 8 (single-token inference), wasting ~22% shared memory and limiting SM occupancy to 5 blocks


## Solution

Introduce m_block_size=8 specialization when prob_m <= 8 and thread_m_blocks == 1, halving the A-tile shared memory stage from 256 to 128 int4s and halving the reduction buffer, reducing total shared memory from 28928B to 22656B (~22%) and increasing theoretical occupancy from 5 to 7 blocks/SM

## Key Lessons

- Halve m_block_size (16→8) when prob_m <= m_block_size/2 in Marlin-style quantized GEMM — reduces total shared memory by ~22% and raises occupancy (5→7 blocks/SM) because both A-tiles and reduction buffer scale linearly with m_block_size
- Simplified microbenchmarks that don't exercise the actual async pipeline (cp_async, barrier acquire/release) may show neutral latency even when shared memory savings improve occupancy — validate occupancy-sensitive optimizations with the real kernel path that is shared-memory-bound
