## Solution

Compute a reordering atom (LayoutAtomQuant) from the WGMMA thread-value layout via compute_memory_reordering_atom<MmaType>(), tile it across the full tensor to produce LayoutB_Reordered, and physically reorder quantized weight data in shared memory so all 16 INT4 elements per thread are contiguous — enabling a single 64-bit vectorized load instead of multiple scattered 16-bit loads

## Key Lessons

- Derive a memory layout from the MMA thread-value layout via compute_memory_reordering_atom and physically reorder narrow-type (INT4) tensors in shared memory — this coalesces scattered sub-bank accesses into full 64-bit vectorized loads, most impactful when the MMA operates on a wider type (FP8) than the stored element type (INT4)
- In CUTLASS CollectiveBuilder, swapping the layout argument (e.g., LayoutB_Transpose → LayoutB_Reordered) is sufficient to propagate the reordering through the entire GEMM pipeline without modifying the MMA or epilogue logic
