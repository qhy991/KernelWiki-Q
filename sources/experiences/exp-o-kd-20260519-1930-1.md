## Solution

Replace the 2D batch-split grid with a 1D flat warp-per-token model where flat_warp_id = (blockIdx.x * blockDim.x + threadIdx.x) >> 5 assigns each warp to exactly one output token, using binary search on workspace_starts to map the flat token index to its owning request, so all 32 lanes cooperatively process one token without divergence or inactive splits

## Key Lessons

- Flat 1D warp-per-token grids eliminate load-balance waste from 2D batch-split grids where short requests leave entire splits idle — every launched warp does exactly one token of useful work regardless of request length distribution
- Binary search on a prefix-sum array (workspace_starts) enables O(log n) per-warp request ownership lookup in flat grids, replacing the O(1) blockIdx.x mapping lost when collapsing the 2D grid to 1D
- Assigning one warp per token instead of one thread per token enables cooperative vectorized loads across all 32 lanes, removing the serial token loop and tid-based if/else branch divergence within a block
