## Solution

Increase kStage from 1 to 2, enabling the async multi-stage pipeline to double-buffer K tiles: the next K tile is prefetched into smem_sel_next via cp.async while the current iteration's MMA proceeds, and smem_sel/smem_sel_next slot alternation prevents read/write conflicts on the K shared memory buffer across iterations

## Key Lessons

- Increasing kStage from 1 to 2 in tiled flash attention enables double-buffered async K-tile prefetch via cp.async, hiding global memory latency behind MMA computation — effective when the KV loop has many iterations (Tc≥32) to amortize the pipeline setup overhead and the extra shared memory for the second K buffer fits within smem limits
- smem_sel/smem_sel_next slot alternation prevents read/write hazards on double-buffered shared memory without requiring explicit barriers between the async copy and the consuming warp, keeping the pipeline latency-free across consecutive iterations
- CP_ASYNC_WAIT_GROUP(1) before P@V (when more tiles remain) waits only for the V tile, not the next K prefetch, preserving overlap between V load and ongoing computation in the next iteration
