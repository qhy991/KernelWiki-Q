## Solution

Added a k_size template parameter (defaulting to 16) with a k_size==32 branch that uses m16n8k32 MMA inline PTX instructions for FP8 (e4m3.e4m3.f32) and int8 (s8.s8.s32.satfinite), processing twice the K-dimension per call and halving the number of MMA instructions needed for 8-bit GEMM workloads.

## Key Lessons

- Use m16n8k32 MMA instructions instead of two sequential m16n8k16 calls for FP8/int8 GEMM workloads when K-dimension is a multiple of 32 — the k32 variant processes twice the K-dimension per instruction, halving MMA count and removing one warp-synchronization barrier per iteration, yielding ~1.3x speedup
- Template the MMA function with a k_size compile-time parameter to select between m16n8k16 and m16n8k32 PTX variants, enabling the compiler to eliminate dead branches and generate optimal code for each precision type at compile time
- For int8 tensor core MMA, use .satfinite accumulator saturation (mma.sync.aligned.m16n8k32.row.col.s32.s8.s8.s32.satfinite) to prevent overflow, and accumulate into int32_t registers rather than float
