## Challenge

Split QKNorm (RMSNorm+weight) and RoPE are implemented as two separate kernels with an intermediate global-memory buffer, forcing ~16MB of data (1024 tokens x 32 heads x 128 dims x 4 bytes) to be written out by the first kernel and read back by the second, creating a memory-bandwidth bottleneck


## Solution

Fuse QKNorm and RoPE into a single kernel where the RMSNorm+weight normalized values stay in the thread-local register array e[EPT] and RoPE rotation is applied directly on those register-held values before a single final write to output, eliminating the intermediate global-memory buffer and one kernel launch overhead

## Key Lessons

- Fuse sequential kernel passes where the first pass produces per-element intermediate results that the second pass consumes element-wise — keeping values in registers eliminates one global-memory write+read round-trip, yielding up to ~2x reduction in memory traffic for memory-bound workloads
- Kernel fusion saves one launch overhead (~5-10 microseconds per invocation), which becomes significant for small-to-medium workloads where launch latency is a non-negligible fraction of total runtime
- Thread-local register arrays (e.g., float e[EPT] where EPT=4) are sufficient to hold intermediate results between computation phases within a fused kernel, avoiding shared or global memory staging
