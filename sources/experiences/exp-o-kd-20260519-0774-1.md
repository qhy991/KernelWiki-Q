## Solution

Replace contiguous-block iteration (threadIdx.x * tokens_per_thread start with sequential range) with stride-based iteration (for i = tid; i < numel; i += stride) so that adjacent threads access adjacent memory addresses, enabling coalesced global memory loads and eliminating the CEILDIV division and compound bounds-check

## Key Lessons

- Prefer stride-based iteration (i = tid; i < numel; i += blockDim.x) over contiguous-block iteration (i = tid * chunk; i < tid * chunk + chunk) for data-parallel loops — adjacent threads access adjacent addresses, enabling memory coalescing on GPU
- Stride-based loops eliminate the CEILDIV division for computing per-thread chunk sizes and simplify bounds-checking from compound conditions (i < numel && i < start + chunk) to a single comparison (i < numel), reducing loop overhead
