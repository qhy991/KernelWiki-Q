## Solution

Compute base_offset once at the (n,d0,h0,w0,fc) corner using ndhwc_offset, then precompute three byte increments (inc_next_s, inc_next_r, inc_next_t) that encode stride*dilation arithmetic, advancing to (n,d0+ft*dilation_d,h0+fr*dilation_h,w0+fs*dilation_w,fc) via three integer multiply-adds instead of four int64 multiplications inside ndhwc_offset

## Key Lessons

- Precompute byte-offset increments for filter position traversal (inc_next_s/r/t) outside the hot loop instead of recomputing full tensor offsets on every access — reduces per-element address computation from ~12 multiply/adds to ~3, applicable to any convolution or stencil with fixed filter dimensions
- Separate the validity bounds check from the address computation by computing base coordinates (dd,hh,ww) without filter offset, then adding dilation terms only for the bounds check — this avoids recomputing the base offset for each filter position
