## Solution

Pre-compute sh_ids[i] / top_k once into a second shared-memory array (sh_precomputed) during the initialization phase, adding one __syncthreads() barrier, then replace the expensive division in the hot loop with a simple shared-memory read — eliminating ~800 integer divisions per thread at the cost of one extra barrier (~100-200 cycles) and doubled shared memory

## Key Lessons

- Pre-compute loop-invariant integer divisions (especially by non-power-of-2 divisors) into shared memory once during initialization rather than repeating them in hot loops — GPU integer division requires multiple ALU cycles with no shift shortcut, so amortizing N divisions into 1+N reads pays off when the loop iteration count is large
- When adding pre-computation shared-memory arrays, split a single extern __shared__ buffer via pointer arithmetic (sh_ids = sh_mem; sh_precomputed = sh_mem + num_rows) to avoid requiring separate shared-memory allocations
- One extra __syncthreads() barrier for pre-computation costs ~100-200 cycles but is negligible compared to eliminating hundreds of integer-division operations — the tradeoff is favorable when repeat count * rows-per-thread >> barrier cost
