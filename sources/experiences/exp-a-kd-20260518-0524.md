## Solution

Replace global singleton handle with an external handle parameter passed as function argument, eliminating per-call null check and static state; handle creation moves to the caller (main) — this is a software-engineering refactor with no GPU-level optimization mechanism

## Key Lessons

- Host-side null-pointer branch on a global cuBLAS handle is amortized to zero by GPU kernel time — branch prediction makes the check effectively free after warmup, so removing it yields no measurable speedup
- External handle parameter is better software engineering (testability, no global state, explicit ownership) but does not constitute a GPU-level optimization — do not expect performance gains from API ergonomic refactors
- When evaluating cuBLAS GEMM refactors, measure at problem sizes where the GPU kernel dominates (here 512x512x512 takes ~7us per call); host-side overhead savings of ~30ns are invisible at this scale
