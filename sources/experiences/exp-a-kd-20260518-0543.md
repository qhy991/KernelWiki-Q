## Solution

Replace scalar half loads with half2 vectorized (f16x4) loads: each thread loads 4 half values per iteration via two half2 registers, reducing loop iterations from 16 to 4 and widening warp-level memory transactions from 64 to 128 bytes for better L2 cache utilization

## Key Lessons

- Vectorize half-precision GEMV loads from scalar half to half2 (f16x4) to reduce loop iteration count by 4x and widen warp-level memory transactions from 64 to 128 bytes — applies whenever the K dimension is a multiple of 4 and memory bandwidth dominates
- Reducing per-thread loop iterations in memory-bound kernels eliminates per-iteration overhead (address arithmetic, branch, loop counter) that is amplified across thousands of threads — particularly effective when the reduction in iterations is large (e.g., 16→4)
- Wider vector loads improve L2 cache utilization by consolidating multiple narrow transactions into fewer wide ones, reducing cache line conflicts and improving effective bandwidth for stride-1 access patterns
