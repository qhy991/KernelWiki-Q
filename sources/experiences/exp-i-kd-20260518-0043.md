## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: 1D grid and block configuration; Thread ID: tid = threadIdx.x + blockIdx.x * blockDim.x; Bounds check: if (tid < N)

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: 1D grid and block configuration; Thread ID: tid = threadIdx.x + blockIdx.x * blockDim.x
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
