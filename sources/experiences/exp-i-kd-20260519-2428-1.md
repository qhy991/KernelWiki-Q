## Solution

Use if-constexpr dispatch based on Mode to route to specialized __device__ functions; each uses cooperative_copy<ThreadBlockSize, MaxVecBits> with AutoCopyAsync for hardware-accelerated async transfers, cp_async_fence/cp_async_wait/__syncthreads barriers for ordering, and shared memory staging for global-to-shared and shared-to-shared modes

## Key Lessons

- Use cooperative_copy<ThreadBlockSize, MaxVecBits> with AutoCopyAsync{} to leverage Ampere cp.async hardware — automatically generates optimal async memory copy instructions based on vectorization width and thread count, eliminating manual LDG/STS pipelining
- Apply cp_async_fence() + cp_async_wait<0>() + __syncthreads() after async copies to shared memory — the fence commits pending async operations, wait<0> blocks until all complete, and syncthreads ensures all threads observe the data before consumption
- Align shared memory pointers to MaxVecBits/8 bytes offset from the extern __shared__ base — ensures vectorized memory access works correctly on hardware that requires alignment proportional to access width
- Use if-constexpr with std::is_same_v for compile-time mode dispatch in template kernels — zero overhead branching that resolves at compile time, ideal for multi-variant cooperative copy
