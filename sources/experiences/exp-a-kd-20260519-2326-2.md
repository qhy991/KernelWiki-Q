## Challenge

The aligned cumulative sum is computed by a single thread (threadIdx.x == 0) iterating serially over all num_experts entries performing DIV_CEIL alignment and accumulation, creating an O(N) bottleneck that scales linearly with expert count and leaves all other threads idle


## Solution

Replace the serial single-thread cumsum loop with CUB BlockScan ExclusiveSum, where each thread computes its own expert's aligned count (DIV_CEIL(count, block_size) × block_size) and the parallel scan produces all prefix sums in O(log N) steps using warp-level shuffle primitives

## Key Lessons

- CUB BlockScan::ExclusiveSum replaces serial O(N) prefix sum loops with O(log N) parallel scan using warp-level primitives — applies whenever a single-thread cumsum bottleneck exists over dozens to hundreds of elements
- Fusing the DIV_CEIL alignment into the per-thread input to BlockScan (aligned_count) eliminates the need for a separate alignment pass, combining alignment and prefix sum into a single collective operation
