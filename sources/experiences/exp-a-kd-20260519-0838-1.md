## Solution

Replace serial global-memory scan with cooperative shared-memory load (moe_block_size/4 threads each load one int4 concurrently into shared memory) followed by warp-level parallel counting where the last 32 threads each count valid tokens in their slice of shared-memory data and reduce via __shfl_down_sync, eliminating the sequential dependency chain and reducing per-element access latency

## Key Lessons

- Replace serial global-memory scan-and-break patterns with cooperative shared-memory load + warp-level parallel counting when data per thread block is small enough to fit in shared memory — eliminates sequential dependency chains and reduces per-element access latency from global-memory to shared-memory levels
- Use __shfl_down_sync warp shuffle reduction (5 stages for 32-lane warp) to reduce O(N) serial counting to O(log N) parallel reduction, applied here to valid-token counting in MOE blocks
- Dedicate a subset of threads (e.g., last warp) for parallel computation on shared-memory data when the data set is small — avoids wasting all 256 threads on a 16-element problem while keeping the cooperative load parallelism
