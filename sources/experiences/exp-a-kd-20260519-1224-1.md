## Solution

Pass the scalar as a device pointer (const float* d_scale) so the kernel reads *d_scale directly from GPU global memory, eliminating the synchronous cudaMemcpy device-to-host transfer entirely and allowing all kernel launches to proceed asynchronously without host-side synchronization barriers

## Key Lessons

- Pass scalar parameters as device pointers instead of using cudaMemcpy device-to-host to read them back — the per-iteration synchronous transfer and implicit stream synchronize dominate runtime when kernel execution is fast relative to the D2H transfer latency
- Keeping scalars on the GPU and dereferencing them inside the kernel (*d_scale) preserves the asynchronous execution model, enabling the CUDA driver to queue back-to-back kernel launches without host-side stalls
- cudaMemcpy(DeviceToHost) on even a single 4-byte scalar forces an implicit cudaStreamSynchronize — the overhead is dominated by synchronization latency, not transfer size, so this pattern is expensive regardless of payload size
