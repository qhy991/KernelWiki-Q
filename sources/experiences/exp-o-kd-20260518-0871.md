## Key Lessons

- Since all stride values are identical across experts (they represent the row/column stride of the original tensor, which is the same for every expert in the group), this resulted in 3 redundant GPU kernel launches that each wrote the same constant to every element of a small tensor.
- Each torch::full() launch incurs CUDA kernel launch overhead (stream synchronization cost, driver dispatch) for what is essentially a trivial fill operation that could be done alongside other per-expert initialization work.
