## Solution

Replace all 12 asm volatile prmt.b32 calls with the __byte_perm CUDA intrinsic, which maps to the same PTX instruction but grants NVCC freedom to reorder and combine permutations for improved instruction scheduling

## Key Lessons

- Replacing asm volatile with the equivalent CUDA intrinsic (__byte_perm) is a valid portability improvement but does not improve performance when the kernel is memory-bound — each thread reads/writes 8 ints (32 bytes) but performs only 12 permutation instructions, making compute negligible and warp-level interleaving already sufficient to hide instruction latency
- When a kernel's compute-to-memory ratio is very low (12 instructions vs 32 bytes I/O per thread), compiler instruction scheduling optimizations have negligible impact because the warp scheduler hides compute latency via memory-latency interleaving across warps
- A ~2% measured delta in GPU microbenchmarks is within typical noise bounds and should not be interpreted as a real performance difference — prefer intrinsic-based code for portability regardless of such noise
