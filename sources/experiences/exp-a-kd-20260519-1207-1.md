## Solution

Replace the hardcoded ldmatrix.x4 load with a templated ldsm_load<count> that selects ldmatrix.m8n8.x2 when only 2 A-fragment rows are needed (m_block_size=8 scenario), cutting shared memory bandwidth per load by 50% and freeing 2 registers per thread per pipeline stage

## Key Lessons

- Match ldmatrix.xN width to actual data requirements rather than always using x4 — ldmatrix.x2 halves shared memory read bandwidth and reduces register pressure by 2 registers per thread per pipeline stage when only 2 A-fragment rows are needed (m_block_size=8 scenario)
- Template the ldmatrix load count as a compile-time parameter so the correct PTX variant (x1/x2/x4) is selected without runtime branching — if constexpr ensures zero overhead
