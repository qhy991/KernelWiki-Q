## Solution

Replace scalar-by-value k_scale/v_scale parameters with const float* pointers, dereferencing them once per block (*k_scale, *v_scale) to read from global memory; the L2 cache serves the single 4-byte load across all threads, and the pointer indirection enables per-invocation dynamic scale values without changing the kernel signature

## Key Lessons

- Passing scalar constants as pointer arguments (const float* with *dereference) instead of by-value parameters enables dynamic per-invocation values without kernel recompilation, at negligible performance cost since the single L2-cached 4-byte load is amortized across all threads in a block
- When a kernel's dominant cost is in its main compute loop (here: 1024 FP8 conversions + memory ops per block), the parameter-passing mechanism for a few scalar constants has negligible impact on overall execution time — focus optimization effort on the loop body instead
