## Challenge

SiLU+mul activation and per-block dynamic quantization are implemented as two separate kernels, forcing a 64MB FP16 intermediate tensor round-trip through global memory between the activation kernel and the quantization kernel, creating a memory bandwidth bottleneck in this memory-bound pipeline


## Solution

Fuse SiLU+mul and per-block quantization into a single kernel that computes the activation in registers, performs shared-memory absmax reduction, broadcasts the per-group scale, and quantizes — eliminating the intermediate FP16 global memory write/read entirely

## Key Lessons

- Fuse activation kernels with their downstream consumers (quantization, reduction) into a single kernel to eliminate intermediate global memory round-trips — applies whenever the activation output feeds directly into a per-element or per-group operation and the intermediate tensor is never reused elsewhere
- Keep intermediate activation results in registers across computation phases (SiLU → absmax reduction → quantize) instead of writing to global memory and reading back — eliminates O(hidden_size * num_tokens) bytes of GM traffic per activation pass
- Per-block dynamic quantization can be naturally merged with upstream elementwise kernels by reusing the same thread-to-element mapping: each thread computes its activation value, participates in shared-memory absmax reduction, then quantizes using the broadcast scale
