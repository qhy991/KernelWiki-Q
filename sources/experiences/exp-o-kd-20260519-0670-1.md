## Solution

Fuse the normalize and scale operations into a single rescale_fused_kernel so each element is read once, divided by its row sum, and multiplied by the scaling factor in the same loop iteration, eliminating the second kernel launch and the redundant global memory load-store cycle

## Key Lessons

- Fuse sequential elementwise kernels that operate on the same data into a single kernel launch — eliminates kernel dispatch overhead and redundant global memory round-trips, yielding significant speedup even for small problem sizes
- Use a boolean flag parameter (e.g., apply_routed_scaling_factor_on_output) to conditionally execute the second phase within a fused kernel, preserving the ability to skip the scale step when not needed without reverting to a separate kernel launch
