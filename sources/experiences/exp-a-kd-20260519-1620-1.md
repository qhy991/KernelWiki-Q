## Solution

Swap GEMM operands to compute C^T(NxM) = B^T(NxK) x A^T(KxM) with TILE_N_SWAP=128 and TILE_M_SWAP=8, making the small M dimension fully utilized as the column dimension of the transposed problem; however on A800/sm_80 the larger ~8.7KB shared memory reduces per-SM occupancy, 32 vs 64 blocks underutilize the GPU, and the transposed B^T tile layout conflicts with HMMA's preferred row-major A / column-major B fragment convention, producing a 0.668x regression

## Key Lessons

- Swap-AB GEMM (reformulating C=A×B as C^T=B^T×A^T to improve tile fit for small dimensions) is architecture-dependent: beneficial on SM100/Blackwell where LayoutB_T aligns with tensor core fragment preferences, but regresses on SM80/Ampere where HMMA expects row-major A and column-major B — always benchmark on target hardware before applying
- Increasing tile dimensions to improve per-block thread utilization can backfire when the larger shared memory footprint (~5KB → ~8.7KB here) reduces per-SM occupancy enough to outweigh utilization gains — model occupancy alongside thread efficiency
- Halving total thread blocks (64 → 32) can leave SMs starved for work even when per-block utilization reaches 100%, especially on GPUs with many SMs — total parallel coverage matters as much as per-block efficiency
