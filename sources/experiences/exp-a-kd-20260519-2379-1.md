## Solution

Replace per-thread global memory rows with warp-grouped atomicAdd into a compact extern __shared__ array (1KB vs 256KB): each warp owns a contiguous slice of expert slots, all threads atomically increment counts in shared memory, eliminating the entire prefix-sum merge phase and its two __syncthreads() barriers

## Key Lessons

- Replace per-thread global memory rows with warp-grouped atomicAdd into shared memory when performing histogram/counting kernels — eliminates the O(threads × bins) global memory footprint and the serial prefix-sum merge phase, yielding orders-of-magnitude speedup when the per-thread row approach overflows shared memory and spills to global memory
- Shared memory atomics (atomicAdd on __shared__ arrays) are fast on modern GPUs and allow all threads to aggregate counts into a single flat array without per-thread privatization — use this pattern whenever the count domain (e.g., number of expert bins) fits in shared memory
- Pad the expert count to a multiple of WARP_SIZE and assign experts_per_warp = 32 so each warp owns exactly one aligned chunk — this avoids bank conflicts and simplifies the warp_idx/expert_offset address calculation to integer division and modulo by a power of two
