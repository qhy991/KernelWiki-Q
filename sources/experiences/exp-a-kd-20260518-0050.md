## Challenge

The before code uses ld.global.cg.v4.f32 (cache-all) while the after code uses ld.global.cs.v4.f32 (streaming) for 128-bit vector loads over a 512MB working set. Both perform the same copy operation with the


## Solution

Replace ld.global.cg with ld.global.cs (cache streaming) hint to minimize L2 cache retention for 128-bit loads that have no temporal locality, following the CUTLASS streaming-load pattern

## Results

- **speedup**: 0.985x (neutral)


## Key Lessons

- Switching from .cg to .cs cache hints provides no benefit in isolated single-kernel streaming benchmarks — the streaming hint only helps when concurrent L2-resident data would be evicted by cache-all loads
- Cache hint selection is a workload-level optimization, not a kernel-level one — validate with realistic multi-kernel scenarios that create actual L2 contention before committing to .cs
