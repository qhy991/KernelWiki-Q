## Challenge

Variance accumulation in rms_norm_kernel uses scalar strided iteration loading one __half element per global memory transaction, resulting in 4096 scalar loads per token block and severe underutilization of available 128-bit memory bandwidth


## Solution

Replace the scalar strided loop with vectorize_read_with_alignment<8> which performs aligned 128-bit vector loads (8 __half elements per transaction), reducing memory transaction count by 8x for aligned rows while falling back to scalar + vector + tail for misaligned addresses

## Key Lessons

- Use vectorized loads (VEC_SIZE=8 for __half → 128-bit) instead of scalar strided loops in variance accumulation to reduce global memory transaction count by the vector width — applies whenever the array length and base address satisfy alignment constraints
- The vectorize_read_with_alignment helper handles misalignment transparently via prefix-scalar + aligned-vector + tail-scalar decomposition — safe to use without manual alignment guarantees on the input pointer
- Vectorizing only one phase of a multi-phase kernel (here: variance but not output normalization) yields a dampened overall speedup (1.122x vs potential per-phase improvement) — both phases should be vectorized for maximum gain
