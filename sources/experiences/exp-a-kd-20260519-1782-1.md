## Solution

Swap-AB reformulation transposes both operands and reverses multiplication order, which changes internal kernel dispatch logic (from next_pow_2 heuristic to direct M comparison on SM90) and requires reversed scale argument order (b_scales before a_scales) for correct transposed epilogue handling

## Key Lessons

- Swap-AB GEMM formulation does not automatically improve performance for small-M problems (M=32) — the observed 2.8% regression is within measurement noise, and the kernel dispatch difference (next_pow_2 vs direct M comparison) only affects SM90, not SM80
- When applying swap-AB transformation in quantized GEMM pipelines, scale argument order must be reversed (b_scales, a_scales) as a correctness requirement for the transposed epilogue — this is not a performance optimization but a necessary correction
