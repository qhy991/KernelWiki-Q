## Solution

Replace per-warp subset scanning with one-thread-per-expert: each thread reads its own expert from shared_scores[tid] directly (1 read vs ~32 strided reads), then participates in warp-level shuffle reduction with #pragma unroll, eliminating the inner loop overhead entirely

## Key Lessons

- Replace per-warp subset scanning with one-thread-per-expert direct access when thread count >= element count — eliminates strided shared memory reads and loop overhead, reducing reads from ~N/WARPS_PER_TOKEN per warp to 1 per thread
- Add #pragma unroll to warp-level shuffle reduction loops — the compiler can fully unroll the fixed-iteration reduction (5 iterations for warpSize=32), improving instruction scheduling and reducing branch overhead
