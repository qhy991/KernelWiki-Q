## Solution

Add two new fields to AllReduceParams: tmp_result_buffers[MAX_RANKS_PER_NODE] providing per-rank uint32_t* scratch space for the first-half reduction result so the kernel avoids overwriting the shared communication buffer, and is_capturing boolean flag set by the host via cudaStreamIsCapturing to select the COPY_INPUT=false path that skips the illegal in-kernel copy during graph capture.

## Key Lessons

- CUDA graph capture prohibits cudaMemcpyAsync on captured streams — provide a per-rank tmp_result_buffers scratch space and an is_capturing host-set flag so the kernel can skip illegal host-side copies and store intermediate two-shot reduction results without clobbering the shared communication buffer
- Typing struct fields precisely (uint32_t* instead of void* for buffer pointers, RankData* for indirection pointers) enables the compiler to generate address-arithmetic-aware code and makes graph-capture buffer resolution via RankData indirection explicit
