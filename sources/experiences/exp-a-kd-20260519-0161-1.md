## Solution

Insert cudaStreamWaitEvent between consecutive kernel launches using cudaEventDisableTiming events to simulate the device-side griddepcontrol.wait ordering that Programmatic Dependent Launch (PDL) provides on SM90a, aiming to reduce the inter-kernel scheduling gap from microseconds to near-zero

## Key Lessons

- On the default stream, kernels already execute serially — adding cudaStreamWaitEvent provides no scheduling advantage and introduces slight API-call overhead, yielding neutral or negative results on hardware without PDL support
- Programmatic Dependent Launch (PDL) with griddepcontrol.wait on SM90a is a hardware-level mechanism that reduces inter-kernel gap to near-zero via a single PTX instruction, but this benefit cannot be replicated on earlier architectures (SM80) using software event-based synchronization
- Use cudaEventDisableTiming when creating events purely for ordering/synchronization (not profiling) to avoid unnecessary hardware counter overhead
