## Solution

Lower the dispatch clamp from min 32 to min 16 so that M=1 maps to the TILE_M=16 kernel instead of TILE_M=32, matching the smallest available tile config and avoiding unnecessary template instantiation with oversized row iteration bounds

## Key Lessons

- When dispatching templated kernels via power-of-2 tile sizes, the clamp minimum must cover the smallest tile config available — clamping to min 32 when a TILE_M=16 path exists causes small-M cases to skip the optimal smaller tile entirely
- In scalar-loop GEMM kernels where the inner row loop early-returns on row>=M, changing TILE_M alone does not affect the dominant K-dimension dot-product cost — real gains from tile-size tuning appear in tiled shared-memory kernels (e.g., Cutlass 2.x) where TileShape controls shared memory footprint and warp scheduling
