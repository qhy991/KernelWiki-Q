## Challenge

Token counting uses only 128 threads with per-thread serial accumulation into a large (num_thread+1)×num_experts 2D shared memory array, where each thread iterates ~12 elements — underutilizing GPU parallelism and wasting shared memory on the multi-row counting structure


## Solution

Replace per-thread 2D array accumulation with 1024 threads all using atomicAdd on a flat shared_counts array of size padded_num_experts, reducing per-thread iterations from ~12 to ~1.5 while eliminating the large (num_thread+1)×num_experts shared memory allocation

## Key Lessons

- Use atomicAdd on shared memory with many threads instead of per-thread private 2D arrays for histogram-style counting — reduced per-thread iteration count (8× more threads) outweighs atomicAdd serialization when contention is low and element count is modest
- Replacing a (num_threads+1)×num_experts 2D counting array with a flat padded_num_experts array cuts shared memory usage dramatically, freeing space for other purposes like CUB BlockScan temp storage
