## Solution

Add a kMaxK template parameter and compute constexpr bool kSingleIteration = kMaxK <= TileN at compile time, so the compiler replaces the loop bound with the literal 1 and folds isLastColumn to true, eliminating the loop overhead and branch entirely

## Key Lessons

- When a tiled kernel's dimension (head_dim, K) is known at compile time and fits in one tile, promote it to a template parameter and use constexpr bool to collapse the loop to a single iteration — the compiler eliminates the loop branch, the iteration counter, and the last-tile guard, yielding significant speedup in tight inner loops
- Use ternary expressions on constexpr bools (kSingleIteration ? 1 : runtime_val) rather than #ifdef — the compiler constant-folds the dead branch while keeping the generic path readable and maintainable
