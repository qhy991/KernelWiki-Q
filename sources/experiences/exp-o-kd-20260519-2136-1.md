## Challenge

On Ada Lovelace (SM89) GPUs, cutlass_scaled_mm_supports_fp8() unconditionally returned false, blocking FP8 CUTLASS kernel dispatch even when CUDA 12.4+ was available and forcing FP8 workloads to fall back to slower torch.mm paths


## Solution

Replace the unconditional return false with return CUDA_VERSION >= 12040, enabling FP8 CUTLASS kernel dispatch on SM89 when the CUDA toolkit version supports the tuned kernels — Ada Lovelace hardware has FP8 tensor cores but they were gated off by an outdated guard

## Key Lessons

- When a capability gate unconditionally disables a code path with a TODO comment indicating future enablement, periodically revisit it as upstream library support (e.g., CUTLASS tuning for SM89 in CUDA 12.4) may have resolved the original concern — stale guards silently prevent hardware acceleration
- FP8 tensor core support in Ada Lovelace (SM89) requires CUDA 12.4+ for properly tuned CUTLASS kernels; gating on CUDA_VERSION ensures the toolkit provides the necessary kernel implementations
