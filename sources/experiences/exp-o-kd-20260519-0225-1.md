## Solution

Switch to kOptimized iterator algorithm with kUnity stride support, which precomputes a delta table (inc_next[]) for pointer advancement across filter dimensions and bitmask predicates (masks_[s][3]) for validity checks, replacing per-tile coordinate recomputation with a single byte offset lookup and a 3-way AND bitmask

## Key Lessons

- For CUTLASS convolution with unit strides (stride={1,1,1}), prefer kOptimized iterator with kUnity stride support over kAnalytic/kStrided — the precomputed delta table replaces per-tile coordinate arithmetic with a single pointer offset lookup, yielding major speedups for 3D convolutions
- kOptimized iterator trades a small amount of precomputation memory (delta table inc_next[] and bitmask masks_[]) for elimination of hot-path coordinate math and stride divisibility checks — beneficial when the convolution kernel is called many times with the same problem geometry
- kUnity stride support is a specialized fast path in CUTLASS for stride=1 convolutions that removes the general-purpose stride handling overhead — always select it when all stride dimensions are 1
