## Solution

Vectorize to f16x4 (half2 pair) loads per thread using reinterpret_cast<half2*> to load 4 half values per iteration, reducing loop iterations from 8 to 2 and improving effective memory bandwidth through wider 64-bit transactions

## Key Lessons

- Vectorize half-precision GEMV from scalar to f16x4 (half2 pair) loads to reduce loop iterations and improve memory throughput — each thread processes 4 elements per iteration via 64-bit loads instead of 16-bit, reducing loop overhead and improving L2 cache line utilization
- Use reinterpret_cast<half2*>(&val)[0] as a zero-copy way to load two adjacent half values as a single half2 register — avoids extra packing instructions when the data is already contiguous in memory
- Reducing warp-level loop iterations in bandwidth-bound kernels directly cuts branch overhead and index arithmetic per element, which compounds into measurable gains even at modest problem sizes like K=256
