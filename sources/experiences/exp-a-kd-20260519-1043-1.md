## Challenge

AWQ INT4 nibble extraction uses 3 explicit shift instructions (>>4, >>8, >>12) per 32-bit word with a single BOTTOM_MASK, requiring each nibble to be shifted to bit position 0-3 before lop3 extraction; this adds shift ALU instructions without reducing the lop3 count


## Solution

Reduce explicit shifts from 3 to 1 by introducing TOP_MASK (0x00f000f0) to extract nibbles at bit positions 4-7 without shifting, then apply BOTTOM_MASK and TOP_MASK on both the original word and a single >>8 shifted copy (top_i4s); nibbles extracted via TOP_MASK remain at bits 4-7, requiring fma.rn.f16x2 with 1/16 scale and -64 bias instead of simple sub.f16x2 for dequantization

## Key Lessons

- Reducing explicit shift count (3→1) in a nibble extraction pipeline where lop3 count stays at 4 produces negligible speedup because SHL/SHR are single-cycle ALU ops and the saved 2 shifts are only ~0.24% of the per-thread instruction mix — applies when shift reduction does not also reduce the count of heavier operations like lop3 or memory transactions
- When eliminating a shift by using TOP_MASK to extract nibbles at non-zero bit positions, the downstream FP16 conversion must change from sub.f16x2 to fma.rn.f16x2 (scale×value+bias) to compensate for the nibble being at bits 4-7 instead of 0-3 — the mask change and conversion change are coupled and cannot be applied independently
