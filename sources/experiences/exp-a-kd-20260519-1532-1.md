## Solution

Eliminate the TopKSort stage entirely since only unsorted top-K indices are required as output, replacing the load-sort-write sequence with a direct strided read from smemIndices to global memory, and removing smemLogits shared memory, TopKSort temp storage, and outLogits output along the way

## Key Lessons

- When downstream consumers only need unsorted top-K indices, eliminate the radix sort entirely — cub::BlockRadixSort on 2048 items costs multiple digit-pass iterations, ~several KB of shared memory temp storage, and 8 extra registers per thread, all avoidable when sorted order is not required
- Removing unused output arrays (outLogits) and their shared memory staging (smemLogits) reduces shared memory footprint for higher occupancy and eliminates 2048 float global memory writes per row — apply this whenever output requirements can be narrowed
