## Challenge

Each activation access recomputes the full NDHWC linear offset via ndhwc_offset() which performs 4 int64 multiplications and 4 additions to convert (n,d,h,w,c) coordinates to a memory address, even though adjacent filter positions differ by fixed byte increments


## Solution

Compute base_offset once at (n, d0, h0, w0, fc) and add precomputed byte increments (inc_next_t/r/s) for filter positions, replacing 4 int64 multiplications per access with 3 int64 multiplies plus 2 adds, and dereferencing via pointer arithmetic instead of array indexing

## Key Lessons

- Precompute byte-level stride increments (inc_next_s/r/t) for tiled convolution filter positions outside the loop, then compute final addresses as base_offset + linear combination of increments — eliminates per-access coordinate recomputation for adjacent filter taps
- Use char-pointer arithmetic for byte-level addressing (reinterpret_cast<const char*>(base) + byte_offset) when element offsets are non-uniform across dimensions — avoids recomputing full multi-dimensional layout offsets per access
