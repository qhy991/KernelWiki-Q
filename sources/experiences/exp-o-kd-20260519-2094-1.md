## Challenge

TopK shortcut kernel writes both indices and logits values to global memory even though only indices are consumed downstream, wasting 320 bytes of global memory writes per kernel call (8 rows × 10 elements × 4 bytes) and 40 bytes of shared memory for the topkVals buffer


## Solution

Remove the outLogits output buffer entirely from the kernel signature, eliminate the shared memory topkVals[10] array, and skip writing logits values to global memory — only the selected indices are written since they are the sole consumer of this shortcut path

## Key Lessons

- Eliminate global memory writes for data that is never consumed downstream — removing the outLogits buffer cut 320 bytes of STG per call (8×10×4B), which directly reduces memory traffic without affecting any computation
- Audit kernel outputs against actual downstream consumers: when a kernel produces both indices and values but the caller only needs indices, the value output path is pure waste in both shared memory footprint and global memory bandwidth
- Small shared memory savings (removing topkVals[10] = 40 bytes) compound with global memory write elimination to produce measurable speedup even on tiny output sizes — every unnecessary write path is a candidate for removal
