## Challenge

Inner loop of scoring kernel checks scoring_func via runtime if-branch on every iteration despite the value being uniform across all threads and constant throughout execution, adding an unnecessary integer comparison and branch instruction per element


## Solution

Replace runtime branching on scoring_func with C++ template parameter and if constexpr dispatch, making the sigmoid call unconditional at compile time and eliminating the per-iteration integer comparison and branch

## Key Lessons

- Template parameters with if constexpr eliminate runtime branching on uniform/constant values — the compiler inlines only the taken path, removing the comparison instruction entirely
- When loop body cost is dominated by transcendental functions (expf in sigmoid), eliminating a single integer branch comparison yields negligible speedup — focus optimization effort on reducing the dominant computation instead
- Use template dispatch for kernel configuration parameters (scoring function, activation type) that are known at launch time and uniform across all threads — this is a zero-cost abstraction pattern applicable to any variant-selection scenario
