## Challenge

The compute_problem_sizes kernel hardcodes the problem-size layout as (M=occurrences, N=2*hidden, K), preventing the MoE dispatch from choosing a transposed GEMM layout that would be more efficient when the expert batch size (M dimension) is small


## Solution

Introduce a compile-time template parameter SWAP_AB with a threshold constant, using if constexpr to select between standard and swapped problem-size layouts at compile time with zero runtime overhead

## Key Lessons

- Use compile-time template bool parameters with if constexpr to introduce layout variants (e.g., SWAP_AB) with zero runtime overhead — the unused branch is eliminated at compile time, producing identical machine code to a hand-written single-layout version
- For MoE dispatch kernels that compute GEMM problem sizes, swapping M and N dimensions based on a batch-size threshold allows downstream cuBLAS/cutlass to pick the more efficient tiling strategy for each regime (large M vs small M)
