## Solution

Insert cudaStreamWaitEvent between consecutive chained kernels to simulate CUTLASS PDL griddepcontrol.wait, enabling device-side ordering that would eliminate the CPU driver scheduling gap between dependent launches on SM90a hardware

## Key Lessons

- On SM80 hardware, adding cudaStreamWaitEvent between same-stream kernels is redundant because CUDA already serializes kernels within a single stream — the explicit wait provides no ordering benefit and introduces ~0.12% overhead from event record/wait operations
- PDL (Programmatic Dependent Launch) with griddepcontrol.wait is an SM90a-only feature that eliminates the CPU driver scheduling gap between dependent launches by performing synchronization entirely on-device — it cannot be effectively simulated on SM80 via host-side event APIs
- When evaluating PDL-style optimizations, test on target hardware (SM90a/H100) rather than proxy mechanisms on older architectures, since the core benefit (eliminating CPU-side launch latency) is only achievable through the hardware-native griddepcontrol instruction
