## Solution

Replace deferred-write software pipeline with immediate compute-and-store pattern that writes output within the same iteration, eliminating cross-iteration state variables (output_ptr, my_out array), the conditional null-pointer branch, and the post-loop final store

## Key Lessons

- Software-pipeline deferred writes only provide latency-hiding benefit when the deferred store volume is large enough to overlap meaningfully with computation — deferring 4 half values (8 bytes) per thread is far too small compared to the per-iteration workload of loading inputs, reduction, and normalization
- Prefer immediate compute-and-store patterns over software-pipeline deferred writes when per-thread output is small (tens of bytes), since the overhead of cross-iteration state (extra registers, null-pointer branch, post-loop final store) outweighs any latency-hiding gain
- Conditional null-pointer guards in GPU loops are cheap via predication but add code complexity for no benefit when the deferred workload is negligible — simpler unconditional immediate stores win on clarity and maintainability
