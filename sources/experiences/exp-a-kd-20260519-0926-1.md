## Challenge

Per-warp subset scanning assigns ~32 experts to each of 12 warps and iterates with WARP_SIZE stride through shared memory, causing redundant shared memory reads, loop overhead, and branch divergence in the inner scanning loop per TopK iteration


## Solution

Replace per-warp subset scanning with one-thread-per-expert direct mapping where 384 threads each hold one expert value, then use warp-level shuffle reduction directly on thread-local values without any inner scanning loop

## Key Lessons

- Map one thread per data element instead of distributing subsets across warps when total thread count >= element count → eliminates inner scanning loops, reduces shared memory read traffic, and removes per-warp partitioning overhead → applies to TopK, argmax, and selection kernels where thread count covers the data domain
- Direct thread-to-element ownership enables self-invalidation (tid == selected) instead of centralized invalidation from a single warp → reduces cross-warp coordination and shared memory write contention
