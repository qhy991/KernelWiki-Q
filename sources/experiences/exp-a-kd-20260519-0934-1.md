## Solution

Replace fixed blockIdx.x starting offset with a rotation scheme: track cumulative tile counts across groups via group_total_tiles and compute head_cta_id = group_total_tiles % gridDim.x, then set blk_offset = (blockIdx.x + (gridDim.x - head_cta_id)) % gridDim.x to rotate which CTAs receive remainder tiles in each group, balancing per-CTA work to [9,8,8,8,8,8,8,8] with stddev=0.331

## Key Lessons

- Rotate CTA starting offsets across groups by tracking cumulative tile counts (group_total_tiles % gridDim.x) rather than using a fixed blockIdx.x offset per group — distributes remainder tiles to different CTAs each group and reduces tail synchronization latency at kernel completion
- Work imbalance caused by non-divisible tile counts compounds across groups when using fixed-offset scheduling; a simple rotation via modular arithmetic eliminates the compounding effect with negligible overhead (two integer accumulators and one modulo per group)
