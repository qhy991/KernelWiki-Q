## Solution

Refine dequant_skip_flop to exclude MXFP8 by adding && !(s_type == kFE8M0fnu) to the kFE4M3fn check, and replace the b_type-based dispatch with an s_type-based dispatch that calls dequant_fp8_scales for all 8-bit scale types (kFE4M3fn and kFE8M0fnu) before applying the dequantized scales to weights

## Key Lessons

- When a compile-time skip condition depends on multiple template parameters (weight type + scale type), each parameter combination must be individually audited — a blanket match on one parameter (b_type == kFE4M3fn) can mask incorrect skips when the other parameter changes the required computation path
- Dispatch on the scale type (s_type) rather than the weight type (b_type) for scale dequantization logic — the scale format (e8m0 vs 16-bit float) determines how scales must be decoded regardless of the weight format
