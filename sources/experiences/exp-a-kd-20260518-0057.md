## Challenge

The after code uses kFragmentsPerIteration=2 (from the conditional kPartitionsK==1 ? DefaultIterators::kFPI : 1), which allocates 2x shared memory (68KB vs 34KB). This cuts the number of __syncthreads() barriers from 16 to 8, but the occupancy reduction from 4 blocks/SM to 2 blocks/SM dominates. ...


## Solution

the optimization is designed to help. The microbenchmark's trivial smem copy loop cannot reproduce this workload characteristic.
