## Challenge

RoPE kernel reads and writes pairs of floats using scalar loads (x[idx*2], x[idx*2+1]) and scalar stores, issuing two separate memory transactions per pair and underutilizing available memory bandwidth


## Solution

Pack two adjacent (x1,x2) pairs into a single float4 vector, loading and storing four floats per memory transaction instead of two, thereby halving the number of global memory loads/stores and doubling memory throughput

## Key Lessons

- Use float4 vectorized loads/stores via reinterpret_cast to coalesce adjacent scalar pairs into a single 128-bit memory transaction, reducing total memory operations by half — applies when elements are contiguous and 16-byte aligned
- When vectorizing RoPE from float2 pairs to float4 packs, each dimension index must be remapped (token_idx*2 and token_idx*2+1) to compute the correct per-pair frequency exponent within the packed quadruple
