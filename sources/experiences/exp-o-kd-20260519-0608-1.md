## Challenge

MoE gate kernel normalizes topk output weights by output_sum in one pass, then requires a separate kernel launch or host-side loop to multiply by routed_scaling_factor, causing each weight to be read and written twice and adding kernel launch overhead


## Solution

Fuse the routed_scaling_factor multiplication into the same unrolled loop that divides by output_sum, gated by a bool flag apply_routed_scaling_factor_on_output, so normalize and scale execute in a single pass over the output array

## Key Lessons

- Fuse sequential passes over the same data (normalize then scale) into a single loop to halve global memory traffic and eliminate an extra kernel launch — applies whenever two post-processing steps operate on the same output array element-by-element
- Use a runtime bool flag to gate fused operations so the kernel remains correct for both code paths without maintaining two separate kernel implementations
