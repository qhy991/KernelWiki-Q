## Challenge

The tree construction kernel uses bool* tree_mask where each boolean occupies a full byte, requiring 8 bytes per row for draft_token_num=8, and further wastes bandwidth on a complex index computation involving a sequential loop over all prior batches' verified_seq_len


## Solution

Replace bool* tree_mask (1 byte per boolean) with uint8_t* bit-packed representation packing 8 booleans per byte, and simplify the index computation from a sequential verified_seq_len accumulation loop plus tree_mask_mode branching to a direct (bid * draft_token_num + tid) * num_bytes_per_item calculation

## Key Lessons

- Pack boolean arrays into bit fields (uint8_t with |= and bit shifts) instead of using bool* when each element is 0/1 — reduces memory footprint and bandwidth by up to 8x with only a small read-modify-write overhead per set operation
- Remove sequential index computation loops (accumulating over prior batches) when switching to a fixed-stride layout — direct stride-based indexing eliminates warp divergence and serial dependencies across thread blocks
