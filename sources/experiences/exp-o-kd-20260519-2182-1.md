## Challenge

FP8 quantization uses two separate kernel launches (segmented_max_reduction for absmax, then scaled_quant_kernel for quantization), requiring two full passes over input data, an extra cudaMemset, and a global atomicMax across all blocks for scale aggregation


## Solution

Fuse absmax reduction and quantization into a single kernel with one block per token, eliminating the second full read of input data, the inter-block global atomicMax, and the cudaMemset overhead; each block computes its own per-token absmax via blockReduceMax, writes the scale, then immediately quantizes the same token's data

## Key Lessons

- Fusing a reduction pass and a transformation pass into a single kernel halves global memory reads and eliminates kernel launch overhead plus inter-block synchronization — especially beneficial when hidden_size is large (≥4096) as the data volume amplifies memory bandwidth savings
- Map one block per independent element (e.g., one block per token) instead of a grid-stride loop over all elements — this eliminates global atomics entirely since each block owns its own output element and reduction is intra-block only
