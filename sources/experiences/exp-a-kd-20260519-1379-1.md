## Solution

Precompute the inverse scale (1.0f / *scale) once per thread at kernel entry and replace per-element float division with multiplication by the precomputed reciprocal, eliminating repeated division instructions inside the inner loop

## Key Lessons

- Precomputing loop-invariant reciprocals (division → multiplication by inverse) is a textbook arithmetic optimization, but in memory-bound kernels the arithmetic pipeline is already underutilized while waiting on memory transactions, so the speedup may be negligible or neutral
- NVCC -O3 can promote repeated uniform scalar loads (e.g., *scale) into registers automatically, reducing the marginal benefit of manual precomputation in stride-loop kernels where each thread processes many elements
- When kernel performance is dominated by HBM traffic (6 bytes per element here: 2-byte half read + 4-byte float write), arithmetic instruction selection has minimal impact on observed throughput
