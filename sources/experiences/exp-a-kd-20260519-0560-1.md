## Challenge

RoPE kernel uses a flat linear index (blockIdx.x * blockDim.x + threadIdx.x) across all token positions, causing threads in the same warp to frequently cross token boundaries during strided memory access, reducing global memory coalescing efficiency


## Solution

Remap threads so each block corresponds to one token position (blockIdx.x = token_pos, threadIdx.x = dimension index), ensuring all threads in a block access the same token row sequentially and produce well-coalesced global memory reads with 512-thread blocks (16 warps per block)

## Key Lessons

- Map one block to one logical row (token position) and use threadIdx.x for the dimension index so that consecutive threads access consecutive memory addresses within the same row, producing coalesced global memory transactions — applies to any per-element transform that operates independently across dimensions within a structured row
- Larger thread blocks (512 vs 256) increase warps per block (16 vs 8), improving SM occupancy and reducing block scheduling overhead when the natural data dimension supports it
