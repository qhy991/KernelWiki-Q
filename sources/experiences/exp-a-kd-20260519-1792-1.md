## Solution

Fuse the fp8→half dequantization and scale multiplication into a single scaled_vec_conversion_fp8_to_half function that performs __nv_cvt_fp8_to_halfraw → half_to_float → multiply by scale → float_to_half in one call, consolidating the conversion logic without changing the underlying hardware instruction stream

## Key Lessons

- Fusing separate dequant + scale steps into a single inline function improves code clarity but produces no performance change when both paths use __forceinline__ PTX helpers that compile to the identical cvt.f16.fp8 → cvt.f32.f16 → mul.f32 → cvt.rn.f16.f32 instruction sequence
- When a kernel is memory-bound (1MB read + 2MB write per element), micro-refactoring of arithmetic instruction ordering within the same instruction stream has negligible effect — performance is dominated by memory throughput, not compute
