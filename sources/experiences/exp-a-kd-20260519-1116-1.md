## Solution

Replace division/modulo with bitwise shift/AND for power-of-2 decompositions (mIdx>>7, mIdx&31, (mIdx>>5)&3, kIdx>>2, kIdx&3) and collapse the 5-stride-variable additive expression into a single shift-OR expression ((mTileIdx*numKTiles+kTileIdx)<<9 | outerMIdx<<4 | innerMIdx<<2 | innerKIdx), also adding __forceinline__ and early-return pattern

## Key Lessons

- Manual replacement of division/modulo by power-of-2 values with bitwise shift/AND is redundant under nvcc -O3, which already performs this lowering — only justified when compiling at -O0 or when the divisor is not a manifest compile-time constant
- Collapsing multi-stride additive offset into a single shift-OR expression improves readability but produces identical PTX when the compiler can see all stride values are derived from known tile sizes
- Adding __forceinline__ to small device helpers has no measurable effect at -O3 because nvcc already inlines such functions, but it serves as a clarity hint for maintainers
