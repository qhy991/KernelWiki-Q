## Solution

Extract the sorting loop into a dedicated moe_token_sort_kernel launched across multiple blocks (<<<actual_blocks, 256>>>) with a grid-stride loop and global atomicAdd on cumsum_buffer, distributing the token-to-expert binning across all available SMs instead of a single block

## Key Lessons

- Split monolithic single-block kernels into separate multi-block kernels when a phase can run independently after a synchronization point — distributing work across SMs improves occupancy and reduces per-thread serial workload
- Use global atomicAdd with a grid-stride loop for token-to-expert binning when token count exceeds a single block's capacity — global atomics spread contention across SMs unlike shared-memory atomics confined to one block
- Reuse the cumsum buffer (computed by the align kernel) as the atomic counter base for the sort kernel — avoids redundant counting and provides correct per-expert offsets without extra memory
