## Solution

Replace full sort with multi-pass radix selection: build a coarse 256-bin FP16 histogram in shared memory to identify the threshold bin separating top-K from the rest, then refine only the ambiguous candidates in that bin using byte-level radix passes on FP32 bits, avoiding global-memory sorting passes entirely

## Key Lessons

- Replace full sort with radix-based selection for top-K when K << N — a coarse histogram identifies the threshold bin in O(N), and only ambiguous candidates need refinement, reducing work from O(N log N) to effectively O(N) → applies when only a subset ranking matters and the coarse histogram provides sufficient discrimination
- Use FP16-quantized 8-bit histograms for the coarse pass and reserve FP32 byte-level radix for the refinement pass — the two-level approach balances speed (few bins, shared memory) with precision (full FP32 resolution only where needed) → applies to selection and partition problems on floating-point data
- CUB device-wide sort incurs multiple global-memory read/write passes; for partial-ordering problems, shared-memory histogram methods avoid nearly all global-memory traffic → applies when the working set fits in shared memory and the output is a small subset of the input
