## Challenge

MoE sum kernel uses a runtime-variable topk loop with plain global-memory loads, causing loop-carried accumulator dependencies and preventing the compiler from overlapping independent load instructions across iterations


## Solution

Make TOPK a compile-time template parameter to enable #pragma unroll, fully unrolling the inner loop into independent __ldg (read-only cache) load-add operations that can be issued simultaneously, bypassing the L1 data cache and eliminating per-iteration branch overhead

## Key Lessons

- Make small iteration counts (e.g. topk=4) compile-time template constants so the compiler can fully unroll the loop — this eliminates branch overhead and exposes independent memory operations for instruction-level parallelism
- Use __ldg() for read-only global-memory accesses whose stride exceeds a cache line — routes loads through the read-only cache, avoiding L1 data cache pollution and enabling the compiler to schedule loads more aggressively
- Fully unrolling a small fixed-count loop with __ldg allows the hardware to overlap multiple independent cache-line requests simultaneously, yielding significant speedup (1.74x) when each iteration accesses a distinct 8KB-stride region
