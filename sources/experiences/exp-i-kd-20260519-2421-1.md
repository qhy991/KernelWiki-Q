## Solution

Use cutlass::Array<cutlass::half_t, kN> as the Element type for vectorized SIMD processing, instantiate CUTLASS operator functors (plus, minus, multiplies, divides, multiply_add) as ternary ops via op(a, b, c), and guard the iteration loop with CUTLASS_PRAGMA_NO_UNROLL to prevent the compiler from unrolling benchmark iterations

## Key Lessons

- Use cutlass::Array<T, kN> as the element type in CUTLASS kernels to get vectorized SIMD loads/stores for free — the Array abstraction maps to PTX vector types (half2, float4, etc.) without manual reinterpret_cast
- CUTLASS operator functors (plus, multiplies, multiply_add) accept three arguments as ternary ops: op(a, b, c) computes a ⊗ b ⊕ c for multiply_add, enabling fused multiply-add patterns in a single functor call
- Use CUTLASS_PRAGMA_NO_UNROLL on iteration loops in benchmark kernels to prevent the compiler from eliminating repeated iterations — ensures each iteration actually executes for accurate throughput measurement
