bf16_reduce_direct

## Solution

The after kernel (upcast_reduce_kernel_bf16) uses the packed array_t<nv_bfloat16,8> type with __align__(16), loading 8 bf16 values (16 bytes) per ld.128 instruction, upcasting all 8 to float32 via __bfloat162float in an unrolled loop, accumulating in array_t<float,8> using native fp32 add, and do...

## Key Lessons

- The fp32 accumulation also provides numerically superior results (closer to the CPU fp32 reference) compared to the direct bf16 __hadd path, though the measured speedup is dominated by the memory throughput improvement rather than the arithmetic path.
