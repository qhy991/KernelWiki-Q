## Solution

Vectorize the inner loop to process 4 half elements per thread per iteration using half2 loads via reinterpret_cast, collapsing 32 scalar iterations into 8 vectorized iterations and doubling effective memory bandwidth from 64 to 128 bytes per warp per iteration

## Key Lessons

- Vectorize half-precision GEMV inner loops with half2 loads (f16x4 pattern) to process 4 elements per thread per iteration — doubles effective memory bandwidth and reduces loop overhead by 4x when K is a multiple of 128
- Use reinterpret_cast<half2*>(&val)[0] as a zero-overhead HALF2 macro to load pairs of adjacent half values as a single 32-bit transaction instead of two separate 16-bit loads
- Manual decomposition of half2 into .x/.y components for multiply-accumulate avoids native half2 arithmetic limitations and gives the compiler full scheduling freedom over paired operations
