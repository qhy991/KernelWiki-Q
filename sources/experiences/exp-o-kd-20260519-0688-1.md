## Solution

Move peer buffer pointers into a separate GPU-resident RankData struct accessed via a single RankData* indirection, eliminating the per-thread buffers[RANKS] local array, the round-robin modulo computation, and reducing register pressure — the one-time 64-byte global load of RankData is amortized across the entire reduction loop

## Key Lessons

- Moving large pointer arrays out of kernel parameter structs into a separate GPU-resident indirection struct (RankData*) reduces per-thread register consumption and kernel parameter block size — applies whenever peer buffer pointers exceed a few entries and are read by every thread in the same way
- Eliminating round-robin modulo remapping ((local_rank + ii) % RANKS) at kernel setup by pre-arranging pointer order in the GPU-resident struct removes integer division instructions from the hot path — applies when the mapping is fixed for the kernel's lifetime
- A single extra global memory load (64 bytes for RankData) read once per thread block is negligible compared to saving 8 registers per thread across hundreds of threads — prefer pointer indirection over embedded arrays when the indirection target fits in a single cache line
