## Solution

Move scale loading inside the per-head warp loop using indexed access k_scale[head * kv_scale_stride], adding a kv_scale_stride parameter so the same kernel handles both per-tensor (stride=0, reads same element) and per-head (stride>0, reads per-head scale) layouts without a dedicated code path

## Key Lessons

- Design quantization-scale kernels with a stride parameter (kv_scale_stride) so a single kernel handles both per-tensor (stride=0) and per-head (stride>0) layouts — when stride=0 the indexed access degrades to reading the same element, preserving the fast-path behavior without a separate kernel
- Repeated reads from a small scale array (1–num_heads floats) at the same address across warps achieve near-100% L1 cache hit rate, making per-iteration global loads effectively free compared to the bulk data copy in the inner loop — do not avoid per-head scale loading for cache concerns alone
