## Solution

Cache input data in shared memory during Pass-1 with bank-conflict padding (32-byte stride alignment) and a __syncwarp barrier, then read from shared memory in Pass-2 to eliminate the redundant global load

## Key Lessons

- Shared memory caching to eliminate redundant global loads is NOT beneficial when the per-warp working set already fits in L1 cache — the smem write instructions, bank-conflict padding overhead, and __syncwarp barrier cost can exceed the savings from avoiding a cached global reload
- At hidden_dim=1024 with fp16 (2KB per warp, 16KB per CTA with 8 warps), L1 cache (128KB) comfortably retains Pass-1 data for Pass-2 reuse, making explicit shared memory caching unnecessary and slightly harmful (~0.6% regression)
- Shared memory optimization decisions must account for the hardware cache hierarchy — when L1 hit rate is already high for the access pattern, adding smem indirection introduces overhead without measurable benefit
