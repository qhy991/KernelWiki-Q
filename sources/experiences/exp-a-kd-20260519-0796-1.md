## Solution

Template GroupReduceMax on THREADS_PER_SUBWARP and guard each shuffle step with `if constexpr` to allow the compiler to eliminate unnecessary shuffles at compile time for smaller group sizes, while keeping all steps for 16-thread subwarps

## Key Lessons

- Template subwarp reduction functions on group size and use `if constexpr` to guard each shuffle step so unnecessary shuffles are eliminated at compile time for smaller groups — produces zero-cost generality when the group size is a compile-time constant
- For THREADS_PER_SUBWARP=16 all 4 XOR-shuffle steps (offsets 8,4,2,1) are required for a complete reduction, so the `if constexpr` optimization has no effect and performance is neutral at this specific size — the benefit appears only for sub-16 group sizes
