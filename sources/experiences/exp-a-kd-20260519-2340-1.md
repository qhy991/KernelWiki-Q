## Challenge

A local constexpr bool variable inside a __global__ kernel function is declared with the 'static' qualifier, which is unnecessary for compile-time dispatch via 'if constexpr' and may affect binary symbol emission without any runtime benefit


## Solution

Remove the 'static' qualifier from the local constexpr bool variable, since 'constexpr' alone is sufficient for 'if constexpr' compile-time dispatch — this is a code hygiene improvement that eliminates unnecessary static storage semantics without changing generated instructions

## Key Lessons

- Removing 'static' from a local constexpr variable in CUDA device code has no runtime effect — 'constexpr' alone fully satisfies the compile-time constant requirement for 'if constexpr' dispatch, so the 'static' qualifier is redundant and only affects symbol emission in the binary
- When benchmarking NVCC codegen differences that only involve storage-class qualifiers on compile-time constants, expect timing deltas within normal GPU benchmark noise (±0.5%) and classify the verdict as neutral rather than attributing the noise to a real optimization
