## Challenge

Implement top-k sampling kernel that returns deterministic top-1 index from top-k candidates


## Solution

One thread per batch element using insertion sort for top-k selection


### Key Changes

- Used insertion sort for top-k selection (O(n*k) complexity)
- Shared memory for storing top-k values and indices
- Deterministic output: always return top-1 from top-k


## Key Lessons

- Test framework expects deterministic output for topk_sampling - return top-1 from top-k
- Insertion sort works well for small k (k=8) and small vocab (160)
- Shared memory size must be declared with compile-time constants for CUDA
- Function signature must match test framework: forward(probs, k)
