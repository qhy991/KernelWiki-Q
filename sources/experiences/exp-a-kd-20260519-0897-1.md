## Solution

Replace the embedded pointer array with a RankData* indirection to a GPU-resident struct, eliminating the per-thread local buffers array (freeing 8 registers), removing round-robin modulo computation, at the cost of a single 64-byte global read that amortizes across the entire reduction loop

## Key Lessons

- Replace embedded pointer arrays in kernel parameter structs with single-pointer indirection to GPU-resident lookup tables to eliminate per-thread local pointer copies (reducing register pressure) and remove remapping arithmetic — applies when multiple peer or resource pointers are passed to kernels via parameter structs
- A single extra global memory load for a small lookup struct (e.g., 64 bytes for 8 pointers) is negligible when amortized across a data-parallel loop processing thousands of elements per thread
