## Solution

Fuse the per-element computation into a single loop: load current_di once (broadcast across elements), then compute (accum[i] - current_di) * attn inline for each element, eliminating the two intermediate register arrays fragment_attn and fragment_di

## Key Lessons

- Fusing per-element computations to eliminate intermediate register arrays reduces register pressure but shows no benefit when the fragment size is small enough (8 elements) that all temporaries stay in registers without spilling — the compiler already schedules loads and arithmetic efficiently
- Register-saving fusion techniques become valuable when fragment sizes grow large enough to exceed register file capacity, causing spills to local memory; microbenchmarks with small fragments may not capture this benefit
- Broadcasting a scalar value (current_di loaded once) instead of replicating it into a full array is a useful pattern even when register pressure is not the bottleneck, as it expresses the data dependency more clearly to the compiler
