## Challenge

Skeleton kernel requires type-safe numeric conversion with element-wise scaling across heterogeneous data types (float, half, bfloat16, int8_t, int2b_t, int4b_t, fp8), with no manual per-element loop that would forfeit vectorized memory throughput


## Solution

Instantiate cutlass::NumericArrayConverter<Destination, Source, Count> and call it with the source and scale_factor Array references in a single expression, letting CUTLASS handle vectorized load, type-safe conversion, element-wise scaling, and vectorized store through the Array assignment operator

## Key Lessons

- CUTLASS NumericArrayConverter<Dst,Src,N> encapsulates vectorized load, type-safe element conversion, and optional scale-factor application in one callable object — prefer it over manual per-element loops to get aligned vector access for free
- Template the vector width (Count) as a compile-time parameter so the compiler can unroll the entire conversion pipeline; power-of-2 counts map directly to hardware vector instructions (half2, float4, etc.)
- CUTLASS Array<T,N> assignment operator handles vectorized store, so '*destination = result' yields an aligned STG without explicit reinterpret_cast or vector-store intrinsics
