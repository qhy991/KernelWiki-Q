## Challenge

Synchronous global-to-shared memory load using a simple strided loop (LDG+STS per thread) with __syncthreads() barrier, which stalls each thread until its load completes before any compute can proceed


## Solution

Replace synchronous LDG+STS loads with cp.async.ca.shared.global PTX instructions allowing the hardware to asynchronously transfer data from global to shared memory, followed by commit_group and wait_all barriers to ensure completion before compute

## Key Lessons

- cp.async only improves throughput when concurrent compute can overlap with async copies — issuing cp.async followed immediately by wait_all with no interleaved work is functionally equivalent to a synchronous load but adds fence/wait instruction overhead
- In a real multi-stage pipelined kernel, cp.async enables issuing global loads for stage N+1 while computing on stage N, but single-stage microbenchmarks with no interleaved compute cannot benefit from async copy
