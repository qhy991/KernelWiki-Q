## Solution

Increase kStage from 1 to 2 to enable multi-stage async pipeline with double-buffered K loading, preloading one K tile before the KV loop and issuing async prefetches for current V and next K inside each iteration — however, with only 4 KV-tile iterations at N=256 the pipeline cannot amortize its setup overhead and the extra 9216 bytes of shared memory adds occupancy pressure

## Key Lessons

- Multi-stage (kStage>1) async software pipelines in tiled attention kernels only pay off when the KV loop has enough iterations (typically Tc>=8) to amortize preload setup and fill the pipeline — at short sequence lengths the synchronous path is equally fast or better
- Double-buffering K tiles increases shared memory by one KV_tile_size per extra stage (e.g., 9216 bytes for head_dim=64, Bc=64, pad=8), which can reduce occupancy and offset any latency-hiding gains on small problem sizes
