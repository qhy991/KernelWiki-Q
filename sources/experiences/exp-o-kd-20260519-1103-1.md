## Challenge

Speculative decoding tree mask uses one bool per element (bool*), requiring 256×32×32 = 262144 bytes and initializing 31 false writes per token, wasting memory bandwidth and allocation size


## Solution

Replace bool-per-element mask with bit-packed uint8_t mask (8 bits per byte), reducing allocation 8x (262KB→32KB), eliminating the 31-element false initialization loop, and using bitwise OR to set individual bits during tree traversal

## Key Lessons

- Bit-pack boolean masks into uint8_t (8 bits per byte) instead of using bool arrays — reduces memory allocation 8x, cuts H2D transfer cost, and eliminates per-element false initialization loops, applying to any kernel with sparse boolean data
- Simplify mask index computation by removing mode-dependent branching and variable-length prefix sums — a flat (batch × token × bytes) layout with num_bytes_per_item parameter is faster to compute and easier to maintain than conditional FULL_MASK/QLEN_ONLY indexing
- Replace per-element mask writes with bitwise OR into pre-zeroed bytes — tree_mask[offset] |= (1 << bit) sets one bit without touching neighbors, avoiding the need to initialize surrounding entries to false
