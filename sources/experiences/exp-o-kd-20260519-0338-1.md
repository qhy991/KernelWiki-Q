## Solution

Split the input range into 64 contiguous chunks, launch one block per chunk so every SM participates, compute each partial sum in shared memory with the same tree reduction, then combine block-level results with atomicAdd into the output

## Key Lessons

- Split a large reduction across multiple blocks with each block owning a contiguous key subrange — this saturates SMs and DRAM bandwidth instead of serializing all work through one block
- Use atomicAdd to combine per-block partial sums into a single output when the number of blocks is small (tens of atomics), avoiding a second kernel launch for the final reduction
- Always memset the output to zero before accumulating partial sums with atomicAdd, otherwise the result includes stale data from prior runs
