## Challenge

bf16×bf16 GEMM writes output to an intermediate bf16 buffer via cublasGemmEx with CUDA_R_16BF output type, then a separate cast kernel reads that buffer to convert bf16→fp32, incurring an extra 6 KB global-memory round-trip and an additional kernel launch overhead


## Solution

Use cublasGemmEx with CUDA_R_32F as the output type so the GEMM writes fp32 results directly, eliminating the intermediate bf16 buffer and the separate cast kernel entirely

## Key Lessons

- cublasGemmEx supports mixed-type output: bf16 inputs with CUDA_R_32F output produces fp32 results directly via fp32 accumulation, avoiding the need for a separate bf16→fp32 cast kernel and its associated global-memory round-trip
- When a downstream consumer requires fp32 precision from bf16 GEMM, always prefer the single-kernel mixed-output cublasGemmEx path over GEMM-to-bf16 plus a separate cast kernel — the eliminated intermediate write/read pays for itself at any problem size
