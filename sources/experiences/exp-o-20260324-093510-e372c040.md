## Challenge

Round 3 optimization attempts for RMS norm kernel after achieving 2.647x in Round 2


## Solution

128-thread kernel with register blocking, 8 float4/thread, fmaf, __ldg for weight

## Key Lessons

- Register blocking (caching input in registers) is the key optimization for RMS norm - eliminates second memory pass
- 128 threads is the sweet spot for hidden_size=4096 on A800 (64 threads too few, 256 too many)
- After register blocking, the kernel is near-optimal: batch_1 hits latency floor ~0.004ms, batch_512 saturates memory bandwidth at ~3043 GBPS
- Shared memory weight caching adds overhead (extra __syncthreads) that negates bandwidth savings
- Persistent kernel hurts large batch throughput by serializing work that could run in parallel
- shfl_xor vs shfl_down for warp reduction makes no measurable difference
- The 0.3 performance_ratio improvement threshold is strict - marginal gains (0.014) don't qualify as keep
