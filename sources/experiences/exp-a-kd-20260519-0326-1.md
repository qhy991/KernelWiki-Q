## Solution

Fuse the two separate compute and update passes into a single q-loop that computes m_delta, updates m_prime, multiplies s_prime, then accumulates s_delta contributions in one pass — plus isfinite() guards on m_delta and s_delta to prevent NaN/Inf propagation

## Key Lessons

- Loop fusion that keeps m_delta, s_delta, m_prime, and s_prime all live in a single scope increases register pressure — when tile dimensions are small (kQ=4, kK=32), register spill to local memory outweighs the saved loop overhead, making separate passes faster
- Adding isfinite() branch guards inside tight fused loops compounds register pressure and disrupts instruction-level parallelism, further degrading throughput in already register-heavy kernels
