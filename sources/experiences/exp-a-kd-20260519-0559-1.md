## Challenge

RoPE kernel reads and writes pairs of floats using scalar 4-byte loads and stores (x[idx*2], out[idx*2]), issuing two separate memory transactions per pair and underutilizing available memory bandwidth


## Solution

Replace scalar float loads/stores with float4 (128-bit) vectorized loads/stores via the FLOAT4 macro, so each thread reads 4 floats in one transaction and processes two consecutive RoPE pairs, halving the number of memory transactions and improving bandwidth utilization

## Key Lessons

- Use float4 (128-bit) vectorized loads/stores via reinterpret_cast<float4*> to coalesce 4 scalar float transactions into one, reducing memory transaction count by 4x — applies to any kernel where consecutive float elements are accessed by the same thread
- Process multiple element pairs per thread (2 RoPE pairs via float4 vs 1 pair via scalar) to increase arithmetic intensity relative to memory traffic, which is especially beneficial for memory-bound kernels like RoPE
