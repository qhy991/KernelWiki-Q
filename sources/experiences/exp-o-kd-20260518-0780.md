## Key Lessons

- This two-kernel approach incurs: (1) one extra kernel launch overhead, (2) a full round-trip of the intermediate activation through global memory (write by silu_and_mul, read by quantization), which is wasteful since the intermediate data is consumed immediately by the next kernel.
- When enabled, the kernel reads the gate and up vectors directly from the doubled-width input (gate||up layout), applies `compute_silu_mul` in-register, and feeds the result directly into `cvt_warp_fp16_to_fp4` without any intermediate global-memory write.
