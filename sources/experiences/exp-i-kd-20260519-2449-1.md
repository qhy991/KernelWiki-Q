## Challenge

Skeleton CUTLASS threadblock epilogue kernel is missing accumulator tile initialization, tile coordinate computation from grid dimensions, problem size derivation, and the actual epilogue operator invocation to write results to output


## Solution

Initialize FragmentC accumulator elements using warp_id * blockDim.x + lane_id as a deterministic per-thread test value, compute tile coordinates from blockIdx, derive problem_size by scaling tile coordinates by the epilogue Shape dimensions (128x128), and invoke the epilogue functor with problem_size, tb_tile_coord, and accum to trigger shared-memory-based output writing

## Key Lessons

- In CUTLASS epilogue kernels, problem_size is derived from tile coordinates scaled by the threadblock tile shape (M, N) — this maps each block's (blockIdx.x, blockIdx.y) to its corresponding output region in the global matrix
- Use CUTLASS_PRAGMA_UNROLL on accumulator initialization loops to ensure compile-time unrolling and avoid loop overhead in the performance-critical epilogue path
- FragmentC element count equals Shape::kCount / (kWarpCount * kWarpSize), distributing accumulator elements evenly across all threads in the threadblock
