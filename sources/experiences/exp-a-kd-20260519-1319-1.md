## Solution

Add kv_scale_stride parameter to generalize scale loading: extend the fast-path branch condition to is_contiguous_heads && kv_scale_stride == 0, move scale dereference into each branch, and add per-head scale indexing (k_scale[head * kv_scale_stride]) in the HND/slow path

## Key Lessons

- Move scale pointer dereference into the branch that uses it rather than loading before the branch — defers the access and naturally supports different indexing strategies (per-tensor vs per-head) without redundant loads
- Extend the fast-path branch condition to include the scale stride check (kv_scale_stride == 0) so the flat contiguous loop is only taken when both layout and scale mode are trivial, preventing incorrect results when per-head scales are present
- Per-head scale indexing via k_scale[head * kv_scale_stride] generalizes both per-tensor (stride=0) and per-head (stride=1) modes with a single stride parameter — applies to any kernel that must support mixed-granularity quantization scales
