## Challenge

Dynamic int8 quantization kernel performs two sequential cub::BlockReduce passes (one for max with cub::Max, one for min with cub::Min) separated by a __syncthreads() barrier, requiring two full reduction traversals over shared memory and doubling synchronization overhead


## Solution

Introduce a MinMax struct carrying both min and max in a single object, accumulate both values simultaneously in the per-thread scan (operator+=), and perform a single BlockReduce<MinMax, 256> pass with a custom binary operator (operator&=), eliminating the __syncthreads() barrier and halving reduction operations

## Key Lessons

- Merging multiple independent reductions (min + max) into a single compound reduction with a custom struct eliminates inter-reduce __syncthreads() barriers and halves shared memory traffic — applies whenever two or more associative reductions scan the same data
- Reducing block size from 1024 to 256 for the merged MinMax reduction lowers per-block shared memory pressure, enabling higher grid-level parallelism and occupancy when the kernel operates over many independent rows
