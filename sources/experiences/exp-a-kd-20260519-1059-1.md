## Solution

Fuse sigmoid, shared-memory bias backup, bias addition, and index assignment into a single Vlen-width loop body so each element's operations complete before moving to the next, while also saving original bias values to shared memory (smem_bias) for downstream topk score restoration

## Key Lessons

- Fusing two unrolled loops into one does not improve ILP when the compiler already interleaves separate #pragma unroll loops into a tightly scheduled instruction stream — the effective instruction order is unchanged
- Shared-memory bias backup (smem_bias) and index assignment (idx_chunk) are overhead in isolation but enable downstream topk score restoration via (thread_max_val - smem_bias[thread_max_idx]), eliminating the need to carry intermediate results through the entire topk iteration — always evaluate fusion in the context of the full kernel, not just the fused sub-region
