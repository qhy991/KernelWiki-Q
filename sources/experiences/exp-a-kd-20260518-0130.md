## Solution

Add a template parameter kMaxK and derive constexpr bool kSingleIteration = kMaxK <= TileN so that when head_dim fits in one tile the loop bound collapses to 1 and the isLastColumn check becomes compile-time true, eliminating the runtime branch and loop prologue entirely

## Key Lessons

- Replace runtime loop bounds with constexpr-derived single-iteration guards when tile dimensions are known at compile time — the compiler folds the loop away entirely, removing branch and comparison overhead from the hot path
- Use constexpr bool guards (e.g., kSingleIteration) to short-circuit runtime checks like isLastColumn into compile-time constants, eliminating per-iteration conditional branches in tiled kernel epilogues
- When kMaxK <= TileN, the loop col < (kSingleIteration ? 1 : head_dim_value) compiles to col < 1, so the compiler unrolls the single iteration and removes the loop prologue entirely — apply this pattern whenever a tile dimension is guaranteed to cover the full extent at a specific template instantiation
