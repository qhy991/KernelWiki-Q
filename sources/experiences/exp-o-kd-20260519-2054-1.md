## Challenge

The init_same_group lambda unconditionally reads g_idx data from shared memory (sh_g_idx_stage) even when has_act_order is false, causing unnecessary shared-memory reads and register pressure for non-act-order quantization paths


## Solution

Add a compile-time constexpr early return at the top of init_same_group that checks !has_act_order; when activation ordering is not used, it immediately sets is_same_group[pipe]=false and same_group_id[pipe]=0 and returns, completely skipping the g_idx shared-memory reads and group comparison logic

## Key Lessons

- Use if constexpr with template/compile-time flags to insert zero-cost early returns that skip entire code paths at compile time — the compiler eliminates the dead branch entirely, avoiding any runtime overhead
- When a kernel supports multiple configuration modes (e.g., act-order vs non-act-order), guard shared-memory reads for mode-specific data behind a compile-time check so unused modes never pay the memory-access cost
