## Challenge

Tried to improve upon r1/v1 (217.5% baseline ratio) for RMS norm with hidden_size=4096

## Key Lessons

- For RMS norm hidden_size=4096: 256 threads with 4 float4/thread is the sweet spot
- Shared memory for weight vector (16KB/block) kills occupancy - avoid for small hidden sizes
- Reducing thread count from 256 to 128 with more work/thread does not help for this problem
- The baseline transplant with float4 vectorization + warp shuffle reduction is already near-optimal
- When 2 consecutive attempts regress, consider switching to a different task for breadth
