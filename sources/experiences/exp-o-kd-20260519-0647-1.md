## Challenge

Blelloch exclusive prefix sum uses shared memory read-modify-write in both up-sweep and down-sweep phases, requiring O(log n) __syncthreads() barriers per phase and incurring shared-memory bank conflicts on every scan step


## Solution

Replace Blelloch shared-memory scan with warp shuffle-based two-level exclusive prefix sum: __shfl_up_sync performs per-warp inclusive scan without shared memory traffic, then cross-warp aggregation uses only 2 __syncthreads() barriers instead of 6, avoiding bank conflicts entirely within each warp-level step

## Key Lessons

- Replace shared-memory Blelloch scan with warp shuffle (__shfl_up_sync) based scan when scan_size fits within one or few warps — eliminates shared memory bank conflicts and reduces per-step traffic from load-compute-store to a single shuffle instruction
- For small scans (n <= WARP_SIZE), warp_exclusive_scan computes the exclusive prefix sum with zero __syncthreads() barriers; for larger scans, a two-level approach (per-warp scan + cross-warp aggregation) reduces total __syncthreads() count from O(2*log n) to O(2) regardless of scan size
- At very small scan sizes (n=8), the speedup from warp shuffle is modest (~6%) because __syncthreads() overhead dominates — the technique becomes more impactful as scan size grows within a warp or when used in hot-path kernels called repeatedly
