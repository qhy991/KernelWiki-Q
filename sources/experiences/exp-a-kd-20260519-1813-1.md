## Solution

Replace the compile-time constexpr pipe_stages=4 with a runtime stages parameter, allowing the caller to select stages=2 for INT8 workloads, halving the pipeline-related shared memory footprint across scales, B-matrix, and A-matrix buffers

## Key Lessons

- Parameterize shared memory pipeline depth as a runtime argument rather than a compile-time constant — this allows tuning stages based on activation type and GPU shared memory capacity without recompilation
- Reducing pipeline stages from 4 to 2 is viable for INT8 activations where the lower compute-to-memory ratio means fewer prefetch stages are needed to hide latency
