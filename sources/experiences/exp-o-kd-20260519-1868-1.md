## Solution

Fuse sigmoid+bias computation inline within the top-2 selection loop of a single kernel, eliminating the intermediate buffer allocation, removing one kernel launch, and keeping per-element sigmoid+bias results in registers instead of writing them to and reading them from global memory

## Key Lessons

- Fuse element-wise preprocessing (sigmoid, bias addition) into the consumer kernel loop when intermediate results are used only once — eliminates the global memory round-trip and kernel launch overhead, yielding compound speedup from reduced memory traffic and better pipeline occupancy
- Kernel fusion is especially beneficial when the intermediate buffer is large (O(num_tokens × num_experts)) and the per-element compute is lightweight (single SFU tanh instruction for sigmoid) — the global memory traffic of the two-kernel approach dominates over any scheduling benefit of separation
