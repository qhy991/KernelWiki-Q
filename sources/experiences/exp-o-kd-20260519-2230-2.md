## Challenge

The inner quantization loop divides each element by scale (FP division), which has ~4x lower throughput than FP multiplication on GPU, creating an instruction bottleneck in the hot per-element loop


## Solution

Replace per-element FP division (/ scale) with multiplication by precomputed reciprocal tmp_scale = 127.0f / block_absmax_val, amortizing the single division across all hidden_size elements while using higher-throughput FP multiply in the hot loop

## Key Lessons

- Replace per-element FP division with multiplication by a precomputed reciprocal (tmp_scale = 127.0f / absmax) in quantization loops — the single division is amortized across all elements while FP multiply has ~4x higher throughput than FP division on GPU hardware
