## Challenge

Baseline top-k sampling kernel used 1024-thread shared-memory tree reduction for vocab_size=160, k=8 - massively over-provisioned


## Solution

One warp (32 threads) per batch item; each thread loads 5 elements (160/32=5); k=8 rounds of warp-level argmax via __shfl_down_sync; no shared memory needed

## Key Lessons

- For small vocab top-k (vocab<=256), one warp per batch item with register-based storage beats shared-memory tree reduction by 2x+
- vocab_size=160 with 32 threads: each thread handles exactly 5 elements (160/32=5), perfect fit for unrolled register loads
- k=8 rounds of warp-level argmax (track used[] array per thread) is efficient for small k values
- __shfl_down_sync for warp-level max-finding eliminates all synchronization barriers
- Baseline over-provisioning (1024 threads for 160 elements) creates massive overhead that warp-level approach avoids
