## Solution

Add a compile-time branched fast path: when zero-points are float (is_zp_float), call sub_zp directly on frag_zpf[k%2][j] which already holds fp16 values from the vectorized int4 load, completely bypassing the dequant unpack+convert step and the intermediate frag_zp register array

## Key Lessons

- When zero-point values are already available in the target arithmetic precision (fp16), bypass the dequant-and-unpack path and consume them directly via a compile-time branched fast path — eliminates unpack+convert arithmetic and reduces register pressure in the matmul hot loop
- Use if constexpr with template predicates (is_zp_float, has_zp) to dispatch between integer and float zero-point handling with zero runtime overhead, keeping both paths type-safe and branch-free at the instruction level
