## Challenge

Blelloch exclusive prefix scan requires O(2*log N) __syncthreads() barriers across up-sweep and down-sweep phases, with indirect shared-memory addressing (offset*(2*tid+1)-1) that causes bank conflicts and irregular access patterns


## Solution

Replace the Blelloch algorithm with a two-level warp exclusive scan using __shfl_up_sync: each warp computes an intra-warp prefix sum via shuffle (no shared memory, no __syncthreads), then warp 0 performs a second-level scan over per-warp partial sums and each thread adds its warp's base offset, reducing total barriers from O(2*log N) to ~3

## Key Lessons

- Replace Blelloch shared-memory prefix scan with __shfl_up_sync-based warp scan when the data fits within a few warps — hardware shuffle eliminates shared-memory bank conflicts and reduces __syncthreads() barriers from O(2*log N) to O(num_warp_levels), yielding significant latency reduction for small-to-medium scan sizes
- Use a two-level scan structure for data spanning multiple warps: first compute per-warp partial scans via shuffle, then a single warp scans the partial sums, and each thread adds its warp's base offset — this keeps the shuffle advantage while handling arbitrary sizes
