## Solution

Replace stride-2 scalar __ldg loads with int4 (128-bit) vectorized loads that fetch multiple gate/up pairs in one transaction, and use int2 (64-bit) vectorized stores for output, exploiting the interleaved memory layout where gate and up values are naturally adjacent

## Key Lessons

- For interleaved data layouts (gate0,up0,gate1,up1,...), use int4/float4 vectorized loads instead of stride-2 scalar __ldg — one int4 fetches multiple gate/up pairs in a single 128-bit transaction, reducing memory instruction count by the vector width factor
- Use int2/int4 reinterpret_cast stores matching the output-to-input ratio (here 1 output per 2 inputs → int2 stores) to coalesce output writes and halve store instruction count
- Include an alignment check guard (16-byte for int4, 8-byte for int2) with a scalar fallback path to handle misaligned pointers safely without crashing
