## Solution

Remove the TopKSort (cub::BlockRadixSort) and its temp storage entirely, replacing the sort+store path with a direct striped-to-global store from smemIndices, since only indices are needed in the output and sorted order is not required

## Key Lessons

- When only unsorted indices are needed (not sorted logit values), the final radix sort pass can be eliminated entirely, saving both shared memory for temp storage and significant compute cycles — applies to top-K selection where ranking order is not required in the output
- Direct striped-to-global store from shared memory arrays avoids an extra local-register load/store round-trip through cub::BlockRadixSort, reducing register pressure and instruction count in the kernel epilogue
