## Solution

Use float4 (128-bit) vectorized loads/stores via the FLOAT4 macro to process 4 float elements per thread in a single memory transaction, reducing total memory transactions by 4x and improving memory bandwidth utilization for the memory-bound ELU operation

## Key Lessons

- Replace scalar float loads/stores with float4 vectorized loads/stores (128-bit) in memory-bound elementwise kernels to reduce memory transaction count by 4x — applies whenever the operation is memory-bound and elements are contiguous and 16-byte aligned
- Scale thread count down proportionally when vectorizing (e.g., 256 threads → 64 threads for x4) to maintain the same total element coverage while reducing scheduling overhead
- Use reinterpret_cast<float4*> casting (FLOAT4 macro) for coalesced vectorized access — each thread issues one 128-bit transaction instead of four 32-bit transactions, better saturating memory bandwidth
