## Solution

Materialize v_in[i] into a local register variable (vin_t src = v_in[i]) before passing to the inlined VecOp, intended to encourage NVCC to emit a single wide load instruction for the entire 8-byte vector pack

## Key Lessons

- Local-copy materialization of a vector-packed global-memory reference does not guarantee a single wide load when NVCC -O3 with __forceinline__ already inlines the operator and sees all element accesses — the compiler emits LDG.8 (or equivalent narrow-wide loads) regardless of the reference source for 8-byte pack widths
- Manual register materialization of vector types provides no measurable benefit at small pack widths (8 bytes / vec_n_t<half,4>) where a single LDG instruction already suffices; the technique is only worth considering when pack widths approach cache-line boundaries where the compiler might split loads
- A 2.3% timing delta on short memory-bound kernels is within normal measurement noise and should not be interpreted as a meaningful performance difference without statistical validation
