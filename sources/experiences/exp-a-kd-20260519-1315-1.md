## Challenge

Split-KV MLA decode attention requires two kernel launches (partial attention + reduction) with intermediate buffers for partial max/sum/output, adding kernel launch overhead and extra memory traffic for the reduction step


## Solution

Simplified to a single-pass kernel that iterates over the full KV sequence in one block, eliminating the reduction kernel and intermediate buffers; however, this reduced total thread blocks from 1024 (BATCH*HEADS*SPLIT_FACTOR) to 256 (BATCH*HEADS), causing severe SM underutilization on the 108-SM A800 that outweighs the reduction overhead savings

## Key Lessons

- Split-KV parallelism creates SPLIT_FACTOR× more thread blocks (1024 vs 256 for batch=16, heads=16, split=4), significantly improving SM occupancy on high-SM-count GPUs like A800 — applies when batch_size × num_heads is too small to saturate available SMs
- Kernel launch overhead and reduction cost can be a worthwhile trade-off when the parallelism gain significantly improves occupancy; at seq_len=1024 with batch=16, the 4× block count increase more than compensates for the extra reduction kernel
