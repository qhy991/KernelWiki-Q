## Solution

Accept lda as a separate kernel parameter and compute independent strides — input_row_stride from lda and output_row_stride from size_k — so input pointers advance by the actual physical memory layout while output pointers use the contiguous output stride, fixing the addressing without changing the per-thread work

## Key Lessons

- When kernel inputs may have a non-contiguous memory layout (lda > logical columns), always accept the leading dimension as a parameter and compute separate input/output strides — a single stride derived from logical width causes silent data corruption in the row addressing
- Column-permutation and gather kernels must distinguish between the logical data width (size_k) and the physical memory stride (lda) because the input and output buffers can have different padding assumptions
- A correctness fix that only changes address computation (no change to per-thread memory access count) has zero performance impact — the same number of loads and stores are issued, just at corrected addresses
