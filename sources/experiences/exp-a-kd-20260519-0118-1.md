## Solution

Reduce pipeline stages from 4 to 1 by computing kStages = min(max_stages, ceil(kMaxK/kKeysPerBlock)) = min(4, ceil(128/128)) = 1 for medium heads where kMaxK equals kKeysPerBlock, cutting shared memory per block from 32KB to 8KB and increasing occupancy from 5 to 20 blocks per SM while also reducing loop iterations and __syncthreads() barriers by 4x

## Key Lessons

- Right-size pipeline stage count to the actual K-dimension needed via kStages = min(max_stages, ceil(kMaxK / kKeysPerBlock)) — when kMaxK <= kKeysPerBlock, only 1 stage is sufficient and over-allocating stages wastes SMEM and slashes occupancy
- Shared memory per block directly controls occupancy: on sm_80 with 164KB SMEM, 4 stages x 8KB = 32KB allows only 5 blocks/SM while 1 stage x 8KB allows 20 blocks/SM — a 4x occupancy gain that dramatically improves latency hiding
- Fewer pipeline stages also proportionally reduces instruction count and __syncthreads() barriers, compounding the occupancy benefit with lower per-block execution overhead
