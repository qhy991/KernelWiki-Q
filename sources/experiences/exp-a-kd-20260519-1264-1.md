## Solution

Adapt tile shape to BM=8, BN=32 for small M workloads, doubling grid parallelism in y-dimension (grid_y=2 for M=16) and widening the x-dimension for better memory coalescing on B-matrix reads (32 consecutive columns per warp vs 16)

## Key Lessons

- M-adaptive tile shapes (wider BN, shorter BM) that benefit CUTLASS tensor-core warp-level tiling do not automatically help simplified shared-memory tiled kernels — the wider tile increases shared memory for s_B (2048 vs 1024 bytes) and cooperative loading overhead without the corresponding warp-level efficiency gain
- Tile shape changes that improve memory coalescing (32 vs 16 consecutive B elements per warp) may still regress if the added shared memory loading cost and reduced per-block compute density outweigh the coalescing benefit
- CUTLASS-style tile shape tuning is architecture-specific: WarpShape<32,64,64> reduces wasted warp computation for small M on tensor cores, but a naive one-element-per-thread kernel cannot replicate this benefit — the optimization is coupled to the MMA pipeline, not just the tile geometry
