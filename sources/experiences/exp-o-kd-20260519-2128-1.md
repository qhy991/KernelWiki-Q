## Solution

Store KV cache in FP8-E5M2 (1 byte) instead of FP16 (2 bytes) by adding a separate cache_t template parameter and converting each half element to FP8 via __nv_cvt_halfraw_to_fp8 before writing, halving write traffic with minimal conversion overhead from the hardware instruction

## Key Lessons

- For write-bandwidth-bound kernels, reducing the per-element store size (e.g., FP16→FP8 via __nv_cvt_halfraw_to_fp8) directly reduces DRAM traffic — the hardware FP8 conversion adds negligible overhead compared to the bandwidth savings, achieving near-theoretical speedup
- Use a separate cache_t template parameter with if constexpr dispatch to make precision downcasting a compile-time decision with zero cost when disabled, keeping the original path bit-identical to the unoptimized version
