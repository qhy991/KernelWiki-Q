## Solution

Replace the conditional branch with inline PTX using predicated mov (@p mov.u32 m, 0) to conditionally zero each mask without branching — however, three separate asm volatile blocks act as optimization barriers, prevent the compiler from merging the clears, add register pressure (.reg .pred and .reg .u32 per block), and total ~18 PTX instructions versus the compiler's efficient predicated store sequence from a single if-statement

## Key Lessons

- Inline PTX asm volatile blocks act as compiler optimization barriers that prevent merging or reordering across sequential operations — the compiler's native predicated code generation for a simple if-statement grouping multiple assignments is often more efficient than hand-written PTX when warp divergence is low
- Branchless predicated-mov patterns are only beneficial when branch divergence is severe and dominates instruction cost; when warp threads are mostly uniform (e.g., 2/3 taking the same path), the compiler's optimized branch code is significantly faster
- Multiple inline asm blocks for related conditional operations (clearing m0, m1, m2 together) each allocate their own .reg temporaries, increasing register pressure compared to a single if-statement that the compiler can optimize as a group — prefer letting the compiler handle groups of conditional assignments
