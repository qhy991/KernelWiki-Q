## Challenge

2D nested loop iteration maps rows to blockIdx.x and columns to threadIdx.x; when numRows is small (e.g., 8) the grid is capped at 8 blocks, leaving an A800's 108 SMs mostly idle despite a large total workload (numRows * colsPerRow)


## Solution

Replace the 2D nested loop with a flat 1D global thread index that strides across all numRows*colsPerRow elements by gridDim.x*blockDim.x, deriving rowIdx via integer division and colIdx via modulo — this decouples thread count from numRows and enables full SM utilization

## Key Lessons

- Flatten 2D nested loops (row-by-blockIdx, col-by-threadIdx) into a single 1D global-index stride when one dimension is small — this decouples launched thread count from the small dimension and enables full SM occupancy
- Prefer globalIdx = blockIdx.x*blockDim.x+threadIdx.x with stride gridDim.x*blockDim.x over hierarchical 2D dispatch when work dimensions are asymmetric (many columns, few rows)
