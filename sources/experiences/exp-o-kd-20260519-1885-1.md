## Challenge

The group accumulation loop uses a runtime variable n_group as its bound, preventing the compiler from unrolling the 16-iteration loop and forcing it to emit branch, counter-increment, and comparison instructions on every iteration


## Solution

Convert n_group from a runtime parameter to a compile-time template parameter NGroup and add #pragma unroll on the group loop, enabling the compiler to flatten all 16 iterations into straight-line code and interleave independent memory loads across iterations

## Key Lessons

- Convert known loop-count parameters to compile-time template constants so the compiler can fully unroll loops — eliminates per-iteration branch/counter overhead and enables instruction-level parallelism across iterations, especially effective when the loop body is lightweight (few ALU ops per iteration)
- Combining a compile-time bound with #pragma unroll gives the compiler both the permission and the information to flatten the loop, allowing it to interleave independent memory loads from different iterations and better hide memory latency
