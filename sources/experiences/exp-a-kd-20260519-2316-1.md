## Solution

Change k_scale/v_scale from scalar by-value float parameters to const float* pointers dereferenced inside the loop body (*k_scale, *v_scale), enabling per-head dynamic scaling via pointer indirection while the NVCC compiler caches the constant-address global load into a register across iterations so no performance regression occurs

## Key Lessons

- Replacing a scalar by-value kernel parameter with a pointer dereferenced inside a loop does not degrade performance when the pointer targets a constant address — the NVCC compiler hoists the repeated global-memory load into a register or L1 cache entry, producing identical loop-body register usage
- Passing scale factors as const float* pointers instead of scalar floats enables per-head or per-block dynamic scaling (the pointer can point into a per-head scale array) without sacrificing the performance of the static-scale case — prefer pointer parameters when API flexibility is needed and the dereferenced value is loop-invariant
