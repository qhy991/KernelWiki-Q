## Challenge

Scalar GELU kernel processes only one half (16-bit) value per thread, leaving each 32-bit coalesced memory segment half-utilized and wasting available memory bandwidth on a memory-bound elementwise kernel


## Solution

Vectorize to half2 processing: each thread loads a full 32-bit half2 (two adjacent half values) via the HALF2 reinterpret_cast macro, computes GELU on both lanes, and stores back as half2 — filling coalesced segments completely and halving the number of memory transactions

## Key Lessons

- Process 2 half values per thread via half2 reinterpret-cast load/store to fill the 32-byte minimum coalesced segment — halves memory transaction count and nearly doubles throughput on memory-bound elementwise kernels
- Scale block size down by the vectorization factor (blockDim.x / elements_per_thread) to maintain the same logical grid coverage while improving per-thread memory efficiency
- Use .x/.y field access on half2 registers to apply scalar operations per lane when no native half2 operator exists for the computation (e.g., custom GELU tanh approximation)
