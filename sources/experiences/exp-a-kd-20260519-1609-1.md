## Challenge

Host-side stride computation and data initialization use NH (num_query_heads) instead of NKV (num_kv_heads) for block_stride and v_cache element count, which would be incorrect when generalizing to grouped-query attention (GQA) where NH != NKV


## Solution

Replace NH with NKV in block_stride computation, v_cache allocation size, and host initialization loops so strides are parameterized by the actual KV head count, enabling correct behavior for GQA while remaining identical for MHA (NH==NKV==8)

## Key Lessons

- Use NKV (num_kv_heads) rather than NH (num_query_heads) for v-cache stride and allocation computations — this correctly handles GQA where NH != NKV and is a zero-cost abstraction for MHA where they are equal
- Verifying stride parameterization correctness in the MHA case (NH==NKV) before deploying to GQA provides confidence that the refactoring preserves both correctness and performance
