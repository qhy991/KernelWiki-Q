## Solution

Cache input data in per-warp shared memory during Pass-1, then read from shared memory in Pass-2 instead of reloading from global memory, eliminating one full global memory read pass and reducing total global memory traffic by approximately 50%

## Key Lessons

- In multi-pass memory-bound kernels where the same data is accessed in multiple passes, cache the data in shared memory between passes to eliminate redundant global memory loads — shared memory bandwidth (~5 TB/s) is roughly 100x higher than global memory (~50-80 GB/s)
- Use __syncwarp() as a lightweight barrier to ensure warp-level shared memory writes are visible before the next pass reads, avoiding the heavier __syncthreads() when each warp owns its own smem region
- Pad shared memory allocations to bank boundaries (smem_padding = 32 for 32 banks × 4 bytes) and give each warp its own contiguous smem region to avoid bank conflicts across warps — this matters most when the per-token data footprint exceeds L1 cache capacity
