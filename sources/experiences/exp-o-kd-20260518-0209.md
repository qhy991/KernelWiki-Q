## Challenge

The before code performs a full linear scan over 64 expert offsets stored in global memory, reading expert_offsets[i] and expert_offsets[i+1] from global on each iteration (up to 64 global memory loads per element)


## Solution

The after code caches all 65 offsets in shared memory via int4-vectorized cooperative loading at block start, then does binary search over the shared-memory array. ## Key Lessons

- Binary search needs only log2(64)=6 comparisons instead of up to 64, and each comparison reads from shared memory (5ns latency) rather than global memory (~400ns latency).
