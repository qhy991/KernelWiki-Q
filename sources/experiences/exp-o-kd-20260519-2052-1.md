## Solution

Replace the unconditional clone with a lazy clone pattern: initialize topk_ids_for_sort as a shallow reference to topk_ids, then clone only inside the expert_map.has_value() branch where preprocessTopkIdLauncher actually mutates the tensor to remap global expert IDs to local ones

## Key Lessons

- Defer GPU tensor clones to only the code paths that actually mutate the data — a shallow reference (direct tensor assignment) suffices for read-only consumers, eliminating both the GPU memory allocation and device-to-device memcpy on the common path
- When a tensor is used read-only in the common case but mutated conditionally, initialize with a shallow alias and clone lazily inside the mutation branch — applies to any MoE routing code where expert remapping is conditional on expert parallelism
