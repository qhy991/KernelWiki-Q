## Challenge

In a MoE sorted-token A-matrix lookup kernel, integer division by top_k is recomputed on every row access inside the inner loop, costing ~20 GPU instruction cycles per division instead of a cheap shared memory read (~5 cycles)


## Solution

Pre-compute all sorted_id / top_k divisions once during the shared-memory loading phase into a separate array (sh_rd_block_sorted_ids), then replace every per-lookup integer division with a simple shared memory read, amortizing the one-time division cost across all iterations

## Key Lessons

- Pre-compute invariant integer divisions once into shared memory rather than recomputing inside hot loops — GPU integer division costs ~20 cycles vs ~5 cycles for a bank-conflict-free shared memory read, so hoisting division out of repeated lookups yields significant speedup when the division result is reused across iterations
- When adding a pre-computed shared memory array, use a single extern __shared__ allocation and partition it via pointer offsets (e.g., sh_data and sh_data + num_rows) to avoid dynamic shared memory allocation limits and keep memory layout contiguous
