## Solution

Replace scalar __half loads with 128-bit vectorized loads via uint4 reinterpret_cast, fetching 8 fp16 elements per LDG.128 transaction and reducing total memory transactions by 8x while keeping the per-element max computation identical

## Key Lessons

- Use uint4 (128-bit) vectorized loads via reinterpret_cast to coalesce 8 fp16 elements into a single LDG.128 transaction — reduces memory transaction count by 8x for bandwidth-bound reduction kernels where compute per element is trivial
- The vectorization factor equals 16 bytes / sizeof(element): fp16 and bf16 both give vec_size=8, yielding identical speedup ratios because the bottleneck is memory transaction count, not compute
