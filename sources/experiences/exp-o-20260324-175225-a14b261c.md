fp32_top_k_sampling_llama3_8b_k8

## Solution

Three-stage optimization: (1) Fix compilation by writing complete self-contained kernel with warp-per-batch + cudaDeviceSynchronize. (2) Remove cudaDeviceSynchronize to reduce ~1ms overhead. (3) Switch to block-per-batch with shared memory inter-warp reduction.

## Key Lessons

- Never call cudaDeviceSynchronize() in the extern C entry point - the test framework handles synchronization and this adds ~1ms overhead
- For top-k sampling with large vocab (128256), block-per-batch is significantly faster than warp-per-batch because it allows more threads to work on each batch element in parallel
- Shared memory inter-warp reduction is required for block-per-batch: each warp writes its local top-k to shared memory, then warp 0 does final reduction
- The auto_wrapper expects topk_kernel(const float* probs, int64_t* samples, int batch_size, int vocab_size, int k) - parameter name 'samples' not 'indices'
- When baseline_comparison is null in test results, check the Baseline/ directory for pre-computed *_results.json files to get reference performance numbers
