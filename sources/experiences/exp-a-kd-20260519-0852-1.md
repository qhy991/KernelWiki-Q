## Solution

Replace Blelloch shared-memory scan with a warp-shuffle (__shfl_up_sync) based two-level exclusive prefix sum: intra-warp scan via shuffle reduction eliminates shared-memory bank conflicts, and a single inter-warp aggregation pass collects warp-level partial sums

## Key Lessons

- Warp shuffle (__shfl_up_sync) based exclusive scan eliminates shared-memory bank conflicts and reduces per-step shared memory traffic compared to Blelloch scan — applies when scan_size fits within or near a single warp (<= 32 elements)
- For small scan sizes (8 elements), __syncthreads() barrier overhead dominates execution time regardless of algorithm choice; warp shuffles provide modest gains because the barrier count remains similar — expect larger speedups only when the shuffle path meaningfully reduces barrier count
- Two-level warp scan (intra-warp via shuffle + inter-warp aggregation) scales to scan sizes beyond one warp while preserving the shuffle advantage within each warp — use when scan_size > WARP_SIZE but still small enough for shared-memory warp_sums aggregation
