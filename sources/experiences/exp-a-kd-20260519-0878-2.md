## Challenge

Scalar padding kernel writes sorted_token_ids one int32 at a time in a stride loop, issuing one STG instruction per element and underutilizing available memory bandwidth for the fill operation


## Solution

Replace scalar int32 stores with 4-wide AlignedArray<int32_t,4> vectorized stores via reinterpret_cast<Vec*> on the output pointer, issuing one 16-byte store per iteration instead of four 4-byte stores, reducing memory transaction count by approximately 4x

## Key Lessons

- Use AlignedArray<T,N> with reinterpret_cast on output pointers to emit vectorized stores (4-wide int32 = 16 bytes per transaction) instead of scalar element writes — reduces memory transaction count by N× for fill/set patterns
- Construct a vector fill constant once (broadcast fill value into all lanes) then assign the whole vector per iteration, rather than writing individual scalars — the compiler can leverage wider STG instructions
