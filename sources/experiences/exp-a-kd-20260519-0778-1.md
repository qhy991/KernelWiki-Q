## Challenge

In the MLA key concatenation kernel, each warp reads the same per-token k_rope data (64 bf16 elements, 128 bytes) from global memory 16 times inside the unrolled HEAD_CHUNK_SIZE loop, issuing redundant loads that could potentially be served by a single register-cached value


## Solution

Introduce a register buffer (k_rope_buf) loaded once before the unrolled loop, then broadcast the register value to all 16 head iterations instead of re-reading k_rope from global memory each time

## Key Lessons

- Register buffering of small repeatedly-accessed data eliminates redundant global memory reads, but yields no measurable benefit when the data already stays resident in L2 cache across the unrolled loop — the 128-byte per-token k_rope fits comfortably in L2 at 2048 tokens
- In isolated benchmarks with low L2 pressure, the compiler may already optimize repeated reads of the same address into register reuse, making explicit register buffering redundant; the optimization becomes valuable only under real-workload L2 contention from large KV caches
