## Solution

Convert pipe_stages from a compile-time constexpr to a runtime parameter (stages), allowing INT8 kernels on Turing to use stages=2 instead of 4, which halves the shared memory allocated to all pipeline buffers (scales, B-matrix)

## Key Lessons

- Convert compile-time pipeline depth constants to runtime parameters to allow per-GPU and per-precision tuning — fewer stages reduce shared memory pressure on GPUs with limited shared memory (e.g., Turing 64KB) without sacrificing throughput for lower-precision activations
