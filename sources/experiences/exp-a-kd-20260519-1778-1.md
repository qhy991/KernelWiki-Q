## Solution

Swap-AB reorders GEMM operands to D = B^T * A^T, which on SM90 hardware can improve TMA tile load efficiency by better aligning the leading-dimension access pattern, but on SM80 PyTorch's cuBLAS backend already selects optimal kernels for both operand orderings so no measurable gain occurs

## Key Lessons

- Swap-AB operand reordering targets SM90 TMA (Tensor Memory Accelerator) tile load efficiency — it provides no benefit on SM80 or earlier architectures that lack TMA hardware, so always verify the target compute capability before applying this technique
- High-level frameworks like PyTorch delegate to cuBLAS which already selects optimal kernel variants for both A*B and B^T*A^T operand orderings, making manual swap-AB redundant when operating through these abstractions
