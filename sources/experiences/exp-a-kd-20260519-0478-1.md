## Solution

Consolidate four separate half2 loads/stores into a single 128-bit LDST128BITS (float4 reinterpret) load/store using a local half[8] pack array, replacing eight scalar .x/.y field assignments with a #pragma unroll loop and four separate bounds checks with one

## Key Lessons

- NVCC at -O3 can coalesce four consecutive half2 loads from aligned contiguous addresses (x[idx+0], x[idx+2], x[idx+4], x[idx+6]) into a single 128-bit load transaction, making explicit LDST128BITS consolidation a code-clarity improvement rather than a performance win — verify with profiling before assuming benefit
- Replacing four separate bounds-checked stores with a single bounds-checked 128-bit store reduces instruction count but the savings are negligible in memory-bound kernels where load/store latency dominates over branch overhead
- LDST128BITS (reinterpret_cast<float4>) is a portable pattern for expressing 128-bit vectorized memory ops in CUDA without relying on hardware-specific intrinsics — use it for code cleanliness even when the compiler already coalesces smaller accesses
