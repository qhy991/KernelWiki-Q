## Solution

Remove the offset_q = N_K - N_Q term from causal_trip_count by switching from CausalMask<false> (IsQBegin=false) to CausalMask<true> (IsQBegin=true), so max_blocks_q = ceil_div((blk+1)*tile_q, tile_k) correctly produces a triangular visit pattern where Q-block b visits only b+1 K-blocks

## Key Lessons

- When Q is at the beginning of the sequence (causal_q_begin=true), use CausalMask<true> without the N_K-N_Q offset — this eliminates unnecessary K-block visits and produces the correct triangular attention pattern
- The offset_q = N_K - N_Q term is only needed when Q starts mid-sequence (IsQBegin=false); applying it unconditionally when Q is at position 0 inflates trip counts by an order of magnitude
- Causal attention trip count must equal blk_coord_q+1 K-blocks for the first Q-tile, forming a triangular workload — verify the dispatch logic against this invariant to catch mask misapplication
