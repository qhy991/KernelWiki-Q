## Solution

Fuse both GEMMs into a single cutlass::gemm::device::B2bGemm kernel where GEMM0's epilogue output stays on-chip in the register file (RF residency) and is directly consumed as GEMM1's A-matrix input, eliminating the intermediate global memory round-trip entirely

## Key Lessons

- Use cutlass::gemm::device::B2bGemm to fuse consecutive GEMMs where one's output feeds the next's input — the register-file residency keeps the intermediate on-chip and eliminates global memory round-trips, which is most beneficial when intermediate tensors are large relative to compute
- B2bGemm requires smaller ThreadblockShapes and WarpShapes for GEMM0 (e.g., 64x64 vs 128x64) because GEMM0's partial output must fit in registers before GEMM1 consumes it — the tile size is constrained by RF capacity, not shared memory
- Kernel fusion speedup diminishes at large problem sizes (1.48x here vs 2.15x on medium) because compute-bound kernels already saturate SM throughput; fusion helps most when memory traffic is the bottleneck relative to arithmetic intensity
