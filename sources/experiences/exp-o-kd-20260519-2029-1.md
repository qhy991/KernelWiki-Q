## Solution

Remove the host-side torch::stable::zero_() call entirely and expand the kernel grid to cover all TMA-aligned padding groups (num_groups_padded = tma_aligned_mn * padded_groups_per_row), so padding threads write zero bytes directly within the quantization kernel, fusing initialization into a single pass

## Key Lessons

- Fuse output buffer initialization into the computation kernel by expanding the grid to cover padding groups that write zeros — eliminates a separate memset kernel launch and full DRAM write pass, yielding measurable speedup whenever the kernel already touches most output bytes
- When TMA alignment introduces padding in the output tensor, extend the kernel grid to cover padded elements rather than pre-zeroing the entire buffer — the extra thread work for padding groups (one zero byte write each) is negligible compared to the removed memset overhead
