## Challenge

Both before and after use the same core algorithm: each thread scans its assigned elements to find local top-2, then a warp-level max reduction finds max1, ballot_sync/popc checks uniqueness of max1, and if max1 is unique a second warp reduction finds max2 by replacing the max1-holding lane's val...


## Solution

the after code uses cg::reduce(tile, val, cg::greater). The underlying hardware instructions are equivalent (both compile to warp shuffle operations), so the measured latency is nearly identical. The cg::reduce abstraction adds no measurable overhead or benefit for this specific reduction pattern...

## Results

- **speedup**: 1.009x (neutral)


## Key Lessons

- The cg::reduce abstraction adds no measurable overhead or benefit for this specific reduction pattern at warp tile granularity.
