## Challenge

Per-group absmax reduction uses shared memory ping-pong with 5 __syncthreads() barriers (4 during reduction stages plus 1 for broadcast), incurring shared memory read/write traffic and barrier synchronization overhead for each group of 16 threads


## Solution

Replace shared memory ping-pong reduction with warp-level __shfl_xor_sync butterfly reduction that exchanges register values directly within a 16-thread subgroup using mask 0xffff, eliminating all shared memory allocation, writes, reads, and __syncthreads() barriers while naturally broadcasting the result to all participating threads

## Key Lessons

- Warp shuffle (__shfl_xor_sync) replaces shared memory reduction with register-to-register exchange, eliminating __syncthreads() barriers and shared memory traffic — applies when the reduction group fits within a single warp (≤32 threads)
- For subgroup sizes smaller than 32 (e.g., 16 threads), use a partial warp mask (0xffff for 16 lanes) with __shfl_xor_sync to avoid interference between independent subgroups sharing a warp
- When global memory access dominates kernel latency (e.g., group_size/threads_per_group = 8 elements per thread), replacing shared memory reduction with warp shuffle yields only marginal speedup since the reduction overhead is a small fraction of total execution time
