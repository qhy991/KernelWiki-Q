## Challenge

The quantization epilogue loop computes quantized values twice: first converting to half and storing into a local float4 variable that is never written to global memory, then re-reading the same input elements to compute int8 output, doubling per-element arithmetic and wasting registers on dead stores


## Solution

Remove the dead half-precision quantization loop entirely, keeping only the int8 output path — eliminating redundant __float2half conversions, dead stores to the local float4 variable, and duplicated element-wise arithmetic

## Key Lessons

- Dead stores to local variables that are never read back or written to global memory may be eliminated by the compiler optimizer, but should still be removed at source level to reduce register pressure and avoid relying on optimizer behavior in register-constrained kernels
- When changing output dtype (e.g., half to int8), remove the old dtype path entirely rather than leaving both paths — the NVCC optimizer may not always eliminate dead stores in complex template kernels, and the residual code confuses maintainers
