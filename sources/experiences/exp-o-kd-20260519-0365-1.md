## Challenge

The embedding kernel unrolls 4 float load/store operations as separate scalar assignments (4 loads + 4 stores = 8 memory instructions), which the compiler may not merge into a single 128-bit wide transaction, resulting in potentially 4 separate 4-byte memory accesses instead of 1 combined 16-byte access


## Solution

Use the LDST128BITS macro (reinterpret_cast<float4*>) to perform a single 128-bit packed load and store, explicitly telling the compiler to emit one wide 16-byte memory transaction instead of four separate 4-byte scalar transactions, reducing memory instruction count from 8 to 2

## Key Lessons

- Use LDST128BITS (reinterpret_cast<float4*>) packed load/store to replace manually unrolled 4-element scalar float sequences — it guarantees a single 128-bit memory transaction instead of relying on the compiler to merge narrow accesses, reducing instruction count from 8 to 2
- Adjust thread indexing from threadIdx.x * 4 to threadIdx.x with 4 * tx stride when switching to packed access, so each thread naturally handles one 128-bit chunk instead of four scalars
