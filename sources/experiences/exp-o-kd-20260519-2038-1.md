## Solution

Add an lda parameter and compute separate input_row_stride (lda-based) and output_row_stride (size_k-based) so input pointers advance by the actual memory stride including padding while output pointers use the contiguous output stride

## Key Lessons

- Never assume input and output share the same row stride — always accept an explicit lda (leading dimension) parameter so the kernel handles both contiguous (lda == size_k) and padded (lda > size_k) memory layouts correctly
- When indexing through int4 (16-byte) pointers, compute row strides in int4 units (lda * sizeof(half) / 16) rather than element counts, since the pointer arithmetic is in int4 granularity not half granularity
- A correctness fix that separates input/output strides has negligible performance cost because both versions perform the same number of memory transactions — the only difference is the address arithmetic
