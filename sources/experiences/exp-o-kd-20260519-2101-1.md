## Solution

Replace the scalar loop with lop3.b32 PTX instructions to extract two pairs of 4-bit values simultaneously and combine them with a magic number (0x64006400) for half-precision representation, followed by sub.f16x2/fma.rn.f16x2 packed arithmetic for final scaling, processing all 8 values in 4 lop3 + 4 half2 instructions instead of 8 scalar iterations

## Key Lessons

- Use lop3.b32 inline PTX with a lookup-table immediate (immLut) to perform bitwise extract-and-transform in a single instruction, eliminating shift-mask-convert sequences when mapping packed int4 values to fp16 bit patterns — applies to any packed low-precision integer to half-precision conversion on SM 75+
- Pack half2 arithmetic (sub.f16x2, fma.rn.f16x2) to process two fp16 values per instruction, doubling throughput for elementwise dequantization — applies whenever the computation is elementwise and operands are naturally paired in half2 layout
