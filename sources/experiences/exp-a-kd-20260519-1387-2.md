## Solution

Expand the kernel grid to cover TMA-aligned padding groups (num_groups_padded) so every scale byte — both valid and padding — is written exactly once inside the kernel; padding groups write zero via the is_valid_group guard, fully eliminating the separate cudaMemsetAsync

## Key Lessons

- Fuse buffer zero-initialization into the computation kernel by expanding the grid to cover all output slots including padding — each thread writes its destination byte exactly once, removing a separate cudaMemsetAsync launch and a full DRAM write pass
- When packing sub-word fields into an aligned buffer with TMA padding requirements, iterate over the padded dimension and use a validity guard (is_valid_group) to distinguish real work from zero-fill, avoiding separate initialization overhead
