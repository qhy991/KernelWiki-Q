## Solution

Conditional dispatch selects a smaller-tile pingpong kernel (BM=8, BK=8, BN=16) for M≤64, attempting to improve occupancy through reduced shared memory and smaller thread blocks; however, on SM80 the approach regresses 0.59× because the hardware lacks warp-specialized TMA async loads and UMMA MMA/epilogue overlap needed to compensate for reduced tile size

## Key Lessons

- Reducing tile BK from 16 to 8 doubles K-loop iterations, adding proportionally more __syncthreads barriers and shared-memory load/store phases — on architectures without async loads, the sync overhead outweighs any occupancy gain from smaller blocks
- A conditional threadIdx guard (threadIdx.x < PP_BK) for A-matrix loading leaves threads idle during that phase; without dedicated load warps or TMA hardware, partial thread utilization directly reduces per-phase memory throughput
- SM120 CUTLASS pingpong optimization relies on hardware-specific warp-specialized TMA async loads and UMMA MMA/epilogue overlap — naively porting only the smaller tile sizes to SM80 without these mechanisms produces a net regression
