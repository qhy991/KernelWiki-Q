## Solution

Replace __shfl_down_sync with __shfl_xor_sync parameterized by a kNumThreads template (kNumThreads=8), computing the starting mask as kNumThreads/2=4 so only 3 productive butterfly rounds execute (mask 4→2→1), eliminating 2 wasted shuffle instructions per reduction

## Key Lessons

- Use __shfl_xor_sync instead of __shfl_down_sync for sub-warp (power-of-2 < 32) reductions — XOR butterfly shuffles naturally respect group boundaries and require exactly log2(groupSize) productive rounds, while __shfl_down_sync with width parameter may execute NOP rounds when the delta exceeds the group size
- Parameterize sub-warp reduction with a kNumThreads template to derive the starting mask as kNumThreads/2 — this makes the shuffle count correct for any power-of-2 group size and eliminates dead shuffle instructions at compile time
