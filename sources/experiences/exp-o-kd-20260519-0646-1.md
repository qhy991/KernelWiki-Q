## Solution

Fuse top-K selection and offset addition into a single kernel that writes intermediate indices to shared memory instead of global memory, adds the per-row offset inline, and writes final adjusted indices directly to global memory, eliminating one kernel launch and the intermediate global-memory round-trip

## Key Lessons

- Fusing a producer-consumer kernel pair where the intermediate data (top-K indices) fits in shared memory eliminates one kernel launch and a global-memory round-trip — applies whenever the intermediate tensor is small (e.g., K values per row) and both kernels operate on the same per-row granularity
- Routing scatter writes through shared memory instead of global memory avoids uncoalesced global stores during the radix refinement stages of top-K selection, improving memory throughput even when the compute-bound scan dominates overall latency
- Kernel fusion savings diminish as the compute-heavy phase lengthens — the fixed-cost savings (one launch + one global round-trip) become a smaller fraction of total latency as input length grows (18.95% at len=4096 vs 11.74% at len=16384)
