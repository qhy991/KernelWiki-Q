## Challenge

The after code replaces a global memory load of the per-group scale with a shared memory broadcast. In the before version, thread 0 writes the scale to global memory and all other threads issue a global memory load to read it back, which has hundreds of cycles of latency and consumes L2 cache ban...


## Solution

In the after version, thread 0 writes the scale to shared_max[0] in addition to global memory, and after __syncthreads(), all threads read from shared memory, which completes in a few cycles. - **speedup**: 1.818x (improved)


## Key Lessons

- The speedup comes from eliminating the global memory access on the critical path between the scale computation and the quantization step.
