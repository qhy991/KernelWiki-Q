## Solution

Remove the outLogits output pointer, the topkVals shared memory array, and all corresponding stores so the kernel only writes indices to global memory — eliminating redundant global stores and shared memory occupancy

## Key Lessons

- When downstream consumers only need indices (not values), remove all value-output paths — eliminating global memory writes for unused data reduces store traffic and shared memory footprint without affecting correctness
- Audit kernel output parameters against actual consumer usage: a parameter that was once needed may become dead after a pipeline refactor, and stale outputs impose a hidden memory-bandwidth tax
