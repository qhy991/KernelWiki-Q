## Solution

Replace runtime cudaGetDeviceProperties() lookup with a compile-time constexpr int kMaxVecBytes = 16 constant, removing the per-iteration API call entirely; the GPU kernel code is unchanged since the runtime value was never passed to the kernel

## Key Lessons

- Move architecture-dependent constants (e.g., max vector bytes based on compute capability) to compile-time constexpr or template parameters instead of querying cudaGetDeviceProperties per iteration — the API call adds host-side overhead invisible to CUDA event timing but measurable in end-to-end latency
- cudaEvent-based benchmarking measures GPU-side execution only; host-side optimizations like removing redundant API calls will not show improvement in event timings even when they reduce real wall-clock dispatch overhead
