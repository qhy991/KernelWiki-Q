## Solution

Replace separate (q & LO) | EX with an explicit inline PTX lop3.b32 instruction using a compile-time lookup table, guaranteeing the 3-input logical operation is emitted as a single instruction regardless of compiler optimization level

## Key Lessons

- NVCC at -O3 already fuses simple (a & b) | c patterns into LOP3 instructions, so explicit PTX lop3.b32 yields no throughput gain when the compiler optimization is active — the benefit is in guaranteeing instruction selection across all compilation contexts and optimization levels
- LOP3 lookup table (0xf0 & 0xcc) | 0xaa encodes the truth table for (a & b) | c — use compile-time constexpr computation to make the intent readable while the assembler receives the numeric LUT
