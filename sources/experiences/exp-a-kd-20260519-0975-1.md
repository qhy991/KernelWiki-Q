## Solution

Hoist the loop-invariant scale computation (null-pointer check, global memory load, and reciprocal division) outside the topk expert loop so it executes once per token instead of once per expert iteration, eliminating 7 redundant divisions per token

## Key Lessons

- Hoisting loop-invariant computations (division, global loads, null checks) outside inner loops is correct but may yield no measurable speedup when the saved operations are dwarfed by the loop body's arithmetic and memory traffic — 7 divisions vs 2048 multiply-load-store per token
- Sub-0.01ms CUDA kernels are dominated by launch overhead and event timing noise; micro-optimizations like eliminating a few scalar divisions cannot be reliably measured at this scale without amortizing over more work per launch
