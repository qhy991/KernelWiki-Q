## Challenge

The f16x8 kernel loads 8 half elements via four separate half2 registers and stores results with four independent boundary-checked half2 writes; replacing this with a single 128-bit LDST128BITS transaction plus local half arrays pack_x[8]/pack_y[8] introduces register spills to local memory and collapses four fine-grained boundary checks into one coarse check that may skip the entire store


## Solution

Replace four named half2 registers and individual HALF2 loads/stores with local half[8] arrays and a single LDST128BITS (float4) 128-bit packed load/store plus a loop, aiming to reduce load/store instruction count — but the local arrays cause register spills to local memory and the single boundary check loses fine-grained partial-tile correctness

## Key Lessons

- Named half2 registers keep 8 half values in GPU registers with zero spills; local half[8] arrays may be spilled to local memory by the compiler when register pressure is high, adding hidden load/store overhead that negates the benefit of wider transactions — prefer named vector registers for small element counts
- Wider 128-bit loads/stores only help when the entire pipeline (register allocation, computation, boundary handling) benefits; collapsing four fine-grained boundary checks into one coarse check can skip valid output writes at tile boundaries, and the register-spill cost of local arrays can outweigh the instruction-count savings
