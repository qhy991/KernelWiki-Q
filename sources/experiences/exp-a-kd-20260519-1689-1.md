## Solution

Guard the entire allocation, memset, and kernel launch sequence behind a needs_alignment runtime flag; when the CUTLASS group gemm path is active (needs_alignment=false), all GPU work is skipped entirely, eliminating per-call cudaMalloc+cudaMemset+launch+cudaFree overhead

## Key Lessons

- Guard GPU allocation and kernel launch behind a runtime condition when the computation is path-dependent — when the condition is false, the entire cudaMalloc+cudaMemset+launch+cudaFree overhead is eliminated, yielding large speedups because allocation fixed cost dominates regardless of buffer size
- In MOE frameworks with multiple backend paths, alignment preprocessing for CUTLASS group gemm can be skipped entirely when the backend handles alignment internally — a needs_alignment flag avoids redundant GPU work and allocator pressure
