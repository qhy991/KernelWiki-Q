Metal FP16 MatMul baseline implementation - Round 1/3 of AutoResearch

## Key Lessons

- float16_t is not a standard C++ type - use float on host, half in Metal kernel
- FP16 precision loss is significant for large matrix multiplication - tolerance of 2e-2 is appropriate
- Metal kernel accepts float buffers but can convert to half internally for FP16 computation
- FP32 accumulation in Metal kernel helps maintain accuracy while using FP16 inputs
