## Challenge

KV cache quantization uses a single shared kv_scale sized for the wider key range [-4,4], forcing the smaller value range [-0.5,0.5] to use only ~16 of the available ~127 int8 quantization levels, severely degrading value reconstruction accuracy


## Solution

Replace the single shared kv_scale with separate k_scale and v_scale, each tuned to its respective data range (k_scale=4.0/127 for keys, v_scale=0.5/127 for values), so each tensor utilizes the full int8 quantization dynamic range and reconstruction error is minimized

## Key Lessons

- Use per-tensor quantization scales matched to each tensor's actual value range rather than a single shared scale — this maximizes utilization of the int8 dynamic range and minimizes reconstruction MSE without affecting kernel throughput
- A quantization-accuracy optimization (reducing MSE) can be orthogonal to kernel-speed optimization — changing only the scale constants preserves identical instruction count, memory bandwidth, and pipeline behavior, resulting in neutral runtime impact
- When key and value tensors have significantly different magnitude ranges (e.g., keys [-4,4] vs values [-0.5,0.5]), a shared scale sized for the wider range wastes most quantization levels on the narrower-range tensor (~16 of 127 levels in this case)
