## Solution

Replace synchronous VLLM_LDG global reads with a cp.async.cg.shared.global PTX instruction-based software pipeline using a 3-stage shared memory buffer: threads asynchronously copy 16-byte chunks from global to shared memory via cp_async4 while compute reads from a different stage's shared buffer, overlapping async loads with compute and hiding global memory latency

## Key Lessons

- Use cp.async.cg.shared.global (cp_async4 for 16-byte) to initiate asynchronous global-to-shared copies that run concurrently with compute — applies whenever a kernel is memory-bound with a repetitive load-compute-store loop over global data
- Multi-stage shared memory buffering (typically 2–3 stages) prevents read-after-write hazards between async load and compute stages by having them operate on different stage offsets, and cp_async_fence + cp_async_wait<N> provide lightweight group-based synchronization without stalling the entire pipeline
- Split threads across input streams (half load gate, half load up) to maximize async copy throughput when multiple independent input tensors must be fetched per iteration
