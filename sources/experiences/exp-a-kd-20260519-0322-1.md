## Challenge

Two independent GEMM kernels each stream the same input matrix x from global memory, writing intermediate results d0 and d1 that are then read back by a third elementwise SiLU-gate kernel, causing redundant global memory traffic for x (read twice), d0, and d1, plus two extra kernel launch overheads


## Solution

Fuse both GEMM accumulators and the SiLU-gated multiply into a single kernel that reads x once per thread, computes acc0 and acc1 in registers, applies silu(acc0)*acc1, and writes all three outputs — eliminating redundant x reads, intermediate global memory round-trips for d0 and d1, and two kernel launch overheads

## Key Lessons

- Fuse dual GEMM kernels sharing the same input matrix into a single kernel that reads x once and computes both accumulators in registers — halves input bandwidth and eliminates intermediate global memory round-trips for the shared operand
- Fusing elementwise epilogue operations (SiLU-gate) into the GEMM kernel avoids writing intermediates to global memory only to read them back immediately — keeps data in registers throughout the computation
- Reducing kernel launch count from 3 to 1 eliminates two rounds of launch overhead, grid scheduling latency, and device synchronization — significant for small-to-medium workloads where launch overhead dominates
