## Solution

Replace scalar float2 loads/stores with a single float4 (128-bit) vectorized load and store per thread, processing two adjacent RoPE pairs simultaneously — halving the number of global memory transactions and improving bandwidth utilization on the 128-bit memory bus

## Key Lessons

- Use float4 (128-bit) vectorized loads/stores instead of scalar float accesses when processing multiple adjacent elements — halves global memory transaction count and saturates the GPU memory bus width, applies to any elementwise or pair-wise kernel with contiguous memory access
- Pack two independent computation pairs into one float4 load/store to increase per-thread work and reduce thread count without sacrificing occupancy — the reduced scheduling overhead and better memory throughput compound for bandwidth-bound kernels like RoPE
