## Challenge

For M=128, the before code still uses the 4x4 cluster (16 SMs) with 256x256 tiles. While M=128 fills half the M tile, the 4x4 cluster in the N dimension is still over-provisioned compared to the actual work available


## Solution

The after code reduces to a 2x1 cluster with a 256x128 tile, which cuts the N tile in half and uses only 2 SMs instead of 16. This reduces inter-SM synchronization cost while still providing enough tile parallelism for M=128, leading to better resource utilization.

## Results

- **speedup**: 1.144x (improved)
