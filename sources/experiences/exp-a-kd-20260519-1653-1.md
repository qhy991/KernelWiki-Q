## Solution

Switch to M-adaptive TileShape 32x64x128 with WarpShape 32x64x64, which matches M=32 exactly, eliminates wasted compute, doubles N-dimension threadblock parallelism from 16 to 32, and increases the K tile from 64 to 128 for better pipeline utilization

## Key Lessons

- Set TileShape.M to match the actual batch dimension exactly when M is small and fixed — oversized M tiles waste compute proportionally to (tile_M - actual_M)/tile_M
- Shrinking the N tile dimension from 128 to 64 doubles the number of concurrent threadblocks along N, improving SM occupancy for tall-skinny GEMM problems
- Increase the K tile dimension when shrinking M and N tiles to maintain pipeline depth — the larger K tile amortizes the fixed cost of loading A/B fragments across more multiply-accumulate operations
