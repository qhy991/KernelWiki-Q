## Challenge

Inner K-loop uses UNRL=2, so each thread processes only 64 K-elements per iteration (stride=32*2) and requires 4 loop iterations for K=256, resulting in high loop overhead and low instruction-level parallelism for the sequential accumulation


## Solution

Increase UNRL from 2 to 4 so each thread processes 128 K-elements per iteration (stride=32*4), halving loop iteration count from 4 to 2 for K=256 and improving instruction-level parallelism through more consecutive loads per iteration

## Key Lessons

- Increasing the inner-loop unroll factor (UNRL) in wavefront-parallel reduction kernels reduces loop iteration count and increases consecutive memory loads per iteration, improving instruction-level parallelism — especially effective when K is moderate and the unrolled stride still fits in the thread's work quantum
- Adaptive heuristic selection of unroll factor based on problem geometry (e.g., sYT=ceil(M/(CuCount*4))) allows the kernel to exploit more ILP when row count per wavefront is low
