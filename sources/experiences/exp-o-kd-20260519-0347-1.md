## Solution

Fuse the reduction and consumer into a single kernel so the shared-memory reduction result (di) is read directly from smem[0] into a register and reused in the same thread block, eliminating the intermediate global-memory buffer and one kernel launch

## Key Lessons

- Fuse a reduction kernel with its consumer into a single kernel when the reduced value fits in shared memory or registers — this eliminates the global-memory write/read round-trip and one kernel launch, which is especially beneficial when the intermediate buffer (di) is small and bandwidth-bound
- When a shared-memory reduction produces a per-block scalar, read it directly from smem[0] into a register for subsequent use rather than writing it to global memory and launching a second kernel to read it back
