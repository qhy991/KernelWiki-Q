## Solution

Enable block-level swizzle (BLOCK_SWIZZLE=true) to interleave thread blocks across N-dimension via a 3D grid, so that blocks computing different M-tiles but adjacent N-tiles are scheduled together to improve L2 cache locality on shared B-matrix data

## Key Lessons

- Block swizzle interleaving via 3D grid decomposition (gridDim.z = N_SWIZZLE) only benefits L2 cache when N is large enough for N_SWIZZLE > 1 — with N=1024 and stride=2048, N_SWIZZLE=1 and the 3D grid degenerates to 2D, producing zero interleaving benefit while adding minor host-side overhead
- Choose the block swizzle stride to be smaller than N so that multiple N-ranges exist and blocks from different M-rows can share L2 cache lines for B-matrix tiles — stride should typically be 1-2x the per-block BN tile size, not larger than N itself
