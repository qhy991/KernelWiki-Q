## Challenge

Host-side mindiv function uses truncating integer division and only checks sparse offsets (3-9), skipping potential WvPrGrp candidates that could yield better CU utilization while preserving the same round count


## Solution

Replace truncating division (N/nPrRnd) with ceiling division ((N+nPrRnd-1)/nPrRnd) and scan the full 0-12 range in a loop, searching from highest index down to find the smallest WvPrGrp that preserves the same round count, reducing idle wavefronts per block

## Key Lessons

- Use ceiling division ((N+d-1)/d) instead of truncating division (N/d) when computing work-group granularity to correctly account for partial rounds that require an extra iteration
- A smaller WvPrGrp that preserves the same round count can reduce idle wavefronts per block on hardware (e.g., AMD wvSplitK) where each wavefront processes multiple rows per round, but may regress on proxy kernels where stride reduction increases the outer loop iteration count
- Host-side work-distribution parameters can have non-obvious performance effects that differ between proxy benchmarks and production kernels — validate on the actual target kernel before concluding a regression represents the real workload
