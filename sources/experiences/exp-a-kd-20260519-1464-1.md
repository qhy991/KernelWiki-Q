## Challenge

Two-pass int8 quantization kernel reads the same half-precision input data from global memory twice (once for absmax reduction, once for quantization), which appears wasteful and motivates a shared-memory caching optimization


## Solution

Cache input data into shared memory during the absmax pass so the quantize pass reads from shared memory instead of global memory, eliminating the second global read — however this regresses by ~7% because the ~1MB working set already fits in L2 cache, the __syncthreads() barrier across 256 threads adds latency, and 8KB shared memory per block risks bank conflicts

## Key Lessons

- Shared memory caching to eliminate a redundant global read can regress performance when the working set already fits in L2 cache — the L2 hit latency (~200 cycles) is low enough that the added __syncthreads() barrier and shared memory bank-conflict risk outweigh the benefit of avoiding a second read
- Adding __syncthreads() to synchronize 256-thread blocks for a two-phase shared memory pattern introduces barrier latency that can exceed the saved global memory bandwidth, especially for small datasets where global reads are already cache-resident
- Shared memory allocation proportional to groups_per_block × group_size (here 16×256×2=8KB) increases per-block resource usage and can reduce occupancy or trigger bank conflicts when threads in the same warp access different group offsets at the same bank index
