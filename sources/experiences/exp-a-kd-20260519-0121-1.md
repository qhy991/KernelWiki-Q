## Solution

Reduce kStages from 4 to 1 for small head dimensions (k<=64) by passing kMaxK=64 to the dispatch, so MakeCustomMma computes kStages=min(4,ceil(64/64))=1, cutting shared memory per block from 33280 to 8320 bytes and enabling 5 blocks/SM (5x occupancy increase) while eliminating 3 redundant __syncthreads barriers and 3x redundant compute passes

## Key Lessons

- Scale pipeline stages (kStages) to actual head dimension rather than using a fixed maximum — when kStages=4 but only 1 stage is needed for small heads, shared memory is wasted at 4x the necessary size, collapsing occupancy to 1 block/SM
- Shared memory per block is a primary occupancy limiter in FMHA kernels — reducing it from 33280 to 8320 bytes allows 5 blocks/SM, providing 5x more active warps to hide memory latency
- Multi-stage pipelines with __syncthreads barriers in the compute loop are pure overhead when the data fits in fewer stages — each unnecessary stage adds a barrier and redundant __hmul passes over already-computed data
