## Challenge

Per-nibble remapping uses a __device__ function with if/else branching on each 4-bit value, introducing warp divergence and extra arithmetic instructions for a deterministic mapping that could be precomputed into a fixed table


## Solution

Precompute a 256-entry byte-level lookup table in CUDA __constant__ memory that maps each input byte directly to its remapped result handling both nibbles simultaneously, replacing the branching per-nibble function and separate lo/hi extraction with a single table access

## Key Lessons

- For small fixed mappings (256 entries or fewer), precompute a lookup table in __constant__ memory instead of computing per-element with branches — eliminates warp divergence and arithmetic overhead, especially effective when the same mapping applies to every element across the kernel
- A 256-entry byte-level LUT can encode two 4-bit nibble mappings into a single lookup, avoiding separate lo/hi extraction, individual mapping, and recombination — the host precomputes the full byte result at LUT upload time
