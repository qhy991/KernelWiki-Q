## Challenge

NaN-propagating max in a shared-memory block reduction uses a multi-instruction C++ path (isnan checks with if-branches) that compiles to FSETP.GTU + FSETP.GT + FSEL + BRA sequences, causing warp divergence when some threads encounter NaN values and take the isnan branch while others proceed through the normal comparison path


## Solution

Replace the multi-instruction isnan+branch sequence with a single PTX max.NaN.f32 inline asm that compiles to one FMNMX.NAN SASS instruction, making the operation branchless and eliminating warp divergence in the reduction loop entirely

## Key Lessons

- Use PTX inline asm max.NaN.f32 (compiles to FMNMX.NAN) instead of isnan+branch sequences for NaN-propagating reductions on SM80+ — the single hardware instruction eliminates warp divergence which is especially costly in iterative shared-memory reduction loops where NaN values cause threads within the same warp to take different branch paths
- Guard PTX inline asm with __CUDA_ARCH__ checks and provide a C++ fallback for older architectures — this maintains correctness across GPU generations while exploiting hardware NaN semantics where available
