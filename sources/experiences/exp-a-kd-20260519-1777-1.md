## Challenge

Standard GEMM with small M dimension (m=8) passes operands in conventional A-then-B order; on SM90 CUTLASS 3.x backends the first operand's row count directly determines TMA tile width, so placing the small-M matrix first can result in suboptimal TMA load granularity


## Solution

Swap the GEMM operand order so the larger-dimension matrix (B^T, 4096 rows) becomes the first operand instead of the small-M matrix (A, 8 rows), enabling wider TMA tile loads on SM90 CUTLASS 3.x; however, on SM80 with cuBLAS this swap has no measurable effect since cuBLAS internally handles operand transposition

## Key Lessons

- Swap-AB operand reordering is only beneficial on SM90 with CUTLASS 3.x TMA loads where the first operand's row count determines TMA tile width — on SM80 or cuBLAS backends, operand order has no measurable effect since transposition is handled internally
- Hardware-specific GEMM optimizations like operand swapping must be validated on the target GPU architecture rather than assumed portable — the same code transformation can be neutral on one generation and significant on another
