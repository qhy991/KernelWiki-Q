## Solution

Vectorize to float4 processing: each thread loads 4 contiguous floats via a single 128-bit coalesced load (FLOAT4), applies swish to each component, and writes back with a single 128-bit coalesced store, reducing total thread count by 4x and improving memory throughput

## Key Lessons

- Vectorize memory-bound elementwise kernels (swish, relu, sigmoid) from scalar f32 to float4 to issue 128-bit coalesced loads/stores — reduces thread count by 4x and improves memory throughput with negligible register pressure increase
- Scale thread index by the vector width (idx * 4 for float4) to maintain correct addressing when increasing per-thread work, and apply the same elementwise operation to each vector component
