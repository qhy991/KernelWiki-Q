## Solution

The after code uses PTX inline asm ld.volatile.global.v2.b32 to bypass L1 cache entirely, which is designed for P2P multi-GPU communication where L1 cannot reflect remote GPU writes; for single-GPU workloads with small working sets this is purely a latency penalty with no freshness benefit

## Key Lessons

- Volatile PTX loads (ld.volatile.global) bypass L1 cache and should only be used for P2P multi-GPU communication where cross-GPU writes must be detected immediately — for single-GPU workloads they only add latency since L1 coherence handles intra-GPU writes within a few cycles
- When the working set fits entirely in L1 cache (4KB << 192KB/SM), prefer regular cached loads; L1 provides ~7x lower latency than L2/global memory and cache bypassing only makes sense when L1 hit rate would be low due to capacity or conflict misses
