## Challenge

The quantization helper uses per-element float division by scale (__half2float(val) / scale) inside the inner loop, which is more expensive than multiplication on GPU hardware and performed redundantly for every element


## Solution

Precompute the reciprocal scale (1.0f / *scale) once per thread outside the loop, then replace per-element division with multiplication in the quant helper, reducing the arithmetic cost from a full FP32 division to a FP32 multiply per element

## Key Lessons

- Precomputing the reciprocal (1.0f / scale) and replacing per-element division with multiplication is a valid arithmetic optimization, but for small kernels (sub-10us, 64K elements) the execution time is dominated by kernel launch overhead and memory traffic, making the arithmetic savings indistinguishable from measurement noise
- Division-to-multiplication conversion shows measurable gains only when the kernel is compute-bound and processes enough elements per thread to amortize the instruction-level difference — on memory-bound small kernels the change is neutral
