## Solution

Use CuTe local_tile to assign 8 half elements (16 bytes) per thread, then copy() to emit LDG.128/STG.128 vectorized memory transactions that load and store four half2 values in a single instruction, amortizing address generation overhead across 8 elements

## Key Lessons

- Vectorized 128-bit memory transactions (LDG.128/STG.128) amortize instruction overhead by loading 8 half values per transaction instead of one — applies whenever data is contiguous and 16-byte aligned
- CuTe copy() auto-generates optimal vectorized load/store instructions based on the tensor's logical shape and memory layout, eliminating manual vectorization bookkeeping
