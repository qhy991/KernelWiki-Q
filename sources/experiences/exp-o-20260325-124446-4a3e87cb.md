## Challenge

Attempted to optimize RMS Norm kernel beyond 207x speedup using __ldg cache hints


## Solution

Tested __ldg for weight loads to utilize read-only cache

## Key Lessons

- RMS Norm kernel with hidden_size=2560 is compute-bound, not memory-bound
- __ldg cache hints do not improve performance when kernel is compute-bound
- Hybrid fast/cached approach with float4 vectorization and warp shuffle is already near-optimal
- Further optimization would require algorithmic changes or different hardware utilization (e.g., tensor cores)
