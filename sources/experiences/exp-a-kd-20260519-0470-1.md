## Solution

Vectorize loads/stores to half2 (two elements per thread) using the HALF2 reinterpret-cast macro, but reduce threads per block from 256 to 128 to keep the same total element coverage — however this occupancy reduction outweighs the bandwidth benefit at this problem size

## Key Lessons

- Half2 vectorization can regress performance when it reduces occupancy by halving the thread count — the lost warp-level latency hiding outweighs the wider memory transactions at small problem sizes
- Vectorized elementwise kernels should keep the same thread count (not halve it) by adjusting the grid stride or loop stride instead, preserving occupancy while gaining wider loads/stores
- At small input sizes (1M elements), memory latency hiding via occupancy is the dominant bottleneck; vectorization benefits only materialize when sufficient warps are available to overlap memory and compute
