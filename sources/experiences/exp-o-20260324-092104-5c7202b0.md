fp32_rms_norm_llama3_8b_hs4096

## Solution

Reduced thread count from 256 to 128, giving each thread 8 float4 vectors (32 floats) to process

## Key Lessons

- For rms_norm with hidden_size=4096, 128 threads outperforms 256 threads significantly (2.46x baseline)
- Increasing work per thread (8 float4 vs 4 float4) improves ILP for memory-bound kernels
- fmaf() helps with sum-of-squares computation accuracy and performance
- Fewer warps means less shared memory and fewer sync barriers
- The baseline 256-thread kernel was not optimal for A800 GPU with this problem size
