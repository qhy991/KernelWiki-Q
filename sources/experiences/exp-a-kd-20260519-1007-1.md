## Solution

Remove the scalar remainder loop entirely and enforce alignment (hidden_dim % vec_size == 0) at the host level, reducing kernel binary size by ~15 instructions and allowing the compiler to schedule the main vectorized loop more aggressively without accounting for the transition to a scalar tail path

## Key Lessons

- Remove scalar remainder/fallback loops when the data dimension is architecturally guaranteed to be aligned — even zero-iteration loops emit instructions (branch, load, clamp, store, loop control) that bloat the kernel binary, pressure the instruction cache, and constrain compiler scheduling of the main vectorized path
- Enforce alignment constraints at the host/caller level (e.g., TORCH_CHECK or runtime guard) rather than embedding scalar fallback paths in the kernel, yielding both cleaner code and tighter GPU binaries
