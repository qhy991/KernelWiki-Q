## Solution

Add an is_a_8bit flag to conditionally compute sh_a_size as 1 byte per element for 8-bit activations instead of always 2 bytes, reducing shared memory per block by 4096 bytes in W4A8 configuration

## Key Lessons

- Parameterize shared memory element size by activation precision (1 byte for int8, 2 bytes for fp16) instead of hardcoding 2 bytes — applies to any quantized GEMM where activation and weight precisions differ
- Occupancy gains from reduced shared memory only translate to throughput when the kernel has enough work items to fill the additional blocks — a grid-stride loop with few tiles may leave extra blocks idle, masking any occupancy benefit
