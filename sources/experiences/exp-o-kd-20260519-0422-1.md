## Solution

Replace synchronous B-matrix global-to-shared FLOAT4 loads with cp.async.ca.shared.global PTX instructions, issuing async copies first and then overlapping them with A-matrix loads and the BK compute loop; CP_ASYNC_COMMIT_GROUP and CP_ASYNC_WAIT_GROUP ensure correctness by committing and then waiting for outstanding async groups before the data is consumed

## Key Lessons

- Use cp.async.ca.shared.global to issue asynchronous global-to-shared memory copies that overlap with register loads and compute, hiding memory latency — applies to any tiled GEMM/conv kernel where global-to-shared transfers are on the critical path
- Issue async copies first in each iteration and wait (CP_ASYNC_WAIT_GROUP) only before the data is consumed, maximizing the overlap window between async transfers and independent work like A-matrix loads and FMAs
- __cvta_generic_to_shared converts a generic shared memory pointer to the shared memory address space required by cp.async PTX instructions — always use it to obtain the uint32_t address for async copy destinations
