## Challenge

RoPE kernel uses a flat linear index (blockIdx.x * blockDim.x + threadIdx.x) to map threads across all token-position pairs, causing threads in the same warp to interleave memory accesses across token rows and preventing coalesced global memory transactions


## Solution

Remap threads so each CUDA block processes one token position (blockIdx.x = token_pos, threadIdx.x = dimension index), ensuring all threads within a warp read from a single contiguous memory segment for the same token, improving coalescing and increasing block count to better saturate SMs

## Key Lessons

- Remap flat 1D thread indices to a per-token block mapping (one block per token, threads per dimension) so that threads within a warp access contiguous memory addresses within the same token row — this coalesces global memory transactions and applies to any elementwise kernel operating on 2D data laid out in row-major order
- Increasing the number of blocks (one per token position) can improve GPU occupancy compared to a flat grid with fewer large blocks — applies when the problem has a natural outer dimension (e.g., sequence length) that can map to blockIdx.x
