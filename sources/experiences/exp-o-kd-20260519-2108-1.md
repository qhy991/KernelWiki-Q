## Solution

Add compile-time detection of StreamKScheduler via cute::is_same_v on TileSchedulerTag, then at runtime configure the scheduler arguments with DecompositionMode::StreamK (splits tiles across SMs for better load balance) and ReductionMode::Nondeterministic (avoids extra synchronization for partial reductions), and pass the configured scheduler to the CUTLASS kernel launch

## Key Lessons

- Use compile-time if constexpr with is_same_v on the TileSchedulerTag to conditionally configure CUTLASS StreamK scheduler arguments — avoids runtime overhead while ensuring the scheduler receives correct DecompositionMode and ReductionMode settings when it is actually instantiated
- StreamK decomposition with Nondeterministic reduction can yield dramatic speedups (15x+) on tall-and-skinny GEMMs where K >> N by splitting tiles across SMs for load balance and eliminating synchronization barriers in partial reductions
- Always pass explicit TileSchedulerArguments to CUTLASS kernel launches when using non-default schedulers — the empty-default argument path silently uses suboptimal settings
