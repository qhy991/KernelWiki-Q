## Solution

Replace pointers_ += offset (array decay) with pointers_[i] += offset inside the loop so each pointer lane is independently advanced, keeping all shared-memory load addresses synchronized after add_pointer_offset

## Key Lessons

- Array names in C++ decay to pointers to their first element — pointers_ += offset increments the decayed base address, not each element; always use pointers_[i] += offset to advance every element of a local pointer array independently
- A correctness bug that reads from wrong addresses can be latency-neutral if the wrong and correct addresses map to the same memory access pattern and instruction count — performance benchmarks alone cannot catch this class of error
