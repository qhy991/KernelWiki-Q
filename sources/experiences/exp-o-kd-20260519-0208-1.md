## Solution

Remove the offset_q term to implement CausalMask<true> (IsQBegin=true) dispatch: when Q starts at position 0, max_blocks_q = ceil_div((blk+1)*tile_q, tile_k) = blk+1, producing the correct triangular trip pattern (1+2+3+4=10 trips instead of 128)

## Key Lessons

- When Q starts at position 0 (IsQBegin=true) in causal attention, the n_k - n_q offset must be omitted from the trip count calculation — including it incorrectly extends the causal window to cover nearly all K-blocks, ballooning work from O(n_q²/2) to O(n_q × n_k)
- Causal mask dispatch should be conditional on Q's starting position: CausalMask<true> for Q at sequence start (no offset), CausalMask<false> for mid-sequence Q (requires offset to align the causal boundary)
