## Challenge

The before code performs 128 element-level extractions from both input arrays (each requiring shift+mask), computes XOR on the extracted 1-bit values, and inserts the result back (conditional read-modify-write)


## Solution

Replace the element-by-element loop (128 iterations of sub-byte get/set) with a word-level XOR loop (4 iterations on uint32_t words), exploiting the fact that XOR commutes through packing — operating on whole storage words produces identical results without any decode/encode overhead

## Results

- **speedup**: 2.527x (improved)


## Key Lessons

- Use word-level (packed) operations instead of element-by-element sub-byte access for bitwise operations on packed data — XOR commutes through packing, so operating on whole storage words eliminates decode/encode overhead while producing identical results
- When data is stored in packed format (multiple elements per word), always prefer operating on the packed representation directly — loop iteration count drops by the packing factor (128 → 4 for 32-bit words), reducing both instruction count and branch overhead
