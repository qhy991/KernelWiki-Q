## Solution

Apply AWQ undo_pack deinterleaving table {0,2,1,3} to remap the sequential bit position to the correct interleaved position before shifting, enabling correct extraction from AWQ-packed weights while maintaining identical runtime cost since the constexpr lookup is compiled to a constant selection

## Key Lessons

- constexpr lookup tables with small fixed indices (e.g., undo_pack[4] = {0,2,1,3}) are resolved at compile time into constant selections, adding zero runtime cost — use them freely for format conversion remapping between GPTQ and AWQ weight layouts
- AWQ vs GPTQ 8-bit packing differs only in the position-to-slot mapping ({0,2,1,3} deinterleaving for AWQ vs sequential for GPTQ); the extraction memory access pattern and shift logic are otherwise identical, so supporting both formats requires only a position remapping table
