## Solution

Fuse the BF16→float conversion into the softmax kernel by calling __bfloat162float inline within each of the three softmax loops (max-reduction, exp-sum, output-write), eliminating the intermediate float buffer allocation and one full global memory round-trip

## Key Lessons

- Fuse type-conversion into compute kernels by inlining __bfloat162float at each use site rather than using a separate conversion kernel — eliminates one full global memory round-trip and one kernel launch, which is significant when per-row work is small (e.g., only 8 expert columns)
- For lightweight kernels operating on few elements per row, kernel launch overhead and intermediate buffer traffic dominate runtime — fusing adjacent passes yields measurable speedup even when the compute itself is unchanged
