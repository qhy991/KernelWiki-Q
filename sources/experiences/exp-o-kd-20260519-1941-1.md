## Solution

Fuse reduce_kernel_float and quantize_kernel_fp8 into a single reduce_kernel_fused_fp8 kernel that computes the normalized accumulator, applies the FP8 out_scale inline (acc *= out_scale where out_scale = 1.0f / *fp8_out_scale_ptr), and writes uint8_t FP8 output directly via __nv_cvt_float_to_fp8 — eliminating the intermediate float buffer allocation, one kernel launch, and the associated global memory write/read round-trip

## Key Lessons

- Fuse a reduce/normalize kernel with a subsequent quantize kernel when the intermediate result is only consumed once — eliminates one kernel launch, one global memory buffer allocation, and a full write-then-read round-trip, yielding consistent speedup across different head sizes
- Apply FP8 out_scale inline during accumulation rather than deferring to a separate quantize pass — the scale multiply is compute-cheap (one FMUL) but the global memory traffic it eliminates (float4 write + uint8 read) is expensive
- Adding a nullptr guard on the scale pointer (fp8_out_scale_ptr != nullptr) makes the fused kernel robust to cases where no scaling is needed, without measurable overhead
