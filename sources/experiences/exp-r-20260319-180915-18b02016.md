## Key Lessons

- Shared memory tiling doesn't automatically improve matmul performance - tile size and memory access patterns matter
- Thread coarsening is complex - each thread computing multiple outputs requires careful index mapping
- For large output spaces, naive implementation may be acceptable if correctness is the primary goal
- Batched matmul benefits from specialized libraries (cuBLAS) rather than manual optimization
