## Challenge

## Key Lessons

- The before kernel uses a compile-time TOPK=8 template parameter with #pragma unroll, which allows nvcc to fully unroll the 8-iteration accumulation loop.
- This regression is the expected trade-off for a general fallback kernel — the before code could not handle non-standard topk values at all (TOPK=-1 produced zeros), so the after code trades some performance for correctness on arbitrary topk.
