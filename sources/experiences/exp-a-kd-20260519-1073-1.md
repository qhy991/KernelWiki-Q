## Solution

Replace scalar per-element half loads and stores with half2 vectorized access via reinterpret_cast<const half2*>(), using __half22float2 to convert two half values per instruction and __float22half2_rn to write two results per store, with #pragma unroll on the inner vec_size/2 loop to eliminate branch overhead and enable instruction-level parallelism

## Key Lessons

- Replace scalar half loads with half2 vectorized loads via reinterpret_cast<const half2*>() to halve the number of global memory transactions — applies whenever contiguous half elements are accessed and addresses are 4-byte aligned
- Use __half22float2 instead of two separate __half2float calls to convert two half values in one hardware instruction, halving type-conversion overhead in bandwidth-bound kernels
- Add #pragma unroll on small fixed-trip-count inner loops (e.g., vec_size/2 = 4 iterations) to eliminate branch overhead and give the compiler freedom to schedule FMA and conversion instructions for pipeline occupancy
