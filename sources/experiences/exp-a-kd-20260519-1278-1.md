## Solution

Replace the two-step bf16 GEMM + cast pipeline with a single cublasGemmEx call using CUDA_R_32F as the output type, letting cuBLAS accumulate in fp32 internally and write fp32 results directly, eliminating the intermediate bf16 global-memory write and the separate cast kernel entirely

## Key Lessons

- cuBLAS supports mixed-type output via cublasGemmEx: bf16 inputs (CUDA_R_16BF) with fp32 output (CUDA_R_32F) and CUBLAS_COMPUTE_32F — use this instead of a separate cast kernel to avoid redundant global-memory traffic and kernel launch overhead
- Eliminating intermediate buffer allocations (e.g., d_C_bf16) when the final output type differs from the input type reduces memory footprint and removes one global-memory round-trip, yielding measurable speedup especially for small batch GEMMs where kernel launch overhead dominates
