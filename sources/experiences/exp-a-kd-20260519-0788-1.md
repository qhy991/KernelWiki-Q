## Solution

Fuse the three separate kernels into a single-pass kernel that reads input_ids once, performs the negate-clamp-lookup-write in-place, eliminating all intermediate global memory buffers (64KB saved) and reducing kernel launch overhead from three to one

## Key Lessons

- Fuse multi-step pipelines that pass data through intermediate global memory buffers into a single kernel — eliminates intermediate global memory traffic (64KB here) and amortizes per-launch driver overhead, yielding compounding speedup for small workloads
- When each step in a pipeline reads the output of the previous step element-wise at the same index, fusion is always safe and beneficial — the intermediate buffers are never reused and add pure overhead
- Kernel launch overhead becomes a dominant cost for small problem sizes (4096 elements); consolidating N launches into 1 can yield near-N× speedup even when per-element work is unchanged
