## Solution

Add SM count awareness to the dispatch condition by requiring both sm_count == 78 (H20 Pingpong proxy target) AND K > 128 before selecting the sequential path, so that high-SM-count GPUs (H100/H200/A800) always use the batched Cooperative-style path regardless of K

## Key Lessons

- Hardware-aware dispatch for library GEMM calls should consider SM count alongside problem dimensions — low-SM-count GPUs (e.g., H20 with 78 SMs) may need a different cuBLAS proxy/strategy than high-SM-count GPUs (H100/H200/A800) for the same problem size
- Adding a hardware guard that does not change the executed code path on the tested configuration produces a neutral verdict, confirming the guard introduces no regression while enabling correct strategy selection on heterogeneous GPU fleets
