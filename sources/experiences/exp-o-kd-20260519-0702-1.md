## Challenge

Per-block reduction uses atomicAdd to a global buffer with 512 threads contending on the same address, then issues __syncthreads and a redundant global read to retrieve the result; the caller must also cudaMemset the buffer before each launch, adding fixed-cost overhead that dominates at smaller workloads


## Solution

Replace atomicAdd-based global reduction with CUB BlockReduce cooperative reduction in shared memory, eliminating atomic contention, the extra global buffer allocation, and the per-launch cudaMemset reset; each block computes its sum via a tree reduction in shared memory, and only thread 0 writes the final result

## Key Lessons

- Replace per-block atomicAdd reduction with CUB BlockReduce to eliminate atomic contention when all threads in a block write the same address — CUB's shared-memory tree reduction is conflict-free and avoids the serialization overhead of 512 competing atomicAdd operations
- CUB BlockReduce removes the need for a separate global atomic buffer and its per-launch cudaMemset reset, reducing both memory footprint and fixed-cost kernel overhead, which is especially significant at smaller workloads where fixed costs dominate total execution time
