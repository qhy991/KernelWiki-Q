## Solution

Replace __shfl_sync with a direct threadIdx.x / 32 computation via canonical_warp_idx() device function, eliminating the shuffle barrier entirely so each thread computes its warp index independently and proceeds without forcing warp convergence

## Key Lessons

- Prefer direct arithmetic (threadIdx.x / 32) over __shfl_sync for computing warp index — the division is computed independently per thread without any barrier, eliminating both shuffle instruction overhead (~20-40 cycles) and reconvergence stalls when threads are divergent
- __shfl_sync forces warp-wide convergence even when no cross-lane communication is needed; if the value being shuffled is already determinable from thread-local data, a direct computation avoids unnecessary synchronization and enables forward progress for divergent threads
- Wrap thread-local index computations in a __device__ helper with __CUDA_ARCH__ guards for host-compilability — provides a clean, reusable abstraction that works in both device and host code paths
