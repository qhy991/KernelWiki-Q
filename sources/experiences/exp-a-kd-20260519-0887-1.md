## Solution

Replace scalar half loads with 16-byte vector loads via vec_t<T,8>::cast_load (memcpy-based 128-bit load), reading 8 half values per transaction instead of one, and iterating over vector elements with an unrolled inner loop for both the absmax and quantize phases

## Key Lessons

- Use 16-byte vector loads (cast_load via memcpy of sizeof(T)*vec_size bytes) instead of scalar element-wise loads when processing contiguous arrays of half/bfloat16 data — reduces memory transactions by 8x for half and 4x for bfloat16
- The vec_t<T,N>::cast_load pattern using memcpy is portable across GPU architectures and avoids alignment issues that reinterpret_cast vector loads can introduce
