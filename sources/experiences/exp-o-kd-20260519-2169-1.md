## Solution

Fuse the next-pass histogram construction directly into the threshold-bin buffering loop by computing and accumulating the next-level radix bin for each buffered element inline using the already-loaded logit value, eliminating the separate histogram-building scan and halving the number of passes over the candidate set

## Key Lessons

- Fuse histogram construction into the data-movement loop when both operate on the same elements — eliminates a redundant second pass over shared-memory buffers and re-reads of global memory, reducing total work by up to half per refinement pass
- When a kernel already loads a value from global memory for classification (binning), compute the next-level classification (next radix bits) inline using the same loaded value rather than re-reading it in a separate pass — the fp32_bits are already in registers
