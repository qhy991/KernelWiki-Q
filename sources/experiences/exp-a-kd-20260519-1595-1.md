## Challenge

The SM89 (Ada Lovelace) branch in the FP8 support gate unconditionally returns false regardless of CUDA version, blocking FP8 CUTLASS kernel dispatch on Ada GPUs and forcing a slower torch.mm fallback for all FP8 workloads


## Solution

Replace the unconditional return false in the SM89 branch with return cuda_version >= 12040, enabling FP8 CUTLASS tensor core dispatch on Ada Lovelace GPUs when CUDA 12.4+ is available, which replaces the slower torch.mm fallback with hardware-accelerated FP8 operations

## Key Lessons

- Host-side dispatch gates that unconditionally disable hardware paths block downstream performance gains → version-conditional checks (cuda_version >= N) allow incremental feature enablement as toolchain support matures → applies to any feature-gating logic tied to GPU capability and CUDA driver version
- Microbenchmarking sub-nanosecond host-side branches produces unreliable speedup numbers dominated by measurement noise → the real performance impact of a dispatch gate change must be measured at the workload level (e.g., end-to-end FP8 matmul throughput) → applies whenever profiling trivial boolean-return functions that gate heavier kernel dispatch
