## Challenge

Blelloch prefix-sum scan over 16 elements uses shared-memory up-sweep and down-sweep phases requiring 8 scan-phase __syncthreads plus 2 auxiliary barriers (10 total) across all 1024 threads, incurring heavy barrier synchronization cost for a data set that fits within a single warp


## Solution

Replace the Blelloch shared-memory scan (8 scan-phase __syncthreads across all 1024 threads) with a warp shuffle exclusive scan using __shfl_up_sync, which requires no barriers within a warp; only 7 __syncthreads remain for cross-warp communication, eliminating 3 full-block synchronization points for a ~10% latency reduction

## Key Lessons

- When scan_size fits within a single warp (≤32 elements), replace Blelloch shared-memory scan with __shfl_up_sync-based exclusive scan — warp shuffle is barrier-free within a warp, eliminating all scan-phase __syncthreads and their full-block synchronization cost
- For multi-warp scans, use a two-level hierarchical approach: intra-warp prefix sum via shuffle, then a single warp accumulates cross-warp offsets — this minimizes __syncthreads to O(num_warps) instead of O(log2(scan_size) × 2)
- Reducing __syncthreads count is especially impactful when thread count is high (e.g., 1024 threads) but only a few threads perform useful work in the scan, as all threads pay the barrier latency regardless of participation
