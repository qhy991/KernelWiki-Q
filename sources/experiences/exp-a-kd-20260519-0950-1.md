## Challenge

The cumsum over 256 expert counts is computed serially by a single thread (threadIdx.x==0) using a sequential O(N) loop with per-iteration div/mod for warp-based shared memory addressing, leaving 1023 threads idle at a barrier


## Solution

Replace the serial O(N) cumsum loop with a parallel Blelloch prefix scan that engages all threads in O(log N) barrier rounds — 8 rounds for 256 experts across up-sweep and down-sweep phases — dramatically reducing the serialization window

## Key Lessons

- Replace single-thread serial O(N) prefix sum with parallel Blelloch scan to engage all threads — the O(log N) barrier depth scales well while the serial loop grows linearly with element count
- Blelloch (work-efficient) scan uses 2N total additions vs Kogge-Stone's N*logN, making it preferred for moderate-sized arrays (256 elements) in shared memory
