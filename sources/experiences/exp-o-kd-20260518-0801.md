## Key Lessons

- When expert_map is absent — the common case without expert parallelism — the clone performs a full GPU device-to-device memcpy of the topk_ids tensor plus a GPU memory allocation, adding latency and memory pressure for no benefit, since sortAndScanExpert only reads the data.
- Applied change: Replace the unconditional clone with a lazy clone pattern: initialize topk_ids_for_sort as a shallow reference to topk_ids (torch::Tensor topk_ids_for_sort = topk_ids), then only clone inside the expert_map.has_value() branch where preprocessTopkIdLauncher actually mutates the data.
