## Solution

Eliminate the separate cast kernel entirely by using cublasGemmEx with CUDA_R_32F as the output type (d_C_fp32, CUDA_R_32F), letting the GEMM epilogue write fp32 directly and removing the intermediate bf16 buffer allocation and the cast kernel launch

## Key Lessons

- Use cublasGemmEx with CUDA_R_32F output type instead of CUDA_R_16BF + separate cast kernel when the final result must be fp32 — the GEMM epilogue can write fp32 directly, eliminating an extra kernel launch and a full memory round-trip through an intermediate buffer
- Removing intermediate buffer allocations (d_C_bf16) reduces memory pressure and simplifies resource management — the 32 KB savings matters at small batch sizes where kernel launch overhead dominates
- When CUBLAS_COMPUTE_32F is already specified for accumulation, switching the output type from CUDA_R_16BF to CUDA_R_32F is essentially free since the internal accumulator is already fp32 — the conversion happens inside the GEMM epilogue rather than in a separate pass
