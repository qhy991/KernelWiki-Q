## Challenge

Same pre-computation pattern but with top_k=7 and 1024 rows (8 rows per thread per inner iteration). Division by 7 is similarly expensive as division by 3 since neither is a power-of-2 allowing shift optimization. The before version executes 1600 divisions per thread across 200 repetitions;


## Solution

the after version computes 8 divisions during init and replaces all 1600 hot-loop divisions with shared memory reads.
