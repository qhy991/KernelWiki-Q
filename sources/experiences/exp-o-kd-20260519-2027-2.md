## Solution

Expand the kernel grid to cover all TMA-aligned padding groups (num_groups_padded = tma_aligned_mn × padded_groups_per_row) so each padding group thread writes a zero byte to the corresponding scale output location, fusing zero-initialization into the computation kernel and eliminating the separate cudaMemsetAsync entirely

## Key Lessons

- Fuse buffer zero-initialization into the computation kernel by expanding the grid to cover TMA-aligned padding elements — padding threads write zeros in the same pass, eliminating a separate cudaMemsetAsync along with its DRAM traversal and launch overhead
- When a kernel writes packed sub-word fields, the padded grid approach ensures every byte in the output buffer is written exactly once by the kernel itself, making the result correct without any prior buffer initialization
