## Challenge

The before code launches two separate kernels: first gather_fp8_cache copies raw FP8 bytes from paged cache blocks into an intermediate buffer using a generic byte-copy loop with 1024 threads, then dequant_fp8_to_bf16 reads from that intermediate buffer and converts FP8->BF16 with 576 threads


## Solution

The after code fuses both steps into a single kernel that reads FP8 data directly from paged cache blocks and writes converted BF16 output, eliminating the intermediate buffer write/read and the second kernel launch. ## Key Lessons

- The speedup comes from: (1) eliminating one kernel launch overhead, (2) avoiding the intermediate buffer write-then-read (656 bytes per token written and then re-read), (3) better cache locality since FP8 read and BF16 write happen in the same kernel pass per token.
