## Solution

Reduce pipeline stages to min(4, ceil(kMaxK/kBlockK))=1 when kMaxK equals kBlockK (kSmemContainsEntireMat=true), cutting SMEM from 32KB to 8KB/block and eliminating 3 redundant compute+sync iterations, which quadruples occupancy (~20 blocks/SM) and removes 3 barrier stalls per kernel invocation

## Key Lessons

- Reduce pipeline stages to min(default, ceil(kMaxK/kBlockK)) when the entire matrix fits in a single shared-memory tile (kSmemContainsEntireMat=true) → eliminates redundant prologue/mainloop load steps and sync barriers → applies to CUTLASS CustomMmaMultistage when kMaxK ≤ kBlockK, especially for small attention head dimensions
- Shared memory per block directly controls SM occupancy: reducing SMEM from 32KB to 8KB per block quadruples active blocks per SM (~5→~20), dramatically improving latency hiding → applies whenever pipeline stages exceed what the problem dimension requires
- Multi-stage pipelines benefit latency hiding only when stages are needed for double/triple buffering; when data fits in one stage, extra stages waste SMEM and add unnecessary __syncthreads barriers without any pipelining advantage
