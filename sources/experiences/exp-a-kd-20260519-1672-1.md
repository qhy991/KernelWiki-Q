## Challenge

Norm-and-quant kernel uses a single scalar scale value for all elements, preventing per-group quantization granularity needed for mixed-precision inference where different channel groups require independent scale factors


## Solution

Replace the single scalar scale with a groupwise scale lookup that computes a per-group scale index (scale_idx) inside the vec4 loop using i*VEC_SIZE/group_size integer division, loading the appropriate scale from a global array while supporting both row-major and transposed scale layouts via compile-time template flags

## Key Lessons

- When group_size is a power-of-two multiple of VEC_SIZE, the per-element group index i*VEC_SIZE/group_size simplifies to a single shift (i/32 for group_size=128, VEC_SIZE=4), making the groupwise scale lookup compile to nearly free integer arithmetic
- Per-group scale arrays that fit entirely in L1 cache (e.g. 32 float values = 128 bytes for hidden_size=4096, group_size=128) add negligible latency over a register-resident scalar — the L1 hit cost is hidden behind the dominant input/weight memory bandwidth
