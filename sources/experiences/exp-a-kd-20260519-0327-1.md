## Challenge

Attention backward pass is split into two separate kernel launches (stage1 for grad_v and tmp_sum accumulation, stage2 for grad_q and grad_k), requiring intermediate softmax-derivative sums (tmp_sum) to be written to global memory via atomicAdd and read back in the second kernel, adding launch overhead and global memory traffic


## Solution

Fuse both kernels into one, reformulate tmp_sum using the chain-rule identity sum_k(attn*grad_attn) = sum_j(output*grad_out) so it can be computed from output/grad_out in shared memory without global atomics, and merge grad_v/grad_k accumulation into a single loop over shared fact

## Key Lessons

- Kernel fusion can regress on small problem sizes (M=64, N=64, K=32) because increased register pressure from more live state (two shared arrays + more accumulator registers) reduces occupancy, outweighing the savings from eliminated launches and global memory traffic
- The chain-rule identity sum_k(attn_ik * grad_attn_ik) = sum_j(output_ij * grad_out_ij) allows computing the softmax-derivative normalization from output and grad_out instead of attn and grad_attn, enabling kernel fusion without an extra global memory pass
- Shared-memory tmp_sum eliminates global atomicAdd contention, but the benefit only materializes when the fused kernel's register footprint does not significantly reduce occupancy compared to the split version — measure before fusing on small tiles
