## Challenge

Standard GEMM dispatch uses next_pow_2(m) clamped to 64 for M-threshold branching, routing all M<=64 cases to a single kernel config without considering N dimension, potentially missing better tile shapes for tall-skinny workloads


## Solution

Swap-AB reformulation (C=A*B becomes D=B^T*A^T) to trigger a dispatch path using direct M comparisons with N-aware sub-branching, enabling selection of CUTLASS tile shapes and cluster sizes tailored to the N dimension on SM90 hardware

## Key Lessons

- Swap-AB GEMM reformulation (C=A*B → D=B^T*A^T) does not improve performance on SM80 cuBLAS because both dispatch paths resolve to the same underlying kernel — the benefit only materializes on SM90 where N-aware sub-branching selects different CUTLASS tile shapes and cluster sizes
- Dispatch logic changes (next_pow_2 threshold vs direct M comparison) are only meaningful when they lead to different kernel configurations; verifying which backend kernel is actually invoked is essential before expecting speedup
