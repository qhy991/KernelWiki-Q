## Challenge

Warp-level top-K uses separate butterfly reduction for values, __ballot_sync to locate the winning lane, and __shfl_sync to broadcast its index — requiring O(WARP_SIZE) shuffles per top-K iteration and repeated conditional logic across K iterations


## Solution

Pack the float value and its int32 index into a single uint64_t word using cub::Traits<T>::TwiddleIn for comparison-safe encoding, then use cooperative_groups::reduce with cg::greater to find the max value+index pair in O(log WARP_SIZE) shuffles per iteration instead of O(WARP_SIZE)

## Key Lessons

- Pack value and index into a single word (uint64_t for fp32, uint32_t for fp16) using cub::Traits<T>::TwiddleIn for comparison-safe bit encoding, so a single cg::reduce call replaces multi-step shuffle+ballot+broadcast sequences in warp top-K — reduces O(WARP_SIZE) shuffles per iteration to O(log WARP_SIZE)
- Use TwiddleIn/TwiddleOut from cub::Traits to convert floating-point values to unsigned integers that preserve ordering under integer comparison (greater), enabling a single integer reduction to correctly find the floating-point maximum along with its index
- Invalidate the previous winner in iterative top-K by comparing packedMax == topK.compValIdx and substituting minValue, which avoids the __ballot_sync winner-detection step entirely
