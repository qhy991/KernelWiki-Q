## Challenge

Scalar byte-by-byte loads in the encoding kernel issue one 1-byte LDG instruction per element, inflating instruction count by 16x compared to a single 16-byte vector load, which dominates execution time for small 16 KB workloads that are instruction-limited rather than bandwidth-limited


## Solution

Replace scalar byte loads/stores with uint4 (16-byte) vectorized loads/stores via reinterpret_cast, processing 16 bytes per iteration and reducing total load instructions by 16x, which directly cuts instruction count for small instruction-bound kernels

## Key Lessons

- Vectorized uint4 loads (16 bytes per instruction) reduce load instruction count by 16x versus scalar byte loads — especially effective for small instruction-bound kernels where reducing instruction count matters more than bandwidth utilization
- reinterpret_cast<uint4*> on byte arrays enables vectorized access without data layout changes; combined with #pragma unroll on the inner byte loop, the compiler generates efficient in-register per-byte processing after a single vector load
