## Solution

Replace scalar float load/store with float4 vectorized loads/stores so each thread processes 4 elements, reducing thread count by 4x and replacing 4 separate 32-bit transactions with a single 128-bit coalesced transaction to better saturate the memory bus

## Key Lessons

- Use float4 vectorized loads/stores to process 4 elements per thread in memory-bound elementwise kernels — reduces total thread count and replaces multiple narrow memory transactions with fewer wide coalesced transactions, better saturating memory bandwidth
- Scale grid dimensions by the vectorization factor (divide by BLOCK*4 instead of BLOCK) to avoid launching excess threads when each thread handles multiple elements
