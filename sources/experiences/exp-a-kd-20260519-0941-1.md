## Solution

Replace the global memory pointer parameter (const float* scale_ptr) with a scalar kernel argument (const float scale_val), moving the reciprocal computation to the host side before kernel launch, so the GPU hardware broadcasts the precomputed value via constant cache or register allocation instead of each thread performing a redundant global load and division

## Key Lessons

- Pass scalar values (scale factors, thresholds) as kernel arguments rather than device pointers — the GPU broadcasts kernel arguments via constant cache or register allocation, eliminating redundant global memory loads and per-thread arithmetic — applies whenever a kernel consumes a single scalar value known at launch time
- For short-running kernels on small tensors, eliminating even minor per-thread overheads (a global load and division) yields measurable speedups because fixed costs represent a larger fraction of total execution time
