## Solution

Extract the sorting phase into a separate multi-block kernel (moe_token_sort_kernel) launched with <<<128, 256>>> using a grid-stride loop and global-memory atomicAdd on cumsum_buffer, achieving near 1:1 thread-to-token mapping that distributes work across all GPU SMs instead of confining it to a single block

## Key Lessons

- Extract scatter/sort phases from monolithic single-block kernels into separate multi-block kernels to scale parallelism with input size — a single block caps utilization at one SM regardless of data volume
- Use grid-stride loops with global-memory atomicAdd for token binning/scatter operations — the broader atomic contention across multiple SMs is far less severe than shared-memory atomics serialized within a single block
- Decompose monolithic kernels by data dependency: keep prefix-sum and metadata computation in a single-block kernel (where sequential dependency exists), then launch independent scatter work as a separate multi-block kernel that reuses the cumsum buffer as atomic counters
