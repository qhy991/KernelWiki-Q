## Solution

Replace plain integer div/mod with FastDivmod using __umulhi (single-cycle on SM80) for index decomposition, and replace per-access ndhwc_offset recomputation with precomputed byte-offset increments added to a base pointer via pointer arithmetic, reducing per-access cost from 4 int64 muls + 4 adds to 3 muls + 2 adds

## Key Lessons

- Replace plain integer div/mod with FastDivmod (using __umulhi + shift + multiply-subtract) for hot-loop index decomposition — __umulhi is single-cycle on SM80 versus ~20 cycles per integer division, compounding to significant savings when multiple decompositions occur per access
- Precompute byte-offset increments for multi-dimensional traversal (inc_next_s, inc_next_r, inc_next_t) and accumulate them onto a base pointer instead of recomputing the full NDHWC linear offset from coordinates on every access — trades one-time setup cost for reduced per-access multiply-add count
- Conv3d activation tile iterators can be integer-ALU-bound rather than memory-bound even at small problem sizes, so reducing instruction count in index arithmetic directly translates to measurable speedup
