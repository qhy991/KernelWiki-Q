## Solution

Replace the scalar K-loop with WMMA 16x16x16 tensor-core fragments: each mma_sync call processes 16 K-elements for an entire 16x16 output tile simultaneously through the hardware matrix multiply unit, reducing total instruction count from 737280 scalar operations to 180 mma_sync calls

## Key Lessons

- Replace scalar multiply-add K-loops with WMMA tensor-core fragments (fragment + load_matrix_sync + mma_sync) to process 16 K-elements per instruction for a 16x16 output tile — reduces scalar instruction count by 16x and leverages dedicated matrix multiply hardware, applicable whenever compute-bound GEMM on tensor-core-capable GPUs is detected
- Use fill_fragment(frag_c, 0.0f) to initialize the accumulator fragment before the K-loop instead of per-thread conditional accumulation — ensures correct hardware-level zero-init across all fragment lanes without branch divergence
- WMMA 16x16x16 tile size with half inputs and float accumulation provides the baseline tensor-core utilization; for further gains, consider WMMA 16x16x32 or MMA PTX to process more K-dimension per instruction, analogous to upgrading from two narrow MFMA calls to one wider MFMA call on AMD hardware
