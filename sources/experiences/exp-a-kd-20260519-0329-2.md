## Solution

Replace shared-memory tree reduction with warp-level __shfl_xor_sync reduction (no shared memory, no __syncthreads within each warp), then write only one partial sum per warp to a tiny shared array for the cross-warp final add — reducing sync points from 8 to 1 and shared memory from 256 to 2 floats

## Key Lessons

- Prefer warp shuffle (__shfl_xor_sync/__shfl_down_sync) over shared-memory tree reduction for intra-warp sums — shuffles are register-to-register with no __syncthreads, eliminating multiple barrier stalls
- When block size is small (e.g., 64 threads = 2 warps), use shuffles for per-warp reduction then a tiny shared array indexed by warp ID for the cross-warp final add — reduces shared memory footprint from O(blockDim) to O(num_warps)
