## Solution

Replace scalar float pointer accesses with vec_n_t<float,4> reinterpret_cast to enable 128-bit vector loads/stores (LDG.128/STG.128), processing 4 elements per iteration via DefaultVecOp with #pragma unroll, reducing total memory transactions by 4x and improving L2 cache line utilization for this bandwidth-bound kernel

## Key Lessons

- For memory-bandwidth-bound kernels with low arithmetic intensity (e.g., single multiply per element), replacing scalar float loads/stores with vec_n_t<float,4> reinterpret_cast to emit LDG.128/STG.128 instructions reduces memory transaction count by 4x and directly improves throughput — applies whenever data is aligned and hidden_size is a multiple of the vector width
- Always include a tail/scalar loop after the vectorized loop to handle remainder elements when hidden_size is not a multiple of VEC_SIZE — this preserves correctness without sacrificing the vectorized path for the majority of elements
- Use DefaultVecOp with a lambda scalar_op and #pragma unroll to cleanly separate the vectorization strategy from the per-element computation, making the pattern reusable across different element-wise operations
