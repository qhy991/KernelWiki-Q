## Key Lessons

- Metal requires explicit threadgroup memory allocation via setThreadgroupMemoryLength
- fp16 has limited range (~65504 max), reduce test sizes to avoid overflow
- Reduction loops need bounds checking to avoid reading garbage memory
- fp16 precision loss of 1-2% is normal for large reductions
- Two-pass reduction (partial sums then final reduce) works well for large arrays
