## Challenge

The before source hard-codes `kBlockSizeI = 128`, which turns the simulated backward tile into an 8-warp block even though `kHeadDim` is only 64


## Solution

The after source changes only that tile choice to `kBlockSizeI = 64`, so it launches 4 warps, halves the block-wide reduction footprint, and avoids paying synchronization and scheduling cost for threads that never contribute useful work.

## Results

- **speedup**: 2.317x (improved)
