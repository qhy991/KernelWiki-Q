## Solution

Reinterpret float pointers as uint4 (128-bit) pointers so each load/store moves 4 floats per memory transaction, reducing total memory instructions by 4x and loop iterations from 8 to 2 per thread, enabling the memory controller to coalesce wider transactions into fewer DRAM bursts

## Key Lessons

- Replace scalar float load/store loops with uint4 (128-bit) vectorized load/store when copying contiguous aligned data — reduces memory instruction count by 4x and cuts loop iterations, improving throughput for bandwidth-bound copy kernels
- reinterpret_cast<const uint4*>(float_ptr) enables 128-bit memory transactions without changing data layout — applies whenever cols is a multiple of 4 and the base pointer is 16-byte aligned
