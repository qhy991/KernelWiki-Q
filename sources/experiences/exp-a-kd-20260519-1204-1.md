## Solution

Pre-compute all sh_ids[i] / top_k results into a second shared memory buffer (sh_precomputed) during a one-time initialization phase, then replace the 200× repeated integer divisions in the hot loop with simple shared memory reads, trading 2× shared memory usage (8KB vs 4KB) for elimination of ~1600 divisions per thread

## Key Lessons

- Pre-compute invariant integer division results into shared memory when the divisor is a non-power-of-2 constant and the operation repeats in a hot loop — integer division costs ~20-40 cycles on NVIDIA GPUs while shared memory reads cost ~5 cycles, so even with 2× shared memory overhead the net speedup can exceed 2×
- Use #pragma unroll on the pre-computation loop to ensure the compiler fully unrolls the division initialization phase, minimizing overhead of the one-time cost
