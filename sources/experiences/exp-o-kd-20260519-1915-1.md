## Challenge

RMS norm variance kernel uses a scalar strided loop where each of 1024 threads reads 4 separate __half values per iteration, issuing 4096 individual 2-byte global memory loads instead of coalesced wider transactions


## Solution

Replace the scalar strided loop with vectorize_read_with_alignment<8> which casts the input pointer to vec_n_t<__half,8> (128-bit) and issues 512 vector loads instead of 4096 scalar loads, reducing global memory transaction count by 8x and lowering per-element latency overhead

## Key Lessons

- Vectorize global memory reads by casting __half pointers to 128-bit (vec_n_t<__half,8>) and issuing one wide load per 8 elements instead of 8 scalar loads — reduces total memory transactions by 8x when memory is aligned and element count is divisible by vector width
- Use a vectorize_read_with_alignment helper with dual lambdas (vec_op + scalar_op) to handle alignment edge cases transparently — cudaMalloc guarantees 16-byte alignment so the fast-path branch is always taken for power-of-two element counts
- Replace scalar strided accumulation loops with vectorized reads even for simple reduction patterns — the reduced transaction count outweighs the overhead of the unrolled per-element extraction inside the vector callback
