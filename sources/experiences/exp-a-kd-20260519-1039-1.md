## Solution

Replace the per-thread counting matrix with atomicAdd to a single shared_counts[num_experts] array, reducing shared memory from O(stride * num_experts) to O(num_experts) and eliminating the serial column-wise prefix sum by distributing reduction across all threads during the counting phase

## Key Lessons

- atomicAdd to shared memory eliminates both O(stride * num_experts) memory overhead and serial reduction loops when histogramming/counting across threads — applies whenever each thread produces partial counts that need global aggregation
- Per-thread counting matrices with serial column-wise prefix sums create a bottleneck proportional to stride * num_experts; atomicAdd parallelizes the reduction into the counting phase, trading serialization for fine-grained atomic contention that scales well for moderate num_experts
- Shared memory footprint reduction from O(stride * num_experts) to O(num_experts) enables using larger thread blocks (1024 vs 256) without exceeding shared memory limits, improving occupancy
