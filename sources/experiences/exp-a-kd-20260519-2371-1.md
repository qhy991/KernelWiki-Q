## Challenge

FP4 quantization kernel unconditionally executes __shfl_xor_sync warp shuffle for cross-thread max reduction even when NUM_THREADS_PER_SF=1 (each thread already covers all 16 elements in the SF vector), wasting instruction slots on unnecessary cross-thread communication


## Solution

Templatize the kernel with THREADS_PER_SF and wrap __shfl_xor_sync in if constexpr (THREADS_PER_SF == 2), completely eliding the shuffle at compile time when each thread covers the full SF vector, eliminating dead cross-thread communication with zero runtime overhead

## Key Lessons

- Use if constexpr with compile-time constants to elide unnecessary warp shuffle operations — when each thread already covers the full data tile (NUM_THREADS_PER_SF==1), __shfl_xor_sync is dead code that if constexpr removes with zero runtime cost
- The measurable impact of single-instruction elision depends on the ratio of eliminated instruction cost to total kernel cost; on SM100 with native FP4 PTX where quantization is a single cvt.rn.satfinite.e2m1x2.f32 per pair, removing dead shuffles becomes proportionally more impactful than on simulated quantization with many float operations
