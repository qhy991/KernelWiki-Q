## Challenge

Hand-written blockReduceMax duplicates the entire reduction structure (warp shuffle, shared-memory staging, cross-warp reduction) from blockReduceSum, differing only in the reduction operator (max vs +), creating code duplication and maintenance burden across separate warpReduceMax and blockReduceMax templates


## Solution

Generalize the reduction template by parameterizing the reduction operator via a function pointer (ReduceFnType<T>), enabling both max and sum reductions to share a single warpReduce/blockReduce implementation; NVCC inlines the function pointer at compile time, producing identical SASS to the hand-written version with zero runtime overhead

## Key Lessons

- Function pointer dispatch for reduction operators in __inline__ __device__ templates is fully inlined by NVCC, producing identical SASS to hand-coded variants — safe to use for code deduplication without any performance penalty
- Parameterizing reduction templates with a function pointer (ReduceFnType<T>) eliminates code duplication between sum/max/min/block-scan variants while maintaining zero overhead, since the compiler resolves the indirect call at compile time for inlined device functions
