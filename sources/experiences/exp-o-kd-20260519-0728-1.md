## Challenge

Static __shared__ declarations cause NVCC to insert alignment padding between arrays of different element types (int16_t to int32_t boundary), wasting shared memory bytes and producing less uniform bank-mapping layouts that increase bank-conflict stalls during the hash-table probe and prefix-sum phases dominating this top-k LRU cache kernel


## Solution

Replace static __shared__ declarations with dynamic extern __shared__ plus an explicit SmemLayout struct that packs all int32_t arrays contiguously first, then all int16_t arrays contiguously after, eliminating inter-type alignment padding and producing a more uniform bank-mapping layout across the hash-table and buffer-scan phases

## Key Lessons

- Replace multiple static __shared__ declarations of mixed element types with a single extern __shared__ buffer plus manual pointer arithmetic packing all same-type arrays contiguously — eliminates compiler-inserted alignment padding between int16_t and int32_t regions, reducing total smem consumption and bank-conflict overlap
- Use a SmemLayout constexpr struct to compute byte offsets and total shared memory size at compile time, ensuring type-safe and maintainable manual layout while passing smem_bytes as the third launch configuration argument to the kernel
- Group same-sized types contiguously in shared memory (all int32_t arrays first, then all int16_t arrays) to avoid inter-type padding and produce more uniform bank-mapping — applies to any kernel with multiple shared arrays of different element sizes
