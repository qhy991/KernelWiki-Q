## Challenge

Scalar element-by-element quantization loop loads one float and stores one int8 per iteration, issuing separate load/store instructions per element and underutilizing available memory bandwidth


## Solution

Use vectorize_with_alignment<16> to load 16 floats (64 bytes) per vector operation in the quantize loop, providing coalesced memory access that partially offsets the occupancy loss from the smaller block size, with alignment-aware prefix/tail handling for arbitrary pointer offsets

## Key Lessons

- vectorize_with_alignment<VEC_SIZE> provides coalesced vector loads (16 floats = 64 bytes per op) for element-wise kernels, improving memory throughput over scalar access — applies whenever input elements are contiguous and the transformation is element-wise
- Vectorized memory access alone cannot compensate for a large reduction in SM occupancy; coalescing benefits are real but bounded by the available warp-level parallelism for latency hiding
