## Solution

Replace the compile-time constant pipe_stages=4 with a runtime stages parameter propagated through get_scales_cache_size, get_kernel_cache_size, and is_valid_config, enabling reduction to stages=2 on memory-constrained GPUs while maintaining stages=4 on GPUs with sufficient shared memory

## Key Lessons

- Parameterize pipeline stage count as a runtime value rather than compile-time constant — enables adapting shared memory footprint to GPU capabilities (stages=4 on Ampere 164KB, stages=2 on Turing 64KB) without recompilation
- Reducing pipeline stages trades prefetch depth for shared memory savings; on memory-constrained GPUs, fewer stages enable valid configurations that would otherwise exceed max_shared_mem
