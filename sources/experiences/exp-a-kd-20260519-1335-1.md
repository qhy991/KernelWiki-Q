## Challenge

Mixture-of-experts gating for 192 experts uses two separate CUDA kernels (moeSoftmax then moeTopK), requiring a 768 KB intermediate softmax probability buffer to be written to and re-read from global memory between kernel launches, dominated by the global memory round-trip latency


## Solution

Fuse softmax and topk into a single topkGatingSoftmax kernel using 8-byte vector loads (BYTES_PER_LDG=8, giving 2 floats per load and THREADS_PER_ROW=96 as a power of 2), with warp-level butterfly shuffle reductions for softmax max/sum and topk argmax all computed in registers, eliminating the intermediate global memory buffer and inter-kernel scheduling overhead

## Key Lessons

- Fuse softmax + topk into a single kernel when the intermediate softmax output is only consumed by the topk stage — eliminates a full global memory round-trip (768 KB write + re-read) and inter-kernel launch overhead, yielding 2.16x speedup for 192-expert MoE gating
- Use 8-byte vector loads (BYTES_PER_LDG=8) to enable non-power-of-2 expert counts (e.g., 192) in fused gating kernels — the resulting THREADS_PER_ROW=96 is a power of 2, satisfying warp shuffle constraints without requiring NUM_EXPERTS to be a power of 2
- Prefer warp-level __shfl_xor_sync butterfly reductions over shared-memory BlockReduce for per-row reductions when THREADS_PER_ROW cleanly divides the warp size — avoids shared memory bank conflicts and sync overhead, and keeps all intermediate values in registers
