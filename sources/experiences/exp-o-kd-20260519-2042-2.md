## Solution

Compute VEC_SIZE dynamically using std::gcd(16/sizeof(scalar_t), hidden_size) to derive the maximum safe vector width as a compile-time-compatible value, ensuring the kernel always uses the widest possible vectorized memory access for any combination of data type and hidden dimension

## Key Lessons

- Use std::gcd(alignment_requirement / sizeof(scalar_t), hidden_size) to compute the maximum safe vector size at runtime — this generalizes hardcoded VEC_SIZE values and automatically selects wider vectors when the data layout permits it
- The gcd-based formula ensures the vector width always divides hidden_size evenly, preventing out-of-bounds access while maximizing memory throughput across different precision types (fp16/fp32/bf16) and model dimensions
