## Solution

Switch to kOptimized iterator algorithm with kUnity stride support, which precomputes a delta table (inc_next[]) for pointer advancement across filter dimensions and bitmask predicates (masks_[s][3]) for validity checks, eliminating per-tile coordinate recomputation in the hot path — advance() applies a single byte offset instead of recomputing filter indices and NDHWC coordinates

## Key Lessons

- Prefer kOptimized iterator algorithm over kAnalytic for CUTLASS convolutions when stride is unity — the precomputed delta table replaces per-tile coordinate recomputation with a single pointer offset, dramatically reducing iterator overhead in the hot path
- Precomputed bitmask predicates (masks_[s][3]) for boundary checks are far cheaper than per-access coordinate-based arithmetic including stride divisibility tests — applies to any implicit-GEMM convolution where the filter traversal pattern is known at kernel launch time
- For 3D convolutions in CUTLASS, the iterator algorithm choice has outsized impact (2.76x here) because the T,R,S filter dimension traversal amplifies per-tile overhead — always benchmark both kAnalytic and kOptimized paths when problem geometry is non-trivial
