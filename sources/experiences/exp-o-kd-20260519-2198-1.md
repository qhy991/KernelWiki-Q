## Solution

Increase BLOCK_SIZE from 16 to 32 so THREAD_GROUP_SIZE becomes 1, eliminating the intra-group shuffle reduction in QK dot products, doubling vector load width to uint4 (128-bit), and halving the number of KV blocks to iterate

## Key Lessons

- Increasing BLOCK_SIZE to match WARP_SIZE (BLOCK_SIZE=32) sets THREAD_GROUP_SIZE=1, eliminating intra-group shuffle reduction in QK dot products — applies whenever each warp processes one full block of tokens and threads can independently own their QK dot computation
- BLOCK_SIZE=32 doubles the vector load width (uint2→uint4, 64→128 bits per load) because VEC_SIZE = 16/(THREAD_GROUP_SIZE×sizeof(element)) increases when THREAD_GROUP_SIZE drops from 2 to 1 — wider loads improve memory throughput for sequential K/V cache access
- Larger BLOCK_SIZE halves the number of KV blocks to iterate for the same sequence length, reducing block-table lookup overhead and loop iteration count — beneficial when block-table indirection is non-negligible relative to compute
