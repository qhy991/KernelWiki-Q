## Solution

Replace the divergent branch with a single predicated PTX inline asm load (ld.global.cg.v4.u32) that either reads 128 bits from global memory or zero-fills the destination registers based on a boolean predicate, eliminating the branch and memset overhead

## Key Lessons

- Predicated PTX loads (ld.global.cg.v4.u32 with @pred guard) can eliminate divergent bounds-checking branches by zero-filling registers when the predicate is false — but when the original branch rarely diverges (e.g., aligned tile dimensions), the benefit is negligible and the asm barrier plus .cg cache hint can introduce measurable overhead
- The .cg (cache-global) hint in ld.global.cg bypasses L1 cache and forces reads from L2/global, which is counterproductive when data fits in L1 — prefer .ca or let the compiler select the default caching policy for accesses that exhibit spatial locality within a tile
- Inline PTX asm blocks act as optimization barriers that prevent the compiler from fusing the load with subsequent computation (e.g., __habs2 on the loaded values), so hand-written predicated loads should be benchmarked against the compiler's own predicated-load generation before committing to the asm path
