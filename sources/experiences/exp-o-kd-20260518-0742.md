## Challenge

Optimization focus: Shared memory caching to eliminate redundant global memory reload in two-pass per-token FP8 quantization kernel.
Problem before: The per_token_quant_fp8_kernel processes each token in two passes: Pass-1 computes the max absolute value across the hidden dimension for scale calc...


## Solution

The optimization is most impactful when the per-token data footprint is large enough that L1 caching cannot serve the second pass.

## Results

- **speedup**: 100x


## Key Lessons

- Since this kernel is fundamentally memory-bound (the arithmetic is lightweight — just abs/max in Pass-1 and multiply/clamp in Pass-2), the redundant global memory loads are the primary performance bottleneck.
- Problem before: The per_token_quant_fp8_kernel processes each token in two passes: Pass-1 computes the max absolute value across the hidden dimension for scale calculation, and Pass-2 applies FP8 quantization using that scale.
