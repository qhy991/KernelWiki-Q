## Solution

Coarsen the loop by processing FPP=2 fragments per outer iteration, using partitioned shared memory (double the SMEM allocation) so both fragments are loaded and stored between the same pair of barriers, halving total barrier count from 32 to 16

## Key Lessons

- Loop coarsening (processing multiple fragments per iteration with partitioned SMEM) directly reduces __syncthreads() barrier count — each barrier forces full thread-block convergence, so halving the count yields measurable speedup when barrier latency dominates the kernel
- Double SMEM allocation to hold multiple fragments is a profitable trade-off when the alternative cost is extra barriers — SMEM is cheap (tens of cycles to allocate) compared to barrier synchronization latency
