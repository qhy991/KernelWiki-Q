## Solution

Fuse renormalization directly into the moeTopK kernel: accumulate the topk weight sum during the selection loop (threadIdx.x==0 accumulates row_sum_for_renormalize per iteration), then perform the inverse-multiply normalization in a final pass within the same kernel, eliminating the second kernel launch and keeping the output values in cache

## Key Lessons

- Fuse lightweight post-processing (renormalization, bias addition, masking) into the producing kernel when the post-process reads only the kernel's own output — the data stays in L2 cache and one kernel launch overhead is eliminated
- Accumulate a running sum during the main computation loop (here, row_sum_for_renormalize during topk selection) rather than re-reading output values afterward, reducing redundant global memory traffic
- A conditional renormalize flag parameter allows the same kernel to serve both fused and unfused code paths without code duplication, maintaining backward compatibility while gaining the fusion speedup
