## Challenge

TopK selection uses CUB BlockReduce invoked k=8 times (once per top element), each requiring shared-memory tempStorage and __syncthreads barriers, while the O(k*k) prior-element masking loop redundantly reads topk_indices from global memory up to 7 times per expert per thread iteration


## Solution

Replace CUB BlockReduce TopK with register-resident local argmax followed by warp-shuffle XOR butterfly reduction across 8 collaborating threads per row, using -10000.f sentinel masking on the register array to exclude already-selected experts without any global memory re-reads

## Key Lessons

- For small-group reductions (≤32 collaborating threads), warp-shuffle XOR butterfly eliminates shared-memory allocation and __syncthreads barriers compared to CUB BlockReduce — applies when cooperative threads fit within a single warp
- Mask already-selected TopK entries by overwriting register-resident values with a large negative sentinel (-10000.f) instead of maintaining a separate exclusion list with global memory reads — avoids O(k*n) redundant global memory accesses per TopK iteration
