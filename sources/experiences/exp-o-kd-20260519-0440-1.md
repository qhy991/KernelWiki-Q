## Challenge

__cvta_generic_to_shared is called on every shared memory element address inside both the prolog and main K-tile loops of a WMMA TF32 SGEMM kernel, issuing redundant address-conversion instructions in the cp.async issue path on every iteration


## Solution

Pre-compute __cvta_generic_to_shared once on the shared memory base pointers (s_a, s_b) before the loops, then derive per-element cp.async destination addresses using pure register byte-offset arithmetic with constexpr stage strides, eliminating repeated cvta calls from the hot path

## Key Lessons

- Hoist __cvta_generic_to_shared out of hot loops by pre-computing the shared memory base pointer once and using byte-offset arithmetic for per-element addressing — eliminates redundant address-conversion instructions and gives the compiler clearer scheduling hints for cp.async issue paths
- For multi-stage shared memory buffers with cp.async, compute stage strides as constexpr (e.g., s_a_stage_offset = BM * (BK + A_PAD)) so per-element byte-offset calculation reduces to pure integer arithmetic with no address-space conversion overhead
