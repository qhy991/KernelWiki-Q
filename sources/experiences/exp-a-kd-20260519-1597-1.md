## Solution

Replace the per-thread 2D token-count matrix with a warp-partitioned static shared_counts[32][8] array where each warp owns 8 experts and threads use atomicAdd to increment counts, eliminating the per-expert prefix-sum pass entirely, removing one __syncthreads barrier, and enabling a 1024-thread launch (16x more parallelism than the original 64 threads)

## Key Lessons

- Warp-partitioned shared memory (e.g., shared_counts[num_warps][experts_per_warp]) with atomicAdd replaces per-thread 2D counting + prefix-sum with a single atomic accumulation phase — eliminates one __syncthreads barrier and the O(blockDim.x × num_experts) prefix-sum loop
- Static shared memory arrays (known size at compile time) avoid the overhead and complexity of extern __shared__ dynamic allocation when the problem dimensions are bounded (e.g., 64 experts → 32 warps × 8 experts)
