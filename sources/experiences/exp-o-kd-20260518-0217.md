## Solution

Replace erff-based exact GELU with tanh-based Taylor approximation (GELU(x) ≈ 0.5x(1 + tanh(√(2/π)(x + 0.044715x³)))) using the single-cycle tanh.approx.f32 PTX hardware intrinsic, eliminating the expensive SFU erff call while maintaining acceptable numerical accuracy

## Key Lessons

- Replace erff() with tanh-based Taylor approximation using tanh.approx.f32 PTX intrinsic for GELU activation — the hardware tanh is a single-cycle instruction vs erff's multi-cycle SFU path, yielding ~2x speedup when the kernel is compute-bound on fp16 data
- The GELU Taylor approximation 0.5x(1+tanh(√(2/π)(x+0.044715x³))) is numerically close enough for inference workloads — applicable whenever exact erf precision is not required and approximate GELU is acceptable
- Wrap PTX inline asm (tanh.approx.f32) in a host/device inline function with a CPU fallback (tanhf) to maintain portability and host-side testing compatibility
