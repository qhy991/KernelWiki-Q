## Solution

Fuse both kernels into a single advance_step_flashinfer_kernel where each thread reads seq_len, computes block_index and block_offset once in registers, then derives all six outputs (tokens, positions, seq_lens, slot_mapping, paged_kv_last_page_len, block_table_bound) from those register-held values without redundant global memory reads

## Key Lessons

- Fuse kernels that operate on the same per-element data and share intermediate computations (seq_len, block_index, block_offset) into a single kernel — eliminates redundant global memory reads and saves ~5-10us per avoided kernel launch, which is significant for lightweight kernels on small batch sizes
- When two kernels read the same input arrays (seq_lens, block_tables) and compute overlapping intermediates, fusing them allows those intermediates to reside in registers across both logical phases, avoiding a round-trip through global memory
