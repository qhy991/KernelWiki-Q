## Challenge

Scalar GELU kernel loads and stores one float per thread using individual 32-bit memory transactions, underutilizing available memory bandwidth on this memory-bound elementwise operation


## Solution

Use float4 vectorized load/store (reinterpret_cast<float4*> via FLOAT4 macro) to process 4 floats per thread in a single 128-bit coalesced memory transaction, reducing total transaction count by 4x and improving memory coalescing for the bandwidth-bound elementwise kernel

## Key Lessons

- Vectorize memory access with float4/float2 loads and stores in memory-bound elementwise kernels — processing 4 elements per thread reduces memory transaction count by 4x and improves coalescing, directly lowering effective latency per element
- Scale thread index by elements_per_thread (idx * 4) and reduce block_size proportionally (256/4=64) to maintain the same total element coverage with fewer threads issuing wider memory transactions
