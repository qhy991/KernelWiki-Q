## Challenge

In tree speculative decoding verification, every draft token must pass a purely stochastic acceptance test (coin < prob_acc), which can reject draft tokens that the target model assigns very high probability (e.g. 0.95+), wasting verification computation and lowering acceptance rates that directly control speculative decoding throughput


## Solution

Replace the pure-stochastic acceptance condition with a combined greedy+stochastic criterion: coin <= prob_acc / threshold_acc || target_prob_single >= threshold_single. A greedy shortcut unconditionally accepts high-confidence tokens, while dividing prob_acc by a tunable threshold_acc (clamped by fmaxf to prevent division-by-zero) scales the stochastic acceptance probability. The < to <= change also accepts the exact boundary case.

## Key Lessons

- Combining a greedy shortcut (target_prob >= threshold_single) with a tunable stochastic threshold (prob_acc / threshold_acc) in speculative decoding verification increases acceptance rates by bypassing stochastic checks for high-confidence tokens — applies to any speculative decoding loop with probability-based acceptance
- Defaulting tunable thresholds to 1.0 and clamping with fmaxf(threshold_acc, 1e-9f) preserves backward compatibility while enabling runtime adjustment of acceptance aggressiveness without code changes
