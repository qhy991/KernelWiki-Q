## Solution

Replace tanh-approximation GELU with exact GELU using ::erf, which computes f * 0.5 * (1.0 + erf(f * M_SQRT1_2)) — fewer arithmetic instructions (one FMUL before the SFU call vs two FMULs plus x^3) and mathematically exact results

## Key Lessons

- Both tanhf and ::erf map to CUDA SFU hardware with similar throughput — when a kernel is memory-bandwidth-bound (3:1 memory-to-compute byte ratio), swapping between equivalent SFU-backed math functions produces no measurable performance difference
- Replacing tanh-approximation GELU with exact erf-based GELU improves numerical accuracy at no performance cost in memory-bound kernels — use the exact formulation whenever correctness matters more than instruction count
