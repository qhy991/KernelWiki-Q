## Challenge

The before kernel (lines 19-25) decomposes each half2 into low and high halves via __low2half/__high2half, then converts each scalar half to float with __half2float — 4 scalar conversion calls per iteration. The after kernel (lines 19-24) uses __half22float2 for a single vector conversion per pair —


## Solution

(Ampere A800), __half22float2 internally expands to the same cvt.rn.f32.f16 PTX instructions as two __half2float calls because the hardware lacks a paired half2-to-float2 conversion unit; each half element is still converted individually by the FPU.
