## Solution

Replace the 128-iteration element-wise get/set loop with a 4-iteration loop that applies the ~ (bitwise NOT) operator directly on each uint32_t storage word, flipping all 32 bits per word in a single instruction and eliminating all sub-byte extraction and insertion overhead

## Key Lessons

- When operating on packed sub-byte data (1-bit, 4-bit, etc.), prefer word-level bitwise operations on the native storage type (uint32_t) over per-element extract-process-insert loops — a single NOT/XOR/AND on a 32-bit word processes 32× to 8× more logical elements in one instruction
- Eliminating sub-byte element access (get/set with shift-mask-read-modify-write) in favor of bulk word-level operations reduces loop iterations by the packing factor and removes per-element branching and conditional bit manipulation overhead
