## Solution

Replace full O(N log N) CUB radix sort with a custom multi-pass radix selection kernel: build a coarse 256-bin FP16 histogram to identify the threshold bin via prefix sum, directly output elements clearly above threshold, discard those below, and refine only the small ambiguous threshold-bin subset across up to 4 byte-level FP32 passes — reducing work from O(N log N) to O(N + threshold_bin_subset)

## Key Lessons

- For top-K selection where K << N, multi-pass radix selection replaces full sort with O(N) histogram + O(threshold_bin_subset) refinement — the speedup grows as N/K increases because most elements are classified above or below threshold in the coarse pass and need no further processing
- Two-level radix precision (coarse FP16 8-bit binning followed by byte-level FP32 refinement) efficiently narrows the ambiguous candidate set: the 256-bin FP16 histogram isolates the threshold bin with ~0.4% precision, so only a small fraction of elements need the expensive multi-byte refinement
