## Solution

Load directly into source array elements by reinterpret_cast-ing the Array struct to a raw float pointer and passing source_ptr[i] as the destination to global_load, eliminating the intermediate tmp variable and the subsequent copy assignment

## Key Lessons

- Eliminating a single register-to-register copy (tmp → source[i]) by loading directly into the target via reinterpret_cast is structurally sound but yields no measurable speedup when warp shuffles and arithmetic dominate kernel runtime — the saved MOV instruction is negligible relative to shuffle and accumulation overhead
- Compilers like nvcc may already coalesce a predicated memcpy load followed by an assignment into a single load into the destination register, making manual elimination of intermediate variables redundant at the instruction level
- reinterpret_cast on a POD array to obtain a raw element pointer is a valid pattern for direct-load APIs, but only worth applying when the load path is on the critical path and the compiler cannot prove the copy is dead
