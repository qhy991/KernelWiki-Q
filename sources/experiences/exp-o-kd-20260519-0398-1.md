## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 pattern), where each thread issues 4 half2 loads per iteration to process 4 elements, collapsing the loop from 32 to 8 iterations and doubling per-warp effective bandwidth from 64 to 128 bytes per iteration

## Key Lessons

- Replace scalar half memory accesses with half2 vectorized loads (f16x4 pattern: 4 half2 loads per iteration = 8 half elements) in bandwidth-bound GEMV/GEMM kernels to double effective memory throughput — the wider 128-byte transactions better saturate the memory bus compared to 64-byte scalar transactions
- Collapsing the K-dimension loop by a vectorization factor reduces per-thread instruction count (fewer branches, index computations) and improves arithmetic intensity by processing multiple multiply-adds per memory load round
- Use reinterpret_cast<half2*> via a HALF2 macro to load pairs of contiguous half values as a single 32-bit transaction without requiring explicit __ldg or __half2float2 intrinsics — works when the source address is 4-byte aligned
