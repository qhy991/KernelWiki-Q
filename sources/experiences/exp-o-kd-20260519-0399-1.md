## Solution

Process 2 FP16 elements per thread using half2 vectorized loads/stores via reinterpret_cast, consolidating two separate 2-byte scalar memory transactions into one 4-byte half2 transaction and halving the total memory transaction count for the same data volume

## Key Lessons

- Use half2 vectorized loads/stores (via reinterpret_cast<half2*>) to process 2 FP16 elements per thread in memory-bound elementwise kernels — halves the number of memory transactions and improves bandwidth utilization when elements are contiguous and 4-byte aligned
- Stride the thread index by 2 (idx = 2 * linear_tid) and halve the grid size when switching from scalar to half2 vectorized processing, keeping total thread count proportional to the reduced workload
