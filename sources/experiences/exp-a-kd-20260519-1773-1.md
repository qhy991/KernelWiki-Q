## Challenge

Standard GEMM call uses CUBLAS_OP_N/CUBLAS_OP_T operand ordering which may not select the optimal cuBLAS kernel for small-M shapes; the swap-AB technique swaps transpose flags to compute the equivalent result via a different internal kernel path


## Solution

Swap the transpose flags from (CUBLAS_OP_N, CUBLAS_OP_T) to (CUBLAS_OP_T, CUBLAS_OP_N) to compute D=B^T*A instead of C=B*A^T, which selects a different cuBLAS internal kernel variant intended to improve memory access for small-M batched GEMM shapes

## Key Lessons

- Swap-AB (transposing both cuBLAS operand flags) is a CUTLASS technique for small-M GEMM shapes where swapping A/B roles can improve tile utilization — but in isolation via cuBLAS it provides no benefit and may slightly degrade due to transposed epilogue writes
- Swap-AB benefit in CUTLASS SM90 comes from the combined effect of swap-AB plus FP8FastAccum schedule and wider K tiles; a cuBLAS-only microbenchmark cannot replicate this advantage
- At M=64 the problem sits at the boundary where swap-AB stops helping — for M≤64 the first operand already has enough rows for good memory utilization in the default kernel path
