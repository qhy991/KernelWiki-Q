softmax_vs_sigmoid_fallback

## Solution

The after moeSigmoid kernel replaces all of this with a single strided loop: read input, compute 1/(1+exp(-x)), write output. No shared memory, no __syncthreads barriers, no multi-pass reads. The ~2x speedup comes from eliminating 2 of 3 global memory read passes, 1 of 2 global write passes, all ...

## Key Lessons

- The before moeSoftmax kernel performs three separate strided passes over global memory: (1) read input to find per-row max via shared-memory reduction with log2(256)=8 __syncthreads barriers, (2) read input again to compute exp(x-max) and write to output, then shared-memory sum reduction with 8 more
