## Solution

Use AlignedArray<float,4> (16-byte) vector loads with interleaved addressing so each thread loads 4 floats per LDG, and 2 threads per row cooperatively cover all 8 experts, yielding full warp utilization with 16 rows processed concurrently per warp

## Key Lessons

- Use AlignedArray<T,N> with 16-byte alignment for vectorized loads when elements are contiguous and type-aligned — 4x the memory throughput per LDG instruction compared to scalar loads
- Interleaved addressing (THREADS_PER_ROW threads cooperate on one row with offset = thread_group_idx * ELTS_PER_LDG) ensures coalesced access across the warp while keeping all threads active, avoiding the 8-of-256 thread utilization problem of scalar strided loops
