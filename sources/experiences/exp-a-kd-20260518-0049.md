## Challenge

The after code uses three separate inline PTX asm volatile blocks (one per mask dimension m0, m1, m2), each containing setp+predicated mov+output mov — roughly 6 PTX instructions per mask element, totaling 18 instructions. The asm volatile barrier prevents the compiler from merging or reordering acr


## Solution

Replace conditional branch with inline PTX predicated mov (setp + @p mov) to eliminate branch divergence, but the approach regresses because asm volatile barriers prevent compiler cross-block optimization and add register pressure, making it slower when warp threads are mostly uniform

## Results

- **speedup**: 0.169x (regression)


## Key Lessons

- Inline PTX asm volatile blocks prevent the compiler from merging or reordering across blocks and add register pressure (.reg .pred, .reg .u32 per block) — only use when the compiler's generated branch code is demonstrably slower due to severe warp divergence
- When warp threads are mostly uniform on a condition (e.g., 2/3 of threads take the same path), the compiler's branch code with SSY/BRA is well-predicted and outperforms explicit predication — do not replace branches with PTX predication without profiling the actual divergence pattern
- Three separate asm volatile invocations for related mask clears lose the compiler's ability to optimize them as a group; a single branch controlling three assignments is cheaper than three independent predicated mov sequences when the branch is coherent within the warp
