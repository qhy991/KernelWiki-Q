## Solution

Fuse the activation and scale stages into a single kernel so the activation result stays in registers for the scale multiply, eliminating the intermediate temp buffer and its global-memory round trip; use a template parameter kProcessRows to adapt rows-per-block to total work size

## Key Lessons

- Fuse producer-consumer kernel pairs into a single kernel when the intermediate tensor fits in registers — eliminates a full global-memory write + read round trip, which dominates cost at medium tensor sizes where compute is not the bottleneck
- Use a compile-time template parameter (kProcessRows) to let the launcher select rows-per-block based on total work (tokens x vectors) — increases occupancy for small problems and preserves memory coalescing for large ones without code duplication
