## Solution

Replace the single-thread serial loop with CUB BlockScan<int32_t,1024> ExclusiveSum to compute the prefix sum in parallel across all 1024 threads, distributing each expert's padded count to a thread and performing a parallel scan tree with log2(1024)=10 depth steps

## Key Lessons

- CUB BlockScan parallel prefix sum over a fixed-size block (1024 threads) carries constant infrastructure overhead (shared-memory scan stages, warp shuffles) regardless of how many inputs are meaningful — when the actual data count is very small (e.g., 8 experts), a serial loop on one thread can be equally fast or faster, making parallel scan an unnecessary complication
- Parallel scan algorithms only pay off when the input size is large enough to amortize their log(N) tree-depth overhead; for scan lengths below ~64–128 elements, the serial O(N) loop on a single thread is often competitive or superior due to zero coordination overhead
