## Challenge

Two separate kernel launches — rescale_kernel divides each row's topk elements by the row sum, then scale_kernel multiplies the same elements by a scaling factor — requiring two full global-memory traversals over the output array and incurring double the kernel launch overhead


## Solution

Fuse rescale (divide by sum) and scale (multiply by factor) into a single kernel that performs both operations per element in one read-modify-write pass, halving global memory traffic and eliminating one kernel launch

## Key Lessons

- Fuse back-to-back elementwise kernels that read/write the same buffer into a single kernel to halve global memory traffic — the saving grows with array size but launch overhead elimination matters most for small kernels
- When fusing kernels that operate on identical row-major data layouts, the per-thread row assignment and loop structure can be reused directly, making fusion a low-risk transformation
- Add a runtime boolean flag (e.g., apply_routed_scaling_factor_on_output) to keep the fused kernel flexible when the second operation is conditionally needed — avoids maintaining separate kernel codepaths
