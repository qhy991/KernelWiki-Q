## Solution

Replace B_PAD=8 padding with B_PAD=0 and apply XOR-permutation swizzle (swizzle_permuted_B_j with (((j>>3)^(i>>2))%2)<<3) to scatter 8-byte chunks across different banks for different row groups, eliminating structural bank conflicts while keeping smem layout compact at BK=16 columns per row

## Key Lessons

- Replacing shared memory padding with XOR-permutation swizzle eliminates bank conflicts more efficiently than static padding — XOR patterns scatter accesses across banks without wasting memory space, which is critical when smem budget limits K_STAGE pipelining depth
- Compacting smem rows from BK+PAD to just BK (e.g., 16 vs 24 columns) frees up to 50% of B smem per tile, enabling higher pipeline stage counts or larger tile sizes that would otherwise be constrained by the 48KB shared memory limit
