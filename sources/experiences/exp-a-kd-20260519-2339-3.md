## Solution

Coarsen the grid to 64 blocks per batch element by having each block compute 4 outputs (TZ=4) using the z-dimension of thread index, reducing block scheduling overhead by 4x while maintaining the same total thread count

## Key Lessons

- Coarsen GPU grids by having each block compute multiple outputs via the thread z-dimension when individual output computations are lightweight — reduces block launch overhead and improves SM occupancy without changing total thread count
- Replace atomic-add output writes (+=) with direct assignment (=) when each output element is written by exactly one block, avoiding atomic transaction overhead
