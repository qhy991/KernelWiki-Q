## Solution

Switch the value parameter from const reference (T const &) to pass-by-value (T) and add a single-argument overload that bakes in the default zero threshold via T(), eliminating the explicit zero variable at each call site

## Key Lessons

- Pass-by-value vs const-reference for small types (<=16 bits like __half) produces no measurable performance difference at -O3 — the compiler inlines both paths and loads the value into a register regardless, so this refactor is purely cosmetic
- For bandwidth-bound kernels (especially half-precision elementwise ops at 2 bytes/element), micro-optimizations in the functor calling convention are completely hidden by memory throughput — focus optimization effort on memory access patterns instead
- A single-argument overload with T() default threshold reduces caller boilerplate without affecting codegen — the compiler evaluates T() (zero initialization) at compile time for __half
