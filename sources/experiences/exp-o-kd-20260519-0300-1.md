## Solution

Replace the __shfl_sync-based warp index broadcast with direct arithmetic (threadIdx.x / 32), which each thread computes independently without requiring warp-level synchronization, thereby eliminating the reconvergence stall after the divergent branch

## Key Lessons

- Prefer direct arithmetic (threadIdx.x / 32) over __shfl_sync for values that are locally computable without cross-lane communication — __shfl_sync forces warp reconvergence after divergent branches while direct computation proceeds independently per-thread
- __shfl_sync acts as an implicit reconvergence barrier that forces all lanes to wait for the slowest branch path — avoid it in hot loops that follow divergent control flow where the value can be derived from per-thread registers alone
