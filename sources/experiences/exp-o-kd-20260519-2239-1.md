## Solution

Add an is_a_8bit boolean parameter to conditionally select 1 byte per A-matrix element for INT8 activations versus 2 bytes for FP16, halving the A-matrix shared memory footprint and enabling previously invalid tile configurations to fit within the 64KB shared memory limit

## Key Lessons

- Size shared memory buffers based on actual element size of the data type rather than using a worst-case constant — INT8 activations need 1 byte versus 2 for FP16, so a data-type-aware multiplier avoids wasting shared memory on architectures with tight limits
- Over-allocated shared memory from hardcoded element sizing can prevent otherwise-valid tile configurations from fitting, blocking the use of higher-occupancy thread configs
