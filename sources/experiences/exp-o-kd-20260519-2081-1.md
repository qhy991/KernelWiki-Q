## Solution

Add ENTRY_SIZE and CTA_SIZE as compile-time template parameters, replacing runtime entry_size/blockDim.x with constexpr values so all loop bounds become compile-time constants, enabling full #pragma unroll and eliminating runtime stride arithmetic

## Key Lessons

- Convert runtime-invariant values (entry sizes, CTA dimensions) into compile-time template parameters so loop bounds become constexpr — this enables full #pragma unroll, eliminates runtime division/modulo for stride arithmetic, and produces tighter straight-line instruction schedules
- Use assert(CTA_SIZE == blockDim.x) to guard consistency at runtime while still letting the compiler treat CTA_SIZE as a compile-time constant for optimization purposes
