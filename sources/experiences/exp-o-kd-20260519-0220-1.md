## Challenge

NDHWC coordinate decomposition uses three sequential integer division/modulo pairs (6 slow / and % operations) to split a flat index into (n, d, h, w), stalling the integer division pipeline on GPU


## Solution

Replace all three integer division/modulo pairs with CUTLASS-style FastDivmod using precomputed multiply-shift sequences (__umulhi + shift), converting 6 slow division operations into 6 fast multiply-shift operations that avoid the GPU's bottlenecked integer division unit

## Key Lessons

- Replace repeated integer division/modulo with precomputed FastDivmod (multiply-shift via __umulhi) when divisors are loop-invariant — integer division on GPU occupies a single pipeline with ~20-cycle latency while __umulhi completes in a few cycles
- Precompute divisor metadata (multiplier, shift_right) once on the host and pass as a Params struct to avoid redundant per-thread computation of the magic constants
- FastDivmod quotient = __umulhi(dividend, multiplier) >> shift_right and remainder = dividend - quotient * divisor is algebraically equivalent to div/mod but uses only fast multiply and subtract hardware
