## Solution

Pre-compute aligned_expert_first_token_offset array on the host (or a separate kernel), then replace the shared memory load, syncthreads barrier, serial accumulation loop, and shuffle broadcast with two O(1) __ldg lookups per thread, eliminating all inter-thread synchronization

## Key Lessons

- Pre-compute per-expert prefix-sum results (aligned offsets) on the host or in a separate lightweight kernel, then look them up with O(1) __ldg reads — eliminates shared memory allocation, __syncthreads barriers, and O(N) serial loops within the hot path, yielding large speedups when N is small enough to fit in read-only cache
- When a warp-level reduction pattern (serial loop on lane 0 + shuffle broadcast) exists only because one thread needs to compute a result that depends on data available to all threads, replace it with each thread independently computing the result via direct global reads — removes both divergence and the broadcast step
- __ldg loads through the read-only texture cache avoid shared memory bank conflicts and syncthreads overhead for small read-only lookup tables (e.g., ~8 int64_t entries), since the data stays in L1/L2 cache across all blocks and never needs explicit shared memory management
