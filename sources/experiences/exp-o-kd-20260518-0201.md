## Challenge

The after version uses (row % 8) in the XOR remapping instead of the full row value, producing a consistent bank-conflict avoidance pattern wit


## Solution

Guard the accumulator pipeline consumer_wait with is_accumulator_needed so that when K=0 the wait is skipped entirely, preventing deadlock without affecting the K>0 path

## Results

- **speedup**: 1.058x (improved)


## Key Lessons

- Guard pipeline consumer_wait calls with a data-availability predicate (e.g., is_accumulator_needed) when the producer may not emit data for certain problem dimensions — prevents deadlock in edge-case problem sizes like K=0 grouped GEMM
- Pipeline wait operations must account for cases where the producer stage is skipped entirely; a conditional guard is cheaper than a timeout or recovery mechanism and preserves correctness for the common path
