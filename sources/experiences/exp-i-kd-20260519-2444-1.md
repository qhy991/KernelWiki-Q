## Solution

Instantiate the CUTLASS LinearCombinationPlanarComplex<float,4,float,float> functor from params, load ArrayPlanarComplex<float,4> via reinterpret_cast for zero-copy access to planar complex memory, apply the functor to compute alpha*accum+beta*source, and store the result through reinterpret_cast — delegating all arithmetic and layout handling to the library functor

## Key Lessons

- Use reinterpret_cast with CUTLASS ArrayPlanarComplex types for zero-copy load/store of planar complex data — the library types exactly match the memory layout of interleaved real/imaginary pairs, avoiding manual element-wise packing
- Delegate epilogue arithmetic (alpha*accum + beta*source) to CUTLASS's pre-built LinearCombinationPlanarComplex functor rather than reimplementing it — the functor internally handles scaling, type conversion, and planar complex layout correctness
- Instantiate the functor from its Params struct once per thread, then call it as a function object — CUTLASS epilogue functors are designed to be stateless callable objects that encapsulate all scaling parameters
