## Challenge

Naive SGEMM kernel has each thread independently read an entire row of A and column of B from global memory across all K iterations, causing massive redundant global memory traffic and becoming gmem-bandwidth bound


## Solution

Use shared memory tiling (block tile + K tile): a 32x32 block cooperatively loads BMxBK and BKxBN tiles into shared memory once per BK step, then all threads reuse that data from smem (~19 TB/s) instead of gmem (~900 GB/s), transforming the kernel from gmem-bound to smem-bound compute

## Key Lessons

- Shared memory tiling in GEMM cooperatively loads BMxBK and BKxBN tiles into smem per BK step so all threads in a block reuse the same data — applies whenever multiple threads access overlapping rows/columns of the input matrices
- The K-dimension tiling loop (bk = 0 to K/BK) with __syncthreads() barriers between load and compute phases ensures correctness: each tile is fully loaded before any thread reads it, and fully consumed before overwriting with the next tile
- Shared memory bandwidth (~19 TB/s) is ~20x higher than global memory bandwidth (~900 GB/s), making smem tiling the first optimization to apply for memory-bound GEMM kernels before considering register tiling or vectorized loads
