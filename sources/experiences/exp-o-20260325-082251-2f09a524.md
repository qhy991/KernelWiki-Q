## Challenge

Improve topk kernel beyond 216.7% ratio for vocab_size=160, k=8 on A800 GPU


## Solution

8 warps per block (256 threads) warp-shuffle approach with __ldg loads

## Key Lessons

- NEVER include cudaDeviceSynchronize in the extern C wrapper - the test framework handles sync and this adds ~4.7ms overhead
- For small vocab (160) with warp-level parallelism, 8 warps/block outperforms 4 warps/block due to better occupancy
- Single-thread approach is slower than warp approach for vocab_size=160 because sequential 160-iteration loop is slower than parallel 32-thread warp reduction
- The warp_id calculation must use the WARPS_PER_BLOCK constant consistently: blockIdx.x * WARPS_PER_BLOCK + (threadIdx.x / WARP_SIZE)
- __ldg loads improve memory access for read-only data
