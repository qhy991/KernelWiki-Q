## Solution

STAGE=4 already uses ~41 KB (with A_PAD=4), and K_STAGE=5 would exceed the 48 KB static limit (~51 KB), causing compilation or launch failure.
Applied change: Replace the static 3D shared memory arrays with a single extern __shared__ float smem[] allocation, then partition it into s_a and s_b region

## Key Lessons

- Expected impact: Enables higher pipeline stages (K_STAGE=5+) that increase global-memory/compute overlap, potentially improving throughput for large K.
