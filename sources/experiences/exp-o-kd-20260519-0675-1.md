## Solution

Scan all index pairs and detect consecutive runs where both src_diff==1 and dst_diff==1, then call transfer_page_direct once per run with num_tokens equal to the full run length, merging O(num_pages) individual copy calls into O(num_runs) larger contiguous transfers

## Key Lessons

- Merge consecutive index pairs into single larger DMA transfers instead of issuing one copy per page — run-length merging reduces DMA dispatch overhead from O(num_pages) to O(num_runs) and enables contiguous memory bus throughput that is significantly higher for large transfers
- Access tensor data via raw data_ptr<int64_t>() instead of repeated .item<int64_t>() calls in hot loops — avoids per-element Python/C++ dispatch overhead and enables pointer arithmetic for consecutive difference checks
- When the caller already sorts host_indices before passing them, consecutive runs are common and run-merging becomes highly effective — exploit known ordering properties of upstream data to batch operations
