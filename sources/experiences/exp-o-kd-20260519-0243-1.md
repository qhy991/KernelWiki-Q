## Solution

Replace per-element bit extraction loop (32 iterations) with direct byte-level scanning (4 iterations): reinterpret the packed storage as raw bytes and OR-check each byte for non-zero, since any non-zero byte implies at least one set bit, reducing iterations by 8x and eliminating shift/mask/division/modulo overhead

## Key Lessons

- For logical OR over packed boolean arrays, scan raw bytes instead of extracting individual bits — any non-zero byte proves at least one bit is true, reducing N iterations to ceil(N/8) and eliminating per-element shift/mask/division/modulo instructions
- Avoid per-element accessor abstractions (operator[] with bit extraction) in hot reduction paths — reinterpret the underlying storage as raw bytes and iterate at the native storage granularity to bypass extraction overhead
- When reducing packed boolean predicates (OR/AND), the byte-level predicate (byte != 0 / byte == 0xFF) is semantically equivalent to checking all constituent bits — applies to any packed-bit reduction where the predicate distributes over groups of bits
