## Challenge

The epilogue kernel reads source[i] from global memory and computes beta*src even when beta=0, wasting 16MB of global memory bandwidth per 4M-element invocation because the runtime-only is_source_needed() check prevents the compiler from eliminating the dead source-loading code path


## Solution

Use compile-time ScaleType::OnlyAlphaScaling to make is_source_needed() return false at compile time, enabling the compiler to eliminate the entire source-loading code path via dead code elimination and reduce total memory traffic from ~48MB (acc+src+out) to 32MB (acc+out)

## Key Lessons

- Compile-time dispatch (e.g., ScaleType enum) eliminates dead code paths that runtime checks cannot, because the compiler can prove values are constant and remove entire load-compute branches — applies whenever a parameter is known to be zero/unused at a higher level
- In bandwidth-bound elementwise epilogue kernels, removing one of three global memory arrays (source) reduces total traffic by ~33%, yielding significant speedup even though the arithmetic reduction is minimal
