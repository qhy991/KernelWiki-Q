## Solution

Replace 4 scalar __half additions with 2 native __half2 vector additions via union type-punning: reinterpret Uint4x2 as __half2[2] and issue two HADD2 instructions, halving the arithmetic instruction count from 4 to 2 per call

## Key Lessons

- Scalar FP16 addition via extract-add-repack loops already compiles efficiently on NVIDIA (4 FADD instructions), so replacing with __half2 HADD2 may not yield speedup when union type-punning introduces register pressure that offsets the instruction count reduction
- Vectorized __half2 arithmetic optimizations that show clear benefit on AMD (where native __half2 addition avoids expensive library function calls) can be neutral or slightly slower on NVIDIA due to different compiler codegen paths — hardware-specific profiling is essential before porting cross-vendor optimizations
- Union-based type punning between Uint4x2 and __half2[2] is a clean zero-cost abstraction for packed FP16 operations on AMD but may not be the optimal codegen path on NVIDIA where direct __half2 intrinsic constructors could be more efficient
