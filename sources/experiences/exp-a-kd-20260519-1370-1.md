## Solution

Replace scalar bf16 loads/stores with 128-bit vectorized int4 loads/stores via reinterpret_cast, collapsing 18 scalar memory operations into only 3 vectorized load/store pairs (2 x 128-bit for NOPE, 1 x 32-bit for ROPE), dramatically reducing dispatched instruction count and pipeline occupancy per warp

## Key Lessons

- Use 128-bit (int4) or 256-bit vectorized loads/stores via reinterpret_cast instead of scalar element loops when copying contiguous memory regions — each vector instruction moves 4-8x more data, reducing total dispatched instructions proportionally
- Compute vec_loads at compile time as DIM * sizeof(DType) / (VEC_SIZE * 32) to let the compiler fully unroll the vectorized loop, eliminating loop overhead entirely
