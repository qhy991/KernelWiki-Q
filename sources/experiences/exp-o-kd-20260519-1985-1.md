## Challenge

Two separate kernels — silu_and_mul_kernel writes a 512KB __half intermediate buffer (64 tokens × 4096 hidden) to global memory, then scaled_quant_kernel reads it all back for int8 quantization — wasting ~1MB of global memory bandwidth (write + read) plus one extra kernel launch overhead


## Solution

Fuse silu_and_mul and scaled_quant into a single fused_silu_mul_quant_kernel where the float silu_result stays in registers and feeds directly into the quantize step, eliminating the entire 512KB intermediate write-read round-trip and one kernel launch

## Key Lessons

- Fuse consecutive kernels where one writes an intermediate that the next immediately reads — keeping the value in registers eliminates a full global memory write-read round-trip and one kernel launch, with benefit scaling proportionally to intermediate buffer size
- When an activation (SiLU, GELU, ReLU) is immediately followed by quantization (int8/int4), fusing them is especially valuable because the intermediate fp16/fp32 result never needs to materialize in global memory at all
- Kernel fusion savings grow with problem size: at 64 tokens × 4096 hidden the ~1MB eliminated traffic (512KB write + 512KB read) dominates latency, making the fusion speedup consistent and predictable across batch dimensions
