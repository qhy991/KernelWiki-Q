## Challenge

The before kernel executes a dequant_4bit function per k-iteration that shifts and masks 4-bit fields from packed integers, then calls __float2half for each extracted value. This adds 8 shift+mask+conversion operations (4 per packed int, 2 packed ints) before the subtraction can proceed


## Solution

The after kernel reads half values directly from memory (simulating the frag_zpf register array already loaded via int4) and applies sub_zp_half immediately, eliminating all unpacking and type-conversion arithmetic.
