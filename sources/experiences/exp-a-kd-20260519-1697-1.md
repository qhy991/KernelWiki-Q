## Solution

Replace scalar float loads with 4-wide vec4_t reinterpretation loads to process four elements per memory transaction, combined with #pragma unroll 4 on the vectorized loop — but the approach regresses because register pressure and instruction overhead from struct-field deconstruction exceed the marginal bandwidth benefit when the working set is L2-cache-resident

## Key Lessons

- Vectorized loads (vec4_t reinterpretation) improve throughput only when the working set exceeds L2 cache capacity — for 64KB inputs that are L2-resident, the added register pressure and instruction overhead from struct-field deconstruction outweighs any memory bandwidth gain
- #pragma unroll 4 on the vectorized loop increases register usage per thread, which can reduce occupancy and harm performance when per-thread work is already fine-grained and the kernel is not memory-bandwidth-bound
- Always profile the actual working set size vs cache hierarchy before applying vectorization — small problem sizes (≤L2) may not benefit from wider loads and can regress due to overhead
