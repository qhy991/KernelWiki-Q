## Challenge

Two-pass dynamic per-token int8 quantization launches two separate kernels (compute_absmax_kernel then static_quant_kernel), requiring two full passes over the input tensor and a global memory round-trip for per-token scales between kernels


## Solution

Fuse absmax computation and quantization into a single kernel that computes per-token absmax via blockReduceMax, broadcasts via shared memory with __syncthreads(), then quantizes in the same kernel invocation, eliminating redundant input traversal and the global memory round-trip for scales

## Key Lessons

- Fuse multi-pass kernels that share input data into a single kernel to eliminate redundant global memory reads and kernel launch overhead — applies whenever Pass 1 produces per-token metadata (scales, norms) that Pass 2 consumes, since the metadata can be communicated via shared memory within the same thread block instead of a global memory round-trip
- Use __shared__ memory + __syncthreads() to broadcast block-level reduction results (e.g., absmax) to all threads within a fused kernel, avoiding the need to write and re-read intermediate results from global memory
