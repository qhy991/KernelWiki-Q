## Solution

Replaces 4 separate half2 loads with a single LDST128BITS 128-bit load into a local array, uses __hmax2 on half2 pairs instead of scalar __hmax, and consolidates stores into one 128-bit write — however the local array staging through .local memory adds overhead that causes regression at small problem sizes

## Key Lessons

- Local array staging (pack_x/pack_y) routes data through .local memory in PTX — the global→local→compute→local→global path adds address computation and local-memory access instructions that direct-register approaches avoid, causing net regression when the working set is cache-resident
- 128-bit packed loads (LDST128BITS) and vectorized intrinsics (__hmax2) reduce instruction count but the benefit only materializes when the eliminated global-memory transactions dominate; for small problem sizes (~2MB) fitting in L2 cache, the transaction overhead is already low and the local-memory staging cost dominates
- Prefer direct register-based half2 computation over local-array staging for bandwidth-bound elementwise kernels on cache-resident data — wider load/store instructions help only when global memory bandwidth is the actual bottleneck, not when intermediate local memory traffic introduces more overhead than the wider transactions save
