## Solution

Fuse the GPU kernel and CPU host function into a single GPU kernel that computes both expert_offsets and 128-aligned blockscale_offsets prefix sums in one serial loop, eliminating cudaDeviceSynchronize, both cudaMemcpy transfers, and the CPU-side loop entirely

## Key Lessons

- Fuse GPU kernel + CPU host computation into a single on-device kernel when the CPU work is lightweight (e.g., serial prefix-sum over a few hundred entries) — the host-device sync and transfer overhead (cudaDeviceSynchronize + bidirectional cudaMemcpy) dominates actual computation time by orders of magnitude
- When a GPU kernel produces intermediate data that a CPU function consumes and feeds back to GPU, evaluate whether the CPU logic (alignment rounding, prefix sums, index arithmetic) can be expressed directly in CUDA — even serial single-thread GPU execution is faster than a full host-device round-trip
