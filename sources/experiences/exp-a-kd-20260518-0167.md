## Solution

Replace wave-based ceiling-division rounding with a simpler two-branch heuristic: if work_blocks ≤ 2×avail_sms return as-is, otherwise use fast_max(work_blocks, 4×avail_sms) to ensure minimum grid occupancy without over-allocation from strict wave alignment

## Results

- **speedup**: 1.001x (neutral)


## Key Lessons

- Simplifying grid-size rounding from strict wave-aligned ceiling division to a fast_max heuristic can produce identical grid sizes for common cases while reducing control-flow complexity — neutral performance but cleaner code
- When sm_occupancy=1 the wave-based formula (ceil(work/gpu_occ)*gpu_occ) and the fast_max(work, avail_sms*4) heuristic coincide for work_blocks near but below 4×SM count — validate across multiple occupancy levels to find cases where they diverge
