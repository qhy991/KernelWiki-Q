## Solution

Replace scalar stride loop with DefaultVecOp<4> wrapper over vec_n_t<float,4>, using reinterpret_cast to issue 128-bit vectorized load/store instructions that process 4 floats per iteration, plus a scalar tail loop for remainder elements, improving memory throughput and L2 cache line utilization

## Key Lessons

- Use vec_n_t<float,4> with reinterpret_cast to issue 128-bit vectorized load/store instructions in memory-bound elementwise kernels — each wide transaction fully consumes a 128-byte L2 cache line instead of loading it for just 4 bytes, improving bandwidth utilization
- Always include a scalar tail loop after the vectorized loop to handle remainder elements when hidden_size is not a multiple of VEC_SIZE — ensures correctness without requiring the caller to pad input
- The DefaultVecOp wrapper pattern separates the vectorization strategy from the per-element operation via a device lambda, making it reusable across different elementwise operations while keeping the vectorized load/store logic identical
