## Solution

Change the C/D operand pointers from uint32_t* to float* and the corresponding inline asm constraints from "r" (integer register) to "f" (float register), directly matching the F32 accumulator register bank that HMMA instructions natively use

## Key Lessons

- Match inline asm constraint letters ("f" for float, "r" for integer) to the native register bank of the PTX instruction's operands — this eliminates cross-register-bank MOV instructions on toolchains that do not optimize the constraint mismatch
- Modern NVCC (12.8+ with -O3) performs correct register bank allocation regardless of C++ pointer type and asm constraint, subsuming this optimization — always validate with SASS disassembly before deploying constraint-only changes
- Historical optimizations targeting register-bank mismatches should be benchmarked on the current toolchain, as compiler improvements may render them neutral or unnecessary
