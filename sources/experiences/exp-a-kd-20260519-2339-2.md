## Challenge

The before kernel uses ~8KB shared memory for pipeline staging plus a shared y_warpwise[TY] array and multiple block.sync() barriers to perform cross-warp reduction across 4 warps, incurring synchronization and shared-memory bank overhead


## Solution

Replace the multi-warp block with a single-warp thread_block_tile so the entire reduction uses cg::thread_block_tile<TX>::shfl_down without any shared memory barriers, then broadcast the result via g.shfl(sum, 0)

## Key Lessons

- Use cg::thread_block_tile instead of shared-memory cross-warp reduction when each output element can be computed by a single warp — eliminates block.sync() barriers and shared-memory bank conflicts
- g.shfl(val, 0) broadcasts the warp-reduced result to all threads without writing to shared memory
