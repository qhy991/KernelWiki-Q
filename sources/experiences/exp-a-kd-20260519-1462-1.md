## Solution

Cache input into shared memory during the absmax pass and quantize from shared memory in a second phase, eliminating the redundant global memory read — however, for ~1MB datasets on A800 (40MB L2), the second global read is already an L2 hit, so the shared-memory copy + __syncthreads() barrier overhead cancels any DRAM savings

## Key Lessons

- Shared-memory caching to eliminate redundant global reads is not beneficial when the working set fits entirely in the GPU L2 cache — the second global read becomes an L2 hit at comparable latency to shared memory, and the explicit smem copy + __syncthreads() barrier adds net overhead
- Before applying shared-memory staging, estimate total data footprint versus L2 cache size: if footprint ≪ L2 capacity (e.g., ~1MB vs 40MB on A800), the multi-pass global approach is already cache-resident and shared memory adds only overhead
- __syncthreads() barriers between smem phases have a fixed cost that must be amortized over significant DRAM savings to be worthwhile — for small per-block working sets the barrier alone can negate the benefit
