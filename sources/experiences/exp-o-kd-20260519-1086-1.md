## Solution

Replace KernelParamsDynamic (runtime mutable int fields) with a templated KernelParams<VPT, NUM_EXPERTS, THREADS_PER_ROW, ROWS_PER_WARP, ROWS_PER_CTA, WARPS_PER_CTA> struct where all fields are static constexpr, pushing parameter computation to compile time and enabling NVCC to fully unroll all #pragma unroll loops (32-iteration sigmoid, 32-iteration bias-add, 3-iteration shuffle reduction) into straight-line code

## Key Lessons

- Convert kernel-internal parameters (VPT, THREADS_PER_ROW, etc.) derived from kernel arguments into compile-time template parameters with static constexpr fields — this gives NVCC exact loop bound knowledge, enabling full unrolling of #pragma unroll loops that would otherwise be treated as variable-trip-count by the compiler
- When kernel launch parameters are known constants at the call site (e.g., num_experts=256, num_expert_group=8 for DeepSeek V3), template specialization eliminates runtime parameter computation overhead and produces better-optimized shuffle reduction sequences with known mask progressions
- The performance impact of compile-time parameter specialization is largest in kernels where loop trip counts dominate computation time — MoE gate kernels with 32-element per-thread sigmoid/bias-add/argmax loops and warp shuffle reductions are prime candidates for this pattern
