## Solution

Replace uint64_t types and "l" register constraints with native double types and "d" (64-bit float register) constraints, matching the DMMA instruction's actual register bank and eliminating cross-bank mov.b64 transfer instructions on older toolchains

## Key Lessons

- Use "d" register constraints instead of "l" for F64 MMA inline ASM operands to match the DMMA hardware register bank — this eliminates mov.b64 cross-bank transfer instructions on older CUDA versions (pre-12.x) where ptxas does not optimize the mismatch
- Modern NVCC (12.8+) with -O3 generates identical SASS regardless of register constraint choice for F64 MMA, but using the correct "d" constraint from the start ensures portability across CUDA toolchain versions
- For PTX inline ASM, always match the C++ type and register constraint to the instruction's native register bank ("d" for double/DMMA, "f" for float, "r" for int32) rather than relying on bit-equivalent integer types
