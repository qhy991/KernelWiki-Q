## Challenge

Optimize RMS Norm CUDA kernel for LLaMA-3-8B hidden_size=4096, target performance_ratio >= 1.3


## Solution

Stripped torch/pybind from baseline, added extern C entry point with adaptive kernel selection (128/256/512 threads based on batch size). Baseline already uses float4 vectorization, warp shuffle reduction, __ldg cache optimization, __fdividef fast division.

## Key Lessons

- The baseline kernel for rms_norm hs=4096 already has excellent optimizations (float4, warp shuffle, __ldg) achieving 217.5x speedup over reference
- Stripping torch headers and adding extern C entry point is sufficient - no additional optimization needed
- Adaptive kernel selection (128/256/512 threads) based on batch size is effective for rms_norm
- head -N to truncate + cat heredoc to append is a clean way to replace torch-dependent host functions
