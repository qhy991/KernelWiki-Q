## Solution

Replace the hardcoded head_size stride with a configurable int64_t head_stride kernel parameter, enabling the kernel to handle non-contiguous tensor layouts directly without host-side copies, while producing identical address arithmetic for the contiguous case

## Key Lessons

- Parameterizing tensor strides (separating logical head_size from physical head_stride) enables kernel reuse across contiguous and non-contiguous memory layouts without measurable performance cost, since the compiler generates equivalent address computation when the runtime values are identical
- Adding an explicit stride parameter even when the common case has stride == size is a low-cost defensive design choice — the extra int64_t parameter adds zero overhead in the common case and avoids host-side transpose/copy when non-contiguous tensors arrive
