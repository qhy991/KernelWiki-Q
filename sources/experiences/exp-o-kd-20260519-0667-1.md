## Solution

Replace scalar float loads/stores with float4 (128-bit) vectorized loads/stores via a VecFloat4 wrapper providing cast_load/cast_store, processing 4 floats per memory transaction to reduce total transactions by 4x and enable fully coalesced 128-bit wide accesses in the bandwidth-bound hidden-dimension copy loop

## Key Lessons

- Replace scalar float loads/stores with float4 (128-bit) vectorized access in bandwidth-bound inner loops — 4x fewer memory transactions and fully coalesced 128-byte accesses directly translate to higher effective throughput when the kernel is memory-bound
- Wrap float4 in a struct with cast_load/cast_store and indexed access operators to keep the element-wise computation logic clean while gaining vectorized memory throughput — applies to any loop that processes contiguous float arrays with per-element computation
