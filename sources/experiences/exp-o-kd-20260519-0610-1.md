## Solution

Replace serial single-thread loop with a parallel segmented prefix sum: 16 active threads each compute a local prefix sum over their segment of 16 experts into shared memory, then a cross-segment phase adds preceding segment totals to each entry, and block_size alignment is applied in a second identical two-phase pass

## Key Lessons

- Replace serial single-thread loops over moderate-sized arrays (64–1024 elements) with segmented parallel prefix sums: divide elements into segments per thread, compute local prefix sums per segment, then aggregate segment totals via a cross-segment prefix — 16 threads each handling 16 elements achieved 2.3x speedup over serial
- Use a shared memory buffer to store per-segment totals for cross-segment aggregation, separated by __syncthreads() barriers — this two-phase pattern (local prefix + cross-segment prefix) generalizes to any segmented scan or reduction on small-to-moderate arrays
- When the computation requires an alignment step (CEILDIV × block_size), apply it as a second pass over the unaligned parallel prefix sum using the same two-phase segmented pattern rather than combining it into the first pass
