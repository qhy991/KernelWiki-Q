## Challenge

The inner epilogue loop over head_dim columns uses a runtime variable (head_dim_value) as the loop bound even when the tile size equals head_dim at compile time, forcing the compiler to generate loop prologue, conditional branch, and runtime comparison on every iteration of the hot path


## Solution

Introduce a constexpr bool kSingleIteration = kMaxK <= TileN template parameter to resolve the loop bound at compile time; when true the loop collapses to a single iteration, eliminating the runtime branch and isLastColumn comparison entirely

## Key Lessons

- Use constexpr or template parameters to replace runtime loop bounds with compile-time constants when the iteration count is statically known (e.g., tile size equals head_dim) — the compiler folds the loop into straight-line code, removing branch overhead and loop prologue
- Fold dependent conditionals (like isLastColumn checks) into the same constexpr guard so that the compiler eliminates not just the loop but also downstream branches that become trivially true or false
