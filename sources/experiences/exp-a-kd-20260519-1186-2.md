## Challenge

Metadata storage allocates only 1 float (4 bytes) per row, which is insufficient for W4A8 quantization that requires per-row activation scale, sub-scale, and quantization descriptors


## Solution

Expand metadata allocation from tb_m * 4 to tb_m * 16 bytes (4 floats per row) to accommodate the full W4A8 quantization descriptor set including activation scale, sub-scale, zero-point, and quantization parameters

## Key Lessons

- W4A8 (4-bit weight, 8-bit activation) quantization requires 4 floats per tile-row of metadata: activation scale, sub-scale, zero-point, and descriptor — allocate metadata proportionally to quantization format complexity, not as a fixed single float per row
- When expanding metadata in shared memory, the kernel's metadata initialization loop bounds must also expand (tb_m → tb_m * 4) to cover the full metadata region
