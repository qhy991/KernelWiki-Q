RMS Norm kernel optimization for hidden_size=512, KernelEvalPlus RN_03 task

## Key Lessons

- For small hidden_size (<=512), single warp per row is optimal - eliminates inter-warp sync
- Warp shuffle (__shfl_down_sync + __shfl_sync broadcast) replaces shared memory reduction entirely
- float4 vectorized loads (4 per thread for 512 elements) maximize memory bandwidth
- __launch_bounds__(32) helps compiler optimize register allocation
- Performance jumped from 84.6% to 244.4% of baseline with this approach
- The key insight: kernel launch overhead dominates for small sizes, so minimize synchronization points
