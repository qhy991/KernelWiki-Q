## Solution

Consolidate 4 separate half2 loads into one 128-bit LDST128BITS transaction and replace 8 scalar __hmax calls with 4 vectorized __hmax2 calls on half2 pairs, staging data through thread-local arrays (pack_x/pack_y) — however the local-memory staging adds overhead that outweighs the benefits at small cache-resident problem sizes

## Key Lessons

- Thread-local array staging (pack_x[8]/pack_y[8]) backed by .local space in PTX introduces extra global-to-local-to-global data movement overhead — for cache-resident workloads at small problem sizes, direct register-based access avoids this penalty even with more instruction count
- __hmax2 vectorized comparison halves the instruction count (8→4) for FP16 ReLU and LDST128BITS consolidates memory transactions, but compute savings can be negated when the staging mechanism adds more overhead than it saves — apply 128-bit packed load/store only when the data path stays in registers or shared memory
- Per-element boundary checks (4 separate if-guards) can be collapsed into a single (idx+7)<N guard when using packed 128-bit access, reducing branch overhead — but this simplification alone does not compensate for local-memory staging costs at small scales
