## Challenge

The quantization kernel computes per-token min and max values using two separate cub::BlockReduce passes (one for Max, one for Min) with an intervening __syncthreads() barrier, wasting shared memory bandwidth and synchronization cycles on a reducible serial dependency


## Solution

Merge the two separate BlockReduce passes into a single pass using a custom MinMax reduction struct that tracks both min and max simultaneously, eliminating one full BlockReduce traversal and one __syncthreads() barrier

## Key Lessons

- Merge independent scalar reductions (Min, Max) into a single struct-based reduction to halve the number of BlockReduce passes and remove intermediate __syncthreads() barriers — applies whenever multiple commutative reductions operate on the same input data
- Define a custom reduction type with operator overloads (+=, &=) to compose naturally with cub::BlockReduce, keeping the kernel code concise while enabling fused multi-field reductions
