## Solution

Pass k_scale and v_scale as const float* pointers instead of scalar-by-value, dereference them once into local register variables at kernel entry, reducing kernel launch parameter overhead and allowing the compiler to treat the cached values as pure register operands

## Key Lessons

- Passing small scalar parameters via const float* pointer and dereferencing once into a local register variable can reduce kernel launch parameter overhead compared to scalar-by-value passing, yielding measurable speedup in compute-bound microbenchmarks (~10.8%)
- Pointer-based parameter passing enables future extensibility to per-head or per-token dynamic scaling without changing the kernel signature — the dereference pattern remains identical regardless of whether the scale tensor contains one or many values
- In isolated microbenchmarks where the kernel body is lightweight, parameter passing overhead is proportionally significant; in full production kernels where memory accesses dominate, this optimization has negligible real-world impact
