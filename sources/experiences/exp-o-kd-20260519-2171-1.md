## Challenge

Zero-point fetch in the Marlin kernel issues num_ints_per_thread (2) separate scalar 4-byte loads from shared memory per k-iteration, totaling 256 load instructions across 128 iterations, incurring high instruction overhead and missing memory coalescing opportunities


## Solution

Consolidate the inner loop's multiple scalar 4-byte loads into a single 16-byte int4 vectorized load per k-iteration, reducing total load instructions from 256 to 128 and enabling the GPU to service the requests through fewer memory transactions

## Key Lessons

- Replace loops of scalar 4-byte shared/global memory loads with a single wider vectorized load (int4 = 16 bytes) when consecutive elements are accessed — halves the instruction count and enables coalescing into fewer memory transactions
- When benchmarking memory access patterns, single-thread configurations amplify instruction-count differences (7.8x here); real multi-thread kernels see smaller but still significant gains from the same vectorization principle
