## Solution

Fuse 3 separate single-destination kernel launches into a single multi-destination kernel that reads each source element once into a register and writes to all 3 destination buffers, eliminating 2 kernel launches and 2/3 of redundant global memory reads

## Key Lessons

- Fusing multiple identical kernel launches (same source, different destinations) into a single read-once-write-many kernel eliminates both redundant global memory reads and per-launch overhead — especially effective for small batch sizes where launch overhead dominates total latency
- Reading source data into a register (int32_t val) before writing to multiple destinations reduces L2 cache pressure since each source cache line is fetched only once instead of N times for N destinations
