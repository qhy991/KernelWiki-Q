## Challenge

The before kernel reads shared memory and immediately consumes the value in a multiply-accumulate within the same dependency chain, preventing the shared memory load latency from being hidden by overlapping with FFMA execution


## Solution

Stage shared memory values into register arrays (r_comp_a[TM], r_comp_b[TN]) via vectorized float4 loads before the compute loop, then use __fmaf_rn to decouple the load latency from the FFMA pipeline and improve instruction-level parallelism

## Key Lessons

- Stage shared memory into register arrays via vectorized float4 loads before the compute inner loop — this decouples smem load latency from FFMA execution, enabling the warp scheduler to overlap the two and improve instruction-level parallelism
- Use __fmaf_rn instead of += * for multiply-accumulate in tight loops — the explicit fused multiply-add intrinsic guarantees a single FFMA instruction without intermediate rounding, improving both throughput and numerical precision
