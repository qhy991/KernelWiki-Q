## Solution

The after kernel introduces offset-based indexing (sorted_token_ids_offset + it, cumsum_offset + expert_id, expert_ids_offset + i) computed from a runtime model_offset parameter to generalize for multi-model LoRA support, but since model_offset is a runtime parameter the compiler cannot constant-fold the zero-valued offsets away, adding extra integer arithmetic to every global memory operation

## Key Lessons

- Runtime offset parameters (model_offset) prevent the compiler from constant-folding zero-valued offsets — each global memory write incurs an extra integer add in the address pipeline even when the offset is always zero at a given call site, so prefer separate kernel variants or template specialization for the zero-offset path instead of a single generalized kernel
- In small kernels where launch overhead and instruction count proportionally dominate total latency, per-access arithmetic overhead from generalized addressing patterns is measurable — consider separate direct-access and offset-access kernel implementations rather than a single runtime-parameterized kernel
