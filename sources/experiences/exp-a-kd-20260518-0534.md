## Challenge

In the before code, rope_only_kernel writes k_rope to global memory and a separate kv_writeback_kernel re-reads those K values from global memory to write to k_buffer


## Solution

The after code keeps post-RoPE K values (y1, y2) in registers after computing them and writes directly to k_buffer in the same kernel, eliminating one global memory round-trip for K.
