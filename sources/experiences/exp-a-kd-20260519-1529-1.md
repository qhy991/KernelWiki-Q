## Solution

Remove the 8KB smemLogits shared memory buffer entirely, eliminate logit writes during candidate population, and replace the TopKSort radix sort with direct smemIndices-to-global store — indices are already ordered by histogram bin from coarse to fine, so a full sort is unnecessary when only index output is required

## Key Lessons

- Eliminate unnecessary output data (e.g., logit values) from kernel shared memory and computation paths when only indices are required — removing the corresponding smem buffer cuts shared memory pressure (~8KB) and can increase occupancy
- When a histogram/bin-based filtering stage already produces coarsely-ordered results, a final radix sort on the full top-K set may be unnecessary for index-only output — replacing sort+store with direct store saves both compute and additional shared memory for sort temp storage
- Audit kernel output contracts early: if downstream consumers only need indices, removing logit storage/write/sort from the entire pipeline eliminates shared memory writes, sort compute, and global memory writes in one coherent change
