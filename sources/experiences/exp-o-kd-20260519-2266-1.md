## Challenge

MoE expert token counting uses 128 threads each writing into a private per-thread 2D tokens_cnts[(threadIdx.x+1, expert_id)] array, requiring each thread to iterate ~12 elements and later perform a serial prefix-sum across all 128 thread rows to merge counts


## Solution

Replace per-thread private 2D counting array with a flat shared_counts array updated via atomicAdd by 1024 threads (8x more threads), reducing per-thread iteration from ~12 to ~1.5 elements and eliminating the subsequent cross-thread-row prefix-sum merge phase

## Key Lessons

- Use atomicAdd on a flat shared array instead of per-thread private counting arrays when the reduction key space (num_experts) is small — eliminates the cross-thread merge phase and scales per-thread work inversely with thread count
- Pad shared array sizes to warp boundaries (padded_num_experts = round_up(num_experts, 32)) to avoid bank conflicts and ensure full-warp initialization
