## Challenge

Sub-warp reduction for 4 active threads uses shared-memory butterfly pattern with __syncthreads() barriers — 3 barriers and 4 shared-memory transactions per iteration, making barrier overhead proportionally expensive for such a small reduction width


## Solution

Replace shared-memory butterfly reduction with a parameterized warp shuffle reduce_sum<kNumThreads> template using __shfl_xor_sync — 2 register-to-register shuffle instructions replace 3 __syncthreads barriers and all shared-memory traffic, yielding 8.4x speedup for the k=4 case

## Key Lessons

- Replace shared-memory butterfly reduction with __shfl_xor_sync warp shuffle for sub-warp reductions — eliminates __syncthreads barriers and shared-memory traffic entirely, producing direct register-to-register transfers that scale with reduction rounds
- Sub-warp reductions with small active thread counts (4–8) benefit disproportionately from warp shuffle because __syncthreads barrier cost is amortized over fewer useful operations, making the relative speedup higher than for full-warp reductions
- Templatize the reduction width via reduce_sum<kNumThreads> with a static_assert power-of-2 check — enables compile-time unrolling of shuffle rounds and safe reuse across different sub-warp widths
