## Challenge

The original code uses __shfl_sync(0xffffffff, threadIdx.x / 32, 0) to broadcast the warp index to all lanes, which acts as an implicit warp-level barrier forcing all 32 threads to converge before proceeding — even though every thread in the warp already computes the identical value (threadIdx.x / 32 is uniform across a warp)


## Solution

Replace the __shfl_sync warp-id broadcast with canonical_warp_idx(), which computes threadIdx.x / NumThreadsPerWarp directly via integer division without invoking a shuffle barrier, eliminating the unnecessary warp convergence requirement and removing shuffle instruction overhead

## Key Lessons

- Avoid __shfl_sync for broadcasting values that are already uniform across a warp (e.g., threadIdx.x / 32) — direct computation is semantically equivalent and removes an implicit convergence barrier that can stall divergent warps for tens to hundreds of cycles
- Prefer named utility functions like canonical_warp_idx() over raw intrinsics with magic numbers — they are self-documenting, include __CUDA_ARCH__ guards for host compatibility, and make the intent explicit
