## Solution

Replace per-warp subset scanning with one-thread-per-expert assignment: each thread reads its own shared_scores[tid] once, then performs direct warp-level reduction via __shfl_down_sync with #pragma unroll, eliminating the inner expert-scanning loop entirely

## Key Lessons

- One-thread-per-expert with direct warp reduction eliminates per-warp inner scanning loops, reducing shared memory reads from ~32 per warp to 1 per thread per top-k iteration — applies when thread count >= element count and threads can each own one element
- #pragma unroll on fixed-iteration warp reduction loops produces a known instruction sequence that amortizes better across many blocks at larger batch sizes
- Consolidating selected-expert storage into persistent shared arrays (selected_experts[]) and deferring output writes until after all top-k iterations reduces write contention vs interleaved global writes inside the loop
