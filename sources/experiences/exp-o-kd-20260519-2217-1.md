## Challenge

In MOE expert dispatch, getMIndices computing aligned expert token offsets was called after expandInputRowsKernelLauncher, forcing the expand kernel to compute aligned row indices on the fly via shared memory cooperative loading, __syncthreads barrier, serial O(num_experts) accumulation on lane 0, and __shfl_sync broadcast to all warp lanes


## Solution

Move getMIndices computation before expandInputRowsKernelLauncher and pass the pre-computed aligned_expert_first_token_offset as a kernel argument, enabling each thread to independently compute its aligned destination row with two __ldg reads instead of shared memory cooperative loading + __syncthreads + serial lane-0 accumulation + __shfl_sync broadcast

## Key Lessons

- Pre-compute alignment offsets in a separate kernel pass before launching the data-movement kernel, then pass the result as a kernel argument — this converts an O(num_experts) serial per-block computation with shared memory + __syncthreads + shuffle into O(1) per-thread __ldg reads, with speedup scaling linearly with expert count
- Reordering host-side kernel launches to move independent computation before dependent data-movement kernels eliminates redundant in-kernel recomputation, removing shared memory pressure and synchronization barriers from the critical data path
