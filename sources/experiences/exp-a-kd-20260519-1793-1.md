## Solution

Fuse the unscaled fp8→half conversion and the scale multiply into a single kernel that reads each fp8 element, converts to half via __nv_cvt_fp8_to_halfraw, promotes to float for the scale multiply, and converts back to half — eliminating the intermediate global memory buffer, one kernel launch, and the associated write+read round-trip

## Key Lessons

- Fuse multi-pass elementwise operations (convert + scale) into a single kernel to eliminate intermediate global memory buffers — each avoided write+read round-trip saves 2× bandwidth for the intermediate array, which dominates cost for memory-bound kernels
- Kernel launch overhead is non-trivial for small-to-medium workloads — removing one launch saves driver-side setup cost and allows the GPU to issue work more efficiently without an intervening synchronization gap
