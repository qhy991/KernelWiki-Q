## Solution

Replace the packed interleaved cache layout [num_blocks, num_heads, head_size/x, block_size, x] with a flat NHD layout [num_blocks, block_size, num_heads, head_size] so that tgt_idx = block_idx*block_stride + block_offset*NH*HS + head_idx*head_size + head_offset produces sequential addresses for adjacent threads, enabling fully coalesced 128-byte memory transactions

## Key Lessons

- Cache layout determines coalescing: NHD layout [blocks, block_size, heads, head_size] produces sequential addresses for adjacent threads writing the same token slot, while packed interleaved layouts scatter writes with block_size stride — always design the cache dimension ordering so the innermost dimensions match the thread iteration pattern
- Strided writes (array[i*stride]) in GPU kernels are a strong signal that the data layout should be restructured to make the stride dimension the outermost dimension, converting scattered access into contiguous access
- Key and value cache can share the same flat layout when both use the same [block_size, num_heads, head_size] ordering — a single tgt_idx formula serves both reads and writes, reducing address arithmetic complexity
