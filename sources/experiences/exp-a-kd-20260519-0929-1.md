## Solution

Replace scalar stride loads with float4 vectorized loads: reinterpret input and bias pointers as float4 arrays so each lane issues 3 coalesced 128-bit loads (covering 12 elements) instead of 12 strided 32-bit loads, dramatically improving memory transaction efficiency and coalescing within each warp

## Key Lessons

- Replace scalar stride loads with float4 vectorized loads when each warp processes a contiguous tile of elements — reduces memory transaction count by 4x and improves coalescing, though the realized speedup depends on what fraction of total kernel time the loading phase occupies
- When the compute-heavy phase (e.g., TopK reduction over 384 experts per top-k iteration) dominates kernel time, memory optimizations in the loading phase yield diminishing returns at larger batch sizes since the compute fraction grows proportionally
