## Challenge

RMS norm kernel hardcodes hidden_size as the input row stride, making it unusable for non-contiguous or padded input tensors where the physical stride differs from the logical hidden dimension size


## Solution

Introduce a separate input_stride kernel parameter so that input row addressing uses blockIdx.x * input_stride + idx instead of the hardcoded hidden_size, decoupling physical memory layout from logical dimension size while preserving identical addresses in the contiguous case

## Key Lessons

- Decoupling the input stride from hidden_size via a separate kernel parameter enables support for padded or non-contiguous tensor layouts with negligible register overhead (one extra int32) and zero performance regression in the contiguous case where stride equals hidden_size
- When adding a configurability parameter that resolves to the same value as the original hardcoded constant, the compiler generates identical memory addresses and the GPU executes identical load/store patterns, making the change purely an API improvement with no compute cost
