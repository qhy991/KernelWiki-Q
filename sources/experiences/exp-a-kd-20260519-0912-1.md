## Challenge

A single-thread <<<1,1>>> kernel iterates sequentially over 256 experts to compute prefix-sum offsets, performing 256 scalar stride-3 global memory reads and 256 dependent additions in one thread with zero parallelism and a serial dependency chain of length N


## Solution

Replace the single-thread sequential loop with a <<<1,256>>> CUB BlockScan that distributes memory reads across 256 threads (one per expert) and computes the inclusive prefix sum cooperatively in shared memory, reducing the serial dependency chain from O(N) additions to O(log N) sequential steps per scan phase

## Key Lessons

- Use CUB BlockScan for parallel prefix-sum instead of single-thread sequential loops — it distributes reads across all block threads and reduces the serial dependency chain from O(N) to O(log N), applicable for cumulative sums or scans that fit within one block or can be chunked
- Chunked BlockScan with shared-memory broadcast handles arrays larger than BLOCK_SIZE — the last thread of each chunk stores the running prefix sum in shared memory so all threads can use it as the base offset for the next chunk, avoiding multi-kernel launch overhead for moderately-sized arrays
