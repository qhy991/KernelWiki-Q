## Challenge

Eight separate kernel launches (one per LoRA adapter group), each with only 8 thread blocks, severely underutilize the 108 SMs on the A800 GPU, incurring 8x kernel launch overhead and leaving most SMs idle between small launches


## Solution

Consolidate all 64 tokens into a single batched_rotary_embedding_kernel launch with 64 thread blocks, using a per-token cos_sin_cache_offsets array to handle different LoRA group cache offsets, amortizing kernel launch overhead from 8 launches to 1

## Key Lessons

- Batch multiple small kernel launches into a single launch with more thread blocks when processing independent groups — kernel launch overhead (driver-side dispatch, stream synchronization, GPU idle gaps) dominates when each launch uses too few blocks to saturate SMs
- Replace a token_offset parameter added per-launch with a per-token offset lookup array (cos_sin_cache_offsets) to enable batched launches while preserving per-group cache addressing — the extra int64 global load per block is negligible compared to eliminating multiple kernel launches
