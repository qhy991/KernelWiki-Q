## Solution

Skip fetch_col_scale_to_shared at pipeline start when dequant_skip_flop is true, deferring the scale fetch to the write-out phase; this frees up a cp_async pipeline slot and reduces contention on the async copy engine during the main pipeline loop in the full Marlin kernel

## Key Lessons

- Deferring a shared-memory fetch from pipeline start to write-out phase frees a cp_async pipeline slot — the benefit is invisible in isolation but material when multiple async copies compete for copy-engine bandwidth in the full kernel
- Lazy/conditional data fetching in async copy pipelines reduces pipeline contention even when total bytes transferred remains unchanged — applies whenever a fetched value is not consumed in the same pipeline stage
