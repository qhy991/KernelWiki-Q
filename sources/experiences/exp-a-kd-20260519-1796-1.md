## Solution

Fuse the two-kernel pipeline into a single kernel that reads half, converts to float, divides by scale, and directly converts the float result to FP8 via __nv_cvt_float_to_fp8—eliminating the intermediate half buffer write/read, the half→float→half round-trip, and one kernel launch entirely

## Key Lessons

- Fuse pre-scale and type-conversion into a single kernel when both are elementwise and operate on the same data — eliminates one kernel launch and halves global memory traffic by removing the intermediate buffer write/read
- Use __nv_cvt_float_to_fp8 directly when the computation already produces a float result, instead of converting float→half→fp8 via __nv_cvt_halfraw_to_fp8 — avoids an unnecessary precision round-trip through half representation
- Kernel fusion is especially beneficial when the intermediate buffer is the same size or larger than the input (2N bytes of half vs N bytes of fp8 output), since eliminating that buffer removes the dominant memory traffic cost
