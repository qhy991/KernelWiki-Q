## Solution

Fuse all five elementwise kernels into a single silu_bw_fused_kernel that loads x1, x2, and dx4 once per element and reuses the computed sigmoid and x3 intermediate in registers, eliminating intermediate global memory writes/reads and kernel launch overhead

## Key Lessons

- Fuse elementwise kernels that share input data and produce intermediate results into a single kernel so intermediates stay in registers instead of global memory — applies whenever multiple elementwise passes read overlapping inputs or when one pass writes an intermediate consumed by the next
- Kernel launch overhead (~5–20 μs per launch) becomes significant when many small elementwise kernels are chained; fusing eliminates this overhead entirely and reduces the number of cudaMalloc allocations for temporary buffers
- Register-resident intermediates (sigmoid value, x3) avoid redundant recomputation across kernels — silu_forward_kernel and dx1_kernel both recompute the same sigmoid from x1, which fusion eliminates by computing it once
