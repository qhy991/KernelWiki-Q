## Solution

Increase VEC_SIZE from 8 to 16 so that 16 FP8 output bytes exactly fill a uint4 (128-bit) register, then replace the 8-iteration scalar byte store loop with a single *(uint4*) wide store that writes all 16 quantized bytes in one coalesced 128-bit memory transaction, halving loop iterations and consolidating stores

## Key Lessons

- Replace scalar byte-width output stores with uint4 (128-bit) vector stores when output elements are contiguous bytes — a single 16-byte coalesced transaction is far more efficient than 8 or 16 individual byte stores, especially when the output type is narrower than the input type
- Use a union of uint8_t[16] and uint4 to build aligned 128-bit output in registers before issuing a single vector store — this avoids alignment issues and lets the compiler emit one STG.128 instruction instead of multiple narrower stores
- Doubling the vector size (8→16) halves the number of loop iterations for both the max-finding and quantization passes, reducing loop overhead and improving instruction-level parallelism when the wider load/store is supported by the hardware
