## Solution

Replace explicit ternary clamp chains with fminf/fmaxf intrinsics and fold the five intermediate variables into a single compact expression: (u + 1.0f) * g / (1.0f + expf(-g * alpha)), reducing source-level complexity while producing equivalent PTX after compiler optimization

## Key Lessons

- Replacing ternary clamp chains with fminf/fmaxf intrinsics improves readability but yields no measurable speedup when the kernel is memory-bound (2 __ldg loads + 1 store per element dominate runtime) and nvcc already lowers ternaries to efficient select instructions at -O3
- Source-level intermediate variable reduction (5 named variables → 2) does not reduce register pressure when the compiler's register allocator already coalesces dead stores into the same physical registers
- Algebraic folding of a SwiGLU expression into a single return statement is a readability improvement, not a performance optimization — the compiler emits identical MUFU.EX2 + FMUL chains regardless of source-level factoring
