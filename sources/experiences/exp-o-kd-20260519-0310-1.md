## Challenge

MoE gated-activation pipeline uses two separate kernels launched sequentially: the first computes SiLU-gated activation and writes the full intermediate tensor to a global-memory temp buffer, then the second kernel reloads that buffer to apply per-expert prequant scaling, incurring a redundant global-memory round-trip that dominates latency on medium-width configurations


## Solution

Fuse gated-activation and per-expert scaling into a single kernel so the activation result (post_act) stays in registers and is immediately multiplied by the prequant scale before a single store to output, eliminating the intermediate global-memory buffer entirely; additionally use a compile-time template parameter kProcessRows with adaptive dispatch based on total work items to improve occupancy across varying problem sizes

## Key Lessons

- Fuse producer-consumer kernel pairs when the intermediate result fits in registers — keeping the activation in a register and immediately scaling it eliminates one full global-memory write+read round-trip, which dominates cost for medium-width MoE activation pipelines
- Use a compile-time template parameter (e.g., kProcessRows) with runtime adaptive dispatch based on total work items (num_tokens * num_vec) to let each thread block process multiple rows when the problem is small, improving GPU occupancy without penalizing large problems
