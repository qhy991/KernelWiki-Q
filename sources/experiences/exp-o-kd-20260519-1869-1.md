## Challenge

MoE candidate selection uses two separate kernels: compute_swb_kernel writes sigmoid(scores)+bias to a global-memory intermediate buffer (TOTAL_ELEMENTS floats), then candidates_before_kernel reads that buffer to find the per-group max, wasting one full global-memory write+read pass and an extra kernel launch


## Solution

Fuse sigmoid+bias computation directly into the candidates kernel: each thread reads raw scores[i] and bias[i], applies sigmoid_accurate inline, adds bias, then participates in the warp reduce-max — eliminating the intermediate swb buffer (TOTAL_ELEMENTS floats = 512KB global memory) and one entire kernel launch

## Key Lessons

- Fuse elementwise preprocessing (sigmoid+bias) into the consuming reduction kernel when each thread processes exactly one element — the inline arithmetic adds only 2-3 instructions per thread while eliminating an entire global-memory write+read pass and a kernel launch
- When num_experts_per_group equals WARP_SIZE (32), each lane handles exactly one expert so the fused loop iterates once and the warp reduce-max covers all candidates with zero wasted lanes
- Eliminating intermediate buffers that are written once and read once is the highest-impact fusion target: the saved global memory traffic (512KB in this case) dominates any added register pressure from inline computation
