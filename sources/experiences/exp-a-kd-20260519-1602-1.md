## Solution

Introduce an explicit input_stride parameter and compute a separate input_token_offset = blockIdx.x * input_stride for input addressing, while retaining token_offset = blockIdx.x * hidden_size for residual addressing, enabling correct handling of non-contiguous layouts with zero performance cost in the contiguous case where stride equals hidden_size

## Key Lessons

- Decouple input addressing from hidden_size via an explicit input_stride parameter — when stride equals hidden_size the compiler generates identical address arithmetic, so generality is free in the contiguous case and enables correctness for padded/strided tensors
- Separate input_token_offset and token_offset allow input and residual to have independent memory layouts, which is necessary when residual connections use a different stride than the main activation tensor
