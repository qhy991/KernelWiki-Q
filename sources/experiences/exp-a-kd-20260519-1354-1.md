## Solution

Swap A and B operands to recast the problem as (N,K)x(K,M) = (3072,1536)x(1536,4), tiled with <128,16,128>; the large dimension 3072 now fills 128-wide tiles with minimal waste and M=4 pads to only 16, reducing total padded work 4x

## Key Lessons

- For small-M batched GEMM (M << N), swap A/B operands so the large dimension N becomes the tiled M dimension — tile padding wastes proportionally less on large dimensions, yielding up to (ceil(M/tile_M)*tile_M)/(ceil(M/tile_N)*tile_N) reduction in padded work
- Dimension swapping (swap_ab) requires no kernel changes — only host-side data repacking and result transposition, making it a zero-risk optimization for any tiled GEMM implementation
- Padded workspace size is the primary overhead indicator for small-dimension GEMM: minimize max(ceil(d/tile))*tile over all dimensions by assigning the smallest physical dimension to the largest tile dimension
