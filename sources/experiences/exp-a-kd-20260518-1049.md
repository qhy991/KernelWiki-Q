float_aligned_4096

## Solution

per thread with fully coalesced scalar loads. The after kernel's vector path reduces iterations to 1 per thread (4096/16=256 vectors), but each iteration processes a 16-element vector struct with an unrolled loop calling the lambda — the NVCC compiler may not inline the lambda through the DefaultRea

## Key Lessons

- The after kernel's vectorize_read_with_alignment<16> fast-path takes the aligned route, loading 16-float (128-byte) vectors and then unrolling them via DefaultReadVecOp which calls the scalar fabsf/fmaxf lambda per element.
