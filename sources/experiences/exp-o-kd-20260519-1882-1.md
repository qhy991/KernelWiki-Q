## Challenge

Exclusive prefix sum over 256 expert counts is computed by a single thread (threadIdx.x==0) in a serial loop with per-iteration shared-memory loads and CEILDIV rounding, leaving all 1023 other threads idle at the barrier waiting for ~256 sequential iterations to complete


## Solution

Replace the single-thread serial prefix sum loop with CUB BlockScan<int32_t, 1024>::ExclusiveSum, which distributes the scan across all 1024 threads via a tree-based reduction with O(log2(1024))=10 parallel depth, eliminating the serial bottleneck entirely

## Key Lessons

- Replace single-thread serial prefix sum loops with CUB BlockScan::ExclusiveSum to distribute scan computation across all block threads — reduces O(N) serial iterations to O(log N) parallel depth, yielding major speedups when N is large relative to thread count
- Each thread reads its own element from shared memory before the scan, eliminating the serial loop's indexed shared-memory access pattern and enabling coalesced parallel reads
- Apply per-element transforms (e.g., CEILDIV rounding) before the parallel scan rather than during — keeps the scan primitive clean and lets BlockScan operate on already-rounded values
