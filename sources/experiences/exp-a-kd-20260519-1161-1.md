## Solution

Convert the runtime n_group parameter to a compile-time template parameter NGroup and add #pragma unroll to the outer loop, enabling full unrolling of the 8-iteration group loop so the compiler can eliminate branch and counter overhead

## Key Lessons

- #pragma unroll with compile-time template parameters increases register pressure by materializing all loop iterations simultaneously — for small loops with simple arithmetic, the loop-control overhead savings can be outweighed by reduced occupancy or register spills
- Loop unrolling is not universally beneficial: when per-iteration work is dominated by memory access latency (64 element loads per thread here), eliminating the branch and counter saves negligible time compared to the memory-bound critical path
- Profile before unrolling — a 0.98x result confirms that runtime loop bounds on small iteration counts are already well-handled by GPU hardware branch prediction, and forcing unroll via template parameters may be counterproductive
