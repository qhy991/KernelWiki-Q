## Challenge

The dequant_kU4B8 function returns a FragB struct (containing nv_bfloat162 data[2]) by value, requiring the compiler to handle an intermediate struct copy/move when the caller assigns the return value, which may introduce unnecessary register-to-register moves under register pressure


## Solution

Replace the return-by-value FragB pattern with an output-pointer parameter (nv_bfloat162* frag_b), writing dequant results directly into caller-provided memory and eliminating the intermediate FragB struct entirely

## Key Lessons

- NVCC applies NRVO (Named Return Value Optimization) for small structs in isolated microbenchmarks, making return-by-value and output-pointer patterns generate equivalent register-level code — the output-pointer optimization only yields measurable benefit when embedded in complex kernels with high register pressure
- BF16 dequant constants (EX=0x43004300, ADD=0xC308C308) use the BF16 exponent bias (0x43) instead of FP16 (0x64), but the return-vs-pointer calling convention optimization is independent of the compute type and produces identical results across precision specializations
