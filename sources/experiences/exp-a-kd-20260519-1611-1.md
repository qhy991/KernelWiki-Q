## Solution

Switch to sequential (non-interleaved) packing where each byte holds two consecutive nibbles in natural order, enabling direct sequential extraction (b0_lo→out[0], b0_hi→out[1], b1_lo→out[2], ...) without the cross-lane deinterleave shuffle; in the full Machete GEMM kernel this layout enables prmt-based wgmma upconversion and reduces shared-memory bank conflicts

## Key Lessons

- Data layout changes (interleaved vs sequential packing) must be evaluated in the full kernel context where they affect shared-memory bank conflicts and hardware upconversion paths (prmt-based wgmma), not in isolated microbenchmarks that only measure nibble extraction overhead
- A regression in a dequant microbenchmark does not necessarily indicate the layout is worse — nvcc register allocation and instruction scheduling artifacts can dominate in narrow tests that strip away the downstream consumer kernel
- Sequential nibble packing simplifies extraction logic (natural byte-lane order) but the interleaved layout may enable better warp-level memory access coalescing or prmt shuffle patterns when consumed by a GEMM kernel — the optimal layout is consumer-kernel-dependent
