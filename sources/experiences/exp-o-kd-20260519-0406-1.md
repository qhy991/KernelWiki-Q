## Solution

Vectorize from scalar half to half2 type: each thread loads two packed halves in a single 32-bit half2 transaction via HALF2 macro, accumulates variance from both components (reg_x.x² + reg_x.y²) into one half before block reduction, and writes both results in one half2 store, halving thread count from K to K/2

## Key Lessons

- Vectorize scalar half loads/stores to half2 to double memory throughput per transaction and halve thread count — applies to any elementwise or normalization kernel where adjacent elements can be processed independently
- Pre-sum multiple partial squares (reg_x.x² + reg_x.y²) into a single half before block reduction to reduce the number of values entering the reduction and improve dynamic range utilization
