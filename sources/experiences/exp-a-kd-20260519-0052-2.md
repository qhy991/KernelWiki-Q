## Solution

Precompute per-row byte-offset increments (inc_next_s/r/t) derived from stride, dilation, and tensor layout constants, then compute the final pointer as base_offset + ft*inc_next_t + fr*inc_next_r + fs*inc_next_s via simple int64 addition instead of calling ndhwc_offset() with four multiplications per access

## Key Lessons

- Precompute byte-offset increments from tensor layout constants (stride × dilation × dimension stride) outside the hot loop, then advance pointers via simple addition instead of recomputing full ND offsets — reduces 4 int64 multiplications per access to 3 additions
- Separate the base pointer computation (called once per row) from the filter-position delta (called per ft/fr/fs) — the base address is invariant across filter iterations, so hoisting it eliminates redundant multiplies
- Use char-pointer arithmetic with byte offsets for pointer advancement in ND tensor access patterns — allows direct addition of precomputed increments without repeated coordinate-to-offset conversion
