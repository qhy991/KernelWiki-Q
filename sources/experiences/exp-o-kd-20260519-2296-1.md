## Challenge

Token sorting for MoE expert assignment runs inside a single-block kernel where each thread sequentially writes to sorted_token_ids using a local prefix-summed count, limiting sorting parallelism to one block regardless of token count


## Solution

Decouple sorting into a dedicated multi-block kernel (up to 65535 blocks x 256 threads) where each thread atomically claims its output position via atomicAdd on the global cumsum_buffer, enabling GPU-wide parallel sorting independent of the alignment kernel's thread count

## Key Lessons

- Decouple sorting phases into a separate multi-block kernel when the original single-block approach bottlenecks on token count — scaling to 65535 blocks x 256 threads yields 22.62x speedup for large expert/token counts
- Use atomicAdd on a pre-computed cumulative-sum buffer (cumsum_buffer) instead of per-thread local prefix sums for scatter-position assignment — the atomic operation replaces the local count-and-increment pattern and enables lock-free parallel scatter across thousands of threads
- Grid-stride loops (for i = tid; i < numel; i += stride) in the multi-block kernel ensure all tokens are processed regardless of how block count maps to numel, making the kernel robust across varying token counts
