## Solution

Replace hardcoded per-N switch dispatch with an adaptive WVSPLIT_TILE heuristic that computes sYT = ceil(M_in / (CuCount * 4)) to estimate ideal tile size per compute unit, then selects YTILE and UNRL based on sYT magnitude: small sYT uses smaller tiles with more unrolling (YTILE=1/UNRL=4) to keep SIMD busy, while larger sYT uses bigger tiles with moderate unrolling to improve per-wavefront efficiency

## Key Lessons

- Compute sYT = ceil(M / (CuCount * constant)) as a workload-to-hardware ratio metric to drive adaptive tile/unroll selection — this ensures tile parameters scale with actual work per compute unit rather than being fixed by output dimension N alone
- For small workloads (sYT <= 1), prefer minimal tile size with heavy unrolling (YTILE=1/UNRL=4) to maximize SIMD occupancy by reducing outer loop overhead and increasing inner-loop instruction-level parallelism
- Include LDS capacity as a gating condition (fit_lds) in tile selection heuristics to prevent selecting tile sizes that exceed shared memory limits, falling back to conservative tile sizes when the LDS budget is tight
