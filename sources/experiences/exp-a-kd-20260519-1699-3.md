## Challenge

The absmax reduction in the before code uses a serial shared-memory tree with multiple __syncthreads barriers (one per tree level, log2(blockDim) levels for 1024 threads), and a global atomicMax across all blocks, creating synchronization overhead that grows with block size and grid size


## Solution

Replace the shared-memory tree reduction with warp shuffle reduction (__shfl_xor_sync) within each warp followed by a single shared-memory write per warp and a final warp-level reduction, eliminating all __syncthreads barriers inside the reduction and removing global atomics by using per-token block-local computation

## Key Lessons

- Warp shuffle reduction (__shfl_xor_sync / __shfl_down_sync) eliminates shared-memory reads/writes and __syncthreads barriers for intra-warp reductions — replaces log2(WARP_SIZE) shared-memory accesses and one __syncthreads with pure register operations
- Block reduction via warp shuffle + one shared-memory write per warp + single __syncthreads + final warp shuffle reduces total barriers from O(log2(blockDim)) to exactly 1, and cuts shared memory usage from O(blockDim) to O(blockDim/WARP_SIZE)
