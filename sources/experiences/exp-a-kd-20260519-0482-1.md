## Challenge

Embedding lookup kernel uses 8 separate scalar half-element load/store instructions per thread (8 loads + 8 stores = 16 memory instructions), causing high instruction dispatch overhead and preventing the hardware from issuing wide coalesced memory transactions


## Solution

Replace 8 individual scalar half load/store pairs with a single 128-bit packed load/store via reinterpret_cast<float4*> (LDST128BITS macro), consolidating 16 memory instructions into 2 wide memory transactions

## Key Lessons

- Use reinterpret_cast<float4*> to pack 8 half values (16 bytes) into a single 128-bit load/store, replacing 8 scalar half memory instructions with one wide transaction — reduces instruction dispatch overhead and guarantees 128-byte memory transaction width
- The LDST128BITS(value) macro pattern (reinterpret_cast<float4*>(&(value))[0]) provides a portable, type-punning-free way to issue 128-bit memory operations on half arrays without explicit pointer arithmetic
