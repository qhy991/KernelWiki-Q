## Challenge

Optimize top-k sampling kernel for vocab_size=256, k=6 on A800 GPU


## Solution

Vectorized float4 loads with __ldg() read-only cache


### Key Changes

- Replaced scalar float loads with float4 vectorized loads (2 float4 per thread = 8 floats)
- Used __ldg() intrinsic for L1 read-only cache optimization
- Added #pragma unroll to inner loops
- Stored topk_val/topk_idx in all lanes (not just lane 0) to avoid extra broadcast


## Key Lessons

- Vectorized float4 loads with __ldg() can dramatically improve memory throughput for small kernels
- For vocab_size=256 with 32 threads/warp, each thread handles exactly 8 floats = 2 float4 loads - perfect alignment
- __ldg() enables L1 read-only cache which is critical for repeated access patterns
- The performance_ratio metric in KernelEvalPlus is percentage-based (515.4 = 515.4% = 5.154x actual speedup)
- curand_init must be avoided - use fast inline LCG RNG instead
- One warp per batch item is optimal for vocab_size=256
