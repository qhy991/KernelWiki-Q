## Solution

Halve block metadata allocation from tb_m*4*2 to tb_m*4 by restructuring how sorted_ids, rd_sorted_ids, and topk_weights share the same shared memory region in the kernel layout

## Key Lessons

- Audit shared memory accounting for buffers that can share or reorganize layout — sorted_ids and topk_weights in MoE kernels can overlap temporally, halving the metadata footprint without correctness impact
- Reducing per-block shared memory directly enables higher SM occupancy when the kernel is shared-memory-bound, yielding throughput gains even when compute logic is unchanged
