## Challenge

Within each head segment the scalar kernel copies one half element at a time, issuing separate 16-bit load and store instructions that underutilize available memory bandwidth when head_size is a multiple of the vector width


## Solution

Use vectorize_with_alignment<VEC_SIZE=8> to perform 128-bit wide (8×half) memory transactions within each head segment, automatically handling alignment via prefix/vectorized-body/tail decomposition so each warp thread copies 8 elements per iteration instead of one

## Key Lessons

- Use vectorize_with_alignment<8> (128-bit / 8-element) for fp16 head-size segments that are multiples of 8 — each thread moves 8 elements per iteration, amortizing instruction overhead and saturating memory bandwidth
- The alignment-safe vectorization pattern (prefix scalar + vectorized body + tail scalar) handles unaligned base pointers gracefully without requiring the caller to guarantee alignment
