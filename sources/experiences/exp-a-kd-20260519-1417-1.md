## Challenge

Naive 8-bit dequantization extracts each byte from a 32-bit word using separate shift, AND-mask, and OR operations (4 shifts + 4 masks + 4 ORs for 4 bytes), then manually ORs the bias bits (0x6400) and packs the resulting half values with additional shifts — a multi-instruction sequence that should be reducible to a single byte-permute instruction


## Solution

Replace the multi-instruction shift-mask-or byte extraction and manual fp16 bias packing with prmt.b32 PTX instruction, which rearranges bytes from the 32-bit quantized word into the fp16 register layout (including bias bits 0x6400) in a single instruction per value pair

## Key Lessons

- prmt.b32 can replace multi-instruction shift-mask-or byte extraction with a single PTX instruction that interleaves bytes from two 32-bit sources into a destination — applicable whenever bytes need rearrangement into fp16 or other packed layouts
- NVCC at -O3 on sm_80 already recognizes naive shift-mask-or byte extraction patterns and emits PRMT or equivalent instructions internally, so explicit PRMT serves as a compiler-behavior correctness guarantee rather than a guaranteed throughput improvement — benchmark before assuming speedup
