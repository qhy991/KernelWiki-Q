## Solution

Fuse all three operations (shuffle, multiply, sum-reduce) into a single kernel where each thread accumulates factor*val in a register-local sum_val across the topk loop, reads input directly by permutation index, and writes output once — eliminating two intermediate buffers (~448MB), two kernel launch overheads, and serializing synchronization barriers between stages

## Key Lessons

- Fusing a gather-shuffle → elementwise-mul → reduction pipeline into a single kernel eliminates intermediate buffer allocation and global memory round-trips — when the intermediate results are consumed only once by the next stage and fit in registers, fusion removes O(N) global memory writes+reads per intermediate stage
- Register-local accumulation (sum_val += factor * val) replaces global-memory intermediates when the reduction dimension (topk) is small enough to keep partial sums in registers — applies whenever reduction dimension is O(1) to O(10)
- Multi-kernel pipelines on large tensors suffer compounding memory bandwidth costs: each intermediate buffer doubles the traffic for that stage (one write + one read), so a 3-kernel chain can incur 4–6× the memory traffic of a fused version
