## Solution

Fuse SiLU+mul and per-block quantization into a single kernel where each thread computes SiLU(gate)*up in registers, participates in a shared-memory absmax reduction to find the per-group scale, then quantizes using that scale — eliminating the FP16 intermediate global memory write/read entirely and removing one kernel launch

## Key Lessons

- Fuse adjacent kernels where one writes an intermediate tensor that the next immediately reads — keeping the activation in registers across the computation+quantization boundary eliminates a full global memory round-trip, which dominates cost for memory-bound operations
- When fusion requires a cross-thread reduction (e.g., per-group absmax), use shared memory for the reduction and scale broadcast — the shared-memory latency is negligible compared to the eliminated global memory traffic
- Thread-block granularity should align with the reduction group size (group_size=128 threads per block) so that the fused kernel maps naturally to the quantization group structure without wasted threads or cross-block coordination
