## Challenge

MoE fused gate kernel passes VPT, THREADS_PER_ROW, and ROWS_PER_WARP as runtime int fields in KernelParamsDynamic, preventing NVCC from fully unrolling #pragma unroll loops that iterate over params.VPT (16 iterations for sigmoid/bias-add/argmax) and params.THREADS_PER_ROW/2 (4-iteration shuffle reduction), incurring loop overhead and blocking instruction-level parallelism across independent VPT elements


## Solution

Convert KernelParamsDynamic (runtime int fields) into a templated KernelParams struct with static constexpr fields, and hoist VPT, NUM_EXPERTS, THREADS_PER_ROW, ROWS_PER_WARP, ROWS_PER_CTA into template parameters on the kernel itself, making all loop bounds compile-time constants that enable NVCC to fully unroll every #pragma unroll loop into straight-line code

## Key Lessons

- Promote kernel configuration constants (VPT, THREADS_PER_ROW, ROWS_PER_WARP, etc.) from runtime struct fields to compile-time template parameters with static constexpr fields — this enables NVCC to fully unroll #pragma unroll loops, eliminating loop overhead and improving ILP scheduling, yielding significant speedup when loop trip counts are moderate (8–32 iterations)
- When all iteration bounds in a CUDA kernel are compile-time constants, the compiler can convert loops into straight-line instruction sequences, allowing it to interleave independent memory loads and arithmetic across loop iterations to better hide latency
- This constexpr-parameter-specialization pattern trades kernel binary size (one specialization per expert/group configuration) for throughput, and is most effective when the kernel is deployed with a small, known set of configurations at production time
