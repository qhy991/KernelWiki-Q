## Solution

ow-loop bound difference (32 vs 16 with early exit at row>=8) does not change the arithmetic intensity. The per-element compute cost dominates over the minor difference in how many loop iterations are skipped by the row

## Key Lessons

- In production Cutlass GEMM, the smaller M16 TileShape reduces shared memory allocation per tile and avoids prefetching rows 9-32 that are never used, which would yield measurable savings for small-batch decode.
