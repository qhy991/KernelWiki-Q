## Solution

Merge the two separate loop sections into a single unified loop over largest_part with an inner conditional (gather_from_rank == NGPUS-1 || idx < part) that guards memory writes, trading a small per-iteration branch overhead (~7% regression) for simplified control flow and improved correctness guarantees in multi-GPU visibility

## Key Lessons

- Unifying separate loop sections into a single loop with an inner conditional simplifies code structure and improves correctness guarantees (e.g., thread ID matching for cross-device visibility), but the added branch inside an unrolled inner loop incurs instruction overhead that scales with iteration count — evaluate whether the clarity gain justifies the performance cost
- In a #pragma unroll inner loop, every conditional evaluation is replicated per iteration with no opportunity to hoist the branch out — prefer keeping conditionals outside unrolled loops or restructuring the loop bounds to avoid per-element branching when the branch is predictable and nearly-always-taken
- For uneven partition handling in collective operations, a separate small overflow block (if tid < remaining) is essentially free because it affects only a handful of threads for a single element, whereas embedding the overflow logic into the main loop body penalizes every thread on every iteration
