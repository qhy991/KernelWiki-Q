## Challenge

When FP8 output is needed for merge_attn_states, the merge kernel writes BF16/FP16 results to a temporary global-memory buffer, then a separate quantization kernel reads that buffer and writes FP8 output, incurring an extra kernel launch and a full global-memory round-trip for the intermediate buffer


## Solution

Fuse FP8 quantization inline by extending the kernel template with output_t and USE_FP8_OUTPUT parameters, performing scaled_fp8_conversion inside the merge kernel and narrowing output_pack_t (uint for FP32→FP8, uint2 for BF16/FP16→FP8) so vector stores match the narrower FP8 element width, eliminating the separate quantization kernel launch

## Key Lessons

- Fusing quantization into the producer kernel (via template parameters and inline scaled conversion) eliminates a separate quantization kernel launch and a full global-memory round-trip — applies whenever the output format differs from the compute format and the conversion is elementwise
- Use std::conditional_t to select output pack types at compile time so vector store width matches the narrower quantized element size, avoiding bank conflicts and uncoalesced stores
- Separate input_pack_t from output_pack_t when input and output have different element widths — the input side keeps wide uint4 loads while the output side narrows to match FP8
