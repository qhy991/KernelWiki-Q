## Key Lessons

- The before code uses 8 explicit asm volatile PTX statements (4 sub.f16x2 + 4 mul.rn.f16x2) operating on individual uint4 fields, each creating a RAW dependency chain where sub feeds directly into mul per field, and the asm volatile barrier prevents cross-field scheduling.
- With full unrolling, nvcc generates essentially the same instruction mix (4 half2 sub + 4 half2 mul) in both cases; the __hsub2/__hmul2 intrinsics lower to sub.f16x2/mul.rn.f16x2 PTX identical to the before variant's explicit asm.
