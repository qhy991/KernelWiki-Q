## Challenge

The before kernel uses cuda::memcpy_async pipeline to copy weight and input tiles into shared memory before computing, but when feat_in==feat_out=256 the tile covers the entire dimension so the pipeline never overlaps load with compute, making all pipeline infrastructure pure overhead


## Solution

Replace shared-memory pipeline loads with direct global-memory vector loads into registers via float4 reinterpret_cast, bypassing shared memory entirely and eliminating all pipeline acquire/commit/wait/release overhead

## Key Lessons

- When the working set fits in registers (one tile covers the full dimension), skip shared-memory pipeline staging and load directly into registers via vectorized reinterpret_cast — pipeline overhead is only justified when load and compute actually overlap
- Prefer float4 (128-bit) vector loads for bf16 data to maximize global-memory throughput per load instruction, loading 8 bf16 elements per transaction
