## Challenge

An intermediate cudaMemcpy(DeviceToHost) between two GPU kernels forces a full stream drain: the host blocks until compute_kernel finishes, copies 4 bytes D2H, then re-awakens to enqueue the scale kernel, adding ~2.4ms of sync overhead that dwarfs the actual ~0.008ms kernel execution time


## Solution

Keep the scalar on device and pass a device pointer (const float* d_scale_ptr) to the scale kernel, which dereferences it inline via *d_scale_ptr, allowing both kernels to be enqueued back-to-back with zero host blocking and no D2H memcpy

## Key Lessons

- Eliminate cudaMemcpy D2H between kernel pipeline stages by keeping scalars on device and passing device pointers — the sync overhead is a fixed cost (~2.4ms) that dominates small-to-medium kernel execution time regardless of workload size
- Kernel dereference of a device scalar (*d_scale_ptr) is nearly free (single global load cached in L1/L2) compared to the full stream-drain + PCIe round-trip of cudaMemcpy D2H — applies whenever a per-tensor scale or bias needs to flow between kernels without host inspection
- Enqueue dependent kernels back-to-back on the same stream to let the CUDA driver schedule them without host involvement — the stream dependency guarantees the producer kernel completes before the consumer reads its output, eliminating the need for explicit host-mediated synchronization
