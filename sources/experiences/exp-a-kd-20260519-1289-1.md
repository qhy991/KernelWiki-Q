## Solution

Pre-compute the reciprocal scale_inv = 1.0f / scale once at kernel entry, then replace all per-element FP32 divisions with FP32 multiplications (val * scale_inv), reducing the expensive division from N invocations to a single one

## Key Lessons

- Replace per-element FP32 division by a constant with a single reciprocal pre-computation (1.0f / scale) followed by multiplication → FP32 division has ~4x lower throughput and ~4x higher latency than FP32 multiply on Ampere → applies whenever a constant divisor is reused across many elements in a kernel
- The benefit of reciprocal pre-computation grows with element count per thread because the single division cost is amortized over more multiplications — most impactful in compute-bound kernels processing millions of elements
