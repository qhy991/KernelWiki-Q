## Challenge

Three separate CUDA kernels (shuffle, multiply, sum-reduce) execute sequentially for MoE weighted gather-and-reduce, routing data through two intermediate global memory buffers (~128MB total), incurring 3 kernel launches, 3 global memory read/write round-trips, and idle gaps between kernel boundaries


## Solution

Fuse all three operations into a single kernel where each thread block handles one output row: gather source rows via permutation index, multiply by weight factors, and accumulate the weighted sum in registers — eliminating both intermediate buffers and reducing global memory traffic from 3 passes to 1 read + 1 write

## Key Lessons

- Fuse sequential kernels that pass data through intermediate global memory buffers into a single kernel with register accumulation — eliminates intermediate writes/reads and kernel launch overhead, yielding multiplicative speedups proportional to the eliminated memory traffic
- When a pipeline of kernels forms a producer-consumer chain (each kernel's output feeds only the next kernel), fusion converts global memory round-trips into register operations that are orders of magnitude faster
- Kernel fusion is especially beneficial for medium-sized workloads where per-kernel overhead (launch latency, synchronization gaps between kernels) constitutes a significant fraction of total execution time
