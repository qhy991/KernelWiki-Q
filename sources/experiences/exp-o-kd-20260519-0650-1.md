## Solution

Replace Blelloch scan with a two-level warp-based exclusive prefix sum: each warp computes an intra-warp exclusive scan via __shfl_up_sync (hardware-synchronized, no __syncthreads needed), warp 0 accumulates cross-warp offsets with another __shfl_up_sync, reducing total barriers from ~2*log2(scan_size) to ~6 fixed __syncthreads() independent of scan_size

## Key Lessons

- Replace Blelloch (work-efficient) parallel prefix scan with warp-level __shfl_up_sync exclusive scan when data fits within a few warps — warp shuffle is hardware-synchronized and eliminates O(log n) __syncthreads() barriers, reducing latency especially for small scan sizes where barrier overhead dominates computation
- Two-level warp scan (intra-warp via __shfl_up_sync + inter-warp via warp 0 scanning warp_sums) scales prefix sum to arbitrary sizes while keeping __syncthreads() count fixed at ~6 regardless of scan_size, versus ~2*log2(n) for Blelloch
