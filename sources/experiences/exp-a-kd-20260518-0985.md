## Challenge

In the before code, the scalar normalization loop runs with 1024 threads, each performing one half-precision load from input and weight and one store to output per stride iteration. These scalar accesses are coalesced at the warp level


## Solution

The after code uses vec_n_t<half,8> packed loads/stores with only 512 threads (hidden_size/VEC_SIZE).
