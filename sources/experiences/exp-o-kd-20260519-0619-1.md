## Challenge

Renormalization of top-K MoE gating weights (dividing each weight by their sum so they sum to 1) required a separate kernel launch after moeTopK completed, forcing top-K weights through an extra global memory write-read-rewrite round-trip and adding full kernel launch overhead


## Solution

Fuse renormalization directly into the moeTopK kernel by accumulating the sum of top-K weights during the existing selection loop (row_sum_for_renormalize), then having thread 0 apply reciprocal multiplication in-place after the loop, eliminating the separate renormalize kernel and its global memory round-trip

## Key Lessons

- Fuse lightweight post-processing (normalization, scaling) into the producer kernel when the computation is a simple scalar accumulation during the main loop followed by a small per-thread fixup — eliminates an entire kernel launch and one global memory round-trip for negligible register cost
- Precompute the reciprocal (1.f / sum) and multiply rather than dividing inside the loop — one division replaces k divisions, and multiplication by reciprocal is faster on GPU
