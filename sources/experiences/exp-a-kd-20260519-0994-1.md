## Solution

Remove the alignas(64) constraint by replacing AlignedStorage with LocalStorage (a plain struct), since NVCC's optimizer already decomposes the fixed-index unrolled loop into per-element register assignments when data flows from inline asm loads directly into inline asm stores with no intermediate arithmetic

## Key Lessons

- Removing unnecessary alignas on local Storage structs has no performance impact when the NVCC optimizer already keeps the data in registers via loop unrolling with fixed indices — alignas only matters if the compiler must spill the struct to local memory as a contiguous block
- When data flows from inline asm loads directly into inline asm stores with no intervening arithmetic, the compiler decomposes the struct into individual register assignments regardless of alignment attributes
- alignas on local variables is only beneficial when it prevents suboptimal local-memory layout or enables vectorized load/store instructions that require alignment — otherwise it is dead annotation
