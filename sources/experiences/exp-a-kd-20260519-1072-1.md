## Solution

Replace scalar stride-256 loads with vectorized 16-byte packed loads via AlignedArray, using VPT=4 and THREADS_PER_ROW=16 so each thread loads 4 contiguous floats in one coalesced transaction with interleaved addressing (vec_thread_read_ptr[ii * THREADS_PER_ROW]) to achieve full 128-byte memory coalescing across the row-subgroup

## Key Lessons

- Use vectorized loads (AlignedArray / float4) with interleaved thread addressing instead of scalar strided loads when threads per row < warp size — contiguous vectorized transactions achieve full coalescing while strided scalar loads waste bandwidth on partial transactions
- The interleaved access pattern (thread i reads at offset i * THREADS_PER_ROW) maps adjacent threads to contiguous memory blocks, ensuring all loads in a warp-subgroup merge into minimal cache-line transactions
