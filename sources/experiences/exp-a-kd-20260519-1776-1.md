## Challenge

Standard GEMM C=A*B for small M=16 with N=1280 may not select the optimal cuBLAS kernel configuration, motivating a swap-AB transformation to D=B^T*A^T to try to trigger a better internal code path


## Solution

Swap AB operands to compute D=B^T*A^T then C=D^T, which is mathematically equivalent but attempts to leverage N-aware kernel selection in cuBLAS/CUTLASS backends for small-M regimes

## Key Lessons

- Swap-AB (computing B^T*A^T instead of A*B) does not improve performance on SM80 cuBLAS because PyTorch's backend already optimizes both operand orderings — the transformation only yields benefit when combined with SM90 CUTLASS 3.x features (TMA, FP8FastAccum, wider K tiles)
- High-level framework calls like torch::matmul delegate operand reordering to the backend library, so manual swap-AB at the PyTorch level is redundant unless the backend is CUTLASS 3.x with SM90-specific scheduling
