## Solution

Replace device_vec with aligned_vector using alignas(16) on an aligned_storage<uint32_t,4> member to guarantee 16-byte alignment, combined with warp_ns::load/store wrapper functions and operator[] access — intended to enable LD.128 instructions via explicit alignment annotation

## Key Lessons

- Adding alignas to vector types does not guarantee wider load instructions when nvcc -O3 already infers 16-byte alignment from cudaMalloc-returned pointers and contiguous stride offsets — the compiler's alias analysis is sufficient for alignment inference in many common allocation patterns
- Refactoring .data[i] access to operator[] is purely syntactic at -O3 optimization level since both resolve to identical memory addresses and the compiler generates the same PTX — do not expect performance changes from access syntax alone
- Wrapping pointer-cast load/store in template functions (warp::load/store) improves readability but has zero performance impact at high optimization levels since the inliner eliminates the abstraction entirely
