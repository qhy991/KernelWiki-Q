## Solution

Parameterize blockReduceSum with a compile-time maxBlockSize template and use if constexpr to skip the shared-memory two-phase path entirely when the block fits in one warp, replacing it with a single parameterized warpReduceSum<T, numLanes> that shuffles only the active lanes (4 rounds for 16 threads instead of 5 rounds for 32)

## Key Lessons

- When block size is known at compile time and fits within a single warp (≤32 threads), use if constexpr to bypass shared memory, __syncthreads, and the two-phase reduction entirely — replacing it with a single parameterized warpReduceSum reduces shuffle rounds and eliminates synchronization overhead
- Parameterize warpReduceSum with a numLanes template to correctly reduce sub-warp thread groups; a full 32-lane shuffle for 16 active threads reads garbage from inactive lanes and wastes one extra shuffle round
- Removing __syncthreads is particularly impactful for small blocks because it forces a full warp-level pipeline drain; compile-time elimination via if constexpr avoids this overhead without runtime cost
