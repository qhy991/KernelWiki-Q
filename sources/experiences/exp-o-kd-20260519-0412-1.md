## Solution

Reorganize the thread-to-data mapping so that blockIdx.x maps to token_pos and threadIdx.x maps to the dimension pair index within that token, causing all threads in a warp to read from the same contiguous row (x[token_pos*N*2 + tid*2]) and producing fully coalesced 32-byte memory access

## Key Lessons

- Map blockIdx.x to the outer dimension (token position) and threadIdx.x to the inner dimension (feature pair index) so that consecutive threads access contiguous memory within the same row — this produces fully coalesced accesses and avoids strided cross-row transactions
- A flat linear index that interleaves tokens in thread order breaks memory coalescing when threads in the same warp compute different token positions that map to non-adjacent memory regions
