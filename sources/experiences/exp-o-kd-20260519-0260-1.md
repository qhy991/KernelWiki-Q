## Solution

Reduce kStages from 4 to 1 for small head dimensions (K<=64) by passing the actual head size to the pipeline stage computation kStages=min(4, ceil(headSize/tileK)), cutting shared memory from 33280 to 8320 bytes/block and enabling 5 blocks/SM instead of 1

## Key Lessons

- Compute pipeline stage count as kStages=min(maxStages, ceil(headDim/tileK)) rather than using a fixed constant — when headDim equals or is smaller than tileK, a single stage suffices and eliminates unnecessary shared memory allocation and synchronization barriers
- Shared memory footprint directly limits occupancy: 33280 bytes/block allows only 1 block/SM under 48KB, while 8320 bytes/block allows 5 blocks/SM — a 5x occupancy increase that hides memory latency without any algorithmic change
- Reducing pipeline stages from N to 1 eliminates (N-1) __syncthreads() barriers and (N-1) redundant compute passes over the same data, compounding the occupancy gain with reduced barrier stall time
