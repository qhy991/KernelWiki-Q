## Solution

Replace atomicAdd + __syncthreads + global read with CUB BlockReduce, which cooperatively reduces per-thread counts in shared memory using an optimized hierarchical reduction algorithm and delivers the final sum to thread 0, eliminating global atomic contention and the synchronization barrier

## Key Lessons

- Prefer CUB BlockReduce over atomicAdd + __syncthreads for per-block sum reductions — shared-memory cooperative reduction eliminates atomic serialization on a single global address and avoids the subsequent global memory read latency
- BlockReduce delivers the final result only to thread 0 (or a designated lane), which naturally preserves the existing thread-0-only downstream logic without restructuring the control flow
