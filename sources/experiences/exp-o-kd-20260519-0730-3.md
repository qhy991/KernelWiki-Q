## Challenge

expert_ids array is filled by assigning each thread to one expert and looping over its block range, causing load imbalance when experts have very different token counts — some threads finish quickly while others do heavy work


## Solution

Replace per-expert assignment with cooperative binary search: all threads stride over the output block indices and binary-search the prefix array to find which expert owns each block, distributing work evenly regardless of expert skew

## Key Lessons

- Replace per-owner work distribution with strided cooperative access when work per owner is skewed — binary-search inversion (search prefix array for owning expert) distributes load evenly across all threads
- When an inverse mapping is needed (fill output from a prefix sum), binary search on the prefix array is a general technique that avoids load imbalance from uneven bucket sizes
