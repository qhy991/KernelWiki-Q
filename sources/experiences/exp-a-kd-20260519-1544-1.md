## Solution

Introduce a K > 3*N heuristic to detect tall-and-skinny shapes and switch from persistent tile scheduling (grid_size=total_tiles) to StreamK scheduling (grid_size=num_sms), which distributes all M*N output elements evenly across every SM for full occupancy

## Key Lessons

- Use the K > 3*N heuristic to detect tall-and-skinny GEMM shapes where persistent tile scheduling produces fewer blocks than available SMs, and switch to StreamK (grid_size=num_sms) to achieve full occupancy — applies whenever the tile count is small relative to SM count
- StreamK distributes output elements (M*N) evenly across all SMs rather than assigning one block per output tile, reducing per-block work from total_elements/total_tiles to total_elements/num_sms — critical when total_tiles << num_sms
- Scheduler dispatch heuristics based on problem geometry (K vs N ratio) can yield order-of-magnitude speedups on shape extremes even when the inner compute loop is unchanged, because GPU utilization dominates arithmetic throughput for these configurations
