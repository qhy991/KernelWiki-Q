## Challenge

An epilogue kernel processes one fragment per loop iteration, requiring two __syncthreads() barriers per fragment (one before write to SMEM, one before read from SMEM), totaling 2*kIterations barriers and incurring significant synchronization overhead


## Solution

Coarsen the loop stride by FPP=2 so each iteration stages multiple fragments into partitioned shared memory before the barrier pair, halving __syncthreads() count from 2*kIterations to 2*(kIterations/FPP) while using 2x SMEM capacity

## Key Lessons

- Loop coarsening that batches multiple fragments inside a single barrier pair reduces __syncthreads() overhead proportionally to the coarsening factor — applies whenever shared memory staging processes independent data tiles in a loop
- Trading shared memory capacity for fewer barriers is worthwhile when the additional SMEM stays within per-block limits and barrier overhead dominates kernel time — profile to confirm the tradeoff is beneficial
