## Solution

Replace all static __shared__ declarations with a single extern __shared__ char buffer and manually place int32_t arrays contiguously first, then int16_t arrays at a computed 2-byte boundary, using a centralized SmemLayout struct for compile-time layout verification and enabling cudaFuncSetAttribute to exceed the 48KB static shared memory cap

## Key Lessons

- Group same-sized types contiguously (int32_t arrays first, then int16_t arrays) when laying out a single extern __shared__ buffer — this eliminates compiler-inserted alignment padding between differently-sized static __shared__ declarations and reduces total shared memory consumption
- Use extern __shared__ with cudaFuncSetAttribute to exceed the 48KB static shared memory cap — necessary for kernels whose shared memory requirements scale with configuration parameters
- Centralize shared memory layout computation in a compile-time struct (e.g., SmemLayout) so offsets and total size are verified at compile time and the layout is reusable across kernel launches
