## Solution

Fuse both kernels into a single advance_step_flashinfer_kernel where each thread reads seq_len and block_tables once, keeps them in registers, and derives all six outputs (tokens, seq_lens, positions, slot_mapping, paged_kv_last_page_len, block_table_bound) without any redundant global memory loads, eliminating one full kernel launch and its associated driver overhead

## Key Lessons

- Fuse kernels that operate on the same per-element data (same query index, same input arrays) into a single kernel to eliminate redundant global memory loads and save one kernel launch's driver overhead (~5-10 us per launch) — applies whenever two kernels share the same thread-to-element mapping and read overlapping input arrays
- When kernel 2 re-reads values that kernel 1 already computed or loaded (e.g., seq_lens, block_tables), fusion lets the second set of computations reuse register-held values instead of issuing fresh global loads, yielding both latency and bandwidth savings
