## Solution

Replace the naive reinterpret_cast with a generic vectorize_with_alignment template that computes pointer misalignment at runtime, processes a scalar prefix to reach alignment boundary, runs the vectorized main body, then handles remaining tail elements — ensuring correctness for any pointer alignment

## Key Lessons

- Alignment-aware vectorization (scalar prefix + vectorized body + scalar tail) adds runtime overhead (misalignment arithmetic, extra branches, potential scalar loops) that can negate its benefit when the pointer is already naturally aligned — as is always the case with cudaMalloc buffers
- cudaMalloc guarantees at least 256-byte alignment, so reinterpret_cast to vec_n_t<float,4>* is always safe for cudaMalloc'd pointers; the alignment-prefix pattern is only valuable when processing sub-buffers or externally-provided pointers of unknown alignment
- With only 4 vector elements per thread (hidden_size=4096, 256 threads), the fixed overhead of alignment calculation dominates — the pattern pays off only when the vectorized body has enough iterations to amortize the prefix/tail overhead
