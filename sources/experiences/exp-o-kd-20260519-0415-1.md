## Solution

Replace scalar f32 load/store with vectorized float4 packed load/store using the FLOAT4 macro (reinterpret_cast<float4*>), so each thread loads 4 floats in one 128-bit transaction and processes two adjacent pairs (x_v.x/x_v.y as first pair, x_v.z/x_v.w as second pair) with separate trigonometric coefficients per dimension index, halving thread count and reducing global memory transactions from 4 to 2 per thread

## Key Lessons

- Vectorized float4 load/store replaces 4 individual 32-bit global memory transactions with a single 128-bit transaction per thread, improving bandwidth utilization — applies to any kernel processing contiguous f32 data where elements can be grouped in multiples of 4
- When vectorizing RoPE to process multiple pairs per thread, each pair's dimension index must be computed independently (token_idx*2 and token_idx*2+1 for the two adjacent pairs) because adjacent pairs correspond to different rotary frequency bands
