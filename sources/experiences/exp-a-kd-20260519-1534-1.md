## Solution

Fuse RoPE computation and KV-cache write into a single kernel (concat_and_cache_mla_rope_fused_kernel) where each thread computes RoPE on its k_pe element and immediately writes the result to the paged KV cache, eliminating the intermediate global-memory round-trip and reducing two kernel launches to one

## Key Lessons

- Fuse producer-consumer kernel pairs where the consumer immediately re-reads the producer's output — eliminates one full global-memory round-trip and one kernel launch, especially beneficial when intermediate data (k_pe: 256 tokens × 128 half = 64KB) would otherwise be written to global memory then immediately re-read
- A single fused kernel with max(rope_threads, cache_threads) threads amortizes launch overhead over more total work per block, improving GPU utilization when individual kernels would launch with underutilized thread counts (embed_dim=64 vs kv_lora_rank=1024)
