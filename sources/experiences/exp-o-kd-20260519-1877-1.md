## Challenge

The SwiGLU activation kernel uses scalar __ldg loads with stride-2 indexing to access interleaved gate/up pairs, issuing two separate 4-byte load instructions per element plus one 4-byte store, which prevents the compiler from merging adjacent loads into wider memory transactions on this memory-bound kernel


## Solution

Replace scalar __ldg loads with int4 (128-bit) vectorized loads that fetch adjacent gate/up pairs in a single transaction, paired with int2 (64-bit) vectorized output stores, reducing input memory transactions by ~4x and output by ~2x with an alignment-checked fast path and scalar fallback for correctness

## Key Lessons

- Use int4/int2 vectorized loads and stores for interleaved data layouts where gate/up pairs are adjacent in memory — a single 128-bit load replaces two scalar __ldg loads, yielding significant speedup on memory-bound kernels
- Always include an alignment check before casting pointers to wider types (int4 requires 16-byte alignment, int2 requires 8-byte), with a scalar fallback path to maintain correctness for unaligned inputs
- For gated activation kernels with a 2:1 input-to-output ratio (e.g., SwiGLU), vectorized input loads paired with proportionally sized output stores (int4 in, int2 out) naturally match the data flow pattern
