## Solution

Implement a double-buffered circular pipeline using PipelineState for automatic pipe index and phase tracking, ClusterTransactionBarrier with arrive_and_expect_tx for TMA producer synchronization, ClusterBarrier for MMA consumer coordination, and warpgroup_arrive/commit/wait primitives that issue WGMMA reading directly from SMEM descriptors without explicit register copies

## Key Lessons

- Use PipelineState<K_PIPE_MAX> for circular pipe indexing with automatic phase tracking — the phase flips each full cycle through K_PIPE_MAX pipes, enabling correct barrier wait semantics without manual phase bookkeeping
- On SM90, WGMMA reads directly from SMEM via descriptors rather than registers — invoke warpgroup_arrive() to issue, warpgroup_commit_batch() to submit to hardware, and warpgroup_wait<0>() to synchronize completion, eliminating explicit SMEM-to-register copy in the mainloop
- Use canonical_warp_idx_sync() and elect_one_sync() to designate a single warp/lane for barrier and TMA operations — only one thread per cluster should invoke TMA copies and barrier arrive to avoid redundant memory transactions
