## Challenge

Logical OR reduction on packed 1-bit booleans (N=32) extracts each bit individually via shift-and-mask in a 32-iteration loop with per-element functor calls, wasting instructions on repeated bit extraction for bits that share the same storage byte


## Solution

Replace 32 per-element bit extractions with 4 byte-level zero-checks: since a byte is non-zero if and only if any of its 8 packed bits is true, scanning whole bytes via `ptr[byte] != 0` eliminates shift-and-mask overhead and functor call dispatch, reducing iterations by 8x

## Key Lessons

- For logical reductions (OR/AND) on packed sub-byte types, aggregate to the storage granularity (byte) instead of extracting individual elements — a non-zero byte implies at least one true bit, collapsing 8 comparisons into one
- Use raw byte pointer reinterpret_cast on packed boolean storage to bypass per-element accessor overhead (shift + mask) when the operation can be expressed at the storage level
- Initialize reduction accumulators to the identity element (false for OR, true for AND) rather than seeding with the first two elements — this avoids special-casing and prevents bugs like the original CUTLASS code that initialized to true, making the result always true
