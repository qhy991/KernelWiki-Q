## Challenge

The inner copy loop in the cache gather kernel uses runtime-variable bounds (entry_size and blockDim.x), preventing the compiler from unrolling iterations and forcing it to generate loop-counter overhead, branch prediction, and runtime division/modulo for stride calculations on every invocation


## Solution

Convert ENTRY_SIZE and CTA_SIZE from runtime function parameters to compile-time template parameters (ENTRY_SIZE_T, CTA_SIZE_T), enabling #pragma unroll on the inner copy loop so the compiler generates fully unrolled straight-line code with no loop-counter overhead and better instruction-level parallelism

## Key Lessons

- Convert loop-bound parameters to compile-time template parameters when their values are known at kernel launch time — the NVCC compiler can then fully unroll the loop (#pragma unroll with known trip count), eliminating loop-counter increments, branch prediction, and generating tighter straight-line code with better ILP
- Template-parameterized kernel launch syntax (kernel<Const1,Const2><<<grid,block>>>(...)) has zero runtime overhead since template instantiation happens at compile time — use it freely for stride sizes, tile dimensions, and CTA sizes that are constant across all launches
- Use assert(TEMPLATE_PARAM == blockDim.x) to catch mismatched template and runtime values at debug time — the assert is compiled out in release builds, giving safety without performance cost
