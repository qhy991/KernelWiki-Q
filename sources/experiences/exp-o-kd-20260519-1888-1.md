## Solution

Replace the 64 sequential cudaMemcpyAsync dispatches with a single CUDA kernel launch using a 2D grid of (num_layers, num_pairs) thread blocks, where each block processes one layer-pair combination by reading block_mapping from GPU memory and copying key and value cache elements in parallel

## Key Lessons

- Batching many small cudaMemcpyAsync calls (each ~2.5µs API overhead) into a single CUDA kernel launch with a 2D grid eliminates per-call dispatch overhead — applies whenever multiple independent D2D copies share the same copy pattern across layers or batches
- Use pointer arrays (int64_t*) on the GPU to let a single kernel access per-layer device pointers via blockIdx indexing, avoiding the need for host-side loops over separate device allocations
- Kernel launch overhead is constant regardless of grid size, so increasing the number of thread blocks (more layers/pairs) increases GPU parallelism without increasing dispatch cost — scales efficiently with model size
