## Challenge

The before kernel performs four warp-cooperative memory operations per work item: load_vec + store_vec for K-cache and load_vec + store_vec for V-cache, totaling 4 global-memory transfers of kElementSize=512 bytes each. The after kernel with kIsMLA=true eliminates the entire V-cache pipeline at c...


## Solution

Introduce InstructionShape_ as a template parameter replacing the hardcoded MatrixShape<16,8> alias, and add static_assert constraints restricting valid shapes to 16x8 and 16x16 columns, enabling the iterator to support both m16n8k8 and m16n8k16 MMA instructions

## Results

- **speedup**: 2.009x (improved)


## Key Lessons

- Parameterize tensor-core instruction shapes as template parameters instead of hardcoding them — enables a single iterator to serve multiple MMA variants (m16n8k8, m16n8k16), letting downstream code select the widest instruction for higher arithmetic throughput
- Use static_assert to constrain template shape parameters to known-valid values — catches misconfiguration at compile time and documents the supported instruction set as part of the type contract
