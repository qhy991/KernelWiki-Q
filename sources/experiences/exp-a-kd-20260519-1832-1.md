## Solution

Replace inline PTX asm shfl with the compiler-visible __shfl_down_sync() intrinsic, collapsing the multi-statement asm+add pattern into a single expression that allows nvcc to freely optimize register allocation and instruction scheduling

## Key Lessons

- Prefer compiler-visible intrinsics (__shfl_down_sync, __shfl_xor_sync) over inline PTX asm for warp shuffle operations — nvcc already generates optimal SHF instructions from intrinsics and retains full freedom for register allocation and instruction scheduling
- Inline asm with register constraints that worked around an AMD-specific v_add_f32 row_shl limitation does not translate to a CUDA benefit; the optimization is cross-vendor portability, not CUDA performance — on NVIDIA hardware the compiler handles both patterns similarly
