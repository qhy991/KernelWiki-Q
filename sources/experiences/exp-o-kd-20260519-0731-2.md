## Challenge

Expert counting loop uses two-level warp-based addressing requiring integer division and modulo per element (warp_idx = expert_id / experts_per_warp; expert_offset = expert_id % experts_per_warp) to index into shared_counts, adding unnecessary arithmetic overhead in the hot counting path


## Solution

Replace the two-level warp-based indexing with direct shared_counts[expert_id] addressing — the expert_id already uniquely identifies the slot, so div/mod and the multiplicative offset are entirely redundant arithmetic that can be eliminated

## Key Lessons

- When shared memory is laid out as a flat array indexed by expert_id, use direct indexing instead of two-level (div/mod) addressing — the intermediate warp layout mapping is redundant when each expert maps to a unique slot
