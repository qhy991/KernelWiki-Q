## Challenge

RMS norm kernel uses scalar element-wise half loads inside a #pragma unroll inner loop, relying on NVCC to coalesce contiguous accesses into vector-width transactions rather than making the vectorized load intent explicit to the compiler


## Solution

Replace scalar half loads with a vec_n_t<__half, VEC_SIZE> struct loaded via reinterpret_cast, making the 128-bit vectorized load explicit to the compiler; simplify the variance loop to unit-stride scalar access since coalescing is straightforward there

## Key Lessons

- Explicit vec_n_t packed loads make vectorized memory access intent clear to the compiler in heavily templated code where NVCC's automatic coalescing heuristics may be less effective — even though in simple kernels the scalar and packed patterns produce identical 128-bit hardware transactions
- When NVCC can prove contiguous access in a #pragma unroll inner loop over half elements, it automatically coalesces scalar loads into 128-bit transactions — explicit vectorization yields no measurable speedup in these straightforward cases
- Simplifying the variance reduction loop to unit-stride scalar access (hidden_size iterations) over a VEC_SIZE-strided unrolled loop is preferable when the computation is reduction-bound and the compiler handles coalescing anyway — fewer address arithmetic instructions per element
