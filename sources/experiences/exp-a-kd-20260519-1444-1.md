## Solution

Generalize the kernel with a template parameter num_bits, deriving all bit-width-dependent values as constexpr (pack_factor=32/num_bits, mask=(1<<num_bits)-1, shift=cur_pos*num_bits) so a single template instantiation replaces per-width kernel variants

## Key Lessons

- Replacing hardcoded bit-width constants with template parameters (num_bits) and constexpr derivation (pack_factor=32/num_bits, mask=(1<<num_bits)-1) produces identical machine code for each instantiation while enabling a single source-of-truth kernel — applies whenever multiple quantization widths share the same packing logic
- if constexpr branches (e.g., if constexpr (num_bits == 8) for output packing) allow specialization within a generalized template without runtime cost, keeping per-width logic isolated and zero-overhead
