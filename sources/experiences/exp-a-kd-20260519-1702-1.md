## Solution

Replace compile-time fixed 32KB shared memory with dynamic shared memory sized at kernel launch time based on device capability and batch size: small batches (<=4 rows) use medium ~22KB preset, medium batches (<=8 rows) use 48KB cap, large batches use full device smem, allowing the kernel to adapt buffered_items to available resources

## Key Lessons

- Dynamic shared memory sizing via cudaFuncSetAttribute + cudaDevAttrMaxSharedMemoryPerBlockOptin allows kernels to adapt buffer capacity to device capability, though occupancy gains are only meaningful when enough CTAs are launched to saturate the GPU
- extern __shared__ with a runtime-sized second dimension ([][1]) plus a passed-in max_buffered_items parameter decouples the buffer size from compile-time constants, enabling batch-size-aware smem tuning without recompilation
