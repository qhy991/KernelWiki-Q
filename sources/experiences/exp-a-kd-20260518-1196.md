## Challenge

The before kernel reconstructs page_stride and head_stride from num_heads and head_size via inline multiplication (bo*num_heads*head_size + hi*head_size), while the after kernel receives these strides as pre-computed int64_t kernel parameters (bo*page_stride + hi*head_stride)


## Solution

. For NHD layout the runtime stride values match the hardcoded products exactly (page_stride=1024, head_stride=128), so both kernels compute identical addresses. The measured 0.32% overhead is within measurement noise for this short kernel (256 tokens, 2 loop iterations per t

## Results

- **speedup**: 0.997x (neutral)


## Key Lessons

- This is not a meaningful regression; the stride-based pattern's benefit is enabling flexible cache layouts (HND) that improve attention-side memory coalescing.
