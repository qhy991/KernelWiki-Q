## Challenge

The after kernel adds an early-exit branch for token_idx >= 128 (prefix_num_tokens): for suffix-only tokens, it performs a single 128b pack load/store to copy suffix_output directly and one scalar LSE read, bypassing the prefix LSE load, ma


## Solution

Replace synchronous FLOAT4 global-to-shared loads for B with cp.async.ca asynchronous copies issued at the start of each loop iteration, committing the async group before A loading and BK compute so B transfers proceed in flight; CP_ASYNC_WAIT_GROUP(0) replaces __syncthreads() as the primary B-completion barrier

## Results

- **speedup**: 1.067x (improved)


## Key Lessons

- Replace synchronous FLOAT4 global-to-shared loads with cp.async.ca for tiles that can be prefetched one iteration ahead — the async copy issues before compute and retires in the background, hiding global memory latency behind BK FMA iterations when the compute loop is long enough
- Decouple async B-tile loading from synchronous A-tile loading in the main loop: issue and commit B async copies first, then perform A loads and compute while B transfers are in flight, using CP_ASYNC_WAIT_GROUP(0) as the completion barrier before the next iteration reuses the shared-memory buffer
