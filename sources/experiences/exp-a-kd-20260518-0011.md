## Key Lessons

- The simulated workload uses simple global memory stores (dst_a[idx] = src_a[idx] + 1.0f) rather than actual TMA async copies.
- Since global stores are not bottlenecked by the warp synchronization pattern in the same way TMA descriptor issues are, the elimination of one __ballot_sync per iteration is invisible against the memory-bound store throughput.
