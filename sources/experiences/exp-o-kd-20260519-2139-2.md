## Solution

Replace the per-thread counter + cumsum lookup with a single atomicAdd on local_offsets[expert_id] stored in shared memory, where atomicAdd returns the pre-increment value as the rank — eliminating two global-memory reads and one write per token, reducing sorting to one shared-memory atomic per token

## Key Lessons

- Use atomicAdd on a pre-initialized offset array to compute scatter ranks in a single operation — atomicAdd returns the old value (the rank) and increments in place, replacing a load-add-store sequence
- Storing local_offsets in shared memory and initializing it from cumsum (read once) converts per-token global-memory rank lookups into shared-memory atomics, which are ~10-100x lower latency than global-memory round-trips
