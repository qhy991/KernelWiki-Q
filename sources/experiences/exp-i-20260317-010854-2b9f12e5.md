fp32_flash_attention_llama3_8b_q8_0_cache512

## Solution

Naive Flash Attention with Q8_0 KV cache

## Key Lessons

- Q8_0 format: each block has float16 scale + 32 int8 values
- For head_dim=128, need 4 blocks per head (128/32)
- Grid indexing must match grid layout: [batch_size, num_heads, 1] means blockIdx.x=batch, blockIdx.y=head
- Online softmax requires tracking max (m) and sum (l) across all tiles
- Performance bottleneck: naive implementation with scalar loads and no tiling optimization
