## Challenge

Implementing Flash Attention with Q8_0 quantized KV cache. Key challenges: 1) Understanding Q8_0 block layout (scale + 32 int8 values), 2) Correct grid indexing for [batch, heads, head_dim] query and [seq_len, heads, head_dim] cache, 3) Online softmax to avoid storing full attention matrix


## Solution

Naive Flash Attention with Q8_0 KV cache


### Key Changes

- Fixed grid indexing: batch_idx = blockIdx.x, head_idx = blockIdx.y
- Correct Q8_0 cache layout: [seq_len, num_heads, head_dim/32]
- Online softmax with m (max) and l (sum) tracking


## Key Lessons

- Q8_0 format: each block has float16 scale + 32 int8 values
- For head_dim=128, need 4 blocks per head (128/32)
- Grid indexing must match grid layout: [batch_size, num_heads, 1] means blockIdx.x=batch, blockIdx.y=head
- Online softmax requires tracking max (m) and sum (l) across all tiles
- Performance bottleneck: naive implementation with scalar loads and no tiling optimization
