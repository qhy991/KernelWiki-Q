## Challenge

Kernel skeleton lacks the SMEM-to-GMEM output store, requiring an SM90 BULK_COPY mechanism for high-throughput shared-to-global memory writes with proper TMA-style synchronization


## Solution

Construct an output GMEM tensor, instantiate Copy_Traits<SM90_BULK_COPY_AUTO> for hardware-optimized bulk store traits, invoke copy(blkcp, sA, gA_out) to perform the SMEM-to-GMEM transfer, then synchronize with tma_store_arrive() and tma_store_wait<0>() which are required even for non-TMA bulk copy stores on SM90

## Key Lessons

- SM90_BULK_COPY_AUTO Copy_Traits provides hardware-auto-tuned bulk store parameters for SMEM-to-GMEM writes on Hopper — instantiate with {} default construction and pass to copy() as the first argument
- tma_store_arrive() + tma_store_wait<0>() is the mandatory synchronization pair for SM90 bulk copy stores (not just TMA stores) — the arrive signals the store operation and wait<0> blocks until completion
