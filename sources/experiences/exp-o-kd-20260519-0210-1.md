## Solution

Add a causal_q_begin boolean flag (default true) to the Options struct so that the causal mask variant can be selected at runtime: CausalMask<true> for the qbegin case computes max_blocks_q as ceil_div((blk_coord_q+1)*tile_q, tile_k), visiting fewer K-tiles when Q is at the beginning of the attention matrix

## Key Lessons

- Selecting the correct causal mask variant (IsQBegin=true vs false) based on Q position in the attention matrix avoids visiting unnecessary K-tiles — use CausalMask<true> when Q is at the beginning (typical prefill), CausalMask<false> when Q is at the end, as the tile visit schedules differ significantly in K-tile count
- Adding a runtime flag for mask variant selection is a low-cost way to fix a semantic dispatch bug where the default causal path used the wrong mask for the common use case
