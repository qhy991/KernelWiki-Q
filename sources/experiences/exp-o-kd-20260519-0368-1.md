## Challenge

Flash attention split-KV kernel uses kStage=1 (single-stage synchronous pipeline), forcing each of the 64 KV loop iterations to stall on global memory loads for both K and V tiles before MMA computation can begin, exposing full global memory latency per iteration


## Solution

Upgrade to kStage=2 multi-stage pipeline with double-buffered K tiles using async cp.async prefetch: the next K tile is prefetched into an alternate shared memory slot (smem_sel_next) while current MMA proceeds, and V tiles are loaded asynchronously with CP_ASYNC_WAIT_GROUP(1) to overlap V load with ongoing P@V computation

## Key Lessons

- Use multi-stage async pipelines (kStage>=2) with double-buffered shared memory slots to overlap global memory loads with MMA computation in tiled attention kernels — the latency hiding effect compounds across many KV loop iterations (64 tiles at seq_len=4096 produces 16.8% speedup)
- Alternate shared memory buffer slots (smem_sel/smem_sel_next) across KV loop iterations to eliminate read/write conflicts on double-buffered K tiles without requiring explicit barriers between the async load and compute phases
- Use CP_ASYNC_WAIT_GROUP(N) to wait only for specific async copy groups rather than all outstanding copies — waiting for only the V tile while allowing the next K prefetch to remain in-flight preserves the load-compute overlap across iterations
