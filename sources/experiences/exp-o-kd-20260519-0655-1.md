## Solution

Fuse all three operations into a single kernel where each thread accumulates factor*val into a register-local sum across the topk loop, reads input directly by permutation index, and writes output only once — eliminating both intermediate buffers and reducing global memory traffic from ~3 full tensor passes to 1 input read + 1 output write

## Key Lessons

- Fuse sequential kernels that produce intermediate tensors into a single kernel — register-local accumulation eliminates massive global memory round-trips for intermediates, which is the dominant cost at large tensor sizes
- When a multi-kernel pipeline passes full tensors through global memory between stages, the intermediate buffer traffic (read+write per stage) typically dwarfs the compute cost — fusion that keeps intermediates in registers can yield 5x+ speedups
- Each eliminated kernel launch also removes one implicit device synchronization barrier, improving pipeline throughput and reducing scheduling overhead
