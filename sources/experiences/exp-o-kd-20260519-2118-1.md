## Solution

Move the write-only sorted_token_ids initialization into a separate threadblock (blockIdx.x == 1) that runs concurrently with the counting/alignment block (blockIdx.x == 0) by launching <<<2, threads>>> instead of <<<1, threads>>>, overlapping init and counting across different SMs with no synchronization needed since the counting block never reads sorted_token_ids

## Key Lessons

- When a kernel contains write-only preamble work independent of the main computation, delegate it to a separate threadblock launched with <<<2, threads>>> — both blocks execute concurrently on different SMs, turning serial init+compute latency into parallel max(init, compute)
- Separating write-only initialization from read-modify-write logic into independent threadblocks requires no inter-block synchronization when the blocks operate on non-overlapping data dependencies
