## Solution

Replace scalar float loads with vectorized float4 loads using PTX inline asm ld.global.cg.v4.u32, processing 4 elements per load instruction to reduce total load instructions by 4x and increase memory throughput; however the kernel remains dominated by atomicAdd contention on the 256-bin histogram, resulting in no net speedup

## Key Lessons

- Vectorized loads (float4) reduce load instruction count by 4x but produce no speedup when the kernel bottleneck is atomicAdd contention rather than memory latency — profile before optimizing to identify the true bottleneck
- PTX inline asm ld.global.cg.v4 with cache-global hint bypasses L1 cache for streaming data that will not be reused; effective for sequential sweeps but the benefit is hidden when atomics dominate execution time
- Atomic contention on a small histogram (256 bins shared across 1024 threads) can dominate kernel runtime regardless of load bandwidth improvements — reducing atomics (e.g., thread-local histograms with a final merge) would be a higher-impact optimization
