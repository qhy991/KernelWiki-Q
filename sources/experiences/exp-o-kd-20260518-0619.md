## Challenge

The before kernel for 64 experts uses shared memory for the count matrix, which is faster than global memory but still requires per-thread row initialization, counting, prefix-sum across 64 thread rows with two __syncthreads() barriers, and then sequential token placement


## Solution

The after kernel launches 8 blocks x 256 threads with a 3-line inner loop: read topk_ids[i], atomicAdd to claim position, write token index.
