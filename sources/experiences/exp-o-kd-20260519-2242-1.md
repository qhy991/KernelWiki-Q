## Solution

Replace hardcoded pipe_stages=4 with a runtime stages parameter set to 2 for Turing (sm75), halving all stage-dependent shared memory allocations (sh_a_size, sh_b_size, sh_s_size, sh_g_idx_size) while non-stage-dependent buffers remain unchanged, yielding ~1.78x total shared memory reduction

## Key Lessons

- Make pipeline stages a runtime parameter instead of a hardcoded constant so the shared memory footprint can be tuned per architecture — Turing (sm75) works with 2 stages while Ampere may need 4, and halving stages directly halves all stage-multiplied buffers
- Shared memory allocation in quantized GEMM is dominated by stage-dependent buffers (A tile, B tile, scales, group index) — reducing pipeline depth is the most effective lever for fitting within per-thread-block shared memory limits on occupancy-constrained hardware
