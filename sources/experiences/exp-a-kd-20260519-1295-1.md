## Solution

Pack 8 half elements into a __align__(16) half8_packed_t struct containing 4 x __half2 fields and issue a single 128-bit store via reinterpret_cast, eliminating three separate STG instructions per iteration

## Key Lessons

- On sm_80+ GPUs, the memory controller automatically coalesces adjacent 64-bit stores from the same warp into full 128-bit transactions, so explicit 128-bit packing of contiguous half2 stores yields negligible improvement in store-heavy microbenchmarks — the hardware already achieves the same effective throughput
- nvcc at -O3 can merge consecutive half2 stores to contiguous aligned addresses into a wider store instruction during compilation, further reducing the gap between separate __half2 writes and explicit 128-bit packed stores
- In real CUTLASS epilogues, alignment affects template-instantiation-level tile partitioning and shared-memory staging beyond just store width — a store-only microbenchmark cannot capture the full performance impact of alignment changes
