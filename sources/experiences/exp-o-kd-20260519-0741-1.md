## Solution

Replace bool* tree_mask with uint8_t* bit-packed representation where each byte holds 8 boolean values as individual bits, using bitwise OR (|= (1 << bit_idx)) to set connection flags and reducing total global memory writes by ~8x for the mask region

## Key Lessons

- Replace bool arrays with bit-packed uint8_t storage when each element carries only 0/1 information — the ~8x reduction in memory footprint and fewer distinct cache lines written to global memory outweighs the small read-modify-write overhead of bitwise OR
- Bit-packing boolean arrays also enables simpler index arithmetic: (bid * draft_token_num + tid) * bytes_per_item replaces mode-dependent branching and per-batch prefix-sum accumulation, reducing both code complexity and register pressure
- Use num_bytes_per_item = ceil(draft_token_num / 8) as a kernel parameter rather than hardcoding the packing factor — supports variable draft sizes without recompilation while keeping the bit-shift arithmetic uniform
