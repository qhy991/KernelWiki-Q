## Solution

Cache the entire expert_first_token_offset array into shared memory via cooperative __ldg loads and a __syncthreads() barrier, then read from smem instead of global memory; however the small array size (72 bytes) means global reads already hit L1 cache and the added synchronization overhead negates any benefit

## Key Lessons

- When accessed data fits in a single L1 cache line (< 128 bytes), direct global memory reads already benefit from cache — adding shared memory caching and __ldg introduces __syncthreads() barrier overhead that can negate or exceed any latency reduction
- __ldg forces reads through the read-only/texture cache path, but for data already L1-resident this adds instruction overhead without latency improvement — always measure to confirm cache misses before applying __ldg and smem caching
