## Challenge

MoE routing uses two separate CUDA kernel launches — one for normalizing topk weights by row sum and another for multiplying by routed_scaling_factor — causing two full global-memory read-write passes over the same output array plus duplicate kernel dispatch overhead


## Solution

Fuse the normalize and scale operations into a single fused_rescale_kernel where each thread divides by row sum and immediately multiplies by the scaling factor on the same register-held value, eliminating one entire global memory round-trip and one kernel launch dispatch

## Key Lessons

- Fusing consecutive elementwise kernels that read and write the same memory region into a single kernel eliminates redundant global memory round-trips — applies whenever two kernels operate on overlapping data with no inter-kernel dependencies beyond simple element-wise transformations
- Eliminating a kernel launch dispatch saves driver overhead (~5-20μs per launch) which is significant for small data volumes where compute time is comparable to dispatch latency
