## Challenge

MoE candidate selection uses two separate kernels: compute_swb_kernel writes sigmoid(scores)+bias to an intermediate global-memory buffer, then candidates_before_kernel reads from that buffer to find per-group max, wasting one full global-memory write+read pass and an extra kernel launch


## Solution

Fuse the sigmoid+bias computation directly into the candidates kernel by reading raw scores and bias arrays, computing sigmoid_accurate and adding bias inline per-element, eliminating the intermediate swb buffer and the separate compute_swb_kernel entirely — for 32 experts per group each thread handles exactly one expert, so inline computation adds only 2-3 instructions while removing a full global-memory pass

## Key Lessons

- Fuse elementwise pre-processing (sigmoid, bias addition) into the downstream reduction kernel when the per-element compute cost is low relative to the memory traffic saved — eliminates one global-memory write+read pass and one kernel launch overhead
- When num_experts_per_group equals WARP_SIZE (32), each thread processes exactly one element, making inline computation essentially free (2-3 extra instructions) compared to the cost of an extra global-memory round trip
- Eliminate intermediate buffers by re-computing values inline when the computation is cheap (sigmoid + add) and the data would otherwise require a full global-memory write followed by a full global-memory read
