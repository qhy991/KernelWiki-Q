## Challenge

FP8 quantization kernel performs a floating-point division (__half2float(val) / scale) per element in the inner loop, repeating the expensive division instruction for every element even though the scale value is uniform across all threads


## Solution

Precompute the reciprocal of scale (1.0f / *scale) once per thread before the loop and replace per-element division with multiplication by the cached inverse, reducing each iteration from one FDIV to one FMUL

## Key Lessons

- Replacing per-element division with precomputed reciprocal multiplication reduces arithmetic cost from FDIV (~16-20 cycles) to FMUL (~4 cycles) per element — but the speedup only materializes when the kernel is compute-bound at steady-state throughput, not when launch overhead or memory transfers dominate
- At small problem sizes (4096 elements, ~24 KB total I/O), kernel launch overhead and CUDA event resolution dwarf single-instruction arithmetic savings; always profile at realistic problem sizes before concluding an optimization is effective
