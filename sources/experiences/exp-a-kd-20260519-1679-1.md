## Solution

Fuse histogram construction into the candidate buffering scan by computing the next-pass radix bin from the already-loaded logit value and accumulating via atomicAdd inside the same branch that buffers the index, eliminating the separate histogram-build pass entirely

## Key Lessons

- Fuse histogram or reduction construction into existing data-movement scans (buffering/filtering loops) rather than running a separate pass over the same elements — halves global memory reads and shared-memory atomics for the candidate set
- In multi-pass radix algorithms, each pass partitions elements into output and candidate buffers; computing the next-pass histogram inline during partition avoids a redundant re-read of identical data and the associated __syncthreads() barrier
