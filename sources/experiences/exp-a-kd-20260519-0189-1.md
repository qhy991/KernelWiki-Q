## Challenge

Warp index is broadcast to all lanes via __shfl_sync(0xffffffff, threadIdx.x / 32, 0) inside a tight loop, forcing a full-warp shuffle barrier each iteration even though all threads in the same warp inherently compute the identical value from threadIdx.x / 32


## Solution

Replace __shfl_sync broadcast with a plain threadIdx.x / 32 integer division wrapped in canonical_warp_idx(), which compiles to a single shift-right instruction with no cross-lane synchronization barrier

## Key Lessons

- Avoid __shfl_sync for values that are invariant across all lanes (e.g., threadIdx.x / 32) — each thread can compute the same result independently via plain arithmetic, eliminating the shuffle barrier and its scheduling constraints
- When a warp-index value is needed inside a tight loop, use a __forceinline__ helper returning threadIdx.x / 32 (compiles to a single SHR instruction) rather than issuing a shuffle every iteration — the per-iteration overhead accumulates measurably in compute-light kernels
