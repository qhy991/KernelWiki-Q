## Challenge

The before source (hierarchical_cta_tile_before.cu) passes cta_tile directly as the TiledCopy tile argument without any compatibility check. ## Solution

Wrap tile selection in a compile-time if-constexpr guard that checks compatible(shape(CTA_Tile), shape(SLayout)); when incompatible, apply product_each(cta_tile) to flatten hierarchical modes (e.g., ((4,32),64) -> (128,64)) so the resulting shape matches the flat SLayout structure

- **speedup**: 0.000x (regression)


## Key Lessons

- Always guard TiledCopy tile selection with a compile-time compatible() check between shape(CTA_Tile) and shape(SLayout) — hierarchical vs flat rank mismatches silently produce incorrect TMA partitioning at runtime
- Use product_each() to flatten hierarchical CTA_Tile modes into their flat products when the SLayout expects a non-nested shape — product_each collapses each sub-tuple to its product (e.g., (4,32) → 128) while preserving already-flat modes
- Leverage if-constexpr with compatible() for zero-cost compile-time dispatch between the fast path (shapes already match) and the fallback path (flattening needed) — no runtime overhead is introduced
