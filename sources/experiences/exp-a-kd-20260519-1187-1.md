## Solution

Replace the single-pass 512-bin histogram with a progressive 4-step refinement using 2048 bins per step, where each step extracts finer bits of the float representation (half top bits, float upper/middle/lower bits), with early exit via __syncthreads_or to stop scan rounds as soon as the threshold bin is found, and isPartialMatch filtering to skip non-candidate elements in later steps

## Key Lessons

- Progressive histogram refinement with increasing bit precision (half top bits → float upper → middle → lower) isolates the top-k threshold more efficiently than a single coarse pass — the first step often narrows candidates sufficiently that later steps process far fewer elements
- Round-based prefix scan with __syncthreads_or early exit avoids scanning all bins when the threshold is found in an early round, saving computation proportional to the number of skipped rounds
- isPartialMatch filtering in progressive steps skips elements whose upper bits don't match the threshold pattern from prior steps, reducing atomicAdd traffic and histogram work in later refinement passes
