## Challenge

The before kernel uses 2 warps (64 threads) per CTA block with shared memory (s_partial[2], s_inv_rms) and two __syncthreads() barriers per iteration: one after writing warp-level partial sums to shared memory, and another after computing the cross-warp inv_rms aggregation


## Solution

The after kernel uses a single warp (32 threads) per block, eliminating all shared memory and __syncthreads() barriers. The sum-of-squares reduction is completed entirely within the warp using __shfl_down_sync, and inv_rms is broadcast to all 32 threads via __shfl_sync from lane 0 — no inter-warp...

## Results

- **speedup**: 1.128x (improved)


## Key Lessons

- The 32-thread block also enables higher GPU occupancy (up to 16 blocks per SM vs 6 for the 64-thread CTA), allowing more tokens to be processed concurrently.
