## Solution

Replace per-expert thread-mapped fill with a cooperative binary search over the prefix-sum array where all blockDim.x threads iterate over block indices and binary-search the prefix array to find the owning expert, distributing work evenly across all threads

## Key Lessons

- Replace per-owner thread-mapped fill with cooperative binary search over the prefix-sum array when the number of owners (experts) is much smaller than thread count — distributes work evenly across all threads and eliminates load imbalance from skewed owner counts
