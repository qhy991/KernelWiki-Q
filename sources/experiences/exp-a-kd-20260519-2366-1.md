## Solution

Add an early-return guard after cache_index resolution: if (cache_index == params.pad_slot_id) { return; } so that thread blocks mapped to padding slots skip all memory loads, arithmetic loops, and output writes entirely

## Key Lessons

- Early-return for padding tokens in SSM/scan kernels eliminates wasted arithmetic and memory traffic → works because padding slots produce discarded outputs so all computation on them is dead work → applies to continuous batching where a fraction of batch entries are padding
- Fixed grid launches still schedule all blocks including those that immediately early-return, incurring scheduling overhead that dilutes per-block savings → for maximum benefit, resize the grid to only valid entries via compaction or mask rather than relying solely on per-block early-exit
