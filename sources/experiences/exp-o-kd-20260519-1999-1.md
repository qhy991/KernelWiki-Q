## Solution

Add an explicit int64_t input_stride parameter to the kernel signature and replace all blockIdx.x * hidden_size offsets with blockIdx.x * input_stride, allowing the kernel to read rows at an arbitrary stride without a prior .contiguous() copy; contiguous inputs (stride == hidden_size) remain unchanged

## Key Lessons

- Replace hardcoded dimension strides with explicit stride parameters so kernels handle non-contiguous (strided) tensor layouts directly — this avoids a costly .contiguous() device-side copy whose overhead scales linearly with tensor size
- When adding stride parameters, use int64_t to match PyTorch's stride representation and relax contiguity checks to only require inner-dimension contiguity (stride(-1) == 1) rather than full tensor contiguity
