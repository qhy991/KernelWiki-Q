tokens_16_256ep

## Solution

The after path's single cublasGemmEx writes 4096 fp32 elements directly, eliminating the 4 KB bf16 write, the 4 KB bf16 read, and the cast kernel launch. The consistent ~20% speedup shows the cast kernel overhead (launch + extra memory traffic) is a fixed fraction of the small-GEMM total time.

## Key Lessons

- At M=16 the before path writes 4096 bf16 elements as GEMM output, then reads them back in the cast kernel and writes 4096 fp32 elements.
