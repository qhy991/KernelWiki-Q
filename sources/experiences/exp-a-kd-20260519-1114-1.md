## Solution

Replace the multi-instruction shift/mask/OR byte rearrangement with a single PRMT (byte-permute) hardware instruction per half2 pair, which selects and interleaves bytes from the packed int32 and the FP16 bias constant 0x64646464 in one hardware cycle, reducing instruction count from 6+ arithmetic ops to 2 PRMT calls

## Key Lessons

- PRMT (byte-permute) replaces multiple shift/mask/OR instructions with a single hardware instruction for byte rearrangement into FP16 layout — reduces instruction count and frees warp scheduler slots in large kernels where pipeline structure constrains compiler scheduling
- Microbenchmark regressions do not always predict real-world kernel performance: nvcc can optimize simple OR/shift patterns in isolated loops, while inline asm acts as a compiler scheduling black box, but in full kernels the instruction count reduction and scheduler slot freeing dominate
