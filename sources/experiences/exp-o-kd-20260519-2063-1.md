## Challenge

MoE top-k routing uses two separate kernel launches — a topkKernel that writes unnormalized weights to global memory, then a renormKernel that reads them back (64KB), computes row-wise sums, and writes normalized values back (64KB) — incurring a redundant global memory round-trip and full kernel launch overhead for very little per-row work


## Solution

Fuse topk selection and renormalization into a single kernel by accumulating selected_sum in shared memory during the iterative argmax loop (one addition per selected weight by thread 0), then dividing all k weights by selected_sum in a short serial loop at the end — eliminating one full kernel launch and the redundant 128KB global memory round-trip

## Key Lessons

- Fuse producer-consumer kernel pairs when the second kernel processes very little data per row (k=2 weights) — kernel launch overhead and global memory round-trips dominate over compute, so eliminating one launch yields multiplicative speedup
- Accumulate running sums in shared memory during an iterative selection loop (selected_sum += best.value) to avoid a separate reduction pass — when k is small, serial accumulation by thread 0 is cheaper than a parallel reduction across warp/block
- For small-k top-k with renormalization, the normalization division can be done serially by thread 0 after the selection loop exits — avoiding any warp-level reduction or atomic overhead in the renormalization phase
