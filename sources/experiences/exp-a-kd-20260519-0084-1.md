## Challenge

Convolution interleaved store pattern uses C++ if-guarded stores (if(guard) { *ptr = val; }) for conditional writes in scattered non-contiguous addresses, potentially leaving performance on the table versus explicit PTX-level predicated stores


## Solution

Replace C++ if-guarded store with inline PTX @p st.global.v4.u32 predicated store via a helper function ptx_global_store_16b, which uses a PTX .reg .pred register and setp.ne.b32 to predicate the vector store directly at the instruction level

## Key Lessons

- The nvcc compiler at -O3 already emits efficient PTX predicated store sequences for simple if-guarded scalar/vector stores — manual inline asm predication provides negligible benefit when the compiler pattern matches the obvious C++ idiom
- When a kernel is memory-latency-bound due to scattered non-contiguous writes across threads, instruction-level micro-optimizations like predication vs. branching are hidden by memory latency and fall within noise margins
