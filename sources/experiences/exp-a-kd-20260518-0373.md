## Challenge

The fp16 case shows the same regression pattern as bf16 for the same structural reason. The before kernel uses blockReduceMax with 256 threads per token and stride=256, giving each thread exactly 1 vector-load iteration and generating 256 concurrent wide memory requests per token


## Solution

The after kernel uses warpReduceMax with 32 threads per token and stride=32, forcing each thread into 8 sequential vector-load iterations with only 32 concurrent requests at any instant. The reduced memory-level parallelism outweighs the benefit of eliminating __syncthreads() on the A800 (sm_80)....
