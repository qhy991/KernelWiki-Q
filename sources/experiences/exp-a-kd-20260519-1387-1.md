## Challenge

Per-group FP8 scale packing uses atomicOr to shift-OR each group's exponent byte into a shared uint32, requiring the output buffer to be pre-zeroed and incurring atomic contention across warps writing adjacent packed slots


## Solution

Replace atomicOr-based 32-bit packing with direct byte-addressed writes using reinterpret_cast<uint8_t*>, where each thread writes its single exponent byte at the exact byte offset without any atomic or shift-OR logic

## Key Lessons

- Replace atomicOr packing of sub-word fields with direct byte-addressed writes via reinterpret_cast<uint8_t*> when each thread writes a non-overlapping byte — eliminates atomic contention, removes the need for shift-OR construction, and avoids requiring a pre-zeroed destination buffer
- Direct byte writes to sub-word lanes of a uint32 buffer are safe when threads write disjoint byte offsets within the same 32-bit word, as the GPU memory controller handles sub-word granularity without data races
