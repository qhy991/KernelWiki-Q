## Solution

Replace scalar half load/store with 128-bit uint4 vectorized accesses that pack 8 half elements per transaction, reducing loop iterations from 16 to 2 per thread and enabling the hardware to coalesce 16-byte transactions into full 32-byte L2/DRAM bursts

## Key Lessons

- Use uint4 (128-bit) vectorized loads/stores for contiguous half-precision memory copies instead of scalar element-by-element access — packs 8 half values per transaction, reducing loop iterations and enabling hardware coalescing into full 32-byte memory bursts
- Vectorized access benefits scale with element size: half (2 bytes) gains more than float (4 bytes) because 128 bits packs 8 half vs 4 float, amplifying the reduction in instruction count and loop iterations
