## Solution

Reorder the kernel prologue to compute local_split_kv from per-batch dim_k/split_kv arrays BEFORE declaring shared memory or constructing global tensor pointers, then add an early return when local_split_kv==1 to skip all resource allocation for short sequences that need no reduction

## Key Lessons

- Move per-CTA parameter computation (local_split_kv, local_dim_k) before __shared__ declarations and global pointer setup so that early-return guards can skip resource allocation entirely — applies to any kernel with variable work per CTA where some CTAs may have no work
- In flash-attention split-KV reduction, short sequences (dim_k <= tile_shape_s) produce local_split_kv==1 meaning no cross-split reduction is needed — an early return at this point avoids shared memory allocation and LSE reduction overhead for the common short-sequence serving path
