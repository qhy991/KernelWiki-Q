## Solution

Replace per-thread counter + cumsum lookup with atomicAdd on local_offsets[expert_id]: thread 0 initializes local_offsets from cumsum, then each token atomically claims its rank in one operation, eliminating the 2D tokens_cnts matrix and simplifying the sorting loop

## Key Lessons

- For MoE token sorting where ordering within an expert does not affect correctness, atomicAdd on a shared offset array eliminates the need for a per-thread 2D counter matrix and prefix-sum lookup, yielding large speedups (2.4–9.7x) by reducing shared-memory footprint and simplifying the sorting loop
- atomicAdd-based position claiming trades deterministic per-thread ordering for a single-operation rank assignment — acceptable whenever downstream consumers (e.g., expert block processing) do not depend on token order within an expert
