## Solution

The after kernel eliminates atomics by giving each thread its own per-expert count row in shared memory, then performs a serial per-expert prefix sum across thread rows followed by a serial cumsum, replacing CUB BlockScan entirely — but this trades 64x more shared memory (2064 ints vs 32 ints) and serial reduction work for the elimination of low-contention atomics that were not a bottleneck

## Key Lessons

- Replacing shared-memory atomicAdd with per-thread counting arrays scales shared memory by thread_count x num_experts — this backfires when atomics have low contention (few expert targets) and the per-thread initialization cost dominates
- CUB BlockScan uses warp-shuffle-based O(log N) prefix sum that outperforms serial O(N) reduction loops for moderate thread counts — do not replace library scan primitives with serial loops without benchmarking
- For MoE alignment with few experts (<=16) and many threads (1024), atomic contention on shared memory is negligible and atomicAdd+CUB is more efficient than contention-free per-thread counting
