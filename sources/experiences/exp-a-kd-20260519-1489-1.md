## Solution

Add a can_vec guard that checks both address alignment (addr & WIDTH-1 == 0) and length alignment (len & VEC_SIZE-1 == 0) before computing any alignment arithmetic, and early-returns into the pure vectorized loop to skip alignment computation, pointer adjustment, and scalar loop branches entirely

## Key Lessons

- Adding a fast-path early return for aligned inputs in a vectorized elementwise kernel is semantically correct and reduces instruction count, but nvcc already eliminates zero-iteration scalar loops via dead-code analysis, so the benefit is invisible at small kernel sizes where launch overhead dominates
- Microbenchmarking sub-millisecond kernels is dominated by launch overhead (~5us) and measurement noise; an optimization that saves only a handful of integer instructions per thread cannot be reliably measured without either larger workloads or kernel-level profiling tools (ncu)
