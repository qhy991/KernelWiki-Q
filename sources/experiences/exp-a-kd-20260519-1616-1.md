## Challenge

KV cache dequantization uses a single shared kv_scale for both key and value dequantization, preventing independent per-tensor scaling when key and value tensors have different dynamic ranges during fp8 quantization


## Solution

Split the shared kv_scale parameter into separate k_scale and v_scale, allowing each dequantization path to use its own per-tensor scale factor for correct fp8-to-fp32 conversion when key and value quantization ranges differ

## Key Lessons

- Separating shared kv_scale into independent k_scale and v_scale per-tensor parameters improves correctness flexibility for fp8 KV cache dequantization with zero performance cost — applies whenever key and value tensors may have different quantization dynamic ranges
- When k_scale == v_scale the numerical output is identical to the shared-scale version, so the refactor is backward-compatible and safe to deploy without conditional logic
