## Challenge

MoE fused-silu-quant kernel dispatches work via global flat iteration across all expert rows, requiring each thread to perform a register-chunk offset-array search (loading 16 uint32 offsets per chunk, up to 64 comparisons across 4 chunks for 64 experts) per element to identify the owning expert, then skips masked rows with continue that only skips the current element rather than terminating iteration within that expert's row range


## Solution

Replace global flat iteration with static per-expert thread assignment computed via integer division/modulo from thread ID (with remainder handling for non-uniform distribution), eliminating all offset-array reads and comparisons, and change masked-row early exit from continue to break within the per-expert iteration scope so threads terminate their loop immediately upon reaching the mask threshold rather than continuing to iterate across remaining global elements

## Key Lessons

- Compute expert assignment from thread ID via integer division/modulo instead of searching offset arrays at runtime — turns O(n_experts) global-memory reads and comparisons per element into O(1) arithmetic, applicable whenever expert boundaries are uniform (rows_per_expert is constant)
- Use per-expert iteration scope with break instead of global iteration with continue for masked-row early exit — when threads are statically assigned to a single expert's row range, break terminates the entire loop upon hitting the mask threshold, eliminating wasted iterations past the valid row boundary
- Handle non-uniform thread-to-expert distribution by assigning remainder threads one extra stride via a bound calculation (bound = remainder * (stride+1)), ensuring all threads are utilized without divergence in the steady-state loop
