## Solution

Process 2 FP16 elements per thread using half2 vectorized loads/stores via reinterpret_cast, consolidating two separate 2-byte scalar memory transactions into one 4-byte half2 transaction and halving the total memory transaction count for the same data volume

## Key Lessons

- Vectorize memory access with half2 (4-byte transactions) instead of scalar half (2-byte transactions) for memory-bandwidth-bound elementwise kernels — halves transaction count and directly reduces latency when the kernel is DRAM-bound
- Use reinterpret_cast<half2*> to load two adjacent half values as a single half2 register when data is naturally aligned, avoiding the need for separate pointer types or special allocation
- Scale the thread index by the vector width (idx = 2 * ...) and halve the grid dimension accordingly to maintain correct work distribution when switching from scalar to vectorized processing
