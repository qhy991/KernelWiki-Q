## Solution

Replace scalar half loads/stores with half2 vectorized accesses: load two consecutive FP16 values via HALF2(), compute both additions per-component using __hadd on the half2 struct fields, and store two results via HALF2(), doubling elements processed per thread and halving memory transactions

## Key Lessons

- Vectorize FP16 memory accesses with half2 to process 2 elements per thread, consolidating two 2-byte scalar transactions into one 4-byte transaction — applies to any bandwidth-bound elementwise kernel operating on contiguous FP16 data
- Scale thread stride by 2 (2*(blockIdx.x*blockDim.x+threadIdx.x)) when using half2 vectorization to avoid overlapping work between adjacent threads
