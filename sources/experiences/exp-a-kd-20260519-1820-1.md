## Solution

Decompose one m16n8k32 MMA call into four m8n8k16 calls by splitting the A matrix into row-pairs (rows 0-7 / rows 8-15) and the B matrix into column-halves, accumulating partial results into the same four accumulator registers across the four sequential calls

## Key Lessons

- A single m16n8k32 MMA on sm80+ can be decomposed into four m8n8k16 calls on sm75 by splitting rows (0-7 / 8-15) and B-column halves, accumulating into the same registers — enables backward-compatible int8 GEMM on Turing hardware
- At micro-benchmark scale (1 thread, 1 block), kernel launch overhead (~2.5us) dominates total latency, making the 1-vs-4 instruction difference unmeasurable — benchmark design must saturate the pipeline with enough work to expose instruction-level differences
- The m8n8k16 decomposition preserves numerical equivalence with m16n8k32 when using satfinite accumulation, since each sub-tile is accumulated independently into the same int32 registers in the correct order
