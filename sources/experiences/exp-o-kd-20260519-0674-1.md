## Challenge

Per-page cudaMemcpy transfers (48 calls of 8KB each) incur redundant driver dispatch and synchronization overhead for each individual call, with the overhead-to-data ratio being extremely high for small 8KB transfers


## Solution

Detect consecutive runs in both source and destination index arrays at runtime and merge each run into a single cudaMemcpy call, reducing 48 individual 8KB transfers into 6 merged 64KB transfers and eliminating 42 redundant dispatch roundtrips

## Key Lessons

- Coalesce contiguous cudaMemcpy calls by detecting consecutive index runs in both source and destination arrays — merging N small transfers into fewer large transfers eliminates per-call driver dispatch overhead proportional to the reduction in call count
- Per-call dispatch overhead dominates total latency for small cudaMemcpy transfers (8KB); merging into 64KB+ transfers amortizes overhead across more data, yielding speedups directly proportional to the reduction in call count
- The run-detection pattern (checking src_idx[i+1]-src_idx[i]==1 && dst_idx[i+1]-dst_idx[i]==1) is a reusable O(n) preprocessing step applicable whenever page-level or block-level copy indices may contain contiguous segments
