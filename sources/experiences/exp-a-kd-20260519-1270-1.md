## Challenge

Scalar half loads and stores issue 2-byte memory accesses per thread, severely underutilizing the 32-byte memory transaction segment width and wasting most of each transaction's bandwidth capacity


## Solution

Pack 8 half values into a 16-byte __align__(16) struct and cast pointers so each thread issues a single LDG.128/STG.128 instruction, fully utilizing memory transaction width and reducing total load/store instruction count by 8x

## Key Lessons

- Scalar half (2-byte) accesses have the worst memory efficiency per transaction — each 2-byte load wastes most of a 32-byte segment, so packing into 128-bit aligned structs yields disproportionately large speedups compared to wider types like float
- Use a templated __align__ struct (array_t<T,sz>) to enable LDG.128/STG.128 instructions from portable CUDA code without inline PTX or __ldg intrinsics — the compiler auto-generates 128-bit loads when the pointer type is 16-byte aligned
- For memory-bound copy/elementwise kernels, vectorized access width is the primary performance lever — reducing instruction count by N× (here 8×) translates almost directly to latency reduction when the kernel is purely bandwidth-limited
