## Solution

Fuse bias addition into the GEMM epilogue by loading the bias vector into a shared memory tile placed in the overlap gap between the B tile buffer and reduction buffer using the Marlin overlap layout formula max(max(sh_b, sh_red), min(sh_b, sh_red) + sh_bias) + sh_a, saving 256 bytes vs naive stacking and eliminating a separate kernel launch

## Key Lessons

- In tiled GEMM kernels the B tile buffer and reduction buffer have non-overlapping peak lifetimes — placing bias in the gap between max(sh_b, sh_red) and min(sh_b, sh_red) + sh_bias enables fused bias without increasing peak shared memory usage (Marlin overlap layout)
- Fusing bias into the GEMM epilogue may regress per-kernel latency due to extra smem allocation and arithmetic, but eliminates a separate bias-add kernel launch and global memory round-trip, providing net system-level throughput improvement in production inference pipelines
