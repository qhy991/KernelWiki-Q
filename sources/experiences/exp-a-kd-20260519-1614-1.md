## Solution

Replace the per-nibble shift/mask/subtract loop with a 16-entry compile-time LUT and three PTX prmt.b32 instructions per 4 nibbles, using bit 3 of each nibble to select between low/high LUT candidates via byte permutation — converting all 8 nibbles in parallel without scalar arithmetic

## Key Lessons

- PTX prmt.b32 LUT-based nibble conversion replaces 8 shift/mask/subtract operations with 6 permutation instructions but may show neutral performance when the compiler fully unrolls the naive loop and instruction scheduler interleaves the scalar ops — the theoretical ILP advantage is absorbed by compiler optimization
- For small input sizes (e.g., 4096 words), kernel launch overhead and memory transfer latency dominate execution time, masking instruction-level optimization differences — LUT/prmt techniques require larger workloads to amortize their register pressure advantage
- Compile-time LUT constants (static constexpr uint32_t) passed as PTX "n" constraints allow the assembler to embed lookup values directly in instruction encodings, eliminating register allocation for the table at runtime
