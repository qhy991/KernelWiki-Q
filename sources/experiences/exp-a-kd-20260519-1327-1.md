## Solution

Replace the conditional vectorization branch with vectorize_with_alignment, which checks both pointer address alignment and length divisibility at runtime, handles misaligned prefixes and tails internally, and accepts a scalar lambda so the kernel body becomes a single generic call without duplicated fallback logic

## Key Lessons

- Replacing a length-only vectorization check with a full address-and-length alignment check (vectorize_with_alignment) yields neutral performance when the per-element computation (float division, double clamping, type cast) dominates — the alignment logic overhead is amortized by the heavy arithmetic, making abstraction cost negligible
- Prefer vectorize_with_alignment over manual conditional branches for robustness: it handles misaligned pointers, prefix/tail elements, and length non-divisibility in one generic call, eliminating duplicated scalar fallback code without measurable performance cost
- When kernel runtime is dominated by arithmetic intensity rather than memory throughput, structural refactoring of the vectorization framework has minimal performance impact even when it adds a few extra branch checks per thread
