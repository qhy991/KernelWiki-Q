## Solution

Replace scalar half loads with 128-bit uint4 vectorized loads that fetch 8 half values (16 bytes) per instruction, enabling each warp to produce 512 bytes per transaction and fully saturate the L2/DRAM memory path, while accumulating into a per-thread float[8] array and converting back to half only at output

## Key Lessons

- Scalar half loads (2 bytes) produce only 64 bytes per warp transaction — half the optimal 128-byte L2 transaction size; use uint4 (128-bit) vectorized loads to fetch 8 half values per instruction and fully saturate the memory path, applicable whenever accessing contiguous half-precision arrays
- Vectorize the inner reduction loop by accumulating into a float[ELEM_PER_THREAD] array from register-resident data unpacked from the loaded vector, then convert back to half only at output — defers precision narrowing and reduces global memory instructions by ELEM_PER_THREAD×
- reinterpret_cast<const uint4*>(half_ptr) and stride by num_vec (cols/8) instead of cols to maintain correct indexing when switching from scalar to 128-bit vectorized access of contiguous half arrays
