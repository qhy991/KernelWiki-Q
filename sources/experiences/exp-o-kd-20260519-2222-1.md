## Solution

Fuse the unscaled fp8→half conversion and the scale multiply into a single kernel that reads each fp8 element, converts to half, promotes to float for the scale multiply, and converts back to half in one pass—eliminating the intermediate global memory buffer and one kernel launch entirely

## Key Lessons

- Fuse conversion + scale kernels into a single kernel when they form a producer-consumer pair over global memory—the fused version eliminates the intermediate buffer write+read (2N bytes halved) and one kernel launch, yielding major speedups for memory-bound dequant patterns
- Prefer __nv_cvt_fp8_to_halfraw followed by a float-domain scale multiply over separate unscaled-convert then scale kernels—the per-element arithmetic is identical but global memory traffic drops by ~50%
