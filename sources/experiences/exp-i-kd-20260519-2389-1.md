## Challenge

Skeleton code provides only commented-out TODO markers for a CUTLASS PredicatedTileIterator-based 2D memory copy kernel, requiring implementation of iterator initialization, fragment-based tile traversal with boundary predication, and correct strided dimension iteration counting


## Solution

Implement a two-phase copy loop using PredicatedTileIterator: zero-initialize the fragment, perform the first load/store outside the loop, then iterate remaining tiles in a for loop with explicit ++iterator increment, using ceiling division on the strided dimension to compute total tile count

## Key Lessons

- Zero-initialize Fragment before first PredicatedTileIterator::load to prevent undefined values when boundary predication leaves some fragment elements unset — applies to all CUTLASS iterator usage with irregular extents
- Use two-phase iteration (first tile outside loop + remaining tiles in for loop with explicit ++iterator) to handle PredicatedTileIterator's forward-only traversal cleanly without separate index tracking
- Ceiling division (extent + stride - 1) / stride computes the number of tiles needed to cover the strided dimension — standard pattern for tile-based iteration in CUTLASS PitchLinear layouts
