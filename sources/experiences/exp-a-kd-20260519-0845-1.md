## Solution

Replace per-element scalar float loads/stores with uint4 (128-bit) packed transactions so each thread processes 4 floats at once, consolidating 4 separate memory instructions into one 16-byte load/store and improving L2 cache line utilization

## Key Lessons

- Replace scalar float loads/stores with uint4 (128-bit) packed transactions in memory-bound element-wise kernels — each thread processes 4 floats in one transaction, reducing memory instruction overhead by 4x and improving L2 cache line utilization
- Use reinterpret_cast<const uint4*>(ptr)[index] for aligned 128-bit loads from float arrays when head_size is a multiple of 4 — the byte offset is (index * 4 * sizeof(float)), computed as pack_offset / pack_size in the uint4 index space
- Scale thread grid by (head_size / pack_size) instead of head_size when switching to packed access — fewer threads cover the same data, which can also reduce warp fragmentation when head_size is small
