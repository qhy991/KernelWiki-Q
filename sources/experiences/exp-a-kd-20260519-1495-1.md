## Solution

Replace static shared memory with dynamic shared memory (extern __shared__) and 2D subscript with flat indexing via index(num_experts, row, col) to enable runtime-sized allocation; however, the runtime stride multiplication in index() introduces per-access overhead that the compiler cannot fully eliminate even when num_experts is effectively constant, causing a 2.55% latency regression

## Key Lessons

- Static shared memory with compile-time 2D array dimensions enables zero-cost stride computation by the compiler — switching to dynamic shared memory with flat indexing introduces a runtime multiplication per access that cannot be fully optimized away even when the stride is effectively constant
- Dynamic shared memory provides runtime flexibility but may regress performance when the compile-time constant size is already known — prefer static allocation when dimensions are fixed at compile time to avoid per-element stride computation overhead
