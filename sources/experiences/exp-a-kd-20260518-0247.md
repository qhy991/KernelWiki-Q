## Key Lessons

- The f32x4 kernel regresses because reducing threads from 1024 to 256 per block decreases occupancy.
- With only 256 threads, the GPU has fewer active warps to hide memory latency, and the wider per-thread access pattern does not compensate for the reduced scheduling depth.
