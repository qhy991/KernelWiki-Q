## Solution

Wrap global loads with __ldg() to explicitly route reads through the read-only data cache; however, on Ampere (sm_80) with -O3, NVCC already recognizes const __restrict__ pointers as read-only and auto-routes through the read-only cache, making explicit __ldg() redundant with no measurable performance gain

## Key Lessons

- On Ampere (sm_80+) with -O3, NVCC automatically routes loads through const __restrict__ pointers via the read-only data cache, making explicit __ldg() redundant — only use __ldg() on older architectures (Kepler/Maxwell/Pascal) where the compiler does not perform this optimization
- Sub-10μs kernel benchmarks measured with cudaEvent timing have noise margins around 1-2%, so speedups below ~2% in this regime are not statistically significant
