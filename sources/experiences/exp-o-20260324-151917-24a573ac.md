fp32_flash_attention_qwen2_5_7b_f16_cache8192

## Solution

Increase TILE_SIZE from 512 to 2048 to reduce outer loop iterations

## Key Lessons

- TILE_SIZE is a critical parameter for flash attention - larger tiles reduce outer loop overhead
- For seq_len=8192, TILE_SIZE=2048 is optimal (4 tiles, fits in 48KB smem)
- V accumulation inner loop is NOT the bottleneck - outer tile loop overhead matters more
- Transposing V in shared memory hurts because loading becomes non-coalesced
- cp.async PTX requires 16-byte aligned addresses - use with caution
- The half2 query + float4 K loading + V_CHUNK=128 combination from r2/v5 is already near-optimal for the inner loop
