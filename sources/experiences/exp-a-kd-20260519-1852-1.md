## Challenge

Residual addition and RMS normalization are implemented as two separate kernel launches, requiring two full passes over the residual array from global memory and incurring double kernel launch overhead for each token batch


## Solution

Fuse residual addition and RMS normalization into a single kernel where the first loop performs add + variance accumulation simultaneously, and the second loop reads residual data that remains L2-cache-hot from the first loop's write, eliminating one kernel launch and one full global memory round-trip

## Key Lessons

- Fuse sequential kernels that share an intermediate tensor into a single kernel — eliminates kernel launch overhead and keeps intermediate results in L2 cache, yielding significant speedup for memory-bound operations at small batch sizes
- When the first loop of a fused kernel writes an intermediate result and the second loop reads it, the data stays L2-cache-hot across the __syncthreads() barrier, turning what would be a full global-memory round-trip into a near-free L2 read
