## Solution

Enable TF32 tensor core acceleration by setting CUBLAS_TF32_TENSOR_OP_MATH math mode and CUBLAS_GEMM_DEFAULT_TENSOR_OP algorithm, which internally converts FP32 inputs to TF32 (19-bit mantissa → 10-bit) for tensor core execution while keeping the external API interface and data types unchanged

## Key Lessons

- Set CUBLAS_TF32_TENSOR_OP_MATH + CUBLAS_GEMM_DEFAULT_TENSOR_OP to enable tensor core acceleration for FP32 GEMM — provides up to ~16x throughput uplift on Ampere+ GPUs for compute-bound large matrix multipulations with only minor precision loss
- TF32 tensor core mode requires both the math mode and the gemm algorithm to be set together — changing only one may not activate tensor cores, as cuBLAS uses the combination of both to select the internal kernel implementation
- cuBLAS tensor core TF32 conversion trades ~3 bits of mantissa precision for dramatically higher throughput — acceptable for most deep learning and HPC workloads where 1e-1 FP32 tolerance is sufficient
