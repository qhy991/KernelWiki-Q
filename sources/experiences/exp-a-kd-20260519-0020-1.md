## Solution

Compute local_split_kv before declaring shared memory, then add an early return when local_split_kv==1 to skip shared memory allocation and the full reduction body entirely for short-sequence batches that need no multi-split reduction

## Key Lessons

- Compute per-CTA runtime parameters (local_split_kv) before declaring shared memory so that early-return guards can skip shared memory allocation entirely — saves resources on CTAs that don't need the reduction path
- Reordering pure computation before shared memory has no measurable overhead on long-sequence paths where the early return doesn't fire, making the reordering strictly beneficial or neutral across all input shapes
- For variable-length attention workloads with mixed short/long sequences, gating shared memory behind per-CTA condition checks can free shared memory for co-scheduled CTAs on the same SM, improving occupancy on heterogeneous batches
