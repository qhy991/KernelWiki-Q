## Challenge

Row-wise dot-product kernel loads input and gradient using scalar float accesses, issuing one 4-byte load per element, which underutilizes available memory bandwidth when data is naturally 16-byte aligned


## Solution

Cast input pointers to float4 and load four elements per LDG instruction, accumulating four multiply-adds per iteration to quadruple effective memory throughput when data is 16-byte aligned

## Key Lessons

- Use float4 reinterpret_cast loads to coalesce four scalar floats into a single 128-bit transaction — applies whenever data is 16-byte aligned and contiguous, cutting load instruction count by 4x
- Vectorized loads reduce loop iteration count proportionally (kCols → kCols/4), lowering branching and address arithmetic overhead in addition to memory throughput gains
