## Solution

Add expert_ids and expert_step fields to Params and insert an early-return guard (if expert_ids[token_id / expert_step] == -1 return) before the memory load/compute section, so masked-token threads skip the two float4 global loads (32 bytes), SiLU computation (4 expf), and float4 store (16 bytes); the one-block-per-token mapping (BLOCK_SIZE=256, num_vecs=256) ensures no warp divergence since each block is either entirely masked or entirely active

## Key Lessons

- Early-return expert filtering in MoE kernels yields large speedups (2.5x+) when many tokens are masked because it eliminates global memory loads, compute, and stores for inactive tokens — applies to any MoE kernel where token-to-expert assignment is known before the main compute section
- One-block-per-token mapping (blockDim == hidden_dim / VEC_SIZE) eliminates warp divergence from early-return guards since each block is either entirely masked or entirely active — prefer this mapping over interleaved thread-to-token schemes when expert filtering is needed
- The observed speedup is bounded below the theoretical max (4x for 75% masked) because the full block grid is still launched and masked blocks incur scheduling overhead — further gains require token compaction (re-launching only active blocks)
