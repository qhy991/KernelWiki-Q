## Challenge

The reshape_and_cache_flash kernel passes FP8 quantization scale factors (k_scale, v_scale) as scalar float parameters in the kernel launch buffer, which limits flexibility when scales need to be device-resident or vary per-token


## Solution

Change k_scale and v_scale from scalar pass-by-value parameters to const float* pointers dereferenced inside the kernel, enabling device-resident scale factors; analysis reveals the measured 2.4x speedup is a microbenchmark noise artifact, not a genuine throughput gain

## Key Lessons

- Changing scalar kernel parameters to device pointers provides API flexibility (device-resident, per-token scales) but offers no intrinsic performance benefit — the scalar path reads from the parameter buffer while the pointer adds one cached global load, both negligible compared to the compute-heavy inner loop
- Single-token microbenchmarks (1 thread block, ~1024 iterations per thread) are dominated by launch overhead and timing granularity; measured speedups of 2x+ in this regime are typically noise artifacts confirmed by contradictory regression in batched cases
