## Solution

Fuse FP8 quantization into the merge kernel via a compile-time USE_FP8_OUTPUT template parameter: each thread computes the merged float32 result and immediately converts to FP8 using float_to_fp8e4 with a pre-inverted scale, then stores via a narrower uint2 pack — eliminating the intermediate BF16 buffer write/read entirely and removing the second kernel launch

## Key Lessons

- Fuse quantization (BF16→FP8) into the compute kernel that produces the data rather than using a separate pass — eliminates an entire round-trip through global memory for the intermediate tensor, yielding up to 1.5x speedup for large tensors where memory traffic dominates
- Use a compile-time bool template parameter (USE_FP8_OUTPUT) with if constexpr to keep both BF16 and FP8 output paths in the same kernel without runtime branching overhead
- When fusing quantization, accumulate in float32 then convert once to the target format — avoids intermediate precision loss from BF16 rounding before the final FP8 cast
