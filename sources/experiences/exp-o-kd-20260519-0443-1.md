## Solution

Replace scalar f32 loads with float4 vectorized loads so each thread fetches 4 elements per iteration, reducing loop iterations by 4x and issuing 128-byte aligned memory transactions that are more efficient for the GPU memory controller to coalesce

## Key Lessons

- Vectorized float4 loads fetch 4 elements per memory transaction, reducing instruction count and enabling 128-byte aligned coalesced access — applies whenever data is contiguous and 16-byte aligned in memory-bound kernels
- Reducing loop iteration count by a factor equal to the vector width (4x for float4) cuts loop overhead, index arithmetic, and branch prediction pressure in addition to the bandwidth benefit
- SGEMV kernels are memory-bound; the primary speedup comes from fewer, wider memory transactions rather than from the FMA computation itself — measure memory throughput to confirm the bottleneck before optimizing compute
