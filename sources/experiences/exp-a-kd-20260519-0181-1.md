## Solution

Annotate the by-value params argument with __grid_constant__ to request constant-memory caching of the parameter struct across the kernel grid, though for a ~24-byte struct that already fits in registers this provides no measurable benefit

## Key Lessons

- __grid_constant__ eliminates local-memory spills for large parameter structs, but provides no benefit when the struct is small enough to fit in registers (~24 bytes / 6 registers) — the register allocator already handles it efficiently
- Memory-bandwidth-bound kernels (simple element-wise multiply) are insensitive to parameter passing overhead because the arithmetic intensity is too low for parameter access to be a bottleneck
