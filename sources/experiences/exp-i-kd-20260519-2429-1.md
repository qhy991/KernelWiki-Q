## Solution

Implement the canonical shared memory staging pattern: coalesced global load into extern shared memory via threadIdx.x indexing, __syncthreads() barrier to ensure all loads complete, then read from shared memory for computation (multiply by 2) and write the result back to global memory

## Key Lessons

- Shared memory staging follows a fixed three-phase pattern: coalesced global load → __syncthreads() barrier → shared memory read/compute/write — this avoids redundant global memory accesses when the same data is reused across computation stages
- Use extern __shared__ for dynamically-sized shared memory when the required capacity depends on launch configuration rather than compile-time constants
- Index shared memory with threadIdx.x directly to ensure coalesced global memory loads and stores — each thread reads/writes its own contiguous element
