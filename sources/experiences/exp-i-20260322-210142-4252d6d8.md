## Challenge

Baseline transplant from torch-dependent code to pure CUDA


## Solution

Mechanical transplant: removed torch headers, kept CUDA kernel logic, added extern C wrapper


### Key Changes

- Removed #include <torch/extension.h>
- Added extern "C" void rms_norm_kernel entry point
- Preserved warp-per-row optimization with __shfl_down_sync


## Key Lessons

- Warp-per-row with 32 threads handling 4 elements each works well for hidden_size=128
- __ldg for read-only cache hints and __fdividef for fast division are preserved
- Entry function name must match expected: rms_norm_kernel for rms_norm op type
