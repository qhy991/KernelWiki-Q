## Solution

Fuse bias addition into the GEMM epilogue: keep the scaled dot-product result in registers, add bias inline, and write the final value to global memory in a single store, eliminating the separate bias-add kernel launch and its full output-tensor read/write round-trip

## Key Lessons

- Fuse elementwise epilogue operations (bias add, activation, residual) into the GEMM kernel's register-stage output path to eliminate separate kernel launch overhead and avoid a full output-tensor read+write round-trip through global memory — applies whenever the epilogue is a simple elementwise transformation on the GEMM result
- Keeping GEMM partial sums in registers and applying transformations before the final global store reduces output memory traffic by roughly half (one write vs write+read+write) for the output tensor
