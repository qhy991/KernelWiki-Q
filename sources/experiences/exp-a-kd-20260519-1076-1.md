## Solution

Replace bool-per-element mask with bit-packed uint8_t mask where each bit represents one tree position, reducing mask memory 8x (65536→8192 bytes), eliminating the cumulative offset loop and FULL_MASK/QLEN_ONLY branching, and using bitwise OR to set individual bits within packed bytes instead of per-element bool writes

## Key Lessons

- Bit-pack boolean masks into uint8_t (1 bit per flag instead of 1 byte per bool) to achieve 8x memory reduction — applies to any kernel that stores per-element on/off flags where the flag count per element is modest (<64) and the mask is accessed in a sequential bit-pattern
- Eliminating per-block cumulative offset loops (iterating over prior blocks' verified_seq_len) in favor of direct index arithmetic (bid * draft_token_num + tid) * stride reduces both instruction count and shared-data-dependent serialization within a warp
- Replace per-element bool initialization loops (writing false to N consecutive entries) with single-byte initialization of the packed word when using bit-packed representation — a memset of 2 bytes replaces 15 individual bool writes per token row
