## Solution

Fuse topk selection and renormalization into a single kernel (fusedTopkRenormKernel) by accumulating selected_sum in a shared variable during the iterative topk argmax loop (one addition per selected weight by thread 0), then dividing all k weights by selected_sum in a short serial loop at kernel end — eliminating one full kernel launch and the associated global memory round-trip for the weight read-modify-write

## Key Lessons

- Fuse lightweight post-processing (renormalization of k=2 floats) into the preceding kernel rather than launching a separate kernel — the per-row work is too small to amortize a second kernel launch, so fusion eliminates both launch overhead and a full global memory round-trip for the read-modify-write pattern
- Accumulate derived values (selected_sum) in shared memory during the main computation loop when the value is naturally produced as a byproduct of each iteration, rather than re-reading and re-computing them in a second pass
