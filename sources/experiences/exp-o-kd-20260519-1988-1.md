## Solution

Replace the per-byte FP32 conversion pipeline with a unified bit-manipulation pass: use a precomputed MASK (0x7F007F00) to extract exponent+mantissa fields from two interleaved FP8 values in one integer AND, shift by 1 bit to widen FP8's 4-bit exponent to FP16's 5-bit exponent, then apply a single __hmul2 with a precomputed bias constant (2^8) — eliminating all std::pow calls and FP32 intermediates

## Key Lessons

- Use integer bit-manipulation (MASK + shift) to widen FP8 exponent fields to FP16 width instead of decomposing into sign/exp/mantissa and calling std::pow — reduces ~12 FP32 instructions to ~5-6 integer/half2 operations and avoids the FP32 pipeline entirely
- Process two interleaved FP8 values packed in a 32-bit integer simultaneously using a single MASK+shift pattern — halves instruction count versus per-byte extraction when data uses Marlin-style interleaved layout
- Apply exponent bias correction via __hmul2 with a precomputed constant rather than per-element float arithmetic — leverages the FP16 ALU pipeline instead of FP32, reducing contention with MMA accumulators in mixed-precision GEMM kernels
