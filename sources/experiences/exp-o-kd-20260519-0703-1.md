## Challenge

Expert offset computation uses a <<<1,1>>> single-thread kernel that sequentially iterates over 256 experts, performing 256 strided global memory reads and 256 serial additions — a serial dependency chain that leaves GPU parallelism entirely unused.


## Solution

Replace the single-thread sequential loop with a <<<1,256>>> parallel CUB BlockScan that distributes 256 strided reads across 256 threads and computes the inclusive prefix sum cooperatively in shared memory, reducing the serial dependency chain from 256 additions to O(log N) steps per warp-level scan phase.

## Key Lessons

- Replace <<<1,1>>> sequential prefix-sum kernels with CUB BlockScan cooperative parallel scan — distributes reads across threads and reduces serial dependency from O(N) to O(log N) additions, applicable whenever the element count fits within one or a few thread blocks
- Use a chunked BlockScan with __shared__ broadcast carry when N exceeds BLOCK_SIZE — the last thread of each chunk writes the partial sum to shared memory, all threads sync, then the carry is added as base_prefix_sum to the next chunk's results
- Pad out-of-bounds threads with zero in cooperative scans (val = index < n ? input[...] : 0) so BlockScan produces correct prefix sums without conditional logic inside the scan primitive
