## Solution

Replace div/mod index decomposition with bitwise shift/AND for power-of-two-aligned tile dimensions, fold five stride multiplications into a single shift-OR expression since the decomposed bit fields occupy non-overlapping positions, and precompute numKTiles on the host to eliminate per-thread integer division

## Key Lessons

- When tile dimensions are power-of-two aligned, replace div/mod with shift/AND and stride multiplications with a single shift-OR expression — the decomposed index fields occupy non-overlapping bit positions, allowing the offset to be assembled via bitwise OR instead of five multiply-add operations
- Five independent stride multiplications provide better instruction-level parallelism than a single dependency chain (multiply then shift then OR) — fully-inlined microbenchmarks may show regression even when the compact formula wins in real non-inline __device__ contexts due to reduced register pressure and eliminated per-call variable setup overhead
- Hoist loop-invariant values like numKTiles (derived from numCols) out of per-thread kernel loops to precomputed host-side parameters — eliminates redundant integer division across all threads and reduces per-invocation register allocation in non-inline device functions
