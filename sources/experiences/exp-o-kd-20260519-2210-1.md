## Solution

Thread 0 writes the computed scale to shared_max[0] in addition to global memory, reusing the already-allocated shared memory slot from the prior reduction; after __syncthreads(), all threads read the scale from shared memory instead of global memory, eliminating the high-latency global load from the critical path

## Key Lessons

- When one thread computes a value needed by all threads in the block, broadcast via shared memory instead of a global memory round-trip — shared memory access completes in a few cycles vs hundreds for global memory, and the global store can be kept for correctness without being on the critical path
- Reuse existing shared memory buffers for broadcasting single-value results after a reduction — shared_max[0] already held the reduction result and is no longer needed, so overwriting it with the scale requires zero additional shared memory allocation
