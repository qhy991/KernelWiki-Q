## Solution

Enable TF32 tensor cores by setting CUBLAS_TF32_TENSOR_OP_MATH math mode and CUBLAS_GEMM_DEFAULT_TENSOR_OP algorithm, which routes FP32 inputs through tensor cores with TF32 internal precision, delivering up to 312 TFLOPS throughput on A800

## Key Lessons

- Set cublasSetMathMode to CUBLAS_TF32_TENSOR_OP_MATH and use CUBLAS_GEMM_DEFAULT_TENSOR_OP algorithm to enable TF32 tensor cores — this two-flag change provides ~5x speedup on Ampere+ GPUs for compute-bound FP32 GEMM with negligible precision loss
- TF32 tensor cores trade 8 bits of mantissa precision (19-bit vs 23-bit) for ~16x higher FLOPS vs FP32 CUDA cores — acceptable for most DNN training and inference workloads
