## Solution

Replace the 2-warp CTA block with a single 32-thread warp, eliminating all shared memory and both __syncthreads() barriers — the full reduction completes via __shfl_down_sync within one warp, and inv_rms is broadcast using __shfl_sync from lane 0, enabling 16 blocks per SM (vs 6) for higher occupancy

## Key Lessons

- For small hidden dimensions (≤64 elements) where one warp suffices for the reduction, use a single-warp block instead of a multi-warp CTA — this eliminates shared memory, __syncthreads() barriers, and cross-warp coordination, while dramatically increasing occupancy (16 blocks/SM vs 6 for 64-thread blocks)
- Use __shfl_sync(mask, value, 0) to broadcast a value from lane 0 to all warp threads as a zero-overhead replacement for shared-memory broadcast with __syncthreads() — applies whenever the producer-consumer relationship is within a single warp
- When per-thread compute is lightweight (1–2 elements), synchronization overhead (__syncthreads) dominates kernel time; reducing block size to remove barriers yields greater speedup than any compute-side optimization
