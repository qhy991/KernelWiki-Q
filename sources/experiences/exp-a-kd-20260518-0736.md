fp8_quant_32k

## Solution

The after code skips the reduction entirely and launches only 32 quant-kernel blocks, reading the data once. The ~1.79x speedup reflects both the eliminated 128KB redundant read and the saved shared-memory reduction + atomic overhead.

## Key Lessons

- The before code dispatches 32 grid blocks for segmented_max_reduction, each with 4KB shared memory and a full reduction tree, followed by 32 blocks for the quant kernel over 32768 floats.
