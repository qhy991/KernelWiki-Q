## Solution

Replace shared-memory butterfly reduction with a templated reduce_sum<kNumThreads> function using __shfl_xor_sync warp shuffle, performing 3 register-to-register shuffle instructions (~4-8 cycles each) with no shared memory access, no barriers, and no warp divergence

## Key Lessons

- Use __shfl_xor_sync warp shuffle for sub-warp reductions instead of shared-memory butterfly patterns — register-to-register shuffles avoid __syncthreads() barriers (~20-40 cycles each) and shared-memory latency, yielding dramatic speedups for small-group reductions
- Parameterize sub-warp reduction with a kNumThreads template parameter — enables compile-time unrolling of shuffle rounds and static_assert validation that thread count is a power of two, making the primitive reusable across different sub-warp widths
- Use an active_mask bitmask ((1u << kNumThreads) - 1u) with __shfl_xor_sync to correctly scope shuffle operations to only active threads — avoids shared-memory predication and eliminates warp divergence from if (threadIdx.x < kActiveThreads) branches within the reduction loop
