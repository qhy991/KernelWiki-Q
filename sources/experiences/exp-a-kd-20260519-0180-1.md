## Solution

Add __grid_constant__ qualifier to the kernel parameter declaration to explicitly request that the compiler place the read-only LargeParams struct into the read-only data cache, avoiding local-memory spills for parameter field accesses

## Key Lessons

- On CUDA 12.8 with -O3, NVCC already places const-qualified kernel parameters into the read-only data cache via its parameter-passing ABI, so explicit __grid_constant__ annotation yields no measurable improvement when the compiler already applies the equivalent optimization
- When a kernel is memory-bandwidth-bound on global memory loads (e.g., params.input[i] in a tight loop), any latency savings from parameter caching are hidden by the dominant global memory access cost — profile before optimizing parameter passing
