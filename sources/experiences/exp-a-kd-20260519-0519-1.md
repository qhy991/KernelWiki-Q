## Solution

Increase per-thread workload from 2 to 8 half elements by using 4 independent half2 register pairs, interleaving 4 load-compute-store sequences per thread to fill the instruction pipeline and reduce total thread count by 4x, which decreases warp scheduling overhead and improves memory system efficiency

## Key Lessons

- Increasing per-thread element count (ILP unrolling) from half2 to half4/half8 creates independent load-compute-store chains that fill the pipeline and hide latency — applies to elementwise activation kernels where per-element work is lightweight and memory-bound
- Reducing total thread count while keeping total work constant decreases warp scheduling overhead and SM resource contention, allowing the memory subsystem to service fewer warps more efficiently
- Use per-half2 boundary checks instead of a single thread-level check when processing multiple elements per thread, to avoid out-of-bounds writes on the tail blocks
