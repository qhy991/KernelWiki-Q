## Challenge

NHWC padding kernel processes each output element as a scalar float, performing one 4-byte load or store per loop iteration, underutilizing available global memory bandwidth while repeating the same index decomposition arithmetic for every single element


## Solution

Replace scalar float loads/stores with float2 vectorized I/O by casting the input/output pointers to float2* and halving the channel dimensions (c_in/2, c_out/2), so each thread processes two adjacent channels per iteration, doubling memory throughput per transaction and halving total loop iterations

## Key Lessons

- When both c_in and c_out are even in an NHWC padding kernel, casting pointers to float2* and halving channel counts doubles memory throughput per transaction — applies to any memory-bound NHWC layout transformation where channels are contiguous and counts are even
- Vectorized I/O reduces global memory transaction count proportionally to vector width but does not eliminate per-element arithmetic overhead (index decomposition), so the achievable speedup is bounded by the memory-bound fraction of kernel runtime
