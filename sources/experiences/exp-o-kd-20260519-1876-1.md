## Solution

Replace scalar __ldg loads and scalar stores with int4 (128-bit) vectorized loads/stores that process VEC_SIZE=2 floats per transaction, guarded by a 16-byte alignment check on all three pointers with a scalar fallback for unaligned or remainder elements

## Key Lessons

- Replace scalar __ldg loads and scalar stores with int4 (128-bit) vectorized loads/stores in memory-bound activation kernels — each int4 transaction moves VEC_SIZE elements, reducing total memory transactions by ~VEC_SIZE× and improving cache line utilization
- Guard vectorized fast paths with a runtime 16-byte alignment check on all participating pointers and provide a scalar fallback for unaligned addresses and remainder elements — ensures correctness for all input alignments without sacrificing the vectorized path when aligned
- Precompute per-token base pointers (x_ptr, y_ptr, out_ptr) outside the loop instead of recomputing token_idx * 2 * d + idx each iteration — reduces address arithmetic and enables clean reinterpret_cast to int4* for the vectorized path
