kfactor8_int8

## Solution

the after path generates the correct bank-conflict-free swizzle addresses for kFactor=8 interleaved layouts. This is a correctness-enabling change, not a latency optimization.

## Key Lessons

- In before/kfactor8_int8_before.cu, the compute_byte_offset_before function declares partition_contiguous_idx, access_contiguous_idx, and access_strided_idx as -1, then enters three if-branches (kFactor==4, ==2, ==1) none of which match kFactor==8, so the indices stay at -1.
