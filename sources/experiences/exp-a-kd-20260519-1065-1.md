## Challenge

A standalone sigmoid kernel writes transformed values element-by-element to a 128KB global-memory workspace buffer via scalar stores, then a separate TopK kernel reads that workspace back — wasting one full write/read round-trip and underutilizing threads (only 8 of 256 threads per block touch data with 8 experts)


## Solution

Fuse sigmoid into the TopK kernel using AlignedArray<float,4> vector loads (16 bytes per LDG) with interleaved addressing so 2 threads per row cooperatively load all 8 experts, compute sigmoid in registers (row_chunk[]), and eliminate the workspace buffer entirely — removing one kernel launch, 128KB of global-memory traffic, and the scalar-load bottleneck

## Key Lessons

- Fuse activation (sigmoid) into the consumer kernel and keep transformed values in registers instead of writing to a global-memory workspace — eliminates one kernel launch and a full write/read round-trip, which dominates latency for small row-width workloads
- Use AlignedArray<float,N> vector loads (16-byte transactions) with interleaved thread addressing so every thread in a sub-group carries useful data — with 8 experts and 2 threads per row, utilization jumps from 3% (8/256) to 100%
- Size the thread-to-row mapping so THREADS_PER_ROW = num_experts / ELTS_PER_LDG, yielding ROWS_PER_WARP = WARP_SIZE / THREADS_PER_ROW concurrent rows per warp for high occupancy without wasted threads
