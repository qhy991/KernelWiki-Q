## Solution

Instantiate cutlass::NumericArrayConverter<Destination, Source, Count> as a local object and apply it to dereferenced source/destination Array pointers — the CUTLASS Array type provides vectorized load/store while NumericArrayConverter generates type-specific PTX for each <Destination,Source> pair, yielding zero-overhead batched conversion across all supported numeric types

## Key Lessons

- cutlass::NumericArrayConverter<Dst,Src,N> wraps type conversion for N elements in a single functor call, automatically generating optimal PTX for each type pair — use it instead of per-element scalar conversion loops for vectorized throughput and type safety
- cutlass::Array<T,N> acts as a value-type wrapper that maps to vectorized load/store (e.g., float4, half2) when N matches hardware vector widths — passing Array pointers and dereferencing them in-kernel gives the compiler full visibility for memory coalescing and alignment
- Template-driven kernels with cutlass::NumericArrayConverter compile to specialized PTX per <Destination,Source,Count> instantiation with zero runtime overhead — all type dispatch and conversion logic is resolved at compile time, making a single kernel template cover all numeric type pairs
