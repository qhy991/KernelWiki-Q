## Solution

Fuse RoPE computation and KV-cache write into a single kernel: compute RoPE on k_pe and immediately write the result directly to the paged KV cache in the same thread, eliminating the intermediate global-memory write/read cycle and kernel launch overhead

## Key Lessons

- Fuse kernels with a producer-consumer relationship via global memory into a single kernel to eliminate redundant global-memory round-trips — applies whenever one kernel's output is immediately consumed by the next kernel without intermediate transformation
- Kernel fusion amortizes launch overhead over more total work per block, especially beneficial when individual kernels are lightweight relative to launch cost or when the fused kernel has larger thread count for the dominant work section
