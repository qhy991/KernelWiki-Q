## Solution

Launch max(num_experts, WARP_SIZE)=32 threads to fill the entire warp, halving per-thread token workload from 1024 to 512; adjust shared memory layout from (num_experts+1)*num_experts to (blockDim.x+1)*num_experts and add if(threadIdx.x < num_experts) guards to prevent excess threads from writing out-of-bounds expert rows

## Key Lessons

- Launch max(num_experts, WARP_SIZE) threads instead of num_experts threads in single-block kernels to eliminate partial-warp scheduling penalty and distribute work across all 32 lanes — applies whenever num_experts < 32 and per-thread work is data-parallel
- When padding thread count beyond num_experts, guard expert-indexed shared memory writes with if(threadIdx.x < num_experts) to prevent excess threads from accessing out-of-bounds rows in the expert-count matrix
- Size shared memory dynamically using blockDim.x rather than a compile-time constant so the layout scales correctly when the launch thread count differs from num_experts
