## Key Lessons

- The wider M tile (64 vs 32) increases sA shared memory to 64x32=8KB, while the narrower N tile reduces sB reuse per computation cycle (only 16 N columns per tile instead of 32).
- The after version uses TILE_M=64 TILE_N=16 with blockDim(16,64), which changes the loading pattern: for sA, blockDim.x=16 < TILE_K=32 forces each thread to loop twice to cover all K elements per row; for sB, blockDim.y=64 > TILE_K=32 means threads with threadIdx.y >= 32 skip sB loading.
