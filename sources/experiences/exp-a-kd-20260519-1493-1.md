## Challenge

Static shared memory arrays sized for worst-case NUM_MAX_EXPERTS=64 allocate 16900 bytes regardless of the runtime num_experts parameter, wasting shared memory and preventing the kernel from adapting to smaller expert counts


## Solution

Replace static __shared__ 2D arrays with extern __shared__ dynamic allocation sized to (num_experts+1)*num_experts at runtime (360 bytes for 8 experts vs 16900 bytes static), using a flat index(total_col, row, col) helper to compute offsets into the contiguous buffer

## Key Lessons

- Use extern __shared__ with runtime-sized allocation instead of static __shared__ arrays sized for worst-case maximums — reduces shared memory from 16900 to 360 bytes for 8 experts, though occupancy benefits only materialize when multiple thread blocks can run concurrently
- Dynamic shared memory requires manual flat index computation (row * cols + col) replacing compile-time 2D array strides — the added multiply per access introduces overhead that can offset memory savings when the kernel is single-block and memory-bound rather than compute-bound
- Reducing shared memory footprint without increasing concurrent blocks (e.g., single-block <<<1,8>>> launch) yields minimal performance gain — the optimization is structurally correct but the execution configuration prevents occupancy improvement
