## Challenge

Per-token-group int8 quantization kernel uses scalar half loads (one element per load instruction) in both the absmax reduction loop and the quantize loop, underutilizing available memory bandwidth and requiring 64 threads per group to cover 64 elements


## Solution

Replace scalar half loads with 16-byte vectorized loads via a templated vec_t struct (vec_size=8 for half), processing 8 elements per load and reducing threads per group from 64 to 16, which enables coalesced 16-byte memory transactions and frees threads for multi-group block packing

## Key Lessons

- Use 16-byte vectorized loads (vec_t<T, 16/sizeof(T)>) instead of scalar loads for half-precision data to maximize memory throughput — each load fetches 8 half elements in a single coalesced transaction, reducing total load instructions by 8x
- Vectorized loads reduce the required threads per group (64 elements / 8 per vector = 8 iterations, 16 threads cover this with stride), which frees threads for multi-group block packing and higher occupancy
