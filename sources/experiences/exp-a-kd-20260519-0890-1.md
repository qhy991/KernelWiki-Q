## Challenge

Naive RMSNorm kernel reads the input tensor x from global memory twice — once in Pass 1 for sum-of-squares accumulation and again in Pass 2 for output computation — wasting 8KB of fp16 memory bandwidth per row for hidden size 8192


## Solution

Cache each thread's fp32 input elements in a register array xi_cache[16] during Pass 1 and reuse them in Pass 2, eliminating the redundant global memory read — each thread stores only 64 bytes of fp32 registers (16 elements), which fits comfortably in the register file without spilling

## Key Lessons

- Cache input elements in per-thread register arrays when a kernel needs multiple passes over the same data — eliminates redundant global memory loads and the savings scale with hidden size since the register cost per thread grows linearly while global bandwidth savings grow at the same rate
- For memory-bandwidth-bound kernels like RMSNorm on large hidden sizes (8192+), the L2 cache cannot effectively buffer input rows across many concurrent tokens, so register caching is strictly better than relying on L2 to serve the second read
- #pragma unroll on the per-thread loop enables the compiler to fully unroll the fixed-trip-count loop and keep xi_cache entries in registers rather than spilling to local memory — essential when the per-thread element count is small (≤16)
