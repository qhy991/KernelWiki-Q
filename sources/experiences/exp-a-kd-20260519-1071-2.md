## Solution

Eliminate shared-memory BlockReduce entirely by assigning each thread an entire row (VPT=16=NUM_EXPERTS), so max and sum reductions become thread-local register loops; for wider rows where THREADS_PER_ROW>1, use __shfl_xor_sync warp shuffle reduction instead of shared-memory BlockReduce to avoid TempStorage allocation and __syncthreads barriers

## Key Lessons

- When row width is small enough for one thread to own the entire row (VPT=NUM_COLS), perform max/sum reductions as thread-local register loops — eliminates shared-memory TempStorage, bank conflicts, and __syncthreads() barriers entirely
- When THREADS_PER_ROW>1 and cross-thread reduction is needed, prefer __shfl_xor_sync warp shuffle over shared-memory BlockReduce — shuffle operates purely on registers without shared-memory intermediaries, and the parameterized THREADS_PER_ROW width keeps reduction scope minimal
