## Solution

Replace scalar per-thread bf16 loads and uint8 stores with int4 (128-bit) vectorized loads of 8 bf16 elements per thread and uint64_t packed stores of 8 fp8 outputs, reducing thread count from 576 to 96 while achieving 8x wider memory transactions that fully utilize available memory bandwidth

## Key Lessons

- Use int4 (128-bit) vectorized loads to read 8 bf16 elements per thread instead of scalar loads — wider memory transactions saturate available memory bandwidth and reduce total load instructions by 8x
- Pack multiple fp8 quantized outputs into a single uint64_t aligned store instead of writing individual bytes — contiguous 8-byte stores are coalesced into efficient 128-bit memory transactions
- Reducing thread count (576→96) while increasing per-thread work (1→8 elements) raises concurrent blocks per SM, improving occupancy when each block's register/shared-memory footprint allows more blocks to coexist
