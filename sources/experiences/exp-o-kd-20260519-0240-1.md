## Challenge

Element-wise logical AND reduction on packed 1-bit booleans extracts each bit individually via byte-index, bit-shift, and mask (128 iterations with per-element LogicalAnd1b functor calls), incurring massive instruction overhead from redundant extraction and narrow per-bit operations


## Solution

Reinterpret the packed storage as raw bytes and scan only ceil(N/8)=16 bytes, checking if any byte equals 0x00 (all 8 bits false), reducing loop iterations 8x and eliminating all bit-shift/mask extraction overhead while using wider byte-level memory accesses

## Key Lessons

- For packed sub-byte types (1-bit, 2-bit, 4-bit), operate on the underlying storage granularity (bytes/words) instead of extracting individual elements — reduces loop iterations by the packing factor and eliminates per-element bit manipulation overhead
- When the reduction operator is logical AND on packed booleans, the entire group reduces to a single byte-level zero-check: any 0x00 byte means the AND over those 8 bits is false, enabling batch processing 8 elements per iteration
- Short-circuit accumulation (OR of zero-checks) allows early exit when any false group is found — useful for divergent data though unrolled loops always complete all iterations
