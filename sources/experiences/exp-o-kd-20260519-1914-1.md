## Solution

Add m_block_size_8 boolean parameter that computes tb_m = thread_m_blocks * 8 when MoE block size is 8, halving the M-dimension tile size and reducing shared memory for A matrix, reduction, and metadata buffers, allowing more concurrent thread blocks per SM

## Key Lessons

- Make tile dimensions configurable based on actual problem size (e.g., MoE block size) rather than using a fixed maximum — smaller tiles reduce shared memory footprint and can unlock higher SM occupancy when the workload does not require full tile capacity
- Conditionally halving the tile dimension (16→8) for small batch scenarios cuts shared memory by roughly half for A and metadata buffers without affecting large-batch paths, as the fallback preserves the original tile size
