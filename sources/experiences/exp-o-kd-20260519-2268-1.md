## Challenge

MoE expert token counting uses a 2D shared-memory tokens_cnts array with per-thread rows and a serial thread-row prefix sum, consuming ~66KB shared memory for 128 experts and limiting parallelism to only 128 threads


## Solution

Replace the 2D per-thread tokens_cnts array with a flat shared_counts array updated via atomicAdd, enabling 1024 threads to count tokens in parallel and reducing shared memory from ~66KB to ~5KB

## Key Lessons

- Replace 2D per-thread count arrays with atomicAdd into a flat shared array to scale thread count beyond the expert dimension — eliminates the serial thread-row prefix sum and reduces shared memory from O(threads*experts) to O(experts)
- Pad shared memory arrays to warp boundaries (padded_num_experts) to avoid bank conflicts when threads access expert counts in parallel
