## Solution

Use a 2D thread grid where each thread computes one output element C[i][j] via a sequential dot-product loop over the K dimension, with column-major indexing (row + col*ld) and final alpha/beta scaling

## Key Lessons

- Column-major GEMM indexing follows row + col*ld for all matrix accesses — A[i + k*lda] picks element (i,k), B[k + j*ldb] picks element (k,j), and C[i + j*ldc] writes element (i,j)
- A reference GEMM with one-thread-per-element and a sequential K-loop is the standard correctness baseline — it directly mirrors the mathematical definition of matrix multiplication and is trivially verifiable
- Bounds checking (i < M && j < N) is essential when the grid dimensions are rounded up to block multiples — threads beyond the matrix edges must be masked to avoid out-of-bounds writes
