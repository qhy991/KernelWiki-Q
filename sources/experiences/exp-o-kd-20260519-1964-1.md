## Solution

Fuse FP8 quantization into the merge kernel by adding a compile-time USE_FP8_OUTPUT template parameter with if constexpr dispatch: the merged float result is directly scaled and cast to FP8 via float_to_fp8e4 instead of writing BF16, eliminating the intermediate buffer and the second kernel launch entirely

## Key Lessons

- Fuse a quantize/cast kernel into the producer kernel when the producer's output is immediately consumed by quantization — eliminates one kernel launch and a full DRAM round-trip through an intermediate buffer, yielding 1.28x speedup even for small tensors
- Use compile-time bool template parameter with if constexpr to support multiple output precisions in one kernel template — zero overhead for the unused path, avoids code duplication
- Use std::conditional_t to select pack types (uint4 for BF16, uint2 for FP8) matching the output element width — ensures correct vectorized store alignment when output type differs from input type
