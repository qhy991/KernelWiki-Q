## Solution

Replace scalar per-element loads and stores with flashinfer::vec_t vectorized 128-bit (16-byte) loads via cast_load/cast_store, processing 8 bf16 elements per transaction and reducing total memory transactions by ~8x

## Key Lessons

- Use flashinfer::vec_t with cast_load/cast_store to coalesce 8 scalar bf16 loads/stores into a single 128-bit vector transaction — applies whenever accessing contiguous bf16 elements in memory-bound kernels
- Compute vec_size as 16/sizeof(scalar_t) and iterate by vector index instead of element index to naturally handle alignment while maximizing per-transaction bandwidth
