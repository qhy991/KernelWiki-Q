## Solution

Fuse the BF16-to-float conversion directly into the softmax kernel by replacing float-pointer reads with inline __bfloat162float calls in each loop pass, eliminating the intermediate buffer entirely and reducing the pipeline from two kernel launches to one

## Key Lessons

- Fuse type-conversion kernels into compute kernels when the conversion is lightweight (single __bfloat162float instruction) — the eliminated intermediate buffer round-trip to global memory dominates the cost, saving 2× memory traffic for the input path
- Kernel fusion eliminates one full kernel launch overhead (grid setup, block scheduling, context switch) in addition to reducing memory traffic — the benefit grows with data size since the intermediate buffer scales linearly
- When each thread already iterates over multiple elements in a stride loop (64 experts / 256 threads = ~0.25 elements, but with stride), adding an inline conversion per element adds negligible register pressure compared to the global memory savings from removing the intermediate buffer
