## Solution

Replace FP32 accumulation MMA with FP16 accumulation (mma.sync.f16 variant) to halve accumulator registers from 32 to 16, then convert fp16 accumulators back to fp32 via __half2float in a backward iteration to avoid overwriting unread data

## Key Lessons

- Reducing MMA accumulator registers (fp32 to fp16) does not improve performance on SM80 when tensor core throughput is the bottleneck rather than register pressure — the m16n8k16 fp32 and fp16 variants have similar hardware throughput on Ampere
- Register count reduction only translates to occupancy gains when the kernel is register-bound near the SM limit; with 4 warps and 8 accumulator elements on a 65536-register SM, both variants fit comfortably so occupancy is unchanged
- Post-MMA fp16-to-fp32 conversion overhead (reinterpret_cast + __half2float per element) can fully offset any minor MMA-loop gains from reduced register footprint — always account for conversion cost when evaluating accumulation precision changes
