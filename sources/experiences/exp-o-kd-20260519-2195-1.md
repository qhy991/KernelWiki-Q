## Solution

Replace the multi-thread read-modify-write self_counter with a compact _flag[kMaxBlocks] array where only thread 0 writes after __syncthreads(), and split the dual-phase peer_counter[2] into separate start/end arrays to eliminate modulo-2 indexing, reducing global memory store traffic from O(ngpus) read-modify-writes to one read + one conditional write per barrier

## Key Lessons

- Replace multi-thread read-modify-write on cache-line-aligned counter arrays with a single compact _flag array where only thread 0 writes after __syncthreads() — eliminates false-sharing L2 write-back traffic and reduces global store count from O(threads) to 1 — applies to barrier synchronization where only arrival detection matters, not per-thread ordering
- Split dual-phase barrier state (modulo-2 alternation via array[2]) into separate start/end arrays — eliminates the modulo index computation and halves the per-phase working set size — applies to two-phase barrier patterns where start and end synchronization never overlap
