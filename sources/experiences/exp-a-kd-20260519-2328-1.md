## Solution

Replace atomicAdd with if (masks[i]) conditional direct assignment to avoid atomic operation cost, but this introduces per-iteration warp divergence across ~24 strided iterations and extra global memory reads for masks[i], causing a 35% regression that outweighs the atomic pipeline overhead

## Key Lessons

- Uniform atomicAdd with no warp divergence can outperform conditional direct writes when the branch condition varies across threads (50% mask rate) — the divergence cost across many strided iterations dominates the atomic serialization overhead
- Fusing a mask into an arithmetic expression (masks[i] * value) avoids an explicit branch and an extra conditional global memory read compared to if (masks[i]), keeping the warp execution path uniform
- Replacing atomics with conditional writes is not universally beneficial: when divergence is high and iteration count is large, the atomic approach's uniform execution and fused arithmetic can be faster despite per-address serialization
