## Challenge

MoE top-K selection and weight renormalization are performed by two separate kernel launches (moeTopK + renormalize_kernel), incurring kernel launch overhead (~5-10us per launch) and an extra global memory round-trip to read and rewrite all num_tokens*topk weight values


## Solution

Fuse renormalization into moeTopK by accumulating row_sum_for_renormalize during the top-K selection loop and dividing each weight by this sum in a post-loop block executed by thread 0, eliminating the separate renormalize kernel launch and its global memory round-trip

## Key Lessons

- Fusing a lightweight post-processing step (e.g., renormalization) into the producer kernel eliminates kernel launch overhead and an extra global memory round-trip — applies when the post-processing uses data already computed or written by the same thread in the same kernel
- Accumulating a sum during the main computation loop and applying a single inverse multiplication at the end keeps the data cache-friendly in L1/L2 and avoids a separate reduction-style kernel that must re-read values from global memory
