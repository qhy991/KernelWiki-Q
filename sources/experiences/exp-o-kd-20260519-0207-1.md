## Challenge

CausalMask<false> trip count adds offset_q = N_K - N_Q into the ceiling division, causing a single Q-block (N_Q=64) to iterate over all 128 K-blocks (N_K=8192) instead of only the 1 block it actually needs under causal semantics


## Solution

Remove offset_q from the trip count formula to use CausalMask<true> semantics, computing max_blocks_q = ceil_div((blk_coord_q+1)*tile_q, tile_k) directly — for the extreme-asymmetry case (1 Q-tile vs 128 K-tiles) this reduces trip count from 128 to 1, eliminating 99.2% of wasted computation

## Key Lessons

- Removing offset_q from the causal mask trip count avoids visiting K-blocks that lie outside the causal window — critical when N_Q << N_K because the offset inflates trip count by roughly N_K/tile_K factor
- CausalMask<true> (IsQBegin=true) semantics that treat the first Q-tile as aligned to the start of the KV cache are correct for decode-phase attention where the single query token should only attend to its own causal window, not the full prefill-length cache
