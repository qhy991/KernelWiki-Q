## Challenge

MOE block-alignment kernel uses a static 2D shared memory array (__shared__ int32_t shared_counts[WARP_SIZE][8]) with hardcoded experts_per_warp=8, limiting the kernel to DeepSeek V3's exact 256-expert/32-warp configuration without flexibility for other expert counts


## Solution

Replace static 2D shared memory with extern dynamically-allocated 1D shared memory and parameterize experts_per_warp as a runtime kernel argument, using flat indexing (warp_id * experts_per_warp + i) that produces identical byte offsets and bank mappings as the original 2D row-major layout — nvcc folds the compile-time constant and equal runtime value into identical PTX

## Key Lessons

- extern __shared__ with runtime-sized 1D flat indexing is performance-neutral versus static 2D shared memory arrays when the stride is identical — nvcc generates equivalent PTX address computation for both forms when the runtime stride equals the compile-time constant
- Row-major 2D access arr[row][col] and flat 1D access arr[row*stride+col] produce identical byte offsets and shared memory bank mappings (bank = flat_index % 32), so converting between them preserves bank conflict behavior exactly
- Dynamic shared memory allocation via extern __shared__ with a runtime size parameter enables kernel generalization to different expert counts without any performance penalty, since the hardware sees the same addresses and bank access patterns
