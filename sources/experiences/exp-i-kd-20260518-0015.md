## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Target: SM 100 (Blackwell); Shared memory: 128KB per CTA; Use TMA (Tensor Memory Access) for K/V loading

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Target: SM 100 (Blackwell); Shared memory: 128KB per CTA
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
