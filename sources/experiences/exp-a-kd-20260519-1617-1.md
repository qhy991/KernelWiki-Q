## Challenge

The KV dequant kernel uses a single shared kv_scale for both key and value FP8 dequantization via fp8::scaled_convert, producing numerically incorrect results when K and V tensors have different dynamic ranges and require distinct per-tensor scale factors


## Solution

Split the shared kv_scale parameter into separate k_scale and v_scale per-tensor scale factors, applying each to its respective tensor during FP8 dequantization so that K and V are independently and correctly rescaled

## Key Lessons

- Use separate per-tensor scale factors for K and V in FP8 dequantization rather than a shared kv_scale — each tensor has independent dynamic ranges and a compromise scale introduces systematic dequantization error in both
- Correctness fixes that only split a pass-by-value scalar parameter have negligible performance impact since the extra float register does not affect occupancy or instruction count for simple kernels
