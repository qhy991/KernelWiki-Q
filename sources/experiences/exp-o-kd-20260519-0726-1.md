## Solution

Fuse rope_only_kernel and kv_writeback_kernel into a single fused_rope_kv_kernel where K RoPE results (y1, y2) remain in registers after computation and are written directly to both k_rope and k_buf in the same kernel, eliminating the redundant global memory store-and-reload of K RoPE data

## Key Lessons

- Fuse producer-consumer kernel pairs when the consumer reads data that the producer just wrote to global memory — keeping values in registers across both operations eliminates a full global memory round-trip, which is especially valuable for memory-bound workloads like RoPE and KV cache operations
- Kernel fusion eliminates not only launch overhead but also redundant global memory traffic; the traffic savings dominate when the data volume per token is large (e.g., KV_HEADS × HEAD_DIM × sizeof(half) per token) and the computation is memory-bound
