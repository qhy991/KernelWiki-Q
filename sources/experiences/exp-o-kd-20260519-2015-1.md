## Challenge

Grid-stride scalar loop computes three integer division/modulo operations per element to decompose a flat index into (token, head, dim), causing high arithmetic overhead and preventing contiguous memory access patterns


## Solution

Assign one warp to each (token, head) pair so the base pointer is computed once per warp using a single division/modulo, eliminating per-element index decomposition and enabling each lane to handle a contiguous chunk of the dimension

## Key Lessons

- Assign one warp per logical entity (token-head pair) instead of one thread per element — the warp computes the base pointer once via a single division/modulo, then each lane processes a contiguous chunk, eliminating redundant per-element index decomposition
- Prefer warp-level (>>5 for warp_id, &31 for lane_id) over thread-level indexing when the work unit maps naturally to a multi-element group with a shared base address
