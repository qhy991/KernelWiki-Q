## Challenge

Flash attention on Apple M1 Pro Metal with seq_len=8192, num_heads=32, head_dim=128

## Solution

Two-pass softmax with one thread per output element, simple sequential loops over seq_len=8192

## Key Lessons

- Metal has limited registers; large local arrays like scores[8192] cause severe register spilling and slowdown to 1000ms+
- Threadgroup reduction across wrong thread boundaries breaks softmax correctness (NMSE=1.0)
- SIMD group reduction requires proper simd_* primitives and data distribution across lanes
- SIMD reductions like simd_max(a, a) with same value don't reduce across lanes - need proper data distribution
- Threadgroup memory allocation for unused arrays still consumes registers and causes spilling
- Two-pass softmax (find max first, then compute softmax) is the only correct approach so far for Metal flash_attention
- Best achieved latency: ~278ms with simple two-pass approach using one thread per output element
