## Challenge

Issuing 128 individual cudaMemcpy calls of 8KB each for page-table-driven memory copy incurs ~2-3μs driver dispatch and synchronization overhead per call, which dominates the actual 8KB data transfer time on the GPU internal bus


## Solution

Scan the index arrays for consecutive runs where both src_diff==1 and dst_diff==1, then merge each run into a single cudaMemcpy call covering all consecutive pages at once, reducing 128 calls to 2 calls of 512KB each

## Key Lessons

- Merge contiguous cudaMemcpy calls by detecting consecutive index runs — eliminates per-call driver dispatch overhead (~2-3μs each) and applies whenever page-table index arrays contain adjacent pages
- Larger contiguous transfers (512KB vs 8KB) achieve significantly better HBM bandwidth utilization because the GPU memory controller amortizes command setup over a larger payload
