## Solution

Fuse the two index_select kernels and the cache_write kernel into a single kernel that reads source data directly using selection indices and writes to the final KV cache slot in one pass, eliminating intermediate global memory buffers and reducing total global memory traffic from 2 source reads + 2 intermediate writes + 2 intermediate reads + 1 cache write to just 1 source read + 1 cache write per element

## Key Lessons

- Fusing a gather (index_select) with a scatter (cache_write) into a single kernel eliminates the intermediate global memory round-trip — applies whenever a pipeline has consecutive kernels where one kernel's output is consumed entirely by the next and never reused
- Kernel fusion reduces global memory traffic proportionally to the size of intermediate tensors: each eliminated intermediate buffer removes one write + one read, which dominates latency for memory-bound copy operations
- Use a lambda or inline helper to handle multiple source-to-destination copy patterns within a fused kernel, keeping per-tile logic parameterized rather than duplicated for each source tensor
