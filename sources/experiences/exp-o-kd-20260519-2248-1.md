## Solution

Reduce cluster shape from 4x4x1 (16 SMs) to 2x1x1 (2 SMs) to minimize inter-SM coordination overhead while maintaining sufficient parallelism for large-M workloads, and simplify the tile index mapping to a 1D cluster decomposition

## Key Lessons

- Reducing cluster shape from 4x4x1 to 2x1x1 for large-M GEMM kernels cuts inter-SM barrier synchronization overhead by 8x while preserving tile occupancy — applies when tile dimensions already provide sufficient parallelism and the cluster width exceeds the useful coordination range
- Simplify 2D cluster index mapping (cluster_id_x/y + local_id_x/y) to 1D (single cluster_id) when CLUSTER_N=1, eliminating integer modulo/division overhead in the index calculation path
