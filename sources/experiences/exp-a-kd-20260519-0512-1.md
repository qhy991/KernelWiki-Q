## Solution

Vectorize the kernel to process 2 half elements per thread using half2 packed loads/stores via reinterpret_cast, halving thread count while each thread does double the work

## Key Lessons

- Explicit half2 vectorization for elementwise activation kernels provides negligible benefit when the GPU memory subsystem already coalesces adjacent scalar half accesses from neighboring threads into full-width transactions — the hardware-level coalescing already achieves the same memory throughput gain
- half2 vectorization is most impactful when it enables half2 arithmetic intrinsics (__hadd2, __hmul2) that the compiler cannot auto-vectorize from scalar code, not merely for memory access packing alone
