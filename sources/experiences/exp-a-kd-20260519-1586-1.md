## Solution

Template the kernel on dst_t and introduce convert_from_half<dst_t> — an identity function for half and a half-to-bfloat16 conversion for __nv_bfloat16 — enabling a single kernel source to support multiple output types with zero overhead for the original fp16 path

## Key Lessons

- Template-based output type dispatch via convert_from_half<dst_t> adds multi-dtype support with zero runtime cost — the compiler inlines the identity specialization to the same PTX as the original hardcoded half path
- Use explicit template specializations (identity for same-type, conversion for cross-type) to keep per-dtype logic clean and let the optimizer eliminate unused branches at compile time
