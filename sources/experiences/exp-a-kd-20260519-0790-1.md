## Solution

Fuse both normalize and scale operations into a single fused_rescale_kernel where each thread performs division by row sum followed immediately by multiplication with the scaling factor on the same register-held value, eliminating one kernel launch and one entire global memory read-write round-trip per element

## Key Lessons

- Fuse sequential elementwise kernels that operate on the same data into a single kernel to eliminate redundant global memory round-trips and kernel launch dispatch overhead — applies whenever kernel B reads what kernel A just wrote
- Keep intermediate results in registers when performing back-to-back arithmetic on the same element — register-held values avoid a global memory write-then-read cycle that doubles memory traffic for no computational benefit
