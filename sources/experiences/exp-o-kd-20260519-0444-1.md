## Solution

Replace scalar f32 loads with vectorized float4 loads using reinterpret_cast<float4*>, enabling each thread to process 4 elements per iteration and coalescing 4 scalar memory transactions into 1 vector transaction for 4x fewer total loads

## Key Lessons

- Vectorized float4 loads coalesce 4 scalar memory transactions into 1 vector transaction, reducing total memory traffic by 4x and improving bandwidth utilization — applies to any bandwidth-bound inner loop with contiguous, 16-byte-aligned access patterns
- Vectorization can regress when the working set exceeds L2 cache (observed at M=2048/K=2048 due to cache thrashing) — evaluate problem dimensions before applying; vectorized loads benefit most when K is moderate enough that the accessed rows fit in cache
