## Solution

After code transposes operands (B^T as A, A^T as B) and calls with shape (N,M,K) instead of (M,N,K), intending to improve tile scheduling — but with a naive one-element-per-thread kernel this swap introduces strided column-major access on B^T, degrading coalescing and causing 2.66x slowdown

## Key Lessons

- swap_ab (transposing operand pointers and problem shape) is a CUTLASS 3.x TMA-specific optimization for SM90 tile schedulers — applying it to naive one-element-per-thread kernels on SM80 or earlier degrades performance by converting coalesced row-major accesses into strided column-major accesses that thrash the cache
- Memory coalescing dominates naive GEMM performance: the original (M=8,N=8192) shape gives grid.y=8 where each warp reads contiguous A rows and coalesced B columns, while the swapped (M=8192,N=8) shape forces 8192 thread block rows to stride across B^T with pitch K=4096, destroying spatial locality
