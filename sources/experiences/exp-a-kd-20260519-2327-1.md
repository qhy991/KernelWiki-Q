## Challenge

MoE sorted_token_ids writing uses atomicAdd for every token including mask=0 entries where atomicAdd(&x, 0) is a data no-op but still exercises the atomic instruction pipeline, adding per-element serialization overhead


## Solution

Replace unconditional atomicAdd with a conditional branch (if masks[i]) that skips mask=0 tokens and uses direct assignment for mask=1 tokens, eliminating all atomic operations from the write path

## Key Lessons

- Conditional branches on ~50% mask rates cause warp divergence whose serialization cost exceeds that of uncontended atomicAdd on sm_80+ hardware, because unique-address atomics are resolved efficiently in L2 cache — prefer uniform atomicAdd over divergent conditional writes when mask rates are near 50% and write addresses are unique
- atomicAdd(&x, 0) is not free but is cheaper than warp divergence when atomics target unique non-overlapping addresses, since sm_80 L2 cache handles non-conflicting atomics without DRAM round-trips
