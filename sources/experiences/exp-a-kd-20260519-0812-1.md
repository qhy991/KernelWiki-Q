## Solution

Replace __shared__ float scale with const float scale, moving the variable into per-thread registers so each warp independently holds its own scale value computed from its own warp_max, eliminating the inter-warp race condition without requiring __syncthreads()

## Key Lessons

- When each warp computes an independent per-token value (e.g., scale from warp_max), use register-local const variables instead of __shared__ memory — shared memory requires explicit synchronization (__syncthreads()) and introduces race conditions when multiple warps write the same location
- __shared__ memory is only appropriate when warps within a CTA must exchange data; for per-warp-private scalars, registers are faster, correct without synchronization, and avoid false-sharing hazards
- Race conditions on __shared__ variables can pass correctness tests under specific warp scheduling orders (e.g., A800 sm_80 sequential warp dispatch) but are provably undefined behavior — correctness validation alone cannot prove absence of data races
