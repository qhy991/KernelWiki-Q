## Solution

Replace per-element scalar float loads/stores with uint4 (128-bit) packed loads/stores so each thread processes 4 contiguous float values in a single 16-byte memory transaction, reducing total load/store instruction count by 4× and improving L2 cache line utilization for the memory-bound merge

## Key Lessons

- Use uint4 (128-bit) packed load/store to consolidate 4 scalar float memory transactions into one 16-byte transaction in memory-bound kernels — reduces load/store instruction overhead and improves L2 cache line utilization when accessing contiguous float arrays
- Adapt the threading granularity when vectorizing: reduce threads-per-head by the pack factor (head_size / pack_size) so each thread covers pack_size contiguous elements, keeping total work constant while improving memory throughput
