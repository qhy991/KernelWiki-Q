## Challenge

RoPE kernel uses scalar float loads and stores (two 32-bit transactions per element pair), causing high memory transaction overhead and underutilizing available memory bandwidth at large sequence lengths


## Solution

Replace scalar float2 loads/stores with float4 vectorized 128-bit transactions, processing two consecutive rotation pairs per thread, which halves the number of memory transactions and maximizes bandwidth utilization under memory-bound conditions

## Key Lessons

- Use float4 vectorized loads/stores (128-bit transactions) instead of scalar float operations in memory-bound kernels — this halves the number of memory transactions and doubles effective bandwidth utilization, especially beneficial at large data volumes where memory pressure is highest
- When vectorizing RoPE from float2 to float4, each thread processes two consecutive rotation pairs requiring two separate angle computations (exp_f_v and exp_s_v) since adjacent pairs use different frequency indices — the angle index mapping must be adjusted from idx to idx*2 and idx*2+1 respectively
