## Challenge

MoE gating softmax kernel uses scalar per-element strided loads across three separate global-memory passes (max-finding, exp-sum, normalize-write), reading each row element three times from global memory; with 256 threads and only 64 elements per row, 192 threads are completely idle per iteration


## Solution

Replace scalar strided loads with vectorized packed loads (VPT=4, BYTES_PER_LDG=16) using an interleaved access pattern where 16 threads each load 4 floats in one 16-byte coalesced transaction; data stays in registers across all three computation phases (max, exp-sum, normalize), and CUB BlockReduce is replaced by warp butterfly __shfl_xor_sync reduction — reducing global memory traffic from 3 reads + 1 write per element to 1 vectorized read + 1 vectorized write

## Key Lessons

- For small power-of-2 row sizes (e.g., 64 elements), use vectorized packed loads (VPT elements per thread) with an interleaved access pattern so THREADS_PER_ROW threads cooperatively load the entire row in coalesced 16-byte transactions — eliminates idle threads and maximizes memory throughput
- Keep row data in registers across all computation phases (max, exp-sum, normalize) instead of re-reading from global memory in separate passes — reduces global memory traffic from 3 reads + 1 write to 1 vectorized read + 1 vectorized write per element
- Replace CUB BlockReduce + shared memory + __syncthreads with warp butterfly __shfl_xor_sync reduction when THREADS_PER_ROW <= WARP_SIZE — eliminates shared memory bank conflicts and synchronization barriers for intra-warp reductions
