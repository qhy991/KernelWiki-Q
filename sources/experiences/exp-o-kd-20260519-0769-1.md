## Solution

Fuse the routed_scaling_factor multiplication into the existing rescale loop by adding a conditional multiply inside the same loop iteration, so each element is divided by output_sum and then multiplied by routed_scaling_factor in a single pass, eliminating the separate scaling kernel

## Key Lessons

- Fusing a second elementwise operation (scaling) into an existing rescale loop eliminates a separate kernel launch and an extra global-memory traversal — applies whenever two independent per-element transforms can be composed into one pass
- Guard fused operations behind a runtime flag (apply_routed_scaling_factor_on_output) so the same kernel handles both gated and non-gated MoE variants without divergence in the common case
