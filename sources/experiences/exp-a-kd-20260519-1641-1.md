## Solution

Introduce typed ld128/st128 template wrappers around __ldg on int4* and a PackedVec struct with .elts[] member access, replacing raw reinterpret_cast chains — the wrappers add type safety and explicit intent while the compiler generates identical ld.global.nc.v4.u32 and st.global.v4.u32 PTX instructions

## Key Lessons

- Typed ld128/st128 wrappers around __ldg + int4 reinterpret_cast are a zero-cost abstraction — NVCC generates identical PTX (ld.global.nc.v4.u32 / st.global.v4.u32) because the wrappers are __forceinline__ with static_assert size checks that resolve at compile time
- A PackedVec struct with .elts[] array member provides named access to packed elements and is equivalent in generated code to raw reinterpret_cast<bf162*>(&vec) — prefer it for readability and type safety without performance penalty
- When comparing before/after kernels that only differ in load/store wrapper style (raw intrinsics vs typed templates), expect neutral performance; verify by checking that PTX instruction sequences are identical rather than benchmarking alone
