## Challenge

MoE fused gate kernel loads 384 expert input/bias values per row using scalar stride accesses (one 32-bit float per iteration with WARP_SIZE stride), issuing 12 separate 32-bit LDG instructions per lane and underutilizing available global memory bandwidth


## Solution

Replace scalar stride loads with float4 vectorized loads via reinterpret_cast<float4*>: each lane loads 3 float4 vectors (VEC_PER_LANE=3, covering VPT=12 elements), issuing 3 coalesced 128-bit LDG.128 transactions instead of 12 separate 32-bit loads, with #pragma unroll on both outer and inner loops to let the compiler schedule instructions efficiently

## Key Lessons

- Replace scalar stride loads (one element per iteration with WARP_SIZE stride) with float4 reinterpret_cast loads when consecutive lanes access contiguous memory — reduces LDG instruction count by 4x and improves memory coalescing by issuing 128-bit transactions instead of 32-bit
- Use VPT/VEC_SIZE to compute VEC_PER_LANE so each lane loads its assigned chunk via lane_id * VEC_PER_LANE + i indexing, naturally mapping adjacent lanes to adjacent 128-byte segments for full coalescing
- Apply #pragma unroll on both the outer vec-per-lane loop and the inner float4 processing loop to eliminate branch overhead and allow the compiler to schedule multiple LDG.128 instructions with independent address computation
