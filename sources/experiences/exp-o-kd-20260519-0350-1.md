## Solution

Replace the two-pass atomic-accumulate-and-reload pattern with a fused computeDi function that reads grad_output and output via alignment-aware vectorized cutlass::Array loads, accumulates Di = sum(do_i * o_i) in registers, and finalizes with __shfl_xor_sync warp reduction before writing to shared memory — eliminating global atomics and the extra tmp_sum_i tensor entirely

## Key Lessons

- Replace global-memory atomic accumulation with register-local dot products and warp shuffle reduction when the reduction result is consumed within the same kernel — eliminates atomic contention and removes the need for an intermediate global tensor
- Use alignment-aware vector load width (128-bit / sizeof(element)) with a scalar fallback path to guarantee wide memory throughput when head_dim is aligned while remaining correct for odd dimensions
- Fusing the Di = sum(do_i * o_i) computation directly from the output and grad_output tensors avoids an extra global store-load round trip and reduces the backward pass from two kernel phases to one
