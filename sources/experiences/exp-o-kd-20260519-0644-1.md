## Solution

Switch LSE layout to row-major [num_tokens, num_heads] with access token_idx * num_heads + head_idx, so adjacent threads within the same token read consecutive LSE entries and achieve coalesced 128-byte memory transactions

## Key Lessons

- Align tensor access index order with the thread iteration order (innermost loop variable) to achieve coalesced reads — when threads iterate token_idx in the inner dimension, index as token_idx * num_heads + head_idx so adjacent threads read contiguous memory
- Match auxiliary tensor layouts (LSE, biases) to the output tensor layout to eliminate transpose overhead and ensure all related accesses follow the same coalescing-friendly pattern
