## Challenge

Attention backward GradV/GradQ/GradK loops always perform runtime column-bound checks (col < p.head_dim_value) even when the head dimension is known at compile time to fit within a single GEMM tile, causing unnecessary loop overhead, prologue scheduling, and isLastColumn branch conditions


## Solution

Introduce constexpr bool kSingleIterationGradV = kMaxK <= MatmulGradV::ThreadblockShape::kN to make the loop bound a compile-time constant; when kMaxK fits in one tile the condition collapses to col < 1, allowing the compiler to eliminate the loop branch, prologue guard, and last-column checks entirely

## Key Lessons

- Replace runtime loop-bound conditions with constexpr predicates derived from compile-time tile dimensions — when the iteration count is provably 1 the compiler eliminates loop overhead, prologue checks, and epilogue branches entirely
- For GEMM-backed attention kernels where head_dim (kMaxK) is a compile-time constant, comparing it against ThreadblockShape::kN at compile time collapses multi-iteration loops into straight-line code for common small head dimensions (64, 128)
