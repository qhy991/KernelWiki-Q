## Challenge

Skeleton kernel needs a two-phase implementation to load bit-packed boolean predicates from unsigned integer words into a CUTLASS PredicateVector template, verify round-trip correctness of each predicate, and repack them back to output words after a synchronization barrier


## Solution

Implement two nested-loop phases: Phase 1 iterates over input words and extracts each bit via shift-and-mask into predicates.set(), immediately verifying with predicates.at(); after __syncthreads(), Phase 2 reads predicates back with predicates.at() and reconstructs each output word by OR-ing shifted boolean values together

## Key Lessons

- Compute word_count as (kPredicates + 31) / 32 to round up to full 32-bit words → ensures all predicates fit even when kPredicates is not a multiple of 32 → applies to any bit-packed boolean array mapping
- Immediate round-trip verification (set followed by at in the same loop) catches PredicateVector storage bugs at the point of insertion → early detection avoids silent corruption during the repack phase → useful in any test/validation kernel
- CUTLASS_PRAGMA_UNROLL on the inner 32-iteration bit loop lets the compiler fully unroll the bit extraction → critical for predicate processing where each iteration is independent and trivially small, removing loop overhead entirely
