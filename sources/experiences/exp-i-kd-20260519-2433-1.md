## Challenge

SM90 Hopper GPU provides bulk memory copy (TMA-like) transactions for efficient global-to-shared memory transfers, but requires proper barrier initialization, transaction byte sizing, and phase-bit synchronization to safely orchestrate the asynchronous copy pipeline


## Solution

Allocate aligned shared memory via SharedStorage, construct CuTe tensors for SMEM and GMEM, initialize a 64-bit shared memory barrier with transaction_bytes = size(sA)*sizeof(T), issue SM90_BULK_COPY_AUTO with the barrier, wait for phase-bit flip, then write output from SMEM to GMEM using a strided loop across threads

## Key Lessons

- SM90 bulk copy requires a three-step barrier pattern: initialize_barrier → set_barrier_transaction_bytes → copy with barrier — the transaction bytes must equal the exact byte count of the SMEM tensor to guarantee the barrier tracks the full transfer
- A single thread (threadIdx.x==0) issues the bulk copy operation with the barrier, then all threads wait on the barrier's phase bit flip to ensure completion — the barrier is initialized with numThreads=1 because only one thread participates in the copy issue
- SharedStorage structs with extern shared memory and reinterpret_cast provide the aligned SMEM pointer distribution needed by CuTe tensors and barrier operations — avoid raw pointer arithmetic for SMEM in CUTLASS/CuTe codepaths
