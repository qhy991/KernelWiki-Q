## Solution

Add a per-thread expert_ids lookup with early return when the token is assigned to no expert (-1), so masked tokens skip the expensive SiLU activation and multiply entirely; when the mask rate is 0%, all threads still pass the check, incurring only a small constant overhead from the additional global load and branch

## Key Lessons

- An early-return filter guard on a separate global-memory array (expert_ids) adds ~3% overhead even when no threads are masked, because the extra global load lives in a different cache line from the main data path and cannot coalesce with the primary load stream
- Per-thread filter branches that evaluate identically across all warps (uniform predicate) avoid warp divergence but still cost instruction slots — the overhead scales with the number of threads, not the number of divergent paths
- When adding conditional skip logic to a compute-heavy kernel, benchmark the zero-filter-hit case separately to quantify the constant overhead; the real benefit only appears when a significant fraction of tokens are masked and the saved compute outweighs the filter cost
