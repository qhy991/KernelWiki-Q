## Solution

Replace scalar __half loads with vectorized HalfVec8 (8×__half = 16 bytes) loads via vectorize_read_with_alignment_impl, reducing 8192 scalar memory transactions to 1024 wider transactions that better utilize the GPU's 16-byte load path

## Key Lessons

- Vectorized loads (HalfVec8 = 8×__half = 16 bytes) reduce global memory transaction count by VEC_SIZE×, yielding measurable bandwidth improvements when hidden_size is large enough for the memory savings to dominate over fixed kernel overhead
- Alignment-aware vectorized read utilities (vectorize_read_with_alignment_impl) handle both aligned fast-path and misaligned prefix/tail fallback, making vectorization safe to apply without guarantees on input alignment or length divisibility
