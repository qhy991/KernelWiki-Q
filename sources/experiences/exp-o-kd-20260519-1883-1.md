## Solution

Replace the serial single-thread prefix sum with CUB BlockScan parallel exclusive prefix sum across all 1024 threads, where each thread owns one expert slot and BlockScan::ExclusiveSum computes the cumulative sum in O(log N) parallel depth, eliminating the single-thread bottleneck

## Key Lessons

- Replace serial single-thread prefix sums with CUB BlockScan parallel scan to eliminate the single-thread bottleneck — even for moderate element counts (64), distributing scan computation across all block threads yields measurable speedup by utilizing otherwise-idle threads
- CUB BlockScan::ExclusiveSum naturally handles out-of-range threads by treating their zero-initialized input as a no-op contribution, eliminating the need for explicit boundary guards in the scan logic itself
- For prefix sums over moderate-sized arrays (32–256 elements) within a single block, parallel BlockScan is preferred over serial loops even when the serial loop is short — the parallel version avoids the warp-level serialization stall that occurs when only one thread is active
