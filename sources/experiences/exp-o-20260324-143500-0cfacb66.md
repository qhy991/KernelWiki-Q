fp32_flash_attention_qwen2_5_7b_f16_cache8192

## Solution

Increased V_CHUNK from 32 to 64, stored V in shared memory as half (not float) to save space (16KB vs 32KB), used float4 loads for V (8 halves per load) for better memory throughput

## Key Lessons

- Storing V in half precision in shared memory halves smem usage, allowing larger V_CHUNK (64 vs 32)
- Larger V_CHUNK reduces the number of sync barriers and improves shared memory reuse
- float4 loads for half data (8 halves per load) significantly improves memory throughput
- With BLOCK_SIZE=512 and V_CHUNK=64: 512 threads load 64*128/8=1024 float4 values, each thread loads 2 float4 = very efficient
- Total smem 18.7KB is well within A800's 48KB limit per block
