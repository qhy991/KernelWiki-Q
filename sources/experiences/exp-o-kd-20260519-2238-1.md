## Solution

Introduce an is_a_8bit flag to conditionally select 1 byte per A-matrix element for INT8 activations versus the default 2 bytes, halving shared memory footprint for the A tile from 16384 to 8192 bytes and freeing space for larger tile sizes or additional pipeline stages

## Key Lessons

- Size shared memory buffers based on the actual data type of the activation (1 byte for INT8 vs 2 bytes for FP16) rather than hardcoding the larger size — the per-tile savings compound across pipeline stages and enable larger tile configs within the shared memory budget
- Parameterize pipeline stage count through function arguments instead of global constants, allowing callers to tune the stages-vs-shared-memory tradeoff per kernel variant
