## Challenge

The fMHA kernel dispatch passes boolean flags (true/false) for kSingleValueIteration instead of concrete kMaxK upper bounds, so MakeCustomMma cannot cap its shared-memory pipeline stages to match actual head dimensions, wasting SMEM and limiting occupancy for small and medium heads


## Solution

Replace boolean kSingleValueIteration flags with concrete kMaxK upper bounds (128 for medium heads, 65536 for large heads, 64 for small heads) so MakeCustomMma computes kStages = ceil(kMaxK/kK) and reduces SMEM pipeline depth for smaller dimensions, increasing occupancy

## Key Lessons

- Pass concrete upper-bound constants (kMaxK) instead of boolean flags to CUTLASS MMA dispatch — enables the pipeline to compute tighter kStages = ceil(kMaxK/kK), reducing SMEM consumption per threadblock and raising occupancy for small dimensions
- Expose tunable constexpr constants (like kMaxK=64 with a comment suggesting 32/16) at the dispatch boundary so downstream tuning is visible without reading internal pipeline code
