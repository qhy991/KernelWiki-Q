## Solution

Add an is_a_8bit boolean parameter to conditionally select 1 byte per element for INT8 activations versus 2 bytes for FP16, correctly sizing the A-matrix shared memory buffer to match the actual data width

## Key Lessons

- When shared memory buffers store heterogeneous data types (INT8 activations vs FP16 weights), add a conditional element-size multiplier based on the activation type — INT8 needs only 1 byte, halving the A-matrix buffer compared to the default 2-byte FP16 assumption
- Shared memory over-allocation from incorrect element sizing compounds with pipeline stages: fixing both (stages 4→2 and elem_size 2→1) yields a 4x reduction in A-matrix shared memory alone
