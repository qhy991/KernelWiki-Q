## Challenge

Standard triple-chevron <<<grid, block, 0, stream>>> kernel launches enforce strict sequential execution of producer-consumer kernels on a stream, preventing any inter-kernel overlap or early-start opportunities even when the consumer could begin processing partial results


## Solution

Replace the triple-chevron launch with cudaLaunchKernelEx and set cudaLaunchAttributeProgrammaticStreamSerialization to enable Programmatic Dependent Launch (PDL), which allows the consumer kernel to begin execution before the producer fully completes when the hardware supports it

## Key Lessons

- cudaLaunchAttributeProgrammaticStreamSerialization (PDL) enables inter-kernel overlap on SM90+ (Hopper) GPUs, but the attribute is silently ignored on SM80 (Ampere) — always verify target architecture support before expecting performance gains from PDL
- PDL requires kernel-side cooperation via cudaGridDependencySynchronize() and cudaTriggerProgrammaticLaunchCompletion() guarded by __CUDA_ARCH__ >= 900 — these barriers are dead code on pre-SM90 hardware and produce no overhead
