## Solution

Introduce a runtime guard (K > 0) that branches between copying valid accumulator data from the buffer and zero-filling the register fragment, ensuring correctness when no MMA data exists and avoiding a global-memory load pipeline in the K=0 path

## Key Lessons

- Guard accumulator buffer reads with a K > 0 check — when the inner dimension is zero no MMA populates the accumulator, so zero-filling registers avoids reading garbage and is the mathematically correct identity element for accumulation
- Register-level zero-fill (reg[i] = 0.0f) is cheaper than a global-memory load pipeline from an uninitialized buffer, so the K=0 branch is both correct and faster — applies to any GEMM epilogue that must handle degenerate tile dimensions
- Always treat accumulator state as undefined when the corresponding MMA did not execute — defensive guards at epilogue boundaries prevent correctness bugs that only manifest under edge-case problem sizes
