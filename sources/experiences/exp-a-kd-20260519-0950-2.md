## Challenge

The token counting loop computes warp_idx and expert_offset via integer division and modulo (expert_id / experts_per_warp, expert_id % experts_per_warp) for every token, adding two expensive integer operations per atomicAdd in the hot counting path


## Solution

Use direct shared_counts[expert_id] indexing in the counting loop, eliminating the per-token div/mod computation and the two-level warp-based addressing scheme entirely

## Key Lessons

- Direct array indexing (shared_counts[expert_id]) eliminates integer div/mod in hot counting loops — prefer direct indexing when the index domain matches the array size, even if the original design grouped elements by warp
