## Challenge

KV cache gather kernel processes each element with immediate read-then-write in a single scalar loop iteration, preventing the GPU memory pipeline from batching independent reads and potentially underutilizing available memory-level parallelism


## Solution

Apply 4x loop unrolling with separated read/write phases: batch all 4 cache reads into local arrays before issuing any writes, aiming to increase memory-level parallelism by letting the hardware pipeline multiple outstanding loads

## Key Lessons

- Loop unrolling with local arrays for read/write phase separation can increase register pressure enough to reduce occupancy, negating the memory pipeline benefit — 8 local arrays (4 index + 2 data arrays × unroll_factor=4) consumed significant registers in this case
- When a kernel already has sufficient occupancy (256 blocks × many warps) to hide memory latency naturally, explicit read batching provides diminishing returns and may not offset its register cost
- Microsecond-scale kernels dominated by launch overhead and scheduling are poor candidates for fine-grained memory pipeline optimizations — the optimization signal falls within measurement noise (under 5% delta)
