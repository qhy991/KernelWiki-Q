## Challenge

NHWC channel-padding kernel processes one float element per thread using scalar loads and stores, requiring twice as many global memory transactions and loop iterations as necessary when both c_in and c_out are even


## Solution

Reinterpret pointers as float2 and halve the channel dimensions (c_in/2, c_out/2) so each thread loads/stores 8 bytes per iteration instead of 4, cutting total loop iterations and global memory transactions by 2x while keeping index decomposition arithmetic identical

## Key Lessons

- When channel dimensions are even, reinterpret float pointers as float2 and halve the logical dimensions — each thread processes 2 adjacent channels per iteration, doubling memory throughput without changing the index arithmetic structure
- Vectorization speedup is bounded by the fraction of time spent on memory I/O versus integer index computation; when heavy modulo/division arithmetic remains unchanged, the gain from wider loads/stores alone is moderate (~14%)
