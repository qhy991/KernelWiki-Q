## Solution

Use VEC_SIZE=8 (128-bit) vectorized loads and stores via vec_n_t<half,8> to transfer 8 half elements per memory transaction, with alignment-aware prefix/tail handling to safely process potentially misaligned addresses

## Key Lessons

- Use 128-bit vectorized loads/stores (8 half or 4 float elements) for contiguous memory copy kernels to increase effective memory bandwidth by up to 4x over scalar transactions — applies whenever source and destination addresses can be expressed as linear offsets from pre-computed base pointers
- Wrap vectorized access in an alignment-aware helper (check address alignment, process misaligned prefix scalar, then vector body, then scalar tail) to handle arbitrary pointer offsets without requiring callers to guarantee alignment
