## Solution

Assign one warp per (token, head) pair so each of the 32 lanes maps to a dimension offset via lane_id, eliminating all per-element integer division/modulo and enabling fully coalesced warp-level memory transactions

## Key Lessons

- Mapping one warp per logical work-item (token,head pair) instead of one thread per element eliminates per-element index arithmetic (division/modulo) and naturally aligns lane_id with contiguous memory offsets — applies whenever the inner dimension is a multiple of warp size
- Use flat_warp_id = (blockIdx.x * blockDim.x + threadIdx.x) >> 5 for warp-centric grid mapping when the workload is naturally keyed by (row, column) pairs rather than individual elements
