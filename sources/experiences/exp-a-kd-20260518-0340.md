sgemm_8x8_m512

## Solution

after code issues 2 CP_ASYNC_CA calls (16 bytes each) via __cvta_generic_to_shared pointer, commits the group, then proceeds with A loading and transpose while the B transfers are in flight. With BM=128/BN=128 tiling, each thread loads 8 values for A and the online transpose writes 8 scalar value...

## Key Lessons

- The after kernel uses cp.async.ca to asynchronously copy B-tile data from global to shared memory in the prologue.
