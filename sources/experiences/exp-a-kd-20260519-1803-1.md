## Challenge

blockReduceSum and warpReduceSum hardcode the addition operator (val += shuffle), preventing reuse of the reduction primitive for other binary operations (max, min) without duplicating the entire template


## Solution

Generalize the reduction primitive by accepting a ReduceFnType<T> function pointer parameter and replacing hardcoded val += with val = fn(val, ...), enabling a single template to serve sum, max, min and other binary operations while NVCC inlines the pointer to produce identical SASS

## Key Lessons

- Passing a __device__ function pointer for the binary operation in reduction templates generalizes the primitive (sum/max/min) without runtime cost — NVCC inlines the pointer and emits identical SASS to the hardcoded operator version
- Generalizing a reduction template via function pointer dispatch is a zero-cost abstraction when the function body is trivial (a + b, max(a,b)); the compiler devirtualizes and inlines it at compile time
