## Solution

Replace the software conversion chain with a single PTX inline asm instruction cvt.rni.sat.s8.f32 that performs round-to-nearest-int with saturation to s8 in one hardware instruction, eliminating rintf/fmaxf/fminf overhead

## Key Lessons

- Use PTX inline asm cvt.rni.sat.s8.f32 instead of software rintf+fmaxf+fminf for float-to-int8 conversion — the single hardware instruction performs round-to-nearest-even with saturation, replacing three separate math intrinsics and reducing instruction count per element
- Use reinterpret_cast<const int8_t&>(dst) to extract the sign-extended int8 from the 32-bit PTX register result without an additional conversion instruction
