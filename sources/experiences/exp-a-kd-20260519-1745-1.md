## Challenge

Uniform kChunkSize=512 chunk iteration requires 6 chunks for seqlen=3000, incurring excessive per-chunk overhead (boundary computation, mask checks) and poor ILP in small trailing chunks, while also misaligning with prefix cache block boundaries


## Solution

Replace uniform kChunkSize chunks with variable-length chunks read from a pre-computed cu_chunk_seqlen array, reducing 6 iterations to 3 (488+2048+464 tokens aligned with prefix cache blocks), halving chunk-loop overhead and enabling better ILP in the large 2048-token middle chunk

## Key Lessons

- Use variable-length chunk boundaries from pre-computed cu_chunk_seqlen metadata instead of uniform kChunkSize division when sequence lengths don't evenly divide — reduces loop iteration count and per-chunk overhead proportional to the number of eliminated chunk boundaries
- Align chunk boundaries with higher-level cache block boundaries (e.g., prefix cache blocks) to avoid misalignment — this enables merging small adjacent chunks into larger ones, improving ILP within each thread's work loop
- Track per-chunk token counts via a running tokens_processed accumulator instead of recomputing chunk offsets from chunk_index * kChunkSize — this naturally supports variable-length chunks without branch divergence on boundary checks
