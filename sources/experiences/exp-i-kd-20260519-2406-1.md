## Solution

Use make_coord(_,_) to slice tiled tensors by blockIdx indices, local_partition with the ThreadLayout to distribute tile elements among threads, make_tensor_like to create a register-backed fragment matching the source partition layout, and the copy() primitive for both GMEM→RMEM and RMEM→GMEM transfers — supporting UniversalCopy, AlignedArray-based, and AutoVectorizingCopy atom configurations

## Key Lessons

- Use make_coord(_,_) to extract a 2D tile slice from a tiled CUTE tensor indexed by blockIdx — this preserves the logical shape while selecting the block-local partition
- Use local_partition(tensor, ThreadLayout{}, threadIdx.x) to distribute a tile's elements among threads according to an arbitrary thread layout — the layout encodes which thread owns which elements
- Use make_tensor_like(source_tensor) to create a register-backed fragment tensor with matching shape and layout, then use copy(src, fragment) / copy(fragment, dst) as the standard CUTE GMEM↔RMEM transfer pattern
