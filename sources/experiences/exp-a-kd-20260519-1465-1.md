## Solution

Replace the hardcoded (kv_lora_rank + pe_dim) stride with a runtime entry_stride parameter, enabling correct addressing for both padded and non-padded cache layouts while producing identical addressing when entry_stride equals kv_lora_rank+pe_dim

## Key Lessons

- Parameterize layout-dependent stride values rather than hardcoding derived constants (kv_lora_rank+pe_dim) — enables correct handling of alignment-padded cache layouts without affecting performance in the unpadded case
- When entry_stride equals (kv_lora_rank+pe_dim) with no padding, both hardcoded and parameterized versions produce identical addressing, so the optimization benefit manifests only in padded cache layouts where correct stride enables efficient 128-byte vector loads in downstream kernels
