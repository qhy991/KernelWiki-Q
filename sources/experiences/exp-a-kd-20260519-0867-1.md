## Solution

The before code uses direct struct copy (row_chunk_vec_ptr[0] = vec_thread_read_ptr[0]) which compiles to a single wide 128-byte load; the after code replaces this with a loop extracting individual elements, which NVCC on sm_80 does not fuse back into a single load despite #pragma unroll, causing a 15.7% regression

## Key Lessons

- Direct struct copy (dst[0] = src[0]) on AlignedArray compiles to a single wide vector load instruction on sm_80, while per-element indexing (src[0][ii]) in an unrolled loop compiles to 32 separate accesses — prefer direct copy when data size matches the vector width exactly
- NVCC on sm_80 does not merge unrolled AlignedArray element accesses back into a single wide load even with #pragma unroll, so loop-based extraction from vectorized pointers is a pessimization rather than an optimization for full-width loads
- When VPT equals MAX_VPT (the full AccessType width), the direct copy has no wasted elements and naturally produces the optimal single-instruction load — element-wise extraction only adds overhead without alignment benefits in this case
