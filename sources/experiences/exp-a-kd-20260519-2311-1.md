## Solution

Replace the runtime float kernel parameter with a constexpr template struct (min_scaling_factor<FP8E4M3FN>::val()) that computes the constant at compile time via __forceinline__, removing the parameter from the kernel launch signature entirely

## Key Lessons

- Replacing a runtime float kernel parameter with a constexpr template struct produces identical PTX after NVCC constant-folding — both paths yield the same precomputed immediate constant → applies when the host-side value is already a compile-time constant, making this a code-clarity and type-safety refactor rather than a performance optimization
- Removing a single 4-byte scalar parameter from the kernel launch buffer does not measurably reduce per-invocation launch overhead when the kernel already has multiple pointer and integer parameters — launch overhead is dominated by driver dispatch and the overall argument buffer, not individual scalar argument savings
