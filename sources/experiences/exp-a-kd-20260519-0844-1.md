## Challenge

The merge_attn_states kernel uses scalar per-element bfloat16 load/store, where each 2-byte access wastes ~62 bytes of a 128-byte memory transaction, severely underutilizing global memory bandwidth for this memory-bound kernel


## Solution

Replace scalar bfloat16 load/store with 128-bit uint4 packed operations, so each thread loads 8 consecutive bf16 elements (16 bytes) in a single transaction, amortizing memory overhead and reducing total global memory transactions by ~8x

## Key Lessons

- Use uint4 (128-bit) packed load/store to process 8 consecutive bfloat16 elements per thread instead of scalar access — this amortizes memory transaction overhead across 8 elements and reduces total global memory transactions by ~8x, yielding significant bandwidth savings for memory-bound kernels
- When converting scalar kernels to vectorized packed access, reindex threads from per-element to per-pack granularity (threads_per_head = head_size / pack_size) so the same total work is distributed across fewer threads each doing proportionally more work per iteration
