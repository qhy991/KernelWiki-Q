## Solution

Fuse all five metadata copy operations into a single kernel launch using a parameter struct passed via __grid_constant__, with sequential #pragma unroll copy loops inside one kernel that handles all data sections — eliminating 4 kernel launch overheads and enabling the GPU to saturate SMs with the larger fused grid

## Key Lessons

- Fusing multiple small independent copy kernels into a single kernel eliminates per-launch overhead (~5us each) and improves SM utilization — applies whenever several lightweight data movements execute sequentially on the same stream
- Use __grid_constant__ to pass large parameter structs to fused kernels efficiently — the struct is stored in constant memory and cached per-SM, avoiding redundant global reads across all threads in a block
- Sized #pragma unroll (e.g., #pragma unroll 8) on grid-stride loops with small iteration counts enables the compiler to fully unroll trivial copy loops, reducing loop overhead and improving instruction-level parallelism
