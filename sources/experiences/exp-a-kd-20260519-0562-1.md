## Challenge

Flat linear indexing (idx = blockIdx.x * blockDim.x + threadIdx.x) interleaves thread pairs from different token positions within the same warp, causing each warp to access non-contiguous memory segments and issuing separate memory transactions for each discontinuous region


## Solution

Remap threads so one block maps to one token position (blockIdx.x = token_pos, threadIdx.x = pair index), ensuring consecutive threads within each warp read/write adjacent pairs in the same token row for fully coalesced memory access

## Key Lessons

- Remap blockIdx to the outer dimension and threadIdx to the inner dimension so that consecutive threads access consecutive memory addresses — this maximizes warp-level coalescing and is applicable whenever a 2D data structure is flattened into a 1D array
- Flat linear indexing across a multi-dimensional domain (idx = block*blockDim + thread with division/modulus to recover coordinates) causes inter-token interleaving within warps; prefer per-token-block mapping to keep warp memory accesses contiguous
