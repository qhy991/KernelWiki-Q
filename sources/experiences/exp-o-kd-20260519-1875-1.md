## Challenge

Warp-level sum reduction uses __shfl_down_sync (5 shuffle ops) which leaves the result only on lane 0, requiring an additional __shfl_sync broadcast (6th shuffle op) to distribute the result — the extra broadcast wastes one shuffle instruction per reduction, compounding over thousands of repeated reductions


## Solution

Replace __shfl_down_sync + separate __shfl_sync broadcast (6 ops) with __shfl_xor_sync butterfly pattern (5 ops) which makes the reduced result available on ALL lanes simultaneously, eliminating the broadcast step and adding #pragma unroll for compile-time loop unrolling

## Key Lessons

- Use __shfl_xor_sync butterfly pattern instead of __shfl_down_sync + __shfl_sync broadcast for warp reductions when the result is needed on all lanes — the butterfly distributes the result across all threads naturally, saving one shuffle operation per reduction call
- Saving a single shuffle instruction per reduction compounds measurably when the reduction is called inside a tight loop (5000 iterations × 8 warps) — instruction-level savings at high repetition counts produce observable speedups
