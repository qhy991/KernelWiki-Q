## Challenge

Implement Flash Attention with online softmax and tiling for LLaMA-3-8B with mixed precision (FP32 query, F16 KV cache)


## Solution

Naive Flash Attention with 1D thread layout (128 threads per block)


### Key Changes

- Used 1D block layout (128 threads) to stay within 1024 thread limit
- Tile size BLOCK_N=32 to fit shared memory within 48KB
- Replaced atomicMax with sequential processing for online softmax
- Proper F16 to FP32 conversion for KV cache


## Key Lessons

- atomicMax doesn't support float - need warp reduction or sequential processing
- Block dimensions must not exceed 1024 threads total
- Shared memory usage: q[BLOCK_D] + k[BLOCK_N][BLOCK_D] + v[BLOCK_N][BLOCK_D] must fit in 48KB
- Online softmax requires careful order of operations: compute m_new first, then l_new, then update output
- F16 KV cache needs __half2float() conversion for FP32 computation
- Naive implementation passes correctness but needs optimization for performance
