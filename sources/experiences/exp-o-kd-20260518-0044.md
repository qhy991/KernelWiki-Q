## Challenge

With 8 warps and 32 rows, the before code has more threads competing for atomicAdd on each s_prime entry. Each row is covered by multiple warps that all need to atomically add their partial sums


## Solution

The after code scales the addition_storage array to 32*8=256 floats and the reduction loop iterates 8 times per row.
