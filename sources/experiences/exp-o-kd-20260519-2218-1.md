## Solution

Swap the two cuBLAS operands and their transpose flags to compute D = B^T * A^T instead of C = A * B; since D = C^T, the result is mathematically equivalent but now the first operand B^T has N=4096 rows, giving cuBLAS a wide problem for efficient memory access and high occupancy

## Key Lessons

- For small-M GEMM (M << N, K), swap operands to compute C^T = B^T * A^T instead of C = A * B — the transposed formulation gives the cuBLAS kernel a wide first operand (N rows instead of M) for better memory throughput and occupancy
- The swap-AB identity C = A*B ⟺ C^T = B^T*A^T lets you freely choose which operand is 'first' in cuBLAS without changing the data layout — exploit this when one dimension (M) is much smaller than the others
