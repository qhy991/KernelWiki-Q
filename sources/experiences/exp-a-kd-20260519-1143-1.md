## Challenge

PagedAttention value cache uses [num_blocks, num_heads, block_size, head_size] layout, causing stride=head_size (64 elements, 256 bytes) between consecutive positions in a block, which prevents coalesced memory transactions when reading across the position dimension during attention decoding


## Solution

Transpose the inner two dimensions to [num_blocks, num_heads, head_size, block_size], making the position dimension stride-1 contiguous so that consecutive positions within a KV block are laid out adjacently in memory, enabling fully coalesced 128-byte memory transactions

## Key Lessons

- Placing the most frequently iterated dimension (position) as the innermost (stride-1) dimension in the KV cache enables coalesced memory access — applies to any paged attention implementation where the position loop is the dominant read pattern
- Isolated microbenchmarks with low occupancy (64 threads/block) and small data volumes (4 KB per head-block) can show false regression for layout optimizations that require high concurrency (many warps issuing coalesced requests) to realize bandwidth gains — validate layout changes under realistic multi-query batched workloads before concluding
