## Solution

Derive alignment from element bit width using 128 / sizeof_bits<ElementAB>, so for __half the computed alignment is 128/16=8 elements (16 bytes), fitting exactly one float4 load/store per work item and eliminating redundant memory transactions

## Key Lessons

- Derive vectorized access alignment from element bit width (128 / sizeof_bits<T>) rather than hardcoding a constant — ensures vector width always matches 128-bit memory access granularity regardless of element type
- Eliminating the 32-byte (256-bit) load path in favor of a single 128-bit transaction per work item reduces load/store instruction count by half and avoids exceeding the GPU's optimal coalesced access boundary
