## Solution

Add is_accumulator_needed guard to the first-iteration accumulator wait condition so that when K=0 the consumer_wait is skipped entirely, preventing deadlock while preserving correct synchronization for K>0 cases

## Key Lessons

- Guard pipeline synchronization waits with a data-dependency predicate (is_accumulator_needed) when the producer may not produce — prevents deadlock in degenerate dimensions (K=0) without affecting the common path
- CUTLASS pipeline consumer_wait calls assume prior producer commits; when the computation loop has zero iterations (K=0), the pipeline never transitions, so the consumer must be conditionally skipped
