## Challenge

Sigmoid and TopK are launched as two separate kernels: sigmoidKernel_scalar writes 128KB of intermediate sigmoid values to a global-memory workspace buffer, then topkKernel_blockreduce reads from that workspace, incurring two kernel launch overheads and a redundant global-memory round trip


## Solution

Fuse sigmoid and TopK into a single kernel (topkGatingSigmoid_kernel) that applies the sigmoid transform in registers immediately after loading input values, eliminating the global-memory workspace buffer and one kernel launch entirely

## Key Lessons

- Fuse elementwise transforms (sigmoid, ReLU, etc.) with downstream reductions (TopK, softmax) into a single kernel so transformed values stay in registers — eliminates global-memory workspace allocation, one kernel launch, and a full write-then-read round trip
- When a producer kernel's only consumer is a reduction-style kernel, the intermediate buffer is pure overhead — fuse and hold the data in register or shared memory instead
