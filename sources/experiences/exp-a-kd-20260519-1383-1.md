## Solution

Replace the scalar while-loop with a vec4_t<__half> (8-byte) load and float4_t (16-byte) store loop, unrolled by 4, to widen per-thread memory access; a scalar tail loop handles remainder elements

## Key Lessons

- Scalar per-thread access already achieves full warp-level coalescing on modern GPUs (A800) when threads read contiguous half values — a warp of 32 threads loads a contiguous 64-byte segment that the memory controller packs into the same number of 32-byte sectors regardless of whether threads issue individual half loads or a combined vec4 load, so vectorization provides no memory throughput benefit at warp level
- Vectorization overhead (reinterpret_cast setup, num_vec_elems computation, tail-loop branch, unroll pragmas) increases instruction count in a memory-bound kernel without reducing the dominant memory transfer time, causing a slight regression — vectorization only helps when it genuinely reduces HBM transactions or enables wider instruction-level parallelism
