## Challenge

Rotary embedding kernel assigns one warp per (token, head) pair, creating 64 warps per token with 32 Q and 32 K heads; each warp independently processes exactly one head, preventing any data reuse across heads that share the same position-dependent cos/sin values and reducing scheduling efficiency


## Solution

Consolidate HEADS_PER_WARP=4 heads onto each warp so one warp iterates over 4 heads sequentially, reducing total warps from 64 to 16 per token; this lowers scheduling overhead and enables sharing position-dependent data across heads within the same warp

## Key Lessons

- Grouping multiple attention heads per warp (HEADS_PER_WARP=4) reduces total warp count from 64 to 16 per token, enabling data sharing and lower scheduling overhead — applies when heads share read-only position-dependent data within the same token
- Reducing warp count can underutilize the GPU if the original grid already provided sufficient occupancy; at 512 tokens the before kernel launches 32768 warps (full GPU), while the after kernel launches only 8192 warps which may not saturate all SMs
