## Challenge

When processing rows whose block count (BPR=129) is not a multiple of blocks-per-warp (BPW=8), the final ib0=128 iteration unconditionally processes both QR=2 sub-rows even though ir=1 maps to bidx=132 which exceeds the valid row length, causing the kernel to accumulate padded zeros and waste one shared-memory read and multiply-accumulate per thread


## Solution

Add a loop-bound condition ib0 + ir * BPW / QR < bpr to the inner ir loop so that out-of-bounds sub-rows are skipped at the final misaligned iteration, trading a per-iteration integer comparison and potential branch divergence across all 34 iterations for one fewer shared-memory read and MAC at the tail

## Key Lessons

- Adding bounds checks to unrolled inner loops for correctness on misaligned dimensions introduces a per-iteration comparison overhead that can offset the savings from skipping one tail iteration — acceptable as a correctness fix but not a performance optimization when the skipped work is a tiny fraction of total work
- When BPR is not a multiple of BPW, the cooperative tile-loading strategy already pads rows with zeros, so the functional difference between with/without bounds check is zero; the bounds check only avoids accumulating padded zeros that were already harmless
