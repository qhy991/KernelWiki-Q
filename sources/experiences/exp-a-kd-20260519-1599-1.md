## Solution

Replace the per-thread prefix-sum counter matrix with warp-level shared_counts[32][8] for counting and atomicAdd on local_offsets[expert_id] for direct position claiming during sorting, eliminating the per-expert prefix-sum pass and enabling 1024-thread parallelism (16x increase over 64 threads)

## Key Lessons

- Replace per-thread prefix-sum counters with atomicAdd on a shared offset array when deterministic per-thread ordering is not required — eliminates O(blockDim.x × num_experts) shared-memory storage, one __syncthreads barrier, and the entire per-expert prefix-sum pass
- atomicAdd on shared-memory local_offsets reduces sorting from three shared-memory operations per token (two reads + one write) to a single atomic read-modify-write, improving throughput when per-expert contention is moderate relative to thread count
- Decoupling thread count from num_experts (64 → 1024 threads) enables 16x higher sorting parallelism when the algorithm no longer requires a 1:1 thread-to-expert mapping — applies to any MoE routing kernel where per-thread deterministic ordering is unnecessary
