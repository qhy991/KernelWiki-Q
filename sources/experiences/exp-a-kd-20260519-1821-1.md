## Solution

Decompose the 16×8×32 output tile into four 8×8×16 sub-tiles by splitting A into row halves (a[0]/a[1] for rows 0-7, a[2]/a[3] for rows 8-15) and B into K-halves (b[0] for k=0-15, b[1] for k=16-31), issuing separate m8n8k16 MMA instructions for each — each still occupies a full tensor core pipeline slot, causing 4x instruction throughput on sm80+

## Key Lessons

- Prefer the largest natively available MMA shape on the target hardware — decomposing m16n8k32 into four m8n8k16 quadruples instruction count because each sub-MMA still consumes a full tensor core pipeline slot, yielding no throughput benefit on sm80+
- Use m8n8k16 decomposition of m16n8k32 only when targeting sm75 (Turing), which lacks both m16n8k32 and m16n8k16 for int8 — on sm80+ (Ampere and later) the single m16n8k32 path is always preferred
- MMA shape decomposition changes the accumulator mapping: m16n8k32 writes c[0..3] in one instruction while four m8n8k16 instructions must carefully route a-fragments and b-fragments to the correct accumulator halves (c[0,1] vs c[2,3]) to produce mathematically equivalent results
