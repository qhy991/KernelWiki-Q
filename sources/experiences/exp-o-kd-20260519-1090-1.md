## Solution

Replace scalar 16-bit bf16 loads/stores with 128-bit uint4 vectorized accesses via a Pack16B union that loads 8 bf16 values per memory transaction (2 packs of 8 = 16 elements per iteration), reducing global memory transaction count by ~8x and adding __ldg read-only cache hints to improve L1 utilization

## Key Lessons

- Replace scalar bf16 loads/stores with uint4 (128-bit) vectorized accesses via a union overlay (Pack16B) to reduce memory transaction count by 8x — applies to any memory-bound kernel processing contiguous bf16 data where the hidden dimension is a multiple of 16
- Use __ldg read-only cache hints on input reads in memory-bound kernels to improve L1 cache utilization by routing through the texture/read-only cache path instead of the standard L1 path
- Vectorize both loads and stores together — scalar stores in the epilogue can negate the bandwidth gains from vectorized loads, so use uint4 reinterpret_cast writes for output as well
