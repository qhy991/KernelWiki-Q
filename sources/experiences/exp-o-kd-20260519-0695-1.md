## Solution

Replace the 4-warp CTA kernel (128 threads, shared memory, 2x __syncthreads()) with a single-warp kernel (32 threads, each processing 4 elements) using warp shuffle reduction and __shfl_sync broadcast for inv_rms, eliminating all shared memory and barrier synchronization while improving occupancy from ~4 to ~16 blocks per SM

## Key Lessons

- For hidden sizes where the total elements fit within a single warp's capacity (≤ 128 elements with 4 per thread in 32 threads), a single-warp kernel with __shfl_down_sync reduction eliminates shared memory and all __syncthreads() barriers entirely — applies when the reduction produces ≤ 32 partial sums
- Broadcasting a scalar via __shfl_sync(mask, val, 0) replaces a shared-memory write + __syncthreads() + shared-memory read pattern, saving one full barrier round-trip per token
- Smaller block sizes (32 vs 128 threads) increase SM occupancy by up to 4x (16 vs 4 concurrent blocks per SM), which is especially beneficial for memory-bound kernels like RMS norm where latency hiding dominates
