## Solution

Replace scalar per-element load loops and scalar store loops with single pointer-cast AlignedArray dereferences using a correctly-sized Vlen=8 type, enabling the compiler to emit LDG.128 and STG.128 instructions that perform 2 aligned 32-byte vector loads and 1 aligned 32-byte vector store instead of 24 separate 4-byte transactions

## Key Lessons

- Use an AlignedArray type whose width exactly matches the per-thread data footprint (Vlen=8) rather than a wider container (MAX_VPT=32) — the type-width mismatch prevents the compiler from emitting a single wide load instruction even when the access pattern is contiguous
- Replace scalar per-element load/store loops with single pointer-cast dereferences (*(ArrayVal*)ptr) to let the compiler emit LDG.128/STG.128 vectorized memory transactions, reducing transaction count from 24 to 3 per thread and achieving 1.46x speedup
- Align the AlignedArray type size to the hardware's 128-bit load/store unit width (4 floats for fp32) so that a Vlen=8 element chunk maps to exactly two LDG.128 instructions per load
