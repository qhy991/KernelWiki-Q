## Challenge

The before code uses 1024 threads per block (min(8192/8, 1024)), launching 256 blocks. With 1024 threads per block, the GPU can only fit a small number of blocks per SM due to register and shared memory constraints


## Solution

Replace scalar f32 loads with vectorized float4 (f32x4) loads so each thread fetches 4 elements per iteration via reinterpret_cast<float4>, reducing memory transactions by 4x and loop trip count by 4x

## Results

- **speedup**: 1.857x (improved)


## Key Lessons

- Use vectorized float4 loads (reinterpret_cast<float4*>) instead of scalar f32 loads in memory-bound inner loops — coalesces 4 scalar transactions into 1 vector transaction, improving memory bandwidth utilization when accesses are contiguous and aligned
- Vectorized loads increase per-thread working set by 4x — can cause cache thrashing at large K (e.g., K=2048) where the working set exceeds L2 capacity, so apply with awareness of problem dimensions
- Adjust loop stride and iteration count in proportion to vector width (divide NUM_WARPS by 4 for float4) to maintain correct element coverage across the full K dimension
