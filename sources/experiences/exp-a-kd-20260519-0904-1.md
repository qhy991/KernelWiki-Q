## Solution

Replace the 4-warp CTA block with a single-warp (32-thread) block where each thread processes 4 elements, eliminating all shared memory and __syncthreads() barriers; warp shuffle (__shfl_down_sync) performs full reduction within the single warp and __shfl_sync broadcasts inv_rms, yielding up to 4x higher occupancy (16 vs 4 CTAs/SM)

## Key Lessons

- For small hidden sizes (≤128) where one warp can cover the dimension, use a single-warp block instead of multi-warp CTA — eliminating shared memory and __syncthreads() barriers while gaining occupancy from smaller blocks (32 threads → up to 16 CTAs/SM vs 128 threads → ~4 CTAs/SM)
- When a full reduction fits within one warp, __shfl_sync(..., val, 0) broadcasts the result to all lanes without shared memory, replacing the write-to-smem + __syncthreads() + read-from-smem pattern
- Increasing per-thread element count (kElemsPerThread from 1 to 4) is a valid strategy to keep a warp fully utilized when downsizing from multi-warp to single-warp — register pressure stays low for 4 fp32 accumulators
