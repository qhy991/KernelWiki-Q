## Challenge

GELU FP32 scalar kernel processes one float per thread with individual 32-bit load/store instructions, underutilizing GPU memory transaction granularity and bandwidth since each thread issues separate uncoalesced or partially coalesced memory transactions


## Solution

Vectorize the kernel to process 4 floats per thread using float4 (128-bit) load/store, so each thread reads and writes 4 adjacent floats in a single coalesced memory transaction, reducing total memory transactions by ~4x and improving bandwidth utilization

## Key Lessons

- Vectorize elementwise kernels with float4 load/store so each thread processes 4 adjacent elements in a single 128-bit coalesced transaction — applies to any memory-bound activation or elementwise operation where computation per element is lightweight
- Scale the global thread index by the vector width (idx * 4) to maintain correct coverage while reducing the number of threads needed by the vectorization factor
- Manual unrolling of computation across float4 .x/.y/.z/.w fields keeps the compute pipeline simple and avoids loop overhead when the vector width is small and fixed
