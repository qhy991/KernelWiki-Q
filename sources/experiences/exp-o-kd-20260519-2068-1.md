## Challenge

The moeTopK fallback kernel for non-power-of-2 expert counts writes top-k weights without normalization, requiring a separate renormalize kernel launch and an extra global memory read+write cycle to normalize the k selected weights to sum to 1


## Solution

Fuse renormalize into moeTopK by adding a renormalize boolean parameter, accumulating selected weight sums during the topk selection loop, and dividing each of the k output weights by that sum after the loop — eliminating a separate kernel launch and global memory round-trip

## Key Lessons

- Fuse lightweight post-processing (e.g., renormalize of k values) into the kernel that produces the output — the per-block overhead of accumulating a sum and dividing k values is negligible compared to eliminating a full kernel launch and global memory round-trip
- Guard fused logic behind a boolean parameter to preserve correctness on code paths that don't need the extra work — the conditional branch cost is near-zero when the condition is uniform across the block
