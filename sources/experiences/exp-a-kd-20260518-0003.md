## Challenge

The 4-bit dequant kernel (dequant_4bit_8) constructs zero-point offset constants z1 and z16 using __float2half_rn(-1024.0f - (float)zero) and __float2half_rn(-1024.0f / 16.0f - (float)zero). These float subtraction/division + float-to-half conversion operations are expensive per-call and the zero-point value is known at kernel level.


## Solution

Replace float-arithmetic constant construction with bit-level IEEE 754 half-precision manipulation. For z1: half_uint16(0xe400 | zero) directly ORs the runtime zero into the known bit pattern for -1024.0f (0xe400), bypassing float arithmetic entirely. For z16: __hsub(__int2half_rn(-64), __int2half_rn(zero)) uses compact integer-to-half mapping plus a half subtraction. Both broadcasts use __half2half2 (lane duplicate) instead of __halves2half2 (two-argument combine).

## Results

- **speedup**: 1.145x (12.6% latency reduction)


## Key Lessons

- When constructing half-precision constants from known base values, use bit-level manipulation (integer OR into known IEEE 754 patterns) instead of float arithmetic + conversion. __float2half_rn is expensive because it goes through the full float rounding pipeline; half_uint16(uint16_t) is free. Applies when the base constant has a known bit representation.
- __int2half_rn is faster than __float2half_rn for integer-valued constants because it uses a compact hardware/table lookup rather than the full float conversion pipeline. When the value is a small integer, prefer __int2half_rn. Applies to any dequant constant construction with integer offsets.
- The combined effect of eliminating float conversion overhead and reducing instruction count for constant setup yielded 12.6% latency reduction in a memory-bound 4-bit dequant kernel. Even small instruction count reductions in hot dequant paths produce measurable gains.
- __half2half2(val) is the dedicated broadcast intrinsic (duplicates a single half into both lanes). When the compiler can prove __halves2half2 gets identical arguments, it optimizes to the same PTX — but __half2half2 makes broadcast intent explicit and avoids relying on compiler optimization.
