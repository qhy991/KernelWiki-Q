## Solution

Replace the hardcoded % 4 divisibility check with a dynamically computed divisor k_tile / mma_k_atom that derives the actual MMA instruction count per mainloop iteration from the tile shape configuration, ensuring correct validation for any CUTLASS tile layout

## Key Lessons

- Never hardcode the number of MMA instructions per mainloop iteration as a constant — compute it dynamically as size<2>(TileShape{}) / TiledMma::tile_size_mnk<2>() since it varies with every tile shape configuration and an incorrect divisor silently accepts invalid or rejects valid promotion intervals
- When validating scheduling parameters derived from hardware tile geometry, always derive constraints from the actual runtime tile configuration rather than assuming a fixed ratio that only holds for one specific tile shape
