## Solution

Fuse both passes into one: read global memory once into shared memory while computing absmax during the copy, then quantize from shared memory, eliminating the redundant global read at the cost of a __syncthreads() barrier and shared memory usage

## Key Lessons

- Shared memory caching to eliminate redundant global reads yields minimal benefit when the working set fits within L2 cache (32MB input vs 40MB L2 on A800) — the second global read already hits L2, not DRAM, so shared memory introduces syncthreads barrier overhead without meaningful DRAM savings
- Fusing computation (absmax) into a copy loop that stages data to shared memory is a valid pattern, but the __syncthreads() barrier cost and shared memory bank conflicts must be weighed against the actual DRAM traffic reduction
- For kernels where the input tensor is comparable to or smaller than L2 cache size, the hardware cache already provides the equivalent of shared memory caching — explicit shared memory staging only helps when the working set clearly exceeds L2 capacity
