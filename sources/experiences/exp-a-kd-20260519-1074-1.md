## Solution

Vectorize the MoE reduce kernel using half2 loads with vec_size=8: replace scalar half loads with half2 reinterpret_cast loads, __half2float with __half22float2 for batch conversion, scalar __float2half stores with __float22half2_rn vectorized stores, and add #pragma unroll on the inner accumulation loop to eliminate branch overhead

## Key Lessons

- Use half2 vectorized loads via reinterpret_cast<const half2*> and __half22float2 to process two fp16 values per load/conversion instruction in gather-reduce kernels — halves memory transactions and conversion instructions, applies whenever input elements are contiguous and 2-byte aligned
- Apply #pragma unroll to inner accumulation loops over fixed-size vector tiles to eliminate loop branch overhead and enable the compiler to pipeline load-compute-store sequences
- Vectorization gains diminish at higher topk values because per-row address computation (permutation lookup, src_row * row_stride + d) is scalar-only and dominates execution time — factor this into expected speedup estimates
