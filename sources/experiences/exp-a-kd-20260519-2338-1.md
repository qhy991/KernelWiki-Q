## Solution

Decompose work into warp-per-head assignments (8 warps for 32 heads, each warp handles one head contiguously), then use vectorized loads/stores with VEC_SIZE=8 via vectorize_with_alignment to copy each head_size=128 segment in 4 vector iterations per thread instead of 128 scalar transactions

## Key Lessons

- Assign one warp per head in KV-cache copy kernels when num_heads divides evenly into blockDim.x/32 — each warp operates on a contiguous head segment, eliminating per-element division/modulo for index decomposition and producing naturally coalesced memory access
- Use vectorize_with_alignment with VEC_SIZE=8 for half-precision head_size=128 copy loops — 8 half elements (16 bytes) per vector transaction reduces memory operation count by 8x and saturates available bandwidth for contiguous segments
- Compute source and destination base pointers once before the head loop (key_src, key_dst) instead of recalculating full offsets per element — reduces address arithmetic from O(n) per element to O(1) per head
