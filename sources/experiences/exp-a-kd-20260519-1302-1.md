## Solution

Cooperatively load the entire expert offset array into shared memory via __ldg before use, so all subsequent reads in the accumulation loop hit smem instead of global memory

## Key Lessons

- For data smaller than one cache line (72 bytes for 9 int64_t) that is already L1-cache resident after the first thread block's access, shared memory caching adds cooperative-load and __syncthreads() overhead without measurable benefit — prefer direct global reads when the working set fits comfortably in L1
- __ldg routes loads through the read-only/texture cache path, which helps under L1 contention or for large strided reads, but for tiny sequentially-accessed arrays the hardware L1 already provides equivalent latency
