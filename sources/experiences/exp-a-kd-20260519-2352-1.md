## Solution

The after code uses __constant__ memory for the LUT, but constant cache can only service one address per warp per clock; with 16 divergent byte indices per thread across 32 warp threads, accesses serialize while the global-memory LUT benefits from L2 cache which handles random accesses in parallel without serialization

## Key Lessons

- __constant__ memory is only beneficial when all threads in a warp access the same address (broadcast pattern); when threads access different addresses, the constant cache serializes per-address servicing, causing severe slowdowns — prefer global memory with L2 cache for divergent-access LUTs
- Small lookup tables (e.g., 256 bytes) that fit entirely in the GPU L2 cache achieve near-constant-cache latency for broadcast patterns while retaining full parallelism for divergent patterns, making global memory the safer default
- Per-element LUT lookups with data-dependent indices (b[i] = lut[b[i]]) are inherently divergent across threads and within each thread's unrolled loop — this anti-broadcast pattern is the worst case for constant-cache utilization and should avoid __constant__ storage
