## Challenge

The per-token group quantization kernel hardcodes row-major scale output addressing via output_s + (block_group_id + local_group_id), requiring a separate kernel implementation to support column-major weight layouts that transpose the scale stride


## Solution

Add a bool IS_COLUMN_MAJOR template parameter (defaulting to false) and use if constexpr to dispatch scale output indexing at compile time — the column-major path remaps global_group_id via (col_idx * scale_stride + row_idx), while the false path compiles to identical instructions as the original code with zero overhead

## Key Lessons

- Use if constexpr (C++17) for compile-time branching to add data layout flexibility without runtime cost — the compiler elides the unused branch entirely, producing identical PTX to a hand-written single-layout kernel
- Template bool parameters with default values extend CUDA kernel functionality backward-compatibly — existing call sites that omit the parameter get the default (false) behavior at zero performance overhead
