## Challenge

The ep_pre_reorder kernel uses a vectorized loop with vec_size=4 to copy and scale hidden states, but lacks a scalar tail loop to process remainder elements when hidden_size is not exactly divisible by vec_size, causing incorrect results for non-aligned dimensions


## Solution

Add a scalar tail loop that processes the remaining (hidden_size % vec_size) elements after the vectorized loop, computing vec_elements = (hidden_size / vec_size) * vec_size as the boundary so the tail iterates zero times when hidden_size is already aligned, ensuring correctness without measurable overhead

## Key Lessons

- Always pair a vectorized loop with a scalar tail loop to handle remainder elements when the problem dimension may not be divisible by the vector width — the tail iterates zero times at aligned sizes and costs only one comparison per thread
- A zero-iteration loop (dead branch) adds negligible overhead even on sub-0.02ms kernels — correctness fixes for non-aligned dimensions are essentially free when the aligned case is the common path
