## Challenge

Each thread performs an O(E) linear scan through a 256-element prefix-sum array to find which expert a token belongs to, requiring up to 256 serial comparisons per token with divergent branch exits and uncoalesced memory reads


## Solution

Replace per-thread linear scan with warp-cooperative binary-style search: each warp iteration probes 32 consecutive prefix-sum entries in parallel, uses __ballot_sync to vote on which lanes still need to move forward, and __clz to locate the boundary lane — reducing serial iterations from O(E) to O(E/32)

## Key Lessons

- Use __ballot_sync for warp-cooperative search over sorted arrays: each lane probes one element, the ballot mask reveals which lanes passed the predicate, and __clz extracts the boundary index — reducing O(E) serial search to O(E/32) warp iterations
- Cooperative warp search scales better than per-thread linear scan as the search space grows because serial overhead grows linearly with E for linear scan but only as E/32 for cooperative search, yielding increasing speedup at larger E
- Prefer warp-wide coalesced reads (32 lanes reading consecutive offsets) over per-thread divergent sequential reads for improved memory throughput on prefix-sum array lookups
