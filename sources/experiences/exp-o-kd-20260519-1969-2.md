## Solution

Replace per-byte conditional nibble remapping with a 256-entry constant-memory lookup table covering all byte values precomputed at kernel startup, combined with uint4 vectorized loads (16 bytes per thread) and a grid-stride loop for efficient GPU execution

## Key Lessons

- Use a constant-memory LUT instead of conditional branches for small bounded mappings — a 256-entry table covers all byte values and leverages the GPU constant cache for uniform low-latency access across all threads in a warp
- Vectorized loads via uint4 (16 bytes per thread) coalesce global memory transactions and reduce load/store instruction count by 16x compared to single-byte access
- Grid-stride loops decouple data size from grid configuration, allowing a single kernel to handle arbitrary input sizes while keeping each thread's working set in registers
