## Challenge

The VecLoad dispatch condition only checks seqlen divisibility (seqlen % kNElts == 0), enabling vectorized loads even in varlen mode where individual sequences have different lengths, risking out-of-bounds or misaligned accesses across sequence boundaries within a batch


## Solution

Add a varlen guard to the VecLoad condition so that vectorized loads are only enabled when both seqlen is divisible by kNElts AND the kernel is not in varlen mode, preventing unsafe vectorized memory accesses across variable-length sequence boundaries

## Key Lessons

- Vectorized load/store dispatch conditions must guard against variable-length (varlen) mode — when sequences in a batch have different lengths, the total seqlen divisibility check is insufficient because individual sequences may not be aligned, risking out-of-bounds vectorized accesses
- A correctness guard that disables an optimization path for unsafe cases may show neutral performance in uniform-length benchmarks — the real impact is preventing silent corruption in production varlen workloads
