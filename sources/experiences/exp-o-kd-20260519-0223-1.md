## Solution

Eliminate the source tensor load entirely by removing the source parameter, beta parameter, and associated multiply-add from the kernel when beta=0 is known at compile time via CUTLASS's ScaleType::OnlyAlphaScaling, reducing memory traffic from ~48MB (acc+src+out) to ~32MB (acc+out) per invocation

## Key Lessons

- Eliminate source tensor loads at compile time via template policy (e.g., ScaleType::OnlyAlphaScaling) rather than relying on runtime branching when scaling factors are known before kernel launch — saves global memory bandwidth proportional to tensor size and enables compiler dead code elimination of the entire load path
- Pass scaling factors as kernel template arguments or value parameters instead of device pointers when they are known before launch — device pointers obscure values from the compiler and block dead code elimination of dependent code paths
