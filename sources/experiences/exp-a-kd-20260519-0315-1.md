## Solution

Remove the redundant cudaMemset calls from both warmup and timed loops since the kernel fully overwrites every output element, eliminating an entire device-memory sweep and reducing total memory traffic per iteration.

## Key Lessons

- Remove cudaMemset before a kernel that unconditionally writes every output element — the memset adds a full device-memory pass with no correctness benefit, costing ~25% overhead for an elementwise kernel that is already memory-bandwidth-bound
- When benchmarking memory-bound kernels, any extra device-memory traversal (even a lightweight memset) appears as a significant fraction of measured time because the kernel itself is already saturating memory bandwidth
