## Solution

Pre-compute source and destination base pointers once outside the loop, then use a vectorized copy utility (VEC_SIZE=8) that loads/stores 8 half values (128 bits) per thread via aligned vec_n_t<half,8> transactions, coalescing 8 scalar 2-byte operations into a single wide memory transaction

## Key Lessons

- Pre-compute source and destination base pointers before the copy loop when the layout makes source and destination contiguous — eliminates per-element div/mod index computation and enables vectorized wide loads/stores
- Use vec_n_t<scalar,VS> (VEC_SIZE=8 for fp16) to coalesce multiple narrow scalar stores into 128-bit wide memory transactions, yielding up to ~1.7x speedup for contiguous half-precision copy kernels
- Alignment-aware vectorization with a scalar prefix/tail fallback (vectorize_with_alignment pattern) safely handles unaligned base addresses while maximizing the vectorized fast path for the majority of elements
