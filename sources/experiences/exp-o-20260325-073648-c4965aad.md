fp32_top_k_sampling_qwen2_5_7b_k8_ne0160

## Solution

One warp (32 threads) per batch item. Each thread loads 5 elements (160/32=5). k=8 rounds of warp-level argmax using __shfl_down_sync to find top-k indices. No shared memory needed.

## Key Lessons

- For vocab_size <= 160, one warp per batch item with warp-shuffle reduction is significantly faster than shared-memory tree reduction
- Use /home/qinhaiyan/anaconda3/envs/Kernel/bin/python to run unified_test_runner.py (has torch 2.8.0+cu128)
- The extern C entry point for topk op_type must be named topk_kernel exactly
- k rounds of warp-wide argmax with masking is correct and efficient for small k (k=8)
