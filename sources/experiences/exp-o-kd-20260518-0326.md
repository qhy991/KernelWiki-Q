## Challenge

The before code issues 16 individual cudaMemcpyAsync calls (8 pairs × 2 for key+value) from the CPU side, each requiring CPU-side dispatch overhead. For small 1KB blocks (256 floats), this per-call overhead dominates the actual transfer time


## Solution

The after code consolidates all 8 pairs into a single kernel launch using a 2D grid where blockIdx.y selects the pair and blockIdx.x selects the layer.
