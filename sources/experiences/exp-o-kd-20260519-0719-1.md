## Challenge

A separate zero_buffer_kernel is launched to zero the cumsum_buffer before each invocation of the MoE block alignment kernel, adding a full kernel launch overhead (~2.7us) despite the align kernel already initializing all cumsum entries through its prefix-sum computation


## Solution

Remove the redundant zero_buffer_kernel launch since moe_align_block_size_kernel explicitly writes cumsum[0]=0 and computes every cumsum[i] from cumsum[i-1], fully initializing the buffer without any dependency on prior contents

## Key Lessons

- Eliminate redundant buffer zeroing kernels when the downstream consumer already initializes every element it reads — the savings come from removing the fixed kernel launch overhead (~2-3us), not from the memory operations
- A prefix-sum (cumsum) kernel that writes cumsum[0]=0 and computes each cumsum[i] from cumsum[i-1] fully initializes its output array, making any prior zeroing of that array unnecessary
