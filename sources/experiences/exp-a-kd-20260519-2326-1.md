## Solution

Replace per-thread 2D counting rows with atomicAdd into a flat padded shared_counts array, enabling the kernel to launch with 1024 threads (8× more parallelism for counting) while reducing shared memory from ~66KB to ~5KB by eliminating the (num_threads+1)×num_experts 2D array

## Key Lessons

- Using atomicAdd into a flat shared memory array instead of per-thread 2D counting rows eliminates the serial thread-row prefix sum aggregation step and reduces shared memory from O(num_threads × num_experts) to O(padded_num_experts), enabling higher thread counts and better occupancy
- Padded shared memory allocation to warp boundaries (padded_num_experts = round_up(num_experts, WARP_SIZE)) avoids bank conflicts when threads access shared_counts in the subsequent scan phase — applies whenever shared arrays are followed by warp-cooperative operations
