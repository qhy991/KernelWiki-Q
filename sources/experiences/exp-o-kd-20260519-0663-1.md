## Solution

Cache the expert_offsets array into shared memory via cooperative int4-vectorized loads at block start — each thread loads 4 uint32_t elements per iteration using 16-byte int4 transactions, and the one-time __syncthreads barrier cost is amortized across all elements processed by the block

## Key Lessons

- Cooperative shared memory loading with int4 vectorized transactions (16 bytes per load) reduces global memory traffic for small read-only lookup tables — applies when a table under 4KB is read repeatedly by all threads in a block
- Use int4 reinterpret_cast loads instead of scalar loads for shared memory population — 4x fewer load transactions with identical data movement, since global memory bandwidth is transaction-granular at 32 bytes minimum
