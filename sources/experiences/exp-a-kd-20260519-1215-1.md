## Solution

Replace the scalar strided loop with vectorize_read_with_alignment<8> which issues 32-byte (256-bit) aligned vector loads via vec_n_t<float, 8>, reducing total memory transactions from 4096 individual 4-byte loads to 512 wider loads while handling misalignment with scalar prefix/tail fallbacks

## Key Lessons

- Vectorized loads (vec_n_t<float,8> = 32-byte reads) reduce total memory transactions by N× where N is the vector width — applies whenever data is aligned and length is divisible by the vector factor, but expect modest gains for already-coalesced float types since warp-level 128-byte transactions already merge scalar float reads efficiently
- Use a generic vectorize_read_with_alignment helper with lambda callbacks (vec_op + scalar_op) to decouple the memory access pattern from the computation logic — this handles misalignment transparently via scalar prefix/tail while maximizing vector throughput on the aligned body
