## Challenge

Warp-level sum reduction uses __shfl_down_sync which leaves the result only on lane 0, requiring an additional __shfl_sync broadcast to distribute the result to all lanes, totaling 6 shuffle operations per reduction call


## Solution

Replace __shfl_down_sync reduction plus __shfl_sync broadcast with __shfl_xor_sync butterfly pattern which completes the reduction in exactly 5 shuffle ops and leaves the result on ALL lanes, eliminating the separate broadcast step

## Key Lessons

- __shfl_xor_sync butterfly pattern leaves the reduction result on all lanes after 5 ops, whereas __shfl_down_sync leaves it only on lane 0 requiring a 6th broadcast — prefer xor when all threads need the result, saving one shuffle instruction per reduction call
- Adding #pragma unroll to the shuffle reduction loop ensures the compiler fully unrolls the 4-iteration loop, eliminating loop overhead and enabling constant-propagation of the mask values
- Per-reduction savings of one shuffle op accumulate to measurable speedups when the reduction is called thousands of times inside a kernel (5000 reductions here yielded 5.6% improvement)
