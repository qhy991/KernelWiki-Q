## Challenge

GEMM kernel launcher always hardcodes PersistentScheduler regardless of problem shape dimensions, with no heuristic to adapt the scheduling strategy based on the ratio of K to N that determines whether enough output tiles exist to fill all SMs


## Solution

Introduce a shape-aware heuristic dispatch that selects PersistentScheduler when K <= 3*N (balanced shapes with sufficient output tiles to occupy all SMs), preserving the optimal scheduling path for this regime while enabling future selection of alternative schedulers for unbalanced shapes

## Key Lessons

- Shape-aware scheduler dispatch using the K/N ratio as a heuristic threshold correctly preserves PersistentScheduler for balanced GEMM shapes (K <= 3*N) where total output tiles >= num_SM — introducing the dispatch logic itself is neutral for shapes already on the optimal path, but enables correct fallback for unbalanced shapes
- Persistent scheduling is beneficial when total_tiles (tiles_m * tiles_n) is large enough to keep all SMs busy; the K <= 3*N heuristic is a proxy for sufficient tile parallelism relative to computational intensity
