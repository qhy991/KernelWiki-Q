## Solution

Replace the single scalar scale with a groupwise scale lookup from a per-group scale array, adding compile-time template parameters (group_size, is_scale_transposed) to compute the correct scale index per vec4 chunk; the scale array (32 groups * 4 bytes = 128 bytes) fits entirely in L1 cache, so the added index arithmetic and load are hidden by the existing memory-bandwidth bottleneck

## Key Lessons

- Groupwise scale lookup adds negligible overhead in memory-bandwidth-bound kernels when the scale array fits in L1 cache (128 bytes for 32 groups) and the lookup cost is amortized across vec4 elements — applicable whenever per-group granularity is needed without changing the dominant memory access pattern
- Compile-time template parameters (group_size, is_scale_transposed) allow a single kernel implementation to support both scalar and groupwise scale modes with zero runtime branching overhead via if constexpr — use this pattern to generalize kernels without sacrificing the scalar-path fast case
