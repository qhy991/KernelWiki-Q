## Key Lessons

- The before code uses key_start - num_keys - num_queries (minus sign), which for key_start=64, num_keys=128, num_queries=128 gives max(0, 64-128-128) = max(0, -192) = 0, incorrectly returning 0 instead of the correct first_query=64.
- The sign fix correctly positions the bottom-right causal diagonal: a key at position key_start should first attend to queries starting at key_start - num_keys + num_queries, which ensures the causal boundary aligns with the bottom-right offset of the attention matrix.
