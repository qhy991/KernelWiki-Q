## Challenge

Chained kernel pairs (main_work + tail) execute sequentially on the same stream with no early completion signal, forcing each successor to wait for the predecessor's tail/epilogue work to finish before starting its own main work, preventing any overlap


## Solution

Record a work-done event (cudaEventRecord with cudaEventDisableTiming) immediately after the main kernel and before the tail kernel, then have the successor chain wait on that event via cudaStreamWaitEvent, simulating SM90a griddepcontrol.launch_dependents to allow successor main work to overlap with predecessor tail work

## Key Lessons

- Programmatic Dependent Launch (PDL) with griddepcontrol.launch_dependents on SM90a signals successor grids at work-exhaustion (before tail/epilogue), enabling overlap — but event-based simulation on SM80 lacks this hardware support and serializes identically, so the technique only benefits Hopper+ GPUs
- Use cudaEventDisableTiming when creating events used purely for synchronization signaling (not elapsed-time measurement) to reduce overhead on the CUDA runtime event pool
- When profiling PDL-like patterns, neutral results on pre-SM90a hardware do not invalidate the optimization — the benefit materializes only with native griddepcontrol instructions on Hopper architecture
