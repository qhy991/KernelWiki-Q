## Solution

Introduce an undo_pack constexpr lookup table {0,4,1,5,2,6,3,7} to remap AWQ's interleaved bit positions back to sequential order before extraction, correcting the deinterleaving logic without changing the memory access pattern or instruction count since the constexpr table is resolved at compile time

## Key Lessons

- constexpr lookup tables indexed by compile-time-resolvable values (e.g., threadIdx-derived indices with small range) are folded into immediate constants by the compiler, producing SASS identical to direct computation — use them freely for readability without performance concern
- AWQ's 4-bit packing interleaves nibble positions (0,4,1,5,2,6,3,7) rather than storing them sequentially; a GPTQ-style sequential cur_pos extraction reads the wrong nibble unless an undo_pack remapping is applied — correctness fix, not a performance optimization
