## Solution

Replace scalar per-element multiply-add loop with WMMA 16x16x16 tensor core fragments: each mma_sync call processes 16 K-elements for an entire 16x16 output tile simultaneously through the dedicated matrix multiply unit, reducing total instructions from 737280 scalar operations to 180 mma_sync calls per tile

## Key Lessons

- Replacing scalar per-element multiply-add loops with WMMA tensor core fragments (mma_sync) reduces instruction count by orders of magnitude for dense matrix operations — applies whenever the workload is a GEMM on half-precision data where tensor cores are available
- WMMA fragment-based GEMM replaces K scalar multiply-adds per output element with K/WMMA_K mma_sync calls per 16x16 output tile, leveraging dedicated matrix multiply hardware that processes an entire 16x16x16 tile in one instruction
- fill_fragment(frag_c, 0.0f) provides a clean zero-initialization of the accumulator fragment, eliminating the need for conditional first-iteration checks in the K-loop
