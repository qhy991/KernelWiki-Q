## Challenge

Boundary-padding threads use an if/else branch: invalid positions zero a 16-byte PVec via memset while valid positions perform a 128-bit reinterpret_cast load, causing warp divergence for boundary threads in the quantization kernel


## Solution

Replace the branch with a predicated PTX ld.global.cg.v4.u32 inline-asm load (ld128_or_zero_cg_u32) that zeroes destination registers when the predicate is false and loads from global memory with .cg hint when true, eliminating branch divergence at the cost of bypassing L1 cache

## Key Lessons

- The .cg (cache-global) hint bypasses L1 and forces L2-only caching on SM80, adding ~30-40ns latency per access — the cost of losing L1 hits for 128-bit loads outweighs any branch-divergence savings on this architecture
- PTX asm volatile blocks act as optimization barriers that prevent the compiler from fusing the load with subsequent half2 abs-max operations, increasing register pressure and breaking instruction-level parallelism — a simple branch may allow better compiler scheduling
- Predicated-load techniques designed for SM100 (where 256-bit ld.global.cg.v8.u32 provides throughput gains) do not generalize to SM80's 128-bit variant — architecture-specific validation is essential before applying cache-hint optimizations
