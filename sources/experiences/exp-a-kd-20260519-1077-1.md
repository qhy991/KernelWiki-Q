## Challenge

Speculative decoding tree mask uses bool-per-element storage (256*32*32 = 262144 bytes), incurring 8x memory overhead vs a bit-packed representation and requiring 31 separate false-bool stores per token for initialization plus a complex index calculation involving per-batch seq_tree_idx accumulation and a FULL_MASK/QLEN_ONLY branch


## Solution

Replace bool-per-element mask with bit-packed uint8_t mask (256*32*4 = 32768 bytes, 8x reduction), simplifying index calculation to a direct stride product and replacing 31 boolean stores with a single byte write, using bitwise OR to set individual bits during tree traversal

## Key Lessons

- Bit-pack boolean masks into uint8_t (8 booleans per byte) instead of using bool-per-element arrays — reduces memory footprint by up to 8x and eliminates per-element initialization loops, applicable whenever mask elements are independently addressable bits
- Simplify index arithmetic when switching to packed representation: stride-based indexing (bid * draft_token_num + tid) * num_bytes_per_item replaces multi-branch offset calculation and removes the need for a runtime mask-mode enum
- Use bitwise OR (|= with bit shift) for individual bit setting in packed masks instead of direct array assignment — preserves adjacent bits already set and avoids read-modify-write hazards when multiple threads write different bits in the same byte
