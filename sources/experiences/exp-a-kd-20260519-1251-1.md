## Challenge

Two separate kernels perform attention output reduction and FP8 quantization: reduce_kernel_float writes float results to a global memory intermediate buffer (~32KB for HEAD_SIZE=256), then quantize_kernel_fp8 reads it back, applies scale, and converts to FP8 — incurring one extra kernel launch and a full write/read round-trip through global memory


## Solution

Fuse reduce_kernel_float and quantize_kernel_fp8 into a single reduce_kernel_fused_fp8 kernel that computes the normalized sum, applies the FP8 out_scale inline (1.0f / *fp8_out_scale_ptr), and writes uint8_t FP8 output directly via __nv_cvt_float_to_fp8 — eliminating the intermediate float buffer and one kernel launch entirely

## Key Lessons

- Fusing a quantization kernel into its producer eliminates both the intermediate global memory buffer (write + read round-trip) and one kernel launch — applies whenever a producer kernel writes float results that a consumer immediately reads and converts to a narrower type (FP8/INT8)
- Kernel fusion speedup is dominated by eliminated global memory traffic rather than compute savings — consistent ~1.42x across HEAD_SIZE=128 and HEAD_SIZE=256 confirms the savings come from removing the intermediate buffer and launch overhead, not from ALU reduction
