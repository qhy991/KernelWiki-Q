## Solution

Reduce pipeline stages from 4 to 2 and parameterize the stages count through function signatures, cutting the dominant stage-dependent shared memory terms (sh_a_size, sh_b_size, sh_s_size, sh_g_idx_size) in half so the config fits within Turing's 64KB shared memory limit

## Key Lessons

- Reducing pipeline stages from 4 to 2 halves the dominant shared memory terms (sh_a_size, sh_b_size) that scale linearly with stage count — on memory-constrained architectures like Turing (64KB shared), fewer stages is preferable to more prefetching when the kernel already approaches the smem ceiling
- Parameterize pipeline stage count as a function argument rather than a global constant, enabling runtime or compile-time selection of the optimal stages value per GPU architecture and tile configuration
