## Challenge

Optimization focus: Shared-memory B matrix access: XOR-based swizzle permutation (B_PAD=0) replaces linear indexing with column padding (B_PAD>=8) to avoid bank conflicts without wasting shared memory.
Problem before: When B_PAD >= 8, the kernel avoids shared-memory bank conflicts by padding each ro


## Solution

=2 and BN=128, the padding costs 128*(16+8)*2 = 6144 bytes vs 128*16*2 = 4096 bytes without padding. The wasted space limits the achievable K_STAGE count and reduces effective smem capacity.
Applied change: The swizzle approach (B_PAD=0) uses the XOR-based address transformation function swizzle_per

## Key Lessons

- Problem before: When B_PAD >= 8, the kernel avoids shared-memory bank conflicts by padding each row of B's shared memory with 8 extra half-values.
