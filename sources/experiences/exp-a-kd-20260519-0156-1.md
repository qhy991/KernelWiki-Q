## Solution

Split the K=2048 dimension into 4 slices of K=512 and launch 4 gemm_partial kernels concurrently on separate CUDA streams; each partial kernel computes a partial sum over its K slice, then a lightweight reduce_k kernel merges the 4 partial buffers into the final C matrix

## Key Lessons

- Split-K decomposition converts a single long-running kernel into multiple short kernels that run concurrently on separate SMs — effective when the grid size is small relative to available SMs and K is large enough to amortize the reduction overhead
- Use CUDA streams with cudaEventRecord/cudaStreamWaitEvent to overlap independent split-K partial GEMMs, then synchronize the default stream before launching the final reduction kernel
- The reduction cost O(M×N×K_SPLITS) is negligible compared to the GEMM cost O(M×N×K) when K_SPLITS << K, making split-K near-ideal for large K dimensions with small M×N output tiles
