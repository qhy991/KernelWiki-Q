## Challenge

Residual addition and RMS normalization are implemented as two separate kernel launches, requiring two full passes over the residual array from global memory (one write by the add kernel, one read by the norm kernel) plus double kernel launch overhead


## Solution

Fuse residual addition and RMS normalization into a single kernel where the first loop simultaneously computes the element-wise add and accumulates variance, and the second loop reads residual from L2 cache (just written in the first loop) instead of a separate global memory pass

## Key Lessons

- Fuse consecutive kernels that share an intermediate tensor into a single kernel to eliminate kernel launch overhead and keep the intermediate data in L2 cache — applies when one kernel writes and the next kernel reads the same buffer
- In a fused kernel, combine element-wise operations with statistical accumulation (e.g., residual add + variance) in the same loop iteration to reduce the number of global memory reads
- After a fused first-loop write, the second loop can read the just-written buffer as an L2-hot access instead of a full global memory round-trip, effectively halving bandwidth cost for the intermediate tensor
