## Solution

Replace scalar half2float loads with explicit vec_n_t<__half, VEC_SIZE> loads via reinterpret_cast, packing VEC_SIZE half values into a struct to force wide vector loads instead of relying on NVCC coalescing — however, NVCC already coalesces unrolled contiguous scalar fp16 loads into 128-bit transactions on A100, making the explicit vectorization redundant

## Key Lessons

- NVCC's coalescing optimizer already merges contiguous unrolled scalar fp16 loads into wide 128-bit memory transactions on sm_80+ — explicit vec_n_t reinterpret_cast vectorization provides no additional benefit when the scalar pattern is already coalesced
- When evaluating vectorized load optimizations, measure before committing — compiler coalescing can eliminate the expected speedup, making explicit vectorization redundant overhead
- The vec_n_t struct approach is useful when the compiler cannot prove contiguity (e.g., strided or indirect access), but for straightforward unrolled index patterns the compiler handles coalescing automatically
