## Challenge

Nibble encoding kernel processes one byte per thread using scalar load/store (in[i] and out[i]), resulting in 32 separate 1-byte memory transactions per warp that cannot fully coalesce into efficient 128-byte sectors, severely underutilizing memory bandwidth on a 16 MB bandwidth-bound workload


## Solution

Replace byte-by-byte scalar load/store with uint4 (16-byte) vectorized load/store: each thread loads 16 bytes via reinterpret_cast<uint4>, processes all bytes locally in registers using kNibbleLUT, then writes back as a single uint4 store, reducing memory transactions by 16x and enabling full 128-byte coalesced transactions per warp

## Key Lessons

- Replace scalar byte-by-byte load/store with uint4 (16-byte) vectorized access in bandwidth-bound elementwise kernels — a warp of 32 threads doing uint4 loads produces 4 coalesced 128-byte transactions (512 bytes total) instead of 32 poorly-coalesced single-byte transactions, reducing memory transaction count by up to 16x
- Vectorized access also reduces total thread count (fewer grid blocks needed), lowering kernel launch overhead and improving overall throughput on large bandwidth-bound workloads that exceed L2 cache capacity
