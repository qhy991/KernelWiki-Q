## Challenge

Optimization focus: Replace scalar / root_two with scalar * half_root_two in GELU operator() across all three template specializations (generic T, float, double).
Problem before: The GELU activation function computed scalar / root_two (i.e., scalar / sqrt(2)) as part of the erf argument. Floating-po


## Solution

ent overhead in a hot path.
Applied change: Replaced the division scalar / root_two with multiplication scalar * half_root_two, where half_root_two = sqrt(2)/2 = 1/sqrt(2). Mathematically these are identical: x / sqrt(2) = x * (sqrt(2)/2). The half_root_two constant was already defined in cutlass::c

## Key Lessons

- Expected impact: Eliminates one floating-point division per GELU evaluation, replacing it with a multiplication.
- The exact speedup depends on whether the division was on the critical path and the overall instruction mix in the epilogue, but microbenchmarks can quantify the per-element improvement.
