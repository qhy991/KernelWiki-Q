## Key Lessons

- Each thread must fetch inputs[i], scales[i], zeros[i] from global memory and store output[i], making the kernel memory-bound rather than arithmetic-bound.
- In the actual Machete kernel context, this optimization matters because the BF16 scale multiply occurs in registers after smem-to-rmem load, inside a tight loop feeding WGMMA tensor cores where arithmetic throughput is the bottleneck rather than global memory bandwidth.
