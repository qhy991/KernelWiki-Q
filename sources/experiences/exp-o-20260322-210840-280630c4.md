## Challenge

v1 kernel used scalar __ldg calls for each weight element, causing 4 separate load transactions


## Solution

Float4 vectorization for weight loads


### Key Changes

- Use float4 for vectorized memory access
- Fix __ldg usage: __ldg(reinterpret_cast<const float4*>(...))
- Single transaction for 4 floats instead of 4 separate loads


## Key Lessons

- __ldg requires the pointer inside, not outside: __ldg(ptr) not *__ldg(ptr)
- Float4 vectorization reduces memory transactions by 4x
- Correct __ldg syntax for vector types: __ldg(reinterpret_cast<const float4*>(ptr))
