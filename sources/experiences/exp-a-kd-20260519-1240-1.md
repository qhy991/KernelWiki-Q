## Solution

Increase YTILE from 2 to 4 via an adaptive heuristic (sYT=ceil(6912/(108*4))=16), doubling rows processed per wavefront per iteration and halving grid-stride loop iterations from 2 to 1, reducing outer-loop overhead while keeping total work per wavefront constant at 16 row-K groups

## Key Lessons

- Increasing YTILE to reduce grid-stride loop iterations yields only marginal gains (~1.5%) when total work per wavefront is unchanged — the outer-loop overhead saved is negligible compared to per-row computation cost, making this a weak tuning knob for reduction kernels with balanced work distribution
- Adaptive YTILE selection based on sYT=ceil(M/(CuCount*YTILE)) heuristics allows the launch config to self-tune to problem dimensions, but when sYT is already small (≤2), doubling YTILE only removes one grid-stride iteration whose cost is dominated by the inner K-loop
