## Challenge

RoPE kernel processes one (x1, x2) pair per thread using scalar float loads and stores, issuing two separate 4-byte memory transactions per pair and leaving memory bandwidth underutilized at larger hidden dimensions


## Solution

Pack two adjacent RoPE pairs into a single float4 (128-bit) vector so each thread loads/stores 4 floats in one coalesced transaction, halving the number of memory operations and improving arithmetic intensity at larger hidden dimensions

## Key Lessons

- Vectorize RoPE (or similar element-pair) kernels with float4 packing to halve memory transactions — each thread processes two adjacent (x1,x2) pairs in a single 128-bit load/store, which is especially effective at larger hidden dimensions where the workload per thread amortizes launch overhead
- When packing two RoPE pairs into float4, compute separate frequency exponents (exp_f_v, exp_s_v) for each pair because adjacent pairs map to different dimension indices (2k and 2k+1) in the rope theta schedule
