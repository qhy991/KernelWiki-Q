## Challenge

Single-query cached KV attention with BLOCK_SIZE=16 uses THREAD_GROUP_SIZE=2, requiring a shuffle reduction step in the QK dot product after FMA accumulation and producing twice as many KV cache blocks for a given sequence length, increasing loop iteration count and block-table indirection overhead


## Solution

Increase BLOCK_SIZE from 16 to 32, setting THREAD_GROUP_SIZE=1 so each thread independently computes its full QK dot product without intra-group shuffle reduction, halves the number of KV cache blocks to iterate (512 tokens → 16 vs 32), and widens the QK vector load from uint2 to uint4 for higher memory throughput

## Key Lessons

- Setting THREAD_GROUP_SIZE=1 (BLOCK_SIZE=32) eliminates the intra-warp shuffle reduction in QK dot product — each thread computes its QK independently, saving shuffle instructions per QK block and removing the thread_group_offset==0 output gate that discards half the computed results
- Larger BLOCK_SIZE halves the number of KV cache blocks for the same sequence length (512 tokens → 16 blocks vs 32), reducing loop iteration count and block-table indirection overhead in both QK and value accumulation phases
- Widening the QK vector load from uint2 (VEC_SIZE=4) to uint4 (VEC_SIZE=8) when THREAD_GROUP_SIZE=1 doubles memory throughput per load instruction for Q and K cache accesses
