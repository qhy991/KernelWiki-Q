## Challenge

Optimization focus: Changed kernel grid launch from per-batch-split (batch_size x num_splits) to per-token (num_tokens CTA blocks), and reduced CTA size from 1024 threads to 64 threads.
Problem before: The original kernel launched with grid(batch_size, num_splits) where num_splits was 2-16 depending


## Solution

4-thread CTA with only 576 scalar elements per entry left most threads idle (576/1024 < 1 iteration per thread). For extremely long sequences with many blocks, the nested loop created significant overhead and poor thread utilization.
Applied change: The rewrite launches grid(num_tokens) with block=6

## Key Lessons

- The token_to_seq mapping array replaces the batch-to-split bookkeeping, allowing direct batch_id lookup per token.
- Measured impact should be confirmed via point-level validation.
