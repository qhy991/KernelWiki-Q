## Challenge

The PTX predicated move approach is designed to avoid branch instructions, but for this simple conditional-zero pat


## Solution

Replace if-else branch with PTX inline assembly using predicate registers (setp.ne.b32 + @p mov.u32) to conditionally zero each mask value without explicit branch instructions, mimicking CUTLASS clear_mask_ pattern

## Results

- **speedup**: 0.999x (neutral)


## Key Lessons

- nvcc already converts trivial conditional-zero patterns (if (cond) { var = 0; }) into predicated or SEL instructions at SASS level — hand-written PTX predication produces identical code and yields no benefit for such simple patterns
- PTX-level predication via inline asm is only worthwhile when the compiler fails to eliminate branch overhead (e.g., complex multi-statement branches, function calls inside conditionals), not for single-statement conditional assignments
- Always profile before and after when attempting low-level PTX optimizations — the compiler optimizer may already emit equivalent SASS, making the manual PTX transformation purely cosmetic with neutral or even negative impact due to reduced readability
