## Challenge

Exceed performance_ratio=217.5 for fp32 RMS norm with batch=512, hidden=4096


## Solution

Persistent grid-stride kernel: each block processes multiple rows

## Key Lessons

- Persistent grid-stride loop is highly effective for RMS norm when the weight tensor fits in L2 cache (16KB for hidden=4096)
- Reducing grid size by 2x (each block handles 2 rows) amortizes weight load cost and improves L2 hit rate
- The key insight: weight tensor is read-only and small enough to stay in L2 across multiple row iterations within a block
- For batch=512, using 256 blocks instead of 512 blocks gave +13.5% speedup with no correctness issues
- The __syncthreads() guard before next row iteration is important for shared memory safety
