## Solution

Replace the shared-memory butterfly reduction with a parameterized reduce_sum<kNumThreads> template using __shfl_xor_sync warp shuffle intrinsics and an active_mask, eliminating all __syncthreads() barriers and shared-memory transactions for direct register-to-register data exchange

## Key Lessons

- Replace shared-memory butterfly reduction with __shfl_xor_sync warp shuffle for sub-warp reductions — eliminates __syncthreads() barriers and shared-memory transactions entirely, enabling direct register-to-register exchange that is orders of magnitude faster when the reduction group fits within a single warp
- Use an active_mask ((1u << kNumThreads) - 1u) with __shfl_xor_sync to correctly constrain sub-warp reductions to only the active threads, avoiding interference from inactive warp lanes
- Parameterized reduce_sum<kNumThreads> with static_assert for power-of-2 validation provides a reusable, compile-time-verified reduction primitive adaptable to any sub-warp group size
