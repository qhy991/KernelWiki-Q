## Challenge

256-bit predicated cache-global load stores results via reinterpret_cast<uint4*>(&out) pointer overlay on a template-parameterized PackedVec_Old struct, creating uint4 temporaries and relying on type-punning through incompatible pointer casts that constitute a strict-aliasing hazard


## Solution

Replace template-parameterized PackedVec_Old with a concrete u32x8_t struct whose d[8] array members are bound directly as inline asm output constraints on SM100+ and individually assigned from uint4 field decomposition on SM80, eliminating all reinterpret_cast type-punning and uint4 temporary construction

## Key Lessons

- Bind inline asm output constraints directly to destination struct member variables instead of loading into temporaries and then storing via reinterpret_cast overlay → eliminates intermediate register-to-uint4 moves and avoids strict-aliasing violations → applies whenever asm output fields map 1:1 to struct members
- Prefer concrete non-template types for fixed-layout containers (u32x8_t with d[8]) over template-parameterized wrappers (PackedVec_Old<Type> with raw[8]) → reduces compile-time instantiation overhead and makes data layout explicit → applies when the container holds a single known element type and the template parameter serves no structural purpose
