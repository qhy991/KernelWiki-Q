## Solution

Replace the unconditional deep clone with a lazy shallow pointer assignment when expert_map is absent: just assign d_topk_ids directly to the sort input variable, eliminating both the cudaMalloc allocation and the device-to-device cudaMemcpy entirely

## Key Lessons

- Use lazy/conditional deep clones instead of unconditional tensor.clone(): when the downstream consumer is read-only and no transformation is needed, a shallow pointer reference eliminates cudaMalloc + cudaMemcpy overhead entirely — applies to any MoE routing or data-parallel path where an expert_map or remapping table may or may not be present
- cudaMalloc per-iteration allocation + cudaFree deallocation creates repeated allocator and synchronization overhead; eliminating the allocation entirely yields outsized speedups (2x here) even for moderate tensor sizes, because the fixed cost of each GPU memory operation dominates the variable copy cost
