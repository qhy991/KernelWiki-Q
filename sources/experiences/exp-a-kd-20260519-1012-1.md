## Challenge

Two separate CUDA kernels perform SiLU+Mul activation and FP4 quantization sequentially: Kernel 1 writes activated PackedVec results to a global memory intermediate buffer, then Kernel 2 reads that same buffer for quantization, causing one full round-trip of unnecessary global memory traffic plus one extra kernel launch overhead


## Solution

Fuse silu_and_mul activation and quantization into a single kernel so that the activated PackedVec data stays in registers between the two operations, eliminating the intermediate global memory write-read round-trip and one kernel launch

## Key Lessons

- Fuse producer-consumer kernel pairs where the intermediate data fits in registers (PackedVec of 8 half values per thread) — eliminates one full global memory round-trip and one kernel launch, yielding significant speedup when computation per element is moderate relative to memory traffic
- Kernel fusion is most beneficial when intermediate data volume is large relative to per-element compute, as the saved global memory bandwidth dominates the improvement; for this 1024x256 problem the 1.36x speedup reflects both launch overhead elimination and memory traffic reduction
- When the intermediate result of one device function (silu_and_mul modifying a PackedVec in-place) is consumed immediately by another device function (simulated_quant_fp4 reading the same PackedVec), keeping data in registers is always preferable to writing through global memory — the compiler can keep the vector in register across both calls within a single kernel
