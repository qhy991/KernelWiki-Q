## Solution

Fuse both expert_offsets and blockscale_offsets prefix-sum computations into a single GPU kernel that reads problem_sizes1 from global memory and writes both output arrays in one serial loop, eliminating the cudaDeviceSynchronize barrier, D2H readback, CPU-side cumsum, and H2D upload entirely

## Key Lessons

- Fuse related prefix-sum computations into a single GPU kernel instead of splitting across GPU kernel + host-side CPU computation — eliminates cudaDeviceSynchronize stalls and D2H/H2D transfer overhead, which dominate latency for small-data kernels where the sync cost exceeds the actual computation cost by orders of magnitude
- When GPU kernel output must feed into a host-side computation, consider whether the host computation can be expressed as additional accumulators in the same kernel loop — lightweight scalar prefix-sums (ceiling-aligned offsets) add negligible register pressure and avoid the entire host-device round-trip
