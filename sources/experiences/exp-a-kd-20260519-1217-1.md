## Solution

Replace scalar variance loop with vectorized reads using vllm's vectorize_read_with_alignment<8> template (256-bit loads, 8 floats per transaction) to reduce global memory transactions in the variance accumulation phase, while the output normalization loop remains unchanged with scalar loads

## Key Lessons

- Vectorizing only one of two equally-sized memory-bound loops (variance accumulation) while leaving the other (output normalization) scalar means the bandwidth saving is halved — for balanced read-heavy kernels, vectorize all loops or the untialized portion will dominate runtime and negate gains
- For fp32 kernels with large hidden sizes (4096), the output normalization loop reads 16KB per token twice; partial vectorization of just one pass yields neutral speedup because the remaining scalar loop consumes the saved budget
- The vectorize_read_with_alignment helper handles misaligned pointers and non-power-of-two lengths gracefully, but the alignment check overhead and scalar fallback paths can reduce the effective speedup when cudaMalloc alignment is not guaranteed to match VEC_SIZE*sizeof(T)
