## Key Lessons

- 4D tensor-matrix multiplication can be implemented as batched GEMM with naive approach
- One thread per output element is sufficient for correctness in level_1 tasks
- Memory layout: A[b,i,j,l] and B[l,k] need proper indexing in kernel
- Output C[b,i,j,k] requires computing total elements = b*i*j*k for grid size
