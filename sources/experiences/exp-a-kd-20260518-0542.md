## Challenge

The before code uses device_vec<half2,2> (8 bytes, no alignas, default alignment=2) with manual static_cast pointer indexing by lane_id, while the after code uses aligned_vector<half2,2> (8 bytes, alignas(8) propagated from the aligned_storage<uint32_t,2> member) with warp::load/store. On sm_80 (Amp


## Solution

Replace scalar half loads with half2 (f16x4) vectorized loads, processing 4 half elements per thread per iteration via HALF2 reinterpret_cast, reducing loop iterations from 8 to 2 and widening each memory access to 64 bits for better L2 cache line utilization

## Results

- **speedup**: 1.017x (neutral)


## Key Lessons

- Vectorize half-precision memory accesses with half2 (64-bit) loads instead of scalar 16-bit loads in bandwidth-bound GEMV kernels — the wider transaction improves L2 cache line utilization and reduces total load instructions
- Collapsing multiple scalar loop iterations into fewer vectorized iterations reduces loop overhead (branch, index arithmetic, boundary checks) proportionally to the vectorization width
- Use reinterpret_cast<half2*>(&val)[0] (wrapped as HALF2 macro) to load two adjacent half values as a single half2 register without requiring data layout changes when the memory is naturally aligned
