## Solution

Replace scalar int32 stores with vectorized int4 stores, packing the fill value into a 4-element int4 vector and writing four int32 values per store instruction to reduce memory transaction count by 4x

## Key Lessons

- Vectorized int4/float4 stores reduce memory transaction count by 4x for contiguous fills, but the speedup is only measurable when the memory write phase dominates kernel execution time — at small buffer sizes (64K int32 = 256 KB), kernel launch overhead and cudaEvent synchronization noise mask any gains
- Always profile the ratio of compute/memory time to launch overhead before applying vectorized stores; for sub-millisecond kernels, the optimization is structurally correct but practically invisible in wall-clock measurement
