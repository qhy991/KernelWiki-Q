## Challenge

The after code introduces the greedy shortcut clause `target_prob_single >= threshold_single` with threshold_single=0.5. In the before code, every draft token must pass the stochastic check `coin < prob_acc` — even tokens with very high target-model probability (0.70-0.95) can be rejected if the ran


## Solution

Add a causal_q_begin boolean flag (default true) to enable runtime dispatch to CausalMask<true> (IsQBegin=true), which computes max_blocks_q as ceil_div((blk_coord_q+1)*tile_q, tile_k) without the offset_q=N_K-N_Q adjustment, avoiding unnecessary tile visits beyond the causal boundary for Q-at-beginning prefill

## Results

- **speedup**: 1.063x (improved)


## Key Lessons

- Use CausalMask<true> (IsQBegin=true) instead of CausalMask<false> when Q is at the beginning of the attention matrix — it computes max_blocks_q without the offset_q=N_K-N_Q adjustment, visiting significantly fewer K-tiles (250→10 trips for asymmetric_qk, 128→1 for small_q_large_cache)
- Provide a runtime causal mask variant flag rather than hardcoding CausalMask<false> — the optimal mask depends on Q position relative to K, which differs between prefill (Q at beginning) and decoding phases
