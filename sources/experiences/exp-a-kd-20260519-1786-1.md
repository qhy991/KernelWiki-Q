## Challenge

Tiled GEMM uses a shallow K-tile depth of 64 (tile 32x32x64), requiring K/64 shared-memory load rounds; this underutilizes arithmetic intensity because each loaded tile pair produces relatively few FMA operations before being evicted


## Solution

Double the K-tile depth from 64 to 128 (tile 32x32x128) to increase arithmetic intensity — each shared-memory load round now sustains twice as many FMA operations, halving the total number of load/synchronize rounds from K/64 to K/128

## Key Lessons

- Increasing K-tile depth (TK) halves the number of shared-memory load/synchronize rounds and raises arithmetic intensity, but the benefit is marginal when occupancy is already sufficient to hide shared-memory load latency — the deeper tile also doubles shared-memory footprint (32*128+128*32=8192 vs 32*64+64*32=4096 half elements), which can reduce per-SM available shared memory and limit occupancy on constrained GPUs
- Tuning a single tiling dimension in isolation may yield only ~1% improvement; meaningful gains in tiled GEMM typically require co-tuning TM, TN, and TK together to balance shared-memory usage, register pressure, and warp-level parallelism
