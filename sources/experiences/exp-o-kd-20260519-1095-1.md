## Solution

Set VPT=16 so each single thread owns all 16 elements of a row, eliminating cross-thread reductions entirely; max and sum are computed locally within the thread from its owned chunk, removing shared TempStorage, two __syncthreads() barriers, and the wasteful 256-thread BlockReduce on 16-element rows

## Key Lessons

- When the row length (num_cols) is small enough for one thread to own all elements (VPT == num_cols, THREADS_PER_ROW=1), eliminate block-level reductions entirely and compute max/sum locally — this removes shared TempStorage, __syncthreads() barriers, and idle threads
- For narrow-row kernels (e.g. MOE softmax with 16 experts), pack multiple rows per warp by setting ROWS_PER_WARP = WARP_SIZE / THREADS_PER_ROW, yielding better occupancy than the one-block-per-row approach
- The butterfly shfl_xor loop compiles away when THREADS_PER_ROW=1 (mask starts at 0), so the same template handles both multi-thread and single-thread-per-row cases without branching
