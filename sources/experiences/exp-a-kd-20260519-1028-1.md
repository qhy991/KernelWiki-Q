## Solution

Replace contiguous-block iteration with stride-based (grid-stride) iteration where thread i reads indices i, i+stride, i+2*stride, ... ensuring adjacent threads access adjacent memory locations for coalesced transactions and simplifying the loop condition to a single i < numel check

## Key Lessons

- Stride-based (grid-stride) iteration ensures adjacent threads read adjacent memory locations, producing coalesced memory transactions — applies to any kernel where threads iterate over a contiguous input array
- Grid-stride loops use a simpler single-condition bound (i < numel) versus the compound check (i < numel && i < start_idx + tokens_per_thread) required by contiguous-block partitioning, reducing per-iteration instruction count and branch overhead
