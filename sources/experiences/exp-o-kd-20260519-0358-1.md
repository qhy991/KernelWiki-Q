## Solution

Use float4 (128-bit) vectorized loads and stores via the FLOAT4 macro to process 4 float elements per thread in a single memory transaction, reducing total memory transactions by 4x and improving bandwidth utilization

## Key Lessons

- Use float4 (128-bit) vectorized loads/stores for memory-bound elementwise kernels to process 4 elements per thread and reduce memory transaction count by 4x — applies to any elementwise operation where input and output are contiguous fp32 arrays
- Scale thread index by the vector width (idx * 4 for float4) and reduce block size proportionally (blockSize / 4) to maintain the same total element coverage with fewer threads
