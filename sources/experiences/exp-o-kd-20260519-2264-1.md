## Solution

Change k_scale and v_scale from scalar float to float* pointer parameters so the kernel dereferences per-token or per-layer scale values from a GPU tensor, enabling dynamic scaling that adapts FP8 quantization accuracy to actual KV cache value magnitudes without changing the kernel execution path significantly

## Key Lessons

- Pass FP8 scale factors as pointers (float*) instead of scalar values to enable dynamic per-token or per-layer scaling — the kernel overhead is negligible (one pointer dereference) but quantization accuracy improves significantly when KV cache magnitudes vary across tokens
- A single global FP8 scale factor cannot accurately represent both small and large values; per-token or per-layer scales reduce quantization error by adapting to the actual dynamic range of each token's key/value data
