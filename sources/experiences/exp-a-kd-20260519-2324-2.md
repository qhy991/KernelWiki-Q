## Solution

Replace the two serial phases with a single CUB BlockScan ExclusiveSum call where each thread computes its own aligned count (DIV_CEIL×block_size) and all threads participate in an O(log n) parallel exclusive scan using warp-level shuffle primitives, eliminating both serial bottlenecks

## Key Lessons

- Replace serial prefix-sum loops with CUB BlockScan ExclusiveSum to achieve O(log n) parallel scan with full thread participation — applies whenever each thread holds one value to be scanned
- Fuse the alignment computation (DIV_CEIL×block_size) into the per-thread input before BlockScan to eliminate a separate serial cumsum phase entirely, reducing two serial passes to one parallel operation
