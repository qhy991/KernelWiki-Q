## Solution

Replace the entire manual scalar-decomposition pipeline (two calls to fp8_e4m3_to_float_manual with branches and powf) with a single vendor intrinsic: load the packed bits into __nv_fp8x2_e4m3 and static_cast to float2, which maps to a hardware-optimized conversion path that handles both elements without branching

## Key Lessons

- Use vendor intrinsics (__nv_fp8x2_e4m3 + static_cast<float2>) instead of manual bit-manipulation for FP8 type conversion — the intrinsic maps to hardware-optimized instructions that eliminate branch overhead and transcendental function calls (powf), yielding significant instruction count reduction
- Packed vector types (fp8x2, fp16x2) should be converted as a unit rather than decomposed into scalar elements — the hardware path for paired conversion is more efficient than two independent scalar conversions
