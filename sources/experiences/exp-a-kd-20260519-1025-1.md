## Solution

Fuse the rescale (divide-by-sum) and scale (multiply-by-factor) into a single kernel that performs both operations in one per-element loop iteration, eliminating one kernel launch and one full global-memory read-modify-write round-trip over the output array

## Key Lessons

- Fusing two elementwise kernels that traverse the same output array into a single kernel halves global memory traffic and removes one kernel launch overhead — applies whenever two kernels read-modify-write the same buffer with no inter-kernel data dependency beyond the buffer itself
- For small workloads (e.g. 24576 floats), kernel launch overhead (~5μs per launch) is a significant fraction of total latency, making fusion particularly impactful even when compute cost is trivial
