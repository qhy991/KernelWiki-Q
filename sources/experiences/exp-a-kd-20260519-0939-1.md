## Challenge

MoE fused gate topk selection loop runs topk=7 full iterations, each requiring a 32-element local argmax scan, warp-shuffle reduction, expert clearing, global memory writes, and a __syncthreads() barrier — the 7th iteration is expensive but could be replaced by a cheap shared-expert assignment when shared experts are available


## Solution

Reduce the cooperative topk loop from 7 to 6 iterations by computing topk_excluding_share_expert_fusion = topk - 1, then assign the 7th slot as a shared expert via a lightweight round-robin index (thread_row % n_share_experts_fusion) with weight = output_sum / routed_scaling_factor, eliminating one full argmax scan + warp shuffle + barrier cycle at the cost of two global memory writes

## Key Lessons

- Replace one expensive topk iteration (local argmax scan + warp shuffle reduction + barrier) with a lightweight shared-expert assignment (round-robin index + two global writes) when the expert selection policy allows non-routed experts — the fused assignment costs O(1) per row vs O(VPT × log(threads_per_row)) for a full topk round
- Remove redundant loop-invariant guards (if k_idx < topk inside a for-k_idx<topk loop) — they add a branch and indent level with zero semantic value, and their removal simplifies the code path for the compiler optimizer
- Redundant static_cast<float> chains on already-float values are no-ops at runtime but can confuse readers and may inhibit compiler analysis — use direct division for same-type arithmetic
