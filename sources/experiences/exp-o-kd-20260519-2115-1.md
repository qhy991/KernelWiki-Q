## Challenge

In the MoE block-alignment kernel launched as <<<1,1024>>>, all 1024 threads first write numel to every position of sorted_token_ids via a global-memory strided loop (16 iterations per thread for 16384 positions), serializing initialization before the expert counting phase can begin


## Solution

Overlap sorted_token_ids global-memory initialization with expert counting by launching <<<2,1024>>> instead of <<<1,1024>>>: block 1 handles the init writes while block 0 immediately begins counting via shared-memory atomicAdds, reducing total latency from init_time+count_time to max(init_time, count_time) with no inter-block synchronization needed since block 0 never reads sorted_token_ids

## Key Lessons

- Delegate independent global-memory initialization to a separate thread block (<<<2,N>>> with blockIdx.x guard + early return) to overlap it with computation on a different SM — works when the init writes to a buffer that the compute block never reads, eliminating the serialization penalty without any inter-block synchronization
- Prefer overlapping coarse-grained work phases across thread blocks rather than trying to overlap within a single block, since CUDA provides no intra-kernel inter-block synchronization primitives — the blockIdx.x guard pattern is simpler and more portable than persistent-thread or cooperative-group approaches
