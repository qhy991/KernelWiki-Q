## Solution

Reconfigure tile shape to BM=32, BN=64, BK=64, TM=1, TN=2 so that M=64 produces 2 fully-utilized blocks (32/32=100% fill) with zero padding waste, while the doubled BK=64 halves K-loop iterations and the reduced shared memory (12KB vs 16KB) enables higher SM occupancy

## Key Lessons

- Choose BM so that M is an integer multiple of BM to eliminate padding waste — when M is small (≤BM), reducing BM to a divisor of M turns partial tiles into fully active tiles, removing all guard overhead and wasted shared memory loads
- Increasing BK halves the number of K-loop iterations, improving the compute-to-memory ratio by amortizing shared memory load latency across more multiply-accumulate operations per tile — effective when shared memory budget permits the larger tile
- Reducing shared memory footprint (16KB→12KB) while maintaining 1024 threads per block allows more concurrent blocks per SM, increasing occupancy and hiding latency — small-M tiles that shrink shared memory unlock this occupancy benefit automatically
