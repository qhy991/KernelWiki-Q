## Solution

Add compile-time maxBlockSize template parameter to blockReduceSum and numLanes parameter to warpReduceSum; use if constexpr to skip shared-memory allocation, __syncthreads, and the second warp reduction entirely when maxBlockSize <= WARP_SIZE, falling through to a single parameterized warpReduceSum

## Key Lessons

- Use if constexpr with a compile-time block-size template parameter to eliminate shared-memory allocation, __syncthreads barriers, and redundant second-stage reductions when the block fits within a single warp — removes dead work that the compiler cannot always prove redundant at runtime
- Parameterize warpReduceSum with a numLanes template and static_assert it to a power-of-2 ceiling — enables correct sub-warp reductions for the second-stage cross-warp finalize without overshuffling inactive lanes
