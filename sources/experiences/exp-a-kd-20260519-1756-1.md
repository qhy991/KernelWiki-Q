## Solution

Lower the dispatch floor from std::max(32, ...) to std::max(16, ...) so that M<=16 problems correctly reach the TILE_M=16 branch, enabling smaller Cutlass TileShape allocations that halve shared memory usage and improve occupancy for small-batch GEMM

## Key Lessons

- When dispatching GEMM tile sizes by power-of-2 M, the floor value in std::max must match the smallest tile configuration available — a floor of 32 with a mp2<=16 branch creates unreachable dead code that silently degrades small-M performance
- The real benefit of correct tile dispatch appears in Cutlass shared-memory allocation and occupancy, not in simplified scalar-access kernels — TileShape<16,...> uses half the shared memory of TileShape<32,...>, allowing more CTAs per SM for small-batch workloads
- Early-return guards (if row>=M return) hide the cost of oversized tile dispatch in correctness benchmarks but waste thread slots — the tile-M dimension should tightly bound actual M to avoid launching threads that immediately exit
