## Solution

Replace the two-phase serial prefix-sum + serial aligned cumsum with a single CUB BlockScan ExclusiveSum that computes the aligned exclusive prefix sum in O(log 64) parallel steps using warp-level shuffle primitives, with each thread computing its own aligned_count before the scan

## Key Lessons

- Use CUB BlockScan ExclusiveSum instead of manual serial prefix-sum loops — it computes in O(log n) via warp-level shuffles and replaces both the cross-thread-row merge and the thread-0 serial cumsum with a single collective operation
- Fuse the block_size alignment into the scan input (compute aligned_count = DIV_CEIL(count, block_size) * block_size per thread before scanning) to eliminate a separate alignment pass
