## Solution

Replace scalar half loads with half2 vectorized loads (f16x4 pattern) using reinterpret_cast<half2*> via a HALF2 macro, processing 4 half elements per iteration through paired half2 loads and inline dot-product arithmetic, reducing loop iterations from 8 to 2 and improving L2 cache line utilization via wider 64-bit memory transactions

## Key Lessons

- Use half2 vectorized loads via reinterpret_cast<half2*>(&val)[0] (HALF2 macro) to fetch two half elements per 64-bit load in GEMV kernels, doubling effective memory throughput — applies when data is contiguous and K is aligned to at least 2
- Reducing loop iterations through wider vectorized loads (scalar→f16x4 cuts iterations by 4x) eliminates per-iteration overhead (branch prediction, index arithmetic) — most impactful when the original iteration count is modest (4–16), where overhead dominates arithmetic latency
- Pair two half2 loads per iteration (f16x4) and compute the 4-element dot product inline via .x/.y field access to avoid extra packing/unpacking instructions — keeps register pressure low while maximizing compute-to-load ratio
