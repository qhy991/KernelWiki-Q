## Solution

Replace self_counter + peer_counter[2] with three dedicated arrays: start and end for the two barrier phases and _flag as a shared per-block counter updated only by thread 0, eliminating modulo-2 alternation, making the two-phase barrier intent explicit, and preventing per-thread counter drift

## Key Lessons

- Replace modulo-2 alternating arrays with semantically named arrays (start/end) for multi-phase barrier synchronization — eliminates error-prone index arithmetic and makes the two-phase protocol explicit and self-documenting
- Use a shared per-block counter (_flag) updated by a single designated thread (thread 0) instead of per-thread self_counter arrays — prevents counter drift across threads and eliminates redundant read-modify-write operations, yielding significant speedup in barrier-heavy workloads
- Consolidating counter management from per-thread updates to a single-thread responsibility reduces total memory footprint (864 → 612 FlagType entries here) and cache pressure in multi-GPU barrier synchronization
