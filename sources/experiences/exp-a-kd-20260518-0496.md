## Solution

The after source pre-allocates the same-size buffers once at lines 107-110, then the benchmark loop at lines 127-132 only launches the kernel. With 4096 tokens, each thread processes 64 tokens instead of 4, so the kernel takes ~0.60ms per invocation (3x longer than the 256-token case at 0.20ms). ...

## Key Lessons
