## Solution

Replace scalar / root_two with scalar * half_root_two (where half_root_two = sqrt(2)/2 = 1/sqrt(2)), a mathematically equivalent strength-reduction that swaps a costly SFU division for a single FFMA multiplication, using the already-defined cutlass::constants::half_root_two constant across all three specializations

## Key Lessons

- Replace floating-point division by a constant with multiplication by its reciprocal (x / c → x * (1/c)) when the reciprocal is precomputed — GPU SFU division has 4x-8x lower throughput than FFMA multiplication, making this a low-risk, universally applicable strength-reduction for hot per-element paths
- Check whether a constant library (e.g., cutlass::constants) already provides the reciprocal form (half_root_two, half_pi, etc.) before introducing a new constant — avoids duplicate definitions and leverages well-tested values across all precision types
