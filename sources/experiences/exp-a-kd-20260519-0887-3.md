## Challenge

The baseline kernel launches one block per group with 128 threads per block (1 group × 128 threads), resulting in low GPU occupancy when many small independent groups leave thread resources underutilized


## Solution

Pack 16 groups per block using 16 threads per group (16 × 16 = 256 threads/block), mapping each group to a warp slice via threadIdx.x division/modulo, which doubles thread count per block and improves SM occupancy by keeping more warps active

## Key Lessons

- Pack multiple small independent work-groups into a single block (groups_per_block × threads_per_group) when each group needs only a few threads — increases threads per block and improves SM occupancy without changing per-group logic
- Use threadIdx.x division/modulo to partition block threads into per-group lanes: local_group_id = threadIdx.x / threads_per_group, lane_id = threadIdx.x % threads_per_group
