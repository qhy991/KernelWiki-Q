## Solution

Replace softmax scoring with per-element sigmoid scoring (1/(1+exp(-x))) via a compile-time ScoringFunc enum, eliminating all warp-level reductions; however the 4 added per-element divisions offset the savings from removing 2 shuffle ops and reduction loops, yielding neutral performance

## Key Lessons

- Replacing softmax with sigmoid in topk gating eliminates warp-level reductions but introduces per-element divisions (1/(1+exp(-x))) that offset the savings — net effect is neutral when THREADS_PER_ROW is small and reduction iterations are few
- At small THREADS_PER_ROW counts (e.g., 2 with 64 experts and VPT=4), butterfly reductions cost only log2(2)=1 shuffle iteration each, so removing them provides negligible arithmetic savings relative to memory traffic dominant workloads
- Use compile-time enum dispatch (if constexpr) to select scoring functions without runtime overhead, enabling flexible kernel templates that support both softmax and sigmoid paths in a single implementation
