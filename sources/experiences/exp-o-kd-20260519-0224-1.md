## Challenge

The is_source_needed() method in LinearCombinationClamp relies solely on a runtime comparison beta_ != ElementCompute(0) to decide whether the source tensor (C matrix) must be loaded during the GEMM epilogue, preventing the compiler from eliminating source-loading code paths at compile time even when alpha-only scaling is intended


## Solution

Add a ScaleType::Kind template parameter (with default ScaleType::Default for backward compatibility) and replace the single runtime check with compile-time if-statements that branch on the Scale enum: NoBetaScaling returns true, OnlyAlphaScaling returns false, and Default falls through to the original runtime beta_ != 0 check, enabling the compiler to eliminate unreachable source-loading branches via dead code elimination

## Key Lessons

- Replace runtime branching with compile-time template parameters (using an enum with a default value) for epilogue scaling modes — the compiler eliminates entire source-loading code paths via dead code elimination, reducing global memory reads from ~48MB to ~32MB per invocation when alpha-only scaling is sufficient
- When adding compile-time optimization knobs to existing CUTLASS epilogue functors, use defaulted template parameters to preserve backward compatibility — call sites that don't specify the new parameter continue using the original runtime behavior
