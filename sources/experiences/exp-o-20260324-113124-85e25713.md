fp32_rms_norm_qwen2_5_7b_hs3584

Cache input float4 values in registers during the first pass (sum_sq computation). Reuse cached values for output computation instead of re-reading from global memory. 256-thread kernel caches up to 4 float4 per thread; 512-thread kernel caches up to 2 float4 per thread. Combined with existing float4, warp shuffle, __frsqrt_rn optimizations.

## Key Lessons

- Register caching is effective for kernels that read input multiple times - eliminates redundant global memory reads
- For hidden_size=3584 with 256 threads: each thread handles 3-4 float4 elements, feasible to cache in registers
- Even highly optimized baselines (v7) can benefit from register caching when the kernel has multiple read passes
- Use explicit variables (c0,c1,c2,c3) rather than arrays for register caching to ensure compiler keeps values in registers
- Adaptive thread count (256 for small batch, 512 for large batch) remains important for occupancy
