## Solution

Reduce kBlockSizeI from 128 to 64 so kThreads drops from 256 (8 warps) to 128 (4 warps), matching the actual kHeadDim=64 work dimension and halving the reduction tree depth and __syncthreads calls

## Key Lessons

- Match block size (threads per block) to the actual data-parallel dimension when the work per thread is gated by a dimension constant — over-provisioning threads wastes warps on idle threads and adds unnecessary synchronization rounds in shared-memory reductions
- Halving the active thread count in a reduction kernel reduces shared-memory footprint, cuts reduction tree depth by one level, and eliminates one __syncthreads barrier per kernel launch, compounding into significant speedup for reduction-heavy workloads
