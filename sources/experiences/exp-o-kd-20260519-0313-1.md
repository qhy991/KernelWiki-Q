## Challenge

The fallback gated activation kernel only produces the activated tensor; AWQ prequant scaling for the FC2 layer requires a separate per-channel scaling kernel launch, doubling memory traffic over the activation output when inter_size is large


## Solution

Reuse the expert-id lookup already needed for gated parameter selection to also resolve per-expert AWQ prequant scale rows, then multiply the scale inside the same vector loop that computes SwiGLU/GEGLU outputs — eliminating the separate scaling kernel pass entirely

## Key Lessons

- Fuse per-channel or per-expert scaling into an existing activation loop when the expert-id lookup is already available — avoids a separate kernel launch and an extra full pass over the activation output, with the biggest gains when inter_size makes that pass expensive
- Share an expensive computation (expert-id lookup via binary search) across multiple consumers (gated parameters + prequant scales) by hoisting it into a common guard condition, trading one extra branch for the elimination of an entire kernel launch
- Guard new functionality behind optional pointers and flags so the fused path degrades gracefully — the prequant_scale_vec nullptr check ensures no overhead when AWQ scaling is not in use
