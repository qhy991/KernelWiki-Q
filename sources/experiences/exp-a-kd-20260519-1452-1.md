## Challenge

Multi-slice GEMM reduction uses an FP16 intermediate buffer to accumulate partial sums across threadblocks, requiring __float2half() and __half2float() conversion instructions on every read and write of each slice, adding arithmetic overhead and accumulating half-precision rounding error across slices


## Solution

Replace the half-typed intermediate buffer with a float-typed buffer, eliminating both __float2half() and __half2float() conversion instructions per element per slice and preserving full FP32 precision throughout the multi-slice accumulation chain

## Key Lessons

- Eliminating FP16 intermediate buffers in multi-pass reduction chains removes __float2half/__half2float conversion overhead and dramatically improves numerical precision (max_rel_error 0.058% → 0.0004%), even when runtime is neutral for small buffers that fit in L2 cache
- For small intermediate buffers (a few KB) that reside in L2 cache, the 2× memory footprint increase from half→float is negligible compared to the conversion instruction savings, making FP32 intermediates strictly preferable for accuracy-critical accumulation patterns
