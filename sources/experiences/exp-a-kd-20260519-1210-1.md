## Solution

Overlap sh_b and sh_red to the same shared memory region (both alias smem + TB_M) since the phase-1 data buffer and phase-2 reduction buffer are separated by __syncthreads and never live together, reducing total shmem from 1600 to 1056 bytes

## Key Lessons

- Aliasing phase-disjoint shared memory buffers (separated by __syncthreads) to the same region reduces per-block shmem footprint, but only yields occupancy gains when shared memory — not thread count — is the occupancy limiter
- When 256 threads/block already caps occupancy at 8 blocks/SM on the target architecture, further reducing shmem below the next occupancy threshold has no practical benefit — profile occupancy before optimizing shared memory layout
