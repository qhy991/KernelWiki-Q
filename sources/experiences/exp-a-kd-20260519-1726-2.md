## Challenge

The quantization loop processes elements one-at-a-time with scalar loads and per-element division by scale, underutilizing available memory bandwidth for the int8 output write phase


## Solution

Replace the scalar quantization loop with vectorize_with_alignment<16> to process 16 elements per vector iteration for improved memory throughput, and precompute the reciprocal inv_s = 1.0/scale to replace per-element floating-point division with multiplication

## Key Lessons

- Vectorized memory access (vectorize_with_alignment<16>) processes multiple elements per memory transaction, improving bandwidth utilization for elementwise transformations — especially effective when input/output types differ and misalignment is handled
- Precompute the reciprocal (1/scale) once and multiply per-element instead of dividing per-element — avoids the much slower FP division instruction on GPU hardware, applies whenever the divisor is loop-invariant
