## Challenge

The reshape_and_cache_flash kernel uses a single scalar loop for both NHD and HND KV-cache layouts, computing head_idx and head_offset via per-element division/modulo and issuing scalar loads/stores. For HND layout where head_stride != head_size, heads are non-contiguous in cache, so the scalar loop misses vectorization opportunity within each contiguous head_size segment.


## Solution

Distribute heads across warps (one warp per head iteration) and use vectorize_with_alignment with stride=32 to perform 128-bit vector loads/stores within each contiguous head_size segment, eliminating per-element division/modulo and pre-computing per-head source/destination pointers outside the inner vectorized loop.

## Key Lessons

- For HND (strided-heads) KV-cache layouts where heads are non-contiguous, distribute heads across warps rather than using a flat thread index — each warp owns one head, enabling vectorized copy within the contiguous head_size segment while respecting inter-head strides
- Pre-computing per-head source and destination pointers outside the inner copy loop eliminates per-element division/modulo arithmetic (i / head_size, i % head_size), reducing integer ALU pressure in memory-bound kernels
- Use vectorize_with_alignment with a warp-wide stride (32) to achieve 128-bit vector loads/stores for the aligned portion of each contiguous segment — applicable whenever a warp operates on a contiguous memory region
