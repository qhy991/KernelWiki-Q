## Solution

Replace fixed ldsm4 with templated ldsm<count> using if constexpr to select ldmatrix.x1, x2, or x4 at compile time, matching the exact number of matrix rows needed by the tile size with zero runtime overhead

## Key Lessons

- Templatize ldmatrix load count via a compile-time parameter and dispatch with if constexpr — the compiler eliminates unused branches, producing the exact ldmatrix.x1/x2/x4 PTX with zero overhead while adapting to tile dimensions at compile time
- Match ldmatrix width to actual tile row count: for small M-block sizes (e.g., m_block_size_8 needs only 2 rows), using ldmatrix.x2 instead of x4 halves shared memory bandwidth per A-fragment load and frees 2 registers per thread per pipeline stage, directly improving occupancy for small-batch GEMM kernels
