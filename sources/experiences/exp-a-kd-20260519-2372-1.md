## Solution

Templatize the kernel on THREADS_PER_SF and guard the shuffle with if constexpr so that when only one thread covers the SF vector (THREADS_PER_SF==1) the shuffle is elided at compile time; for THREADS_PER_SF==2 the shuffle still executes and is functionally identical to the unconditional version

## Key Lessons

- Use if constexpr with template parameters to conditionally emit warp shuffle instructions at compile time — when THREADS_PER_SF==1 the shuffle is completely elided, eliminating a redundant cross-lane exchange without any runtime branch cost
- Guarding shuffle operations behind compile-time conditions enables a single kernel template to handle both single-thread and multi-thread SF vector configurations without sacrificing performance in either case
