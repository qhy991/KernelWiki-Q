## Solution

Replace hardcoded constants with WARP_SIZE/2 for the initial mask and introduce VLLM_SHFL_XOR_SYNC macro that maps to __shfl_xor_sync on NVIDIA or __shfl_xor on ROCm, producing identical SASS on CUDA hardware while enabling AMD GPU portability

## Key Lessons

- Replace hardcoded warp-size constants (16, 32) with WARP_SIZE/2 calculations so the same reduction pattern adapts to AMD GFX9 where warp size is 64 — produces identical SASS on NVIDIA with zero performance cost
- Wrap vendor-specific shuffle intrinsics (__shfl_xor_sync on CUDA, __shfl_xor on ROCm) in a portable macro so cross-platform kernels compile without #ifdef guards scattered through the reduction loop body
