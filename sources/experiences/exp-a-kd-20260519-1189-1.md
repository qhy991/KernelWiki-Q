## Challenge

Scalar stride loop in histogram kernel loads one float per thread iteration, motivating exploration of vectorized float4 loads to improve memory throughput


## Solution

Introduced vectorized_process helper using float4 wide loads with alignment handling (skip_cnt, remainder), but the overhead of pointer alignment computation, union-based decomposition, and branch logic for edge elements outweighs memory bandwidth gains for 32K floats (~128KB) where L1 cache covers the entire working set

## Key Lessons

- Vectorized (float4) loads add alignment handling overhead (skip_cnt computation, reinterpret_cast, union decomposition, remainder branches) that does not amortize for small working sets (~128KB) where L1 cache already provides effective coalescing of scalar loads
- When each thread processes few elements (~64 for 32K data / 512 threads), the instruction overhead of vectorization scaffolding dominates any bandwidth savings — prefer simple stride loops that let the hardware cache handle coalescing naturally
