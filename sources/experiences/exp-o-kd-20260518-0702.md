## Solution

Applied change: The rewrite uses cp_async4_pred to bulk-load all sorted_token_ids int4 values into shared memory in one asynchronous batch (moe_block_size/4 threads each load one int4). After cp_async_fence + cp_async_wait + syncthr

## Key Lessons

- The topk-weight and expert-id division is also distributed across all moe_block_size threads rather than only tid4-aligned threads.
- This meant all moe_block_size/4 int4 values were read from global memory one at a time in a serial break-on-first-invalid pattern, creating a global-memory access bottleneck and a sequential dependency chain that prevented parallel counting.
