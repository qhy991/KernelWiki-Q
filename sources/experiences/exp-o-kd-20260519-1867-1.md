## Challenge

Mixture-of-experts top-2 routing uses two separate kernel launches: a compute_scores_with_bias_kernel that writes sigmoid(scores)+bias to an intermediate global-memory buffer (2×TOTAL_ELEMENTS float transactions), followed by a topk_with_k2 kernel that reads from that buffer — incurring an extra kernel launch overhead and a full round-trip through global memory for data that is consumed exactly once.


## Solution

Fuse sigmoid_accurate() and per-group bias addition directly into the top-2 selection loop inside a single kernel, eliminating the intermediate scores_with_bias buffer allocation and the second kernel launch — the per-element cost of 2–3 extra SFU/arithmetic instructions is far smaller than the saved 2×TOTAL_ELEMENTS global-memory write+read round-trip.

## Key Lessons

- Fusing elementwise pre-processing (sigmoid + bias) into a reduction/topk loop eliminates an intermediate global-memory buffer and an entire kernel launch, saving 2N global-memory transactions at the cost of a few extra ALU instructions per element — net win when buffer size dominates compute
- When num_experts_per_group ≤ WARP_SIZE each lane processes at most one element so the per-element SFU+ALU chain (tanhf + multiply + add) is fully pipelined and does not increase critical-path latency of the warp reduction
