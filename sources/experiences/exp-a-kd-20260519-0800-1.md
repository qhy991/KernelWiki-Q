## Solution

Replace scalar stride-16 __half loads with 16-byte vectorized loads via a half8_vec_t struct using reinterpret_cast<uint4> to load 8 __half values per instruction, then iterate over the 8 elements in an unrolled inner loop for both absmax scan and quantization passes

## Key Lessons

- Naive vectorization via reinterpret_cast<uint4> loads can regress performance when nvcc already auto-coalesces simple stride-16 scalar patterns — the compiler's scalar path optimizations (hardware fp16-to-float conversion, auto-coalescing) can outperform explicit vec_t loading for small workloads with only 8 iterations per thread
- Vectorized loads increase register pressure by forcing 8 simultaneous __half-to-float conversions and intermediate values into an unrolled inner loop, which may cause register spills that degrade performance more than the reduced load instruction count saves
- A simple regular stride access pattern (stride equal to warp subgroup size) is often better optimized by nvcc than an explicitly vectorized struct-based approach, especially when the total iteration count per thread is small (8 iterations)
