## Solution

Warp-level multi-token packing assigns each warp to one token using warpReduceMax and register-local scales, eliminating __syncthreads and blockReduceMax overhead but reducing CTA count from 32 to 4 (32 tokens / 8 per CTA), causing severe GPU under-utilization and 34x latency regression for small batches

## Key Lessons

- Warp-level multi-token packing in per-token quantization eliminates blockReduceMax and __syncthreads overhead but reduces CTA count by kTokensPerCTA — only beneficial when num_tokens is large enough (>= sm_count * 2 * kTokensPerCTA) to maintain sufficient grid-level parallelism across SMs
- For small-batch quantization workloads, the baseline one-token-per-CTA approach provides better GPU utilization through more independent CTAs that can saturate multiple SMs, even with the cost of block-level synchronization
- Grid-level parallelism (total CTA count) dominates over per-CTA synchronization overhead when the CTA count drops below the number of SMs — 4 CTAs cannot saturate even a single SM's concurrent-CTA capacity on modern GPUs
