## Solution

Reorder the shared memory layout so that all 16 INT4 elements owned by one thread occupy 8 contiguous bytes (at offset thread_idx_in_row * 8 within the row), enabling a single 64-bit vectorized load instead of 16 scattered byte loads, which coalesces into fewer wider shared memory transactions and reduces bank-conflict stalls

## Key Lessons

- Reorder shared memory layout to match the WGMMA thread-value access pattern so each thread's elements are contiguous — this converts many narrow scattered loads into one wide vectorized load, reducing bank-conflict stalls and instruction count
- For INT4 sub-byte types, host-side reorder (reorder_data) permutes elements at packing granularity so that all values owned by one thread occupy contiguous packed bytes — the reorder cost is amortized once at load time while the kernel benefits from contiguous access on every iteration
