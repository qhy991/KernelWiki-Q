## Solution

Replace 2D batch-split grid with 1D flat warp-per-token model where each warp processes exactly one token cooperatively; binary search on workspace_starts maps flat token index to request ID with O(log num_reqs) overhead, enabling uniform work distribution where every launched warp produces real output

## Key Lessons

- Warp-per-token scheduling (1 warp = 1 token) eliminates serial token loops and guarantees all launched warps produce real output, avoiding wasted SM slots from uneven split sizes in 2D batch-split grids — applies whenever processing variable-length sequences with per-token work
- Binary search on workspace_starts replaces direct blockIdx-based batch indexing in flat grids, adding only O(log num_reqs) overhead per warp while enabling uniform 1D work distribution across all tokens
- Using 256-thread blocks (8 warps) instead of 576-thread blocks aligns with warp-size multiples, avoids partial-warps, and gives the scheduler more block-level granularity for occupancy
