## Challenge

The before constructor_loop_before.cu computes each thread's shared-memory origin offset via a triple-nested unrolled loop iterating over (inst_m_idx, inner_idx, access_m_idx) with a conditional branch (if access_idx == ldsm_vec_num) to select the matching iteration


## Solution

The after constructor_loop_after.cu replaces this with three direct integer-division/modulo operations on ldsm_vec_num to extract the same indices.
