## Solution

Fuse the separate GPU expert_offsets kernel and the CPU blockscale_offsets computation into a single GPU kernel (compute_expert_blockscale_offsets) that reads problem_sizes1 from device memory and computes both prefix sums in one serial loop, eliminating the cudaDeviceSynchronize, D2H/H2D transfers, and host-side computation entirely

## Key Lessons

- Fuse GPU kernel + host-side CPU computation into a single kernel when they share the same input data — eliminates cudaDeviceSynchronize stalls and D2H/H2D transfer overhead, which dominate latency for small-data workloads
- When CPU-GPU sync is the bottleneck (not compute), the speedup from kernel fusion can be orders of magnitude rather than incremental — the ~271x speedup here comes from removing a ~2ms sync stall, not from faster arithmetic
- Prefix-sum computations over small arrays (64-256 elements) are cheap enough to run on a single GPU thread in a serial loop — the overhead of launching a parallel scan would exceed the computation cost
