## Solution

Add sm_occupancy > 1 guard before the modulo-based wave-boundary condition so that when sm_occupancy==1 the scheduler skips the SK wave formation branch and falls through to the correct fallback that combines the last full wave with the partial wave

## Key Lessons

- Guard modulo-based wave-boundary conditions with an sm_occupancy > 1 check — when sm_occupancy==1, x%1==0 always equals 1-1, making the condition vacuously true and triggering incorrect scheduling paths
- Integer modulo tautologies (n % 1 == 0) are a common source of silent bugs in occupancy-aware dispatch logic — always verify boundary conditions for sm_occupancy values of 1 and 2 where modulo arithmetic degenerates
