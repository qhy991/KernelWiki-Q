## Challenge

Implement kernel following the CUTLASS_device_function interface pattern with constraints: Element can be cutlass::Array<half_t, kN> or cutlass::Quaternion<T>; Operator must be a CUTLASS primitive (plus, minus, multiplies, divides, multiply_add); kN is the array length parameter

## Key Lessons

- Interface pattern 'CUTLASS_device_function': the implementation follows the specified skeleton contract. Key constraints: Element can be cutlass::Array<half_t, kN> or cutlass::Quaternion<T>; Operator must be a CUTLASS primitive (plus, minus, multiplies, divides, multiply_add)
- CUTLASS-based implementations use template parameters for compile-time specialization: data types, layouts, tile sizes, and warp configurations are all template arguments rather than runtime parameters
