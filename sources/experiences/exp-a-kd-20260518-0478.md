## Challenge

The after kernel caches each fp32 input element in the xi_cache register array during Pass 1 and reuses those registers in Pass 2, eliminating the second global-memory load of xr[i] that the before kernel performs


## Solution

Consolidate four separate half2 loads/stores into a single 128-bit LDST128BITS (float4 reinterpret) load/store and replace per-element scalar ELU calls with a pragma-unrolled loop, reducing instruction count and boundary checks

## Results

- **speedup**: 1.054x (improved)


## Key Lessons

- NVCC at -O3 can coalesce four consecutive half2 loads from contiguous aligned addresses into a single 128-bit memory transaction, making explicit LDST128BITS packing yield negligible gains on memory-bound elementwise kernels when the original access pattern is already coalesced
- Explicit 128-bit packed load/store reduces source-level instruction count and boundary checks (4→1), which is cleaner code but may not translate to measurable speedup when the compiler already merges the transactions
- For memory-bound activation kernels processing 8 half elements per thread, the bottleneck is memory bandwidth not instruction count — coalescing already-sequential half2 accesses provides no additional bandwidth reduction
