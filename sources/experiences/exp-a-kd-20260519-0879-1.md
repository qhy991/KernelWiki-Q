## Solution

Replace scalar int32_t stores with 4-wide AlignedArray<int32_t,4> vectorized stores via reinterpret_cast<Vec*>, writing 4 elements per thread iteration to reduce total store instructions and loop iterations by 4x

## Key Lessons

- CUDA's memory coalescing already merges adjacent 32-bit scalar writes from consecutive warp threads into 128-byte wide transactions — vectorized stores provide diminishing returns when warp-level coalescing already achieves near-full memory bus utilization
- For small arrays (<64KB), kernel launch overhead (~5μs) dominates execution time, making memory-level optimizations like vectorized stores invisible in benchmarks — vectorization benefits appear only when the workload is throughput-bound on large arrays
- Benchmarking fill/memset kernels on tiny arrays (4K elements) cannot distinguish scalar vs vectorized approaches because launch overhead masks the actual store latency — use larger arrays (≥1M elements) to measure memory throughput effects
