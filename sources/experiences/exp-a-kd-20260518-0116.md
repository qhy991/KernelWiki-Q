## Challenge

In the before code, atomicMaxFloat receives max_val * scaling, which introduces an extra FMUL before each atomicCAS loop iteration. The atomicMaxFloat implementation uses an atomicCAS loop that may retry, and each retry recomputes fmaxf(val, old_val) where val already includes the scaling multiply


## Solution

The after code pre-scales the fragment with scaling * kLog2e before the max-reduction, so atomicMaxFloat receives the already-scaled max value directly.
