## Challenge

MoE gating uses two separate kernel launches — a scalar sigmoid kernel writes to a 1MB global-memory workspace buffer (64 experts × 4096 rows × 4 bytes), then a TopK kernel reads back from it — incurring a full global-memory round-trip and double launch overhead


## Solution

Fuse sigmoid and TopK into a single kernel where each thread computes sigmoid on its loaded chunk and keeps the result in registers (row_chunk[]), then performs TopK on the same register-resident data — eliminating the 1MB workspace allocation, the global-memory write/read cycle, and the second kernel launch overhead

## Key Lessons

- Fuse producer-consumer kernels when intermediate data fits in registers — eliminating the global-memory workspace round-trip can yield order-of-magnitude speedups by removing both the write-pass latency and the allocation overhead
- Keeping intermediate results register-resident across computation phases (sigmoid → TopK) avoids redundant global memory traffic and lets the GPU schedule the fused work more efficiently
