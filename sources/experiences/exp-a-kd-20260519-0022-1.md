## Solution

Replace max() with fmax() in both the per-thread LSE maximum loop and the warp shuffle reduction — fmax(NaN, x) returns x (the non-NaN operand), so NaN values are automatically skipped and the correct maximum among valid elements is computed

## Key Lessons

- Use fmax()/fmin() instead of max()/min() when computing reductions over data that may contain NaN — fmax(NaN, x) = x while max(NaN, x) is NaN, so fmax provides NaN-safe semantics at negligible performance cost
- Correctness fixes like NaN-safe reduction operators may show neutral performance but prevent silent numerical corruption in attention kernels with variable-length or padded sequences
