## Solution

Add cudaMemsetAsync to zero the output buffer before launching split accumulation kernels, ensuring correct atomic accumulation when multiple splits contribute partial results

## Key Lessons

- When using atomic accumulation with split-KV or fused reduction, the output buffer must be zeroed via cudaMemsetAsync before launching accumulation kernels — stale data from prior runs will otherwise be added to new partial results, producing silently incorrect output
- cudaMemsetAsync adds measurable overhead in microbenchmarks with lightweight kernels (15.5% regression here), but the cost is negligible relative to real FMHA kernel execution time — correctness outweighs the small latency increase
- Use cudaMemsetAsync (asynchronous, stream-ordered) rather than cudaMemset (synchronous) for buffer zeroing before kernel launches, as it avoids an unnecessary device synchronize and overlaps with preceding operations on the same stream
