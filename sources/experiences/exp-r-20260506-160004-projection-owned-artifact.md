Some artifacts (like verification results, canonical state files) are projection-owned and cannot be edited with generic filesystem tools. The agent must use the verifier, record_benchmark_artifacts, or state_core APIs.

## Challenge

Agent tries to edit projection-owned artifacts (verification results, canonical state files) via mcp_filesystem_write_file or mcp_filesystem_edit_file, but gets blocked: 'Canonical state artifacts are projection-owned and cannot be edited with generic filesystem tools.'

### Symptoms

- `Canonical state artifacts are projection-owned and cannot be edited with generic filesystem tools`
- `Use the verifier, record_benchmark_artifacts, or state_core APIs instead`
- `Agent wastes steps trying to manually edit these files`

## Solution

Do not try to edit projection-owned artifacts directly. Use the appropriate API for each type of artifact.

### Implementation

```cuda
1. Identify which files are projection-owned (verification results, canonical state)
2. For verification results: run the canonical verification command
3. For benchmark artifacts: use record_benchmark_artifacts tool
4. For state updates: use state_core APIs
5. Only source code files (kernel.cu, main.cpp, solution.json) can be edited directly
6. If you need to update verification results, re-run verification
```

## Key Lessons

- Projection-owned artifacts cannot be edited with generic filesystem tools.
- Verification results are generated by running verification, not by manual edits.
- Use the appropriate API for each artifact type.
- Only source code files can be edited directly via filesystem tools.
- Trying to edit projection-owned files wastes steps and never succeeds.
