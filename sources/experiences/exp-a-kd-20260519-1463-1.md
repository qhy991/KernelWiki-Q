## Challenge

Two-pass int8 group quantization kernel reads half-precision input from global memory twice — once for absmax computation and once for quantization — doubling global memory traffic when the dataset exceeds L2 cache capacity


## Solution

Cache input in shared memory during the first pass (copy global→shared while computing absmax in a single loop), then quantize from shared memory in the second phase, eliminating the redundant global memory read of input data

## Key Lessons

- Shared-memory caching to avoid redundant global reads yields no benefit when the working set fits entirely in L2 cache — on an A800 with 40MB L2, a ~1MB dataset is already cached after the first pass, so the second global read hits L2 not DRAM, and the added smem copy plus __syncthreads() barrier overhead (~4%) negates any theoretical bandwidth savings
- Apply shared-memory staging for redundant global reads only when the dataset exceeds L2 cache capacity — for small working sets that are L2-resident, the two-pass global approach is competitive because it avoids shared memory pressure and barrier synchronization overhead
