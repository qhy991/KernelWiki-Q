## Solution

Fuse renormalization into the topk gating softmax kernel by accumulating the topk weight sum in a register (row_sum_for_renormalize) during the top-K selection loop, then performing the inverse multiplication in-place after the loop — eliminating the separate kernel launch and its global memory traffic

## Key Lessons

- Fuse lightweight post-processing (e.g., renormalization of topk weights) into the producing kernel when the accumulator state is already in registers — eliminates a separate kernel launch and avoids a full global memory round-trip for the data
- Accumulating a running sum in a scalar register during a topk selection loop costs negligible overhead vs. writing to global memory, but saves an entire second kernel's worth of memory traffic and launch latency
