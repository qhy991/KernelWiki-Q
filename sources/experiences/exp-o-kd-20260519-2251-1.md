## Solution

Replace hardcoded head_size stride with a configurable head_stride parameter (int64_t) in address computation, changing head_idx * head_size to head_idx * head_stride. The host function derives head_stride from query.stride(-2) for tensors with an explicit heads dimension, enabling the kernel to navigate arbitrary head strides and eliminating .contiguous()/.copy_() overhead entirely.

## Key Lessons

- Replace hardcoded dimension strides with configurable stride parameters derived from tensor.stride() so kernels can operate directly on non-contiguous tensor layouts — applies whenever the Python wrapper currently calls .contiguous() before kernel dispatch, which is a signal the kernel assumes a fixed layout
- Computing stride from tensor.stride(-2) at host side and passing it as a kernel parameter has zero performance cost for contiguous tensors (stride == size) while enabling large speedups for padded/sliced layouts by eliminating full tensor copies
