## Challenge

Warp-level integer reduction using __reduce_add_sync (SM80+ cooperative groups intrinsic) needs SM75 compatibility, but replacing it with a manual __shfl_down_sync cascade introduces 5 sequential shuffle-sync pairs that are 2.5x slower than the hardware-optimized intrinsic on Ampere


## Solution

Replace __reduce_add_sync with a manual cascaded __shfl_down_sync reduction using compile-time constexpr guards (MOE_BLOCK_SIZE thresholds) for SM75 compatibility, trading Ampere-specific hardware optimization for broader GPU support at a measured 2.5x performance cost

## Key Lessons

- __reduce_add_sync on SM80+ is hardware-optimized by the CUDA runtime and significantly faster (2.5x) than a manual __shfl_down_sync cascade — when SM80+ is guaranteed, prefer the cooperative groups intrinsic over manual shuffle implementations
- Manual __shfl_down_sync cascade enables SM75 compatibility but incurs substantial performance cost on SM80+ — use architecture dispatch (#if __CUDA_ARCH__ >= 800) to select the optimal reduction implementation per target SM version
