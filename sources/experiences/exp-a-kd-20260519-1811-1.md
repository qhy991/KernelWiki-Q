## Solution

Convert the compile-time pipe_stages constant into a runtime stages parameter propagated through the shared memory sizing call chain, and add an is_a_8bit flag to halve activation shared memory (1 byte/elem vs 2) when 8-bit activations are used

## Key Lessons

- Parameterize shared memory pipeline stages as a runtime value instead of a compile-time constant — enables a single binary to adapt shared memory sizing across GPU generations with different shared memory budgets without recompilation
- Conditional element sizing via is_a_8bit flag (1 byte/elem for INT8 vs 2 for FP16) in shared memory calculation — directly reduces activation buffer footprint when lower precision is acceptable, freeing shared memory for larger tiles or more pipeline stages
