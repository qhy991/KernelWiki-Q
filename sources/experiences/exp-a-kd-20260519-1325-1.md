## Challenge

The absmax kernel uses an all-or-nothing can_vectorize branch: when the element count is not divisible by the vector width (16), it falls back entirely to scalar loads, abandoning vectorization for ~98.7% of data that is actually aligned


## Solution

Replace the conditional can_vectorize branch with vectorize_read_with_alignment, which handles misalignment by processing a scalar prefix to reach alignment, vectorizing the aligned middle portion, and finishing with a scalar tail — enabling partial vectorization for any element count regardless of divisibility

## Key Lessons

- Alignment-aware partial vectorization captures vector-load benefits for the aligned subset of data even when total length is not divisible by vector width — but the lambda/functor abstraction and prefix/tail scalar overhead can offset gains at small problem sizes where kernel launch and synchronization dominate timing
- Partial vectorization (e.g., vectorizing 1008 of 1021 elements) shows neutral results on single-block grids because each thread processes only ~2 vector iterations — the technique scales better on multi-block grids with larger inputs where the fixed overhead is amortized over more vector packs per thread
