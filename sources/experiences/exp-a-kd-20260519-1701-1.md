## Solution

Replace 256-bin histogram with 2048-bin (11-bit half-precision binning) and use cub::BlockScan ExclusiveSum instead of manual cumulative sum, aiming to resolve TopK in fewer refinement passes by using finer-grained bins that distinguish more values in a single pass

## Key Lessons

- Increasing histogram resolution from 256 to 2048 bins reduces refinement passes but increases shared memory pressure (8KB to 32KB+) and cub::BlockScan TempStorage overhead — the tradeoff only benefits longer sequences where multiple refinement passes are costly; for short decode sequences (seq_len=4096), the overhead dominates
- cub::BlockScan ExclusiveSum provides a cleaner API than manual cumulative sum, but its TempStorage scales with thread count (1024 threads) — evaluate the shared memory budget impact before replacing hand-rolled scans in latency-sensitive decode paths
