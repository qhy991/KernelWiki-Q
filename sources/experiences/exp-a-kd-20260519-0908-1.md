## Solution

Replace scalar int32 loads with int4 (128-bit) vectorized loads, fetching four int32 values per global memory transaction when the input pointer is 16-byte aligned, with a scalar fallback for unaligned addresses, reducing total memory transactions by 4x in the aligned path

## Key Lessons

- Use int4 (128-bit) vectorized loads instead of scalar int32 loads when iterating over contiguous int32 arrays in memory-bound kernels — reduces global memory transaction count by 4x, improving bandwidth utilization for workloads where each thread processes many elements
- Include a runtime alignment check (pointer % 16 == 0) with a scalar fallback path when vectorizing loads via reinterpret_cast — cudaMalloc guarantees 256-byte alignment but user-provided pointers or sub-allocations may not meet int4 alignment requirements
- A 4x reduction in memory transactions translates to modest (5.5%) end-to-end speedup when the kernel is memory-bound but also spends time in CUB BlockReduce and other non-load overhead — the speedup scales with the fraction of time spent in the load-heavy loop
