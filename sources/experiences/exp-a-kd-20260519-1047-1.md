## Solution

Pack 8 tokens per CTA with one warp (32 threads) per token, replacing blockReduceMax with warpReduceMax and __shared__ scale with register-local scale to eliminate __syncthreads, but reducing CTA count from 2048 to 256 which loses CTA-level parallelism across 108 SMs and causes 0.585x regression

## Key Lessons

- Warp-level token packing eliminates __syncthreads overhead but can regress when it reduces total CTA count below the level needed to saturate all SMs with multiple concurrent CTAs — CTA scheduling parallelism across SMs often outweighs per-CTA synchronization savings
- Without shared-memory staging to reduce global memory traffic (e.g., double-buffering input tiles), warp packing alone does not halve memory bandwidth and each thread's 8x longer loop (32 vs 4 iterations) directly increases latency with no compensating throughput gain
