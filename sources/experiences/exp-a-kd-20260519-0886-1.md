## Solution

Replace global loop with round-robin thread assignment that statically partitions threads across experts at kernel launch, eliminating the offset array entirely and converting the continue-based early exit to a break that terminates the per-expert loop when masked rows are reached

## Key Lessons

- Round-robin thread assignment eliminates per-element offset array lookups and converts continue to break for early exit within an expert's range, but shows no measurable speedup when per-element compute (silu_and_mul + quantization + packing) dominates over the eliminated indexing overhead
- When per-element work is heavy (32 bytes bf16 read + 8 silu via __expf + 8 multiplications + absmax + quantization + packing), eliminating O(n_experts) search across only 6 entries represents under 2% of total work and falls within measurement noise — optimization effort should target the dominant compute bottleneck instead
- Static thread-to-expert partitioning removes the offsets kernel parameter entirely and guarantees sequential row iteration within each expert, enabling break-based early exit; this structural improvement simplifies the kernel but does not translate to speedup unless the eliminated indexing overhead is a significant fraction of total execution time
