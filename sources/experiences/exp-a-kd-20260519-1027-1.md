## Solution

Replace contiguous-block iteration with stride-based (grid-stride) iteration where thread i processes indices i, i+stride, i+2*stride..., promoting coalesced global memory reads since adjacent threads access adjacent elements, and simplifying loop control to a single variable stride loop

## Key Lessons

- Stride-based iteration (thread i processes i, i+stride, i+2*stride...) promotes memory coalescing because adjacent threads access adjacent addresses on each iteration, whereas contiguous-block iteration scatters accesses when elements-per-thread > 1 — however, when threads ≈ elements (1 element per thread), the coalescing benefit vanishes since both patterns produce the same memory layout
- Grid-stride loops are simpler (single loop variable, no CEILDIV, no double-bound check) and generalize naturally to arbitrary element counts without edge-case handling
