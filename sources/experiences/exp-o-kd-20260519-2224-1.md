## Challenge

fp8-to-float conversion and per-element scaling are implemented as two separate kernel launches with an intermediate 4N-byte global memory buffer (d_float_tmp), doubling memory traffic and incurring redundant kernel launch overhead

## Key Lessons

- Fuse elementwise type-conversion and arithmetic kernels into a single kernel when they form a producer-consumer chain â€” eliminates one kernel launch and the intermediate global memory round-trip, which dominates latency for memory-bound operations
- Removing an intermediate global memory buffer (4N bytes for float) halves the total DRAM traffic for memory-bound elementwise pipelines, yielding speedups far beyond what instruction-level optimization alone can achieve
