## Challenge

Scalar ELU kernel processes one float element per thread with separate 32-bit load/store instructions, underutilizing available memory bandwidth and requiring 4x more memory transactions than necessary for the memory-bound elementwise operation


## Solution

Use float4 (128-bit) vectorized loads/stores via the FLOAT4 macro so each thread processes 4 elements in a single memory transaction, reducing total memory transactions by 4x and improving memory bandwidth utilization for the bandwidth-bound ELU operation

## Key Lessons

- Use float4 vectorized loads/stores for memory-bound elementwise kernels — each thread processes 4 elements per memory transaction, reducing total transactions by 4x and saturating memory bandwidth more effectively
- Scale thread index by the vectorization factor (idx * 4) and reduce block size proportionally (blockSize / 4) to maintain the same grid coverage while issuing fewer, wider memory operations
- Reuse the same scalar compute function (elu) inside a vectorized load/store loop — vectorization only affects memory access, not the arithmetic, so the compute logic remains unchanged
