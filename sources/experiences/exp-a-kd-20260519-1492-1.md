## Solution

Replace the scalar stride loop with vectorize_read_with_alignment<16> to load 16 floats (64 bytes) per vector iteration, reducing iteration count from ~16 to ~1 per thread — but the 64-byte vector load has higher latency and register cost than 16 coalesced 4-byte scalar loads, and the per-iteration overhead (alignment check, reinterpret_cast, vec_op dispatch) becomes proportionally dominant with only 1 iteration

## Key Lessons

- Vectorized loads (float4/float16) help when each thread processes many iterations so the amortized load throughput gain outweighs per-instruction overhead; with only ~1 vector iteration per thread the overhead of alignment checks, reinterpret_cast, and vec_op dispatch dominates
- Coalesced scalar loads already achieve near-peak memory bandwidth for aligned contiguous access patterns — replacing them with explicit vector loads does not improve throughput when the hardware coalescing unit handles the scalar pattern efficiently
- Vector load width should be matched to work per thread: a VEC_SIZE=16 template that reduces iterations from 16 to 1 eliminates the loop-amortization window that normally hides the vector load's higher latency and register pressure
