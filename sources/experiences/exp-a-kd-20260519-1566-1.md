## Challenge

Naive top-k uses O(N*K) iterative selection: K=16 iterations each scan all N=256 elements with a warp max-reduction, a used-flag array in shared memory (~3KB total), and a block-level __syncthreads() per iteration, causing excessive synchronization and redundant memory traffic


## Solution

Replace O(N*K) iterative selection with WarpSelect incremental algorithm: read input in rounds of WARP_SIZE, use __ballot_sync to skip rounds where no candidate exceeds the k-th threshold, accumulate candidates into a warp-sized buffer, and trigger warp-level bitonic sort/merge only when the buffer fills — reducing sync from 16 block-level __syncthreads to ~8 warp-level __syncwarp and cutting shared memory from ~3KB to 256 bytes

## Key Lessons

- Replace O(N*K) iterative-selection top-k with WarpSelect incremental approach that reads input in warp-sized rounds and maintains a sorted top-k buffer — reduces time complexity from O(N*K) to O(N log k) by avoiding repeated full scans
- Use __ballot_sync to skip entire input rounds when no candidate exceeds the current k-th threshold — as the top-k buffer fills, fewer candidates qualify and later rounds are skipped at zero cost
- Prefer warp-level bitonic sort/merge with __syncwarp over block-level __syncthreads for intra-warp algorithms — eliminates unnecessary inter-warp synchronization barriers and cuts shared memory from O(N) to O(WARP_SIZE)
