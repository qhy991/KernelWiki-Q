## Challenge

The before code's two-step path (bf16 GEMM + cast kernel) requires cuBLAS to write 2048 bf16 values to global memory, then the cast kernel reads those 2048 bf16 values and writes 2048 fp32 values


## Solution

The after code's single cublasGemmEx with CUDA_R_32F output avoids the intermediate bf16 write and the separate read+write pass for the cast. The saved global-memory traffic and one fewer kernel launch account for the ~19% speedup at M=8.
