## Challenge

The SiLU*up intermediate result is stored to shared memory via shared_result[tid] and reloaded later for quantization, creating an unnecessary shared memory store-load round-trip that could increase latency and waste shared memory space


## Solution

Remove the shared_result array entirely and keep the SiLU*up result in a register variable across the absmax reduction and scale broadcast, storing only fabsf(result) to shared memory for the reduction and using the register-held result directly in the final quantization step

## Key Lessons

- Keeping intermediate values in registers versus shared memory shows no measurable improvement because nvcc -O3 already performs register promotion that eliminates unnecessary shared memory store-load round-trips when the value remains live in a register
- Manual register caching of values that the compiler can trivially keep live across __syncthreads barriers is a low-impact optimization — the compiler's own dead-store elimination and register allocation typically handles this case
