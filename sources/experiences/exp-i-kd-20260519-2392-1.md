## Solution

Map threads 2D (threadIdx.x for rows, threadIdx.y for columns) with bounds checking, compute column-major offset as i + j * ldm, and generate pseudo-random values via integer Lehmer-style PRNG ((offset + seed) * 16807 % 16 - 8) for fast deterministic initialization

## Key Lessons

- 2D thread mapping (threadIdx.x for rows, threadIdx.y for columns) maps naturally onto 2D matrix layouts — for column-major storage, consecutive threads in x-dimension access consecutive memory addresses, producing coalesced global stores
- Integer Lehmer-style PRNG (multiplier 16807, modular arithmetic) generates deterministic pseudo-random values with minimal GPU instruction cost — suitable for initialization kernels where statistical quality is unimportant but speed and reproducibility matter
- Column-major offset computed as i + j * ldm (row index + column index × leading dimension) is the standard BLAS/LAPACK stride convention — use it consistently with CUTLASS column-major layouts to avoid transpose bugs
