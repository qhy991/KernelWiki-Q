Task level_1/task_4: Matrix-vector multiplication (GEMV) with A(256, 131072) and B(131072, 1)

## Key Lessons

- For GEMV with small M (256) and large K (131072), assign one block per row
- Use strided access pattern: each thread processes K/TILE_SIZE elements
- Simple reduction in shared memory works well for this problem size
- Avoid complex multi-row-per-block logic when M is small
