## Challenge

Signal buffer allocated with standard cudaMalloc relies on cache-coherent memory for cross-kernel visibility; on AMD MI200 the L2 cache can hold stale data, requiring an uncached allocation path (hipDeviceMallocUncached) that has no direct NVIDIA equivalent


## Solution

The after code attempts to use an uncached allocation path (hipExtMallocWithFlags with hipDeviceMallocUncached on ROCm) to bypass L2 cache for signal visibility, but on NVIDIA this falls back to identical cudaMalloc since no uncached flag exists, resulting in no performance difference

## Key Lessons

- hipDeviceMallocUncached bypasses the L2 cache on AMD MI200 to ensure cross-GPU signal visibility, but NVIDIA GPUs provide hardware cache coherence via NVLink making uncached allocation unnecessary — platform-specific cache behavior determines whether this optimization matters
- When porting ROCm-specific allocation flags to CUDA, there may be no equivalent API (no cudaMallocUncached exists); the correct fallback is standard cudaMalloc, which produces identical results on NVIDIA hardware
- Cache-coherent interconnect (NVLink on NVIDIA) versus non-coherent L2 (MI200) is a fundamental architectural difference that determines whether signal buffers need uncached allocation for correctness
