## Challenge

The topK shortcut-path kernel writes both outIndices and outLogits to global memory per element, but outLogits values are never consumed downstream — this doubles global memory traffic (one indices write + one logits read-modify-write per element) in an already memory-bound operation, wasting ~8KB of writes per row


## Solution

Remove the outLogits output parameter entirely along with the smemLogits shared memory array; the kernel writes only outIndices, eliminating all redundant global memory writes and the corresponding logits reads from the shortcut path, halving memory traffic

## Key Lessons

- Eliminate dead output parameters and their associated shared-memory buffers when downstream consumers do not use them — removing unused global writes directly reduces memory traffic in memory-bound kernels
- Audit kernel I/O contracts for unused outputs before optimizing computation; removing a single unused float output array can yield >20% speedup when the kernel is memory-bound on global store throughput
