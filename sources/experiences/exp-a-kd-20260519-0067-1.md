## Solution

Switch from IteratorAlgorithm::kAnalytic to IteratorAlgorithm::kOptimized, which precomputes a delta table for all spatial dimensions before the main GEMM loop, replacing repeated per-tile integer division/modulo with direct indexed table lookups that reduce register pressure and enable more predictable memory access patterns

## Key Lessons

- Prefer IteratorAlgorithm::kOptimized over kAnalytic in CUTLASS Conv3d kernels — the precomputed delta table eliminates per-tile integer division/modulo overhead, which is especially significant for 3D convolutions where filter offsets span three spatial axes
- Precomputed lookup tables for frequently recomputed index arithmetic reduce both instruction count and register pressure in hot GEMM inner loops — the benefit scales with the number of spatial dimensions in convolution problems
