## Challenge

Residual addition and RMS normalization are implemented as two separate kernel launches, requiring two full global-memory passes over the hidden_size=8192 data per token — one pass to add input+residual and write back, then a second pass to re-read the result for variance computation and normalization


## Solution

Fuse residual addition and RMS normalization into a single fused_add_rms_norm_kernel: loop 1 computes input+residual, accumulates variance, and writes the sum to residual in one pass; loop 2 reads back the L2-cache-hot residual values for normalization and writes the result to input, eliminating one kernel launch and one full global-memory round-trip

## Key Lessons

- Fuse adjacent kernels that read/write overlapping data regions into a single kernel — eliminates kernel launch overhead and ensures intermediate results stay L2-cache hot between computation phases, yielding 1.6x speedup for residual-add + rms-norm fusion
- When fusing kernels, merge the first kernel's write loop with the second kernel's reduction/scan loop so each element is loaded once, accumulated, and stored — applies to any pattern where kernel A writes data that kernel B immediately re-reads
- Kernel fusion speedup diminishes as per-kernel work grows (hidden_size 8192 gets 1.61x vs 4096 gets 1.66x) because the fixed launch-overhead savings amortize over longer compute — fusion is most impactful when kernels are memory-bound with small payloads
