## Solution

Generalize the kernel into a template <bool STRIDE_I_ZERO, bool STRIDE_J_ZERO> multi-path dispatch that extracts the quantization core into a vec_quant lambda and selects among full-row vectorized, group-vectorized, and scalar-fallback paths at compile time, so the per-tensor specialization (STRIDE_J_ZERO=true) resolves to identical machine code as the original

## Key Lessons

- C++ template bool parameters enable zero-overhead multi-path dispatch — when the compiler instantiates with known true/false values, dead code elimination removes all unused branches, producing code identical to a hand-written single-path kernel
- Lambda-based abstraction for repeated GPU computation patterns (vec_quant, get_inv_scale) enables code reuse across dispatch paths without runtime cost, since CUDA lambdas are fully inlined by NVCC at compile time
