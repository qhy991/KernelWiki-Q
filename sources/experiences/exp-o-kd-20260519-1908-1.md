## Challenge

In the MoE Marlin GEMM kernel's hot A-matrix fetch loop, every iteration computes sh_block_sorted_ids[row] / top_k via integer division to map a token row to its expert, wasting multiple ALU cycles per row across every pipeline stage and slice


## Solution

Pre-compute sorted_token_ids / top_k once into a separate shared memory array sh_rd_block_sorted_ids during initialization, then replace the per-iteration integer division with a simple shared memory read, trading N expensive divisions per pipeline stage for N cheap loads amortized by a single init pass

## Key Lessons

- Pre-compute invariant integer divisions (e.g., sorted_ids / top_k) into a dedicated shared memory array during initialization instead of recomputing them in hot loops — GPU integer division costs ~20 ALU cycles while a shared memory read costs ~1 cycle, making the amortization worthwhile whenever the result is used more than once per element
- Decompose composite index arithmetic (a_gl_rd_delta_i * i + a_gl_rd + a_gl_rd_delta_o * a_off) into separate row and column components so that pre-computed lookup results integrate cleanly without recomputing the division from the flattened index
