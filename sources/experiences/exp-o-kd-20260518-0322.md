broadcast_fma

## Solution

The AFTER version (broadcast_fma_after.cu) uses the h0_h0 function (PTX mov.b32 {%1,%1}) to broadcast the scalar into both lanes of a half2, then performs fma.rn.f16x2 PTX asm which computes scalar*a+c for both lanes simultaneously in a single instruction, followed by add.f16x2 for accumulation. ...
