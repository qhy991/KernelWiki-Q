## Challenge

The before code directly computes (65,4096)x(4096,4096) with contiguous row-major access on A (lda=65) and contiguous column-major access on B (lda=4096). The after code swaps via CUBLAS_OP_T, computing (4096,4096)x(4096,65) where B is accessed transposed with lda=4096 and A transposed with lda=65. 


## Solution

or M=65, the A matrix transposed access (stride 65 in a 65x4096 matrix) is nearly contiguous because 65 rows span most of one 128-byte cache line per column, but B transposed access still introduces some uncoalescing overhead on SM80. The smaller regression (~8.8%) compared to M=3 reflects that M=65
