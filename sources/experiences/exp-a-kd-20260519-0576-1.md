## Solution

Set CUBLAS_TF32_TENSOR_OP_MATH math mode and CUBLAS_GEMM_DEFAULT_TENSOR_OP algorithm hint to explicitly instruct cuBLAS to route the FP32 GEMM through TF32 tensor core kernels, reducing mantissa from 23 to 10 bits for higher FLOPs throughput on Ampere+ GPUs

## Key Lessons

- Explicitly setting CUBLAS_TF32_TENSOR_OP_MATH + CUBLAS_GEMM_DEFAULT_TENSOR_OP forces cuBLAS to use tensor core TF32 kernels for FP32 GEMM on Ampere+ GPUs, yielding ~1.15x speedup even when auto-dispatch already partially utilizes tensor cores — applies when reduced FP32 mantissa (TF32 ~3 decimal digits) is acceptable for the workload
- cuBLAS auto-dispatch under CUBLAS_DEFAULT_MATH may already route to tensor cores on some shapes, but explicit TF32 math mode provides more consistent kernel selection across different matrix dimensions and cuBLAS versions
