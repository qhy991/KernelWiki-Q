## Challenge

QKNorm (RMSNorm + weight) kernel writes ~16MB of normalized values to a global-memory intermediate buffer, then the NeoX RoPE kernel reads that buffer back in a separate launch, creating a full global-memory round-trip for this memory-bound pipeline


## Solution

Fuse the two kernels into one: keep RMSNorm+weight results in the register array e[EPT], apply NeoX RoPE via __shfl_xor directly on those register-held values, and write final output once — eliminating the ~16MB intermediate buffer round-trip

## Key Lessons

- Fuse consecutive kernels connected by a global-memory intermediate buffer by keeping intermediate results in per-thread registers — eliminates one full global-memory round-trip and applies whenever the intermediate data fits in registers (EPT elements per thread)
- Kernel fusion yields the largest speedups on memory-bound pipelines where intermediate buffer traffic dominates runtime; the measured 1.458x speedup here is consistent with roughly halving global-memory traffic for the ~16MB intermediate buffer
- __shfl_xor register-to-register swaps between warp lanes are negligible compared to global memory accesses, making them effectively free when used to apply cross-lane transformations (like NeoX RoPE rotation) on register-held data within a fused kernel
