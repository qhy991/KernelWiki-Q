## Solution

Replace the three-operation software path (rintf + fmaxf/fminf + cast) with a single PTX inline assembly instruction cvt.rni.sat.s8.f32 that performs rounding-to-nearest-even, saturation to [-128, 127], and float-to-int8 conversion in one hardware operation, reducing per-element instruction count and improving instruction-level parallelism

## Key Lessons

- Replace multi-operation software float-to-int8 conversion (rintf + clamp + cast) with inline PTX cvt.rni.sat.s8.f32 — the hardware instruction fuses rounding, saturation, and type conversion into a single operation, reducing instruction count and enabling better ILP — applies to any kernel performing per-element quantization with clamping
- Use reinterpret_cast<const int8_t&>(uint32_t) to extract the low byte from the PTX destination register after cvt.sat.s8 — the .sat.s8 modifier writes only the lower 8 bits into a 32-bit register, so the reinterpret is zero-cost
