## Challenge

Scalar bf16 multi-GPU reduction loads one 2-byte nv_bfloat16 per memory transaction, issuing 8x more load instructions than necessary and severely underutilizing available 128-bit memory bandwidth


## Solution

Use __align__(16) array_t<nv_bfloat16,8> packed type to cast input pointers and load 8 bf16 values (16 bytes) per ld.128 instruction, reducing memory instruction count by 8x while iterating over n_packed = n/8 elements

## Key Lessons

- Use __align__(16) packed struct types (array_t<T, 16/sizeof(T)>) to enable 128-bit ld.128/st.128 instructions for 2-byte types — reduces memory instruction count by 8x for bf16 and applies to any memory-bound kernel accessing contiguous low-precision data
- Cast input pointers to packed types (e.g. (const PBF *)ptr) and iterate over n_packed = n/(16/sizeof(T)) to naturally coalesce memory access without manual vector intrinsics
