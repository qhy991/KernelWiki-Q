## Solution

Refactor monolithic dispatch into a templated dispatch_scaled_mm that routes per-tensor/per-token/per-channel scales to an fp8_func and blockwise scales (with 128x128 group sizes validated via ceil_div) to a dedicated blockwise_func, enabling sub-tile quantization granularity without modifying the dispatch logic itself

## Key Lessons

- Templated dispatch with function parameters (Fp8Func, Int8Func, BlockwiseFunc) enables extending scaling mode support without modifying the dispatch logic — new quantization granularities are added by registering new function handlers rather than changing shared dispatch code
- Blockwise scaling with 128x128 group sizes aligns with typical CUTLASS MMA tile dimensions on SM100 architectures, enabling sub-tile dequantization for improved FP8 accuracy while keeping the tile-level compute path unchanged
- Separating dispatch routing from kernel implementation via templates keeps the per-tensor fast path zero-cost — no virtual dispatch or branching overhead is added to the existing hot path when blockwise scales are not used
