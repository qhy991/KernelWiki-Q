## Challenge

Two separate kernels (rescale_kernel and scale_kernel) each traverse the full output array, causing two complete global-memory read-modify-write cycles and two kernel launch overheads for what is elementwise division-then-multiplication on the same data


## Solution

Fuse rescale (divide by sum) and scale (multiply by routed_scaling_factor) into a single kernel that performs both operations per element in one read-modify-write pass, halving global memory traffic and eliminating one kernel launch

## Key Lessons

- Fuse consecutive elementwise kernels that operate on the same data into a single kernel to halve global memory traffic — applies whenever two passes read/write the same array and there are no inter-element dependencies between the passes
- Kernel fusion savings are dominated by reduced global memory traffic at larger scales; at smaller scales, eliminating the extra kernel launch overhead contributes a larger relative speedup
- A bool flag (apply_routed_scaling_factor_on_output) allows conditional fusion when the scaling step may not always be needed, preserving flexibility without sacrificing the fused fast path
