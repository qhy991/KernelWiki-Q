## Challenge

The before code uses __float2half followed by __nv_cvt_halfraw_to_fp8, routing through the FP16 datapath. On SM80 (A100/A800), the half-to-fp8 conversion leverages well-optimized FP16 execution units with short pipeline latency


## Solution

The after code calls __nv_cvt_float_to_fp8 directly, which must pass through the FP32-to-FP8 conversion pipeline that incurs additional rounding and saturation logic in hardware.
