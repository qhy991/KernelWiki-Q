## Summary

Adds DeepGemm contiguous-mode support for the DeepEP normal-mode expert parallel MoE implementation. This enables the use of DeepGemm's optimized FP8 grouped GEMM kernels in the contiguous memory layout path used by DeepEP's normal dispatch mode, providing improved MoE computation efficiency for DeepSeek models.

## Problem

DeepEP (Deep Expert Parallelism) in normal mode was not leveraging DeepGemm's high-performance contiguous-mode grouped GEMM kernels. The existing implementation used standard MoE kernels that did not fully exploit FP8 tensor cores with the contiguous memory layout expected by DeepEP's normal dispatch path.

## Solution / Techniques

- Integrates DeepGemm contiguous-mode grouped GEMM into the DeepEP normal-mode token dispatcher
- Supports FP8 quantization with per-token-group scaling for expert parallel MoE
- Maintains compatibility with the existing DeepEP dispatch infrastructure
- Validated accuracy on GSM8K benchmark
- 52 commits addressing code style, FP8 compatibility, auto-mode testing, and performance benchmarking

```python
# DeepEP normal mode with DeepGemm contiguous grouped GEMM
# Token dispatcher routes tokens to experts using contiguous memory layout
# DeepGemm performs FP8 grouped GEMM on contiguous expert buffers
```

## Performance

The contiguous mode passed debugging in DeepEP normal mode with correct accuracy on GSM8K. Performance benchmarking against the normal baseline was identified as a remaining optimization target. The contiguous layout enables better memory access patterns for grouped GEMM operations compared to the default scattered layout.
