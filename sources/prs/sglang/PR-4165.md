## Summary

Integrates DeepGemm, a high-performance FP8 GEMM library with JIT compilation, into the sgl-kernel build system. This enables optimized tensor operations for Hopper and Blackwell GPUs by deferring kernel compilation to runtime for hardware-specific optimization.

## Problem

SGLang lacked integration with DeepGemm's high-performance GEMM kernels. The build system needed to be extended to incorporate DeepGemm as a third-party library alongside the existing AOT-compiled sgl-kernel components, enabling JIT-compiled FP8 matrix multiplications optimized for specific tensor dimensions and hardware.

## Solution / Techniques

- Modified `setup.py` to build DeepGemm alongside AOT sgl-kernel components
- Symlinks CUTLASS and CuTe header files required for JIT compilation at runtime
- Copies DeepGemm files into the Python package during wheel creation
- JIT compilation strategy defers kernel compilation until runtime, allowing dynamic optimization based on tensor dimensions and hardware characteristics
- Requires setuptools >= 75.0 for compatibility

## Key Code

```python
# DeepGemm JIT compilation - kernels are compiled at runtime
# targeting specific tensor dimensions and hardware capabilities
import deep_gemm
# Uses CUTLASS headers for JIT: cute.h, cutlass.h
```

## Performance

Benchmarks on Hopper/Blackwell GPUs:

| Matrix Size (m) | TFLOPS |
|-----------------|--------|
| 64 | 193-423 |
| 128 | 347-789 |
| 4096 | 595-1392 |

Grouped operations achieve 1.3+ TFLOPS at production scale, supporting tensor parallelism patterns common in large language model inference.
