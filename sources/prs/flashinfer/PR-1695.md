## Summary

Adds a GEMM kernel overlapped with two-shot allreduce operations using multimem instructions within the CuTe DSL framework. This fuses distributed communication with computation to reduce latency in tensor-parallel inference on Blackwell GPUs, leveraging SM100's GDC (Global Data Crossbar) and TMA multicast capabilities.

## Problem

In tensor-parallel LLM inference, GEMM operations followed by allreduce communication create a sequential bottleneck. The GEMM must complete before allreduce can begin, wasting GPU cycles. A fused GEMM+allreduce kernel can overlap computation and communication, reducing end-to-end latency for distributed inference.

## Solution / Techniques

- Implements GEMM + two-shot allreduce in CuTe DSL (CUTLASS Python DSL)
- Uses multimem instructions for efficient multi-GPU memory access
- Two-shot allreduce: first shot reduces partial results, second shot broadcasts
- Leverages SM100 GDC for cross-SM data movement during allreduce
- TMA multicast for efficient data distribution across SMs
- Single fused kernel eliminates GEMM-allreduce synchronization overhead
