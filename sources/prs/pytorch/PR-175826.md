## Summary
Migrates PyTorch's inductor CI infrastructure from CUDA 12.8 to CUDA 13.0, updates GPU architecture mappings to include B200 (sm100) benchmarking, and removes redundant duplicate CI jobs that previously maintained parallel CUDA 12.8 and CUDA 13.0 pipelines.

## Problem
PyTorch's inductor CI used CUDA 12.8 as its primary toolkit version with separate `-cuda13` variant jobs running in parallel. This duplication was wasteful and the CUDA 12.8 baseline was outdated given that CUDA 13.0 had become the target for Blackwell (B200/sm100) GPU support. The architecture lists in CI workflows also contained incorrect GPU-to-compute-capability mappings.

## Solution / Techniques
Three key changes:
1. **Docker image migration**: All inductor benchmark workflows switched from `pytorch-linux-jammy-cuda12.8-cudnn9-py3-gcc11-inductor-benchmarks` to the CUDA 13.0 variant.
2. **Duplicate job removal**: Redundant `-cuda13` jobs removed from 5 workflow files (inductor-periodic, inductor-micro-benchmark, inductor-perf-compare, inductor-perf-test-nightly, trunk).
3. **Architecture alignment**: GPU compute capabilities corrected to match actual hardware -- A10G: 8.6, A100: 8.0, H100: 9.0, B200: 10.0, T4: 7.5.

## Key Code

```yaml
# .github/workflows/inductor-perf-test-b200.yml

# GPU architecture mapping corrected across workflows:
# A10G: 8.6, A100: 8.0, H100: 9.0, B200: 10.0, T4: 7.5
```

## Performance
This PR enables benchmarking of PyTorch's torch.compile/inductor stack on B200 (Blackwell/sm100) GPUs with the correct CUDA 13.0 toolkit. The CI consolidation reduces duplicate build and test time. The B200 perf test workflow specifically targets compute capability 10.0 for Blackwell architecture performance regression tracking.
