## Summary

Introduces blockwise and groupwise GEMM implementations targeting the Blackwell SM100 architecture, alongside improvements to existing Hopper FP8 GEMM kernels. Adds new examples (example 81) for Blackwell blockwise and groupwise GEMM and extends Hopper grouped GEMM with blockwise scaling support (example 68).

## Problem

CUTLASS lacked native blockwise and groupwise scaled GEMM kernels for the Blackwell SM100 architecture. Quantized inference workloads using FP8 and FP4 block-scaled formats on Blackwell required new warp-specialized collective mainloop implementations that leverage SM100-specific hardware features such as TCGen05 MMA instructions and TMEM for accumulator management.

## Solution / Techniques

The PR adds several new kernel components:

1. **SM100 Blockwise Scaling Collective MMA** (`sm100_mma_warpspecialized_blockwise_scaling.hpp`): A new warp-specialized mainloop that handles block-scaled data types with proper scale factor promotion intervals on Blackwell.

2. **SM100 Blockwise Scale Layout** (`sm100_blockwise_scale_layout.hpp`): Defines the memory layout for scale factors used in blockwise quantization on SM100.

3. **New Blackwell Examples**:
   - `81_blackwell_gemm_blockwise.cu` - Demonstrates blockwise scaled GEMM on SM100
   - `81_blackwell_gemm_groupwise.cu` - Demonstrates groupwise scaled GEMM on SM100

4. **Hopper Improvements**: Adds warmup loop support for profiling and a static `ScalePromotionInterval` configuration for FP8 GEMMs.

## Key Code

```cpp
// Configuration for blockwise scaling on SM100

    ElementAccumulator,

```

## Performance

The PR adds profiling infrastructure with warmup iterations to Hopper FP8 GEMM examples. The Blackwell blockwise GEMM kernels leverage TCGen05 instructions and TMEM-based accumulator double buffering for high throughput on SM100. Specific performance numbers were not included in the PR but these kernels are foundational for CUTLASS 3.8's Blackwell support.
