## Summary

DeepGEMM's `[Public release 26/04]` ships the Mega MoE feature plus FP4 Indexer plus a batch of smaller features and fixes. The Mega MoE kernel fuses dispatch / linear-1 / SwiGLU / linear-2 / combine into a single mega-kernel that overlaps NVLink communication with tensor-core computation; only FP8 × FP4 MoE is supported in this release, and PyTorch ≥ 2.9 is required.

## Why this is the DeepGEMM full-ledger anchor for Round 4

This PR is the most substantive DeepGEMM change in the post-`2026-04-17` window: it crosses the kernel-feature boundary (Mega MoE adds a new SM100 fused MoE mega-kernel; FP4 Indexer is a new FP4 quantization path) rather than being a bugfix. It serves as the primary DeepGEMM upstream-code anchor for the wiki's `wiki/kernels/deepgemm.md` page in the Round 4 refresh.
